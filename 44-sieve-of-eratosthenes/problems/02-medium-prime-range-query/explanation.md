# 구간 소수 개수 쿼리 - 해설

## 접근 방식

1. 에라토스테네스의 체로 1부터 N까지의 소수 여부를 전처리한다
2. 누적합 배열 prefix를 구성한다: prefix[i] = 1부터 i까지의 소수 개수
3. 각 쿼리 [L, R]에 대해 prefix[R] - prefix[L-1]로 O(1)에 응답한다

## 핵심 아이디어

에라토스테네스의 체와 누적합(Prefix Sum)을 결합하는 것이 핵심이다. 체로 소수 여부를 O(N log log N)에 전처리한 뒤, 소수 개수의 누적합을 구해두면 임의의 구간 쿼리를 O(1)에 처리할 수 있다. 이는 Q개의 쿼리를 각각 독립적으로 세는 O(Q × N) 방식보다 훨씬 효율적이다.

누적합의 원리: [L, R] 구간의 소수 개수는 "1부터 R까지의 소수 개수"에서 "1부터 L-1까지의 소수 개수"를 빼면 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log log N + Q) |
| 공간 복잡도 | O(N) |

전처리(체 + 누적합)에 O(N log log N), 각 쿼리 응답에 O(1)이므로 총 O(N log log N + Q)이다. N = 10^6, Q = 10^5일 때 충분히 빠르다.

## 대안적 접근

- 누적합 없이 매 쿼리마다 구간을 순회하면 O(Q × N)으로 시간 초과가 발생할 수 있다.
- 세그먼트 트리를 사용할 수도 있지만, 이 문제에서는 정적 쿼리이므로 누적합이 더 간단하고 효율적이다.
