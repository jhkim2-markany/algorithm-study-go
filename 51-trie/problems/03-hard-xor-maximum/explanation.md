# XOR 최댓값 - 해설

## 접근 방식

1. 비트 트라이(Bit Trie)를 생성한다. 각 노드는 비트 0과 1에 대한 두 개의 자식을 가진다.
2. 배열의 모든 수를 최상위 비트(30번째)부터 최하위 비트(0번째)까지 비트 단위로 트라이에 삽입한다.
3. 각 수에 대해 트라이를 탐색하며 XOR 결과가 최대가 되는 상대 수를 찾는다.
4. 탐색 시 각 비트 위치에서 현재 비트와 반대되는 경로를 우선 선택한다. 반대 비트를 선택하면 해당 위치의 XOR 결과가 1이 되어 값이 커진다.
5. 반대 비트 경로가 없으면 같은 비트 경로로 이동한다.
6. 모든 수에 대한 XOR 최댓값 중 가장 큰 값을 출력한다.

## 핵심 아이디어

XOR 결과를 최대화하려면 상위 비트부터 가능한 한 1을 만들어야 한다. 이는 그리디 전략과 트라이의 결합이다.

비트 트라이는 숫자를 이진수로 표현하여 트라이에 저장한다. 일반 트라이가 문자열의 문자를 노드로 사용하는 것처럼, 비트 트라이는 각 비트(0 또는 1)를 노드로 사용한다.

XOR 최댓값을 구할 때, 최상위 비트부터 탐욕적으로 반대 비트를 선택하는 것이 최적이다. 상위 비트의 가중치가 하위 비트의 가중치 합보다 크기 때문이다. 예를 들어 30번째 비트의 값(2³⁰)은 0~29번째 비트의 합(2³⁰ - 1)보다 크다.

브루트포스로 모든 쌍을 비교하면 O(N²)이지만, 비트 트라이를 사용하면 O(N × B)로 줄일 수 있다 (B: 비트 수).

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × B) |
| 공간 복잡도 | O(N × B) |

- N: 배열의 크기
- B: 비트 수 (10⁹ < 2³⁰이므로 B = 31)
- N이 최대 100,000이고 B = 31이므로 약 3,100,000번 연산으로 충분히 빠르다

## 대안적 접근

분할 정복 방법으로도 풀 수 있다. 최상위 비트를 기준으로 배열을 두 그룹(해당 비트가 0인 그룹, 1인 그룹)으로 나누고, 두 그룹 사이에서 XOR 최댓값을 재귀적으로 구한다. 시간 복잡도는 동일하게 O(N × B)이다.
