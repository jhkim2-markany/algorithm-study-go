# 접두사 검색 - 해설

## 접근 방식

1. 트라이 자료구조를 생성한다.
2. N개의 단어를 트라이에 삽입한다. 각 단어의 문자를 하나씩 따라가며 노드를 생성한다.
3. 각 질의 문자열에 대해 트라이에서 해당 경로가 존재하는지 확인한다.
4. 질의 문자열의 모든 문자를 따라갈 수 있으면 접두사가 존재하는 것이므로 "YES"를 출력한다.
5. 중간에 경로가 끊기면 해당 접두사를 가진 단어가 없으므로 "NO"를 출력한다.

## 핵심 아이디어

이 문제는 트라이의 가장 기본적인 활용인 접두사 검색이다. 트라이에 단어를 삽입하면 공통 접두사를 공유하는 노드가 자연스럽게 합쳐진다. 접두사 검색 시에는 단어의 끝(isEnd) 여부를 확인할 필요 없이, 경로가 존재하기만 하면 된다.

브루트포스로 각 질의마다 모든 단어와 비교하면 O(M × N × L)이 되지만, 트라이를 사용하면 각 질의를 O(L)에 처리할 수 있어 훨씬 효율적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × L + M × L) |
| 공간 복잡도 | O(N × L × 26) |

- N × L: 트라이 구축 비용
- M × L: 질의 처리 비용
- L: 문자열의 최대 길이

N, M이 최대 10,000이고 L이 최대 100이므로 충분히 빠르게 동작한다.
