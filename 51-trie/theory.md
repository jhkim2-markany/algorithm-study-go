# 트라이

## 개념

트라이(Trie)는 문자열 집합을 저장하고 탐색하기 위한 트리 형태의 자료구조이다. "Retrieval"에서 유래한 이름으로, 접두사 트리(Prefix Tree)라고도 부른다. 각 노드는 문자 하나를 나타내며, 루트에서 특정 노드까지의 경로가 하나의 문자열(또는 접두사)을 구성한다.

해시맵으로 문자열을 저장하면 정확한 일치 검색은 빠르지만, 접두사 기반 검색에는 비효율적이다. 트라이는 공통 접두사를 공유하는 구조 덕분에 접두사 검색, 자동완성, 사전 순 정렬 등에서 뛰어난 성능을 보인다.

## 선수 학습

트라이를 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [13-tree](../13-tree/) - 트리 자료구조의 기본 개념, 노드와 간선, 트리 순회 방법
- [26-string-algorithm](../26-string-algorithm/) - 문자열 처리의 기초, 패턴 매칭 등 문자열 알고리즘 기법

## 동작 원리

트라이의 핵심 연산은 삽입(Insert), 검색(Search), 접두사 검색(StartsWith)이다.

### 1. 노드 구조

각 노드는 자식 노드에 대한 포인터 배열(또는 맵)과 해당 노드가 단어의 끝인지를 나타내는 플래그를 가진다.

```
TrieNode {
    children: [26]TrieNode  // 알파벳 소문자 기준
    isEnd: bool              // 이 노드에서 단어가 끝나는지 여부
}
```

### 2. 삽입 (Insert)

문자열을 한 글자씩 따라가며 노드를 생성한다:

1. 루트 노드에서 시작한다
2. 현재 문자에 해당하는 자식 노드가 없으면 새로 생성한다
3. 해당 자식 노드로 이동한다
4. 문자열의 모든 문자를 처리할 때까지 2~3을 반복한다
5. 마지막 노드에 isEnd = true를 표시한다

### 3. 검색 (Search)

문자열이 트라이에 존재하는지 확인한다:

1. 루트 노드에서 시작한다
2. 현재 문자에 해당하는 자식 노드가 없으면 false를 반환한다
3. 해당 자식 노드로 이동한다
4. 문자열의 모든 문자를 처리할 때까지 2~3을 반복한다
5. 마지막 노드의 isEnd가 true이면 true, 아니면 false를 반환한다

### 4. 접두사 검색 (StartsWith)

주어진 접두사로 시작하는 단어가 존재하는지 확인한다:

1. 루트 노드에서 시작한다
2. 현재 문자에 해당하는 자식 노드가 없으면 false를 반환한다
3. 해당 자식 노드로 이동한다
4. 접두사의 모든 문자를 처리할 때까지 2~3을 반복한다
5. 모든 문자를 성공적으로 따라갔으면 true를 반환한다 (isEnd 확인 불필요)

### 예시

"app", "apple", "apt", "bat"을 삽입한 트라이 구조:

```
       (root)
      /      \
     a        b
     |        |
     p        a
    / \       |
   p   t*     t*
   |
   l
   |
   e*
```

`*` 표시는 isEnd = true인 노드이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 삽입 | O(L) |
| 검색 | O(L) |
| 접두사 검색 | O(L) |
| 삭제 | O(L) |
| 공간 복잡도 | O(N × L × Σ) |

- L: 문자열의 길이
- N: 저장된 문자열의 개수
- Σ: 알파벳 크기 (영소문자 기준 26)

공간 복잡도에서 최악의 경우 모든 문자열이 공통 접두사를 공유하지 않으면 O(N × L × Σ)이지만, 실제로는 공통 접두사 공유로 인해 훨씬 적은 공간을 사용한다.

## 적합한 문제 유형

- 문자열 집합에서 접두사 검색이 필요한 문제
- 자동완성 시스템 구현 문제
- 사전(Dictionary) 기반 문자열 검색 문제
- 문자열의 공통 접두사 관련 문제
- XOR 최댓값/최솟값을 구하는 문제 (비트 트라이)
- 문자열 집합의 사전 순 정렬 문제
- 접두사 코드(Prefix-free code) 판별 문제

## 단계별 추적 (Trace)

### 예시 입력

문자열 집합 {"cat", "car", "card", "dog"}을 트라이에 삽입하고, "car", "ca", "card"를 검색한다.

### 삽입 과정

**"cat" 삽입:**

```
(root)
  └─ c
     └─ a
        └─ t*
```

**"car" 삽입:**

```
(root)
  └─ c
     └─ a
        ├─ t*
        └─ r*
```

**"card" 삽입:**

```
(root)
  └─ c
     └─ a
        ├─ t*
        └─ r*
           └─ d*
```

**"dog" 삽입:**

```
(root)
  ├─ c
  │  └─ a
  │     ├─ t*
  │     └─ r*
  │        └─ d*
  └─ d
     └─ o
        └─ g*
```

`*` = isEnd (단어의 끝)

### 검색 과정

**Search("car"):**

```
root → 'c' 존재 → 이동
     → 'a' 존재 → 이동
     → 'r' 존재 → 이동
     → isEnd = true ✓ → 결과: true (단어 존재)
```

**Search("ca"):**

```
root → 'c' 존재 → 이동
     → 'a' 존재 → 이동
     → isEnd = false ✗ → 결과: false (접두사만 존재, 완전한 단어 아님)
```

**StartsWith("ca"):**

```
root → 'c' 존재 → 이동
     → 'a' 존재 → 이동
     → 모든 문자 탐색 완료 → 결과: true (접두사 존재)
```

**Search("card"):**

```
root → 'c' 존재 → 이동
     → 'a' 존재 → 이동
     → 'r' 존재 → 이동
     → 'd' 존재 → 이동
     → isEnd = true ✓ → 결과: true
```

## 실전 팁

### 활용 노하우

- 접두사 관련 문제가 나오면 트라이를 먼저 떠올리자. 해시맵으로는 접두사 검색이 비효율적이다
- XOR 최댓값 문제는 비트 트라이로 풀 수 있다. 각 수를 이진수로 변환하여 상위 비트부터 트라이에 삽입하고, 그리디하게 반대 비트를 선택한다
- 자동완성 구현 시 각 노드에 해당 접두사로 시작하는 단어 수를 저장하면 효율적이다
- Go에서는 `[26]*TrieNode` 배열 대신 `map[byte]*TrieNode`를 사용하면 알파벳 크기에 제한받지 않는다

### 자주 하는 실수

- isEnd 플래그 설정 누락: 삽입 시 마지막 노드에 isEnd = true를 반드시 표시해야 한다. 누락하면 Search가 false를 반환한다
- Search와 StartsWith 혼동: Search는 isEnd까지 확인하고, StartsWith는 경로 존재만 확인한다
- 메모리 초과: 문자열 수가 많고 길이가 길면 노드 수가 폭발적으로 증가한다. 배열 기반 트라이는 `[26]*Node`로 노드당 26개 포인터를 사용하므로 메모리에 주의하자
- 삭제 구현 시 다른 단어에 영향: 공유 접두사 노드를 함부로 삭제하면 다른 단어가 손상된다

### 엣지 케이스

- 빈 문자열 삽입: 루트 노드의 isEnd를 true로 설정한다
- 이미 존재하는 단어 재삽입: isEnd만 true로 설정하면 되므로 중복 처리가 자연스럽다
- 한 단어가 다른 단어의 접두사인 경우: "car"와 "card"처럼 중간 노드에도 isEnd가 설정될 수 있다
- 단일 문자 단어: 루트의 직접 자식에 isEnd가 설정된다

## 관련 알고리즘 비교

| 알고리즘 | 검색 시간 | 접두사 검색 | 공간 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 트라이 | O(L) | O(L) | O(N×L×Σ) | 접두사 검색, 자동완성, XOR 최댓값 |
| 해시맵 | O(L) 평균 | O(N×L) | O(N×L) | 정확한 일치 검색 |
| 이진 탐색 (정렬 배열) | O(L log N) | O(L log N) | O(N×L) | 정렬된 문자열 집합 검색 |
| 접미사 트라이 | O(L) | O(L) | O(N²) | 부분 문자열 검색 |
| 아호-코라식 | O(N+M+Z) | - | O(N×Σ) | 다중 패턴 동시 매칭 |

- 접두사 기반 검색이 핵심이면 → 트라이
- 정확한 일치 검색만 필요하면 → 해시맵 (공간 효율적)
- XOR 관련 문제면 → 비트 트라이
- 다중 패턴 매칭이면 → 아호-코라식 (트라이 기반)
- 부분 문자열 검색이면 → 접미사 배열 또는 접미사 트라이
