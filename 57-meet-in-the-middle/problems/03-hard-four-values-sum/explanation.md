# 합이 0인 네 값 조합 수 세기 - 해설

## 접근 방식

1. 네 배열을 두 그룹으로 나눈다: (A, B)와 (C, D)
2. A와 B의 모든 쌍의 합을 구한다 → 리스트 AB (크기 N²)
3. C와 D의 모든 쌍의 합을 구한다 → 리스트 CD (크기 N²)
4. CD를 정렬한다
5. AB의 각 원소 x에 대해, CD에서 -x의 개수를 이분 탐색으로 센다
6. 모든 개수를 합산하여 출력한다

## 핵심 아이디어

4개 배열에서 하나씩 선택하는 나이브한 접근은 O(N⁴)으로 N=4000일 때 시간 초과이다. 중간에서 만나기(MITM)를 적용하여 배열을 2개씩 묶으면 O(N² log N)으로 해결할 수 있다.

개수를 세기 위해 `sort.SearchInts`를 두 번 사용한다:
- `lo = SearchInts(cd, target)`: target 이상인 첫 위치 (lower_bound)
- `hi = SearchInts(cd, target+1)`: target+1 이상인 첫 위치 (upper_bound)
- `hi - lo`: target과 정확히 같은 원소의 개수

이 기법은 정렬된 배열에서 특정 값의 출현 횟수를 O(log N)에 구하는 표준 패턴이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N² log N) |
| 공간 복잡도 | O(N²) |

- AB, CD 생성: O(N²)
- CD 정렬: O(N² log N)
- AB 순회 × 이분 탐색: O(N² × log(N²)) = O(N² log N)

## 대안적 접근

- 해시맵 방식: CD의 합을 해시맵에 저장하면 O(N²) 평균 시간에 해결 가능하지만, 해시 충돌 시 최악 O(N⁴)이 될 수 있다
- N ≤ 1000이면 해시맵 방식이 더 간단하고 빠를 수 있다
