# 목표값에 가장 가까운 부분집합 합 - 해설

## 접근 방식

1. 배열을 전반부와 후반부로 반씩 나눈다
2. 각 절반의 모든 부분집합 합을 비트마스크로 열거한다
3. 후반부의 합 리스트(sumB)를 정렬한다
4. 전반부의 각 합 a에 대해, sumB에서 S - a에 가장 가까운 값을 이분 탐색으로 찾는다
5. 이분 탐색 결과의 idx와 idx-1 위치를 모두 확인하여 최소 차이를 갱신한다

## 핵심 아이디어

존재 판정 문제와 달리, 정확히 일치하는 값이 아닌 "가장 가까운 값"을 찾아야 한다. `sort.SearchInts`는 target 이상인 첫 위치를 반환하므로, 해당 위치(idx)와 바로 이전 위치(idx-1)를 모두 확인해야 한다.

- `sumB[idx]`: need 이상인 가장 작은 값 → 합이 S 이상인 후보
- `sumB[idx-1]`: need 미만인 가장 큰 값 → 합이 S 미만인 후보

두 후보 중 S와의 차이가 더 작은 쪽을 선택한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^(N/2) × N) |
| 공간 복잡도 | O(2^(N/2)) |

## 대안적 접근

- 투 포인터 방식: sumA를 오름차순, sumB를 내림차순 정렬 후 투 포인터로 탐색할 수도 있다
- N ≤ 20이면 모든 부분집합을 직접 열거하여 O(2^N)으로 해결 가능하다
