# 중간에서 만나기 (Meet in the Middle)

## 개념

중간에서 만나기(Meet in the Middle, MITM)란 탐색 공간을 **반으로 분할**하여 각각 독립적으로 브루트포스 탐색한 뒤, 두 결과를 **합치는(merge)** 최적화 기법이다.

핵심 아이디어는 지수적 탐색 공간 O(2^N)을 O(2^(N/2))로 줄이는 것이다. 예를 들어 N=40인 부분집합 문제에서 2^40 ≈ 10^12은 시간 초과이지만, 2^20 ≈ 10^6은 충분히 처리 가능하다.

**MITM의 3단계:**
1. **분할(Split):** 입력 집합을 절반으로 나눈다 (A = 전반부, B = 후반부)
2. **열거(Enumerate):** A와 B 각각에 대해 모든 부분집합의 결과를 브루트포스로 구한다
3. **합치기(Merge):** 정렬 + 이분 탐색 또는 해시맵을 사용하여 A의 결과와 B의 결과를 조합한다

대표적인 적용 분야:
- **부분집합 합(Subset Sum):** N개 원소에서 합이 S인 부분집합 존재 여부
- **가장 가까운 합:** 목표값에 가장 가까운 부분집합 합 찾기
- **네 수의 합(4SUM):** 네 배열에서 합이 0인 조합 수 세기
- **배낭 문제 변형:** N이 작고 값이 큰 경우의 배낭 문제

## 동작 원리

### 1. 부분집합 합 문제에서의 MITM

N개의 정수 배열에서 합이 정확히 S인 부분집합이 존재하는지 판정한다.

1. 배열을 전반부 A[0..N/2-1]와 후반부 B[N/2..N-1]로 나눈다
2. A의 모든 부분집합 합을 열거한다 → 리스트 sumA (크기 최대 2^(N/2))
3. B의 모든 부분집합 합을 열거한다 → 리스트 sumB (크기 최대 2^(N/2))
4. sumB를 정렬한다
5. sumA의 각 원소 a에 대해, sumB에서 S - a를 이분 탐색한다
6. 찾으면 "존재", 모두 실패하면 "불가능"

### 2. 부분집합 열거 방법

N/2개 원소의 모든 부분집합을 비트마스크로 열거한다.

```
half = N / 2
for mask = 0 to 2^half - 1:
    sum = 0
    for i = 0 to half - 1:
        if mask의 i번째 비트가 1이면:
            sum += arr[i]
    sumA에 sum 추가
```

### 3. 합치기 전략

**방법 1: 정렬 + 이분 탐색**
- sumB를 정렬한 뒤, sumA의 각 원소에 대해 이분 탐색
- 시간: O(2^(N/2) × log(2^(N/2))) = O(2^(N/2) × N/2)

**방법 2: 해시맵**
- sumB를 해시맵에 저장한 뒤, sumA의 각 원소에 대해 O(1) 조회
- 시간: O(2^(N/2)) 평균, 해시 충돌 시 최악 O(2^N)

**방법 3: 투 포인터 (정렬된 경우)**
- sumA를 오름차순, sumB를 내림차순 정렬 후 투 포인터로 합치기
- 시간: O(2^(N/2) × log(2^(N/2)))

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (정렬+이분탐색) | O(2^(N/2) × N) |
| 시간 복잡도 (해시맵) | O(2^(N/2)) 평균 |
| 공간 복잡도 | O(2^(N/2)) |

- 완전 탐색 O(2^N) 대비 지수부가 절반으로 줄어든다
- N=40일 때: 2^40 ≈ 10^12 → 2^20 ≈ 10^6으로 약 100만 배 개선

## 적합한 문제 유형

- N ≤ 40 정도의 부분집합 합 문제
- 두 그룹으로 나눌 수 있는 조합 최적화 문제
- 4SUM, 합이 특정 값인 조합 수 세기
- 배낭 문제에서 N이 작고(≤ 40) 값이 큰 경우
- 양방향 탐색이 가능한 최단 경로 문제 (상태 공간 탐색)
- "N개 중 선택/비선택" 구조의 문제에서 N이 30~40 범위

## 단계별 추적 (Trace)

### 예시: 부분집합 합 존재 판정

배열 `[3, 1, 4, 1, 5, 9]`, 목표 합 S = 10

**1단계: 분할**

```
A = [3, 1, 4]  (전반부, 인덱스 0~2)
B = [1, 5, 9]  (후반부, 인덱스 3~5)
```

**2단계: A의 모든 부분집합 합 열거**

```
mask  선택 원소     합
000   (없음)        0
001   {3}           3
010   {1}           1
011   {3,1}         4
100   {4}           4
101   {3,4}         7
110   {1,4}         5
111   {3,1,4}       8

sumA = [0, 3, 1, 4, 4, 7, 5, 8]
```

**3단계: B의 모든 부분집합 합 열거**

```
mask  선택 원소     합
000   (없음)        0
001   {1}           1
010   {5}           5
011   {1,5}         6
100   {9}           9
101   {1,9}         10
110   {5,9}         14
111   {1,5,9}       15

sumB = [0, 1, 5, 6, 9, 10, 14, 15]
```

**4단계: sumB 정렬**

```
sumB(정렬) = [0, 1, 5, 6, 9, 10, 14, 15]
```

**5단계: 합치기 (이분 탐색)**

```
a=0  → S-a=10, sumB에서 10 탐색 → 찾음! ✓
```

a=0일 때 바로 발견: A에서 아무것도 선택하지 않고, B에서 {1,9}를 선택하면 합 = 10.

**결과:** 합이 10인 부분집합이 존재한다 → `{1, 9}` (인덱스 3, 5)

### 추가 검증

다른 조합도 확인:
- a=1 → S-a=9, sumB에서 9 탐색 → 찾음 (A={1}, B={9}) → 합 1+9=10 ✓
- a=4 → S-a=6, sumB에서 6 탐색 → 찾음 (A={3,1}, B={1,5}) → 합 4+6=10 ✓
- a=5 → S-a=5, sumB에서 5 탐색 → 찾음 (A={1,4}, B={5}) → 합 5+5=10 ✓

## 실전 팁

### 활용 노하우

- N ≤ 20이면 단순 브루트포스(2^20 ≈ 10^6)로 충분하다. MITM은 N이 30~40 범위일 때 진가를 발휘한다
- 부분집합 열거 시 비트마스크를 사용하면 코드가 간결해진다: `for mask := 0; mask < (1 << half); mask++`
- 합치기 단계에서 "개수 세기"가 필요하면 `sort.SearchInts`로 lower_bound와 upper_bound를 구해 차이를 계산한다
- 4SUM 문제는 배열 2개씩 묶어 합을 구한 뒤 MITM을 적용하면 O(N² log N)에 해결된다
- 메모리가 부족할 수 있으므로 2^(N/2)의 크기를 미리 계산하여 확인하자 (N=40이면 2^20 ≈ 100만 개)

### 자주 하는 실수

- 분할 시 전반부와 후반부의 크기를 잘못 계산하는 실수. `half = N/2`로 나누고 나머지는 후반부에 포함시킨다
- 이분 탐색에서 중복 원소를 처리하지 않는 실수. 개수를 세야 하는 문제에서는 lower_bound와 upper_bound를 모두 사용한다
- 빈 부분집합(mask=0)을 빠뜨리는 실수. mask=0도 유효한 부분집합(아무것도 선택하지 않음)이다
- 오버플로우: 원소 값이 크면 합이 int64 범위를 넘을 수 있다. Go에서는 `int`가 64비트이므로 대부분 안전하지만, 명시적으로 확인하자

### 엣지 케이스

- N = 0 또는 빈 배열: 합이 0인 부분집합만 존재 (빈 집합)
- N = 1: 원소 하나만 있으므로 선택/비선택 2가지만 확인
- 모든 원소가 0: 모든 부분집합의 합이 0이므로 S=0이면 2^N개 조합
- 목표 합 S가 매우 크거나 음수: 음수 원소가 있으면 합이 음수가 될 수 있으므로 범위 주의
- 원소에 음수가 포함된 경우: 합의 범위가 넓어지므로 해시맵 방식이 유리할 수 있다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 적합한 N 범위 | 특징 |
| --- | --- | --- | --- |
| 완전 탐색 (브루트포스) | O(2^N) | N ≤ 20 | 단순하지만 N이 크면 불가능 |
| 중간에서 만나기 (MITM) | O(2^(N/2) × N) | N ≤ 40 | 탐색 공간 반분할로 지수부 절반 |
| DP (배낭) | O(N × S) | S ≤ 10^7 정도 | 합의 범위가 작을 때 효율적 |
| 백트래킹 + 가지치기 | O(2^N) 최악 | N ≤ 25~30 | 가지치기로 평균 성능 개선 |

**선택 기준:**
- N ≤ 20 → 완전 탐색
- N ≤ 40, 합의 범위가 큼 → MITM
- N이 크고 합의 범위가 작음 (S ≤ 10^7) → DP
- N ≤ 25~30, 가지치기가 효과적 → 백트래킹

## 변형 및 최적화

### 4SUM (네 수의 합)

네 배열 A, B, C, D에서 a+b+c+d = 0인 조합의 수를 구하는 문제이다.

1. A+B의 모든 합을 구한다 → 리스트 AB
2. C+D의 모든 합을 구한다 → 리스트 CD
3. AB의 각 원소 x에 대해 CD에서 -x의 개수를 센다

시간 복잡도: O(N² log N), 공간 복잡도: O(N²)

### 양방향 BFS

상태 공간 탐색에서 시작점과 끝점 양쪽에서 BFS를 수행하여 중간에서 만나는 기법이다. 탐색 공간이 b^d일 때 2×b^(d/2)로 줄어든다.

### 가중치 부분집합 최적화

부분집합 열거 시 가지치기를 추가하여 실제 탐색 공간을 줄일 수 있다. 예를 들어 합이 S를 초과하면 더 이상 원소를 추가하지 않는다.
