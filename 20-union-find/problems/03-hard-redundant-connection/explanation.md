# 여분의 간선 - 해설

## 접근 방식

1. N개의 노드를 각각 독립된 집합으로 초기화한다.
2. 간선을 입력 순서대로 하나씩 읽으며 Union 연산을 수행한다.
3. Union 연산 시 두 노드가 이미 같은 집합에 속하면, 해당 간선이 사이클을 형성하는 여분의 간선이다.
4. 문제 조건에 따라 가장 마지막에 발견된 사이클 간선을 답으로 출력한다.

## 핵심 아이디어

트리는 N개의 노드와 N-1개의 간선으로 이루어진 연결 그래프이다. 여기에 간선이 하나 추가되면 정확히 하나의 사이클이 생긴다. 이 사이클에 포함된 간선 중 하나를 제거하면 다시 트리가 된다.

유니온 파인드를 사용하면 간선을 추가할 때 사이클 형성 여부를 O(α(N))에 판별할 수 있다. 두 노드의 루트가 같다면 이미 연결되어 있으므로, 해당 간선을 추가하면 사이클이 생긴다.

이 문제에서는 사이클이 정확히 하나이므로, 사이클을 형성하는 간선도 정확히 하나이다. 간선을 입력 순서대로 처리하면 자연스럽게 "입력에서 가장 마지막에 주어진" 조건을 만족한다. 사이클에 포함된 간선 중 입력 순서가 가장 뒤인 간선이 Union 실패를 일으키기 때문이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × α(N)) |
| 공간 복잡도 | O(N) |

N개의 간선에 대해 각각 Find/Union 연산을 수행하므로 O(N × α(N)) ≈ O(N) 시간이 소요된다. parent와 rank 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- DFS로 사이클을 찾은 뒤, 사이클에 포함된 간선 중 입력 순서가 가장 뒤인 것을 선택하는 방법도 가능하다. 그러나 유니온 파인드 방식이 구현이 더 간단하고 직관적이다.
- 방향 그래프에서의 여분의 간선 문제는 더 복잡하며, 진입 차수 분석과 유니온 파인드를 결합해야 한다.
