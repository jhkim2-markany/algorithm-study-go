# 네트워크 연결 - 해설

## 접근 방식

1. 모든 컴퓨터를 최소 비용으로 연결하는 문제는 최소 신장 트리(MST) 문제이다.
2. 크루스칼 알고리즘을 사용한다: 간선을 비용 기준으로 오름차순 정렬한다.
3. 비용이 작은 간선부터 순서대로 확인하며, 두 노드가 아직 연결되지 않았으면 해당 간선을 선택한다.
4. 유니온 파인드로 두 노드의 연결 여부를 판별하고, 선택 시 Union 연산으로 합친다.
5. N-1개의 간선을 선택하면 MST가 완성된다.
6. 모든 간선을 확인해도 N-1개를 선택하지 못하면 연결 불가능(-1)이다.

## 핵심 아이디어

크루스칼 알고리즘은 그리디 전략과 유니온 파인드를 결합한 MST 알고리즘이다. 핵심은 "비용이 가장 작은 간선부터 선택하되, 사이클을 형성하는 간선은 건너뛴다"는 것이다.

사이클 판별에 유니온 파인드가 사용된다. 두 노드의 루트가 같으면 이미 같은 연결 요소에 속하므로 해당 간선을 추가하면 사이클이 생긴다. 루트가 다르면 안전하게 간선을 추가할 수 있다.

N개의 노드를 모두 연결하려면 정확히 N-1개의 간선이 필요하다. 따라서 N-1개의 간선을 선택한 시점에서 알고리즘을 종료할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M log M) |
| 공간 복잡도 | O(N + M) |

간선 정렬에 O(M log M), M번의 Union/Find 연산에 O(M × α(N))이 소요된다. 정렬이 지배적이므로 전체 시간 복잡도는 O(M log M)이다. 간선 배열에 O(M), 유니온 파인드에 O(N) 공간이 필요하다.

## 대안적 접근

- 프림(Prim) 알고리즘을 사용할 수도 있다. 우선순위 큐 기반 프림은 O(M log N)이며, 간선이 많은 밀집 그래프에서는 프림이 더 효율적일 수 있다.
- 간선이 적은 희소 그래프에서는 크루스칼이 구현이 간단하고 효율적이다.
