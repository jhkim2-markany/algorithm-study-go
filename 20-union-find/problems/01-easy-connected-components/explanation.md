# 연결 요소 개수 - 해설

## 접근 방식

1. N개의 노드를 각각 독립된 집합으로 초기화한다.
2. M개의 간선을 순서대로 읽으며, 각 간선의 두 노드를 Union 연산으로 같은 집합에 합친다.
3. 모든 간선을 처리한 후, 루트가 자기 자신인 노드의 수를 세면 연결 요소의 개수가 된다.

## 핵심 아이디어

유니온 파인드의 가장 기본적인 활용 문제이다. 그래프의 각 간선은 두 노드를 같은 연결 요소로 묶는 역할을 하므로, Union 연산과 자연스럽게 대응된다.

경로 압축과 랭크 기반 합치기를 적용하면 각 Union/Find 연산이 거의 O(1)에 수행되므로, M개의 간선을 처리하는 데 O(M × α(N)) ≈ O(M) 시간이 소요된다.

최종적으로 `find(i) == i`인 노드를 세면 서로 다른 루트의 수, 즉 연결 요소의 개수를 구할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M × α(N)) |
| 공간 복잡도 | O(N) |

초기화에 O(N), M번의 Union 연산에 O(M × α(N)), 연결 요소 세기에 O(N) 시간이 소요된다. parent와 rank 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- DFS/BFS로 각 연결 요소를 탐색하는 방법도 가능하다. 시간 복잡도는 O(N + M)으로 동일하지만, 유니온 파인드는 간선이 동적으로 추가되는 온라인 쿼리 상황에서 더 유리하다.
