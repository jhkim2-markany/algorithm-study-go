# 유니온 파인드

## 개념

유니온 파인드(Union-Find)는 서로소 집합(Disjoint Set Union, DSU)이라고도 불리며, 여러 원소를 겹치지 않는 집합들로 분류하고 관리하는 자료구조이다. 두 가지 핵심 연산을 제공한다:

- **Find**: 특정 원소가 속한 집합의 대표 원소(루트)를 찾는다
- **Union**: 두 원소가 속한 집합을 하나로 합친다

유니온 파인드의 핵심 특성은 다음과 같다:
- **트리 기반 표현**: 각 집합을 트리로 표현하며, 트리의 루트가 집합의 대표 원소이다
- **경로 압축(Path Compression)**: Find 연산 시 방문한 모든 노드를 루트에 직접 연결하여 이후 탐색을 빠르게 한다
- **랭크/크기 기반 합치기(Union by Rank/Size)**: Union 연산 시 작은 트리를 큰 트리 아래에 붙여 트리의 높이를 최소화한다
- **거의 상수 시간 연산**: 경로 압축과 랭크 기반 합치기를 함께 사용하면 연산당 평균 O(α(N)) 시간이 소요된다 (α는 아커만 함수의 역함수)

유니온 파인드는 그래프의 연결 요소 판별, 사이클 탐지, 크루스칼 알고리즘(최소 신장 트리), 네트워크 연결 문제 등에 널리 사용된다.

## 동작 원리

### 초기화

1. N개의 원소 각각을 독립된 집합으로 초기화한다.
2. 각 원소의 부모를 자기 자신으로 설정한다: `parent[i] = i`
3. 각 원소의 랭크(트리 높이)를 0으로 설정한다: `rank[i] = 0`

### Find 연산 (경로 압축)

1. 현재 노드의 부모가 자기 자신이면 루트이므로 반환한다.
2. 그렇지 않으면 부모를 따라 재귀적으로 루트를 찾는다.
3. **경로 압축**: 재귀 반환 시 현재 노드의 부모를 루트로 직접 갱신한다.
4. 이후 같은 노드에 대한 Find 호출은 O(1)에 가까워진다.

```
Find(x):
    if parent[x] ≠ x:
        parent[x] = Find(parent[x])  // 경로 압축
    return parent[x]
```

### Union 연산 (랭크 기반 합치기)

1. 두 원소 x, y의 루트를 각각 Find로 찾는다.
2. 두 루트가 같으면 이미 같은 집합이므로 아무 작업도 하지 않는다.
3. 두 루트가 다르면 랭크가 낮은 트리를 랭크가 높은 트리 아래에 붙인다.
4. 랭크가 같으면 한쪽을 다른 쪽 아래에 붙이고, 붙인 쪽의 랭크를 1 증가시킨다.

```
Union(x, y):
    rootX = Find(x)
    rootY = Find(y)
    if rootX == rootY: return
    if rank[rootX] < rank[rootY]:
        parent[rootX] = rootY
    else if rank[rootX] > rank[rootY]:
        parent[rootY] = rootX
    else:
        parent[rootY] = rootX
        rank[rootX] += 1
```

### 크기 기반 합치기 (Union by Size)

랭크 대신 각 집합의 크기(원소 수)를 기준으로 합칠 수도 있다. 작은 집합을 큰 집합에 합치면 트리의 높이 증가를 최소화할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| Find (경로 압축 없음) | O(N) |
| Find (경로 압축) | O(α(N)) ≈ O(1) |
| Union (랭크/크기 기반) | O(α(N)) ≈ O(1) |
| 초기화 | O(N) |
| 공간 복잡도 | O(N) |

α(N)은 아커만 함수의 역함수로, 실질적으로 모든 입력에 대해 5 이하의 값을 가진다. 따라서 경로 압축과 랭크 기반 합치기를 함께 사용하면 M번의 연산에 대해 총 O(M × α(N)) ≈ O(M) 시간이 소요된다.

## 적합한 문제 유형

- 그래프의 연결 요소(Connected Components) 개수 구하기
- 그래프에서 사이클 탐지
- 크루스칼 알고리즘을 이용한 최소 신장 트리 구성
- 네트워크 연결/분리 판별
- 동적 연결성(Dynamic Connectivity) 문제
- 여분의 간선(Redundant Connection) 찾기
- 집합의 동적 병합이 필요한 문제
- 온라인 쿼리로 집합 관계를 판별하는 문제

## 단계별 추적 (Trace)

### 예시 입력

정점 5개 (0~4), 연산 순서: Union(0,1), Union(2,3), Union(1,3), Find(0), Find(4)

### 실행 과정

**초기 상태:**

```
parent: [0, 1, 2, 3, 4]
rank:   [0, 0, 0, 0, 0]

각 원소가 독립 집합:
{0}  {1}  {2}  {3}  {4}
```

**Union(0, 1):**

```
Find(0) = 0, Find(1) = 1  (루트가 다름 → 합침)
rank 동일 → parent[1] = 0, rank[0] = 1

parent: [0, 0, 2, 3, 4]
rank:   [1, 0, 0, 0, 0]

트리 구조:
  0       2   3   4
  |
  1
```

**Union(2, 3):**

```
Find(2) = 2, Find(3) = 3  (루트가 다름 → 합침)
rank 동일 → parent[3] = 2, rank[2] = 1

parent: [0, 0, 2, 2, 4]
rank:   [1, 0, 1, 0, 0]

트리 구조:
  0       2       4
  |       |
  1       3
```

**Union(1, 3):**

```
Find(1): parent[1]=0, parent[0]=0 → 루트 = 0
Find(3): parent[3]=2, parent[2]=2 → 루트 = 2
루트가 다름 → 합침, rank 동일 → parent[2] = 0, rank[0] = 2

parent: [0, 0, 0, 2, 4]
rank:   [2, 0, 1, 0, 0]

트리 구조:
    0           4
   / \
  1   2
      |
      3
```

**Find(0):**

```
parent[0] = 0 → 자기 자신이 루트 → 반환 0
```

**Find(4):**

```
parent[4] = 4 → 자기 자신이 루트 → 반환 4
0과 4는 다른 집합에 속함
```

**경로 압축 예시 - Find(3):**

```
Find(3): parent[3]=2 → Find(2): parent[2]=0 → Find(0): 루트
경로 압축 적용: parent[3] = 0, parent[2] = 0 (이미 0)

압축 후 트리:
    0           4
  / | \
 1  2  3
```

## 실전 팁

### 활용 노하우

- Union-Find는 "같은 그룹인지 판별" + "그룹 합치기"가 핵심이므로, 문제에서 "연결", "같은 집합", "그룹"이라는 키워드가 나오면 먼저 떠올리자
- 크루스칼 알고리즘에서 사이클 판별 용도로 가장 많이 사용된다
- 집합의 크기를 관리하려면 Union by Size를 사용하고, `size[]` 배열을 루트 노드에만 유지하면 된다
- 오프라인 쿼리에서 간선을 추가하면서 연결 여부를 판별할 때 매우 효율적이다
- 2차원 격자에서 인접 칸을 합칠 때 `(row * cols + col)`로 1차원 인덱스로 변환하여 사용한다

### 자주 하는 실수

- Find 함수에서 경로 압축을 빠뜨리면 최악 O(N)이 되어 TLE 발생
- Union 시 Find를 호출하지 않고 원소 자체를 비교하는 실수 (반드시 루트끼리 비교해야 함)
- `parent` 배열 초기화를 잊는 경우 (`parent[i] = i`로 반드시 초기화)
- Union by Rank와 Union by Size를 혼용하면 안 된다 (하나만 선택)
- 0-indexed와 1-indexed를 혼동하여 배열 범위를 벗어나는 경우

### 엣지 케이스

- 같은 원소끼리 Union하는 경우 (이미 같은 집합 → 무시)
- 모든 원소가 하나의 집합으로 합쳐지는 경우
- 원소가 1개뿐인 경우
- Union 없이 Find만 반복하는 경우 (각 원소가 독립 집합)
- 정점 번호가 0부터 시작하는지 1부터 시작하는지 확인

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| Union-Find | 집합 합치기/판별 O(α(N)), 경로 압축으로 거의 상수 시간 | 동적으로 집합을 합치면서 같은 집합 여부를 판별할 때 |
| BFS/DFS 연결 요소 | 그래프 전체 탐색 O(V+E), 정적 그래프에서 연결 요소 탐색 | 그래프가 고정되어 있고 한 번만 연결 요소를 구할 때 |
| Kruskal (MST) | Union-Find 기반, 간선 정렬 후 사이클 없이 선택 | 최소 신장 트리를 구할 때 (Union-Find 활용) |
| Prim (MST) | 우선순위 큐 기반, 정점 중심 확장 | 밀집 그래프에서 MST를 구할 때 |
| Weighted Union-Find | 간선에 가중치(차이값)를 부여하여 상대적 관계 관리 | 원소 간 상대적 차이/관계를 관리해야 할 때 |
