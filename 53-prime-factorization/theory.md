# 소인수분해

## 개념

소인수분해(Prime Factorization)란 1보다 큰 자연수를 소수(Prime Number)의 곱으로 나타내는 것이다. 산술의 기본 정리(Fundamental Theorem of Arithmetic)에 의해, 모든 1보다 큰 자연수는 소수의 곱으로 유일하게 표현된다(순서 무시).

예를 들어 60 = 2² × 3 × 5이다. 소인수분해는 약수 개수 계산, GCD/LCM 계산, 오일러 파이 함수 등 다양한 정수론 문제의 기반이 된다.

## 동작 원리

### 1. 시행 나눗셈 (Trial Division)

가장 기본적인 소인수분해 방법으로, 2부터 √N까지의 소수로 나누어 보는 방식이다.

1. 소인수 후보 d를 2부터 시작한다
2. N이 d로 나누어 떨어지면, d로 더 이상 나누어지지 않을 때까지 반복하여 나눈다
3. 나눈 횟수가 d의 지수가 된다
4. d를 1 증가시키고 d² ≤ N인 동안 2~3을 반복한다
5. 반복 종료 후 N > 1이면 N 자체가 마지막 소인수이다

핵심 원리: N이 합성수라면 √N 이하의 소인수를 반드시 가진다. 따라서 √N까지만 확인하면 나머지 하나의 소인수(있다면)는 자동으로 결정된다.

### 2. 최적화된 시행 나눗셈

기본 시행 나눗셈에서 짝수를 별도 처리하여 탐색 범위를 절반으로 줄인다.

1. 먼저 2로 나누어 떨어지는 만큼 나눈다
2. 이후 d = 3부터 시작하여 d += 2 (홀수만 탐색)로 진행한다
3. d² ≤ N인 동안 나누어 떨어지면 반복하여 나눈다
4. 반복 종료 후 N > 1이면 마지막 소인수이다

### 3. 최소 소인수(SPF) 체를 이용한 소인수분해

에라토스테네스의 체를 변형하여 각 수의 최소 소인수(Smallest Prime Factor)를 전처리한 뒤, 이를 이용해 O(log N)에 소인수분해하는 방법이다.

**전처리 단계 (SPF 배열 구축):**

1. 크기 N+1의 배열 spf를 생성하고 spf[i] = i로 초기화한다
2. 2부터 √N까지 순회한다
3. spf[i] == i이면 i는 소수이다
4. i의 배수 j (i²부터 시작)에 대해, spf[j]가 아직 갱신되지 않았으면 spf[j] = i로 설정한다

**소인수분해 단계:**

1. 수 x에 대해 spf[x]가 최소 소인수이다
2. x를 spf[x]로 나누어 떨어지지 않을 때까지 나눈다
3. x가 1이 될 때까지 반복한다

이 방법은 여러 수를 반복적으로 소인수분해해야 할 때 매우 효율적이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시행 나눗셈 (단일 수) | O(√N) |
| 최적화 시행 나눗셈 (단일 수) | O(√N) (상수 개선) |
| SPF 체 전처리 | O(N log log N) |
| SPF 기반 소인수분해 (단일 수) | O(log N) |
| SPF 기반 Q개 쿼리 | O(N log log N + Q log N) |
| 공간 복잡도 (시행 나눗셈) | O(1) |
| 공간 복잡도 (SPF 체) | O(N) |

## 적합한 문제 유형

- 주어진 수를 소인수분해하여 출력하는 문제
- 약수의 개수, 약수의 합을 구하는 문제 (소인수분해 활용)
- 여러 수에 대해 반복적으로 소인수분해가 필요한 문제 (SPF 체 활용)
- 오일러 파이 함수(φ(n))를 계산하는 문제
- GCD/LCM을 소인수분해로 구하는 문제
- 특정 범위 내 수들의 소인수 관련 통계를 구하는 문제
- N!의 소인수분해 (르장드르 공식 활용)

## 단계별 추적 (Trace)

### 예시 입력

N = 360을 시행 나눗셈과 SPF 체 두 가지 방법으로 소인수분해한다.

### 방법 1: 최적화된 시행 나눗셈

```
N = 360

d=2: 360 ÷ 2 = 180  (2의 개수: 1)
     180 ÷ 2 = 90   (2의 개수: 2)
     90  ÷ 2 = 45   (2의 개수: 3)
     45  ÷ 2 → 나누어지지 않음
     → 2³

d=3: 45  ÷ 3 = 15   (3의 개수: 1)
     15  ÷ 3 = 5    (3의 개수: 2)
     5   ÷ 3 → 나누어지지 않음
     → 3²

d=5: 5² = 25 > 5? → 아니오 (25 > 5)
     5   ÷ 5 = 1    (5의 개수: 1)
     → 5¹

d=7: 7² = 49 > 1 → 종료 (N=1이므로 마지막 소인수 없음)

결과: 360 = 2³ × 3² × 5¹
```

### 방법 2: SPF 체를 이용한 소인수분해

**SPF 배열 (일부):**

```
인덱스:  2  3  4  5  6  7  8  9  10 ...
spf:    2  3  2  5  2  7  2  3  2  ...
```

**소인수분해 과정:**

```
x = 360

spf[360] = 2 → 360 ÷ 2 = 180
spf[180] = 2 → 180 ÷ 2 = 90
spf[90]  = 2 → 90  ÷ 2 = 45
spf[45]  = 3 → 45  ÷ 3 = 15
spf[15]  = 3 → 15  ÷ 3 = 5
spf[5]   = 5 → 5   ÷ 5 = 1 → 종료

결과: 360 = 2³ × 3² × 5¹ ✓
```

SPF 체 방식은 각 단계에서 나눌 소수를 즉시 알 수 있어 O(log N)에 완료된다.

## 실전 팁

### 활용 노하우

- 단일 수의 소인수분해는 시행 나눗셈으로 충분하다. 2를 먼저 처리하고 홀수만 탐색하면 상수가 절반으로 줄어든다
- 여러 수를 반복적으로 소인수분해해야 하면 SPF 체를 전처리하자. 전처리 O(N log log N), 쿼리 O(log N)이다
- 약수의 개수: 소인수분해 결과가 p₁^a₁ × p₂^a₂ × ... 이면 약수의 개수는 (a₁+1)(a₂+1)...이다
- N!의 소인수분해에서 소수 p의 지수는 르장드르 공식 `⌊N/p⌋ + ⌊N/p²⌋ + ⌊N/p³⌋ + ...`으로 구한다

### 자주 하는 실수

- √N까지만 탐색하는 것을 잊는 실수: `d*d <= N` 조건을 사용하자. `d <= sqrt(N)`은 부동소수점 오차가 발생할 수 있다
- 반복 종료 후 N > 1 처리 누락: 마지막에 남은 N이 1보다 크면 그 자체가 소인수이다. 이를 빠뜨리면 큰 소인수를 놓친다
- SPF 체에서 배열 크기 부족: 소인수분해할 최대 값까지 배열을 할당해야 한다
- 1을 소인수분해하려는 실수: 1은 소인수가 없다. 입력이 1인 경우를 별도 처리하자

### 엣지 케이스

- N = 1: 소인수가 없다. 빈 결과를 반환한다
- N이 소수인 경우: √N까지 나누어지는 수가 없고, 마지막에 N 자체가 유일한 소인수이다
- N = 2: 가장 작은 소수. 시행 나눗셈에서 d=2로 바로 처리된다
- N이 큰 소수의 제곱인 경우 (예: 49 = 7²): √N = 7에서 정확히 나누어진다
- N이 매우 큰 경우 (10¹⁸): 시행 나눗셈은 O(√N) ≈ O(10⁹)으로 시간 초과. 폴라드 로 알고리즘 등 고급 기법이 필요하다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 적합한 상황 |
| --- | --- | --- |
| 시행 나눗셈 | O(√N) | 단일 수 소인수분해 (N ≤ 10¹²) |
| SPF 체 + 소인수분해 | O(N log log N) 전처리 + O(log N) 쿼리 | 다수의 수 반복 소인수분해 (N ≤ 10⁷) |
| 에라토스테네스의 체 | O(N log log N) | 범위 내 소수 목록 생성 |
| 소수 판정 (밀러-라빈) | O(k log² N) | 소수 여부만 판별 (소인수분해 아님) |
| 폴라드 로 알고리즘 | O(N^(1/4)) 기대 | 매우 큰 수의 소인수분해 (N ≤ 10¹⁸) |

- 단일 수 소인수분해 (N ≤ 10¹²) → 시행 나눗셈
- 다수의 수 반복 소인수분해 (N ≤ 10⁷) → SPF 체
- 소수 목록만 필요하면 → 에라토스테네스의 체
- 소수 여부만 판별하면 → 밀러-라빈
- 매우 큰 수 (N > 10¹²) → 폴라드 로 알고리즘
