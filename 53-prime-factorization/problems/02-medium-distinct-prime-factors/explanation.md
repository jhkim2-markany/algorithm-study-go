# 서로 다른 소인수 개수 쿼리 - 해설

## 접근 방식

1. SPF(최소 소인수) 체를 구축하여 각 수의 최소 소인수를 전처리한다
2. 2부터 N까지 각 수에 대해 SPF 배열을 이용하여 서로 다른 소인수의 개수를 계산하고 저장한다
3. 각 쿼리에 대해 전처리된 결과를 O(1)에 응답한다

## 핵심 아이디어

Q개의 쿼리를 각각 시행 나눗셈으로 처리하면 O(Q × √N)이 되어 시간 초과가 발생할 수 있다. 대신 SPF 체를 전처리하면 각 수의 소인수분해를 O(log N)에 수행할 수 있다.

더 나아가, 2부터 N까지 모든 수의 서로 다른 소인수 개수를 미리 계산해 두면 각 쿼리를 O(1)에 처리할 수 있다. SPF 배열을 이용한 소인수분해는 최소 소인수로 나누고, 같은 소인수를 모두 제거한 뒤 다음 최소 소인수로 넘어가는 방식이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log log N + N log N + Q) |
| 공간 복잡도 | O(N) |

- SPF 체 구축: O(N log log N)
- 전처리 (각 수의 소인수 개수): O(N log N)
- 쿼리 응답: O(Q)

## 대안적 접근

- 에라토스테네스의 체를 변형하여 각 소수 p의 배수에 대해 카운트를 1씩 증가시키는 방법도 있다. 이 경우 전처리가 O(N log log N)으로 더 효율적이다.
- 쿼리 수가 적고 N이 매우 크다면, SPF 체 대신 각 쿼리마다 시행 나눗셈을 사용하는 것이 메모리 측면에서 유리할 수 있다.
