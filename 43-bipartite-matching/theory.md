# 이분 매칭

## 개념

이분 매칭(Bipartite Matching)은 이분 그래프(Bipartite Graph)에서 최대 매칭을 구하는 문제이다. 이분 그래프란 정점을 두 그룹으로 나눌 수 있고, 같은 그룹 내의 정점끼리는 간선이 없는 그래프를 말한다. 매칭(Matching)이란 어떤 정점도 두 번 이상 사용되지 않는 간선의 집합이며, 최대 매칭은 간선 수가 최대인 매칭이다.

이분 매칭은 작업 배정, 자원 할당, 스케줄링 등 "한쪽 그룹의 원소를 다른 쪽 그룹의 원소에 1:1로 대응시키는" 문제에 널리 활용된다. 대표적인 알고리즘으로 DFS 기반의 헝가리안 알고리즘(Kuhn's Algorithm)이 있으며, 최대 유량(Maximum Flow)으로도 풀 수 있다.

쾨니그 정리(König's Theorem)에 의해, 이분 그래프에서 최대 매칭의 크기는 최소 버텍스 커버의 크기와 같다. 이 정리는 이분 매칭 문제의 이론적 기반이 된다.

## 동작 원리

이분 매칭의 핵심은 증가 경로(augmenting path)를 반복적으로 찾아 매칭을 확장하는 것이다.

### 1. 이분 그래프와 매칭

이분 그래프 G = (L ∪ R, E)에서:
- L: 왼쪽 정점 그룹
- R: 오른쪽 정점 그룹
- E: L과 R 사이의 간선 집합

매칭 M ⊆ E는 M에 포함된 간선들이 어떤 정점도 공유하지 않는 간선 집합이다. 매칭에 포함되지 않은 정점을 자유 정점(free vertex)이라 한다.

### 2. 증가 경로 (Augmenting Path)

증가 경로란 자유 정점에서 시작하여 자유 정점에서 끝나는 경로로, 매칭에 포함된 간선과 포함되지 않은 간선이 번갈아 나타나는 경로이다. 증가 경로를 찾으면 경로 상의 매칭/비매칭 간선을 뒤집어 매칭 크기를 1 증가시킬 수 있다.

### 3. 헝가리안 알고리즘 (Kuhn's Algorithm)

DFS를 이용하여 증가 경로를 찾는 알고리즘이다:

1. 모든 정점을 매칭되지 않은 상태로 초기화한다
2. 왼쪽 그룹의 각 정점 u에 대해:
   a. 방문 배열을 초기화한다
   b. DFS로 u에서 시작하는 증가 경로를 찾는다
   c. 증가 경로가 존재하면 매칭을 갱신하고 매칭 수를 1 증가시킨다
3. DFS 과정:
   a. u와 연결된 오른쪽 정점 v를 순회한다
   b. v가 아직 방문하지 않았으면 방문 표시한다
   c. v가 매칭되지 않았거나, v의 현재 매칭 상대에서 다른 증가 경로를 찾을 수 있으면 u-v를 매칭한다

### 4. 최대 유량으로의 변환

이분 매칭은 최대 유량 문제로 변환할 수 있다:

1. 가상의 소스 S를 추가하고 왼쪽 그룹의 모든 정점에 용량 1의 간선을 연결한다
2. 가상의 싱크 T를 추가하고 오른쪽 그룹의 모든 정점에서 T로 용량 1의 간선을 연결한다
3. 원래 이분 그래프의 간선은 용량 1로 설정한다
4. S에서 T로의 최대 유량이 최대 매칭의 크기이다

### 5. 쾨니그 정리 (König's Theorem)

이분 그래프에서 최대 매칭의 크기 = 최소 버텍스 커버의 크기이다.

- 버텍스 커버(Vertex Cover): 모든 간선의 적어도 한 끝점을 포함하는 정점 집합
- 최소 버텍스 커버: 크기가 최소인 버텍스 커버

최소 버텍스 커버는 최대 매칭을 구한 후, 잔여 그래프에서 교대 경로 분석을 통해 구할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 헝가리안 알고리즘 (Kuhn) | O(V × E) |
| Hopcroft-Karp 알고리즘 | O(E × √V) |
| 최대 유량 변환 (Edmonds-Karp) | O(V × E²) |
| 공간 복잡도 | O(V + E) |

헝가리안 알고리즘은 왼쪽 정점 수만큼 DFS를 수행하며, 각 DFS는 O(E)이므로 전체 O(V × E)이다. Hopcroft-Karp는 BFS와 DFS를 결합하여 O(E × √V)로 더 빠르지만 구현이 복잡하다. 코딩테스트에서는 헝가리안 알고리즘이 주로 사용된다.

## 적합한 문제 유형

- 작업을 작업자에게 1:1로 배정하는 문제
- 이분 그래프에서 최대 매칭 크기를 구하는 문제
- 최소 버텍스 커버를 구하는 문제 (쾨니그 정리 활용)
- 최대 독립 집합을 구하는 문제 (전체 정점 수 - 최소 버텍스 커버)
- 스케줄링 및 자원 할당 문제
- 체스판 위의 배치 문제 (흑백 칸을 이분 그래프로 모델링)
- 네트워크 유량으로 변환 가능한 매칭 문제

## 단계별 추적 (Trace)

### 예시: 헝가리안 알고리즘으로 최대 매칭 구하기

이분 그래프:
- 왼쪽 그룹 L = {1, 2, 3}
- 오른쪽 그룹 R = {A, B, C}
- 간선: 1-A, 1-B, 2-A, 2-C, 3-B

```
초기 상태: 매칭 없음
  match[A] = -1, match[B] = -1, match[C] = -1

[라운드 1] 왼쪽 정점 1에서 증가 경로 탐색
  DFS(1):
    1 → A: A가 미매칭 → 매칭 성공!
    match[A] = 1
  매칭 수: 1

  현재 매칭: 1─A

[라운드 2] 왼쪽 정점 2에서 증가 경로 탐색
  DFS(2):
    2 → A: A가 이미 1과 매칭됨
      → 1에서 다른 경로 탐색 (DFS(1))
        1 → B: B가 미매칭 → 매칭 성공!
        match[B] = 1
      → A를 2에게 양보 가능!
    match[A] = 2
  매칭 수: 2

  현재 매칭: 1─B, 2─A

[라운드 3] 왼쪽 정점 3에서 증가 경로 탐색
  DFS(3):
    3 → B: B가 이미 1과 매칭됨
      → 1에서 다른 경로 탐색 (DFS(1))
        1 → A: A가 이미 2와 매칭됨
          → 2에서 다른 경로 탐색 (DFS(2))
            2 → C: C가 미매칭 → 매칭 성공!
            match[C] = 2
          → A를 1에게 양보 가능!
        match[A] = 1
      → B를 3에게 양보 가능!
    match[B] = 3
  매칭 수: 3

  최종 매칭: 1─A, 2─C, 3─B
```

### 증가 경로 시각화

```
라운드 3에서 찾은 증가 경로:

  3 ──→ B ──→ 1 ──→ A ──→ 2 ──→ C
  (미매칭) (매칭) (매칭) (매칭) (매칭) (미매칭)

  비매칭 간선: 3→B, 1→A, 2→C
  매칭 간선:   B→1, A→2

  경로 상의 매칭/비매칭을 뒤집으면:
  새 매칭: 3─B, 1─A, 2─C  (매칭 크기 +1)
```

## 실전 팁

### 활용 노하우

- 이분 매칭 문제인지 판별하는 핵심: "한쪽 그룹의 원소를 다른 쪽에 1:1로 대응"시키는 구조가 보이면 이분 매칭을 의심하자
- 체스판 문제에서 흑/백 칸을 두 그룹으로 나누면 이분 그래프가 된다. 격자 위의 배치 문제에서 자주 활용된다
- 최소 버텍스 커버 = 최대 매칭 (쾨니그 정리), 최대 독립 집합 = 전체 정점 수 - 최대 매칭. 이 관계를 기억하면 다양한 문제를 풀 수 있다
- 헝가리안 알고리즘은 구현이 간단하고 대부분의 코딩테스트 제한 내에서 충분히 빠르다

### 자주 하는 실수

- 방문 배열(visited)을 각 라운드마다 초기화하지 않는 경우. 매 왼쪽 정점마다 visited를 새로 초기화해야 한다
- 이분 그래프가 아닌 그래프에 이분 매칭을 적용하려는 경우. 먼저 이분 그래프인지 확인하자
- 간선 방향을 잘못 설정하여 왼쪽→오른쪽이 아닌 양방향으로 탐색하는 경우
- match 배열의 인덱스를 왼쪽/오른쪽 그룹 중 어느 쪽 기준으로 관리하는지 혼동하는 경우
- 정점 번호가 0부터 시작하는지 1부터 시작하는지에 따른 인덱스 오류

### 엣지 케이스

- 간선이 하나도 없는 경우 — 최대 매칭 = 0
- 완전 이분 그래프인 경우 — 최대 매칭 = min(|L|, |R|)
- 한쪽 그룹의 크기가 0인 경우
- 한 정점에서 모든 상대 정점으로 간선이 있는 경우
- 매칭이 불가능한 정점이 존재하는 경우 (연결된 간선이 없는 정점)

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| 헝가리안 알고리즘 (Kuhn) | O(V × E) | DFS 기반, 구현 간단 | 코딩테스트에서 가장 추천 |
| Hopcroft-Karp | O(E × √V) | BFS+DFS 결합, 더 빠름 | 정점/간선이 매우 많을 때 |
| 최대 유량 (Ford-Fulkerson) | O(V × E) | 이분 매칭을 유량 문제로 변환 | 가중치 매칭, 복잡한 제약 조건이 있을 때 |
| 가중치 이분 매칭 (Hungarian Method) | O(V³) | 비용이 최소/최대인 매칭을 구함 | 배정 문제에서 비용 최적화가 필요할 때 |
| 일반 그래프 매칭 (Edmonds' Blossom) | O(V³) | 이분 그래프가 아닌 일반 그래프에서 매칭 | 이분 그래프가 아닌 경우 (코테에서는 거의 출제 안 됨) |
