# 업무 배정 최적화 - 해설

## 접근 방식

1. 직원을 왼쪽 그룹, 프로젝트를 오른쪽 그룹으로 하는 이분 그래프를 구성한다.
2. 헝가리안 알고리즘으로 최대 매칭을 구한다.
3. 매칭 결과에서 각 프로젝트에 배정된 직원 번호를 추출하여 출력한다.
4. 매칭되지 않은 프로젝트는 0으로 표시한다.

## 핵심 아이디어

기본 이분 매칭에서 한 단계 더 나아가, 매칭 수뿐만 아니라 실제 매칭 결과를 출력해야 한다. 헝가리안 알고리즘에서 `matchR[v]` 배열은 오른쪽 정점(프로젝트) v에 매칭된 왼쪽 정점(직원)을 저장하므로, 알고리즘 종료 후 이 배열을 순회하면 배정 결과를 바로 얻을 수 있다.

주의할 점은 매칭 결과가 유일하지 않을 수 있다는 것이다. 같은 최대 매칭 수를 가지는 여러 배정 방법이 존재할 수 있으며, 알고리즘은 그 중 하나를 출력한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × E) |
| 공간 복잡도 | O(N + M + E) |

N명의 직원 각각에 대해 DFS를 수행하며, 각 DFS는 최대 O(E)의 간선을 탐색한다. 매칭 결과 출력은 O(M)이다.

## 대안적 접근

Hopcroft-Karp 알고리즘을 사용하면 O(E × √V)로 더 빠르게 풀 수 있다. BFS로 레벨 그래프를 구성한 후 DFS로 여러 증가 경로를 동시에 찾는 방식이다. N, M이 매우 큰 경우에 유리하지만, 이 문제의 제약 조건(N, M ≤ 1,000)에서는 헝가리안 알고리즘으로 충분하다.
