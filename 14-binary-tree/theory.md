# 이진 트리

## 개념

이진 트리(Binary Tree)는 각 노드가 최대 두 개의 자식 노드를 가지는 트리 자료구조이다. 두 자식을 각각 왼쪽 자식(Left Child)과 오른쪽 자식(Right Child)이라 부른다.

이진 트리의 주요 종류는 다음과 같다:
- **정 이진 트리(Full Binary Tree)**: 모든 노드가 0개 또는 2개의 자식을 가지는 트리
- **완전 이진 트리(Complete Binary Tree)**: 마지막 레벨을 제외한 모든 레벨이 완전히 채워져 있고, 마지막 레벨은 왼쪽부터 채워진 트리
- **포화 이진 트리(Perfect Binary Tree)**: 모든 내부 노드가 2개의 자식을 가지고, 모든 리프 노드가 같은 깊이에 있는 트리
- **이진 탐색 트리(BST, Binary Search Tree)**: 왼쪽 서브트리의 모든 값 < 현재 노드 값 < 오른쪽 서브트리의 모든 값을 만족하는 트리

이진 트리는 수식 파싱, 허프만 코딩, 이진 탐색 트리, 힙 등 다양한 분야에서 활용되며, 코딩테스트에서 순회, 복원, 경로 탐색 등의 문제로 자주 출제된다.

## 동작 원리

이진 트리 문제의 핵심은 재귀적 구조를 활용한 순회와 서브트리 분할이다.

### 이진 트리 표현

1. **배열 표현**: 완전 이진 트리의 경우 배열로 표현한다. 인덱스 i의 왼쪽 자식은 2*i, 오른쪽 자식은 2*i+1이다.
2. **노드 구조체 표현**: 각 노드가 값, 왼쪽 자식 포인터, 오른쪽 자식 포인터를 가진다.
3. **입력 기반 표현**: 각 노드의 왼쪽/오른쪽 자식 번호가 주어지는 형태로 표현한다.

### 이진 트리 순회

1. **전위 순회(Preorder)**: 현재 노드 → 왼쪽 서브트리 → 오른쪽 서브트리. 트리를 복사하거나 직렬화할 때 사용한다.
2. **중위 순회(Inorder)**: 왼쪽 서브트리 → 현재 노드 → 오른쪽 서브트리. BST에서 정렬된 순서를 얻을 때 사용한다.
3. **후위 순회(Postorder)**: 왼쪽 서브트리 → 오른쪽 서브트리 → 현재 노드. 서브트리의 결과를 먼저 계산한 뒤 합칠 때 사용한다.

### 이진 트리 복원

1. 전위 순회 + 중위 순회 결과가 주어지면 원래 트리를 유일하게 복원할 수 있다.
2. 전위 순회의 첫 번째 원소가 루트이다.
3. 중위 순회에서 루트의 위치를 찾으면, 그 왼쪽이 왼쪽 서브트리, 오른쪽이 오른쪽 서브트리이다.
4. 서브트리의 크기를 이용하여 전위 순회 결과도 분할한 뒤 재귀적으로 복원한다.

### 서브트리 기반 재귀

1. 이진 트리의 많은 문제는 "왼쪽 서브트리의 결과"와 "오른쪽 서브트리의 결과"를 합쳐서 현재 노드의 답을 구하는 패턴이다.
2. 리프 노드(자식이 없는 노드)가 기저 조건이 된다.
3. 후위 순회 방식으로 자식을 먼저 처리하고 부모를 처리하는 것이 일반적이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(N) |
| 시간 복잡도 (평균) | O(N) |
| 시간 복잡도 (최악) | O(N) |
| 공간 복잡도 | O(N) |

이진 트리 순회는 모든 노드를 한 번씩 방문하므로 O(N)이다. 노드 저장에 O(N) 공간이 필요하며, 재귀 호출 스택은 트리의 높이에 비례하여 균형 트리에서 O(log N), 편향 트리에서 O(N)이다.

## 적합한 문제 유형

- 이진 트리의 전위/중위/후위 순회 결과 출력
- 전위 + 중위 순회 결과로 트리 복원
- 이진 트리의 높이, 노드 수, 리프 수 계산
- 이진 트리에서의 경로 합, 최대 경로 합
- 이진 탐색 트리(BST)의 삽입, 탐색, 삭제
- 이진 트리의 대칭 여부, 동일 여부 판별
- 이진 트리의 직렬화/역직렬화


## 단계별 추적 (Trace)

### 예시 입력

전위 순회와 중위 순회 결과로 이진 트리를 복원하는 과정을 추적한다.

```
전위 순회: [1, 2, 4, 5, 3, 6, 7]
중위 순회: [4, 2, 5, 1, 6, 3, 7]
```

### 실행 과정

**단계 1: 루트 결정**

전위 순회의 첫 번째 원소 = 1 → 루트

중위 순회에서 1의 위치 = 인덱스 3

```
중위: [4, 2, 5] | 1 | [6, 3, 7]
       왼쪽 서브트리   루트   오른쪽 서브트리
```

**단계 2: 왼쪽 서브트리 복원**

전위: [2, 4, 5], 중위: [4, 2, 5]

루트 = 2, 중위에서 2의 위치 = 인덱스 1

```
중위: [4] | 2 | [5]
       좌    루트   우
```

- 왼쪽 자식 = 4 (리프)
- 오른쪽 자식 = 5 (리프)

**단계 3: 오른쪽 서브트리 복원**

전위: [3, 6, 7], 중위: [6, 3, 7]

루트 = 3, 중위에서 3의 위치 = 인덱스 1

```
중위: [6] | 3 | [7]
       좌    루트   우
```

- 왼쪽 자식 = 6 (리프)
- 오른쪽 자식 = 7 (리프)

**최종 복원된 트리**

```
        1
       / \
      2   3
     / \ / \
    4  5 6  7
```

**순회 결과 확인**

```
전위 (루트→좌→우): 1 → 2 → 4 → 5 → 3 → 6 → 7  ✓
중위 (좌→루트→우): 4 → 2 → 5 → 1 → 6 → 3 → 7  ✓
후위 (좌→우→루트): 4 → 5 → 2 → 6 → 7 → 3 → 1
```

## 실전 팁

### 활용 노하우

- 이진 트리 문제는 대부분 재귀로 풀린다. "현재 노드에서 무엇을 하고, 왼쪽/오른쪽 자식에게 무엇을 위임할지"를 먼저 생각하면 풀이가 명확해진다
- 트리 복원 문제에서 중위 순회의 루트 위치를 빠르게 찾기 위해 해시맵을 사용하면 O(N)에 복원할 수 있다
- BST의 중위 순회는 정렬된 결과를 주므로, BST 검증 문제에서 중위 순회 후 정렬 여부를 확인하는 방법이 유용하다
- 트리의 높이, 지름, 경로 합 등은 후위 순회 패턴(자식 먼저 처리 후 부모에서 합산)으로 해결한다

### 자주 하는 실수

- 재귀 기저 조건에서 `nil` 노드 처리를 빠뜨려 런타임 에러가 발생하는 경우가 많다. 항상 `node == nil` 체크를 먼저 한다
- 트리 복원 시 전위/중위 배열의 인덱스 범위를 잘못 계산하여 off-by-one 에러가 발생한다. 서브트리 크기를 정확히 계산해야 한다
- BST에서 "왼쪽 자식 < 부모 < 오른쪽 자식"만 확인하면 안 된다. 왼쪽 서브트리의 모든 값이 부모보다 작아야 하므로 범위를 전달하며 검증해야 한다

### 엣지 케이스 주의사항

- 노드가 1개인 트리: 루트만 존재하며 자식이 없는 경우
- 편향 트리(Skewed Tree): 모든 노드가 한쪽 자식만 가지는 경우, 높이가 N이 되어 재귀 깊이가 깊어진다
- 빈 트리(nil): 입력이 비어있는 경우를 처리해야 한다
- 동일한 값이 여러 개 있는 경우: BST에서 중복 값 처리 정책(왼쪽 또는 오른쪽)을 명확히 해야 한다

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 이진 트리 순회 | 전위/중위/후위 세 가지 순회 방식, 재귀 기반 | 트리 구조 탐색, 직렬화, 복원 |
| 이진 탐색 트리 (BST) | 왼쪽 < 루트 < 오른쪽 성질, 탐색 O(log N) | 정렬된 데이터 관리, 범위 검색 |
| 힙 (완전 이진 트리) | 부모 ≥ 자식(최대 힙), 배열로 표현 | 우선순위 큐, 최댓값/최솟값 빠른 접근 |
| 일반 트리 | 자식 수 제한 없음, 인접 리스트 표현 | 계층 구조, 파일 시스템, 조직도 |
| 세그먼트 트리 | 구간 쿼리 특화, 완전 이진 트리 구조 | 구간 합, 구간 최솟값 등 쿼리 |
| 트라이 (Trie) | 문자열 저장 특화 트리 | 문자열 검색, 자동 완성, 사전 |
