# 이진 트리 복원 - 해설

## 접근 방식

1. 전위 순회의 첫 번째 원소가 현재 서브트리의 루트임을 이용한다
2. 중위 순회에서 루트의 위치를 찾아 왼쪽/오른쪽 서브트리를 분할한다
3. 왼쪽 서브트리의 크기를 이용하여 전위 순회 결과도 분할한다
4. 재귀적으로 왼쪽 서브트리, 오른쪽 서브트리를 복원한다
5. 후위 순회 순서(왼쪽 → 오른쪽 → 현재)로 결과를 기록한다

## 핵심 아이디어

이진 트리 복원의 핵심은 전위 순회와 중위 순회의 성질을 결합하는 것이다.

전위 순회에서 첫 번째 원소는 항상 루트이다. 이 루트를 중위 순회에서 찾으면, 루트 왼쪽에 있는 원소들이 왼쪽 서브트리, 오른쪽에 있는 원소들이 오른쪽 서브트리를 구성한다.

중위 순회에서 루트의 인덱스를 해시맵에 미리 저장하면 O(1)에 탐색할 수 있어, 전체 시간 복잡도를 O(N)으로 유지할 수 있다.

실제로 트리 구조체를 만들지 않고, 재귀 호출 순서를 후위 순회 순서로 배치하면 복원과 동시에 후위 순회 결과를 얻을 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 노드를 정확히 한 번씩 처리하므로 O(N)이다. 해시맵과 결과 배열에 O(N) 공간이 필요하며, 재귀 호출 스택은 트리의 높이에 비례한다.

## 대안적 접근

- 해시맵 대신 매번 중위 순회 배열을 선형 탐색하면 O(N²)이 된다. N이 작을 때는 이 방법으로도 충분하다.
- 후위 순회 + 중위 순회 결과가 주어지는 경우에도 유사한 방법으로 복원할 수 있다. 후위 순회의 마지막 원소가 루트라는 점만 다르다.
- 전위 순회 + 후위 순회만으로는 이진 트리를 유일하게 복원할 수 없다. 한쪽 자식만 있는 노드에서 왼쪽/오른쪽을 구분할 수 없기 때문이다.
