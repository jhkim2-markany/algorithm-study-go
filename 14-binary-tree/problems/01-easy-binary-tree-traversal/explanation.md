# 이진 트리 순회 - 해설

## 접근 방식

1. 각 노드의 왼쪽 자식과 오른쪽 자식을 배열에 저장한다
2. 루트(1번 노드)에서 시작하여 세 가지 순회를 각각 수행한다
3. 전위 순회: 현재 노드를 먼저 방문하고, 왼쪽 → 오른쪽 순서로 재귀 호출한다
4. 중위 순회: 왼쪽 서브트리를 먼저 방문하고, 현재 노드를 방문한 뒤, 오른쪽 서브트리를 방문한다
5. 후위 순회: 왼쪽 → 오른쪽 서브트리를 먼저 방문하고, 마지막에 현재 노드를 방문한다

## 핵심 아이디어

이진 트리 순회의 핵심은 "현재 노드를 언제 처리하느냐"에 따라 순회 종류가 결정된다는 것이다. 세 순회 모두 왼쪽 → 오른쪽 순서로 서브트리를 방문하지만, 현재 노드의 처리 시점만 다르다.

이진 트리는 왼쪽/오른쪽 자식이 명확히 구분되므로, 일반 트리와 달리 인접 리스트 대신 left[], right[] 배열로 간결하게 표현할 수 있다. 자식이 없는 경우 -1로 표시하여 기저 조건으로 활용한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 순회는 모든 노드를 정확히 한 번씩 방문하므로 O(N)이다. 자식 배열과 결과 배열 저장에 O(N) 공간이 필요하며, 재귀 호출 스택은 트리의 높이에 비례한다.

## 대안적 접근

- 재귀 대신 스택을 사용한 반복적 순회가 가능하다. 편향 트리에서 재귀 깊이가 깊어지는 문제를 방지할 수 있다.
- 모리스 순회(Morris Traversal)를 사용하면 O(1) 추가 공간으로 중위 순회를 수행할 수 있다. 스레드 이진 트리 개념을 활용한다.
