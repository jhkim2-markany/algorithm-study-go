# 힙과 우선순위 큐

## 개념

힙(Heap)은 완전 이진 트리(Complete Binary Tree) 형태의 자료구조로, 부모 노드와 자식 노드 사이에 일정한 대소 관계가 성립한다. 최소 힙(Min Heap)에서는 부모가 항상 자식보다 작거나 같고, 최대 힙(Max Heap)에서는 부모가 항상 자식보다 크거나 같다.

우선순위 큐(Priority Queue)는 각 원소에 우선순위가 부여되어, 우선순위가 가장 높은 원소를 먼저 꺼내는 추상 자료형이다. 힙은 우선순위 큐를 효율적으로 구현하는 대표적인 자료구조이다.

Go 표준 라이브러리의 `container/heap` 패키지는 `heap.Interface`를 구현하면 최소 힙을 제공한다. 이 인터페이스는 `sort.Interface`(Len, Less, Swap)에 `Push`와 `Pop` 메서드를 추가한 것이다.

### 힙의 종류

- **최소 힙(Min Heap)**: 루트에 최솟값이 위치한다. `Less(i, j)`에서 `a[i] < a[j]`를 반환하면 된다.
- **최대 힙(Max Heap)**: 루트에 최댓값이 위치한다. `Less(i, j)`에서 `a[i] > a[j]`를 반환하면 된다.

### container/heap 인터페이스

```go
type Interface interface {
    sort.Interface          // Len, Less, Swap
    Push(x interface{})     // 원소 추가
    Pop() interface{}       // 원소 제거 (마지막 원소 반환)
}
```

`heap.Push`, `heap.Pop` 함수를 통해 힙 속성을 유지하면서 원소를 삽입·삭제한다. 직접 `Push`/`Pop` 메서드를 호출하면 힙 속성이 깨지므로 반드시 `heap` 패키지의 함수를 사용해야 한다.

## 동작 원리

힙은 배열로 구현하며, 인덱스 `i`의 노드에 대해 다음 관계가 성립한다:

- 부모 노드: `(i - 1) / 2`
- 왼쪽 자식: `2*i + 1`
- 오른쪽 자식: `2*i + 2`

### 삽입 (Push)

1. 배열의 맨 끝에 새 원소를 추가한다
2. 추가된 원소를 부모와 비교하며 힙 속성이 만족될 때까지 위로 올린다 (sift up)

### 삭제 (Pop)

1. 루트 원소(최솟값 또는 최댓값)를 꺼낸다
2. 배열의 마지막 원소를 루트로 이동시킨다
3. 루트에서 자식과 비교하며 힙 속성이 만족될 때까지 아래로 내린다 (sift down)

### 힙 구성 (Heapify)

기존 배열을 힙으로 변환할 때는 마지막 비리프 노드부터 루트까지 역순으로 sift down을 수행한다. 이 과정의 시간 복잡도는 O(N)이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 삽입 (Push) | O(log N) |
| 삭제 (Pop) | O(log N) |
| 최솟값/최댓값 조회 | O(1) |
| 힙 구성 (Heapify) | O(N) |
| 공간 복잡도 | O(N) |

N개의 원소를 하나씩 삽입하면 O(N log N)이지만, 배열을 한 번에 힙으로 변환하면 O(N)에 가능하다.

## 적합한 문제 유형

- K번째 최솟값/최댓값 구하기
- 정렬된 여러 리스트를 하나로 합치기
- 데이터 스트림에서 중앙값 유지하기
- 다익스트라 최단 경로 알고리즘의 우선순위 큐
- 허프만 코딩 등 빈도 기반 최적화
- 작업 스케줄링에서 우선순위 기반 처리
- 상위/하위 K개 원소 관리 (Top-K 문제)


## 단계별 추적 (Trace)

### 예시: 최소 힙에 원소 삽입 및 삭제

초기 빈 최소 힙에 `[5, 3, 8, 1, 2]`를 순서대로 삽입한 뒤, 2번 Pop한다.

### 실행 과정

```
=== 삽입 과정 ===

Push(5):
    [5]
     5

Push(3): 3 < 5 → sift up
    [3, 5]
     3
    /
   5

Push(8): 8 > 3 → 그대로
    [3, 5, 8]
       3
      / \
     5   8

Push(1): 1 < 5 → sift up → 1 < 3 → sift up
    [1, 3, 8, 5]
       1
      / \
     3   8
    /
   5

Push(2): 2 < 3 → sift up → 2 > 1 → 정지
    [1, 2, 8, 5, 3]
         1
        / \
       2   8
      / \
     5   3

=== 삭제 과정 ===

Pop(): 루트(1) 제거, 마지막 원소(3)를 루트로 이동, sift down
    [3, 2, 8, 5]  → 3 > 2 → swap
    [2, 3, 8, 5]
       2
      / \
     3   8
    /
   5
    반환값: 1

Pop(): 루트(2) 제거, 마지막 원소(5)를 루트로 이동, sift down
    [5, 3, 8]  → 5 > 3 → swap
    [3, 5, 8]
       3
      / \
     5   8
    반환값: 2

최종 힙 상태: [3, 5, 8]
```

### 배열 인덱스와 트리 구조 대응

```
배열: [1, 2, 8, 5, 3]
인덱스: 0  1  2  3  4

인덱스 관계:
  부모(i) = (i-1)/2
  왼쪽자식(i) = 2*i+1
  오른쪽자식(i) = 2*i+2

       [0]=1
      /     \
   [1]=2   [2]=8
   /   \
[3]=5 [4]=3
```

## 실전 팁

### 활용 노하우

- Go에서 `container/heap`을 사용할 때, `heap.Push()`와 `heap.Pop()`을 반드시 사용하라. 슬라이스의 `append`/직접 삭제는 힙 속성을 깨뜨린다
- "K번째 큰/작은 원소"가 나오면 크기 K의 힙을 유지하는 패턴을 사용한다
- 다익스트라에서 우선순위 큐를 사용할 때, 이미 처리된 노드를 건너뛰는 lazy deletion 패턴을 익혀두자
- 두 개의 힙(최대 힙 + 최소 힙)으로 중앙값을 실시간 유지하는 패턴은 자주 출제된다

### 자주 하는 실수

- **Less 함수 방향 혼동**: 최소 힙은 `a[i] < a[j]`, 최대 힙은 `a[i] > a[j]`이다. 방향을 반대로 하면 최대/최소가 뒤바뀐다
- **Pop 반환값 타입 캐스팅 누락**: `heap.Pop()`은 `interface{}`를 반환하므로 타입 단언이 필요하다
- **heap.Init() 호출 누락**: 기존 슬라이스를 힙으로 사용하려면 반드시 `heap.Init()`을 먼저 호출해야 한다
- **인덱스 0 기반 vs 1 기반 혼동**: Go의 `container/heap`은 0 기반 인덱스를 사용한다

### 엣지 케이스 주의사항

- 힙이 비어 있을 때 Pop 시도
- 모든 원소가 동일한 값인 경우
- 원소가 1개뿐인 힙에서의 연산
- 음수 값이 포함된 경우 (최소 힙에서 음수가 루트)
- 구조체를 힙 원소로 사용할 때 비교 기준 필드 설정

## 관련 알고리즘 비교

| 자료구조/알고리즘 | 삽입 | 삭제(최솟값) | 최솟값 조회 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 최소 힙 | O(log N) | O(log N) | O(1) | 동적으로 최솟값 반복 추출 |
| 정렬 배열 | O(N) | O(1) | O(1) | 삽입이 드물고 조회가 빈번 |
| 비정렬 배열 | O(1) | O(N) | O(N) | 삽입이 빈번하고 삭제가 드문 경우 |
| BST (균형) | O(log N) | O(log N) | O(log N) | 임의 원소 검색도 필요 |
| 정렬 | O(N log N) | - | - | 한 번에 전체 정렬 후 순서대로 처리 |

**선택 기준:**
- 원소가 동적으로 추가/삭제되며 최솟값/최댓값을 반복 추출 → 힙
- 전체 데이터를 한 번 정렬 후 순서대로 처리 → 정렬
- 최솟값/최댓값뿐 아니라 임의 원소 검색도 필요 → 균형 BST
- Top-K 문제 → 크기 K의 힙 유지
- 다익스트라 최단 경로 → 최소 힙 기반 우선순위 큐
