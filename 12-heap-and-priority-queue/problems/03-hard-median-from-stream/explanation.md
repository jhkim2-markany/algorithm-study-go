# 데이터 스트림의 중앙값 - 해설

## 접근 방식

1. 두 개의 힙을 사용한다: 작은 쪽 절반을 저장하는 최대 힙과 큰 쪽 절반을 저장하는 최소 힙
2. 새 값이 들어오면 최대 힙의 루트와 비교하여 적절한 힙에 삽입한다
3. 두 힙의 크기 차이가 1 이하가 되도록 균형을 맞춘다
4. 최대 힙의 루트가 항상 중앙값이 되도록 유지한다

## 핵심 아이디어

데이터를 정렬된 상태로 유지하면서 중앙값을 빠르게 구하기 위해 두 개의 힙을 사용한다.

- **최대 힙(왼쪽)**: 중앙값 이하의 값들을 저장한다. 루트에 이 그룹의 최댓값이 위치한다.
- **최소 힙(오른쪽)**: 중앙값 초과의 값들을 저장한다. 루트에 이 그룹의 최솟값이 위치한다.

두 힙의 크기를 `maxH.Len() >= minH.Len()`이 되도록 유지하면, 중앙값은 항상 최대 힙의 루트이다.

- 전체 개수가 홀수: 최대 힙이 1개 더 많으므로 최대 힙의 루트가 정확한 중앙값
- 전체 개수가 짝수: 두 힙의 크기가 같으므로 가운데 두 값 중 작은 값(최대 힙의 루트)이 중앙값

이 방식은 매 삽입마다 O(log N)에 중앙값을 갱신할 수 있어, 정렬을 매번 수행하는 O(N log N)보다 훨씬 효율적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

N개의 원소를 삽입하며 각 삽입마다 힙 연산 O(log N)을 수행하므로 전체 시간 복잡도는 O(N log N)이다. 모든 원소를 힙에 저장하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **정렬된 배열 유지**: 이진 탐색으로 삽입 위치를 찾고 삽입한다. 삽입 시 O(N) 이동이 필요하여 전체 O(N²)이다.
- **균형 이진 탐색 트리(AVL, Red-Black)**: 중앙값을 O(log N)에 구할 수 있지만, 구현이 복잡하다.
- **세그먼트 트리/펜윅 트리**: 값의 범위가 제한적일 때 빈도 배열과 함께 사용하여 K번째 원소를 O(log V)에 구할 수 있다.
