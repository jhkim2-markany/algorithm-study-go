# 정렬된 리스트 합치기 - 해설

## 접근 방식

1. K개의 정렬된 리스트를 입력받는다
2. 각 리스트의 첫 번째 원소를 최소 힙에 삽입한다
3. 힙에서 최솟값을 꺼내 결과에 추가한다
4. 꺼낸 원소가 속한 리스트의 다음 원소를 힙에 삽입한다
5. 힙이 빌 때까지 3~4단계를 반복한다

## 핵심 아이디어

K개의 정렬된 리스트를 합칠 때, 각 리스트의 현재 위치에서 가장 작은 값을 효율적으로 찾아야 한다. 단순히 K개의 값을 매번 비교하면 O(K)가 걸리지만, 최소 힙을 사용하면 O(log K)에 최솟값을 찾을 수 있다.

힙에는 항상 각 리스트에서 아직 처리하지 않은 첫 번째 원소가 들어 있으므로, 힙의 크기는 최대 K이다. 원소를 꺼낸 후 해당 리스트의 다음 원소를 삽입하면 힙 크기가 유지된다.

이 방식은 K-way merge라고 불리며, 외부 정렬(External Sort)에서도 핵심적으로 사용되는 기법이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log K) |
| 공간 복잡도 | O(K) |

전체 원소 수를 N이라 하면, 각 원소마다 힙 삽입/삭제(O(log K))를 수행하므로 시간 복잡도는 O(N log K)이다. 힙의 크기가 최대 K이므로 공간 복잡도는 O(K)이다.

## 대안적 접근

- **전체 합치기 후 정렬**: 모든 원소를 하나의 배열에 넣고 정렬한다. 시간 복잡도 O(N log N)으로, K가 클 때 힙 방식보다 비효율적이다.
- **2-way merge 반복**: 리스트를 두 개씩 합치는 과정을 반복한다. 시간 복잡도 O(N log K)로 동일하지만, 중간 결과를 저장할 추가 공간이 필요하다.
