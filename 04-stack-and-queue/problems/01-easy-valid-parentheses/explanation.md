# 괄호 검사 - 해설

## 접근 방식

1. 문자열을 왼쪽부터 한 문자씩 순회한다
2. 여는 괄호(`(`, `{`, `[`)를 만나면 스택에 Push한다
3. 닫는 괄호(`)`, `}`, `]`)를 만나면 스택의 맨 위 원소와 짝이 맞는지 확인한다
4. 짝이 맞으면 스택에서 Pop하고, 맞지 않으면 올바르지 않은 문자열이다
5. 문자열을 모두 순회한 후 스택이 비어 있으면 올바른 괄호 문자열이다

## 핵심 아이디어

괄호 검사는 스택의 가장 대표적인 활용 문제이다. "가장 나중에 열린 괄호가 가장 먼저 닫혀야 한다"는 조건이 스택의 LIFO 특성과 정확히 일치한다.

여는 괄호를 스택에 쌓아두고, 닫는 괄호가 나올 때마다 스택의 맨 위(가장 최근에 열린 괄호)와 비교하면 된다. 세 가지 실패 조건을 모두 검사해야 한다:
- 닫는 괄호가 나왔는데 스택이 비어 있는 경우 (대응하는 여는 괄호 없음)
- 닫는 괄호와 스택 맨 위의 여는 괄호 종류가 다른 경우
- 문자열을 모두 처리한 후 스택에 여는 괄호가 남아 있는 경우

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

문자열의 각 문자를 한 번씩 순회하므로 시간 복잡도는 O(N)이다. 최악의 경우 모든 문자가 여는 괄호일 때 스택에 N개가 쌓이므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **카운터 방식**: 괄호가 한 종류(`(`, `)`)만 있는 경우, 스택 대신 정수 카운터로 여는 괄호 수를 추적할 수 있다. 다만 여러 종류의 괄호가 섞인 경우에는 스택이 필수적이다.
- **문자열 치환**: 짝이 맞는 괄호 쌍(`()`, `{}`, `[]`)을 반복적으로 제거하여 빈 문자열이 되는지 확인하는 방법도 있다. 직관적이지만 시간 복잡도가 O(N²)으로 비효율적이다.
