# 주식 가격 - 해설

## 접근 방식

1. 각 날짜의 인덱스를 순서대로 스택에 Push한다
2. 새로운 날짜를 처리할 때, 스택 맨 위 날짜의 가격보다 현재 가격이 낮으면 가격이 떨어진 것이다
3. 스택에서 Pop하면서 해당 날짜의 답(가격이 떨어지지 않은 기간)을 계산한다
4. 모든 날짜를 처리한 후 스택에 남은 날짜들은 끝까지 가격이 떨어지지 않은 경우이다

## 핵심 아이디어

이 문제는 모노톤 스택(Monotone Stack)을 활용하는 대표적인 문제이다. 스택에 "아직 가격이 떨어지는 시점을 찾지 못한 날짜"의 인덱스를 저장한다.

새로운 날짜 i를 처리할 때, 스택 맨 위 날짜 j의 가격보다 i의 가격이 낮으면 j에서 처음으로 가격이 떨어진 날이 i이다. 따라서 j의 답은 `i - j`가 된다. 이 과정을 스택이 비거나 맨 위 가격이 현재 가격 이하가 될 때까지 반복한다.

모든 날짜를 순회한 후 스택에 남아 있는 날짜들은 끝까지 가격이 떨어지지 않은 경우이므로, 마지막 날짜까지의 거리를 답으로 설정한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 날짜의 인덱스는 스택에 최대 한 번 Push되고 한 번 Pop되므로, 전체 연산 횟수는 O(N)이다. 스택과 결과 배열에 각각 O(N) 공간이 필요하다.

## 대안적 접근

- **브루트포스**: 각 날짜에 대해 이후 날짜를 순회하며 가격이 떨어지는 시점을 찾는 O(N²) 방법이 있다. 구현은 간단하지만 N이 100,000일 때 시간 초과가 발생할 수 있다.
- **뒤에서부터 순회**: 배열을 뒤에서부터 순회하면서 스택을 활용하는 방법도 가능하다. 각 날짜에서 스택을 이용해 다음으로 가격이 떨어지는 날을 찾는다.
