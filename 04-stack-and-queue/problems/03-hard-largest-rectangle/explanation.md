# 가장 큰 직사각형 - 해설

## 접근 방식

1. 막대를 왼쪽부터 순서대로 순회한다
2. 스택에 막대의 인덱스를 저장하되, 높이가 오름차순을 유지하도록 관리한다
3. 현재 막대의 높이가 스택 맨 위 막대보다 낮으면, 스택에서 Pop하면서 직사각형 넓이를 계산한다
4. Pop된 막대의 높이를 직사각형의 높이로, 좌우 경계를 이용해 너비를 계산한다
5. 모든 막대를 순회한 후 스택에 남은 막대들도 동일하게 처리한다

## 핵심 아이디어

이 문제는 스택을 활용한 히스토그램 최대 직사각형 문제로, 모노톤 스택의 고전적인 응용이다.

핵심 관찰은 다음과 같다: 어떤 막대 h를 높이로 하는 최대 직사각형의 너비는, h보다 낮은 막대가 처음 나타나는 왼쪽 경계와 오른쪽 경계 사이의 거리이다.

스택에 높이가 오름차순인 막대의 인덱스를 유지한다. 현재 막대의 높이가 스택 맨 위보다 낮으면, 스택 맨 위 막대의 오른쪽 경계가 현재 위치임을 의미한다. Pop한 후 새로운 스택 맨 위가 왼쪽 경계가 된다.

마지막에 높이 0인 가상의 막대를 추가하면, 스택에 남은 모든 막대를 자연스럽게 처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 막대는 스택에 최대 한 번 Push되고 한 번 Pop되므로 전체 시간 복잡도는 O(N)이다. 스택에 최대 N개의 인덱스가 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **분할 정복**: 구간을 반으로 나누어 왼쪽, 오른쪽, 걸치는 경우를 각각 계산하는 O(N log N) 방법이 있다. 세그먼트 트리와 결합하면 효율적이지만 구현이 복잡하다.
- **브루트포스**: 모든 막대 쌍에 대해 최소 높이를 구하는 O(N²) 방법이 있다. N이 100,000일 때 시간 초과가 발생한다.
