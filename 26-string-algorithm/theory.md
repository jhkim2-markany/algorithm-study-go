# 문자열 알고리즘

## 개념

문자열 알고리즘(String Algorithm)은 문자열 내에서 특정 패턴을 검색하거나, 문자열의 구조적 특성을 분석하는 알고리즘 기법의 총칭이다. 텍스트 검색, 데이터 압축, 생물정보학 등 다양한 분야에서 핵심적으로 사용된다.

주요 문자열 알고리즘은 다음과 같다:

- **브루트포스 패턴 매칭**: 텍스트의 모든 위치에서 패턴을 하나씩 비교하는 가장 단순한 방법이다
- **KMP(Knuth-Morris-Pratt) 알고리즘**: 실패 함수(failure function)를 사전에 계산하여, 불일치 발생 시 불필요한 비교를 건너뛰는 효율적인 패턴 매칭 알고리즘이다
- **접미사 배열(Suffix Array)**: 문자열의 모든 접미사를 사전순으로 정렬한 배열로, LCP(Longest Common Prefix) 배열과 함께 다양한 문자열 문제를 효율적으로 해결한다
- **라빈-카프(Rabin-Karp) 알고리즘**: 해시 함수를 이용하여 패턴과 텍스트의 부분 문자열을 비교하는 방법으로, 다중 패턴 검색에 유리하다

## 동작 원리

### 브루트포스 패턴 매칭

1. 텍스트의 첫 번째 위치부터 시작한다.
2. 현재 위치에서 패턴의 첫 문자부터 하나씩 비교한다.
3. 모든 문자가 일치하면 해당 위치를 결과에 기록한다.
4. 불일치가 발생하면 텍스트의 다음 위치로 이동하여 다시 비교한다.
5. 텍스트 끝까지 반복한다.

### KMP 알고리즘

KMP 알고리즘은 두 단계로 동작한다:

**1단계: 실패 함수(failure function) 계산**

1. 패턴 자체에서 접두사와 접미사가 일치하는 최대 길이를 각 위치마다 계산한다.
2. 패턴의 첫 번째 위치의 실패 함수 값은 0이다.
3. 현재 위치의 문자와 이전 일치 길이 다음 문자를 비교한다.
4. 일치하면 실패 함수 값을 1 증가시킨다.
5. 불일치하면 이전 실패 함수 값을 따라가며 다시 비교한다.

**2단계: 패턴 검색**

1. 텍스트와 패턴을 동시에 순회한다.
2. 현재 문자가 일치하면 두 포인터를 모두 전진시킨다.
3. 패턴의 끝까지 일치하면 매칭 위치를 기록하고, 실패 함수를 이용해 다음 비교 위치로 이동한다.
4. 불일치가 발생하면 실패 함수를 참조하여 패턴의 비교 위치를 조정한다. 텍스트 포인터는 되돌아가지 않는다.

### 접미사 배열

1. 문자열의 모든 접미사를 생성한다 (인덱스 0부터 시작하는 접미사, 인덱스 1부터 시작하는 접미사, ...).
2. 모든 접미사를 사전순으로 정렬한다.
3. 정렬된 접미사의 시작 인덱스를 배열에 저장한다. 이것이 접미사 배열이다.
4. 인접한 접미사 간의 최장 공통 접두사(LCP) 길이를 계산하여 LCP 배열을 구성한다.
5. 접미사 배열과 LCP 배열을 활용하여 패턴 검색, 최장 반복 부분 문자열 등의 문제를 해결한다.

### 라빈-카프 알고리즘 (참고)

1. 패턴의 해시 값을 계산한다.
2. 텍스트에서 패턴 길이만큼의 윈도우를 슬라이딩하며 해시 값을 계산한다.
3. 해시 값이 일치하면 실제 문자열을 비교하여 확인한다.
4. 롤링 해시를 사용하여 윈도우 이동 시 O(1)에 해시를 갱신한다.

## 복잡도

| 구분 | 브루트포스 | KMP | 접미사 배열 | 라빈-카프 |
| --- | --- | --- | --- | --- |
| 시간 복잡도 (최선) | O(N) | O(N + M) | O(N log N) | O(N + M) |
| 시간 복잡도 (평균) | O(N × M) | O(N + M) | O(N log N) | O(N + M) |
| 시간 복잡도 (최악) | O(N × M) | O(N + M) | O(N log² N) | O(N × M) |
| 공간 복잡도 | O(1) | O(M) | O(N) | O(1) |

- N: 텍스트 길이, M: 패턴 길이
- KMP는 실패 함수 전처리에 O(M), 검색에 O(N)이 소요되어 총 O(N + M)이다
- 접미사 배열은 정렬 방식에 따라 O(N log N) ~ O(N log² N)이며, SA-IS 알고리즘을 사용하면 O(N)에 구성 가능하다
- 라빈-카프는 해시 충돌이 없으면 O(N + M)이지만, 최악의 경우 O(N × M)이다

## 적합한 문제 유형

- 문자열 내 패턴 검색 (단일/다중 패턴)
- 최장 반복 부분 문자열 찾기
- 문자열 주기 판별
- 접두사-접미사 일치 문제
- 문자열 정렬 및 사전순 비교
- 최장 공통 부분 문자열 (LCS with suffix array)
- 회문(팰린드롬) 판별
- 문자열 압축 및 인코딩
- DNA 서열 분석 등 생물정보학 문제

## 단계별 추적 (Trace)

### 예시 입력 - KMP 알고리즘

텍스트: "ABABDABACDABABCABAB", 패턴: "ABABCABAB"

### 1단계: 실패 함수 계산

```
패턴: A B A B C A B A B
인덱스: 0 1 2 3 4 5 6 7 8

i=1: B vs A → 불일치, fail[1] = 0
i=2: A vs A → 일치, fail[2] = 1
i=3: B vs B → 일치, fail[3] = 2
i=4: C vs A → 불일치, j=fail[0]=0, C vs A → 불일치, fail[4] = 0
i=5: A vs A → 일치, fail[5] = 1
i=6: B vs B → 일치, fail[6] = 2
i=7: A vs A → 일치, fail[7] = 3
i=8: B vs B → 일치, fail[8] = 4

실패 함수: [0, 0, 1, 2, 0, 1, 2, 3, 4]
```

### 2단계: 패턴 검색

```
텍스트: A B A B D A B A C D A B A B C A B A B
패턴:   A B A B C
        i=0,1,2,3 일치, i=4 불일치 (D vs C)
        j = fail[3] = 2 → 패턴을 2칸 이동

텍스트: A B A B D A B A C D A B A B C A B A B
패턴:       A B A B C
            i=2,3 일치 (이전 매칭 재활용), i=4 불일치 (D vs C)
            j = fail[1] = 0

... (중간 과정 생략)

텍스트: A B A B D A B A C D A B A B C A B A B
패턴:                       A B A B C A B A B
                            i=0~8 모두 일치!

매칭 위치: 인덱스 10에서 패턴 발견
```

## 실전 팁

### 활용 노하우

- 단일 패턴 검색에는 KMP가 가장 안정적이다 (최악에도 O(N+M) 보장)
- 다중 패턴 검색에는 Aho-Corasick(트라이 + KMP 확장)이 효율적이다
- 문자열 주기 판별은 KMP 실패 함수로 해결 가능: 주기 = `len - fail[len-1]`
- 접미사 배열은 구현이 복잡하지만, 최장 반복 부분 문자열 등 고급 문제에 필수이다
- 라빈-카프의 롤링 해시는 문자열 비교를 O(1)로 줄여주지만, 해시 충돌에 주의해야 한다
- Go에서 문자열은 UTF-8 바이트 배열이므로, 아스키 문자만 다루는 경우 `[]byte`로 변환하면 빠르다

### 자주 하는 실수

- KMP 실패 함수 계산에서 인덱스 시작점을 잘못 설정하는 경우 (0-indexed vs 1-indexed)
- 라빈-카프에서 해시 충돌 시 실제 문자열 비교를 하지 않아 오답 발생
- 접미사 배열 정렬에서 비교 함수를 잘못 구현하여 TLE 또는 오답 발생
- 패턴 길이가 텍스트 길이보다 긴 경우를 처리하지 않는 경우
- 빈 문자열이나 길이 1 문자열에 대한 예외 처리 누락

### 엣지 케이스

- 패턴이 텍스트와 완전히 동일한 경우
- 패턴이 텍스트보다 긴 경우 (매칭 불가)
- 텍스트나 패턴이 빈 문자열인 경우
- 모든 문자가 동일한 경우 (예: "AAAA"에서 "AA" 검색 → 겹치는 매칭)
- 패턴이 텍스트의 맨 처음 또는 맨 끝에만 존재하는 경우

## 관련 알고리즘 비교

| 알고리즘 | 전처리 | 검색 시간 | 공간 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 브루트포스 | 없음 | O(N×M) | O(1) | 짧은 문자열, 간단한 구현 |
| KMP | O(M) | O(N) | O(M) | 단일 패턴 검색, 안정적 성능 |
| 라빈-카프 | O(M) | O(N) 평균 | O(1) | 다중 패턴 검색, 해시 기반 |
| 접미사 배열 | O(N log N) | O(M log N) | O(N) | 다양한 문자열 구조 분석 |
| Aho-Corasick | O(Σ패턴) | O(N + 매칭수) | O(Σ패턴) | 다중 패턴 동시 검색 |
| Z 알고리즘 | O(N) | O(N) | O(N) | KMP 대안, Z-배열 활용 |
