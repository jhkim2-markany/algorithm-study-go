# 접미사 배열 - 해설

## 접근 방식

1. 문자열의 모든 접미사를 생성하고, 이를 사전순으로 정렬하여 접미사 배열(SA)을 구성한다.
2. 효율적인 구성을 위해 접미사 배열 정렬 시 O(N log² N) 알고리즘을 사용한다. 각 단계에서 2^k 길이의 접두사를 기준으로 정렬하며, 이전 단계의 순위를 활용하여 비교한다.
3. 접미사 배열이 구성되면, Kasai 알고리즘을 사용하여 O(N)에 LCP 배열을 계산한다.
4. Kasai 알고리즘은 SA의 역배열(rank)을 이용하여, 이전 LCP 값에서 1을 빼고 시작하는 최적화를 적용한다.

## 핵심 아이디어

접미사 배열 구성의 핵심은 "더블링(doubling)" 기법이다. 처음에는 각 접미사의 첫 1글자로 순위를 매기고, 다음에는 2글자, 4글자, ... 순으로 순위를 확장한다. 각 단계에서 이전 순위를 키로 사용하여 정렬하므로, 비교 연산이 O(1)에 수행된다.

LCP 배열 계산에는 Kasai의 알고리즘이 핵심이다. 이 알고리즘은 다음 관찰에 기반한다: SA에서 rank[i]의 LCP 값이 h이면, rank[i+1]의 LCP 값은 최소 h-1이다. 이 성질을 이용하면 전체 LCP 배열을 O(N)에 계산할 수 있다.

접미사 배열과 LCP 배열을 함께 사용하면 최장 반복 부분 문자열, 서로 다른 부분 문자열의 개수, 최장 공통 부분 문자열 등 다양한 문자열 문제를 효율적으로 해결할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log² N) |
| 공간 복잡도 | O(N) |

접미사 배열 구성에 O(N log² N) (더블링 log N 단계 × 정렬 N log N, 순위 비교 O(1)로 최적화 시 O(N log N)), LCP 배열 계산에 O(N)이 소요된다. 접미사 배열, 순위 배열, LCP 배열에 각각 O(N) 공간이 필요하다.

## 대안적 접근

- 단순 정렬: 모든 접미사를 직접 정렬하면 O(N² log N)이지만 구현이 간단하다.
- SA-IS 알고리즘: O(N) 시간에 접미사 배열을 구성할 수 있지만 구현이 복잡하다.
- DC3(Skew) 알고리즘: O(N) 시간의 또 다른 선형 시간 알고리즘이다.
- 접미사 트리: 접미사 배열과 동등한 정보를 제공하지만 메모리 사용량이 더 크다.
