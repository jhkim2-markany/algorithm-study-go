# 문자열 패턴 매칭 - 해설

## 접근 방식

1. 텍스트의 각 위치에서 패턴과 문자를 하나씩 비교하는 브루트포스 방식을 사용한다.
2. 텍스트의 인덱스 0부터 N-M까지 순회하며, 각 위치에서 패턴의 모든 문자가 일치하는지 확인한다.
3. 일치하는 위치를 1-based 인덱스로 변환하여 저장한다.
4. 불일치가 발생하면 즉시 다음 위치로 넘어간다.

## 핵심 아이디어

가장 직관적인 문자열 검색 방법인 브루트포스 패턴 매칭이다. 텍스트의 모든 가능한 시작 위치에서 패턴을 비교한다. 불일치가 발생하면 바로 중단하고 다음 위치로 이동하므로, 평균적으로는 O(N)에 가까운 성능을 보인다.

이 문제의 제약 조건(|T| ≤ 100,000)에서는 브루트포스로도 충분히 해결 가능하다. 다만 최악의 경우(예: 텍스트 "AAAAAA...", 패턴 "AAAB")에는 O(N × M)이 될 수 있으므로, 더 큰 입력에서는 KMP 등의 효율적인 알고리즘이 필요하다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(N) |

텍스트의 각 위치(N-M+1개)에서 최대 M번 비교하므로 최악 시간 복잡도는 O(N × M)이다. 매칭 위치를 저장하는 배열에 최대 O(N) 공간이 필요하다.

## 대안적 접근

- KMP 알고리즘을 사용하면 O(N + M)에 해결할 수 있다.
- 라빈-카프 알고리즘으로 해시 기반 비교를 하면 평균 O(N + M)에 해결 가능하다.
- 보이어-무어 알고리즘은 실제 텍스트 검색에서 가장 빠른 성능을 보이는 경우가 많다.
