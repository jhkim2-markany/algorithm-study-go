# KMP 문자열 검색 - 해설

## 접근 방식

1. 패턴의 실패 함수(failure function)를 먼저 계산한다. 실패 함수는 패턴의 각 위치에서 접두사와 접미사가 일치하는 최대 길이를 저장한다.
2. 텍스트를 한 번 순회하면서 패턴과 비교한다.
3. 불일치가 발생하면 실패 함수를 참조하여 패턴의 비교 위치를 조정한다. 이때 텍스트 포인터는 절대 되돌아가지 않는다.
4. 패턴 전체가 일치하면 시작 위치를 기록하고, 실패 함수를 이용해 다음 가능한 매칭 위치로 이동한다.

## 핵심 아이디어

KMP 알고리즘의 핵심은 실패 함수(부분 일치 테이블)에 있다. 브루트포스에서는 불일치가 발생하면 텍스트 포인터를 한 칸만 전진시키고 패턴을 처음부터 다시 비교하지만, KMP는 이미 일치한 부분의 정보를 활용한다.

예를 들어 패턴 "ABABC"에서 4번째 문자 'C'에서 불일치가 발생하면, 이미 일치한 "ABAB"의 접두사 "AB"와 접미사 "AB"가 같으므로 패턴의 3번째 위치부터 비교를 재개할 수 있다. 이렇게 하면 텍스트의 각 문자를 최대 한 번만 비교하게 되어 O(N + M) 시간에 검색이 완료된다.

실패 함수 계산 자체도 KMP와 동일한 원리를 사용한다. 패턴을 자기 자신과 매칭하는 것으로 볼 수 있으며, O(M) 시간에 계산된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(M) |

실패 함수 계산에 O(M), 텍스트 검색에 O(N)이 소요되어 총 O(N + M)이다. 실패 함수 배열에 O(M) 공간이 필요하다. 브루트포스의 O(N × M)에 비해 최악의 경우에도 선형 시간을 보장한다.

## 대안적 접근

- 브루트포스 매칭은 구현이 간단하지만 최악 O(N × M)이므로 큰 입력에서는 시간 초과가 발생할 수 있다.
- 라빈-카프 알고리즘은 해시 기반으로 평균 O(N + M)이지만, 해시 충돌 시 최악 O(N × M)이다.
- 아호-코라식(Aho-Corasick) 알고리즘은 다중 패턴 검색에 적합하며, KMP를 트라이 구조로 확장한 것이다.
