# 정렬

## 개념

정렬(Sorting)은 데이터를 특정 기준에 따라 오름차순 또는 내림차순으로 나열하는 알고리즘이다. 코딩테스트에서 가장 빈번하게 등장하는 기본 알고리즘이며, 탐색, 그리디, 투 포인터 등 다른 알고리즘의 전처리 단계로도 자주 활용된다.

정렬 알고리즘은 비교 기반 정렬과 비비교 기반 정렬로 나뉜다. 비교 기반 정렬은 원소 간 대소 비교를 통해 순서를 결정하며, 이론적 하한이 O(N log N)이다. 대표적으로 버블 정렬, 병합 정렬, 퀵 정렬이 있다.

## 동작 원리

### 버블 정렬 (Bubble Sort)

인접한 두 원소를 비교하여 순서가 잘못되어 있으면 교환하는 과정을 반복한다.

1. 배열의 처음부터 끝까지 인접한 두 원소를 비교한다
2. 앞의 원소가 뒤의 원소보다 크면 두 원소를 교환한다
3. 한 번의 순회가 끝나면 가장 큰 원소가 맨 뒤로 이동한다
4. 정렬이 완료될 때까지 위 과정을 반복한다

### 병합 정렬 (Merge Sort)

배열을 반으로 나누어 각각 정렬한 뒤, 두 정렬된 배열을 합치는 분할 정복 방식이다.

1. 배열을 절반으로 분할한다
2. 각 부분 배열을 재귀적으로 정렬한다
3. 두 정렬된 부분 배열을 하나로 병합한다
4. 병합 시 두 배열의 앞쪽 원소를 비교하여 작은 것부터 결과 배열에 넣는다

### 퀵 정렬 (Quick Sort)

피벗(pivot)을 기준으로 배열을 두 부분으로 분할하고, 각 부분을 재귀적으로 정렬한다.

1. 배열에서 피벗 원소를 선택한다
2. 피벗보다 작은 원소는 왼쪽, 큰 원소는 오른쪽으로 분할한다
3. 왼쪽과 오른쪽 부분 배열을 재귀적으로 정렬한다
4. 분할 과정에서 이미 정렬이 진행되므로 별도의 병합이 필요 없다

## 복잡도

| 구분 | 버블 정렬 | 병합 정렬 | 퀵 정렬 |
| --- | --- | --- | --- |
| 시간 복잡도 (최선) | O(N) | O(N log N) | O(N log N) |
| 시간 복잡도 (평균) | O(N²) | O(N log N) | O(N log N) |
| 시간 복잡도 (최악) | O(N²) | O(N log N) | O(N²) |
| 공간 복잡도 | O(1) | O(N) | O(log N) |
| 안정 정렬 | 예 | 예 | 아니오 |

버블 정렬은 구현이 단순하지만 O(N²)으로 느리다. 병합 정렬은 항상 O(N log N)을 보장하지만 추가 메모리가 필요하다. 퀵 정렬은 평균적으로 가장 빠르지만 최악의 경우 O(N²)이 될 수 있다. Go 표준 라이브러리의 `sort` 패키지는 내부적으로 인트로소트(Introsort) 변형을 사용한다.

## 적합한 문제 유형

- 데이터를 오름차순/내림차순으로 정렬하는 기본 문제
- 커스텀 비교 함수를 사용한 다중 조건 정렬 문제
- 정렬 후 이진 탐색, 투 포인터 등을 적용하는 전처리 문제
- 역전 쌍(Inversion) 개수를 세는 문제 (병합 정렬 활용)
- 좌표 압축, 상대 순서 등 정렬 기반 변환 문제
- K번째 원소 찾기 등 부분 정렬 문제


## 단계별 추적 (Trace)

### 예시: 버블 정렬

입력: `[5, 3, 8, 1, 2]`

```
=== 1회차 (i=0) ===
[5, 3, 8, 1, 2]  → 5>3 교환 → [3, 5, 8, 1, 2]
[3, 5, 8, 1, 2]  → 5<8 유지 → [3, 5, 8, 1, 2]
[3, 5, 8, 1, 2]  → 8>1 교환 → [3, 5, 1, 8, 2]
[3, 5, 1, 8, 2]  → 8>2 교환 → [3, 5, 1, 2, 8]  ← 8이 맨 뒤로

=== 2회차 (i=1) ===
[3, 5, 1, 2, 8]  → 3<5 유지 → [3, 5, 1, 2, 8]
[3, 5, 1, 2, 8]  → 5>1 교환 → [3, 1, 5, 2, 8]
[3, 1, 5, 2, 8]  → 5>2 교환 → [3, 1, 2, 5, 8]  ← 5가 제자리

=== 3회차 (i=2) ===
[3, 1, 2, 5, 8]  → 3>1 교환 → [1, 3, 2, 5, 8]
[1, 3, 2, 5, 8]  → 3>2 교환 → [1, 2, 3, 5, 8]  ← 3이 제자리

=== 4회차 (i=3) ===
[1, 2, 3, 5, 8]  → 1<2 유지 → [1, 2, 3, 5, 8]  ← 정렬 완료

최종 결과: [1, 2, 3, 5, 8]
```

### 예시: 병합 정렬

입력: `[5, 3, 8, 1, 2]`

```
분할 단계 (재귀적으로 반씩 나눔):
                [5, 3, 8, 1, 2]
               /                \
         [5, 3, 8]            [1, 2]
        /        \            /    \
     [5, 3]     [8]        [1]   [2]
     /    \
   [5]   [3]

병합 단계 (정렬하면서 합침):
   [5]   [3]  → 비교: 3<5 → [3, 5]
     [3, 5]  [8]  → 비교: 3<8, 5<8 → [3, 5, 8]
        [1]  [2]  → 비교: 1<2 → [1, 2]
   [3, 5, 8]  [1, 2]  → 병합:
     ↓ 비교: 1<3 → 1 선택
     ↓ 비교: 2<3 → 2 선택
     ↓ 비교: 3 선택, 5 선택, 8 선택
     → [1, 2, 3, 5, 8]

최종 결과: [1, 2, 3, 5, 8]
```

## 실전 팁

### 활용 노하우

- **Go 표준 라이브러리 활용**: 코딩테스트에서는 `sort.Slice()`나 `sort.Ints()`를 적극 활용하자. 직접 정렬을 구현할 필요가 거의 없다
- **커스텀 정렬**: `sort.Slice(arr, func(i, j int) bool { ... })`로 다중 조건 정렬을 간결하게 구현할 수 있다
- **정렬 후 이진 탐색**: 정렬 + `sort.SearchInts()`로 O(N log N + Q log N)에 다수의 탐색 쿼리를 처리할 수 있다
- **역전 쌍 세기**: 병합 정렬의 병합 과정에서 역전 쌍(inversion)의 개수를 O(N log N)에 셀 수 있다

### 자주 하는 실수

- **불안정 정렬 사용**: 동일한 값의 상대적 순서가 중요한 문제에서 퀵 정렬(불안정)을 사용하면 오답이 된다. Go의 `sort.Slice()`는 불안정 정렬이므로, 안정 정렬이 필요하면 `sort.SliceStable()`을 사용하자
- **비교 함수의 일관성**: 커스텀 비교 함수에서 `a < b`와 `a > b`를 혼용하면 정렬 결과가 예측 불가능해진다. 하나의 기준으로 통일하자
- **정렬 기준 누락**: 다중 조건 정렬에서 1차 기준이 같을 때 2차 기준을 빠뜨리면 결과가 달라질 수 있다
- **원본 배열 변경**: 정렬은 원본 배열을 변경한다. 원본이 필요하면 복사본을 만들어 정렬하자

### 엣지 케이스 주의사항

- **빈 배열**: 길이 0인 배열에 대한 정렬은 아무 작업도 하지 않아야 한다
- **원소가 1개**: 이미 정렬된 상태이다
- **모든 원소가 동일**: 정렬 후에도 동일한 배열이어야 한다. 안정 정렬 여부를 확인하자
- **이미 정렬된 배열**: 퀵 정렬의 최악 케이스(O(N²))가 될 수 있다. 피벗 선택 전략을 확인하자
- **역순 정렬된 배열**: 버블 정렬의 최악 케이스이다

## 관련 알고리즘 비교

| 알고리즘 | 평균 시간 | 최악 시간 | 공간 | 안정성 | 적합한 상황 |
| --- | --- | --- | --- | --- | --- |
| 버블 정렬 | O(N²) | O(N²) | O(1) | 안정 | 교육용, 거의 정렬된 소규모 데이터 |
| 선택 정렬 | O(N²) | O(N²) | O(1) | 불안정 | 교환 횟수를 최소화해야 할 때 |
| 삽입 정렬 | O(N²) | O(N²) | O(1) | 안정 | 거의 정렬된 데이터, 소규모 데이터 |
| 병합 정렬 | O(N log N) | O(N log N) | O(N) | 안정 | 안정 정렬이 필요하거나 최악 보장이 필요한 경우 |
| 퀵 정렬 | O(N log N) | O(N²) | O(log N) | 불안정 | 평균적으로 가장 빠른 범용 정렬 |
| 힙 정렬 | O(N log N) | O(N log N) | O(1) | 불안정 | 추가 메모리 없이 O(N log N) 보장이 필요한 경우 |
| 계수 정렬 | O(N+K) | O(N+K) | O(K) | 안정 | 값의 범위(K)가 작은 정수 데이터 |

**선택 기준**:
- 코딩테스트 일반 → Go `sort.Slice()` 사용 (내부적으로 인트로소트)
- 안정 정렬 필요 → `sort.SliceStable()` 또는 병합 정렬
- 역전 쌍 세기 → 병합 정렬 변형
- 값 범위가 작은 정수 → 계수 정렬
