# 커스텀 정렬 - 해설

## 접근 방식

1. N명의 학생 정보를 구조체 배열에 저장한다
2. 다중 조건 비교 함수를 정의한다
3. `sort.SliceStable`을 사용하여 안정 정렬을 수행한다
4. 정렬된 순서대로 이름을 출력한다

## 핵심 아이디어

다중 조건 정렬은 비교 함수에서 우선순위가 높은 조건부터 순서대로 비교하는 것이 핵심이다. 두 원소의 현재 조건 값이 같으면 다음 조건으로 넘어가고, 다르면 해당 조건에 따라 순서를 결정한다.

Go의 `sort.SliceStable`은 안정 정렬을 보장하므로, 모든 비교 조건이 같은 원소들의 상대적 순서가 유지된다. 이 문제에서는 4번째 조건(이름 사전순)까지 명시되어 있으므로 `sort.Slice`를 사용해도 동일한 결과를 얻을 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

N명의 학생을 정렬하는 데 O(N log N), 학생 정보 저장에 O(N)의 공간이 필요하다. 비교 함수 내부의 문자열 비교는 이름 길이가 최대 10이므로 상수 시간으로 취급할 수 있다.

## 대안적 접근

- **구조체 정렬 인터페이스**: Go의 `sort.Interface`를 구현하여 `Len`, `Less`, `Swap` 메서드를 직접 정의할 수 있다. 코드가 길어지지만 재사용성이 높다.
- **키 변환 정렬**: 각 학생의 정렬 키를 하나의 정수로 변환하여 단일 조건으로 정렬할 수 있다. 점수 범위가 0~100이므로 `kor*1000000 + (100-eng)*10000 + math*100` 같은 복합 키를 만들 수 있다.
