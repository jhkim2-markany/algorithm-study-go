# 역전 카운트 - 해설

## 접근 방식

1. 배열을 입력받는다
2. 병합 정렬을 수행하면서 역전 개수를 동시에 센다
3. 병합 단계에서 왼쪽 배열의 원소가 오른쪽 배열의 원소보다 클 때 역전이 발생한다
4. 누적된 역전 개수를 출력한다

## 핵심 아이디어

병합 정렬의 병합(merge) 단계를 활용하여 역전 개수를 효율적으로 센다. 두 정렬된 부분 배열을 병합할 때, 왼쪽 배열의 원소 `arr[i]`가 오른쪽 배열의 원소 `arr[j]`보다 크면, 왼쪽 배열에서 `i`부터 `mid`까지의 모든 원소도 `arr[j]`보다 크다(왼쪽 배열이 이미 정렬되어 있으므로). 따라서 한 번의 비교로 `mid - i + 1`개의 역전을 한꺼번에 셀 수 있다.

브루트포스로 모든 쌍을 비교하면 O(N²)이지만, 병합 정렬을 활용하면 O(N log N)에 해결할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

병합 정렬의 시간 복잡도가 O(N log N)이고, 역전 개수를 세는 작업은 병합 과정에 포함되므로 추가 시간이 들지 않는다. 임시 배열을 위한 O(N) 추가 공간이 필요하다.

## 대안적 접근

- **브루트포스**: 이중 반복문으로 모든 쌍 (i, j)를 검사한다. O(N²)이므로 N이 500,000일 때 시간 초과가 발생한다.
- **BIT(Binary Indexed Tree)**: 좌표 압축 후 BIT를 사용하여 역전 개수를 셀 수 있다. 시간 복잡도는 동일하게 O(N log N)이지만 구현이 다소 복잡하다.
