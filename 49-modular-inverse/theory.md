# 모듈로 곱셈 역원 (Modular Inverse)

## 개념

모듈로 곱셈 역원(Modular Multiplicative Inverse)은 정수 a와 모듈러 M에 대해 a × x ≡ 1 (mod M)을 만족하는 정수 x를 구하는 것이다. 이 x를 a의 모듈러 역원이라 하며, a⁻¹ (mod M)으로 표기한다.

일반적인 나눗셈 a/b는 모듈러 연산에서 직접 수행할 수 없다. 대신 b의 모듈러 역원 b⁻¹을 구한 뒤 a × b⁻¹ (mod M)으로 계산한다. 이것이 모듈러 나눗셈의 핵심이다.

모듈러 역원이 존재하려면 gcd(a, M) = 1이어야 한다. 즉, a와 M이 서로소여야 한다. M이 소수이면 1 ≤ a < M인 모든 a에 대해 역원이 존재한다.

코딩테스트에서 모듈러 역원은 이항 계수 nCr mod p, 확률의 모듈러 표현, 분수의 모듈러 계산 등에 핵심적으로 사용된다.

## 동작 원리

### 1. 페르마 소정리를 이용한 역원 계산

M이 소수일 때, 페르마 소정리에 의해 a^(M-1) ≡ 1 (mod M)이 성립한다. 양변을 a로 나누면:

```
a^(M-2) ≡ a⁻¹ (mod M)
```

따라서 a의 모듈러 역원은 a^(M-2) mod M이다.

1. M이 소수인지 확인한다
2. 빠른 거듭제곱(분할 정복 거듭제곱)으로 a^(M-2) mod M을 계산한다
3. 결과가 a의 모듈러 역원이다

이 방법은 M이 소수일 때만 사용할 수 있으며, 코딩테스트에서 M = 10⁹+7인 경우가 대부분이므로 가장 많이 사용된다.

### 2. 확장 유클리드 알고리즘을 이용한 역원 계산

M이 소수가 아닌 경우에도 gcd(a, M) = 1이면 역원을 구할 수 있다.

확장 유클리드 알고리즘은 ax + My = gcd(a, M) = 1을 만족하는 x, y를 구한다. 이때 ax ≡ 1 (mod M)이므로 x가 역원이다.

1. 확장 유클리드 알고리즘으로 ax + My = 1의 해 x를 구한다
2. x가 음수이면 x + M으로 양수로 변환한다
3. 결과가 a의 모듈러 역원이다

### 3. 모듈러 나눗셈

a / b (mod M)을 계산하는 과정:

1. b의 모듈러 역원 b⁻¹을 구한다 (페르마 소정리 또는 확장 유클리드)
2. a × b⁻¹ mod M을 계산한다
3. 결과가 a / b mod M이다

### 4. 이항 계수의 모듈러 계산

nCr mod p (p는 소수)를 계산하는 과정:

1. 팩토리얼 배열을 전처리한다: fact[i] = i! mod p
2. 역팩토리얼 배열을 전처리한다: inv_fact[i] = (i!)⁻¹ mod p
3. nCr = fact[n] × inv_fact[r] × inv_fact[n-r] mod p로 계산한다

역팩토리얼은 fact[n]의 역원을 먼저 구한 뒤, inv_fact[i] = inv_fact[i+1] × (i+1) mod p로 역순으로 계산하면 효율적이다.

### 5. 역원 배열 전처리

1부터 N까지의 모듈러 역원을 O(N)에 전처리하는 방법:

1. inv[1] = 1로 초기화한다
2. i = 2부터 N까지: inv[i] = -(M / i) × inv[M % i] mod M
3. 이 점화식은 M = (M/i) × i + (M%i)에서 유도된다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 페르마 소정리 역원 (단일) | O(log M) |
| 확장 유클리드 역원 (단일) | O(log M) |
| 팩토리얼/역팩토리얼 전처리 | O(N + log M) |
| 이항 계수 쿼리 (전처리 후) | O(1) |
| 역원 배열 전처리 (1~N) | O(N) |
| 공간 복잡도 | O(N) |

단일 역원 계산은 빠른 거듭제곱 또는 확장 유클리드 모두 O(log M)이다. 팩토리얼 전처리는 O(N)이고 역원 한 번 O(log M)이 추가된다. 전처리 후 이항 계수 쿼리는 O(1)이다.

## 적합한 문제 유형

- 이항 계수 nCr mod p 계산 문제
- 확률을 모듈러 분수로 표현하는 문제 (P × Q⁻¹ mod M)
- 조합론 문제에서 큰 수의 나눗셈이 필요한 경우
- 카탈란 수, 스털링 수 등 조합적 수열의 모듈러 계산
- 분수의 합을 모듈러로 구하는 문제
- 행렬식의 모듈러 계산 문제
- 다항식 보간(라그랑주 보간)의 모듈러 계산

## 단계별 추적 (Trace)

### 예시 입력

a = 3, M = 7 (소수)에 대해 모듈러 역원을 구한다.

### 실행 과정: 페르마 소정리

```
a⁻¹ ≡ a^(M-2) mod M
3⁻¹ ≡ 3^5 mod 7

빠른 거듭제곱으로 3^5 mod 7 계산:
  exp = 5 (이진수: 101)

  result = 1, base = 3

  비트 0 (1): result = 1 × 3 = 3 mod 7 = 3,  base = 3² = 9 mod 7 = 2
  비트 1 (0): result 변화 없음,                 base = 2² = 4 mod 7 = 4
  비트 2 (1): result = 3 × 4 = 12 mod 7 = 5,  base = 4² = 16 mod 7 = 2

  결과: 3⁻¹ mod 7 = 5
```

**검증: 3 × 5 = 15 ≡ 1 (mod 7) ✓**

### 실행 과정: 확장 유클리드 알고리즘

```
3x + 7y = gcd(3, 7) = 1 을 만족하는 x를 구한다.

extGCD(3, 7):
  extGCD(7 mod 3, 3) = extGCD(1, 3)
    extGCD(3 mod 1, 1) = extGCD(0, 1)
      반환: gcd=1, x=0, y=1

    역추적: x = 1 - (3/1)×0 = 1, y = 0
    반환: gcd=1, x=1, y=0

  역추적: x = 0 - (7/3)×1 = -2, y = 1
  반환: gcd=1, x=-2, y=1

x = -2 → (-2 + 7) mod 7 = 5

결과: 3⁻¹ mod 7 = 5 ✓
```

### 이항 계수 계산 예시

5C2 mod 7을 계산한다.

```
팩토리얼 전처리 (mod 7):
  fact[0]=1, fact[1]=1, fact[2]=2, fact[3]=6, fact[4]=3, fact[5]=1

역팩토리얼 전처리:
  inv_fact[5] = fact[5]⁻¹ mod 7 = 1⁻¹ mod 7 = 1
  inv_fact[4] = inv_fact[5] × 5 mod 7 = 1 × 5 = 5
  inv_fact[3] = inv_fact[4] × 4 mod 7 = 5 × 4 = 20 mod 7 = 6
  inv_fact[2] = inv_fact[3] × 3 mod 7 = 6 × 3 = 18 mod 7 = 4
  inv_fact[1] = inv_fact[2] × 2 mod 7 = 4 × 2 = 8 mod 7 = 1
  inv_fact[0] = inv_fact[1] × 1 mod 7 = 1

5C2 = fact[5] × inv_fact[2] × inv_fact[3] mod 7
     = 1 × 4 × 6 mod 7
     = 24 mod 7 = 3

검증: 5C2 = 10, 10 mod 7 = 3 ✓
```

## 실전 팁

### 활용 노하우

- M = 10⁹+7인 문제가 대부분이므로, 페르마 소정리 기반 역원 계산을 템플릿으로 준비하자
- 이항 계수를 여러 번 구해야 하면 팩토리얼/역팩토리얼 배열을 전처리하자. 전처리 O(N), 쿼리 O(1)이다
- 역원 배열 전처리 점화식 `inv[i] = -(M/i) * inv[M%i] % M`은 1~N까지의 역원을 O(N)에 구할 수 있어 매우 유용하다
- 분수 a/b를 모듈러로 표현할 때는 항상 `a * modInverse(b, M) % M`으로 계산한다

### 자주 하는 실수

- M이 소수가 아닌데 페르마 소정리를 적용하는 실수: M이 소수인지 반드시 확인하자. 소수가 아니면 확장 유클리드를 사용해야 한다
- 오버플로우: `a * b % M`에서 a, b가 int64 범위를 넘을 수 있다. Go에서는 int64로 충분하지만, 곱셈 전에 각각 모듈러를 취하자
- 역원이 존재하지 않는 경우: gcd(a, M) ≠ 1이면 역원이 없다. a가 M의 배수인 경우를 체크하자
- 역팩토리얼 계산 순서 오류: `inv_fact[N]`을 먼저 구한 뒤 역순으로 `inv_fact[i] = inv_fact[i+1] * (i+1) % M`으로 계산해야 한다

### 엣지 케이스

- a = 0인 경우: 0의 역원은 존재하지 않는다. 나눗셈에서 분모가 0이 되는 경우를 사전에 체크하자
- a = 1인 경우: 1의 역원은 항상 1이다
- nCr에서 r > n인 경우: 결과는 0이다. 팩토리얼 배열 접근 전에 범위를 확인하자
- nCr에서 r = 0 또는 r = n인 경우: 결과는 1이다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 조건 | 적합한 상황 |
| --- | --- | --- | --- |
| 페르마 소정리 | O(log M) | M이 소수 | 코딩테스트 대부분 (M=10⁹+7) |
| 확장 유클리드 | O(log M) | gcd(a,M)=1 | M이 소수가 아닌 경우 |
| 역원 배열 전처리 | O(N) | M이 소수 | 1~N 전체 역원이 필요한 경우 |
| 뤼카 정리 | O(p² + log_p(n)) | p가 소수, n이 매우 큼 | n이 매우 크고 p가 작은 이항 계수 |

- M이 소수이고 단일 역원이면 → 페르마 소정리 (가장 간단)
- M이 소수가 아니면 → 확장 유클리드
- 이항 계수를 반복 계산하면 → 팩토리얼/역팩토리얼 전처리
- n이 매우 크고 p가 작으면 → 뤼카 정리
