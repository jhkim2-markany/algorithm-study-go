# 이항 계수 모듈러 계산 - 해설

## 접근 방식

1. 팩토리얼 배열 fact[i] = i! mod M을 전처리한다
2. 역팩토리얼 배열 invFact[i] = (i!)⁻¹ mod M을 전처리한다
3. 각 쿼리에 대해 C(N, R) = fact[N] × invFact[R] × invFact[N-R] mod M으로 O(1)에 계산한다

## 핵심 아이디어

이항 계수 C(N, R) = N! / (R! × (N-R)!)에서 나눗셈을 모듈러 역원으로 대체한다. M이 소수이므로 페르마 소정리를 적용하여 (k!)⁻¹ = (k!)^(M-2) mod M으로 역원을 구한다.

역팩토리얼 배열의 효율적 전처리가 핵심이다. fact[N]의 역원 하나만 구한 뒤, invFact[i] = invFact[i+1] × (i+1) mod M 점화식으로 역순 계산하면 역원 계산을 한 번만 수행하면 된다.

이 관계식은 (i!)⁻¹ = ((i+1)!)⁻¹ × (i+1)에서 유도된다. (i+1)! = (i+1) × i! 이므로 양변의 역원을 취하면 성립한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (전처리) | O(N + log M) |
| 시간 복잡도 (쿼리당) | O(1) |
| 시간 복잡도 (전체) | O(N + Q) |
| 공간 복잡도 | O(N) |

N은 최대 입력값, Q는 쿼리 수이다. 전처리에 O(N)과 역원 한 번 O(log M)이 필요하고, 이후 각 쿼리는 O(1)이다.

## 대안적 접근

- 파스칼 삼각형: C(N, R) = C(N-1, R-1) + C(N-1, R)로 DP 테이블을 구성하면 O(N²) 시간과 공간이 필요하다. N이 크면 비효율적이다.
- 뤼카 정리: M이 작은 소수일 때 N > M인 경우에 사용한다. 이 문제에서는 M = 10⁹+7이므로 N < M이 보장되어 뤼카 정리가 불필요하다.
