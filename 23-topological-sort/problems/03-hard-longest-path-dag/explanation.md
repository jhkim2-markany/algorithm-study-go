# DAG 최장 경로 - 해설

## 접근 방식

1. 일반 그래프에서 최장 경로 문제는 NP-Hard이지만, DAG에서는 위상 정렬을 이용하여 다항 시간에 해결할 수 있다.
2. 먼저 Kahn 알고리즘으로 위상 정렬을 수행하여 정점의 처리 순서를 구한다.
3. 시작 정점의 거리를 0으로 초기화하고, 나머지 정점은 음의 무한대로 초기화한다.
4. 위상 정렬 순서대로 각 정점에서 나가는 간선을 통해 인접 정점의 최장 거리를 갱신한다.
5. 최종적으로 각 정점의 dist 값이 시작 정점에서의 최장 경로 길이가 된다.

## 핵심 아이디어

DAG에서 위상 정렬과 동적 프로그래밍(DP)을 결합하는 것이 핵심이다. 위상 정렬 순서는 "어떤 정점을 처리할 때 그 정점으로 들어오는 모든 간선의 출발 정점이 이미 처리되었음"을 보장한다. 이 성질 덕분에 각 정점을 한 번만 처리하면서 최장 거리를 정확히 계산할 수 있다.

DP 점화식은 다음과 같다:
- `dist[v] = max(dist[v], dist[u] + w(u, v))` (간선 u → v, 가중치 w)

시작 정점에서 도달할 수 없는 정점은 dist 값이 음의 무한대로 유지되므로, 이를 -1로 변환하여 출력한다.

예제 1에서 정점 1에서 정점 6까지의 최장 경로는 1→2→4→6 (3+6+2=11)이다. 1→3→5→6 경로(2+1+5=8)보다 길다.

일반 그래프에서는 사이클 때문에 최장 경로를 무한히 늘릴 수 있어 NP-Hard이지만, DAG에서는 사이클이 없으므로 위상 정렬 순서로 한 번의 순회만으로 해결된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

위상 정렬에 O(N + M), DP 갱신에 O(N + M)이 소요되어 전체 시간 복잡도는 O(N + M)이다. 인접 리스트에 O(N + M), 거리 배열과 위상 정렬 결과에 O(N)의 공간이 필요하다.

## 대안적 접근

- 최단 경로를 구하려면 DP 갱신에서 max 대신 min을 사용하면 된다. DAG에서는 최단/최장 경로 모두 O(V + E)에 해결 가능하다.
- DFS 기반 위상 정렬을 사용한 후 역순으로 DP를 수행해도 동일한 결과를 얻을 수 있다.
- 메모이제이션을 활용한 Top-down DP로도 풀 수 있다. 각 정점에서 도달 가능한 최장 거리를 재귀적으로 계산하는 방식이다.
