# 작업 순서 정하기

**난이도:** 중

## 문제 설명

N개의 작업과 M개의 선행 관계가 주어진다. 각 선행 관계는 "작업 A를 완료해야 작업 B를 시작할 수 있다"를 의미한다. 모든 작업을 수행할 수 있는 유효한 순서를 하나 출력하시오.

유효한 순서가 여러 개 존재하면 그 중 아무거나 하나를 출력한다. 만약 선행 관계에 순환이 있어 모든 작업을 수행할 수 없으면 -1을 출력한다.

## 입력 형식

- 첫째 줄에 작업의 수 N과 선행 관계의 수 M이 주어진다
- 둘째 줄부터 M개의 줄에 각 선행 관계 A B가 주어진다 (A를 완료해야 B를 시작 가능)

## 출력 형식

- 유효한 작업 순서를 공백으로 구분하여 한 줄에 출력한다
- 순환이 있어 불가능하면 -1을 출력한다

## 제약 조건

- 1 ≤ N ≤ 100,000
- 0 ≤ M ≤ 200,000
- 1 ≤ A, B ≤ N
- A ≠ B
- 같은 선행 관계가 중복으로 주어지지 않는다
- 시간 제한: 1초
- 메모리 제한: 256MB

## 예제

### 예제 입력 1

```
6 6
1 2
1 3
2 4
3 4
4 5
3 6
```

### 예제 출력 1

```
1 2 3 4 6 5
```

### 예제 입력 2

```
3 3
1 2
2 3
3 1
```

### 예제 출력 2

```
-1
```
