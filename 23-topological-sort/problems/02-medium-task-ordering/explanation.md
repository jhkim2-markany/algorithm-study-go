# 작업 순서 정하기 - 해설

## 접근 방식

1. 작업 간 선행 관계를 방향 그래프로 모델링한다. 간선 A → B는 "A를 완료해야 B를 시작 가능"을 의미한다.
2. Kahn 알고리즘(BFS 기반 위상 정렬)을 사용하여 유효한 작업 순서를 구한다.
3. 진입 차수가 0인 작업부터 큐에 넣고, 처리할 때마다 인접 작업의 진입 차수를 감소시킨다.
4. 모든 작업이 처리되면 결과를 출력하고, 그렇지 않으면 사이클이 존재하므로 -1을 출력한다.

## 핵심 아이디어

위상 정렬을 직접 수행하여 유효한 순서를 출력하는 문제이다. 사이클 판별과 함께 실제 정렬 결과를 구해야 한다는 점에서 기본 문제보다 한 단계 발전된 형태이다.

Kahn 알고리즘에서 큐에 여러 정점이 동시에 들어갈 수 있으므로, 꺼내는 순서에 따라 다양한 유효한 결과가 나올 수 있다. 문제에서 아무 유효한 순서나 출력하면 되므로 일반 큐를 사용한다.

예제 1에서 작업 1은 선행 작업이 없으므로 가장 먼저 수행한다. 이후 작업 2, 3이 수행 가능해지고, 작업 4는 2와 3이 모두 완료된 후에 수행할 수 있다. 작업 5는 4 이후, 작업 6은 3 이후에 수행 가능하다.

만약 사전 순으로 가장 빠른 순서를 구해야 한다면 일반 큐 대신 최소 힙(우선순위 큐)을 사용하면 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

Kahn 알고리즘은 모든 정점과 간선을 한 번씩 처리하므로 O(N + M)이다. 인접 리스트에 O(N + M), 진입 차수 배열과 큐에 O(N), 결과 배열에 O(N)의 공간이 필요하다.

## 대안적 접근

- DFS 기반 위상 정렬을 사용할 수도 있다. DFS 후위 순회의 역순이 위상 정렬 결과가 된다.
- 사전 순으로 가장 빠른 순서가 필요하면 큐 대신 최소 힙을 사용하여 O(N log N + M)에 해결할 수 있다.
- 가능한 모든 위상 정렬 결과를 구하려면 백트래킹을 사용해야 하며, 최악의 경우 지수 시간이 소요된다.
