# 위상 정렬

## 개념

위상 정렬(Topological Sort)은 방향 비순환 그래프(DAG, Directed Acyclic Graph)에서 정점들을 선행 관계를 만족하도록 일렬로 나열하는 알고리즘이다. 간선 (u, v)가 존재하면 정렬 결과에서 u가 반드시 v보다 앞에 위치해야 한다.

위상 정렬의 핵심 특성은 다음과 같다:
- **DAG 전용**: 사이클이 존재하는 그래프에서는 위상 정렬이 불가능하다. 따라서 위상 정렬의 가능 여부로 사이클 존재를 판별할 수 있다.
- **결과의 비유일성**: 하나의 DAG에 대해 여러 유효한 위상 정렬 결과가 존재할 수 있다.
- **선행 관계 보장**: 정렬 결과에서 모든 간선의 방향이 왼쪽에서 오른쪽으로 향한다.
- **두 가지 대표 알고리즘**: 진입 차수 기반의 Kahn 알고리즘(BFS)과 DFS 기반 위상 정렬이 있다.

위상 정렬은 작업 스케줄링, 수강 과목 순서 결정, 빌드 시스템의 의존성 해결, 스프레드시트 셀 계산 순서 등에 널리 사용된다.

## 동작 원리

### Kahn 알고리즘 (BFS 기반)

진입 차수(in-degree)를 이용하여 BFS 방식으로 위상 정렬을 수행한다.

1. 모든 정점의 진입 차수를 계산한다.
2. 진입 차수가 0인 정점을 모두 큐에 넣는다.
3. 큐에서 정점을 하나 꺼내 결과 리스트에 추가한다.
4. 꺼낸 정점에서 나가는 모든 간선을 제거한다 (인접 정점의 진입 차수를 1 감소).
5. 진입 차수가 0이 된 정점을 큐에 추가한다.
6. 큐가 빌 때까지 3~5를 반복한다.
7. 결과 리스트의 크기가 정점 수와 같으면 위상 정렬 성공, 그렇지 않으면 사이클이 존재한다.

### DFS 기반 위상 정렬

DFS의 후위 순회(post-order) 역순을 이용하여 위상 정렬을 수행한다.

1. 모든 정점을 미방문 상태로 초기화한다.
2. 미방문 정점을 하나 선택하여 DFS를 시작한다.
3. DFS에서 현재 정점의 모든 인접 정점을 재귀적으로 방문한다.
4. 현재 정점의 모든 인접 정점 방문이 끝나면 현재 정점을 스택에 push한다.
5. 모든 정점에 대해 2~4를 반복한다.
6. 스택에서 pop하는 순서가 위상 정렬 결과이다.
7. DFS 중 현재 경로에서 이미 방문 중인 정점을 다시 만나면 사이클이 존재한다.

### 사이클 판별

- **Kahn 알고리즘**: 결과 리스트에 포함된 정점 수가 전체 정점 수보다 적으면 사이클이 존재한다.
- **DFS 기반**: 방문 상태를 미방문(0), 방문 중(1), 방문 완료(2)로 구분하여, 방문 중인 정점을 다시 만나면 사이클이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (Kahn) | O(V + E) |
| 시간 복잡도 (DFS) | O(V + E) |
| 공간 복잡도 (Kahn) | O(V + E) |
| 공간 복잡도 (DFS) | O(V + E) |

두 알고리즘 모두 모든 정점과 간선을 한 번씩 처리하므로 시간 복잡도는 O(V + E)이다. 인접 리스트 저장에 O(V + E), 진입 차수 배열 또는 방문 배열에 O(V), 큐 또는 재귀 스택에 O(V)의 공간이 필요하다.

## 적합한 문제 유형

- 작업 스케줄링 (선행 작업이 있는 작업 순서 결정)
- 수강 과목 순서 결정 (선수 과목 관계)
- 빌드 시스템 의존성 해결 (컴파일 순서)
- DAG에서의 최장/최단 경로 (위상 정렬 + DP)
- 사이클 판별 (방향 그래프에서 사이클 존재 여부)
- 이벤트 순서 결정 (인과 관계가 있는 이벤트 정렬)
- 스프레드시트 셀 계산 순서 결정

## 단계별 추적 (Trace)

### 예시 입력 - Kahn 알고리즘 (BFS)

정점 6개 (0~5), 간선: 0→1, 0→2, 1→3, 2→3, 3→4, 2→5

### 실행 과정

```
인접 리스트:
  0 → [1, 2]
  1 → [3]
  2 → [3, 5]
  3 → [4]
  4 → []
  5 → []

진입 차수 계산:
  in[0]=0, in[1]=1, in[2]=1, in[3]=2, in[4]=1, in[5]=1

초기 큐 (진입 차수 0): [0]
결과: []

Step 1: 큐에서 0 꺼냄 → 결과에 추가
  간선 제거: 0→1 (in[1]: 1→0), 0→2 (in[2]: 1→0)
  진입 차수 0이 된 정점: 1, 2 → 큐에 추가
  큐: [1, 2],  결과: [0]

Step 2: 큐에서 1 꺼냄 → 결과에 추가
  간선 제거: 1→3 (in[3]: 2→1)
  큐: [2],  결과: [0, 1]

Step 3: 큐에서 2 꺼냄 → 결과에 추가
  간선 제거: 2→3 (in[3]: 1→0), 2→5 (in[5]: 1→0)
  진입 차수 0이 된 정점: 3, 5 → 큐에 추가
  큐: [3, 5],  결과: [0, 1, 2]

Step 4: 큐에서 3 꺼냄 → 결과에 추가
  간선 제거: 3→4 (in[4]: 1→0)
  진입 차수 0이 된 정점: 4 → 큐에 추가
  큐: [5, 4],  결과: [0, 1, 2, 3]

Step 5: 큐에서 5 꺼냄 → 결과에 추가
  큐: [4],  결과: [0, 1, 2, 3, 5]

Step 6: 큐에서 4 꺼냄 → 결과에 추가
  큐: [],  결과: [0, 1, 2, 3, 5, 4]

결과 크기(6) == 정점 수(6) → 사이클 없음, 위상 정렬 성공
위상 정렬 결과: [0, 1, 2, 3, 5, 4]
```

### DFS 기반 추적

```
방문 순서 (후위 역순):
  DFS(0) → DFS(1) → DFS(3) → DFS(4)
    4 완료 → 스택 push 4
    3 완료 → 스택 push 3
    1 완료 → 스택 push 1
  → DFS(2) → DFS(5)
    5 완료 → 스택 push 5
    2 완료 → 스택 push 2
  0 완료 → 스택 push 0

스택 (top→bottom): [0, 2, 5, 1, 3, 4]
위상 정렬 결과: [0, 2, 5, 1, 3, 4]  (유효한 다른 순서)
```

## 실전 팁

### 활용 노하우

- "순서", "선행 조건", "의존성"이라는 키워드가 나오면 위상 정렬을 떠올리자
- Kahn 알고리즘은 사이클 판별이 자연스럽고, 사전순 최소 위상 정렬이 필요하면 큐 대신 우선순위 큐(최소 힙)를 사용한다
- DAG에서 최장 경로를 구할 때 위상 정렬 + DP 조합이 매우 효과적이다
- 위상 정렬 결과가 유일한지 판별하려면 매 단계에서 큐에 원소가 정확히 1개인지 확인한다
- DFS 기반은 구현이 간결하지만, 사이클 판별을 위해 방문 상태를 3가지(미방문/방문중/완료)로 관리해야 한다

### 자주 하는 실수

- 사이클이 있는 그래프에서 위상 정렬을 시도하고 결과를 그대로 사용하는 경우 (반드시 사이클 체크 필요)
- Kahn 알고리즘에서 진입 차수 배열 초기화를 잘못하는 경우
- DFS 기반에서 방문 상태를 2가지(방문/미방문)로만 관리하여 사이클을 탐지하지 못하는 경우
- 비연결 그래프에서 모든 컴포넌트를 처리하지 않는 경우
- 결과 리스트 크기와 정점 수를 비교하지 않아 사이클을 놓치는 경우

### 엣지 케이스

- 간선이 없는 그래프 (모든 정점이 독립 → 아무 순서나 유효)
- 정점이 1개인 경우
- 선형 체인 형태의 DAG (유일한 위상 정렬 결과)
- 사이클이 존재하는 경우 (위상 정렬 불가능)
- 자기 자신으로의 간선(self-loop)이 있는 경우 (사이클로 판별)

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| Kahn (BFS 기반) | 진입 차수 활용, 사이클 판별 용이, 사전순 정렬 가능 | 사전순 최소 위상 정렬, 명시적 사이클 판별 |
| DFS 기반 위상 정렬 | 후위 역순, 구현 간결, 재귀 사용 | 단순 위상 정렬, SCC와 결합 시 |
| BFS (레벨 탐색) | 비가중치 최단 경로, 레벨별 탐색 | 최단 거리, 레벨 기반 문제 |
| DFS (깊이 탐색) | 경로 탐색, 백트래킹 | 경로 존재 여부, 사이클 탐지 |
| SCC (Tarjan/Kosaraju) | 강한 연결 요소 분해 후 DAG 축약 | 방향 그래프 구조 분석 후 위상 정렬 |
