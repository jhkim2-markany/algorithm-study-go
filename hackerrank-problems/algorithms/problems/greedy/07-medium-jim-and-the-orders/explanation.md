# Jim과 주문 - 해설

## 접근 방식

1. 각 고객의 완성 시각(order[i] + prep[i])을 계산한다
2. 완성 시각을 기준으로 고객을 정렬한다
3. 완성 시각이 같으면 고객 번호가 작은 순서로 정렬한다
4. 정렬된 순서대로 고객 번호를 출력한다

## 핵심 아이디어

각 고객의 버거 완성 시각은 주문 시각과 준비 시간의 합으로 결정된다. 이 값을 기준으로 정렬하면 서빙 순서를 구할 수 있다. 동일한 완성 시각의 경우 고객 번호 순서로 처리한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

완성 시각 계산에 O(N), 정렬에 O(N log N)이 소요된다.

## 대안적 접근

- **우선순위 큐**: 완성 시각을 키로 하는 최소 힙을 사용할 수 있다. 정렬과 동일한 시간 복잡도이지만 코드가 더 복잡해진다.
- **카운팅 정렬**: 완성 시각의 범위가 제한적이라면 카운팅 정렬로 O(N + M)에 해결할 수 있다.
