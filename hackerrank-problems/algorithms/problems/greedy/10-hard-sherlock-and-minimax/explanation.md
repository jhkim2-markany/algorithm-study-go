# Sherlock과 MiniMax - 해설

## 접근 방식

1. 배열을 정렬한다
2. 후보 M 값을 생성한다: P, Q, 그리고 인접한 두 원소의 중간값
3. 각 후보에 대해 min(|arr[i] - M|)을 계산한다
4. 이 값이 최대인 M을 선택한다

## 핵심 아이디어

min(|arr[i] - M|)을 최대화하려면 M이 배열의 원소들로부터 최대한 멀리 떨어져야 한다. 정렬된 배열에서 인접한 두 원소 사이의 중간점이 최적 후보가 된다. 또한 범위의 양 끝점(P, Q)도 후보에 포함해야 한다. 각 후보에 대해 이진 탐색으로 가장 가까운 원소를 찾아 최소 거리를 계산한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

정렬에 O(N log N), 후보 생성 및 평가에 O(N)이 소요된다. 후보의 수는 최대 N+1개이다.

## 대안적 접근

- **삼분 탐색**: min(|arr[i] - M|) 함수는 구간별로 단봉(unimodal)이므로, 각 구간에서 삼분 탐색을 적용할 수 있다.
- **이진 탐색**: 답(최소 거리)을 이진 탐색하고, 해당 거리를 달성할 수 있는 M이 [P, Q]에 존재하는지 확인하는 방법도 가능하다.
