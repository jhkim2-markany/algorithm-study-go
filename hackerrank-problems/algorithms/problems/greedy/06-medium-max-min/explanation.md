# Max Min - 해설

## 접근 방식

1. 배열을 오름차순으로 정렬한다
2. 크기 K인 슬라이딩 윈도우를 사용한다
3. 각 윈도우에서 최댓값(arr[i+k-1])과 최솟값(arr[i])의 차이를 계산한다
4. 모든 윈도우 중 최소 차이를 반환한다

## 핵심 아이디어

정렬된 배열에서 K개의 연속된 원소를 선택하면, 그 부분 배열의 최댓값과 최솟값의 차이가 최소화된다. 비연속적인 K개를 선택하면 차이가 더 커질 수밖에 없다. 이것이 그리디 선택의 근거이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(1) |

정렬에 O(N log N), 슬라이딩 윈도우 탐색에 O(N)이 소요된다.

## 대안적 접근

- **브루트포스**: N개에서 K개를 선택하는 모든 조합을 시도한다. 시간 복잡도 O(C(N,K))로 매우 비효율적이다.
- **이진 탐색**: 불공정도를 이진 탐색하여 해당 불공정도 이하로 K개를 선택할 수 있는지 확인하는 방법도 있으나, 정렬 + 슬라이딩 윈도우가 더 간단하다.
