# 쌍 찾기 - 해설

## 접근 방식

1. 배열의 모든 원소를 해시셋에 저장한다
2. 배열을 순회하면서 각 원소에 대해 (원소 + K)가 해시셋에 있는지 확인한다
3. 존재하면 쌍의 개수를 증가시킨다

## 핵심 아이디어

해시셋을 사용하면 각 원소에 대해 차이가 K인 상대 원소의 존재 여부를 O(1)에 확인할 수 있다. 모든 원소가 서로 다르므로 중복 처리가 필요 없다. 각 원소 x에 대해 x + K만 확인하면 쌍을 한 번만 세게 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

해시셋 구성에 O(N), 탐색에 O(N)이므로 전체 O(N)이다.

## 대안적 접근

- **정렬 + 투 포인터**: 배열을 정렬한 뒤 두 포인터로 차이가 K인 쌍을 찾는다. 시간 O(N log N), 공간 O(1).
- **정렬 + 이진 탐색**: 배열을 정렬한 뒤 각 원소에 대해 이진 탐색으로 상대 원소를 찾는다. 시간 O(N log N).
