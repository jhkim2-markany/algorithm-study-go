# 최소 손실 - 해설

## 접근 방식

1. 가격을 정렬된 자료구조(BST 또는 정렬 배열)에 역순으로 삽입한다
2. 각 가격을 삽입할 때, 자료구조에서 해당 가격보다 약간 큰 값을 찾는다
3. 그 차이가 현재까지의 최소 손실보다 작으면 갱신한다

## 핵심 아이디어

매입은 매도보다 먼저 일어나야 하고, 매입가 > 매도가여야 한다. 가격 배열을 뒤에서부터 처리하면, 이미 삽입된 값들은 현재 값보다 나중 연도의 가격이다. 현재 가격(매입가)보다 약간 작은 값이 아닌, 정렬된 구조에서 현재 값 바로 위의 값을 찾아 차이를 구한다. 이는 이진 탐색 또는 균형 BST로 효율적으로 수행할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 원소의 삽입과 탐색이 O(log N)이므로 전체 O(N log N)이다.

## 대안적 접근

- **정렬 + 인덱스 비교**: 가격을 정렬하고 인접한 값들의 차이를 확인하되, 원래 인덱스 순서(매입이 매도보다 먼저)를 만족하는지 검증한다. 시간 O(N log N).
- **브루트포스**: 모든 (i, j) 쌍(i < j)에서 price[i] > price[j]인 경우의 최소 차이를 구한다. 시간 O(N²).
