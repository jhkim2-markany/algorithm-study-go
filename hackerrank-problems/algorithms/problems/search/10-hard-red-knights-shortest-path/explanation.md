# 레드 나이트의 최단 경로 - 해설

## 접근 방식

1. BFS로 시작 위치에서 목표 위치까지의 최단 경로를 탐색한다
2. 6가지 이동 방향을 우선순위 순서(UL, UR, R, LR, LL, L)로 탐색한다
3. 각 셀에 도달했을 때의 부모 셀과 이동 방향을 기록한다
4. 목표에 도달하면 경로를 역추적하여 출력한다

## 핵심 아이디어

BFS는 최단 경로를 보장하며, 이동 방향을 우선순위 순서로 탐색하면 같은 최단 거리의 경로 중 사전순으로 가장 빠른 경로를 자동으로 찾게 된다. BFS에서 같은 레벨의 노드들은 동일한 거리를 가지므로, 먼저 큐에 들어간 경로가 사전순으로 앞선다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N²) |
| 공간 복잡도 | O(N²) |

모든 셀을 최대 한 번 방문하므로 O(N²)이다. 방문 배열과 부모 정보에 O(N²) 공간이 필요하다.

## 대안적 접근

- **양방향 BFS**: 시작점과 끝점에서 동시에 BFS를 수행하여 탐색 공간을 줄일 수 있다. 다만 경로 복원과 우선순위 처리가 복잡해진다.
- **A* 탐색**: 목표까지의 추정 거리를 휴리스틱으로 사용하여 탐색을 가속할 수 있다.
