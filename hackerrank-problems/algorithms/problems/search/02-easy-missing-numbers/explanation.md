# 빠진 숫자 - 해설

## 접근 방식

1. 원본 배열의 각 숫자 빈도를 해시맵으로 센다
2. 첫 번째 배열의 각 숫자 빈도를 해시맵에서 차감한다
3. 빈도가 양수인 숫자들을 수집한다
4. 결과를 오름차순으로 정렬하여 출력한다

## 핵심 아이디어

두 배열의 빈도 차이를 비교하는 문제이다. 해시맵으로 원본 배열의 빈도를 세고, 첫 번째 배열의 원소를 하나씩 차감하면 빈도가 남아 있는 숫자가 빠진 숫자이다. 숫자 범위가 100 이하로 제한되어 있으므로 배열로도 구현 가능하다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M + K log K) |
| 공간 복잡도 | O(K) |

N, M은 각 배열의 크기, K는 고유 숫자의 수이다. 빈도 계산에 O(N + M), 정렬에 O(K log K)가 소요된다. K ≤ 101이므로 정렬 비용은 무시할 수 있다.

## 대안적 접근

- **정렬 후 비교**: 두 배열을 정렬한 뒤 투 포인터로 비교할 수 있다. 시간 O(M log M), 공간 O(1).
- **카운팅 배열**: 숫자 범위가 100 이하이므로 크기 101의 배열로 빈도를 셀 수 있다. 해시맵보다 상수 시간이 빠르다.
