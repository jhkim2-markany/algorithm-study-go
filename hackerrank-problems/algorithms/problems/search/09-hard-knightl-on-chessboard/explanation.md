# KnightL 체스판 - 해설

## 접근 방식

1. 1 ≤ a, b ≤ N-1인 모든 (a, b) 쌍에 대해 BFS를 수행한다
2. 각 BFS에서 (0, 0)부터 시작하여 KnightL(a, b)의 이동 규칙에 따라 탐색한다
3. (N-1, N-1)에 도달하면 최소 이동 횟수를 기록하고, 도달 불가능하면 -1을 기록한다
4. KnightL(a, b)와 KnightL(b, a)의 결과는 동일하므로 대칭성을 활용할 수 있다

## 핵심 아이디어

BFS는 비가중 그래프에서 최단 경로를 보장한다. KnightL(a, b)의 이동은 8가지 방향이 가능하다: (±a, ±b)와 (±b, ±a). 체스판 크기가 최대 25×25이므로 각 BFS의 상태 공간은 625개로 매우 작다. 대칭성(KnightL(a,b) = KnightL(b,a))을 활용하면 계산량을 절반으로 줄일 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N² × N²) = O(N⁴) |
| 공간 복잡도 | O(N²) |

(N-1)² 개의 (a, b) 쌍에 대해 각각 O(N²) BFS를 수행한다. N ≤ 25이므로 최대 약 25⁴ ≈ 390,625 연산으로 충분히 빠르다.

## 대안적 접근

- **양방향 BFS**: 시작점과 끝점에서 동시에 BFS를 수행하여 만나는 지점을 찾는다. 탐색 공간을 줄일 수 있지만, N이 작아 효과가 미미하다.
- **A* 탐색**: 맨해튼 거리 등의 휴리스틱을 사용하여 탐색을 가속할 수 있다.
