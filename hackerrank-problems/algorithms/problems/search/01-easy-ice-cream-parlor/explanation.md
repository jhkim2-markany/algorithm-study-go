# 아이스크림 가게 - 해설

## 접근 방식

1. 해시맵을 사용하여 각 가격과 해당 인덱스를 저장한다
2. 배열을 순회하면서 현재 가격의 보수(M - 현재 가격)가 해시맵에 있는지 확인한다
3. 보수가 존재하면 두 인덱스를 출력한다
4. 존재하지 않으면 현재 가격과 인덱스를 해시맵에 추가한다

## 핵심 아이디어

Two Sum 문제의 변형이다. 해시맵을 사용하면 각 원소에 대해 보수를 O(1)에 찾을 수 있다. 배열을 한 번만 순회하면서 해시맵에 이전에 본 가격들을 저장하고, 현재 가격의 보수가 이미 저장되어 있는지 확인한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

배열을 한 번 순회하므로 O(N)이고, 해시맵에 최대 N개의 원소를 저장하므로 공간 복잡도도 O(N)이다.

## 대안적 접근

- **정렬 + 투 포인터**: 가격을 정렬한 뒤 양쪽 끝에서 투 포인터로 탐색할 수 있다. 시간 O(N log N), 공간 O(N). 원래 인덱스를 보존해야 하므로 별도 배열이 필요하다.
- **브루트포스**: 이중 반복문으로 모든 쌍을 확인한다. 시간 O(N²), 공간 O(1). N이 작을 때만 적합하다.
