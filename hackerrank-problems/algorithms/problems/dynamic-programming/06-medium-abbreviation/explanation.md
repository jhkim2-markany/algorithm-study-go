# Abbreviation - 해설

## 접근 방식

1. dp[i][j]를 a의 처음 i글자로 b의 처음 j글자를 만들 수 있는지 여부로 정의한다
2. a[i-1]이 대문자인 경우:
   - a[i-1] == b[j-1]이면 dp[i][j] = dp[i-1][j-1]
   - 아니면 dp[i][j] = false (대문자는 삭제할 수 없음)
3. a[i-1]이 소문자인 경우:
   - 대문자로 변환하여 b[j-1]과 매칭: upper(a[i-1]) == b[j-1]이면 dp[i-1][j-1]
   - 삭제: dp[i-1][j]
   - 둘 중 하나라도 true이면 dp[i][j] = true
4. dp[len(a)][len(b)]가 최종 답이다

## 핵심 아이디어

대문자는 반드시 매칭에 사용해야 하고(삭제 불가), 소문자는 대문자로 변환하여 매칭하거나 삭제할 수 있다. 이 두 가지 선택을 DP로 추적하면 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(|a| × |b|) |
| 공간 복잡도 | O(|a| × |b|) |

2차원 DP 테이블을 채우므로 시간과 공간 모두 O(|a| × |b|)이다. 공간은 1차원으로 최적화할 수 있다.

## 대안적 접근

- **재귀 + 메모이제이션**: 탑다운 방식으로 구현할 수 있다.
- **공간 최적화**: dp 테이블을 1차원으로 줄여 O(|b|) 공간으로 해결할 수 있다.
