# 최장 공통 부분 수열 - 해설

## 접근 방식

1. dp[i][j]를 첫 번째 수열의 처음 i개와 두 번째 수열의 처음 j개의 LCS 길이로 정의한다
2. a[i-1] == b[j-1]이면 dp[i][j] = dp[i-1][j-1] + 1
3. 아니면 dp[i][j] = max(dp[i-1][j], dp[i][j-1])
4. dp 테이블을 역추적하여 실제 LCS를 복원한다

## 핵심 아이디어

LCS는 동적 프로그래밍의 대표적인 문제이다. 두 수열의 원소가 같으면 LCS에 포함하고, 다르면 한쪽을 줄인 경우 중 더 긴 LCS를 선택한다. dp 테이블 완성 후 역추적으로 실제 수열을 복원한다.

역추적: dp[i][j]에서 시작하여, a[i-1] == b[j-1]이면 해당 원소를 LCS에 추가하고 (i-1, j-1)로 이동, 아니면 dp[i-1][j]와 dp[i][j-1] 중 큰 쪽으로 이동한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(N × M) |

2차원 dp 테이블을 채우므로 O(N × M)이다. 역추적을 위해 전체 테이블을 유지해야 하므로 공간도 O(N × M)이다.

## 대안적 접근

- **공간 최적화**: LCS 길이만 필요하면 O(min(N, M)) 공간으로 가능하나, 역추적이 필요하면 전체 테이블이 필요하다.
- **Hunt-Szymanski 알고리즘**: 알파벳 크기가 작을 때 더 효율적인 알고리즘이 있다.
