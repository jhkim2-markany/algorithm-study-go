# 동전 교환 문제 - 해설

## 접근 방식

1. dp[i]를 금액 i를 만드는 방법의 수로 정의한다
2. dp[0] = 1로 초기화한다 (금액 0을 만드는 방법은 아무 동전도 사용하지 않는 1가지)
3. 각 동전에 대해, 해당 동전 액면가부터 n까지 순회하며 dp 테이블을 갱신한다
4. dp[n]이 최종 답이다

## 핵심 아이디어

동전의 종류별로 순회하면서 dp 테이블을 갱신하면, 같은 조합을 중복 계산하지 않는다. 예를 들어 동전 [1, 2, 3]으로 금액 4를 만들 때, {1,1,2}와 {2,1,1}을 같은 조합으로 취급한다. 이는 동전을 "순서대로" 고려하기 때문이다.

점화식: dp[j] += dp[j - coin] (각 동전 coin에 대해)

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(N) |

N은 금액, M은 동전의 종류 수이다. 각 동전에 대해 금액 1부터 N까지 순회하므로 O(N × M)이다. 1차원 dp 배열만 사용하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **2차원 DP**: dp[i][j]를 i번째 동전까지 사용하여 금액 j를 만드는 방법의 수로 정의할 수 있다. 공간 복잡도가 O(N × M)으로 증가하지만 이해하기 쉽다.
- **재귀 + 메모이제이션**: 탑다운 방식으로 구현할 수 있으나, 바텀업 방식이 더 효율적이다.
