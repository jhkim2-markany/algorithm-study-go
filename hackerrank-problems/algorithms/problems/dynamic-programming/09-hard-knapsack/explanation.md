# Knapsack - 해설

## 접근 방식

1. dp[j]를 합이 정확히 j가 되는 것이 가능한지 여부로 정의한다
2. dp[0] = true로 초기화한다
3. 각 원소에 대해, 해당 값부터 K까지 순회하며 dp 테이블을 갱신한다
4. dp[K], dp[K-1], ..., dp[0] 중 true인 가장 큰 인덱스가 답이다

## 핵심 아이디어

무한 배낭 문제(Unbounded Knapsack)의 변형이다. 각 원소를 무한히 사용할 수 있으므로, 동전 교환 문제와 유사한 방식으로 접근한다. dp[j]가 true이면 합 j를 만들 수 있다는 의미이다.

또는 dp[j]를 합이 j 이하인 최대 합으로 정의하여 직접 최적값을 추적할 수도 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × K) |
| 공간 복잡도 | O(K) |

N개의 원소에 대해 K까지 순회하므로 O(N × K)이다. 1차원 dp 배열만 사용하므로 O(K)이다.

## 대안적 접근

- **BFS/DFS**: 가능한 합을 탐색하는 방식으로 접근할 수 있으나, 중복 상태가 많아 비효율적이다.
- **그리디**: 가장 큰 원소부터 사용하는 그리디는 최적해를 보장하지 않는다.
