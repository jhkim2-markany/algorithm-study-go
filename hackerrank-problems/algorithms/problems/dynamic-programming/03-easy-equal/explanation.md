# Equal - 해설

## 접근 방식

1. "나머지에게 주기"를 "한 명에게서 빼기"로 변환한다
2. 모든 값을 최솟값 기준으로 맞추되, 최솟값에서 0~4를 추가로 뺀 값도 시도한다
3. 각 목표값에 대해, 각 원소를 목표값까지 줄이는 데 필요한 최소 연산 횟수를 계산한다
4. 차이 d에 대해: d/5 + (d%5)/2 + (d%5)%2 번의 연산이 필요하다
5. 모든 목표값 중 최소 총 연산 횟수를 선택한다

## 핵심 아이디어

한 명을 제외한 나머지에게 k를 주는 것은 한 명에게서 k를 빼는 것과 동일하다. 따라서 각 원소를 특정 목표값까지 줄이는 문제로 변환된다. 목표값은 배열의 최솟값 이하여야 하며, 최솟값에서 0, 1, 2, 3, 4를 뺀 5가지 후보를 시도하면 최적해를 보장한다.

각 차이 d를 줄이는 최소 연산: 5로 나눈 몫 + 나머지를 2로 나눈 몫 + 최종 나머지(0 또는 1)

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

5가지 목표값 각각에 대해 N개의 원소를 순회하므로 O(5N) = O(N)이다. 추가 공간은 상수만 사용한다.

## 대안적 접근

- **완전 탐색**: 가능한 모든 목표값을 시도할 수 있으나, 범위가 넓어 비효율적이다.
- **그리디**: 가장 큰 단위(5)부터 사용하는 그리디 방식이 각 차이에 대한 최소 연산을 보장한다.
