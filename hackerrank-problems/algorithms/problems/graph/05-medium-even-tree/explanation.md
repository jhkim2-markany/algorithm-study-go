# 짝수 트리 - 해설

## 접근 방식

1. 루트(노드 1)에서 DFS를 수행하여 각 서브트리의 크기를 계산한다
2. 서브트리 크기가 짝수인 간선은 제거할 수 있다
3. 루트를 제외하고 서브트리 크기가 짝수인 노드의 수가 답이다

## 핵심 아이디어

서브트리의 노드 수가 짝수이면, 해당 서브트리를 부모로부터 분리해도 양쪽 모두 짝수 노드를 가진다. 전체 노드 수가 짝수이므로, 짝수 크기의 서브트리를 분리하면 나머지도 짝수가 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

DFS로 모든 노드를 한 번씩 방문하므로 O(N)이다.

## 대안적 접근

- **BFS + 리프부터 계산**: 리프 노드부터 시작하여 부모 방향으로 서브트리 크기를 누적하는 방법도 있다.
- **그리디**: 서브트리 크기가 짝수인 간선을 탐욕적으로 제거하는 것이 최적임을 증명할 수 있다.
