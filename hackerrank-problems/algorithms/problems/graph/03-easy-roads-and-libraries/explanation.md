# 도로와 도서관 - 해설

## 접근 방식

1. 도로 비용이 도서관 비용 이상이면, 모든 도시에 도서관을 건설하는 것이 최적이다
2. 그렇지 않으면, 연결 요소를 구한다
3. 각 연결 요소에 도서관 1개 + (요소 크기 - 1)개의 도로를 수리한다
4. 총 비용 = Σ(cLib + (컴포넌트 크기 - 1) × cRoad)

## 핵심 아이디어

그래프의 연결 요소(Connected Component)를 구하는 문제이다. 각 연결 요소 내에서는 스패닝 트리를 구성하면 (크기 - 1)개의 도로만 필요하다. 도로 비용이 도서관 비용보다 비싸면 모든 도시에 도서관을 짓는 것이 더 저렴하다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

BFS/DFS로 연결 요소를 구하는 데 O(N + M)이 소요된다.

## 대안적 접근

- **Union-Find**: 연결 요소를 효율적으로 관리할 수 있다. 경로 압축과 랭크 기반 합치기를 사용하면 거의 O(α(N))에 동작한다.
- **크루스칼 MST**: 최소 스패닝 트리를 구한 뒤 비용을 계산할 수도 있지만, 이 문제에서는 연결 요소 크기만 알면 충분하다.
