# DFS: 그리드에서 연결된 셀 - 해설

## 접근 방식

1. 그리드의 모든 셀을 순회한다
2. 값이 1이고 미방문인 셀을 발견하면 DFS를 시작한다
3. DFS에서 8방향(상하좌우 + 대각선)으로 인접한 1인 셀을 탐색한다
4. 각 DFS에서 방문한 셀 수를 세고, 최댓값을 갱신한다

## 핵심 아이디어

그리드 기반 연결 요소(Connected Component) 문제이다. 8방향 인접을 고려하여 DFS로 연결된 영역의 크기를 구한다. 방문 체크를 통해 같은 셀을 중복 방문하지 않도록 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(N × M) |

각 셀을 최대 한 번 방문하므로 O(N × M)이다. 재귀 호출 스택과 방문 배열에 O(N × M) 공간이 필요하다.

## 대안적 접근

- **BFS**: 큐를 사용하여 같은 결과를 얻을 수 있다. 재귀 깊이 제한이 걱정될 때 유용하다.
- **Union-Find**: 연결 요소를 관리할 수 있지만, 이 문제에서는 DFS/BFS가 더 직관적이다.
