# 알고리즘 (Algorithms)

## 개념

알고리즘은 주어진 문제를 해결하기 위한 체계적인 절차이다. 문제의 특성에 맞는 알고리즘을 선택하면 시간과 공간 복잡도를 크게 줄일 수 있다. 이 문서에서는 코딩 테스트에서 자주 등장하는 4가지 핵심 알고리즘 유형을 다룬다.

### 그리디 (Greedy)

그리디 알고리즘은 매 단계에서 현재 상황에서 가장 좋아 보이는 선택(지역 최적해)을 반복하여 전체 문제의 최적해(전역 최적해)를 구하는 방법이다. 미래를 고려하지 않고 현재 시점에서 최선의 결정을 내린다.

- **탐욕적 선택 속성 (Greedy Choice Property)**: 각 단계의 최적 선택이 전체 최적해로 이어진다
- **최적 부분 구조 (Optimal Substructure)**: 전체 문제의 최적해가 부분 문제의 최적해를 포함한다
- 그리디가 최적해를 보장하려면 위 두 조건이 모두 성립해야 한다

### 동적 프로그래밍 (Dynamic Programming, DP)

동적 프로그래밍은 큰 문제를 작은 부분 문제로 나누어 풀되, 이미 계산한 결과를 저장하여 중복 계산을 피하는 방법이다. 재귀적 구조를 가진 문제에서 부분 문제가 겹칠 때 효과적이다.

- **최적 부분 구조 (Optimal Substructure)**: 큰 문제의 최적해가 작은 부분 문제의 최적해로 구성된다
- **중복 부분 문제 (Overlapping Subproblems)**: 같은 부분 문제가 여러 번 반복하여 등장한다
- **메모이제이션 (Memoization)**: 탑다운 방식으로 재귀 호출 결과를 캐싱한다
- **타뷸레이션 (Tabulation)**: 바텀업 방식으로 작은 문제부터 테이블을 채워나간다

### 그래프 (Graph)

그래프 알고리즘은 정점(Vertex)과 간선(Edge)으로 구성된 그래프 구조에서 탐색, 최단 경로, 연결 요소 등을 구하는 방법이다. 현실 세계의 네트워크, 경로, 관계를 모델링하는 데 널리 사용된다.

- **BFS (Breadth-First Search)**: 너비 우선 탐색. 큐를 사용하여 가까운 노드부터 탐색한다
- **DFS (Depth-First Search)**: 깊이 우선 탐색. 스택(또는 재귀)을 사용하여 깊은 노드부터 탐색한다
- **다익스트라 (Dijkstra)**: 가중치가 있는 그래프에서 단일 출발점 최단 경로를 구한다
- **플로이드-워셜 (Floyd-Warshall)**: 모든 쌍 최단 경로를 구하는 알고리즘이다. 3중 루프로 경유지를 통한 거리를 갱신한다
- **위상 정렬 (Topological Sort)**: DAG(방향 비순환 그래프)에서 선행 관계를 만족하는 정렬 순서를 구한다
- **크루스칼/프림 (Kruskal/Prim)**: 최소 신장 트리(MST)를 구하는 알고리즘이다
- **유니온-파인드 (Union-Find)**: 서로소 집합을 관리하여 연결 요소를 효율적으로 판별한다

### 탐색 (Search)

탐색 알고리즘은 주어진 데이터에서 원하는 값이나 조건을 만족하는 원소를 찾는 방법이다. 정렬된 데이터에서는 이진 탐색으로 O(log N)에 찾을 수 있고, 격자나 그래프에서는 BFS/DFS를 활용한다.

- **이진 탐색 (Binary Search)**: 정렬된 배열에서 탐색 범위를 절반씩 줄여가며 목표값을 찾는다
- **매개변수 탐색 (Parametric Search)**: 최적화 문제를 결정 문제로 변환하여 이진 탐색으로 해결한다
- **완전 탐색 (Exhaustive Search)**: 가능한 모든 경우를 탐색하여 답을 찾는다
- **투 포인터 (Two Pointer)**: 두 개의 포인터를 이동시키며 조건을 만족하는 쌍을 찾는다

## 동작 원리

### 그리디

그리디 알고리즘은 다음 단계를 따른다:

1. 문제를 여러 단계(선택)로 분해한다
2. 각 단계에서 현재 상태를 기준으로 최적의 선택을 한다
3. 선택한 결과를 누적하며 다음 단계로 진행한다
4. 모든 단계를 완료하면 최종 결과를 반환한다

그리디의 핵심은 "되돌아가지 않는다"는 것이다. 한 번 선택하면 그 선택을 번복하지 않는다.

```
예시: 거스름돈 문제 (동전: 500, 100, 50, 10원)
거스름돈: 1260원

단계 1: 500원 × 2 = 1000원 (남은: 260원)
단계 2: 100원 × 2 = 200원  (남은: 60원)
단계 3: 50원  × 1 = 50원   (남은: 10원)
단계 4: 10원  × 1 = 10원   (남은: 0원)

총 동전 수: 2 + 2 + 1 + 1 = 6개
```

### 동적 프로그래밍

DP는 두 가지 방식으로 구현할 수 있다:

**탑다운 (메모이제이션)**:
1. 큰 문제에서 시작하여 재귀적으로 부분 문제를 호출한다
2. 이미 계산한 결과는 메모 테이블에 저장한다
3. 같은 부분 문제가 다시 호출되면 저장된 결과를 반환한다

**바텀업 (타뷸레이션)**:
1. 가장 작은 부분 문제부터 시작한다
2. DP 테이블을 순서대로 채워나간다
3. 최종적으로 원래 문제의 답을 테이블에서 읽는다

```
예시: 피보나치 수열 F(5)

탑다운 호출 트리 (메모이제이션 없이):
  F(5) → F(4) + F(3)
  F(4) → F(3) + F(2)    ← F(3) 중복!
  F(3) → F(2) + F(1)    ← F(2) 중복!

메모이제이션 적용 시:
  F(5) → F(4) + F(3)
  F(4) → F(3) + F(2)
  F(3) → 캐시에서 반환 (이미 계산됨)

바텀업:
  dp[0] = 0
  dp[1] = 1
  dp[2] = dp[0] + dp[1] = 1
  dp[3] = dp[1] + dp[2] = 2
  dp[4] = dp[2] + dp[3] = 3
  dp[5] = dp[3] + dp[4] = 5
```

### 그래프

그래프 탐색의 두 가지 기본 방식:

**BFS (너비 우선 탐색)**:
1. 시작 노드를 큐에 넣고 방문 표시한다
2. 큐에서 노드를 꺼내 인접 노드를 확인한다
3. 방문하지 않은 인접 노드를 큐에 넣고 방문 표시한다
4. 큐가 빌 때까지 반복한다

**DFS (깊이 우선 탐색)**:
1. 시작 노드를 스택에 넣고(또는 재귀 호출) 방문 표시한다
2. 현재 노드의 인접 노드 중 방문하지 않은 노드로 이동한다
3. 더 이상 이동할 수 없으면 이전 노드로 되돌아간다 (백트래킹)
4. 모든 노드를 방문할 때까지 반복한다

```
그래프:
  1 --- 2
  |     |
  3 --- 4 --- 5

BFS (시작: 1): 1 → 2, 3 → 4 → 5
  큐 변화: [1] → [2,3] → [3,4] → [4] → [5] → []

DFS (시작: 1): 1 → 2 → 4 → 3 → 5
  스택 변화: [1] → [2] → [4] → [3] → [5] → []
```

**다익스트라 (최단 경로)**:
1. 시작 노드의 거리를 0, 나머지를 무한대로 초기화한다
2. 우선순위 큐(최소 힙)에서 거리가 가장 짧은 노드를 꺼낸다
3. 해당 노드의 인접 노드에 대해 거리를 갱신한다 (완화, Relaxation)
4. 모든 노드를 처리할 때까지 반복한다

**플로이드-워셜 (모든 쌍 최단 경로)**:

모든 정점 쌍 (i, j) 사이의 최단 경로를 한 번에 구하는 알고리즘이다. 경유지 k를 하나씩 추가하면서 거리를 갱신하는 3중 루프 구조를 사용한다.

1. 거리 행렬 `dist[i][j]`를 초기화한다 (직접 연결된 간선의 가중치, 없으면 ∞, 자기 자신은 0)
2. 경유지 k를 0부터 V-1까지 순회한다 (가장 바깥 루프)
3. 모든 쌍 (i, j)에 대해 `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`로 갱신한다
4. 모든 경유지를 처리하면 `dist[i][j]`에 i→j 최단 거리가 저장된다

```
의사코드:
  // 초기화
  for i = 0 to V-1:
    for j = 0 to V-1:
      if i == j: dist[i][j] = 0
      else if edge(i,j) exists: dist[i][j] = weight(i,j)
      else: dist[i][j] = ∞

  // 3중 루프: 경유지 k를 통한 갱신
  for k = 0 to V-1:        // 경유지
    for i = 0 to V-1:      // 출발지
      for j = 0 to V-1:    // 도착지
        dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

핵심 아이디어: "i에서 j로 직접 가는 것"보다 "i에서 k를 거쳐 j로 가는 것"이 더 짧으면 갱신한다. 경유지를 하나씩 늘려가며 모든 쌍의 최단 거리를 점진적으로 완성한다.

```
예시: 4개 정점 가중치 방향 그래프

초기 거리 행렬:
     0    1    2    3
0 [  0,   3,   ∞,   7 ]
1 [  ∞,   0,   2,   ∞ ]
2 [  5,   ∞,   0,   1 ]
3 [  2,   ∞,   ∞,   0 ]

k=0 (정점 0을 경유지로):
  dist[1][3] = min(∞, dist[1][0]+dist[0][3]) = min(∞, ∞+7) = ∞ (변화 없음)
  dist[2][1] = min(∞, dist[2][0]+dist[0][1]) = min(∞, 5+3) = 8
  dist[3][1] = min(∞, dist[3][0]+dist[0][1]) = min(∞, 2+3) = 5

k=1 (정점 1을 경유지로):
  dist[0][2] = min(∞, dist[0][1]+dist[1][2]) = min(∞, 3+2) = 5
  dist[3][2] = min(∞, dist[3][1]+dist[1][2]) = min(∞, 5+2) = 7

k=2 (정점 2를 경유지로):
  dist[0][3] = min(7, dist[0][2]+dist[2][3]) = min(7, 5+1) = 6
  dist[1][3] = min(∞, dist[1][2]+dist[2][3]) = min(∞, 2+1) = 3
  dist[1][0] = min(∞, dist[1][2]+dist[2][0]) = min(∞, 2+5) = 7

k=3 (정점 3을 경유지로):
  dist[2][1] = min(8, dist[2][3]+dist[3][1]) = min(8, 1+5) = 6

최종 거리 행렬:
     0    1    2    3
0 [  0,   3,   5,   6 ]
1 [  7,   0,   2,   3 ]
2 [  3,   6,   0,   1 ]
3 [  2,   5,   7,   0 ]
```

**위상 정렬 (Topological Sort)**:

DAG(Directed Acyclic Graph, 방향 비순환 그래프)에서 모든 간선 (u, v)에 대해 u가 v보다 앞에 오도록 정점을 나열하는 알고리즘이다. 선행 관계가 있는 작업의 실행 순서를 결정할 때 사용한다.

**Kahn's Algorithm (BFS + 진입차수)**:
1. 모든 정점의 진입차수(in-degree)를 계산한다
2. 진입차수가 0인 정점을 큐에 넣는다 (선행 조건이 없는 작업)
3. 큐에서 정점을 꺼내 결과 리스트에 추가한다
4. 해당 정점에서 나가는 간선을 제거하고, 연결된 정점의 진입차수를 1 감소시킨다
5. 진입차수가 0이 된 정점을 큐에 넣는다
6. 큐가 빌 때까지 반복한다
7. 결과 리스트의 크기가 정점 수와 같으면 위상 정렬 성공, 아니면 사이클이 존재한다

```
의사코드 (Kahn's Algorithm):
  // 진입차수 계산
  for each vertex v:
    inDegree[v] = 0
  for each edge (u, v):
    inDegree[v]++

  // 진입차수 0인 정점을 큐에 삽입
  queue = []
  for each vertex v:
    if inDegree[v] == 0:
      queue.push(v)

  result = []
  while queue is not empty:
    u = queue.pop()
    result.append(u)
    for each neighbor v of u:
      inDegree[v]--
      if inDegree[v] == 0:
        queue.push(v)

  if len(result) != V:
    // 사이클 존재 → 위상 정렬 불가
```

```
예시: 과목 선수과목 관계

  과목: A, B, C, D, E, F
  선수과목: A→B, A→C, B→D, C→D, D→E, C→F

  그래프:
    A → B → D → E
    A → C → D
         C → F

  진입차수: A=0, B=1, C=1, D=2, E=1, F=1

단계 1: 진입차수 0 → 큐 = [A]
  꺼냄: A → 결과 = [A]
  A→B: inDegree[B] = 0 → 큐에 추가
  A→C: inDegree[C] = 0 → 큐에 추가
  큐 = [B, C]

단계 2: 꺼냄: B → 결과 = [A, B]
  B→D: inDegree[D] = 1
  큐 = [C]

단계 3: 꺼냄: C → 결과 = [A, B, C]
  C→D: inDegree[D] = 0 → 큐에 추가
  C→F: inDegree[F] = 0 → 큐에 추가
  큐 = [D, F]

단계 4: 꺼냄: D → 결과 = [A, B, C, D]
  D→E: inDegree[E] = 0 → 큐에 추가
  큐 = [F, E]

단계 5: 꺼냄: F → 결과 = [A, B, C, D, F]
  큐 = [E]

단계 6: 꺼냄: E → 결과 = [A, B, C, D, F, E]
  큐 = [] → 종료

위상 정렬 결과: A → B → C → D → F → E
(결과 크기 6 == 정점 수 6 → 사이클 없음, 정렬 성공)
```

### 탐색

**이진 탐색**:
1. 정렬된 배열에서 왼쪽(lo)과 오른쪽(hi) 경계를 설정한다
2. 중간값(mid)을 계산하여 목표값과 비교한다
3. 목표값이 중간값보다 작으면 오른쪽 경계를 줄이고, 크면 왼쪽 경계를 늘린다
4. 목표값을 찾거나 범위가 비면 종료한다

```
예시: 정렬된 배열 [1, 3, 5, 7, 9, 11, 13]에서 7 찾기

단계 1: lo=0, hi=6, mid=3 → arr[3]=7 → 찾음!

예시: 같은 배열에서 6 찾기

단계 1: lo=0, hi=6, mid=3 → arr[3]=7 → 6 < 7 → hi=2
단계 2: lo=0, hi=2, mid=1 → arr[1]=3 → 6 > 3 → lo=2
단계 3: lo=2, hi=2, mid=2 → arr[2]=5 → 6 > 5 → lo=3
단계 4: lo=3 > hi=2 → 종료 (찾지 못함)
```

**매개변수 탐색**:
1. 답이 될 수 있는 범위를 설정한다 (lo, hi)
2. 중간값(mid)이 조건을 만족하는지 판별한다
3. 조건을 만족하면 더 좋은 답을 찾기 위해 범위를 조정한다
4. 범위가 수렴할 때까지 반복한다

### 자주 사용되는 기법

- **정렬 후 그리디**: 데이터를 특정 기준으로 정렬한 뒤 그리디 선택을 적용한다
- **활동 선택 문제 (Activity Selection)**: 종료 시간 기준 정렬 후 겹치지 않는 활동을 최대한 선택한다
- **DP 점화식 도출**: 문제의 상태를 정의하고, 상태 간 전이 관계(점화식)를 수립한다
- **DP 공간 최적화**: 이전 행만 참조하는 경우 2차원 테이블을 1차원으로 줄인다
- **인접 리스트**: 그래프를 인접 리스트로 표현하여 메모리를 절약한다
- **위상 정렬 (Topological Sort)**: DAG에서 선행 관계를 만족하는 순서를 구한다
- **Lower Bound / Upper Bound**: 이진 탐색의 변형으로 특정 값 이상/초과인 첫 위치를 찾는다
- **유니온-파인드 (Union-Find)**: 경로 압축과 랭크 기반 합치기로 거의 O(1)에 집합 연산을 수행한다

### 예시 1: 그리디 — 활동 선택 문제

입력: 활동 목록 (시작, 종료) = [(1,4), (3,5), (0,6), (5,7), (3,9), (5,9), (6,10), (8,11), (8,12), (2,14)]

```
1단계: 종료 시간 기준 정렬
  (1,4), (3,5), (0,6), (5,7), (3,9), (5,9), (6,10), (8,11), (8,12), (2,14)

2단계: 첫 번째 활동 선택
  선택: (1,4) ← 종료 시간이 가장 빠름
  현재 종료 시간: 4

3단계: 다음 활동 검토
  (3,5) → 시작 3 < 종료 4 → 겹침, 건너뜀
  (0,6) → 시작 0 < 종료 4 → 겹침, 건너뜀
  (5,7) → 시작 5 ≥ 종료 4 → 선택!
  현재 종료 시간: 7

4단계: 계속 검토
  (3,9) → 시작 3 < 종료 7 → 건너뜀
  (5,9) → 시작 5 < 종료 7 → 건너뜀
  (6,10) → 시작 6 < 종료 7 → 건너뜀
  (8,11) → 시작 8 ≥ 종료 7 → 선택!
  현재 종료 시간: 11

5단계: 나머지 검토
  (8,12) → 시작 8 < 종료 11 → 건너뜀
  (2,14) → 시작 2 < 종료 11 → 건너뜀

최종 선택: (1,4), (5,7), (8,11) → 최대 3개 활동
```

### 예시 2: DP — 0/1 배낭 문제

입력: 용량 W=7, 물건 = [(무게=1, 가치=1), (무게=3, 가치=4), (무게=4, 가치=5), (무게=5, 가치=7)]

```
DP 테이블 (dp[i][w] = i번째 물건까지 고려, 용량 w일 때 최대 가치):

        w=0  w=1  w=2  w=3  w=4  w=5  w=6  w=7
물건0:   0    1    1    1    1    1    1    1
물건1:   0    1    1    4    5    5    5    5
물건2:   0    1    1    4    5    6    6    9
물건3:   0    1    1    4    5    7    8    9

점화식: dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])

물건2, w=7 계산:
  넣지 않는 경우: dp[1][7] = 5
  넣는 경우: dp[1][7-4] + 5 = dp[1][3] + 5 = 4 + 5 = 9
  dp[2][7] = max(5, 9) = 9

최종 답: dp[3][7] = 9
역추적: 물건1(가치4) + 물건2(가치5) = 9
```

### 예시 3: 그래프 — BFS 최단 경로

입력: 그래프 (인접 리스트), 시작 노드 = 0

```
그래프:
  0 --- 1 --- 3
  |     |
  2 --- 4 --- 5

인접 리스트:
  0: [1, 2]
  1: [0, 3, 4]
  2: [0, 4]
  3: [1]
  4: [1, 2, 5]
  5: [4]

BFS 실행 (시작: 0):

단계 0: 큐 = [0], 방문 = {0}, 거리 = {0:0}

단계 1: 꺼냄 = 0
  인접: 1 (미방문) → 큐에 추가, 거리[1] = 1
  인접: 2 (미방문) → 큐에 추가, 거리[2] = 1
  큐 = [1, 2], 방문 = {0, 1, 2}

단계 2: 꺼냄 = 1
  인접: 0 (방문됨) → 건너뜀
  인접: 3 (미방문) → 큐에 추가, 거리[3] = 2
  인접: 4 (미방문) → 큐에 추가, 거리[4] = 2
  큐 = [2, 3, 4], 방문 = {0, 1, 2, 3, 4}

단계 3: 꺼냄 = 2
  인접: 0 (방문됨) → 건너뜀
  인접: 4 (방문됨) → 건너뜀
  큐 = [3, 4]

단계 4: 꺼냄 = 3
  인접: 1 (방문됨) → 건너뜀
  큐 = [4]

단계 5: 꺼냄 = 4
  인접: 1 (방문됨) → 건너뜀
  인접: 2 (방문됨) → 건너뜀
  인접: 5 (미방문) → 큐에 추가, 거리[5] = 3
  큐 = [5], 방문 = {0, 1, 2, 3, 4, 5}

단계 6: 꺼냄 = 5
  인접: 4 (방문됨) → 건너뜀
  큐 = [] → 종료

최단 거리: {0:0, 1:1, 2:1, 3:2, 4:2, 5:3}
```

### 예시 4: 탐색 — 매개변수 탐색 (나무 자르기)

문제: 높이 H로 톱을 설정하면 H보다 높은 나무만 잘린다. 최소 M미터의 나무를 가져가려면 H의 최댓값은?

입력: 나무 높이 = [20, 15, 10, 17], 필요한 나무 M = 7

```
탐색 범위: lo=0, hi=20 (가장 높은 나무)

단계 1: mid = 10
  잘리는 양: (20-10) + (15-10) + (10-10) + (17-10) = 10+5+0+7 = 22
  22 ≥ 7 → 조건 만족, 더 높은 H 시도
  lo = 11

단계 2: mid = 15
  잘리는 양: (20-15) + (15-15) + 0 + (17-15) = 5+0+0+2 = 7
  7 ≥ 7 → 조건 만족, 더 높은 H 시도
  lo = 16

단계 3: mid = 18
  잘리는 양: (20-18) + 0 + 0 + 0 = 2
  2 < 7 → 조건 불만족, 더 낮은 H 시도
  hi = 17

단계 4: mid = 16
  잘리는 양: (20-16) + 0 + 0 + (17-16) = 4+0+0+1 = 5
  5 < 7 → 조건 불만족
  hi = 15

단계 5: lo=16 > hi=15 → 종료

최적 H = 15 (최소 7미터를 가져갈 수 있는 최대 높이)
```

## 실전 팁

### 활용 노하우

- **그리디 정당성 검증**: 그리디를 적용하기 전에 "지역 최적이 전역 최적을 보장하는가?"를 반드시 확인하자. 반례를 하나라도 찾으면 그리디가 아닌 DP를 고려해야 한다
- **DP 상태 정의가 핵심**: DP 문제를 풀 때 가장 중요한 것은 상태(state)를 어떻게 정의하느냐이다. `dp[i]`가 무엇을 의미하는지 명확히 정의한 뒤 점화식을 세우자
- **DP 테이블 초기화 주의**: `dp[0]`이나 `dp[0][0]`의 초기값을 잘못 설정하면 전체 결과가 틀어진다. 기저 조건(base case)을 먼저 정확히 설정하자
- **그래프 표현 방식 선택**: 간선이 적으면 인접 리스트, 정점이 적고 간선이 많으면 인접 행렬을 사용한다. 대부분의 코딩 테스트에서는 인접 리스트가 효율적이다
- **BFS는 가중치 없는 최단 경로**: 모든 간선의 가중치가 동일하면 BFS로 최단 경로를 구할 수 있다. 가중치가 다르면 다익스트라를 사용하자
- **이진 탐색의 경계 조건**: `lo <= hi`인지 `lo < hi`인지, `mid`를 포함하여 범위를 줄이는지 제외하는지에 따라 결과가 달라진다. 무한 루프에 빠지지 않도록 경계 조건을 신중히 설정하자
- **매개변수 탐색 패턴**: "최솟값의 최댓값" 또는 "최댓값의 최솟값"이라는 문구가 보이면 매개변수 탐색을 떠올리자
- **유니온-파인드 최적화**: 경로 압축(Path Compression)과 랭크 기반 합치기(Union by Rank)를 함께 적용하면 거의 O(1)에 연산이 가능하다
- **플로이드-워셜 사용 조건**: 정점 수가 적을 때(V ≤ 500) 모든 쌍 최단 경로를 구해야 하면 플로이드-워셜이 가장 간단하다. 경유지 k가 가장 바깥 루프여야 한다는 점을 반드시 기억하자
- **위상 정렬과 사이클 감지**: Kahn's Algorithm에서 결과 리스트의 크기가 정점 수보다 작으면 사이클이 존재한다. 사이클 감지와 위상 정렬을 동시에 수행할 수 있다

### 자주 하는 실수

- **그리디 오적용**: 그리디가 최적해를 보장하지 않는 문제에 그리디를 적용하는 실수. 예를 들어 0/1 배낭 문제는 그리디로 풀 수 없다 (분할 가능 배낭만 가능)
- **DP 점화식 방향 오류**: 바텀업 DP에서 테이블을 채우는 순서가 잘못되면 아직 계산되지 않은 값을 참조하게 된다. 점화식의 의존 관계를 확인하고 올바른 순서로 채우자
- **DP 인덱스 범위 초과**: `dp[i-1]`이나 `dp[i][j-weight]`에서 인덱스가 음수가 되지 않도록 경계 검사를 해야 한다
- **그래프 방문 체크 누락**: BFS/DFS에서 방문 배열을 사용하지 않으면 무한 루프에 빠진다. 특히 양방향 그래프에서 주의하자
- **다익스트라에 음수 가중치**: 다익스트라는 음수 가중치 간선이 있으면 올바른 결과를 보장하지 않는다. 음수 가중치가 있으면 벨만-포드를 사용하자
- **이진 탐색 무한 루프**: `lo`와 `hi`가 수렴하지 않는 경우. `mid = (lo + hi) / 2`에서 `lo = mid`로 설정하면 `lo == mid`일 때 무한 루프가 발생할 수 있다. `lo = mid + 1`로 설정하자
- **정수 오버플로**: `mid = (lo + hi) / 2`에서 `lo + hi`가 오버플로될 수 있다. `mid = lo + (hi - lo) / 2`로 계산하자

### 엣지 케이스 주의사항

- **빈 입력**: 배열이 비어있거나 그래프에 노드가 없는 경우를 처리해야 한다
- **원소가 1개**: 배열에 원소가 1개일 때 이진 탐색, 투 포인터 등이 올바르게 동작하는지 확인하자
- **모든 값이 동일**: 배열의 모든 원소가 같을 때 정렬, 이진 탐색, 그리디 등의 동작을 확인하자
- **연결되지 않은 그래프**: 그래프가 여러 연결 요소로 나뉘어 있을 때 모든 노드를 방문하려면 각 연결 요소에서 BFS/DFS를 시작해야 한다
- **자기 자신으로의 간선 (Self-loop)**: 그래프에 자기 자신으로의 간선이 있을 때 무한 루프에 빠지지 않도록 주의하자
- **DP 값이 매우 큰 경우**: 경우의 수를 구하는 문제에서 결과가 매우 커질 수 있다. 모듈러 연산(`% 1000000007`)을 적용해야 하는지 확인하자
- **음수 값**: 배열에 음수가 포함된 경우 그리디나 이진 탐색의 전제 조건이 깨질 수 있다

## 관련 알고리즘 비교

### 알고리즘 유형별 비교

| 알고리즘 | 핵심 특징 | 시간 복잡도 | 장점 | 단점 | 적합한 상황 |
| --- | --- | --- | --- | --- | --- |
| 그리디 | 매 단계 최선의 선택 | O(N log N) | 구현 간단, 빠른 실행 | 최적해 보장 안 될 수 있음 | 지역 최적 = 전역 최적인 경우 |
| DP | 부분 문제 결과 저장 | O(N × M) | 최적해 보장 | 메모리 사용량 큼 | 중복 부분 문제가 있는 경우 |
| BFS | 너비 우선 탐색 | O(V + E) | 최단 경로 보장 (무가중치) | 메모리 사용량 큼 | 최단 거리, 레벨별 탐색 |
| DFS | 깊이 우선 탐색 | O(V + E) | 메모리 효율적 | 최단 경로 보장 안 됨 | 연결 요소, 사이클 감지 |
| 다익스트라 | 가중치 최단 경로 | O((V+E) log V) | 가중치 그래프 최단 경로 | 음수 가중치 불가 | 양수 가중치 최단 경로 |
| 플로이드-워셜 | 모든 쌍 최단 경로 | O(V³) | 구현 간단, 음수 가중치 허용 | V가 크면 느림 | 정점 수가 적은 모든 쌍 최단 경로 |
| 위상 정렬 | DAG 선행 관계 정렬 | O(V + E) | 사이클 감지 가능 | DAG에서만 가능 | 작업 순서, 의존성 해결 |
| 이진 탐색 | 범위 절반씩 축소 | O(log N) | 매우 빠른 탐색 | 정렬 필요 | 정렬된 데이터에서 탐색 |
| 완전 탐색 | 모든 경우 시도 | O(2^N) ~ O(N!) | 항상 정답 보장 | 매우 느림 | 입력이 작은 경우 |

### 문제 유형별 알고리즘 선택 기준

| 문제 키워드 | 추천 알고리즘 | 이유 |
| --- | --- | --- |
| "최소 동전 수", "최소 비용" (특수 조건) | 그리디 | 탐욕적 선택이 최적해를 보장하는 경우 |
| "경우의 수", "최솟값/최댓값" (일반) | DP | 중복 부분 문제를 효율적으로 해결 |
| "최단 거리" (무가중치) | BFS | 가중치 없는 그래프에서 최단 경로 보장 |
| "최단 거리" (가중치) | 다익스트라 | 양수 가중치 그래프에서 최단 경로 |
| "모든 쌍 최단 거리" (V ≤ 500) | 플로이드-워셜 | 3중 루프로 모든 쌍 최단 경로 |
| "작업 순서", "선수과목", "의존성" | 위상 정렬 | DAG에서 선행 관계를 만족하는 정렬 |
| "연결 요소", "사이클" | DFS / 유니온-파인드 | 그래프 구조 분석에 적합 |
| "최소 신장 트리" | 크루스칼 / 프림 | MST 전용 알고리즘 |
| "정렬된 배열에서 찾기" | 이진 탐색 | O(log N) 탐색 |
| "최솟값의 최댓값" | 매개변수 탐색 | 최적화 → 결정 문제 변환 |
| "합이 K인 쌍" | 투 포인터 | 정렬 후 O(N) 탐색 |
| "모든 경우" (N ≤ 20) | 완전 탐색 / 비트마스크 | 입력이 작아 전수 조사 가능 |

### 그리디 vs DP 선택 기준

```
문제 분석 시작
  │
  ├─ 탐욕적 선택 속성이 성립하는가?
  │   ├─ 예 → 그리디 (더 빠르고 간단)
  │   └─ 아니오 / 불확실 → 아래로
  │
  ├─ 중복 부분 문제가 있는가?
  │   ├─ 예 → DP
  │   └─ 아니오 → 분할 정복 또는 완전 탐색
  │
  └─ 반례를 찾을 수 있는가?
      ├─ 예 → 그리디 불가, DP 사용
      └─ 아니오 → 그리디 시도 후 검증
```

### BFS vs DFS 선택 기준

| 기준 | BFS | DFS |
| --- | --- | --- |
| 최단 경로 | 보장 (무가중치) | 보장하지 않음 |
| 메모리 | O(너비) — 넓은 그래프에서 큼 | O(깊이) — 깊은 그래프에서 큼 |
| 구현 | 큐 사용 | 재귀 또는 스택 |
| 적합한 문제 | 최단 거리, 레벨별 탐색 | 연결 요소, 백트래킹, 위상 정렬 |
| 무한 그래프 | 가까운 해를 먼저 찾음 | 무한 루프 위험 |

**선택 기준 요약**:
- 매 단계 최선의 선택으로 충분하면 → 그리디
- 이전 선택이 이후에 영향을 주고 중복 계산이 있으면 → DP
- 가중치 없는 그래프에서 최단 거리를 구하면 → BFS
- 연결 요소를 찾거나 모든 경로를 탐색하면 → DFS
- 가중치 있는 그래프에서 최단 거리를 구하면 → 다익스트라
- 정렬된 데이터에서 빠르게 찾으면 → 이진 탐색
- 최적화 문제를 결정 문제로 바꿀 수 있으면 → 매개변수 탐색
- 입력이 작고 모든 경우를 봐야 하면 → 완전 탐색
