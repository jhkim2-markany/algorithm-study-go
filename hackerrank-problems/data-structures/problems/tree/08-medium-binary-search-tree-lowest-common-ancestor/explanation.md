# 이진 탐색 트리 최소 공통 조상 - 해설

## 접근 방식

1. 루트부터 시작한다
2. 두 값이 모두 현재 노드보다 작으면 왼쪽으로 이동한다
3. 두 값이 모두 현재 노드보다 크면 오른쪽으로 이동한다
4. 두 값이 현재 노드의 양쪽에 걸치면 현재 노드가 LCA이다
5. 하나의 값이 현재 노드와 같아도 현재 노드가 LCA이다

## 핵심 아이디어

BST의 정렬 성질을 활용한다. 현재 노드의 값이 v1과 v2 사이에 있으면(또는 같으면), 두 노드는 현재 노드의 서로 다른 서브트리에 위치하므로 현재 노드가 LCA가 된다. 두 값이 같은 쪽에 있으면 해당 방향으로 계속 탐색한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(H) |
| 공간 복잡도 | O(H) |

루트에서 LCA까지 한 경로만 탐색하므로 시간 복잡도는 트리의 높이 H에 비례한다. 재귀 호출 스택도 O(H)이다.

## 대안적 접근

- **반복적 구현**: while 루프를 사용하면 공간 복잡도를 O(1)로 줄일 수 있다.
- **일반 이진 트리 LCA**: BST가 아닌 일반 이진 트리에서는 양쪽 서브트리를 모두 탐색해야 하므로 O(N) 시간이 필요하다.
