# 이진 트리 후위 순회 - 해설

## 접근 방식

1. 루트 노드부터 시작한다
2. 왼쪽 서브트리를 재귀적으로 후위 순회한다
3. 오른쪽 서브트리를 재귀적으로 후위 순회한다
4. 현재 노드의 데이터를 출력한다
5. 노드가 nil이면 반환한다

## 핵심 아이디어

후위 순회(Postorder)는 "왼쪽 → 오른쪽 → 루트" 순서로 방문하는 트리 순회 방법이다. 자식 노드를 모두 처리한 후에 현재 노드를 처리하므로, 트리 삭제나 후위 표기식 변환 등에 활용된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(H) |

N개의 노드를 한 번씩 방문하므로 시간 복잡도는 O(N)이다. 재귀 호출 스택의 깊이는 트리의 높이 H에 비례하므로 공간 복잡도는 O(H)이다.

## 대안적 접근

- **두 개의 스택을 이용한 반복적 순회**: 스택 두 개를 사용하여 재귀 없이 후위 순회를 구현할 수 있다.
- **하나의 스택과 플래그**: 방문 여부를 추적하는 플래그와 함께 스택 하나로도 구현 가능하다.
