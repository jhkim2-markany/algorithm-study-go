# 이것은 이진 탐색 트리인가? - 해설

## 접근 방식

1. 각 노드에 허용되는 값의 범위(min, max)를 전달한다
2. 루트는 (-∞, +∞) 범위를 가진다
3. 왼쪽 자식은 (min, 현재 노드 값) 범위를 가진다
4. 오른쪽 자식은 (현재 노드 값, max) 범위를 가진다
5. 노드의 값이 범위를 벗어나면 유효하지 않다

## 핵심 아이디어

단순히 왼쪽 자식 < 루트 < 오른쪽 자식만 확인하면 안 된다. 왼쪽 서브트리의 모든 노드가 루트보다 작아야 하므로, 각 노드에 허용 범위를 전달하여 검증해야 한다. 재귀적으로 범위를 좁혀가며 모든 노드가 올바른 범위에 있는지 확인한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(H) |

모든 노드를 한 번씩 방문하므로 시간 복잡도는 O(N)이다. 재귀 호출 스택의 깊이는 트리의 높이 H에 비례한다.

## 대안적 접근

- **중위 순회 검증**: BST를 중위 순회하면 오름차순이 되어야 한다. 이전 방문 노드의 값을 추적하면서 현재 값이 항상 더 큰지 확인한다.
- **반복적 범위 검증**: 스택을 사용하여 반복적으로 범위를 검증할 수 있다.
