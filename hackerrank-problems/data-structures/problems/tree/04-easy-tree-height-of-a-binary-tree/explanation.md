# 이진 트리의 높이 - 해설

## 접근 방식

1. 루트 노드부터 시작한다
2. 노드가 nil이면 -1을 반환한다
3. 왼쪽 서브트리의 높이를 재귀적으로 구한다
4. 오른쪽 서브트리의 높이를 재귀적으로 구한다
5. 두 높이 중 큰 값에 1을 더하여 반환한다

## 핵심 아이디어

트리의 높이는 루트에서 가장 먼 리프까지의 간선 수이다. 재귀적으로 왼쪽과 오른쪽 서브트리의 높이를 구한 후, 더 큰 값에 1을 더하면 현재 노드를 루트로 하는 서브트리의 높이가 된다. nil 노드의 높이를 -1로 정의하면 리프 노드의 높이가 0이 되어 정의와 일치한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(H) |

모든 노드를 한 번씩 방문하므로 시간 복잡도는 O(N)이다. 재귀 호출 스택의 깊이는 트리의 높이 H에 비례한다.

## 대안적 접근

- **BFS(레벨 순회)**: 큐를 사용하여 레벨별로 순회하면서 레벨 수를 세면 높이를 구할 수 있다.
- **반복적 DFS**: 명시적 스택과 깊이 정보를 함께 저장하여 최대 깊이를 추적할 수 있다.
