# 이진 탐색 트리 삽입 - 해설

## 접근 방식

1. 루트부터 시작하여 삽입할 위치를 찾는다
2. 삽입할 값이 현재 노드보다 작으면 왼쪽으로 이동한다
3. 삽입할 값이 현재 노드보다 크면 오른쪽으로 이동한다
4. nil 위치를 찾으면 새 노드를 생성하여 삽입한다
5. 재귀적으로 부모-자식 관계를 유지하며 루트를 반환한다

## 핵심 아이디어

BST 삽입은 탐색과 동일한 경로를 따라가다가 빈 위치에 새 노드를 추가하는 연산이다. 재귀적 구현에서는 nil 노드에 도달하면 새 노드를 생성하여 반환하고, 부모 노드가 이를 자식으로 연결한다. 기존 노드의 구조는 변경되지 않는다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(H) |
| 공간 복잡도 | O(H) |

삽입 위치를 찾기 위해 루트에서 리프까지 한 경로만 탐색하므로 시간 복잡도는 트리의 높이 H에 비례한다. 균형 트리에서 H = O(log N), 편향 트리에서 H = O(N)이다.

## 대안적 접근

- **반복적 삽입**: while 루프를 사용하여 부모 노드를 추적하면서 삽입 위치를 찾을 수 있다. 공간 복잡도가 O(1)이 된다.
- **자가 균형 트리**: AVL 트리나 레드-블랙 트리를 사용하면 항상 O(log N) 시간에 삽입할 수 있다.
