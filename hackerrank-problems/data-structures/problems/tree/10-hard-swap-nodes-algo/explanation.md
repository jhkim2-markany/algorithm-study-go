# 노드 교환 알고리즘 - 해설

## 접근 방식

1. 인덱스 기반 입력으로 이진 트리를 구성한다
2. 각 쿼리 K에 대해 깊이가 K의 배수인 모든 노드를 찾는다
3. 해당 노드들의 왼쪽과 오른쪽 자식을 교환한다
4. 교환 후 중위 순회를 수행하여 결과를 출력한다
5. 교환은 누적되므로 이전 교환 결과 위에 새 교환이 적용된다

## 핵심 아이디어

트리를 BFS 또는 DFS로 순회하면서 각 노드의 깊이를 추적한다. 깊이가 K의 배수(K, 2K, 3K, ...)인 노드에서 왼쪽과 오른쪽 자식 포인터를 교환한다. 교환은 실제로 트리 구조를 변경하므로, 다음 쿼리는 변경된 트리에 적용된다.

인덱스 기반 트리 구성에서는 배열을 사용하여 각 인덱스에 해당하는 노드를 저장하고, 자식 관계를 설정한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(T × N) |
| 공간 복잡도 | O(N) |

각 쿼리마다 모든 노드를 순회하여 깊이를 확인하고 교환하므로 O(N), T개의 쿼리에 대해 O(T × N)이다. 트리 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **깊이별 노드 그룹화**: 트리 구성 시 각 깊이별로 노드를 미리 그룹화해두면, 쿼리 처리 시 해당 깊이의 노드만 빠르게 접근할 수 있다.
- **BFS 레벨 순회**: 레벨별로 순회하면서 교환 대상 레벨에서만 교환을 수행할 수 있다.
