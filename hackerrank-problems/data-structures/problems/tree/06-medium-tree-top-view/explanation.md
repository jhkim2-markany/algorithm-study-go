# 이진 트리 탑 뷰 - 해설

## 접근 방식

1. BFS로 트리를 레벨 순서로 순회한다
2. 각 노드에 수평 거리(horizontal distance)를 부여한다
3. 루트의 수평 거리는 0, 왼쪽 자식은 -1, 오른쪽 자식은 +1
4. 각 수평 거리에서 처음 방문한 노드만 맵에 저장한다
5. 수평 거리를 정렬하여 왼쪽에서 오른쪽 순서로 출력한다

## 핵심 아이디어

탑 뷰는 각 수직선(수평 거리)에서 가장 위에 있는 노드를 선택하는 문제이다. BFS를 사용하면 레벨이 낮은(위에 있는) 노드가 먼저 방문되므로, 각 수평 거리에서 처음 방문한 노드가 탑 뷰에 해당한다. 맵을 사용하여 이미 방문한 수평 거리를 추적한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

BFS로 N개의 노드를 방문하는 데 O(N), 수평 거리를 정렬하는 데 O(N log N)이 소요된다. 큐와 맵에 최대 N개의 노드를 저장하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **DFS + 레벨 추적**: DFS로 순회하면서 각 수평 거리에서 가장 작은 레벨의 노드를 저장할 수 있다.
- **TreeMap 사용**: 정렬된 맵을 사용하면 별도의 정렬 없이 순서대로 출력할 수 있다.
