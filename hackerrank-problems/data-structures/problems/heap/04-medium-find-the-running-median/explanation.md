# 실시간 중앙값 찾기 - 해설

## 접근 방식

1. 두 개의 힙을 사용한다: 최대 힙(왼쪽 절반)과 최소 힙(오른쪽 절반)
2. 새 원소가 들어오면 적절한 힙에 삽입한다
3. 두 힙의 크기 차이가 1 이하가 되도록 균형을 맞춘다
4. 중앙값은 두 힙의 루트에서 계산한다

## 핵심 아이디어

두 개의 힙(최대 힙 + 최소 힙)을 사용하여 수열을 반으로 나눈다. 최대 힙에는 작은 절반, 최소 힙에는 큰 절반을 저장한다. 이렇게 하면 중앙값을 O(1)에 조회하고, 새 원소 삽입을 O(log N)에 처리할 수 있다.

- 최대 힙의 루트: 작은 절반의 최댓값
- 최소 힙의 루트: 큰 절반의 최솟값
- 두 힙의 크기가 같으면: 중앙값 = (최대 힙 루트 + 최소 힙 루트) / 2
- 크기가 다르면: 더 큰 힙의 루트가 중앙값

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 원소 삽입 시 힙 연산에 O(log N)이 걸리므로 전체 O(N log N)이다. N개의 원소를 저장하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **매번 정렬**: 매번 정렬하면 O(N^2 log N)으로 비효율적이다.
- **균형 이진 탐색 트리**: AVL 트리나 레드-블랙 트리를 사용하면 O(N log N)에 가능하지만 구현이 복잡하다.
- **정렬된 배열 유지**: 이진 탐색으로 삽입 위치를 찾으면 O(N^2)이다.
