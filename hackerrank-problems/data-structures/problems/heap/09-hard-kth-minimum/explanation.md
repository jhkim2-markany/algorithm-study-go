# 구간 K번째 최솟값 - 해설

## 접근 방식

1. 머지 소트 트리(Merge Sort Tree)를 구성한다
2. 각 쿼리에 대해 값에 대한 이분 탐색을 수행한다
3. 후보 값에 대해 구간 내 해당 값 이하인 원소 수를 센다
4. 정확히 k개 이상이 되는 최소 값이 답이다

## 핵심 아이디어

머지 소트 트리는 세그먼트 트리의 각 노드에 해당 구간의 정렬된 배열을 저장하는 자료구조이다. 이를 이용하면 구간 내 특정 값 이하인 원소의 개수를 O(log^2 N)에 계산할 수 있다.

k번째 원소를 찾기 위해 정렬된 전체 값 목록에서 이분 탐색한다. 각 후보 값에 대해 구간 [l, r]에서 해당 값 이하인 원소 수를 세고, 그 수가 k 이상이 되는 최소 값을 찾는다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N + Q log^3 N) |
| 공간 복잡도 | O(N log N) |

머지 소트 트리 구성에 O(N log N), 각 쿼리마다 값 이분 탐색 O(log N) × 구간 카운트 O(log^2 N)이 걸린다.

## 대안적 접근

- **퍼시스턴트 세그먼트 트리**: 좌표 압축 후 퍼시스턴트 세그먼트 트리를 사용하면 각 쿼리를 O(log N)에 처리할 수 있다.
- **Mo's 알고리즘 + 힙**: 오프라인 쿼리 처리로 O(N√N log N)에 가능하다.
- **웨이블릿 트리**: 구간 k번째 원소를 O(log N)에 찾을 수 있지만 구현이 복잡하다.
