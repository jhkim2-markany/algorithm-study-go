# QHEAP1 - 해설

## 접근 방식

1. 최소 힙을 구현한다
2. 쿼리 타입에 따라 처리한다:
   - 타입 1: 힙에 원소를 추가한다
   - 타입 2: 힙에서 특정 원소를 삭제한다 (lazy deletion 또는 직접 삭제)
   - 타입 3: 힙의 최솟값을 출력한다
3. 삭제 시 삭제 표시를 해두고, 최솟값 조회 시 삭제된 원소를 건너뛴다

## 핵심 아이디어

Go의 `container/heap` 패키지를 사용하여 최소 힙을 구현한다. 특정 원소 삭제가 필요하므로, 삭제할 원소를 맵에 기록해두고(lazy deletion) 최솟값 조회 시 삭제된 원소를 힙에서 제거하는 방식을 사용한다. 이렇게 하면 삭제 연산을 O(log N)에 처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q log Q) |
| 공간 복잡도 | O(Q) |

각 쿼리마다 힙 연산에 O(log Q)가 걸리므로 전체 O(Q log Q)이다. 힙에 최대 Q개의 원소가 저장될 수 있으므로 공간 복잡도는 O(Q)이다.

## 대안적 접근

- **직접 삭제**: 힙에서 원소를 찾아 직접 삭제하면 O(N)이 걸린다. 작은 입력에서는 가능하지만 비효율적이다.
- **균형 이진 탐색 트리**: 삽입, 삭제, 최솟값 조회 모두 O(log N)에 가능하지만 구현이 복잡하다.
