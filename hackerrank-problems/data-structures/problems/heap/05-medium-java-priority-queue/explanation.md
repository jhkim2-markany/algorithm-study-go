# 우선순위 큐 - 해설

## 접근 방식

1. 학생 구조체를 정의하고, 우선순위 비교 규칙을 구현한다
2. `container/heap` 패키지를 사용하여 우선순위 큐를 구현한다
3. 각 이벤트를 순서대로 처리한다:
   - ENTER: 학생을 힙에 삽입한다
   - SERVED: 힙에서 최우선순위 학생을 제거한다
4. 모든 이벤트 처리 후 힙에 남은 학생들을 우선순위 순서대로 출력한다

## 핵심 아이디어

우선순위 큐(힙)를 사용하면 삽입과 삭제를 O(log N)에 수행할 수 있다. Go의 `container/heap` 패키지로 커스텀 비교 함수를 가진 힙을 구현한다. Less 함수에서 CGPA 내림차순 → 이름 오름차순 → ID 오름차순 순으로 비교하면 된다.

CGPA가 높을수록 우선순위가 높으므로 최대 힙처럼 동작해야 한다. Less 함수에서 CGPA를 내림차순으로 비교하면 `container/heap`이 최소 힙으로 동작하더라도 CGPA가 높은 학생이 먼저 나온다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 이벤트마다 힙 삽입/삭제에 O(log N)이 걸린다. 최대 N개의 이벤트를 처리하므로 전체 O(N log N)이다. 힙에 최대 N명의 학생이 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **정렬 기반**: 매번 정렬하면 O(N^2 log N)으로 비효율적이다.
- **이진 탐색 트리**: 삽입/삭제/최솟값 접근이 O(log N)이지만 힙보다 구현이 복잡하다.
