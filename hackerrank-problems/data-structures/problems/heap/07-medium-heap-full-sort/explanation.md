# 힙 정렬 - 해설

## 접근 방식

1. 배열을 최대 힙으로 구성한다 (Build Max Heap)
2. 힙의 루트(최댓값)를 배열의 마지막 원소와 교환한다
3. 힙 크기를 1 줄이고 루트에서 Heapify를 수행한다
4. 교환 후 배열 상태를 출력한다
5. 힙 크기가 1이 될 때까지 반복한다

## 핵심 아이디어

힙 정렬은 최대 힙의 성질을 이용한 정렬 알고리즘이다. 최대 힙에서 루트는 항상 최댓값이므로, 루트를 배열의 끝으로 보내고 힙 크기를 줄이면 정렬된 부분이 뒤에서부터 채워진다. Heapify 연산은 부모 노드와 자식 노드를 비교하여 힙 성질을 복원한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(1) |

Build Max Heap에 O(N), 각 교환 후 Heapify에 O(log N)이 걸린다. N-1번 교환하므로 전체 O(N log N)이다. 제자리 정렬이므로 추가 공간은 O(1)이다.

## 대안적 접근

- **퀵 정렬**: 평균 O(N log N)이지만 최악 O(N^2)이다. 힙 정렬은 항상 O(N log N)을 보장한다.
- **병합 정렬**: O(N log N)이지만 O(N) 추가 공간이 필요하다.
- **Go 표준 라이브러리**: `sort.Ints()`를 사용하면 간단하지만, 이 문제는 힙 정렬 과정을 직접 구현해야 한다.
