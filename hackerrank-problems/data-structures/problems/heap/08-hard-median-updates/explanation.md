# 중앙값 갱신 - 해설

## 접근 방식

1. 두 개의 힙(최대 힙 + 최소 힙)을 사용하여 리스트를 관리한다
2. 추가 연산: 적절한 힙에 삽입하고 균형을 맞춘다
3. 삭제 연산: lazy deletion으로 삭제 표시 후 균형을 맞춘다
4. 중앙값 조회: 두 힙의 루트에서 계산한다

## 핵심 아이디어

Running Median 문제의 확장으로, 삭제 연산이 추가된다. 두 개의 힙(최대 힙 + 최소 힙)을 사용하되, 삭제 시 lazy deletion을 적용한다. 삭제할 원소를 맵에 기록해두고, 힙의 루트가 삭제된 원소이면 제거한다.

핵심은 두 힙의 "유효한 크기"를 별도로 관리하는 것이다. 실제 힙 크기와 삭제된 원소 수를 추적하여 균형을 맞춘다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 연산마다 힙 삽입/삭제에 O(log N)이 걸린다. Lazy deletion으로 인해 힙 크기가 최대 N이므로 전체 O(N log N)이다.

## 대안적 접근

- **균형 이진 탐색 트리**: 삽입, 삭제, 중앙값 조회 모두 O(log N)에 가능하지만 Go에서 직접 구현해야 한다.
- **정렬된 슬라이스**: 이진 탐색으로 삽입/삭제 위치를 찾으면 O(N)이 걸린다.
- **세그먼트 트리**: 좌표 압축 후 세그먼트 트리로 k번째 원소를 찾을 수 있다.
