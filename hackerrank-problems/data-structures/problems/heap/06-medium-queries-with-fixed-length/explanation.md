# 고정 길이 쿼리 - 해설

## 접근 방식

1. 각 쿼리마다 길이 d의 슬라이딩 윈도우를 사용한다
2. 슬라이딩 윈도우 내의 최댓값을 효율적으로 추적한다
3. 모든 윈도우의 최댓값 중 최솟값을 구한다

## 핵심 아이디어

슬라이딩 윈도우 최댓값 문제를 힙(우선순위 큐)으로 풀 수 있다. 최대 힙을 사용하여 윈도우 내의 최댓값을 O(log N)에 구한다. 윈도우가 이동할 때 범위를 벗어난 원소는 지연 삭제(lazy deletion) 방식으로 처리한다.

힙의 top이 현재 윈도우 범위 밖이면 pop하고, 범위 안이면 그것이 현재 윈도우의 최댓값이다. 이 방식으로 각 윈도우의 최댓값을 구한 뒤, 그 중 최솟값을 추적한다.

덱(deque)을 사용하면 O(N)에 풀 수 있지만, 힙을 사용하는 방법도 O(N log N)으로 충분히 효율적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q × N log N) |
| 공간 복잡도 | O(N) |

각 쿼리마다 N개의 원소를 힙에 삽입/삭제하므로 O(N log N)이다. Q개의 쿼리를 처리하므로 전체 O(Q × N log N)이다.

## 대안적 접근

- **덱(Deque)**: 슬라이딩 윈도우 최댓값을 O(N)에 구할 수 있다. 전체 O(Q × N)이다.
- **세그먼트 트리**: 구간 최댓값 쿼리를 O(log N)에 처리할 수 있다. 전체 O(Q × N log N)이다.
- **완전 탐색**: 각 윈도우마다 최댓값을 직접 구하면 O(Q × N × d)이다.
