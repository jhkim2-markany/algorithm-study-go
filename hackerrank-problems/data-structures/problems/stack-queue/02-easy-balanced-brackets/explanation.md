# 균형 잡힌 괄호 - 해설

## 접근 방식

1. 스택을 사용하여 여는 괄호를 저장한다
2. 닫는 괄호를 만나면 스택의 최상위와 짝이 맞는지 확인한다
3. 짝이 맞으면 팝하고, 맞지 않으면 균형이 깨진 것이다
4. 문자열을 모두 처리한 후 스택이 비어있으면 균형 잡힌 문자열이다

## 핵심 아이디어

스택의 LIFO(Last In, First Out) 특성을 활용하면 가장 최근에 열린 괄호와 현재 닫는 괄호의 짝을 효율적으로 확인할 수 있다. 여는 괄호는 스택에 푸시하고, 닫는 괄호를 만나면 스택 최상위의 여는 괄호와 매칭한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

문자열을 한 번 순회하므로 시간 복잡도는 O(N)이다. 최악의 경우 모든 문자가 여는 괄호이면 스택에 N개가 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **카운터 방식**: 각 괄호 종류별 카운터를 사용할 수 있으나, `([)]`처럼 순서가 잘못된 경우를 감지하지 못한다.
- **문자열 치환**: 짝이 맞는 괄호를 반복적으로 제거하는 방법도 있으나, 시간 복잡도가 O(N²)이 된다.
