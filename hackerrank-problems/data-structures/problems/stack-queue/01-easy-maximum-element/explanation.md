# 최대 원소 - 해설

## 접근 방식

1. 메인 스택과 함께 현재까지의 최대값을 추적하는 보조 스택을 사용한다
2. 원소를 푸시할 때 보조 스택의 최상위와 비교하여 더 큰 값을 보조 스택에 푸시한다
3. 팝할 때 두 스택 모두에서 팝한다
4. 최대값 쿼리 시 보조 스택의 최상위를 반환한다

## 핵심 아이디어

보조 스택(max stack)을 사용하면 O(1)에 최대값을 조회할 수 있다. 보조 스택의 i번째 원소는 메인 스택의 하위 i개 원소 중 최대값을 나타낸다. 이를 통해 팝 연산 후에도 최대값을 즉시 알 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 쿼리를 O(1)에 처리하므로 전체 시간 복잡도는 O(N)이다. 보조 스택에 최대 N개의 원소가 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **매번 순회**: 쿼리 3마다 스택 전체를 순회하여 최대값을 찾을 수 있으나, 시간 복잡도가 O(N²)이 되어 비효율적이다.
- **세그먼트 트리**: 범위 최대값 쿼리를 지원하지만, 이 문제에서는 과도한 구현이다.
