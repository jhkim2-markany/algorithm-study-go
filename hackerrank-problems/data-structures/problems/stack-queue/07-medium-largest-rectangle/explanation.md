# 히스토그램에서 가장 큰 직사각형 - 해설

## 접근 방식

1. 스택을 사용하여 높이가 증가하는 인덱스를 유지한다
2. 현재 높이가 스택 최상위의 높이보다 작으면, 스택에서 팝하며 넓이를 계산한다
3. 팝한 높이를 기준으로 너비는 현재 인덱스와 새 스택 최상위 사이의 거리이다
4. 모든 건물을 처리한 후 스택에 남은 인덱스에 대해서도 넓이를 계산한다

## 핵심 아이디어

단조 증가 스택(monotone stack)을 사용하면 각 막대가 확장할 수 있는 좌우 경계를 효율적으로 찾을 수 있다. 스택에는 높이가 증가하는 순서로 인덱스를 저장한다. 현재 막대의 높이가 스택 최상위보다 작으면, 스택 최상위 막대의 오른쪽 경계가 현재 위치임을 의미한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 인덱스는 스택에 최대 한 번 푸시되고 한 번 팝되므로 시간 복잡도는 O(N)이다. 스택에 최대 N개의 인덱스가 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **분할 정복**: 구간의 최소 높이를 기준으로 분할하여 재귀적으로 풀 수 있다. 세그먼트 트리를 사용하면 O(N log N)에 해결 가능하다.
- **브루트포스**: 모든 쌍의 구간을 확인하면 O(N²)이지만, N이 크면 시간 초과가 발생한다.
