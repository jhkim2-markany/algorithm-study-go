# 간단한 텍스트 편집기 - 해설

## 접근 방식

1. 현재 문자열 상태를 관리한다
2. 연산 1(append)과 연산 2(delete) 수행 전에 현재 상태를 스택에 저장한다
3. 연산 4(undo) 시 스택에서 이전 상태를 복원한다
4. 연산 3(print) 시 해당 인덱스의 문자를 출력한다

## 핵심 아이디어

스택을 사용하여 문자열의 이전 상태를 저장하면 undo 연산을 O(1)에 수행할 수 있다. 연산 1과 2를 수행하기 전에 현재 문자열을 스택에 푸시하고, undo 시 스택에서 팝하여 복원한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q × |S|) |
| 공간 복잡도 | O(Q × |S|) |

각 연산에서 문자열 복사가 발생할 수 있으므로 최악의 경우 O(Q × |S|)이다. 스택에 최대 Q개의 문자열 상태가 저장되므로 공간 복잡도도 O(Q × |S|)이다.

## 대안적 접근

- **역연산 저장**: 각 연산의 역연산을 저장하여 undo를 수행할 수 있다. append의 역은 delete, delete의 역은 삭제된 문자열의 append이다. 이 방식은 공간 효율이 더 좋다.
- **로프(Rope) 자료구조**: 대규모 문자열 편집에 효율적인 로프 자료구조를 사용할 수 있으나, 구현이 복잡하다.
