# 두 개의 스택으로 큐 구현 - 해설

## 접근 방식

1. 입력 스택(inStack)과 출력 스택(outStack) 두 개를 사용한다
2. enqueue: 입력 스택에 푸시한다
3. dequeue/peek: 출력 스택이 비어있으면 입력 스택의 모든 원소를 출력 스택으로 옮긴다
4. 출력 스택의 최상위에서 팝 또는 피크한다

## 핵심 아이디어

두 스택을 사용하면 LIFO를 FIFO로 변환할 수 있다. 입력 스택에서 출력 스택으로 원소를 옮기면 순서가 뒤집히므로, 출력 스택에서 팝하면 가장 먼저 들어온 원소가 나온다. 출력 스택이 비어있을 때만 옮기므로 각 원소는 최대 두 번 이동한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (분할 상환) | O(1) per operation |
| 공간 복잡도 | O(N) |

각 원소는 입력 스택에 한 번 푸시, 출력 스택에 한 번 푸시되므로 분할 상환 시간 복잡도는 연산당 O(1)이다. 전체 원소를 저장하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **연결 리스트**: 연결 리스트를 사용하면 O(1)에 enqueue/dequeue가 가능하지만, 이 문제는 스택만 사용하는 것이 조건이다.
- **원형 배열**: 고정 크기 큐를 원형 배열로 구현할 수 있으나, 동적 크기 지원이 어렵다.
