# 독성 식물 - 해설

## 접근 방식

1. 스택을 사용하여 각 식물이 죽는 날짜를 추적한다
2. 왼쪽에서 오른쪽으로 순회하며 각 식물의 사망일을 계산한다
3. 현재 식물이 스택 최상위보다 크면 1일차에 죽는다
4. 스택에서 현재 식물보다 크거나 같은 원소를 팝하며 최대 사망일을 추적한다

## 핵심 아이디어

각 식물이 죽는 날짜를 스택을 사용하여 효율적으로 계산한다. 식물 i가 죽는 날짜는 왼쪽에서 자신보다 작거나 같은 식물을 찾을 때까지의 경로에서 가장 늦게 죽는 식물의 사망일 + 1이다. 스택에 (살충제 양, 사망일) 쌍을 저장하여 이를 추적한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 식물은 스택에 최대 한 번 푸시되고 한 번 팝되므로 시간 복잡도는 O(N)이다. 스택에 최대 N개의 원소가 저장되므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **시뮬레이션**: 매일 죽는 식물을 제거하는 시뮬레이션을 수행할 수 있으나, 최악의 경우 O(N²)이 된다.
- **연결 리스트**: 연결 리스트를 사용하여 죽은 식물을 효율적으로 제거할 수 있으나, 날짜 계산이 복잡해진다.
