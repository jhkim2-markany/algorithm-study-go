# 두 스택 게임 - 해설

## 접근 방식

1. 먼저 스택 A에서 가능한 만큼 원소를 꺼낸다
2. 그 다음 스택 A에서 하나씩 되돌리면서 스택 B에서 가능한 만큼 꺼낸다
3. 각 조합에서의 총 개수를 비교하여 최대값을 구한다

## 핵심 아이디어

투 포인터 방식으로 접근한다. 스택 A에서 i개를 꺼내고 스택 B에서 j개를 꺼내는 모든 유효한 조합을 탐색한다. 먼저 A에서 최대한 꺼낸 뒤, A에서 하나씩 줄이면서 B에서 추가로 꺼낼 수 있는 개수를 확인한다. B의 포인터는 단조 증가하므로 전체 시간 복잡도는 O(n + m)이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(n + m) |
| 공간 복잡도 | O(1) |

A의 포인터는 n에서 0으로, B의 포인터는 0에서 m으로 이동하므로 시간 복잡도는 O(n + m)이다. 추가 공간은 포인터 변수만 사용하므로 O(1)이다.

## 대안적 접근

- **이진 탐색**: A에서 i개를 꺼낸 후 B에서 꺼낼 수 있는 최대 개수를 이진 탐색으로 찾을 수 있다. 시간 복잡도는 O(n log m)이다.
- **동적 프로그래밍**: dp[i][j]를 A에서 i개, B에서 j개 꺼낸 상태로 정의할 수 있으나, O(n × m) 공간이 필요하다.
