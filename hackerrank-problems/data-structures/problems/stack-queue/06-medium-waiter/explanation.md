# 웨이터 - 해설

## 접근 방식

1. 에라토스테네스의 체로 필요한 소수를 미리 구한다
2. 각 반복에서 현재 스택의 접시를 위에서부터 확인한다
3. i번째 소수로 나누어지면 B_i 스택에, 아니면 A_i 스택에 넣는다
4. 모든 반복 후 B_1~B_Q와 마지막 A_Q를 순서대로 출력한다

## 핵심 아이디어

스택의 LIFO 특성을 활용하여 접시의 순서를 관리한다. 각 반복에서 현재 스택(A_{i-1})의 최상위부터 접시를 꺼내 소수 판별 결과에 따라 B_i 또는 A_i에 넣는다. 출력 시 각 B 스택과 마지막 A 스택의 최상위부터 출력한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × Q) |
| 공간 복잡도 | O(N + Q) |

각 반복에서 최대 N개의 접시를 확인하므로 시간 복잡도는 O(N × Q)이다. 접시와 스택 저장에 O(N + Q) 공간이 필요하다.

## 대안적 접근

- **비트마스크**: 각 접시의 소수 나눗셈 결과를 비트마스크로 저장하여 한 번에 분류할 수 있으나, 구현이 복잡해진다.
- **큐 사용**: 스택 대신 큐를 사용하면 순서가 달라지므로 문제 조건에 맞지 않는다.
