# 같은 높이의 스택 - 해설

## 접근 방식

1. 각 스택의 총 높이를 계산한다
2. 세 스택의 높이가 같아질 때까지 가장 높은 스택에서 최상위 블록을 제거한다
3. 세 높이가 같아지면 그 값이 답이다

## 핵심 아이디어

그리디 방식으로 항상 가장 높은 스택에서 블록을 제거한다. 세 스택의 높이가 같아지려면 높은 쪽을 줄여야 하므로, 매 단계에서 가장 높은 스택의 최상위 블록을 제거하는 것이 최적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(n1 + n2 + n3) |
| 공간 복잡도 | O(1) |

각 블록은 최대 한 번만 제거되므로 전체 시간 복잡도는 O(n1 + n2 + n3)이다. 추가 공간은 인덱스 변수만 사용하므로 O(1)이다.

## 대안적 접근

- **해시셋 방식**: 각 스택의 가능한 높이를 누적합으로 계산하여 해시셋에 저장한 뒤, 세 셋의 교집합에서 최대값을 찾을 수 있다. 공간 복잡도가 O(n1 + n2 + n3)으로 증가한다.
- **이진 탐색**: 목표 높이를 이진 탐색으로 찾을 수도 있으나, 구현이 복잡해진다.
