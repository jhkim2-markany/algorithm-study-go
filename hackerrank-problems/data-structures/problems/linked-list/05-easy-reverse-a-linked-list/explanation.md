# 연결 리스트 뒤집기 - 해설

## 접근 방식

1. 세 개의 포인터(prev, current, next)를 준비한다
2. current가 nil이 아닌 동안 반복한다
3. next에 current.Next를 저장한다
4. current.Next를 prev로 변경한다 (방향 반전)
5. prev와 current를 한 칸씩 전진시킨다
6. 순회가 끝나면 prev를 새 헤드로 반환한다

## 핵심 아이디어

연결 리스트를 뒤집는 핵심은 각 노드의 `Next` 포인터 방향을 반대로 바꾸는 것이다. 이를 위해 현재 노드의 다음 노드를 미리 저장해두고, 현재 노드의 `Next`를 이전 노드로 변경한다. 이 과정을 리스트 끝까지 반복하면 전체 리스트가 뒤집힌다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

모든 노드를 한 번씩 방문하므로 시간 복잡도는 O(N)이다. 포인터 3개만 사용하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **재귀적 뒤집기**: 재귀 호출로 리스트 끝까지 도달한 뒤 돌아오면서 포인터를 변경할 수 있다. 공간 복잡도가 O(N)이 된다.
- **스택 활용**: 모든 노드를 스택에 넣고 꺼내면서 새 리스트를 구성할 수 있으나, O(N) 추가 공간이 필요하다.
