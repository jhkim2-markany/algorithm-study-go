# 이중 연결 리스트 뒤집기 - 해설

## 접근 방식

1. 현재 노드를 헤드로 초기화한다
2. 각 노드에서 Next와 Prev 포인터를 교환한다
3. 교환 후 원래의 Prev 방향(교환 후 Next)으로 이동한다
4. 마지막으로 처리한 노드가 새로운 헤드가 된다

## 핵심 아이디어

이중 연결 리스트를 뒤집는 핵심은 각 노드의 `Next`와 `Prev` 포인터를 교환하는 것이다. 단일 연결 리스트와 달리 이중 연결 리스트는 양방향 포인터를 가지므로, 두 포인터를 단순히 swap하면 방향이 반전된다. 모든 노드의 포인터를 교환한 후 마지막 노드가 새로운 헤드가 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

모든 노드를 한 번씩 방문하여 포인터를 교환하므로 시간 복잡도는 O(N)이다. 임시 변수 하나만 사용하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **데이터만 교환**: 양 끝에서 시작하여 데이터 값만 교환하는 방법이 있다. 다만 이중 연결 리스트에서 끝 노드를 찾는 데 O(N)이 필요하고, 포인터 교환이 더 효율적이다.
- **새 리스트 생성**: 원래 리스트를 순회하면서 새 이중 연결 리스트를 역순으로 구성할 수 있으나, O(N) 추가 공간이 필요하다.
- **재귀적 접근**: 재귀적으로 끝까지 도달한 뒤 돌아오면서 포인터를 교환할 수 있으나, 호출 스택 오버헤드가 발생한다.
