# 연결 리스트 끝에 노드 삽입 - 해설

## 접근 방식

1. 새로운 노드를 생성한다
2. 리스트가 비어 있으면 새 노드를 헤드로 반환한다
3. 리스트가 비어 있지 않으면 마지막 노드까지 순회한다
4. 마지막 노드의 Next를 새 노드로 설정한다
5. 헤드를 반환한다

## 핵심 아이디어

연결 리스트의 끝에 노드를 삽입하려면 먼저 마지막 노드를 찾아야 한다. 마지막 노드는 `Next`가 `nil`인 노드이다. 마지막 노드를 찾은 후 그 노드의 `Next`를 새 노드로 설정하면 삽입이 완료된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

마지막 노드를 찾기 위해 리스트 전체를 순회해야 하므로 시간 복잡도는 O(N)이다. 새 노드 하나만 생성하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **tail 포인터 유지**: 별도의 tail 포인터를 유지하면 O(1)에 삽입할 수 있다. 다만 이 문제에서는 헤드 포인터만 주어진다.
- **재귀적 접근**: 재귀적으로 마지막 노드를 찾아 삽입할 수 있으나, 호출 스택 오버헤드가 발생한다.
