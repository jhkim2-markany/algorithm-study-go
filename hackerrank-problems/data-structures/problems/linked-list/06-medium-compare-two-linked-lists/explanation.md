# 두 연결 리스트 비교 - 해설

## 접근 방식

1. 두 리스트의 헤드부터 동시에 순회를 시작한다
2. 각 위치에서 두 노드의 데이터를 비교한다
3. 데이터가 다르면 즉시 0을 반환한다
4. 한쪽 리스트가 먼저 끝나면 0을 반환한다
5. 둘 다 동시에 끝나면 1을 반환한다

## 핵심 아이디어

두 리스트가 동일하려면 길이가 같고 모든 대응 노드의 데이터가 같아야 한다. 두 포인터를 동시에 이동시키면서 비교하면 한 번의 순회로 판별할 수 있다. 순회 종료 후 두 포인터가 모두 nil인지 확인하여 길이 동일성도 검증한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(min(N, M)) |
| 공간 복잡도 | O(1) |

두 리스트 중 짧은 쪽의 길이만큼 순회하므로 시간 복잡도는 O(min(N, M))이다. 포인터 두 개만 사용하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **슬라이스 변환 후 비교**: 두 리스트를 각각 슬라이스로 변환한 뒤 비교할 수 있으나, O(N+M) 추가 공간이 필요하다.
- **재귀적 비교**: 재귀적으로 각 노드를 비교할 수 있으나, 호출 스택으로 인해 공간 복잡도가 증가한다.
