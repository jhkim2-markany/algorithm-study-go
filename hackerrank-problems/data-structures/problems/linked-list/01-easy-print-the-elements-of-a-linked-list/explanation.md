# 연결 리스트의 원소 출력 - 해설

## 접근 방식

1. 헤드 포인터부터 시작하여 연결 리스트를 순회한다
2. 현재 노드의 데이터를 출력한다
3. 다음 노드로 이동한다
4. 현재 노드가 nil이 될 때까지 반복한다

## 핵심 아이디어

연결 리스트의 가장 기본적인 연산인 순회(traversal)를 수행하는 문제이다. 헤드 포인터에서 시작하여 각 노드의 `Next` 포인터를 따라가며 데이터를 출력하면 된다. 리스트의 끝은 `Next`가 `nil`인 노드로 판별한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

N개의 노드를 한 번씩 방문하므로 시간 복잡도는 O(N)이다. 추가 메모리 없이 포인터 하나로 순회하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **재귀적 순회**: 재귀 함수를 사용하여 각 노드를 방문할 수 있다. 다만 재귀 호출 스택으로 인해 공간 복잡도가 O(N)이 된다.
- **슬라이스 변환 후 출력**: 연결 리스트를 먼저 슬라이스로 변환한 뒤 출력할 수도 있으나, 불필요한 메모리 사용이 발생한다.
