# 정렬된 두 연결 리스트 병합 - 해설

## 접근 방식

1. 더미 헤드 노드를 생성하여 병합 결과의 시작점으로 사용한다
2. 두 리스트의 현재 노드를 비교하여 더 작은 값을 결과 리스트에 연결한다
3. 한쪽 리스트가 끝나면 나머지 리스트를 그대로 연결한다
4. 더미 헤드의 Next를 반환한다

## 핵심 아이디어

두 정렬된 리스트를 병합하는 것은 머지 소트의 병합 단계와 동일하다. 두 리스트의 현재 노드 중 더 작은 값을 선택하여 결과 리스트에 추가하는 과정을 반복한다. 더미 헤드를 사용하면 첫 번째 노드 처리를 별도로 하지 않아도 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(1) |

두 리스트의 모든 노드를 한 번씩 방문하므로 시간 복잡도는 O(N+M)이다. 기존 노드의 포인터만 변경하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **재귀적 병합**: 재귀적으로 두 리스트의 현재 노드를 비교하며 병합할 수 있다. 코드가 간결하지만 호출 스택으로 인해 공간 복잡도가 O(N+M)이 된다.
- **새 리스트 생성**: 새 노드를 생성하며 병합할 수 있으나, 불필요한 메모리 할당이 발생한다.
