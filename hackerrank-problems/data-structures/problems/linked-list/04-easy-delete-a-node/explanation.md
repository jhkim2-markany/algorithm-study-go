# 노드 삭제 - 해설

## 접근 방식

1. 삭제할 위치가 0이면 헤드를 다음 노드로 변경한다
2. 그렇지 않으면 삭제할 노드의 이전 노드까지 순회한다
3. 이전 노드의 Next를 삭제할 노드의 Next로 연결한다

## 핵심 아이디어

연결 리스트에서 노드를 삭제하려면 삭제할 노드의 이전 노드를 찾아야 한다. 이전 노드의 `Next` 포인터를 삭제할 노드의 `Next`로 변경하면 해당 노드가 리스트에서 제거된다. 헤드 노드를 삭제하는 경우는 별도로 처리해야 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

삭제할 위치까지 순회해야 하므로 최악의 경우 시간 복잡도는 O(N)이다. 추가 메모리를 사용하지 않으므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **더미 헤드 사용**: 더미 노드를 헤드 앞에 추가하면 헤드 삭제를 별도로 처리하지 않아도 된다.
- **재귀적 접근**: 재귀적으로 위치를 추적하며 삭제할 수 있으나, 호출 스택 오버헤드가 발생한다.
