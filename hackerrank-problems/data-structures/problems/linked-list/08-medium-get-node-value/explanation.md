# 끝에서 N번째 노드 값 구하기 - 해설

## 접근 방식

1. 두 개의 포인터(fast, slow)를 헤드에 위치시킨다
2. fast 포인터를 positionFromTail만큼 먼저 이동시킨다
3. 이후 두 포인터를 동시에 이동시킨다
4. fast가 마지막 노드에 도달하면 slow가 가리키는 노드가 정답이다

## 핵심 아이디어

투 포인터(Two Pointer) 기법을 사용한다. fast 포인터를 먼저 positionFromTail만큼 이동시키면, fast와 slow 사이의 간격이 positionFromTail이 된다. 이후 두 포인터를 동시에 이동시키면 fast가 끝에 도달했을 때 slow는 끝에서 positionFromTail번째 위치에 있게 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

리스트를 한 번만 순회하므로 시간 복잡도는 O(N)이다. 포인터 두 개만 사용하므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **리스트 길이 계산 후 접근**: 먼저 리스트 길이를 구한 뒤, (길이 - 1 - positionFromTail)번째 노드에 접근한다. 리스트를 두 번 순회해야 한다.
- **스택 활용**: 모든 노드를 스택에 넣고 positionFromTail번 pop하면 원하는 노드를 얻을 수 있다. O(N) 추가 공간이 필요하다.
