# 자료구조 (Data Structures)

## 개념

자료구조는 데이터를 효율적으로 저장하고 관리하기 위한 구조이다. 문제의 특성에 맞는 자료구조를 선택하면 시간과 공간 복잡도를 크게 줄일 수 있다. 이 문서에서는 코딩 테스트에서 자주 등장하는 4가지 핵심 자료구조를 다룬다.

### 연결 리스트 (Linked List)

연결 리스트는 각 노드가 데이터와 다음 노드를 가리키는 포인터로 구성된 선형 자료구조이다. 배열과 달리 메모리상에 연속적으로 저장되지 않으며, 포인터를 통해 논리적으로 연결된다.

- **단일 연결 리스트 (Singly Linked List)**: 각 노드가 다음 노드만 가리킨다
- **이중 연결 리스트 (Doubly Linked List)**: 각 노드가 이전 노드와 다음 노드를 모두 가리킨다
- **원형 연결 리스트 (Circular Linked List)**: 마지막 노드가 첫 번째 노드를 가리켜 순환 구조를 형성한다

### 트리 (Tree)

트리는 노드들이 부모-자식 관계로 연결된 계층적 자료구조이다. 최상위 노드를 루트(root)라 하며, 자식이 없는 노드를 리프(leaf)라 한다.

- **이진 트리 (Binary Tree)**: 각 노드가 최대 2개의 자식을 가지는 트리
- **이진 탐색 트리 (BST)**: 왼쪽 서브트리의 모든 값 < 부모 < 오른쪽 서브트리의 모든 값
- **균형 이진 트리 (AVL, Red-Black Tree)**: 높이 균형을 유지하여 탐색 성능을 보장하는 트리
- **완전 이진 트리 (Complete Binary Tree)**: 마지막 레벨을 제외한 모든 레벨이 완전히 채워진 트리

### 힙 (Heap)

힙은 완전 이진 트리 기반의 자료구조로, 부모 노드와 자식 노드 간에 대소 관계가 성립한다. 우선순위 큐를 구현하는 데 주로 사용된다.

- **최소 힙 (Min Heap)**: 부모 노드의 값이 자식 노드의 값보다 항상 작거나 같다
- **최대 힙 (Max Heap)**: 부모 노드의 값이 자식 노드의 값보다 항상 크거나 같다
- 배열로 구현하며, 인덱스 `i`의 자식은 `2i+1`, `2i+2`, 부모는 `(i-1)/2`이다

### 스택과 큐 (Stack & Queue)

스택과 큐는 데이터의 삽입과 삭제 순서가 정해진 선형 자료구조이다.

- **스택 (Stack)**: LIFO(Last In, First Out) — 가장 나중에 넣은 데이터가 가장 먼저 나온다
- **큐 (Queue)**: FIFO(First In, First Out) — 가장 먼저 넣은 데이터가 가장 먼저 나온다
- **덱 (Deque)**: 양쪽 끝에서 삽입과 삭제가 모두 가능한 자료구조

## 동작 원리

### 연결 리스트

연결 리스트의 핵심 연산은 포인터 조작이다:

1. **삽입**: 새 노드를 생성하고, 이전 노드의 포인터를 새 노드로, 새 노드의 포인터를 다음 노드로 연결한다
2. **삭제**: 삭제할 노드의 이전 노드가 삭제할 노드의 다음 노드를 가리키도록 포인터를 변경한다
3. **탐색**: 헤드부터 시작하여 포인터를 따라 순차적으로 이동하며 원하는 값을 찾는다
4. **역순**: 각 노드의 포인터 방향을 반대로 뒤집는다

```
삽입 (중간):
  Before: A → B → D
  After:  A → B → C → D
          (B.next = C, C.next = D)

삭제:
  Before: A → B → C → D
  After:  A → B → D
          (B.next = D, C는 해제)
```

### 트리

트리의 핵심 연산은 순회(Traversal)와 탐색이다:

1. **전위 순회 (Preorder)**: 루트 → 왼쪽 → 오른쪽
2. **중위 순회 (Inorder)**: 왼쪽 → 루트 → 오른쪽 (BST에서 정렬된 순서)
3. **후위 순회 (Postorder)**: 왼쪽 → 오른쪽 → 루트
4. **레벨 순회 (Level Order)**: BFS를 사용하여 레벨별로 방문

```
        4
       / \
      2   6
     / \ / \
    1  3 5  7

전위: 4 → 2 → 1 → 3 → 6 → 5 → 7
중위: 1 → 2 → 3 → 4 → 5 → 6 → 7  (정렬 순서)
후위: 1 → 3 → 2 → 5 → 7 → 6 → 4
레벨: 4 → 2 → 6 → 1 → 3 → 5 → 7
```

### 힙

힙의 핵심 연산은 삽입(Push)과 추출(Pop)이다:

1. **삽입 (Heapify Up)**: 배열 끝에 새 원소를 추가한 뒤, 부모와 비교하며 위로 올린다
2. **추출 (Heapify Down)**: 루트를 제거하고 마지막 원소를 루트에 놓은 뒤, 자식과 비교하며 아래로 내린다
3. **힙 구성 (Build Heap)**: 배열을 힙으로 변환한다. 하단부터 heapify하면 O(N)에 가능하다

```
최소 힙 삽입 (값 2 삽입):
  [3, 5, 7, 8, 10]
  → [3, 5, 7, 8, 10, 2]     (끝에 추가)
  → [3, 5, 2, 8, 10, 7]     (2와 7 교환)
  → [2, 5, 3, 8, 10, 7]     (2와 3 교환)

최소 힙 추출:
  [2, 5, 3, 8, 10, 7]
  → 2를 반환, [7, 5, 3, 8, 10]  (마지막 원소를 루트로)
  → [3, 5, 7, 8, 10]            (7과 3 교환)
```

### 스택과 큐

스택과 큐의 핵심 연산은 삽입과 삭제이다:

1. **스택 Push/Pop**: 배열 또는 연결 리스트의 한쪽 끝에서만 삽입/삭제
2. **큐 Enqueue/Dequeue**: 한쪽 끝에서 삽입, 반대쪽 끝에서 삭제
3. **Peek/Front**: 삭제하지 않고 최상위/최전방 원소를 확인

```
스택 (LIFO):
  Push 1, Push 2, Push 3
  상태: [1, 2, 3] ← top
  Pop → 3, Pop → 2, Pop → 1

큐 (FIFO):
  Enqueue 1, Enqueue 2, Enqueue 3
  상태: front → [1, 2, 3] ← rear
  Dequeue → 1, Dequeue → 2, Dequeue → 3
```

### 자주 사용되는 기법

- **더미 노드 (Dummy Node)**: 연결 리스트에서 헤드 삽입/삭제를 간소화하기 위해 가짜 헤드 노드를 사용
- **투 포인터 (Two Pointer)**: 연결 리스트에서 중간 노드 찾기, 사이클 감지(Floyd's Algorithm)에 활용
- **재귀 순회**: 트리 순회를 재귀적으로 구현하여 코드를 간결하게 작성
- **모노톤 스택 (Monotone Stack)**: 스택에 단조 증가/감소 순서를 유지하여 Next Greater Element 등의 문제를 O(N)에 해결
- **두 개의 힙**: 최대 힙과 최소 힙을 동시에 사용하여 중앙값을 O(log N)에 관리
- **스택 두 개로 큐 구현**: 입력 스택과 출력 스택을 사용하여 큐의 FIFO 동작을 구현

## 적합한 문제 유형

### 연결 리스트

- 노드 삽입, 삭제, 역순 등 포인터 조작 문제
- 두 정렬된 리스트 병합
- 사이클 감지 및 시작점 찾기
- 리스트의 중간 노드 찾기
- 리스트를 특정 값 기준으로 분할

### 트리

- 트리 순회 (전위, 중위, 후위, 레벨)
- 이진 탐색 트리의 삽입, 삭제, 검증
- 최소 공통 조상 (LCA) 찾기
- 트리의 높이, 지름, 너비 계산
- 서브트리 비교 및 대칭 검사

### 힙

- 상위 K개 원소 찾기 (Top-K)
- 실시간 중앙값 계산 (Running Median)
- 작업 스케줄링 (최소 대기 시간)
- 다중 정렬 리스트 병합 (K-way Merge)
- 우선순위 기반 시뮬레이션

### 스택과 큐

- 괄호 유효성 검사 (Balanced Brackets)
- 히스토그램에서 최대 직사각형 넓이
- 다음 큰 원소 (Next Greater Element)
- BFS 탐색 (큐 활용)
- 실행 취소/되돌리기 (Undo/Redo)
- 슬라이딩 윈도우 최댓값 (덱 활용)

## 단계별 추적 (Trace)

### 예시 1: 단일 연결 리스트 역순

입력: `1 → 2 → 3 → 4 → nil`

```
초기 상태:
  prev = nil
  curr = 1 → 2 → 3 → 4 → nil

단계 1: curr = 1
  next = curr.next = 2
  curr.next = prev = nil
  prev = curr = 1
  curr = next = 2

  상태: nil ← 1    2 → 3 → 4 → nil
        prev       curr

단계 2: curr = 2
  next = curr.next = 3
  curr.next = prev = 1
  prev = curr = 2
  curr = next = 3

  상태: nil ← 1 ← 2    3 → 4 → nil
              prev       curr

단계 3: curr = 3
  next = curr.next = 4
  curr.next = prev = 2
  prev = curr = 3
  curr = next = 4

  상태: nil ← 1 ← 2 ← 3    4 → nil
                   prev       curr

단계 4: curr = 4
  next = curr.next = nil
  curr.next = prev = 3
  prev = curr = 4
  curr = next = nil

  상태: nil ← 1 ← 2 ← 3 ← 4
                        prev  curr = nil (종료)

최종 결과: 4 → 3 → 2 → 1 → nil
```

### 예시 2: 최소 힙에서 삽입과 추출

```
초기 힙: [1, 3, 5, 7, 9]

        1
       / \
      3   5
     / \
    7   9

--- 삽입: 2 ---

배열 끝에 추가: [1, 3, 5, 7, 9, 2]
인덱스 5의 부모 = (5-1)/2 = 2 → 값 5
2 < 5 → 교환: [1, 3, 2, 7, 9, 5]
인덱스 2의 부모 = (2-1)/2 = 0 → 값 1
2 > 1 → 교환 불필요, 종료

결과: [1, 3, 2, 7, 9, 5]

        1
       / \
      3   2
     / \ /
    7  9 5

--- 추출 (최솟값 1 제거) ---

루트를 마지막 원소로 교체: [5, 3, 2, 7, 9]
인덱스 0에서 heapify down:
  자식: 인덱스 1(값 3), 인덱스 2(값 2)
  최소 자식 = 2 (인덱스 2)
  5 > 2 → 교환: [2, 3, 5, 7, 9]
  인덱스 2에서 계속:
  자식 없음 → 종료

결과: [2, 3, 5, 7, 9], 반환값: 1

        2
       / \
      3   5
     / \
    7   9
```

### 예시 3: 스택을 이용한 괄호 유효성 검사

입력: `"({[]})"`

```
문자열: ( { [ ] } )
인덱스: 0 1 2 3 4 5

단계 0: '(' → 여는 괄호 → 스택에 push
  스택: ['(']

단계 1: '{' → 여는 괄호 → 스택에 push
  스택: ['(', '{']

단계 2: '[' → 여는 괄호 → 스택에 push
  스택: ['(', '{', '[']

단계 3: ']' → 닫는 괄호 → 스택 top = '[' → 매칭 → pop
  스택: ['(', '{']

단계 4: '}' → 닫는 괄호 → 스택 top = '{' → 매칭 → pop
  스택: ['(']

단계 5: ')' → 닫는 괄호 → 스택 top = '(' → 매칭 → pop
  스택: []

스택이 비어 있으므로 → 유효한 괄호 (true)
```

### 예시 4: BST에서 값 탐색

입력: BST에서 값 5 탐색

```

       / \
      3   10
     / \    \
    1   6   14
       / \  /
      4  7 13

탐색 경로:
  현재 노드: 8 → 5 < 8 → 왼쪽으로
  현재 노드: 3 → 5 > 3 → 오른쪽으로
  현재 노드: 6 → 5 < 6 → 왼쪽으로
  현재 노드: 4 → 5 > 4 → 오른쪽으로
  오른쪽 자식 없음 → 값 5는 트리에 존재하지 않음

탐색 횟수: 4회 (트리 높이만큼)
```

## 실전 팁

### 활용 노하우

- **연결 리스트 문제는 그림을 그려라**: 포인터 조작은 머릿속으로만 하면 실수하기 쉽다. 노드와 화살표를 그려가며 풀면 정확도가 올라간다
- **더미 노드 활용**: 연결 리스트에서 헤드가 변경될 수 있는 문제(삽입, 삭제, 병합)에서는 더미 노드를 만들어 `dummy.next`를 반환하면 코드가 깔끔해진다
- **트리 문제는 재귀로 시작하라**: 대부분의 트리 문제는 재귀적 구조와 잘 맞는다. 먼저 재귀로 풀고, 스택 오버플로가 우려되면 반복문으로 변환하자
- **힙은 Go의 `container/heap` 활용**: Go 표준 라이브러리의 `heap.Interface`를 구현하면 커스텀 힙을 쉽게 만들 수 있다
- **스택 문제의 패턴 인식**: "가장 가까운 큰/작은 원소", "괄호 매칭", "히스토그램" 키워드가 보이면 스택을 떠올리자
- **큐는 BFS의 핵심**: 그래프나 격자에서 최단 거리를 구할 때 큐 기반 BFS를 사용한다

### 자주 하는 실수

- **연결 리스트 포인터 순서 실수**: 삽입 시 `new.next = curr.next`를 먼저 하고 `curr.next = new`를 해야 한다. 순서가 바뀌면 원래 다음 노드에 대한 참조를 잃는다
- **트리 nil 체크 누락**: 재귀 함수에서 `node == nil` 체크를 빠뜨리면 nil 포인터 참조 에러가 발생한다. 재귀의 기저 조건을 항상 먼저 작성하자
- **힙에서 인덱스 계산 오류**: 0-based 인덱스에서 자식은 `2i+1`, `2i+2`, 부모는 `(i-1)/2`이다. 1-based와 혼동하지 말자
- **스택 비어있을 때 Pop**: 스택이 비어있는 상태에서 Pop을 시도하면 런타임 에러가 발생한다. Pop 전에 항상 `len(stack) > 0`을 확인하자
- **큐에서 방문 체크 시점**: BFS에서 큐에 넣을 때 방문 표시를 해야 한다. 큐에서 꺼낼 때 하면 같은 노드가 여러 번 큐에 들어가 메모리 초과가 발생할 수 있다

### 엣지 케이스 주의사항

- **빈 리스트/트리**: 노드가 0개인 경우를 반드시 처리해야 한다. 연결 리스트의 `head == nil`, 트리의 `root == nil` 체크
- **노드가 1개인 경우**: 역순, 삭제 등의 연산에서 노드가 1개일 때 특수 처리가 필요할 수 있다
- **힙에 동일한 값**: 힙에 같은 값이 여러 개 있을 때 삭제 연산이 올바르게 동작하는지 확인하자
- **스택/큐 오버플로**: 배열 기반 구현에서 크기 제한을 초과하는 경우. Go의 슬라이스는 동적 확장되므로 일반적으로 문제없지만, 메모리 제한에 주의
- **순환 참조**: 연결 리스트에서 의도치 않은 순환이 생기면 무한 루프에 빠진다. 사이클 감지 로직을 고려하자
- **정수 오버플로**: 트리의 BST 검증 시 범위를 `math.MinInt64`, `math.MaxInt64`로 설정할 때 오버플로에 주의

## 관련 알고리즘 비교

### 자료구조 선택 기준

| 자료구조 | 핵심 특징 | 장점 | 단점 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 배열 | 인덱스 기반 접근 | O(1) 랜덤 접근 | 삽입/삭제 O(N) | 인덱스로 빠르게 접근해야 할 때 |
| 연결 리스트 | 포인터 기반 연결 | O(1) 삽입/삭제 (위치 알 때) | O(N) 탐색 | 빈번한 삽입/삭제가 필요할 때 |
| 스택 | LIFO 순서 | O(1) push/pop | 중간 접근 불가 | 역순 처리, 괄호 매칭, DFS |
| 큐 | FIFO 순서 | O(1) enqueue/dequeue | 중간 접근 불가 | 순서대로 처리, BFS |
| 이진 탐색 트리 | 정렬된 계층 구조 | O(log N) 탐색/삽입/삭제 | 편향 시 O(N) | 정렬된 데이터의 동적 관리 |
| 힙 | 최솟값/최댓값 빠른 접근 | O(1) 최솟값/최댓값 조회 | 임의 원소 탐색 O(N) | 우선순위 기반 처리, Top-K |
| 해시 테이블 | 키-값 매핑 | O(1) 평균 탐색/삽입 | 순서 보장 안 됨 | 빠른 조회가 필요할 때 |

### 문제 유형별 자료구조 선택

| 문제 키워드 | 추천 자료구조 | 이유 |
| --- | --- | --- |
| "역순", "되돌리기", "괄호" | 스택 | LIFO 특성이 역순 처리에 적합 |
| "최단 거리", "레벨별" | 큐 (BFS) | FIFO 특성이 레벨 순회에 적합 |
| "K번째 큰/작은", "중앙값" | 힙 | 최솟값/최댓값을 효율적으로 관리 |
| "정렬된 순서 유지" | BST / 균형 트리 | 삽입/삭제 후에도 정렬 유지 |
| "빈번한 삽입/삭제" | 연결 리스트 | 포인터 변경만으로 O(1) 삽입/삭제 |
| "다음 큰 원소", "히스토그램" | 모노톤 스택 | 단조 순서를 유지하며 O(N) 해결 |
| "슬라이딩 윈도우 최댓값" | 덱 | 양쪽 끝에서 O(1) 삽입/삭제 |
| "우선순위 스케줄링" | 힙 (우선순위 큐) | 우선순위가 높은 작업을 빠르게 추출 |

### 자료구조 간 변환 관계

```
배열 ←→ 연결 리스트: 순차 접근 vs 동적 삽입/삭제
스택 → 재귀: 스택은 재귀 호출 스택을 명시적으로 구현한 것
큐 → BFS: BFS는 큐를 사용한 그래프 탐색
힙 → 정렬: 힙 정렬은 힙을 이용한 O(N log N) 정렬
BST → 정렬 배열: 중위 순회하면 정렬된 배열을 얻을 수 있음
```

**선택 기준 요약**:
- 순서가 중요하고 역순 처리가 필요하면 → 스택
- 순서대로 처리하거나 최단 거리를 구하면 → 큐
- 최솟값/최댓값을 반복적으로 구해야 하면 → 힙
- 정렬된 상태에서 동적으로 삽입/삭제하면 → BST
- 중간 삽입/삭제가 빈번하면 → 연결 리스트
- 특별한 요구사항이 없으면 → 배열 (가장 단순하고 캐시 친화적)
