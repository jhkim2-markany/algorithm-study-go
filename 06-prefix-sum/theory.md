# 누적합

## 개념

누적합(Prefix Sum)은 배열의 처음부터 각 위치까지의 합을 미리 계산하여 저장해두는 기법이다. 이를 통해 임의의 구간 합을 O(1)에 구할 수 있다.

원본 배열 `A[0..N-1]`이 주어졌을 때, 누적합 배열 `P[0..N]`은 다음과 같이 정의한다:

- `P[0] = 0`
- `P[i] = A[0] + A[1] + ... + A[i-1]` (i ≥ 1)

이렇게 전처리하면 구간 `[l, r]`의 합은 `P[r+1] - P[l]`로 O(1)에 계산할 수 있다. 매번 구간을 순회하며 합을 구하는 O(N) 방식 대비 쿼리가 많을수록 큰 이점을 가진다.

### 1차원 누적합

1차원 배열에서의 누적합은 가장 기본적인 형태이다. 배열을 한 번 순회하며 누적합 배열을 구축한 뒤, 구간 합 쿼리에 O(1)로 응답한다.

```
배열 A:    [3, 1, 4, 1, 5, 9]
누적합 P:  [0, 3, 4, 8, 9, 14, 23]

구간 [1, 3]의 합 = P[4] - P[1] = 9 - 3 = 6
(A[1] + A[2] + A[3] = 1 + 4 + 1 = 6)
```

### 2차원 누적합

2차원 배열(행렬)에서도 누적합을 확장할 수 있다. 2차원 누적합 배열 `P[i][j]`는 `(0,0)`부터 `(i-1,j-1)`까지의 직사각형 영역의 합을 저장한다.

전처리:
- `P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + A[i-1][j-1]`

`(r1,c1)`부터 `(r2,c2)`까지의 부분 행렬 합:
- `sum = P[r2+1][c2+1] - P[r1][c2+1] - P[r2+1][c1] + P[r1][c1]`

포함-배제 원리를 활용하여 겹치는 영역을 보정하는 것이 핵심이다.

## 동작 원리

### 1차원 누적합 구축

1. 크기 N+1의 누적합 배열 P를 생성하고 P[0] = 0으로 초기화한다
2. i = 1부터 N까지 순회하며 `P[i] = P[i-1] + A[i-1]`을 계산한다
3. 구간 [l, r]의 합이 필요하면 `P[r+1] - P[l]`을 반환한다

### 2차원 누적합 구축

1. 크기 (N+1)×(M+1)의 누적합 배열 P를 생성하고 0행, 0열을 0으로 초기화한다
2. 각 칸 (i, j)에 대해 포함-배제 원리로 누적합을 계산한다
3. 부분 행렬 합 쿼리에 포함-배제 공식을 적용하여 O(1)에 응답한다

### 누적합 + 해시맵 기법

누적합과 해시맵을 결합하면 "합이 K인 부분배열의 개수"와 같은 문제를 O(N)에 풀 수 있다. 현재까지의 누적합에서 K를 뺀 값이 이전에 등장한 적 있는지를 해시맵으로 확인하는 방식이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 전처리 시간 (1D) | O(N) |
| 구간 합 쿼리 (1D) | O(1) |
| 전처리 시간 (2D) | O(N×M) |
| 부분 행렬 합 쿼리 (2D) | O(1) |
| 공간 복잡도 (1D) | O(N) |
| 공간 복잡도 (2D) | O(N×M) |

전처리에 배열 크기에 비례하는 시간이 소요되지만, 이후 각 쿼리는 O(1)에 처리된다. Q개의 쿼리가 주어지면 전체 시간 복잡도는 O(N + Q) (1D) 또는 O(N×M + Q) (2D)로, 매번 구간을 순회하는 O(N×Q) 방식보다 훨씬 효율적이다.

## 적합한 문제 유형

- 배열의 구간 합을 여러 번 구해야 하는 문제
- 합이 특정 값인 부분배열의 개수를 세는 문제
- 2차원 행렬에서 부분 행렬의 합을 구하는 문제
- 구간 합의 최댓값/최솟값을 구하는 문제
- 배열의 구간에 동일한 값을 더하는 연산이 반복되는 문제 (차분 배열과 결합)
- 연속 부분배열의 평균이 특정 조건을 만족하는지 판별하는 문제


## 단계별 추적 (Trace)

### 예시: 1차원 누적합 구축 및 구간 합 쿼리

입력: `A = [3, 1, 4, 1, 5, 9]`

```
누적합 배열 구축:

인덱스:    0    1    2    3    4    5
배열 A:  [ 3,   1,   4,   1,   5,   9 ]

P[0] = 0
P[1] = P[0] + A[0] = 0 + 3 = 3
P[2] = P[1] + A[1] = 3 + 1 = 4
P[3] = P[2] + A[2] = 4 + 4 = 8
P[4] = P[3] + A[3] = 8 + 1 = 9
P[5] = P[4] + A[4] = 9 + 5 = 14
P[6] = P[5] + A[5] = 14 + 9 = 23

인덱스:    0    1    2    3    4    5    6
누적합 P: [ 0,   3,   4,   8,   9,  14,  23 ]

구간 합 쿼리 예시:
  [1, 3]의 합 = P[4] - P[1] = 9 - 3 = 6
  검증: A[1]+A[2]+A[3] = 1+4+1 = 6 ✓

  [0, 5]의 합 = P[6] - P[0] = 23 - 0 = 23
  검증: 3+1+4+1+5+9 = 23 ✓

  [2, 4]의 합 = P[5] - P[2] = 14 - 4 = 10
  검증: A[2]+A[3]+A[4] = 4+1+5 = 10 ✓
```

### 예시: 2차원 누적합

입력 행렬:

```
A = [ 1  2  3 ]
    [ 4  5  6 ]
    [ 7  8  9 ]

2차원 누적합 구축 (P[i][j] = (0,0)~(i-1,j-1) 영역의 합):

P = [ 0   0   0   0 ]
    [ 0   1   3   6 ]
    [ 0   5  12  21 ]
    [ 0  12  27  45 ]

구축 과정 (포함-배제):
  P[1][1] = P[0][1] + P[1][0] - P[0][0] + A[0][0] = 0+0-0+1 = 1
  P[1][2] = P[0][2] + P[1][1] - P[0][1] + A[0][1] = 0+1-0+2 = 3
  P[2][1] = P[1][1] + P[2][0] - P[1][0] + A[1][0] = 1+0-0+4 = 5
  ...

쿼리: (1,1)~(2,2) 영역의 합 (A의 5,6,8,9)
  = P[3][3] - P[1][3] - P[3][1] + P[1][1]
  = 45 - 6 - 12 + 1 = 28
  검증: 5+6+8+9 = 28 ✓

  ┌─────────────┐
  │ 1  2  3     │
  │ 4 [5  6]    │  ← 쿼리 영역
  │ 7 [8  9]    │
  └─────────────┘
```

## 실전 팁

### 활용 노하우

- **인덱스 설계**: 누적합 배열을 N+1 크기로 만들고 P[0]=0으로 시작하면 구간 합 공식이 `P[r+1] - P[l]`로 깔끔해진다. 0-indexed 배열에서 특히 유용하다
- **차분 배열(Difference Array)**: 구간에 동일한 값을 더하는 연산이 반복될 때, 차분 배열을 사용하면 각 업데이트를 O(1)에 처리하고 마지막에 누적합으로 복원할 수 있다
- **누적합 + 해시맵 조합**: "합이 K인 부분배열 개수" 문제는 `현재 누적합 - K`가 이전에 등장한 횟수를 해시맵으로 관리하면 O(N)에 풀 수 있다
- **모듈러 누적합**: 나머지 연산과 결합하면 "합이 K로 나누어 떨어지는 부분배열" 문제를 풀 수 있다

### 자주 하는 실수

- **인덱스 오프바이원(Off-by-one)**: 구간 [l, r]의 합을 구할 때 `P[r] - P[l]`이 아니라 `P[r+1] - P[l]`이다. 인덱스 설계를 명확히 하자
- **2차원 포함-배제 부호 실수**: 2차원 누적합에서 `+`, `-` 부호를 잘못 적용하면 결과가 틀린다. 공식을 외우기보다 벤 다이어그램으로 이해하자
- **원본 배열 변경 후 누적합 미갱신**: 누적합은 전처리 기법이므로, 원본 배열이 변경되면 누적합을 다시 계산해야 한다. 동적 업데이트가 필요하면 세그먼트 트리를 사용하자
- **오버플로**: 누적합은 원소들의 합이므로 값이 매우 커질 수 있다. int64를 사용하자

### 엣지 케이스 주의사항

- **전체 구간 쿼리**: [0, N-1] 구간의 합은 `P[N] - P[0] = P[N]`이다
- **단일 원소 쿼리**: [i, i] 구간의 합은 `P[i+1] - P[i] = A[i]`이다
- **빈 배열**: 배열이 비어 있으면 누적합 배열은 `[0]`만 포함한다
- **음수 포함**: 누적합은 음수 원소가 있어도 정상 동작한다. 단, 누적합이 단조 증가하지 않으므로 이진 탐색은 적용할 수 없다
- **1×1 행렬**: 2차원 누적합에서 1×1 행렬은 원소 자체가 답이다

## 관련 알고리즘 비교

| 기법 | 전처리 | 쿼리 | 업데이트 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 누적합 | O(N) | O(1) | O(N) 재구축 | 정적 배열의 구간 합 쿼리가 많은 경우 |
| 세그먼트 트리 | O(N) | O(log N) | O(log N) | 구간 합 쿼리 + 동적 업데이트가 모두 필요한 경우 |
| 펜윅 트리 (BIT) | O(N) | O(log N) | O(log N) | 세그먼트 트리보다 구현이 간단하고 메모리 효율적 |
| 희소 배열 | O(N log N) | O(1) | 불가 | 구간 최솟값/최댓값 쿼리 (합이 아닌 경우) |
| 차분 배열 | O(N) | O(N) 복원 | O(1) | 구간 업데이트가 많고 최종 결과만 필요한 경우 |

**선택 기준**:
- 배열이 변하지 않고 구간 합 쿼리만 → 누적합
- 배열이 변하면서 구간 합 쿼리 → 세그먼트 트리 또는 펜윅 트리
- 구간에 값을 더하는 업데이트가 많으면 → 차분 배열
- 구간 최솟값/최댓값 → 희소 배열 또는 세그먼트 트리
