# 2차원 누적합 - 해설

## 접근 방식

1. N×M 행렬을 입력받는다
2. (N+1)×(M+1) 크기의 2차원 누적합 배열을 포함-배제 원리로 구축한다
3. 각 쿼리 (r1, c1, r2, c2)에 대해 포함-배제 공식으로 부분 행렬 합을 O(1)에 계산한다
4. 결과를 출력한다

## 핵심 아이디어

2차원 누적합은 1차원 누적합을 2차원으로 확장한 것이다. 핵심은 포함-배제 원리(Inclusion-Exclusion Principle)이다.

### 누적합 구축

`prefix[i][j]`는 (1,1)부터 (i,j)까지의 직사각형 영역의 합이다:

```
prefix[i][j] = prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1] + matrix[i][j]
```

`prefix[i-1][j]`와 `prefix[i][j-1]`을 더하면 `prefix[i-1][j-1]` 영역이 두 번 더해지므로 한 번 빼준다.

### 부분 행렬 합 쿼리

(r1, c1)부터 (r2, c2)까지의 합:

```
sum = prefix[r2][c2] - prefix[r1-1][c2] - prefix[r2][c1-1] + prefix[r1-1][c1-1]
```

전체 영역에서 위쪽과 왼쪽 영역을 빼되, 겹치는 좌상단 영역이 두 번 빠지므로 한 번 더해준다.

이 공식을 이해하면 어떤 직사각형 영역의 합이든 O(1)에 구할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N×M + Q) |
| 공간 복잡도 | O(N×M) |

누적합 배열 구축에 O(N×M), 각 쿼리 응답에 O(1)이므로 전체 시간 복잡도는 O(N×M + Q)이다. 누적합 배열 저장에 O(N×M) 공간이 필요하다. 누적합 없이 매 쿼리마다 부분 행렬을 순회하면 O(N×M×Q)가 되어 시간 초과가 발생한다.

## 대안적 접근

- **1차원 누적합 반복 적용**: 각 행에 대해 1차원 누적합을 구축한 뒤, 쿼리마다 r1~r2 행을 순회하며 열 방향 구간 합을 더한다. 쿼리당 O(N)으로 2차원 누적합보다 느리다.
- **2차원 세그먼트 트리**: 행렬 원소의 갱신이 필요한 경우 사용한다. 쿼리 O(log N × log M), 갱신 O(log N × log M)이다. 갱신이 없는 이 문제에서는 2차원 누적합이 더 간단하고 효율적이다.
