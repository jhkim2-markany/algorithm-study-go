# 구간 합 - 해설

## 접근 방식

1. N개의 정수를 입력받아 배열에 저장한다
2. 누적합 배열을 구축한다: `prefix[i] = arr[0] + arr[1] + ... + arr[i-1]`
3. 각 쿼리 (l, r)에 대해 `prefix[r] - prefix[l-1]`로 구간 합을 O(1)에 계산한다
4. 결과를 출력한다

## 핵심 아이디어

누적합(Prefix Sum)의 기본 원리를 활용한다. 배열의 처음부터 각 위치까지의 합을 미리 계산해두면, 임의의 구간 합을 뺄셈 한 번으로 구할 수 있다.

구간 `[l, r]`의 합은 "처음부터 r까지의 합"에서 "처음부터 l-1까지의 합"을 빼면 된다:

```
sum(l, r) = prefix[r] - prefix[l-1]
```

만약 누적합 없이 매 쿼리마다 구간을 순회하면 O(N×M)이 되어 시간 초과가 발생할 수 있다. 누적합을 사용하면 전처리 O(N) + 쿼리 O(M)으로 전체 O(N+M)에 해결할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N) |

N은 배열 크기, M은 쿼리 수이다. 누적합 배열 구축에 O(N), 각 쿼리 응답에 O(1)이므로 전체 시간 복잡도는 O(N + M)이다. 누적합 배열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **단순 반복**: 매 쿼리마다 l부터 r까지 순회하여 합을 구한다. 시간 복잡도 O(N×M)으로 쿼리가 많으면 비효율적이다.
- **세그먼트 트리**: 구간 합 쿼리와 함께 원소 갱신이 필요한 경우 세그먼트 트리를 사용한다. 쿼리 O(log N), 갱신 O(log N)이다. 이 문제처럼 갱신이 없는 경우에는 누적합이 더 간단하고 빠르다.
