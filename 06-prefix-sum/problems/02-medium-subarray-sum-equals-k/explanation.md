# 합이 K인 부분배열 - 해설

## 접근 방식

1. 배열을 순회하며 누적합을 계산한다
2. 해시맵에 각 누적합 값의 등장 횟수를 기록한다
3. 현재 누적합에서 K를 뺀 값이 해시맵에 존재하면, 해당 횟수만큼 합이 K인 부분배열이 존재한다
4. 전체 개수를 출력한다

## 핵심 아이디어

누적합과 해시맵을 결합하는 기법이다. 핵심 관찰은 다음과 같다:

부분배열 `A[i+1..j]`의 합이 K라면, `prefix[j] - prefix[i] = K`이다. 이를 변형하면 `prefix[i] = prefix[j] - K`이다.

따라서 현재 위치 j까지의 누적합 `prefix[j]`를 계산한 뒤, `prefix[j] - K`라는 값이 이전에 몇 번 등장했는지를 해시맵으로 확인하면 된다. 등장 횟수만큼 합이 K인 부분배열이 존재한다.

초기에 `freq[0] = 1`을 설정하는 이유는, 배열의 처음부터 현재 위치까지의 합이 정확히 K인 경우를 처리하기 위함이다.

음수가 포함된 배열에서는 투 포인터 방식이 동작하지 않으므로, 이 누적합 + 해시맵 기법이 필수적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

배열을 한 번 순회하므로 시간 복잡도는 O(N)이다. 해시맵에 최대 N개의 서로 다른 누적합 값이 저장될 수 있으므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **브루트포스**: 모든 부분배열의 합을 계산하여 K와 비교한다. 시간 복잡도 O(N²)으로 N이 클 때 비효율적이다.
- **투 포인터**: 배열의 모든 원소가 양수인 경우에만 사용 가능하다. 음수가 포함되면 구간을 줄여도 합이 감소한다는 보장이 없어 적용할 수 없다.
