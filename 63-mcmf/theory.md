# 최소 비용 최대 유량 (MCMF)

## 개념

최소 비용 최대 유량(Minimum Cost Maximum Flow, MCMF)이란 유량 네트워크에서 소스(S)에서 싱크(T)로 **최대 유량**을 보내되, 그 **총 비용을 최소화**하는 알고리즘이다.

일반 최대 유량 문제에서는 "얼마나 많이 보낼 수 있는가"만 관심사지만, MCMF에서는 각 간선에 **용량(capacity)**과 **단위 비용(cost)**이 함께 주어진다. 유량 1을 보낼 때마다 해당 간선의 비용이 발생하므로, 같은 양의 유량을 보내더라도 어떤 경로를 선택하느냐에 따라 총 비용이 달라진다.

핵심 아이디어:
- 최대 유량 알고리즘(Ford-Fulkerson)에서 증가 경로를 찾을 때, **아무 경로**가 아닌 **최소 비용 경로**를 선택한다
- 최소 비용 경로는 잔여 그래프(residual graph)에서 SPFA(Shortest Path Faster Algorithm) 또는 Bellman-Ford로 찾는다
- 음수 비용 간선(역방향 간선)이 존재하므로 다익스트라 대신 Bellman-Ford 계열을 사용한다

대표적인 MCMF 활용:
- **할당 문제**: N명의 작업자를 N개의 작업에 1:1 배정할 때 최소 비용
- **수송 문제**: 여러 공급지에서 여러 수요지로 물건을 운송할 때 최소 비용
- **프로젝트 선택**: 이익과 비용이 있는 프로젝트들에서 최대 이익 선택
- **이분 매칭 + 비용 최적화**: 가중치가 있는 이분 매칭 문제

## 동작 원리

### 1. 네트워크 모델링

각 간선 (u, v)에 두 가지 속성이 있다:
- **용량(capacity)**: 해당 간선으로 보낼 수 있는 최대 유량
- **비용(cost)**: 유량 1을 보낼 때 드는 비용

간선 (u, v, cap, cost)를 추가할 때, 역방향 간선 (v, u, 0, -cost)도 함께 추가한다. 역방향 간선의 비용이 음수인 이유는 유량을 취소하면 비용도 환불되기 때문이다.

### 2. SPFA 기반 증가 경로 탐색

잔여 그래프에서 S→T 최소 비용 경로를 SPFA로 찾는다.

1. 거리 배열 `dist[]`를 INF로 초기화하고, `dist[S] = 0`으로 설정한다
2. SPFA(큐 기반 Bellman-Ford)를 수행한다:
   - 큐에서 노드 u를 꺼낸다
   - u의 모든 인접 간선 (u→v)에 대해, 잔여 용량 > 0이고 `dist[u] + cost < dist[v]`이면:
     - `dist[v] = dist[u] + cost`로 갱신한다
     - `prev[v] = u`로 경로를 기록한다
     - v가 큐에 없으면 큐에 추가한다
3. `dist[T] == INF`이면 더 이상 증가 경로가 없으므로 종료한다

### 3. 경로를 따라 유량 보내기

SPFA로 찾은 최소 비용 경로를 따라 유량을 보낸다.

1. 경로 상의 간선들 중 최소 잔여 용량 `f`를 구한다
2. 경로를 따라가며:
   - 정방향 간선의 잔여 용량을 `f`만큼 감소시킨다
   - 역방향 간선의 잔여 용량을 `f`만큼 증가시킨다
3. 총 유량에 `f`를 더하고, 총 비용에 `f × dist[T]`를 더한다

### 4. 반복

더 이상 증가 경로가 없을 때까지 2~3단계를 반복한다.

```
function MCMF(S, T):
    totalFlow = 0
    totalCost = 0
    while SPFA(S, T)로 최소 비용 경로가 존재:
        f = 경로 상 최소 잔여 용량
        경로를 따라 유량 f를 보낸다
        totalFlow += f
        totalCost += f × dist[T]
    return (totalFlow, totalCost)
```

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| SPFA 기반 MCMF | O(V × E × F) |
| SPFA 1회 | O(V × E) |
| 공간 복잡도 | O(V + E) |

여기서 V는 정점 수, E는 간선 수, F는 최대 유량이다. 매 반복마다 최소 1의 유량을 보내므로 최대 F번 반복하고, 각 반복에서 SPFA는 O(V×E)이다. 실제로는 SPFA가 평균적으로 훨씬 빠르게 동작하며, 대부분의 코딩 테스트 문제에서 충분한 성능을 보인다.

## 적합한 문제 유형

- N명의 작업자를 N개의 작업에 최소 비용으로 1:1 배정하는 문제 (할당 문제)
- 여러 공급지에서 여러 수요지로 물건을 최소 비용으로 운송하는 문제 (수송 문제)
- 이익과 비용이 있는 프로젝트들에서 최대 순이익을 구하는 문제 (프로젝트 선택)
- 가중치가 있는 이분 매칭에서 최소(또는 최대) 비용 매칭을 구하는 문제
- 격자에서 K개의 경로를 겹치지 않게 선택하여 비용을 최소화하는 문제
- 간선에 용량과 비용이 모두 있는 네트워크 최적화 문제

## 단계별 추적 (Trace)

### 예시: 2명의 작업자를 2개의 작업에 최소 비용으로 배정

작업자 A, B를 작업 X, Y에 1:1 배정한다. 비용은 다음과 같다:

```
         작업 X   작업 Y
작업자 A:   3       8
작업자 B:   5       2
```

최적 배정: A→X(비용 3), B→Y(비용 2), 총 비용 = 5

**네트워크 구성:**

```
노드: S(소스), A, B, X, Y, T(싱크)

간선 (용량, 비용):
  S → A  (1, 0)    S → B  (1, 0)
  A → X  (1, 3)    A → Y  (1, 8)
  B → X  (1, 5)    B → Y  (1, 2)
  X → T  (1, 0)    Y → T  (1, 0)

그래프:
        ┌──(1,3)──→ X ──(1,0)──┐
  S ──(1,0)──→ A                T
  │            └──(1,8)──→ Y ──(1,0)──┘
  │     ┌──(1,5)──→ X ──────────┘
  └──(1,0)──→ B                 │
               └──(1,2)──→ Y ───┘
```

**반복 1: SPFA로 최소 비용 경로 탐색**

```
초기: dist = [0, INF, INF, INF, INF, INF]
       (S=0, A=1, B=2, X=3, Y=4, T=5)

SPFA 수행:
  S(0) → A: dist[A] = 0+0 = 0
  S(0) → B: dist[B] = 0+0 = 0
  A(0) → X: dist[X] = 0+3 = 3
  A(0) → Y: dist[Y] = 0+8 = 8
  B(0) → X: dist[X] = min(3, 0+5) = 3 (갱신 안 됨)
  B(0) → Y: dist[Y] = min(8, 0+2) = 2 (갱신!)
  X(3) → T: dist[T] = 3+0 = 3
  Y(2) → T: dist[T] = min(3, 2+0) = 2 (갱신!)

최소 비용 경로: S → B → Y → T (비용 = 2)
경로 상 최소 잔여 용량: min(1, 1, 1) = 1

유량 1을 보낸다:
  totalFlow = 1, totalCost = 1 × 2 = 2
```

**반복 1 후 잔여 그래프:**

```
간선 잔여 용량 변화:
  S → B: 1 → 0  (역방향 B → S: 0 → 1)
  B → Y: 1 → 0  (역방향 Y → B: 0 → 1)
  Y → T: 1 → 0  (역방향 T → Y: 0 → 1)
```

**반복 2: SPFA로 최소 비용 경로 탐색**

```
잔여 그래프에서 SPFA 수행:
  S → A: dist[A] = 0 (S→B는 잔여 용량 0이므로 사용 불가)
  A → X: dist[X] = 0+3 = 3
  A → Y: dist[Y] = 0+8 = 8 (B→Y도 잔여 용량 0)
  X → T: dist[T] = 3+0 = 3

최소 비용 경로: S → A → X → T (비용 = 3)
경로 상 최소 잔여 용량: min(1, 1, 1) = 1

유량 1을 보낸다:
  totalFlow = 2, totalCost = 2 + 1 × 3 = 5
```

**반복 3: SPFA 탐색**

```
S에서 T로 가는 잔여 용량이 있는 경로가 없다 → 종료
```

**최종 결과:**

```
최대 유량: 2
최소 비용: 5

배정 결과:
  A → X (비용 3)
  B → Y (비용 2)
  총 비용: 5
```

## 실전 팁

### 활용 노하우

- MCMF 문제의 핵심은 **네트워크 모델링**이다. 문제를 읽고 소스, 싱크, 간선의 용량과 비용을 정확히 설정하는 것이 가장 중요하다
- 할당 문제는 "작업자→작업" 이분 그래프에 소스와 싱크를 추가하는 전형적인 패턴이다. 이 패턴을 외워두면 빠르게 모델링할 수 있다
- 최대 비용 최대 유량이 필요하면 모든 비용에 -1을 곱한 뒤 MCMF를 수행하고, 결과 비용에 다시 -1을 곱하면 된다
- 간선을 추가할 때 정방향과 역방향을 항상 쌍으로 추가한다. 인접 리스트에서 짝수/홀수 인덱스로 쌍을 관리하면 편리하다
- Go에서 SPFA 큐는 슬라이스로 구현한다. `inQueue` 배열로 중복 삽입을 방지하자

### 자주 하는 실수

- 역방향 간선의 비용을 `-cost`가 아닌 `cost`로 설정하는 실수. 역방향 간선은 유량 취소를 의미하므로 비용도 음수여야 한다
- 간선 쌍의 인덱스를 잘못 관리하는 실수. 간선을 추가할 때 `len(edges)`가 짝수인지 확인하고, `i ^ 1`로 역방향 간선에 접근하자
- SPFA에서 `inQueue` 배열을 갱신하지 않는 실수. 큐에서 꺼낼 때 `inQueue[u] = false`로 반드시 초기화하자
- 소스나 싱크로의 간선 비용을 0이 아닌 값으로 설정하는 실수. 소스/싱크 간선은 보통 비용 0, 용량만 설정한다
- 노드 번호를 잘못 매핑하는 실수. 소스, 싱크, 작업자, 작업 등 노드 번호를 명확히 정의하고 시작하자

### 엣지 케이스

- 유량이 0인 경우: S에서 T로 가는 경로가 없으면 유량 0, 비용 0
- 모든 간선 비용이 0인 경우: 일반 최대 유량과 동일한 결과
- 비용이 음수인 간선이 있는 경우: SPFA가 음수 가중치를 처리하므로 정상 동작
- 노드가 1개인 경우 (S = T): 유량 0, 비용 0
- 용량이 매우 큰 경우: 유량 F가 커지면 반복 횟수가 증가하므로 시간 초과 주의

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| MCMF (SPFA 기반) | 최소 비용 경로로 유량 전송, O(VEF) | 비용이 있는 유량 문제 전반 |
| 최대 유량 (Edmonds-Karp) | BFS로 증가 경로 탐색, O(VE²) | 비용 없이 최대 유량만 필요 |
| 헝가리안 알고리즘 | 이분 매칭 최소 비용, O(N³) | N×N 할당 문제 (정방 행렬) |
| 이분 매칭 (Hopcroft-Karp) | 최대 매칭만, O(E√V) | 비용 없는 이분 매칭 |
| 프로젝트 선택 (최소 컷) | 최대 유량 = 최소 컷 활용 | 이익/비용 기반 선택 문제 |

- 비용 없이 최대 유량만 → **최대 유량 (Edmonds-Karp)**
- 비용이 있는 유량 문제 → **MCMF**
- N×N 1:1 할당 최소 비용 → **헝가리안 알고리즘** 또는 **MCMF**
- 비용 없는 이분 매칭 → **Hopcroft-Karp**
- 이익/비용 기반 프로젝트 선택 → **최소 컷** 또는 **MCMF**
