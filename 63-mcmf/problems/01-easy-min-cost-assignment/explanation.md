# 최소 비용 할당 - 해설

## 접근 방식

1. 할당 문제를 유량 네트워크로 모델링한다
2. 소스(S) → 작업자 → 작업 → 싱크(T) 구조의 이분 그래프를 구성한다
3. MCMF를 수행하여 최소 비용 최대 유량을 구한다
4. 최대 유량 = N (모든 작업자가 배정됨), 최소 비용이 답이다

## 핵심 아이디어

할당 문제는 MCMF의 가장 기본적인 응용이다. N명의 작업자와 N개의 작업을 이분 그래프로 연결하고, 소스와 싱크를 추가하면 유량 네트워크가 된다.

각 간선의 용량을 1로 설정하면 "1:1 배정" 조건이 자동으로 만족된다. 소스에서 각 작업자로 용량 1의 간선을 연결하면 각 작업자는 최대 1개의 작업만 받을 수 있고, 각 작업에서 싱크로 용량 1의 간선을 연결하면 각 작업에는 최대 1명만 배정된다.

네트워크 구성:
- 노드: S(소스), 작업자 1~N, 작업 1~N, T(싱크)
- S → 작업자 i: 용량 1, 비용 0
- 작업자 i → 작업 j: 용량 1, 비용 cost[i][j]
- 작업 j → T: 용량 1, 비용 0

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N³) |
| 공간 복잡도 | O(N²) |

노드 수 V = 2N+2, 간선 수 E = O(N²), 최대 유량 F = N이므로 SPFA 기반 MCMF의 복잡도는 O(V×E×F) = O(N × N² × N) = O(N⁴)이지만, 실제로 SPFA가 평균적으로 빠르게 동작하여 N ≤ 100에서 충분하다.

## 대안적 접근

- 헝가리안 알고리즘: O(N³)으로 할당 문제를 직접 풀 수 있다. 정방 행렬 전용이지만 MCMF보다 효율적이다
- 브루트포스: N!개의 순열을 모두 시도하면 O(N! × N)으로 N ≤ 10 정도에서만 가능하다
