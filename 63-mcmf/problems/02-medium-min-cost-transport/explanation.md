# 최소 비용 수송 - 해설

## 접근 방식

1. 수송 문제를 유량 네트워크로 모델링한다
2. 소스(S) → 공장 → 창고 → 싱크(T) 구조를 구성한다
3. 각 간선의 용량과 비용을 적절히 설정한다
4. MCMF를 수행하여 최소 비용을 구한다

## 핵심 아이디어

수송 문제는 할당 문제의 일반화이다. 할당 문제에서는 모든 간선의 용량이 1이지만, 수송 문제에서는 공급량과 수요량에 따라 용량이 달라진다.

네트워크 구성:
- S → 공장 i: 용량 = supply[i], 비용 = 0 (공장이 생산할 수 있는 최대량)
- 공장 i → 창고 j: 용량 = min(supply[i], demand[j]), 비용 = cost[i][j]
- 창고 j → T: 용량 = demand[j], 비용 = 0 (창고가 필요로 하는 양)

MCMF를 수행하면 최대 유량 = 전체 수요량이 되고(공급이 충분하므로), 이때의 최소 비용이 답이다.

공급량 > 수요량인 경우 남는 물건은 자동으로 운송되지 않는다. 소스에서 공장으로의 간선 용량이 supply[i]이지만, 싱크 쪽의 수요 제한으로 인해 필요한 만큼만 유량이 흐른다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(P × W × D × (P+W)) |
| 공간 복잡도 | O(P × W) |

D는 전체 수요량의 합이다. 노드 수 V = P+W+2, 간선 수 E = O(P×W), 최대 유량 F = D이므로 SPFA 기반 MCMF의 복잡도는 O(V×E×F)이다. P, W ≤ 50이고 D ≤ 5000이므로 충분히 빠르다.

## 대안적 접근

- 선형 프로그래밍: 수송 문제는 LP로 정식화할 수 있으며, 심플렉스법으로 풀 수 있다
- 수송 심플렉스: 수송 문제에 특화된 심플렉스 변형으로 일반 LP보다 효율적이다
- 헝가리안 알고리즘: P = W이고 모든 공급/수요가 1인 특수한 경우에만 적용 가능하다
