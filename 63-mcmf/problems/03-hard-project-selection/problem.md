# 프로젝트 선택

**난이도:** 상

## 문제 설명

N개의 프로젝트와 M명의 직원이 있다. 각 프로젝트 i를 수행하면 `profit[i]`의 이익을 얻는다. 각 직원 j를 고용하면 `salary[j]`의 비용이 든다.

프로젝트를 수행하려면 해당 프로젝트에 필요한 직원들을 반드시 고용해야 한다. 프로젝트 i가 요구하는 직원 목록이 주어진다. 한 직원은 여러 프로젝트에 참여할 수 있으며, 한 번 고용하면 비용은 한 번만 지불한다.

프로젝트를 적절히 선택하고 필요한 직원을 고용하여 **순이익(이익 합 - 비용 합)**을 최대화하라.

## 입력 형식

- 첫째 줄에 프로젝트 수 N과 직원 수 M이 주어진다.
- 둘째 줄에 각 프로젝트의 이익 `profit[1], profit[2], ..., profit[N]`이 공백으로 구분되어 주어진다.
- 셋째 줄에 각 직원의 급여 `salary[1], salary[2], ..., salary[M]`이 공백으로 구분되어 주어진다.
- 넷째 줄부터 N개의 줄에 각 프로젝트가 요구하는 직원 정보가 주어진다. 각 줄의 첫 번째 정수 K는 필요한 직원 수이고, 이어서 K개의 직원 번호가 주어진다.

## 출력 형식

- 최대 순이익을 출력한다. 아무 프로젝트도 선택하지 않는 것이 최적이면 0을 출력한다.

## 제약 조건

- 1 ≤ N, M ≤ 100
- 1 ≤ profit[i] ≤ 10,000
- 1 ≤ salary[j] ≤ 10,000
- 0 ≤ K ≤ M
- 직원 번호는 1 이상 M 이하

## 예제

### 예제 입력 1

```
3 3
10 8 6
5 3 4
2 1 2
1 2
2 2 3
```

### 예제 출력 1

```
12
```

### 예제 입력 2

```
2 2
5 3
10 10
1 1
1 2
```

### 예제 출력 2

```
0
```

## 힌트

- 예제 1: 프로젝트1(이익10, 직원1,2 필요)과 프로젝트2(이익8, 직원2 필요)를 선택하면 이익=18, 비용=5+3=8, 순이익=10. 프로젝트1,2,3 모두 선택하면 이익=24, 비용=5+3+4=12, 순이익=12
- 예제 2: 어떤 프로젝트를 선택해도 직원 비용이 이익보다 크므로 아무것도 선택하지 않는 것이 최적
