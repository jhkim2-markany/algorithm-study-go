# 프로젝트 선택 - 해설

## 접근 방식

1. 프로젝트 선택 문제를 **최대 가중 폐합(Maximum Weight Closure)** 문제로 모델링한다
2. 이익 노드(프로젝트)는 소스에, 비용 노드(직원)는 싱크에 연결한다
3. 최대 유량(= 최소 컷)을 구한다
4. 최대 순이익 = 전체 프로젝트 이익의 합 - 최소 컷

## 핵심 아이디어

이 문제는 "이익이 있는 항목을 선택하되, 선택하면 반드시 비용이 드는 항목도 함께 선택해야 하는" 구조이다. 이는 최대 가중 폐합(Maximum Weight Closure) 문제의 전형적인 형태이다.

**최대 가중 폐합 정리:**
방향 그래프에서 폐합(closure)이란 선택한 노드 집합에서 나가는 간선이 없는 부분집합이다. 각 노드에 가중치(양수 또는 음수)가 있을 때, 가중치 합이 최대인 폐합을 구하는 문제는 최소 컷으로 변환된다.

네트워크 구성:
- S → 프로젝트 i: 용량 = profit[i] (이 간선이 잘리면 프로젝트를 포기)
- 직원 j → T: 용량 = salary[j] (이 간선이 잘리면 직원을 고용)
- 프로젝트 i → 직원 j: 용량 = INF (프로젝트 선택 시 직원 고용 강제)

최소 컷의 의미:
- S 쪽 컷에 포함된 프로젝트 간선 = 포기한 프로젝트의 이익 손실
- T 쪽 컷에 포함된 직원 간선 = 고용한 직원의 급여 비용
- INF 간선은 절대 잘리지 않으므로, 프로젝트를 선택하면 필요한 직원이 반드시 고용된다

따라서: 최대 순이익 = Σprofit - 최소 컷

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(V² × E) |
| 공간 복잡도 | O(V + E) |

Edmonds-Karp 알고리즘 기준 O(V²E)이다. V = N+M+2, E = O(N×M)이므로 N, M ≤ 100에서 충분히 빠르다.

## 대안적 접근

- MCMF로 직접 풀기: 프로젝트 선택에 비용을 부여하고 MCMF를 수행할 수도 있지만, 최소 컷 모델링이 더 간결하다
- 브루트포스: 2^N개의 프로젝트 부분집합을 모두 시도하면 O(2^N × M)으로 N ≤ 20 정도에서만 가능하다
- 그리디: 의존 관계가 있으므로 단순 그리디로는 최적해를 보장할 수 없다
