# 덱 (Deque)

## 개념

덱(Deque, Double-Ended Queue)은 양쪽 끝에서 삽입과 삭제가 모두 가능한 선형 자료구조이다. "Double-Ended Queue"의 줄임말로, 스택(LIFO)과 큐(FIFO)의 기능을 모두 포함하는 일반화된 자료구조이다.

일반 큐는 뒤에서 삽입하고 앞에서 삭제하지만, 덱은 앞(front)과 뒤(rear) 양쪽 모두에서 삽입과 삭제가 가능하다. 이러한 유연성 덕분에 슬라이딩 윈도우 최댓값/최솟값 문제에서 핵심적으로 사용되며, 모노톤 덱(Monotone Deque) 기법의 기반이 된다.

### 주요 연산

- **PushFront**: 덱의 앞에 원소를 추가한다
- **PushBack**: 덱의 뒤에 원소를 추가한다
- **PopFront**: 덱의 앞 원소를 제거하고 반환한다
- **PopBack**: 덱의 뒤 원소를 제거하고 반환한다
- **Front**: 덱의 앞 원소를 제거하지 않고 확인한다
- **Back**: 덱의 뒤 원소를 제거하지 않고 확인한다
- **IsEmpty**: 덱이 비어 있는지 확인한다

## 동작 원리

### 1. 배열 기반 구현

Go에서는 슬라이스를 활용하여 덱을 구현할 수 있다:

- **PushBack**: `append(deque, value)`로 뒤에 추가
- **PushFront**: `append([]int{value}, deque...)`로 앞에 추가 (비효율적)
- **PopFront**: `deque = deque[1:]`로 앞에서 제거
- **PopBack**: `deque = deque[:len(deque)-1]`로 뒤에서 제거

단, PushFront가 O(N)이므로 성능이 중요한 경우 `container/list`(이중 연결 리스트)를 사용하거나 원형 배열로 구현한다.

### 2. 원형 배열 기반 구현

고정 크기 배열에서 front와 rear 포인터를 관리하여 양쪽 삽입/삭제를 O(1)에 수행한다:

1. 배열과 front, rear 인덱스를 초기화한다
2. PushFront: front를 한 칸 앞으로 이동하고 해당 위치에 데이터를 저장한다
3. PushBack: rear 위치에 데이터를 저장하고 rear를 한 칸 뒤로 이동한다
4. PopFront: front 위치의 데이터를 반환하고 front를 한 칸 뒤로 이동한다
5. PopBack: rear를 한 칸 앞으로 이동하고 해당 위치의 데이터를 반환한다
6. 인덱스는 배열 크기로 모듈러 연산하여 순환시킨다

### 3. 모노톤 덱 (Monotone Deque)

슬라이딩 윈도우 최댓값/최솟값 문제에서 핵심적으로 사용되는 기법이다:

1. 덱에 인덱스를 저장한다
2. 새 원소를 추가할 때, 덱의 뒤에서부터 현재 원소보다 작은(또는 큰) 원소를 모두 제거한다
3. 현재 원소의 인덱스를 덱의 뒤에 추가한다
4. 윈도우 범위를 벗어난 인덱스는 덱의 앞에서 제거한다
5. 덱의 앞에 있는 인덱스가 현재 윈도우의 최댓값(또는 최솟값)이다

이 기법을 사용하면 크기 K인 슬라이딩 윈도우의 최댓값/최솟값을 O(N)에 구할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| PushFront (원형 배열/연결 리스트) | O(1) |
| PushBack | O(1) |
| PopFront | O(1) |
| PopBack | O(1) |
| Front / Back | O(1) |
| 공간 복잡도 | O(N) |

슬라이스 기반 PushFront는 O(N)이지만, 원형 배열이나 이중 연결 리스트를 사용하면 모든 연산이 O(1)이다. 코딩테스트에서는 슬라이스 기반으로도 충분한 경우가 많으며, 모노톤 덱 기법에서는 각 원소가 최대 한 번 삽입되고 한 번 삭제되므로 전체 시간 복잡도가 O(N)이다.

## 적합한 문제 유형

- 슬라이딩 윈도우 최댓값/최솟값 문제 (모노톤 덱)
- 양쪽에서 삽입/삭제가 필요한 시뮬레이션 문제
- 회문(팰린드롬) 검사 문제
- 작업 스케줄링에서 양방향 처리가 필요한 문제
- 구간 내 최솟값/최댓값을 효율적으로 관리하는 문제
- 히스토그램 최대 직사각형의 변형 문제
- BFS 변형에서 가중치가 0 또는 1인 그래프 탐색 (0-1 BFS)

## 단계별 추적 (Trace)

### 예시 입력

배열 A = [1, 3, -1, -3, 5, 3, 6, 7], 윈도우 크기 K = 3에 대해 모노톤 덱으로 슬라이딩 윈도우 최댓값을 구한다.

### 실행 과정

덱에는 인덱스를 저장하며, 덱의 앞이 현재 윈도우의 최댓값 인덱스이다.

```
i=0: A[0]=1
  덱: []  → PushBack(0) → 덱: [0]
  윈도우 미완성

i=1: A[1]=3
  A[1]=3 ≥ A[덱.Back=0]=1 → PopBack → 덱: []
  PushBack(1) → 덱: [1]
  윈도우 미완성

i=2: A[2]=-1
  A[2]=-1 < A[덱.Back=1]=3 → 유지
  PushBack(2) → 덱: [1, 2]
  윈도우 [0,2]: 최댓값 = A[덱.Front=1] = 3 ✓

i=3: A[3]=-3
  A[3]=-3 < A[덱.Back=2]=-1 → 유지
  PushBack(3) → 덱: [1, 2, 3]
  윈도우 [1,3]: 덱.Front=1 ≥ 1 → 유효
  최댓값 = A[1] = 3 ✓

i=4: A[4]=5
  A[4]=5 ≥ A[덱.Back=3]=-3 → PopBack → 덱: [1, 2]
  A[4]=5 ≥ A[덱.Back=2]=-1 → PopBack → 덱: [1]
  A[4]=5 ≥ A[덱.Back=1]=3  → PopBack → 덱: []
  PushBack(4) → 덱: [4]
  윈도우 [2,4]: 최댓값 = A[4] = 5 ✓

i=5: A[5]=3
  A[5]=3 < A[덱.Back=4]=5 → 유지
  PushBack(5) → 덱: [4, 5]
  윈도우 [3,5]: 최댓값 = A[4] = 5 ✓

i=6: A[6]=6
  A[6]=6 ≥ A[덱.Back=5]=3 → PopBack → 덱: [4]
  A[6]=6 ≥ A[덱.Back=4]=5 → PopBack → 덱: []
  PushBack(6) → 덱: [6]
  윈도우 [4,6]: 최댓값 = A[6] = 6 ✓

i=7: A[7]=7
  A[7]=7 ≥ A[덱.Back=6]=6 → PopBack → 덱: []
  PushBack(7) → 덱: [7]
  윈도우 [5,7]: 최댓값 = A[7] = 7 ✓
```

**결과: [3, 3, 5, 5, 6, 7]**

## 실전 팁

### 활용 노하우

- 슬라이딩 윈도우 최댓값/최솟값 문제가 나오면 모노톤 덱을 바로 떠올리자. O(N)에 해결할 수 있다
- 모노톤 덱에서 덱에 값 대신 인덱스를 저장하면 윈도우 범위 관리가 쉬워진다
- Go에서는 슬라이스로 덱을 구현하되, PopFront는 `deque = deque[1:]`로 처리한다. 코딩테스트에서는 이 정도로 충분하다
- 0-1 BFS에서 덱은 가중치 0 간선은 앞에, 가중치 1 간선은 뒤에 삽입하는 용도로 사용된다

### 자주 하는 실수

- 모노톤 덱에서 비교 방향 혼동: 최댓값을 구할 때는 뒤에서 현재보다 작은 원소를 제거하고, 최솟값을 구할 때는 현재보다 큰 원소를 제거한다
- 윈도우 범위 벗어난 인덱스 제거 누락: 덱의 앞 원소가 현재 윈도우 범위 밖이면 PopFront로 제거해야 한다
- 빈 덱에서 Front/Back 접근: 덱이 비어 있는지 항상 확인한 후 접근하자
- 슬라이스 기반 PushFront의 O(N) 비용: 빈번한 PushFront가 필요하면 원형 배열이나 `container/list`를 사용하자

### 엣지 케이스

- 윈도우 크기 K = 1: 각 원소 자체가 최댓값/최솟값이다
- 윈도우 크기 K = N: 전체 배열의 최댓값/최솟값 하나만 반환한다
- 모든 원소가 동일한 경우: 덱에 모든 인덱스가 유지되며, 결과는 모두 같은 값이다
- 배열이 단조 증가/감소하는 경우: 덱에 항상 하나의 원소만 유지되거나 모든 원소가 유지된다

## 관련 알고리즘 비교

| 자료구조/기법 | 삽입/삭제 | 슬라이딩 윈도우 최댓값 | 적합한 상황 |
| --- | --- | --- | --- |
| 덱 (모노톤 덱) | O(1) 양쪽 | O(N) | 슬라이딩 윈도우 최댓값/최솟값 |
| 스택 | O(1) 한쪽 | 불가 | LIFO 연산, 괄호 매칭 |
| 큐 | O(1) 한쪽 삽입/삭제 | 불가 | FIFO 연산, BFS |
| 우선순위 큐 (힙) | O(log N) | O(N log N) | 동적 최댓값/최솟값 |
| Sparse Table | O(N log N) 전처리 | O(N) | 정적 배열 구간 쿼리 |
| 세그먼트 트리 | O(log N) | O(N log N) | 동적 구간 쿼리 |

- 슬라이딩 윈도우 최댓값/최솟값이면 → 모노톤 덱 (O(N), 최적)
- 양쪽 삽입/삭제가 필요하면 → 덱
- 동적으로 최댓값/최솟값이 필요하면 → 우선순위 큐
- 정적 배열 구간 쿼리면 → Sparse Table
- 0-1 BFS면 → 덱 (가중치 0은 앞, 1은 뒤)
