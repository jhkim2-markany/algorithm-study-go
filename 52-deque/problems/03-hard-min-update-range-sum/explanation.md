# 최솟값 갱신 구간 합 - 해설

## 접근 방식

1. 모노톤 덱을 사용하여 크기 K인 모든 윈도우의 최솟값 배열 windowMin을 구한다.
2. 각 인덱스 i에 대해, i가 포함되는 윈도우 범위 [max(0, i-K+1), min(N-K, i)]를 계산한다.
3. 해당 범위에서 windowMin의 최댓값이 B[i]가 된다.
4. 구간 최댓값 쿼리를 효율적으로 처리하기 위해 Sparse Table을 구축한다.
5. 모든 B[i]의 합을 구한다.

## 핵심 아이디어

이 문제는 두 가지 핵심 기법을 결합한다:

**1단계 - 모노톤 덱으로 슬라이딩 윈도우 최솟값:**
단조 증가 덱을 유지하면서 크기 K인 각 윈도우의 최솟값을 O(N)에 구한다. 덱의 앞에 항상 현재 윈도우의 최솟값 인덱스가 위치한다.

**2단계 - Sparse Table로 구간 최댓값 쿼리:**
windowMin 배열에 대해 Sparse Table을 O(N log N)에 전처리하면, 임의 구간의 최댓값을 O(1)에 응답할 수 있다. 각 인덱스 i에 대해 B[i]를 구하는 쿼리를 N번 수행하므로 전체 O(N)이다.

B[i]의 의미는 "원소 i가 포함되는 모든 윈도우의 최솟값 중 가장 큰 값"이다. 이는 원소 i 주변에서 가장 유리한 윈도우를 선택했을 때의 최솟값을 나타낸다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N log N) |

- 슬라이딩 윈도우 최솟값: O(N)
- Sparse Table 구축: O(N log N)
- N개의 구간 최댓값 쿼리: O(N)
- 전체: O(N log N)

## 대안적 접근

- 브루트포스: 각 i에 대해 모든 포함 윈도우를 순회하며 최솟값의 최댓값을 구함 → O(NK), N이 크면 시간 초과
- 세그먼트 트리: windowMin 배열에 대해 구간 최댓값 쿼리 → O(N log N), Sparse Table과 동일한 복잡도이나 상수가 더 큼
