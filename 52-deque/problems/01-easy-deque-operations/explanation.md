# 덱 기본 연산 - 해설

## 접근 방식

1. Go 슬라이스를 이용하여 덱을 구현한다.
2. 각 명령을 문자열로 읽어 switch 문으로 분기 처리한다.
3. push_front는 슬라이스 앞에 원소를 추가하고, push_back은 append로 뒤에 추가한다.
4. pop_front, pop_back, front, back은 덱이 비어 있는 경우를 먼저 확인하고 처리한다.
5. bufio를 사용하여 입출력 성능을 최적화한다.

## 핵심 아이디어

덱의 기본 연산을 슬라이스로 직접 구현하는 문제이다. Go 슬라이스의 특성을 활용하면 간결하게 구현할 수 있다:

- `append(deque, x)`: 뒤에 추가 O(1) amortized
- `append([]int{x}, deque...)`: 앞에 추가 O(N) — 모든 원소를 한 칸씩 뒤로 밀어야 한다
- `deque[1:]`, `deque[:len-1]`: 앞/뒤에서 제거 O(1)

N이 최대 10,000이므로 PushFront의 O(N) 비용도 전체적으로 O(N²) = 10⁸ 이내에 처리 가능하다. 더 큰 입력에서는 이중 연결 리스트나 원형 배열 기반 구현이 필요하다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N²) (최악, push_front가 많은 경우) |
| 공간 복잡도 | O(N) |

- N: 명령의 수
- push_front를 제외한 모든 연산은 O(1)이다
- 전체 시간 복잡도는 push_front 호출 횟수에 따라 달라진다
