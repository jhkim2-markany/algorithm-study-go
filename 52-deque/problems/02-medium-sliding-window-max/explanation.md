# 슬라이딩 윈도우 최댓값 - 해설

## 접근 방식

1. 모노톤 덱(단조 감소 덱)을 사용하여 슬라이딩 윈도우의 최댓값을 효율적으로 관리한다.
2. 덱에는 배열의 인덱스를 저장하며, 덱의 앞쪽에 항상 현재 윈도우의 최댓값 인덱스가 위치하도록 유지한다.
3. 새 원소를 처리할 때마다 세 가지 작업을 수행한다:
   - 윈도우 범위를 벗어난 인덱스를 덱의 앞에서 제거한다
   - 현재 원소보다 작거나 같은 원소의 인덱스를 덱의 뒤에서 제거한다
   - 현재 인덱스를 덱의 뒤에 추가한다
4. 윈도우가 완성된 시점부터 덱의 앞에 있는 인덱스가 가리키는 값이 최댓값이다.

## 핵심 아이디어

모노톤 덱의 핵심은 "현재 원소보다 작은 이전 원소는 절대 최댓값이 될 수 없다"는 관찰이다. 새 원소가 들어올 때 덱의 뒤에서부터 자신보다 작은 원소를 모두 제거하면, 덱은 항상 단조 감소 순서를 유지한다. 따라서 덱의 앞이 항상 최댓값이 된다.

각 원소는 덱에 최대 한 번 삽입되고 한 번 삭제되므로, 전체 시간 복잡도는 O(N)이다. 이는 매 윈도우마다 K개 원소를 비교하는 O(NK) 브루트포스보다 훨씬 효율적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(K) |

- N: 배열의 크기
- K: 윈도우 크기
- 각 원소가 덱에 최대 한 번 삽입, 한 번 삭제되므로 전체 O(N)
- 덱에는 최대 K개의 인덱스가 저장된다

## 대안적 접근

- 브루트포스: 매 윈도우마다 K개 원소 중 최댓값을 탐색 → O(NK), N이 크면 시간 초과
- 세그먼트 트리: 구간 최댓값 쿼리로 해결 가능 → O(N log N), 구현이 복잡
- 멀티셋(균형 이진 탐색 트리): 삽입/삭제/최댓값 조회 → O(N log K), Go에서는 직접 구현 필요
