# 슬라이딩 윈도우 최댓값

**난이도:** 중

## 문제 설명

N개의 정수로 이루어진 배열과 윈도우 크기 K가 주어진다. 크기 K인 윈도우를 배열의 왼쪽에서 오른쪽으로 한 칸씩 이동시키면서, 각 위치에서 윈도우 안에 있는 수 중 최댓값을 구하라.

예를 들어 배열이 [1, 3, -1, -3, 5, 3, 6, 7]이고 K = 3이면:

| 윈도우 위치 | 윈도우 내용 | 최댓값 |
| --- | --- | --- |
| [1 3 -1] -3 5 3 6 7 | 1, 3, -1 | 3 |
| 1 [3 -1 -3] 5 3 6 7 | 3, -1, -3 | 3 |
| 1 3 [-1 -3 5] 3 6 7 | -1, -3, 5 | 5 |
| 1 3 -1 [-3 5 3] 6 7 | -3, 5, 3 | 5 |
| 1 3 -1 -3 [5 3 6] 7 | 5, 3, 6 | 6 |
| 1 3 -1 -3 5 [3 6 7] | 3, 6, 7 | 7 |

## 입력 형식

- 첫째 줄에 배열의 크기 N과 윈도우 크기 K가 주어진다 (1 ≤ K ≤ N ≤ 1,000,000)
- 둘째 줄에 N개의 정수가 공백으로 구분되어 주어진다 (각 정수의 절댓값은 1,000,000 이하)

## 출력 형식

- 각 윈도우 위치에서의 최댓값을 공백으로 구분하여 한 줄에 출력한다

## 제약 조건

- 1 ≤ K ≤ N ≤ 1,000,000
- 각 정수의 절댓값은 1,000,000 이하
- 시간 제한: 2초

## 예제

### 예제 입력 1

```
8 3
1 3 -1 -3 5 3 6 7
```

### 예제 출력 1

```
3 3 5 5 6 7
```

### 예제 입력 2

```
5 2
5 4 3 2 1
```

### 예제 출력 2

```
5 4 3 2
```

### 예제 입력 3

```
4 4
1 2 3 4
```

### 예제 출력 3

```
4
```
