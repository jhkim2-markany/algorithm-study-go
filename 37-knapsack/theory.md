# 배낭 문제

## 개념

배낭 문제(Knapsack Problem)는 제한된 용량의 배낭에 물건들을 담아 가치의 합을 최대화하는 조합 최적화 문제이다. 각 물건은 무게(weight)와 가치(value)를 가지며, 배낭의 용량을 초과하지 않으면서 가치 합이 최대가 되도록 물건을 선택해야 한다.

배낭 문제는 동적 프로그래밍의 가장 대표적인 응용 중 하나이며, 코딩테스트에서 자주 출제된다. 문제의 조건에 따라 0/1 배낭, 완전 배낭, 분할 배낭 등 다양한 변형이 존재한다.

## 선수 학습

배낭 문제를 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

## 동작 원리

### 1. 0/1 배낭 문제 (0/1 Knapsack)

각 물건을 넣거나(1) 넣지 않거나(0) 두 가지 선택만 가능하다.

1. `dp[i][w]` = 처음 i개 물건 중에서 용량 w 이하로 담을 때의 최대 가치
2. 물건 i를 넣지 않는 경우: `dp[i][w] = dp[i-1][w]`
3. 물건 i를 넣는 경우 (w >= weight[i]): `dp[i][w] = dp[i-1][w-weight[i]] + value[i]`
4. 두 경우 중 큰 값을 선택: `dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])`

1차원 배열로 공간 최적화가 가능하다. 용량을 역순으로 순회하면 된다.

### 2. 완전 배낭 문제 (Unbounded Knapsack)

각 물건을 무한히 사용할 수 있다.

1. `dp[w]` = 용량 w 이하로 담을 때의 최대 가치
2. 각 물건에 대해 용량을 정순으로 순회한다
3. `dp[w] = max(dp[w], dp[w-weight[i]] + value[i])`

0/1 배낭과 달리 용량을 정순으로 순회하여 같은 물건을 여러 번 선택할 수 있게 한다.

### 3. 분할 배낭 문제 (Fractional Knapsack)

물건을 쪼개서 일부만 담을 수 있다.

1. 각 물건의 단위 무게당 가치(value/weight)를 계산한다
2. 단위 가치가 높은 순서로 정렬한다
3. 배낭에 여유가 있는 한 물건을 통째로 넣는다
4. 마지막 물건은 남은 용량만큼만 쪼개서 넣는다

분할 배낭은 그리디 알고리즘으로 최적해를 구할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 0/1 배낭 (물건 N개, 용량 W) | O(N × W) |
| 완전 배낭 (물건 N개, 용량 W) | O(N × W) |
| 분할 배낭 (물건 N개) | O(N log N) |
| 0/1 배낭 공간 (1차원 최적화) | O(W) |
| 0/1 배낭 공간 (2차원) | O(N × W) |

## 적합한 문제 유형

- 제한된 용량/비용 내에서 가치를 최대화하는 문제
- 물건 선택 문제 (넣거나 넣지 않거나)
- 동전 교환, 부분합 등 배낭 변형 문제
- 그룹별로 하나씩 선택하는 그룹 배낭 문제
- 무한히 사용 가능한 아이템의 최적 조합 문제
- 예산 내 최대 이익 문제


## 단계별 추적 (Trace)

### 예시: 0/1 배낭 문제

```
입력:
  배낭 용량 W = 7
  물건 3개:
    물건 1: 무게=3, 가치=4
    물건 2: 무게=4, 가치=5
    물건 3: 무게=2, 가치=3

DP 테이블 (dp[i][w] = 처음 i개 물건, 용량 w일 때 최대 가치):

초기: dp[0][*] = 0

물건 1 (무게=3, 가치=4):
  w:  0  1  2  3  4  5  6  7
  dp: 0  0  0  4  4  4  4  4
  → w=3부터 물건1을 넣을 수 있음: dp[1][3] = max(0, 0+4) = 4

물건 2 (무게=4, 가치=5):
  w:  0  1  2  3  4  5  6  7
  dp: 0  0  0  4  5  5  5  9
  → w=4: max(dp[1][4]=4, dp[1][0]+5=5) = 5
  → w=7: max(dp[1][7]=4, dp[1][3]+5=9) = 9 ← 물건1+물건2

물건 3 (무게=2, 가치=3):
  w:  0  1  2  3  4  5  6  7
  dp: 0  0  3  4  5  7  8  9
  → w=2: max(dp[2][2]=0, dp[2][0]+3=3) = 3
  → w=5: max(dp[2][5]=5, dp[2][3]+3=7) = 7 ← 물건1+물건3
  → w=7: max(dp[2][7]=9, dp[2][5]+3=8) = 9 ← 물건1+물건2 유지

결과: dp[3][7] = 9 (물건 1과 물건 2 선택, 무게=3+4=7, 가치=4+5=9)
```

### 1차원 배열 최적화 추적

```
같은 문제를 1차원 배열로 풀기 (역순 순회):

초기: dp = [0, 0, 0, 0, 0, 0, 0, 0]
              w=0 1  2  3  4  5  6  7

물건 1 (무게=3, 가치=4) - w를 7→3 역순:
  w=7: dp[7] = max(dp[7], dp[7-3]+4) = max(0, 0+4) = 4
  w=6: dp[6] = max(0, dp[3]+4) = max(0, 0+4) = 4
  w=5: dp[5] = max(0, dp[2]+4) = max(0, 0+4) = 4
  w=4: dp[4] = max(0, dp[1]+4) = max(0, 0+4) = 4
  w=3: dp[3] = max(0, dp[0]+4) = max(0, 0+4) = 4
  dp = [0, 0, 0, 4, 4, 4, 4, 4]

물건 2 (무게=4, 가치=5) - w를 7→4 역순:
  w=7: dp[7] = max(dp[7]=4, dp[3]+5=9) = 9
  w=6: dp[6] = max(4, dp[2]+5=5) = 5
  w=5: dp[5] = max(4, dp[1]+5=5) = 5
  w=4: dp[4] = max(4, dp[0]+5=5) = 5
  dp = [0, 0, 0, 4, 5, 5, 5, 9]

물건 3 (무게=2, 가치=3) - w를 7→2 역순:
  w=7: dp[7] = max(9, dp[5]+3=8) = 9
  w=6: dp[6] = max(5, dp[4]+3=8) = 8
  w=5: dp[5] = max(5, dp[3]+3=7) = 7
  w=4: dp[4] = max(5, dp[2]+3=3) = 5
  w=3: dp[3] = max(4, dp[1]+3=3) = 4
  w=2: dp[2] = max(0, dp[0]+3=3) = 3
  dp = [0, 0, 3, 4, 5, 7, 8, 9]

결과: dp[7] = 9 ✓
```

## 실전 팁

### 활용 노하우

- 0/1 배낭은 1차원 배열 + 역순 순회로 공간을 O(W)로 줄이자
- 완전 배낭은 정순 순회, 0/1 배낭은 역순 순회 — 이 차이를 확실히 기억하자
- 물건 수 N이 작고(≤ 20) 용량 W가 매우 크면 비트마스크 DP나 Meet in the Middle을 고려한다
- "최소 비용으로 목표 달성" 문제도 배낭의 변형이다 (가치 대신 비용을 최소화)
- 부분합 문제(Subset Sum)는 가치 = 무게인 특수한 0/1 배낭이다

### 자주 하는 실수

- 0/1 배낭에서 용량을 정순으로 순회하여 같은 물건을 여러 번 선택하는 실수
- 완전 배낭에서 역순으로 순회하여 물건을 한 번만 선택하는 실수
- DP 배열 초기화를 잘못하는 실수: "정확히 W를 채우는" 문제와 "W 이하로 채우는" 문제의 초기값이 다르다
- 물건의 무게가 0인 경우를 처리하지 않는 실수
- 용량 W가 매우 클 때(10^9) 배열을 직접 만들려는 실수 → 다른 접근법 필요

### 엣지 케이스

- 물건이 0개인 경우: 최대 가치는 0
- 배낭 용량이 0인 경우: 아무것도 담을 수 없으므로 가치 0
- 모든 물건의 무게가 배낭 용량보다 큰 경우: 가치 0
- 물건이 1개인 경우: 넣을 수 있으면 넣고, 없으면 0
- 모든 물건의 무게가 같은 경우: 가치 순으로 정렬하여 상위 W/weight개 선택

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 물건 재사용 | 물건 분할 | 특징 |
| --- | --- | --- | --- | --- |
| 0/1 배낭 (DP) | O(NW) | 불가 | 불가 | 가장 기본적인 형태 |
| 완전 배낭 (DP) | O(NW) | 가능 | 불가 | 동전 교환 문제와 동일 |
| 분할 배낭 (그리디) | O(N log N) | 불가 | 가능 | 단위 가치 정렬 |
| 그룹 배낭 | O(NWK) | 그룹당 1개 | 불가 | 그룹별 선택 |
| Meet in the Middle | O(2^(N/2) × N) | 불가 | 불가 | N ≤ 40, W가 매우 클 때 |

**선택 기준:**
- 물건을 넣거나 빼거나 → 0/1 배낭
- 물건을 무한히 사용 가능 → 완전 배낭
- 물건을 쪼갤 수 있음 → 분할 배낭 (그리디)
- N이 작고 W가 매우 큼 → Meet in the Middle (57번 참조)
- 그룹에서 하나씩 선택 → 그룹 배낭
