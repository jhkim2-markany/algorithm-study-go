# 확률론

## 개념

확률론(Probability Theory)은 사건이 발생할 가능성을 수치적으로 계산하고, 기댓값(Expected Value)을 구하여 문제를 해결하는 알고리즘 분야이다. 코딩테스트에서는 주로 다음과 같은 확률 개념을 활용한다:

- **확률(Probability)**: 특정 사건이 발생할 가능성을 0과 1 사이의 값으로 표현한다. P(A) = (사건 A가 발생하는 경우의 수) / (전체 경우의 수)
- **기댓값(Expected Value)**: 확률 변수의 평균적인 값으로, E[X] = Σ xᵢ × P(xᵢ)로 계산한다
- **조건부 확률(Conditional Probability)**: 사건 B가 발생한 조건에서 사건 A가 발생할 확률로, P(A|B) = P(A ∩ B) / P(B)이다
- **기댓값의 선형성(Linearity of Expectation)**: E[X + Y] = E[X] + E[Y]가 항상 성립하며, X와 Y의 독립 여부와 무관하다

확률 문제는 수학적 공식으로 직접 계산하거나, 동적 프로그래밍(확률 DP)을 활용하여 상태별 확률/기댓값을 점화식으로 구하는 방식으로 풀 수 있다.

## 동작 원리

### 1. 확률 계산의 기본

확률을 계산하는 기본 원리는 다음과 같다:

1. 전체 표본 공간(Sample Space)의 크기를 구한다
2. 원하는 사건(Event)에 해당하는 경우의 수를 구한다
3. P(사건) = (사건의 경우의 수) / (전체 경우의 수)로 확률을 계산한다

주요 확률 법칙:
- 덧셈 법칙: P(A ∪ B) = P(A) + P(B) - P(A ∩ B)
- 곱셈 법칙: P(A ∩ B) = P(A) × P(B|A)
- 여사건: P(A') = 1 - P(A)

### 2. 기댓값 계산

기댓값은 확률 변수의 가중 평균이다:

1. 확률 변수 X가 취할 수 있는 모든 값 x₁, x₂, ..., xₙ을 나열한다
2. 각 값의 확률 P(X = xᵢ)를 구한다
3. E[X] = Σ xᵢ × P(X = xᵢ)를 계산한다

기댓값의 선형성을 활용하면 복잡한 기댓값도 쉽게 구할 수 있다:
- E[X₁ + X₂ + ... + Xₙ] = E[X₁] + E[X₂] + ... + E[Xₙ]
- 이 성질은 X들이 독립이 아니어도 항상 성립한다

### 3. 확률 DP (Probability DP)

상태 전이에 확률이 포함된 동적 프로그래밍이다:

1. 상태를 정의한다 (예: dp[i] = 상태 i에 도달할 확률 또는 상태 i에서의 기댓값)
2. 각 상태에서 전이 가능한 다음 상태와 전이 확률을 정의한다
3. 점화식을 세운다:
   - 확률 전파: dp[next] += dp[cur] × P(cur → next)
   - 기댓값 역추적: dp[cur] = Σ (dp[next] + cost) × P(cur → next)
4. 초기 상태를 설정하고 점화식을 적용한다

### 4. 조건부 확률과 베이즈 정리

조건부 확률을 활용한 문제 풀이:

1. P(A|B) = P(A ∩ B) / P(B)로 조건부 확률을 계산한다
2. 베이즈 정리: P(A|B) = P(B|A) × P(A) / P(B)
3. 전확률 공식: P(B) = Σ P(B|Aᵢ) × P(Aᵢ) (Aᵢ가 표본 공간의 분할일 때)

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 단순 확률 계산 | O(N) |
| 확률 DP (1차원) | O(N × M) |
| 확률 DP (2차원) | O(N × M × K) |
| 기댓값의 선형성 활용 | O(N) |
| 공간 복잡도 | O(상태 수) |

단순 확률 계산은 경우의 수를 세는 데 비례한다. 확률 DP는 상태 수와 각 상태에서의 전이 수에 비례하며, 기댓값의 선형성을 활용하면 복잡한 문제도 O(N)에 풀 수 있는 경우가 있다.

## 적합한 문제 유형

- 주사위, 동전 등 확률 실험의 결과를 계산하는 문제
- 특정 상태에 도달할 확률을 구하는 문제
- 게임이나 과정의 기댓값(평균 횟수, 평균 비용)을 구하는 문제
- 확률 DP로 상태별 확률/기댓값을 계산하는 문제
- 기댓값의 선형성을 활용하여 복잡한 기댓값을 분해하는 문제
- 조건부 확률을 활용한 문제

## 단계별 추적 (Trace)

### 예시: 확률 DP로 기댓값 계산

주사위를 던져서 6이 나올 때까지의 기대 횟수를 구한다.

```
입력: 6면 주사위, 6이 나오면 종료

상태 정의: dp = 6이 나올 때까지의 기대 횟수

점화식 유도:
  dp = (1/6) × 1 + (5/6) × (1 + dp)
  dp = 1/6 + 5/6 + (5/6) × dp
  dp - (5/6) × dp = 1
  (1/6) × dp = 1
  dp = 6

결과: 기대 횟수 = 6회
```

### 예시: 확률 DP 테이블 추적

동전을 3번 던져서 앞면이 2번 이상 나올 확률을 구한다.

```
입력: 동전 3번, 앞면 확률 = 0.5

상태: dp[i][j] = i번째 던지기까지 앞면이 j번 나올 확률

초기: dp[0][0] = 1.0

1번째 던지기:
  dp[1][0] = dp[0][0] × 0.5 = 0.5  (뒷면)
  dp[1][1] = dp[0][0] × 0.5 = 0.5  (앞면)

2번째 던지기:
  dp[2][0] = dp[1][0] × 0.5 = 0.25
  dp[2][1] = dp[1][0] × 0.5 + dp[1][1] × 0.5 = 0.25 + 0.25 = 0.5
  dp[2][2] = dp[1][1] × 0.5 = 0.25

3번째 던지기:
  dp[3][0] = dp[2][0] × 0.5 = 0.125
  dp[3][1] = dp[2][0] × 0.5 + dp[2][1] × 0.5 = 0.125 + 0.25 = 0.375
  dp[3][2] = dp[2][1] × 0.5 + dp[2][2] × 0.5 = 0.25 + 0.125 = 0.375
  dp[3][3] = dp[2][2] × 0.5 = 0.125

결과: P(앞면 ≥ 2) = dp[3][2] + dp[3][3] = 0.375 + 0.125 = 0.5
```

### 예시: 기댓값의 선형성 활용

N명이 모자를 벗어 섞은 뒤 다시 쓸 때, 자기 모자를 쓰는 사람 수의 기댓값을 구한다.

```
입력: N = 4

기댓값의 선형성 적용:
  X = X₁ + X₂ + X₃ + X₄
  (Xᵢ = i번째 사람이 자기 모자를 쓰면 1, 아니면 0)

  E[Xᵢ] = P(i번째 사람이 자기 모자를 쓸 확률) = 1/N = 1/4

  E[X] = E[X₁] + E[X₂] + E[X₃] + E[X₄]
       = 1/4 + 1/4 + 1/4 + 1/4
       = 1

결과: N에 관계없이 기댓값은 항상 1
```

## 실전 팁

### 활용 노하우

- 기댓값의 선형성은 매우 강력한 도구이다. 복잡한 기댓값을 개별 확률 변수의 기댓값 합으로 분해하라
- 확률을 분수로 표현해야 할 때는 모듈러 역원을 사용하라 (P/Q mod M = P × Q^(M-2) mod M)
- "~할 때까지의 기대 횟수" 문제는 점화식을 세워 역추적하는 패턴이 많다
- 여사건 확률(1 - P)을 활용하면 계산이 간단해지는 경우가 많다

### 자주 하는 실수

- 확률 DP에서 전이 확률의 합이 1이 되지 않는 실수
- 기댓값 DP에서 순환 참조가 발생하는 경우 수식 정리를 하지 않는 실수
- 부동소수점 오차를 고려하지 않는 실수 (오차 허용 범위 확인 필요)
- 독립이 아닌 사건에 곱셈 법칙을 잘못 적용하는 실수

### 엣지 케이스

- 확률이 0 또는 1인 경우 (확정적 사건)
- 기댓값이 무한대가 되는 경우 (종료 확률이 0인 상태)
- 모든 경우의 확률이 동일한 경우 (균등 분포)
- N = 1인 경우의 확률 DP

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 확률 DP | O(상태 수 × 전이 수) | 상태 전이에 확률이 포함된 문제 |
| 기댓값의 선형성 | O(N), 수학적 분해 | 복잡한 기댓값을 개별 합으로 분해 가능할 때 |
| 조합론 (경우의 수) | O(N) ~ O(N²) | 확률 = 경우의 수 / 전체로 계산 가능할 때 |
| 마르코프 체인 | O(S³), 행렬 거듭제곱 | 상태 전이가 반복되는 확률 과정 |
| 몬테카를로 시뮬레이션 | O(시행 횟수) | 정확한 계산이 어려울 때 근사값 |
| 포함 배제 원리 | 조건부 확률 계산 | 여러 사건의 합집합 확률 |
