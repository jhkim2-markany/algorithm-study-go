# 조건부 확률과 기댓값 - 해설

## 접근 방식

1. 비트마스크로 현재까지 모은 아이템 집합을 표현한다 (i번째 비트가 1이면 i번째 아이템 보유)
2. dp[mask]를 "이미 모은 아이템 집합이 mask일 때, 모든 아이템을 모으기까지의 기대 횟수"로 정의한다
3. 모든 아이템을 모은 상태(full mask)에서의 기댓값은 0이다
4. 역순으로 각 상태의 기댓값을 계산한다
5. 이미 가진 아이템을 다시 뽑는 자기 참조를 수식으로 정리한다

## 핵심 아이디어

이 문제는 쿠폰 수집가 문제(Coupon Collector's Problem)의 일반화 버전이다. 각 아이템의 확률이 다를 수 있으므로, 비트마스크 DP를 사용하여 모든 가능한 수집 상태를 관리한다.

핵심은 자기 참조(self-loop) 처리이다. 현재 상태 mask에서 이미 가진 아이템을 다시 뽑으면 상태가 변하지 않으므로:

dp[mask] = 1 + pRepeat × dp[mask] + Σ (prob[i] × dp[mask | (1<<i)])

이를 정리하면:
- dp[mask] × (1 - pRepeat) = 1 + Σ (prob[i] × dp[mask | (1<<i)])
- dp[mask] = (1 + Σ (prob[i] × dp[mask | (1<<i)])) / (1 - pRepeat)

여기서 pRepeat는 이미 가진 아이템을 다시 뽑을 확률이고, (1 - pRepeat)는 새로운 아이템을 뽑을 확률이다.

예제 1에서 N=2, 확률 각 0.5인 경우: dp[00] = 3.0이다. 첫 뽑기에서 아무 아이템이나 얻고(기대 1회), 나머지 하나를 얻기까지 기대 1/0.5 = 2회이므로 총 3회이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^N × N) |
| 공간 복잡도 | O(2^N) |

2^N개의 상태 각각에서 N개의 아이템을 확인하므로 O(2^N × N)이다. N ≤ 20이므로 약 2천만 연산으로 시간 내에 해결 가능하다.

## 대안적 접근

- 포함-배제 원리: 각 아이템의 확률이 동일한 경우, 쿠폰 수집가 공식 E = N × H(N) (H(N)은 조화급수)으로 O(N)에 풀 수 있다. 하지만 확률이 다른 일반적인 경우에는 비트마스크 DP가 필요하다.
- Min-Max 포함-배제: E[max] = Σ (-1)^(|S|+1) × E[min of S]를 활용하여 부분집합의 최소 기댓값으로 분해할 수도 있다.
