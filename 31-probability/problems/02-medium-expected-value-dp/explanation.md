# 기댓값 계산 DP - 해설

## 접근 방식

1. dp[i]를 "i번 칸에서 N번 칸에 도착하기까지의 기대 주사위 횟수"로 정의한다
2. N번 칸에서의 기댓값은 0이다 (이미 도착)
3. N-1번 칸부터 역순으로 기댓값을 계산한다
4. 각 칸에서 주사위 눈 1~6에 대해 다음 칸의 기댓값을 합산한다
5. N을 넘어가는 경우 제자리에 머무므로 자기 참조가 발생하며, 이를 수식으로 정리한다

## 핵심 아이디어

기댓값 DP에서 자기 참조(self-loop)를 처리하는 것이 핵심이다. i번 칸에서 주사위를 던졌을 때 N을 넘어가면 제자리에 머무므로, dp[i]의 점화식에 dp[i] 자신이 포함된다.

점화식: dp[i] = 1 + (1/6) × (Σ dp[next] + stay × dp[i])

이를 정리하면:
- 6 × dp[i] = 6 + Σ dp[next] + stay × dp[i]
- (6 - stay) × dp[i] = 6 + Σ dp[next]
- dp[i] = (6 + Σ dp[next]) / (6 - stay)

여기서 stay는 N을 넘어가서 제자리에 머무는 주사위 눈의 개수이다.

예제 1에서 N=2이면 1번 칸에서 주사위 1이 나와야만 도착하고 나머지는 제자리이므로, dp[1] = (6 + 0) / (6 - 5) = 6이다.

예제 2에서 N=7이면 1번 칸에서 주사위 6이 나오면 바로 도착하므로, 기댓값은 1이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 칸에서 최대 6가지 전이를 확인하므로 상수 시간이고, N개의 칸을 순회하므로 전체 O(N)이다.

## 대안적 접근

- 행렬 거듭제곱: 상태 전이를 행렬로 표현하고 행렬 거듭제곱으로 풀 수 있지만, N이 작으므로 단순 DP가 더 효율적이다.
- 시뮬레이션: 몬테카를로 시뮬레이션으로 근사값을 구할 수 있지만, 정확한 값이 필요하므로 DP가 적합하다.
