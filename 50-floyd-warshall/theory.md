# 플로이드-워셜 (Floyd-Warshall)

## 개념

플로이드-워셜(Floyd-Warshall) 알고리즘은 가중치 그래프에서 모든 정점 쌍 사이의 최단 경로를 구하는 동적 프로그래밍 알고리즘이다. 3중 루프라는 단순한 구조로 구현할 수 있으며, 음수 가중치 간선도 처리할 수 있다.

21-shortest-path에서 다루는 다익스트라(Dijkstra)와 벨만-포드(Bellman-Ford)가 단일 출발점 최단 경로를 구하는 반면, 플로이드-워셜은 한 번의 실행으로 모든 정점 쌍(i, j)의 최단 거리를 동시에 구한다. 정점 수 V가 작을 때(V ≤ 500 정도) 매우 효율적이며, 도달 가능성 판별, 경로 복원, 음수 사이클 탐지 등 다양한 응용이 가능하다.

핵심 아이디어는 "중간 정점 집합을 점진적으로 확장"하는 것이다. 정점 0~k만 경유할 수 있을 때의 최단 거리를 구한 뒤, k를 하나씩 늘려가며 최종적으로 모든 정점을 경유할 수 있는 최단 거리를 완성한다.

## 선수 학습

플로이드-워셜을 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [21-shortest-path](../21-shortest-path/) - 최단 경로의 기본 개념, 다익스트라와 벨만-포드 알고리즘, 완화(relaxation) 원리

## 동작 원리

### 1. 거리 행렬 초기화

V개의 정점이 있는 그래프에서 V×V 크기의 거리 행렬 dist를 초기화한다.

1. dist[i][j] = 간선 (i, j)의 가중치 (직접 연결된 경우)
2. dist[i][j] = ∞ (직접 연결되지 않은 경우)
3. dist[i][i] = 0 (자기 자신까지의 거리)

### 2. 3중 루프를 통한 최단 거리 갱신

중간 정점 k를 0부터 V-1까지 순회하며, 모든 정점 쌍 (i, j)에 대해 k를 경유하는 경로가 더 짧은지 확인한다.

```
for k := 0; k < V; k++ {
    for i := 0; i < V; i++ {
        for j := 0; j < V; j++ {
            if dist[i][k] + dist[k][j] < dist[i][j] {
                dist[i][j] = dist[i][k] + dist[k][j]
            }
        }
    }
}
```

- k가 가장 바깥 루프에 위치해야 한다 (동적 프로그래밍의 상태 전이 순서)
- 각 단계에서 "정점 0~k를 경유할 수 있을 때의 최단 거리"가 완성된다

### 3. 경로 복원

최단 경로의 실제 경유 정점을 추적하려면 next 행렬을 함께 관리한다.

1. 초기화: next[i][j] = j (직접 연결된 경우), next[i][j] = -1 (연결 없음)
2. 갱신 시: dist[i][j]가 갱신되면 next[i][j] = next[i][k]로 업데이트
3. 경로 추적: i에서 j까지의 경로는 i → next[i][j] → next[next[i][j]][j] → ... → j

### 4. 음수 가중치 처리

플로이드-워셜은 음수 가중치 간선을 자연스럽게 처리한다. 단, 음수 사이클이 존재하면 최단 거리가 정의되지 않는다.

- 음수 사이클 탐지: 알고리즘 수행 후 dist[i][i] < 0인 정점 i가 있으면 음수 사이클이 존재한다
- 음수 사이클의 영향을 받는 정점 쌍: dist[i][k] + dist[k][j] < dist[i][j]이면서 k가 음수 사이클에 포함된 경우, (i, j) 쌍의 최단 거리는 -∞이다

### 5. 도달 가능성 판별

플로이드-워셜을 변형하면 그래프의 도달 가능성(Transitive Closure)을 구할 수 있다.

1. reach[i][j] = true (직접 연결된 경우 또는 i == j)
2. 갱신: reach[i][j] = reach[i][j] || (reach[i][k] && reach[k][j])
3. 수행 후 reach[i][j]가 true이면 i에서 j로 도달 가능하다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(V³) |
| 공간 복잡도 | O(V²) |
| 경로 복원 추가 공간 | O(V²) |

V는 정점의 수이다. 3중 루프로 인해 시간 복잡도가 O(V³)이므로, 정점 수가 500 이하인 경우에 적합하다. 간선 수 E와 무관하게 항상 O(V³)이므로, 밀집 그래프에서 상대적으로 유리하다.

## 적합한 문제 유형

- 모든 정점 쌍 사이의 최단 거리를 구하는 문제
- 경유 정점을 추적하여 실제 최단 경로를 복원하는 문제
- 음수 가중치가 포함된 그래프의 최단 경로 문제
- 음수 사이클 존재 여부를 판별하는 문제
- 그래프의 도달 가능성(Transitive Closure)을 구하는 문제
- 정점 수가 적고 모든 쌍 정보가 필요한 문제 (V ≤ 500)
- 특정 정점을 반드시 경유하는 최단 경로 문제

## 단계별 추적 (Trace)

### 예시 입력

4개의 정점(0~3)과 다음 간선을 가진 방향 그래프:

```
0 → 1 (가중치 3)
0 → 3 (가중치 7)
1 → 0 (가중치 8)
1 → 2 (가중치 2)
2 → 0 (가중치 5)
2 → 3 (가중치 1)
3 → 0 (가중치 2)
```

### 실행 과정

**초기 거리 행렬 (∞ = INF)**

```
     0    1    2    3
0 [  0    3   INF   7  ]
1 [  8    0    2   INF ]
2 [  5   INF   0    1  ]
3 [  2   INF  INF   0  ]
```

**k=0 (정점 0을 경유)**

```
dist[1][3] = min(INF, dist[1][0]+dist[0][3]) = min(INF, 8+7) = 15
dist[2][1] = min(INF, dist[2][0]+dist[0][1]) = min(INF, 5+3) = 8
dist[2][3] = min(1, dist[2][0]+dist[0][3]) = min(1, 5+7) = 1 (변화 없음)
dist[3][1] = min(INF, dist[3][0]+dist[0][1]) = min(INF, 2+3) = 5
dist[3][2] = min(INF, dist[3][0]+dist[0][2]) = min(INF, 2+INF) = INF

     0    1    2    3
0 [  0    3   INF   7  ]
1 [  8    0    2    15 ]
2 [  5    8    0    1  ]
3 [  2    5   INF   0  ]
```

**k=1 (정점 1을 경유)**

```
dist[0][2] = min(INF, dist[0][1]+dist[1][2]) = min(INF, 3+2) = 5
dist[3][2] = min(INF, dist[3][1]+dist[1][2]) = min(INF, 5+2) = 7

     0    1    2    3
0 [  0    3    5    7  ]
1 [  8    0    2    15 ]
2 [  5    8    0    1  ]
3 [  2    5    7    0  ]
```

**k=2 (정점 2를 경유)**

```
dist[0][3] = min(7, dist[0][2]+dist[2][3]) = min(7, 5+1) = 6
dist[1][0] = min(8, dist[1][2]+dist[2][0]) = min(8, 2+5) = 7
dist[1][3] = min(15, dist[1][2]+dist[2][3]) = min(15, 2+1) = 3
dist[3][0] = min(2, dist[3][2]+dist[2][0]) = min(2, 7+5) = 2 (변화 없음)
dist[3][3] = min(0, dist[3][2]+dist[2][3]) = min(0, 7+1) = 0 (변화 없음)

     0    1    2    3
0 [  0    3    5    6  ]
1 [  7    0    2    3  ]
2 [  5    8    0    1  ]
3 [  2    5    7    0  ]
```

**k=3 (정점 3을 경유)**

```
dist[0][0] = min(0, dist[0][3]+dist[3][0]) = min(0, 6+2) = 0 (변화 없음)
dist[1][0] = min(7, dist[1][3]+dist[3][0]) = min(7, 3+2) = 5
dist[2][0] = min(5, dist[2][3]+dist[3][0]) = min(5, 1+2) = 3
dist[2][1] = min(8, dist[2][3]+dist[3][1]) = min(8, 1+5) = 6

최종 거리 행렬:
     0    1    2    3
0 [  0    3    5    6  ]
1 [  5    0    2    3  ]
2 [  3    6    0    1  ]
3 [  2    5    7    0  ]
```

## 실전 팁

### 활용 노하우

- 정점 수 V ≤ 500이고 모든 쌍 최단 거리가 필요하면 플로이드-워셜이 최적이다. 구현이 3중 루프로 매우 간단하다
- "특정 정점을 반드시 경유하는 최단 경로"는 dist[i][k] + dist[k][j]로 바로 구할 수 있다
- 도달 가능성(Transitive Closure) 문제는 가중치 대신 bool 값으로 OR 연산을 적용하면 된다
- 음수 사이클 탐지: 알고리즘 수행 후 dist[i][i] < 0인 정점이 있으면 음수 사이클이 존재한다

### 자주 하는 실수

- k 루프를 가장 바깥에 두지 않는 실수: `for k → for i → for j` 순서가 반드시 지켜져야 한다. 순서가 바뀌면 DP 상태 전이가 깨진다
- INF 값 설정 오류: INF를 너무 크게 설정하면 `dist[i][k] + dist[k][j]`에서 오버플로우가 발생한다. INF = 10⁹ 정도로 설정하고, 갱신 전에 `dist[i][k] < INF && dist[k][j] < INF` 조건을 확인하자
- 자기 자신 거리 초기화 누락: dist[i][i] = 0으로 초기화해야 한다
- 무방향 그래프에서 양방향 간선 처리 누락: 간선 (u, v, w)에 대해 dist[u][v]와 dist[v][u] 모두 설정해야 한다

### 엣지 케이스

- 정점이 1개인 경우: dist[0][0] = 0만 존재한다
- 연결되지 않은 정점 쌍: dist[i][j]가 INF로 남아 있으면 도달 불가능하다
- 자기 자신으로의 음수 사이클: dist[i][i] < 0이면 해당 정점이 음수 사이클에 포함된다
- 중복 간선: 같은 (u, v) 쌍에 여러 간선이 있으면 최소 가중치만 사용한다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 음수 가중치 | 적합한 상황 |
| --- | --- | --- | --- |
| 플로이드-워셜 | O(V³) | 가능 (음수 사이클 탐지) | 모든 쌍 최단 거리, V ≤ 500 |
| 다익스트라 (V번 반복) | O(V × (V+E) log V) | 불가 | 모든 쌍, 음수 없는 희소 그래프 |
| 벨만-포드 (V번 반복) | O(V² × E) | 가능 | 모든 쌍, 음수 가중치 희소 그래프 |
| 존슨 알고리즘 | O(VE + V(V+E) log V) | 가능 (리웨이팅) | 모든 쌍, 음수 가중치 희소 그래프 |

- 정점 수가 적고 (V ≤ 500) 모든 쌍이 필요하면 → 플로이드-워셜 (구현 간단)
- 단일 출발점이면 → 다익스트라 또는 벨만-포드
- 희소 그래프 + 음수 가중치 + 모든 쌍이면 → 존슨 알고리즘
- 밀집 그래프 + 모든 쌍이면 → 플로이드-워셜 (간선 수와 무관)
