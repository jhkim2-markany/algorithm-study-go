# 모든 쌍 최단 거리 - 해설

## 접근 방식

1. N×N 크기의 거리 행렬을 초기화한다. 자기 자신은 0, 나머지는 무한대로 설정한다.
2. M개의 간선을 입력받으며, 같은 출발-도착 쌍에 여러 도로가 있으면 최솟값만 저장한다.
3. 플로이드-워셜 알고리즘의 3중 루프를 수행한다. 중간 정점 k를 바깥 루프에 두고, 모든 (i, j) 쌍에 대해 k를 경유하는 경로가 더 짧으면 갱신한다.
4. 결과를 출력할 때 도달 불가능한 경우(INF)는 0으로 출력한다.

## 핵심 아이디어

플로이드-워셜의 기본 형태를 그대로 적용하는 문제이다. 핵심은 3중 루프에서 중간 정점 k가 가장 바깥 루프에 위치해야 한다는 점이다. 이는 동적 프로그래밍의 상태 전이 순서를 보장하기 위함이다.

같은 출발-도착 쌍에 여러 간선이 있을 수 있으므로, 입력 시 최솟값만 저장하는 처리가 필요하다. 또한 오버플로우를 방지하기 위해 INF 값끼리의 덧셈을 피해야 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N³) |
| 공간 복잡도 | O(N²) |

N이 최대 100이므로 3중 루프는 100³ = 1,000,000번 연산으로 충분히 빠르다.

## 대안적 접근

각 정점에서 다익스트라를 N번 실행하면 O(N × (N + M) log N)으로도 풀 수 있다. 하지만 N이 작고 모든 쌍 정보가 필요한 경우 플로이드-워셜이 구현이 더 간단하다.
