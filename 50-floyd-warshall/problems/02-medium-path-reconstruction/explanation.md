# 경로 복원 - 해설

## 접근 방식

1. N×N 크기의 거리 행렬 dist와 경로 복원용 next 행렬을 초기화한다.
2. 간선을 입력받아 dist와 next를 설정한다. next[i][j] = j (직접 연결된 경우).
3. 플로이드-워셜 3중 루프를 수행하며, dist가 갱신될 때 next[i][j] = next[i][k]로 함께 갱신한다.
4. 각 질의에 대해 dist[s][e]로 최소 비용을 확인하고, next 행렬을 따라가며 경로를 복원한다.

## 핵심 아이디어

경로 복원의 핵심은 next 행렬이다. next[i][j]는 "i에서 j로 가는 최단 경로에서 i 바로 다음에 방문할 정점"을 저장한다.

dist[i][j]가 k를 경유하여 갱신되면, i에서 j로 가는 최단 경로는 "i에서 k까지의 최단 경로 + k에서 j까지의 최단 경로"로 구성된다. 따라서 i 다음에 방문할 정점은 i에서 k로 갈 때의 다음 정점, 즉 next[i][k]가 된다.

경로 복원 시에는 시작 정점에서 next를 반복적으로 따라가면 도착 정점에 도달한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N³ + Q×N) |
| 공간 복잡도 | O(N²) |

플로이드-워셜에 O(N³), 각 질의의 경로 복원에 최대 O(N)이 소요된다. N ≤ 200이므로 충분히 빠르다.

## 대안적 접근

각 질의마다 다익스트라를 실행하고 경로를 역추적할 수도 있다. 하지만 질의 수가 많고 N이 작은 경우 플로이드-워셜로 전처리하는 것이 효율적이다.
