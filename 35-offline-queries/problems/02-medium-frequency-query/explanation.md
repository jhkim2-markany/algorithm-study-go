# 구간 빈도수 쿼리 - 해설

## 접근 방식

1. Mo's 알고리즘으로 구간 쿼리를 오프라인 처리한다
2. 각 값의 등장 횟수를 관리하는 cnt 배열과, 특정 빈도를 가진 값의 개수를 관리하는 freqCnt 배열을 함께 유지한다
3. 원소를 추가/제거할 때 cnt와 freqCnt를 동시에 갱신한다
4. 쿼리의 답은 freqCnt[K]로 O(1)에 구한다

## 핵심 아이디어

이 문제의 핵심은 "빈도의 빈도"를 관리하는 것이다. cnt[v]는 값 v의 등장 횟수이고, freqCnt[f]는 등장 횟수가 정확히 f인 서로 다른 값의 개수이다. 원소를 추가할 때 cnt[v]가 c에서 c+1로 변하면, freqCnt[c]를 1 감소시키고 freqCnt[c+1]을 1 증가시킨다. 이렇게 하면 각 쿼리에 대해 freqCnt[K]를 바로 읽으면 되므로 답을 O(1)에 구할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + Q) × √N) |
| 공간 복잡도 | O(N + Q) |

Mo's 알고리즘의 포인터 이동에 O((N + Q) × √N)이 소요되며, add/remove 연산이 O(1)이므로 전체 복잡도는 동일하다. freqCnt 배열 덕분에 각 쿼리의 답도 O(1)에 구할 수 있다.

## 대안적 접근

- 값의 범위가 작다면 각 값별로 등장 위치를 저장하고, 이진 탐색으로 구간 내 등장 횟수를 구한 뒤 K와 비교할 수 있다. 하지만 모든 값에 대해 반복해야 하므로 비효율적이다.
