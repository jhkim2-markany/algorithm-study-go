# 구간 역전 쌍 카운팅 - 해설

## 접근 방식

1. Mo's 알고리즘으로 구간 쿼리를 오프라인 처리한다
2. BIT(Binary Indexed Tree)를 사용하여 현재 구간에 포함된 값들의 빈도를 관리한다
3. 원소를 추가/제거할 때 BIT를 이용하여 역전 쌍의 변화량을 O(log N)에 계산한다
4. 오른쪽에 추가할 때는 "현재 구간에서 새 값보다 큰 값의 개수"가 역전 쌍 증가분이다
5. 왼쪽에 추가할 때는 "현재 구간에서 새 값보다 작은 값의 개수"가 역전 쌍 증가분이다

## 핵심 아이디어

역전 쌍 문제에서 Mo's 알고리즘을 적용할 때, add/remove 연산의 방향(왼쪽/오른쪽)에 따라 역전 쌍 변화량 계산이 달라진다는 점이 핵심이다.

- 오른쪽 끝에 값 v를 추가하면, 기존 구간에서 v보다 큰 값들과 역전 쌍이 형성된다. BIT에서 전체 개수 - v 이하 개수로 O(log N)에 구한다.
- 왼쪽 끝에 값 v를 추가하면, 기존 구간에서 v보다 작은 값들과 역전 쌍이 형성된다. BIT에서 v-1 이하의 개수로 O(log N)에 구한다.

홀짝 최적화를 적용하여 홀수 블록에서는 R을 내림차순으로 정렬하면, R 포인터의 방향 전환이 줄어들어 실제 수행 시간이 개선된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + Q) × √N × log N) |
| 공간 복잡도 | O(N + Q) |

Mo's 알고리즘의 포인터 이동에 O((N + Q) × √N)이 소요되고, 각 이동마다 BIT 연산에 O(log N)이 필요하므로 전체 복잡도는 O((N + Q) × √N × log N)이다.

## 대안적 접근

- 머지 소트 트리(Merge Sort Tree)를 사용하면 각 쿼리를 O(log² N)에 처리할 수 있다. 전체 복잡도는 O(Q × log² N)으로, 쿼리 수가 많고 N이 클 때 Mo's 알고리즘보다 유리할 수 있다.
