# 구간 서로 다른 수 개수 - 해설

## 접근 방식

1. 모든 쿼리를 미리 입력받아 Mo's 알고리즘 순서로 정렬한다
2. 블록 크기 B = √N으로 설정하고, L의 블록 번호 기준 1차 정렬, R 기준 2차 정렬한다
3. 투 포인터(curL, curR)를 유지하며 구간을 확장/축소한다
4. 원소를 추가할 때 빈도가 0→1이 되면 서로 다른 수를 증가시킨다
5. 원소를 제거할 때 빈도가 1→0이 되면 서로 다른 수를 감소시킨다
6. 처리 완료 후 원래 쿼리 순서에 맞게 답을 출력한다

## 핵심 아이디어

Mo's 알고리즘은 쿼리를 √N 크기의 블록 단위로 정렬하여, 투 포인터의 총 이동 거리를 최소화한다. 같은 블록 내에서 R은 단조 증가하므로 R 포인터의 이동이 O(N)이고, L 포인터는 블록 내에서 최대 √N만큼 이동한다. 빈도 배열(cnt)을 사용하여 add/remove 연산을 O(1)에 수행하는 것이 핵심이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + Q) × √N) |
| 공간 복잡도 | O(N + Q) |

쿼리 정렬에 O(Q log Q), 포인터 이동에 O((N + Q) × √N)이 소요된다. add/remove가 O(1)이므로 전체 복잡도는 O((N + Q) × √N)이다.

## 대안적 접근

- 세그먼트 트리 + 오프라인 처리: 쿼리를 R 기준으로 정렬하고, 각 값의 마지막 등장 위치를 세그먼트 트리에 기록하여 구간 합으로 서로 다른 수를 구할 수 있다. 시간 복잡도는 O((N + Q) × log N)이다.
