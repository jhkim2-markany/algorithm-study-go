# 오프라인 쿼리

## 개념

오프라인 쿼리(Offline Query)는 모든 쿼리를 미리 받아놓은 뒤, 입력 순서가 아닌 효율적인 순서로 재배열하여 처리하는 기법이다. 온라인 쿼리가 각 쿼리를 입력 즉시 처리하는 것과 달리, 오프라인 쿼리는 전체 쿼리를 한꺼번에 볼 수 있다는 점을 활용하여 시간 복잡도를 크게 줄인다.

대표적인 오프라인 쿼리 알고리즘으로 Mo's 알고리즘이 있다. Mo's 알고리즘은 배열에 대한 구간 쿼리를 제곱근 분할(sqrt decomposition) 기반으로 정렬하여, 투 포인터를 이동시키며 답을 구한다. 업데이트 없이 구간 쿼리만 주어지는 문제에서 매우 효과적이다.

## 동작 원리

### 1. 오프라인 쿼리의 기본 아이디어

온라인 처리에서는 쿼리가 들어올 때마다 독립적으로 처리해야 하지만, 오프라인 처리에서는 다음과 같은 전략을 사용한다:

1. 모든 쿼리를 미리 입력받는다
2. 쿼리를 효율적인 순서로 정렬한다
3. 정렬된 순서대로 쿼리를 처리한다
4. 원래 입력 순서에 맞게 답을 출력한다

### 2. Mo's 알고리즘

Mo's 알고리즘은 배열에 대한 구간 쿼리 [L, R]을 효율적으로 처리하는 오프라인 알고리즘이다.

**블록 분할:**

1. 배열의 크기 N에 대해 블록 크기 B = √N으로 설정한다
2. 각 쿼리의 L 값이 속하는 블록 번호를 기준으로 쿼리를 정렬한다
3. 같은 블록 내에서는 R 값을 기준으로 오름차순 정렬한다

**쿼리 처리:**

1. 현재 구간 [curL, curR]을 유지한다
2. 다음 쿼리 [L, R]로 이동할 때, curL과 curR을 한 칸씩 이동시킨다
3. 원소를 추가(add)하거나 제거(remove)하며 현재 답을 갱신한다
4. 목표 구간에 도달하면 답을 기록한다

**정렬 기준의 핵심:**
- L을 블록 단위로 묶으면, 같은 블록 내에서 R이 단조 증가하므로 R 포인터의 총 이동 거리가 O(N)이 된다
- L 포인터는 같은 블록 내에서 최대 √N만큼 이동하므로, Q개 쿼리에 대해 총 O(Q × √N)이다

### 3. 포인터 이동 (Add / Remove)

Mo's 알고리즘의 핵심은 구간에 원소를 추가하거나 제거하는 연산을 O(1) 또는 O(log N)에 수행하는 것이다.

1. 구간을 오른쪽으로 확장할 때: curR을 증가시키고 arr[curR]을 추가한다
2. 구간을 왼쪽으로 확장할 때: curL을 감소시키고 arr[curL]을 추가한다
3. 구간을 오른쪽에서 축소할 때: arr[curR]을 제거하고 curR을 감소시킨다
4. 구간을 왼쪽에서 축소할 때: arr[curL]을 제거하고 curL을 증가시킨다

### 4. 홀짝 최적화 (Odd-Even Optimization)

블록 번호가 홀수인 쿼리는 R을 내림차순으로 정렬하면, R 포인터의 방향 전환 횟수가 줄어들어 실제 수행 시간이 개선된다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| Mo's 알고리즘 (쿼리 정렬) | O(Q log Q) |
| Mo's 알고리즘 (쿼리 처리) | O((N + Q) × √N) |
| Mo's 알고리즘 (전체) | O((N + Q) × √N) |
| 공간 복잡도 | O(N + Q) |

블록 크기 B = √N일 때 최적이며, add/remove 연산이 O(1)인 경우 위 복잡도가 성립한다. add/remove가 O(log N)이면 전체 복잡도는 O((N + Q) × √N × log N)이 된다.

## 적합한 문제 유형

- 배열의 구간에서 서로 다른 원소의 개수를 구하는 문제
- 구간 내 특정 조건을 만족하는 원소의 빈도를 구하는 문제
- 업데이트 없이 다수의 구간 쿼리를 처리하는 문제
- 구간 내 원소의 통계(합, 최빈값, 중앙값 등)를 구하는 문제
- 세그먼트 트리로 풀기 어려운 비가환적 구간 연산 문제
- 쿼리를 정렬하여 스위핑으로 처리할 수 있는 문제
- 구간 내 역전 쌍(inversion) 개수를 구하는 문제


## 단계별 추적 (Trace)

### 예시: Mo's 알고리즘으로 구간 내 서로 다른 원소 개수 구하기

```
배열: arr = [1, 2, 1, 3, 2, 1]  (N = 6)
블록 크기: B = √6 ≈ 2

쿼리 목록 (0-indexed):
  Q0: [0, 3]  Q1: [1, 4]  Q2: [2, 5]  Q3: [0, 1]

1단계: 쿼리 정렬 (L의 블록 번호 기준, 같은 블록이면 R 기준)
  Q3: [0, 1] → 블록 0, R=1
  Q0: [0, 3] → 블록 0, R=3
  Q1: [1, 4] → 블록 0, R=4
  Q2: [2, 5] → 블록 1, R=5

정렬 결과: Q3 → Q0 → Q1 → Q2

2단계: 포인터 이동하며 쿼리 처리

초기 상태: curL = 0, curR = -1, cnt = {}, answer = 0

--- Q3: [0, 1] ---
  curR: -1 → 1 (오른쪽 확장)
    add(arr[0]=1): cnt[1]=1, answer=1
    add(arr[1]=2): cnt[2]=1, answer=2
  결과: ans[Q3] = 2

--- Q0: [0, 3] ---
  curR: 1 → 3 (오른쪽 확장)
    add(arr[2]=1): cnt[1]=2, answer=2 (이미 존재)
    add(arr[3]=3): cnt[3]=1, answer=3
  결과: ans[Q0] = 3

--- Q1: [1, 4] ---
  curL: 0 → 1 (왼쪽 축소)
    remove(arr[0]=1): cnt[1]=1, answer=3 (아직 존재)
  curR: 3 → 4 (오른쪽 확장)
    add(arr[4]=2): cnt[2]=2, answer=3
  결과: ans[Q1] = 3

--- Q2: [2, 5] ---
  curL: 1 → 2 (왼쪽 축소)
    remove(arr[1]=2): cnt[2]=1, answer=3
  curR: 4 → 5 (오른쪽 확장)
    add(arr[5]=1): cnt[1]=2, answer=3
  결과: ans[Q2] = 3

3단계: 원래 순서로 출력
  Q0: 3, Q1: 3, Q2: 3, Q3: 2
```

### 포인터 이동 시각화

```
배열 인덱스:  0  1  2  3  4  5
배열 값:     [1, 2, 1, 3, 2, 1]

Q3 [0,1]:   [==]                  → 서로 다른 수: 2
Q0 [0,3]:   [========]            → 서로 다른 수: 3
Q1 [1,4]:      [========]         → 서로 다른 수: 3
Q2 [2,5]:         [========]      → 서로 다른 수: 3
```

## 실전 팁

### 활용 노하우

- 블록 크기는 보통 `√N`으로 설정하지만, 쿼리 수 Q가 많으면 `N / √Q`가 더 효율적일 수 있다
- add/remove 연산을 O(1)로 만드는 것이 핵심이다. 빈도 배열(cnt)을 활용하자
- 쿼리 정렬 시 홀짝 최적화를 적용하면 실제 수행 시간이 20~30% 개선된다
- Mo's 알고리즘은 업데이트가 없는 문제에만 적용 가능하다. 업데이트가 있으면 Mo's with Update(시간 축 추가)를 사용한다
- 쿼리 결과를 원래 순서로 복원하기 위해 쿼리에 인덱스를 함께 저장해야 한다

### 자주 하는 실수

- 쿼리 정렬 후 원래 인덱스를 저장하지 않아 결과 순서가 뒤섞이는 실수
- 블록 크기를 0으로 설정하여 나눗셈 오류가 발생하는 실수 (N=0인 경우)
- add와 remove의 순서를 잘못 구현하는 실수: 확장 시 먼저 포인터를 이동하고 add, 축소 시 먼저 remove하고 포인터 이동
- curL, curR 초기값 설정 오류: curL=0, curR=-1로 시작해야 빈 구간에서 출발한다
- 빈도 배열의 크기를 배열 값의 최대값이 아닌 배열 크기로 잡는 실수

### 엣지 케이스

- 쿼리의 L == R인 경우: 단일 원소 구간
- 배열의 모든 원소가 같은 경우: 모든 구간의 서로 다른 원소 수가 1
- 쿼리가 전체 배열을 커버하는 경우: [0, N-1]
- N = 1인 경우: 블록 크기가 1이 되어 정렬 기준이 단순해짐
- 쿼리 수가 0인 경우: 아무 처리도 하지 않음

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 업데이트 지원 | 적합한 상황 |
| --- | --- | --- | --- |
| Mo's 알고리즘 | O((N+Q)√N) | 불가 | 정적 배열의 구간 쿼리 |
| 세그먼트 트리 | O(Q log N) | 가능 | 결합 가능한 연산의 구간 쿼리 |
| BIT (펜윅 트리) | O(Q log N) | 가능 | 누적합 기반 구간 쿼리 |
| Mo's with Update | O((N+Q)^(5/3)) | 가능 | 업데이트가 있는 구간 쿼리 |
| 머지 소트 트리 | O(Q log²N) | 불가 | 구간 내 k번째 수 등 |

**선택 기준:**
- 구간 쿼리 + 업데이트 없음 + 비가환 연산 → Mo's 알고리즘
- 구간 쿼리 + 업데이트 있음 + 결합 가능 연산 → 세그먼트 트리
- 구간 합/최솟값 등 단순 연산 → BIT 또는 세그먼트 트리
- 구간 내 서로 다른 원소 수 (오프라인) → Mo's 알고리즘
- 구간 내 서로 다른 원소 수 (온라인) → 퍼시스턴트 세그먼트 트리
