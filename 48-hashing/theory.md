# 해싱 (Hashing)

## 개념

해싱(Hashing)은 임의 길이의 데이터를 고정 크기의 정수 값(해시 값)으로 변환하는 기법이다. 코딩테스트에서 해싱은 주로 문자열 해싱에 초점을 맞추며, 다항식 해싱(Polynomial Hashing)을 기반으로 문자열 비교, 패턴 매칭, 부분 문자열 검색 등을 효율적으로 수행한다.

기본적인 해시맵(05-hash)이 키-값 저장에 초점을 맞추는 반면, 이 장에서 다루는 해싱은 문자열을 수학적으로 변환하여 O(1)에 비교하는 알고리즘적 기법에 집중한다. 대표적으로 라빈-카프(Rabin-Karp) 알고리즘은 롤링 해시를 이용하여 문자열 패턴 매칭을 O(N+M)에 수행한다.

해시 충돌(서로 다른 문자열이 같은 해시 값을 가지는 경우)을 줄이기 위해 이중 해싱(Double Hashing)이나 큰 소수 모듈러를 사용하는 것이 일반적이다.

## 동작 원리

### 1. 다항식 해싱 (Polynomial Hashing)

문자열 S = s₀s₁...sₙ₋₁을 다항식으로 해석하여 해시 값을 계산한다:

```
hash(S) = (s₀ × p⁰ + s₁ × p¹ + s₂ × p² + ... + sₙ₋₁ × pⁿ⁻¹) mod M
```

- p: 기저(base), 보통 31 또는 37 등 소수를 사용한다
- M: 모듈러, 보통 10⁹+7 또는 10⁹+9 등 큰 소수를 사용한다
- sᵢ: 문자의 정수 값 (예: 'a' = 1, 'b' = 2, ...)

### 2. 롤링 해시 (Rolling Hash)

문자열의 연속 부분 문자열 해시를 효율적으로 계산하는 기법이다. 이전 해시 값에서 O(1)에 다음 해시 값을 구한다.

1. 접두사 해시 배열 H를 전처리한다: H[i] = hash(S[0..i-1])
2. 거듭제곱 배열 P를 전처리한다: P[i] = p^i mod M
3. 부분 문자열 S[l..r]의 해시를 O(1)에 구한다:
   - hash(S[l..r]) = (H[r+1] - H[l] × P[r-l+1]) mod M

### 3. 라빈-카프 알고리즘 (Rabin-Karp)

롤링 해시를 이용한 문자열 패턴 매칭 알고리즘이다.

1. 패턴 P의 해시 값을 계산한다
2. 텍스트 T에서 길이 |P|인 모든 부분 문자열의 해시를 롤링 해시로 계산한다
3. 해시 값이 일치하면 실제 문자열을 비교하여 확인한다 (해시 충돌 방지)
4. 해시 값이 다르면 확실히 다른 문자열이므로 건너뛴다

### 4. 이중 해싱 (Double Hashing)

해시 충돌 확률을 줄이기 위해 서로 다른 두 해시 함수를 동시에 사용한다.

1. 서로 다른 기저 p₁, p₂와 모듈러 M₁, M₂를 선택한다
2. 각 문자열에 대해 두 개의 해시 값 (h₁, h₂)를 계산한다
3. 두 해시 값이 모두 일치해야 같은 문자열로 판정한다
4. 충돌 확률이 1/M₁ × 1/M₂로 크게 감소한다

### 5. 해시 충돌 처리

해시 충돌은 서로 다른 입력이 같은 해시 값을 가지는 현상이다.

- 큰 소수 모듈러 사용: M을 10⁹+7 이상의 큰 소수로 설정한다
- 이중 해싱: 두 개의 독립적인 해시 함수를 사용한다
- 해시 일치 시 실제 비교: 해시가 같으면 원본 문자열을 직접 비교한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 다항식 해시 계산 | O(N) |
| 접두사 해시 전처리 | O(N) |
| 부분 문자열 해시 쿼리 | O(1) |
| 라빈-카프 패턴 매칭 (평균) | O(N + M) |
| 라빈-카프 패턴 매칭 (최악) | O(NM) |
| 공간 복잡도 | O(N) |

N은 텍스트 길이, M은 패턴 길이이다. 라빈-카프의 최악 시간은 모든 위치에서 해시 충돌이 발생하는 경우이며, 이중 해싱을 사용하면 실질적으로 O(N+M)에 가깝다.

## 적합한 문제 유형

- 문자열 패턴 매칭 문제 (라빈-카프)
- 부분 문자열 비교 문제 (롤링 해시)
- 가장 긴 공통 부분 문자열 찾기 (이분 탐색 + 해싱)
- 문자열 중복 검사 문제
- 회문(팰린드롬) 판별 문제
- 여러 문자열의 동일 여부를 빠르게 비교하는 문제
- 부분 문자열의 개수를 세는 문제

## 단계별 추적 (Trace)

### 예시 입력

텍스트 T = "abcabcabc", 패턴 P = "cab"에 대해 라빈-카프 알고리즘으로 패턴 매칭을 수행한다.
기저 p = 31, 모듈러 M = 1000000007, 문자 값: a=1, b=2, c=3

### 실행 과정

**1단계: 패턴 해시 계산**

```
hash(P) = hash("cab")
        = 3×31⁰ + 1×31¹ + 2×31²
        = 3 + 31 + 1922
        = 1956
```

**2단계: 접두사 해시 배열 H 전처리**

```
H[0] = 0
H[1] = 1                          (a)
H[2] = 1 + 2×31 = 63              (ab)
H[3] = 63 + 3×31² = 2945          (abc)
H[4] = 2945 + 1×31³ = 32736       (abca)
H[5] = 32736 + 2×31⁴ = 1878178   (abcab)
H[6] = ...                        (abcabc)
```

**3단계: 거듭제곱 배열 P 전처리**

```
P[0]=1, P[1]=31, P[2]=961, P[3]=29791, ...
```

**4단계: 각 위치에서 길이 3 부분 문자열 해시 비교**

```
위치 0: hash("abc") = (H[3] - H[0]×P[3]) mod M
        → 패턴 해시 1956과 비교 → 불일치

위치 1: hash("bca") = (H[4] - H[1]×P[3]) mod M
        → 패턴 해시 1956과 비교 → 불일치

위치 2: hash("cab") = (H[5] - H[2]×P[3]) mod M
        → 패턴 해시 1956과 비교 → 일치! ✓
        → 실제 문자열 비교로 확인: "cab" == "cab" ✓

위치 5: hash("cab") → 일치! ✓
```

**결과: 패턴 "cab"은 위치 2, 5에서 발견됨**

## 실전 팁

### 활용 노하우

- 코딩테스트에서 문자열 비교가 빈번하면 해싱을 먼저 고려하자. 접두사 해시 전처리 후 O(1)에 부분 문자열을 비교할 수 있다
- 이중 해싱을 기본으로 사용하면 해시 충돌 확률이 극히 낮아진다. 기저 (31, 37), 모듈러 (10⁹+7, 10⁹+9) 조합이 일반적이다
- 가장 긴 공통 부분 문자열 문제는 이분 탐색 + 해싱으로 O(N log N)에 풀 수 있다
- 문자 값을 0이 아닌 1부터 시작하자. 'a'=0으로 하면 "a", "aa", "aaa"의 해시가 모두 0이 되는 문제가 발생한다

### 자주 하는 실수

- 모듈러 연산에서 음수 처리 누락: `(H[r+1] - H[l] * P[r-l+1]) % M`이 음수가 될 수 있으므로 `((결과 % M) + M) % M`으로 보정해야 한다
- 기저와 모듈러를 너무 작게 설정: 충돌 확률이 높아진다. 모듈러는 10⁹ 이상의 소수를 사용하자
- 해시 일치 시 실제 문자열 비교를 생략: 해시 충돌 가능성이 있으므로 반드시 원본 비교를 해야 한다 (단, 이중 해싱 사용 시 생략 가능)
- 거듭제곱 배열 크기 부족: 문자열 최대 길이만큼 미리 계산해야 한다

### 엣지 케이스

- 빈 문자열: 해시 값이 0이 되므로 별도 처리가 필요하다
- 패턴이 텍스트보다 긴 경우: 매칭 결과가 없음을 바로 반환한다
- 모든 문자가 동일한 문자열: 해시 충돌이 빈번할 수 있으므로 이중 해싱이 필수적이다
- 단일 문자 패턴: 해싱보다 단순 순회가 더 효율적이다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| 라빈-카프 (해싱) | O(N+M) 평균 | 롤링 해시, 다중 패턴 가능 | 부분 문자열 비교, 다중 패턴 매칭 |
| KMP | O(N+M) | 실패 함수 기반, 최악 보장 | 단일 패턴 매칭 (안정적) |
| 아호-코라식 | O(N+M+Z) | 트라이 + 실패 링크 | 다중 패턴 동시 매칭 |
| Z 알고리즘 | O(N+M) | Z 배열 기반 | 패턴 매칭, 문자열 주기 |
| 접미사 배열 | O(N log N) | 정렬 기반 | 부분 문자열 검색, 사전 순 정렬 |

- 단일 패턴 매칭이면 → KMP (최악 시간 보장)
- 부분 문자열 비교가 빈번하면 → 해싱 (O(1) 비교)
- 다중 패턴 동시 매칭이면 → 아호-코라식
- 가장 긴 공통 부분 문자열이면 → 이분 탐색 + 해싱
