# 가장 긴 공통 부분 문자열 - 해설

## 접근 방식

1. 두 문자열의 접두사 해시를 각각 전처리한다
2. 이분 탐색으로 공통 부분 문자열의 길이를 결정한다
3. 각 길이에 대해 해싱으로 공통 부분 문자열 존재 여부를 확인한다

## 핵심 아이디어

"길이 L인 공통 부분 문자열이 존재하면, 길이 L-1인 공통 부분 문자열도 반드시 존재한다"는 단조성을 이용하여 이분 탐색을 적용한다.

특정 길이 L에 대해 공통 부분 문자열이 존재하는지 확인하는 방법:
1. 문자열 A의 길이 L인 모든 부분 문자열의 이중 해시를 해시 집합에 저장한다
2. 문자열 B의 길이 L인 부분 문자열의 해시가 집합에 있는지 확인한다
3. 접두사 해시를 전처리해두면 각 부분 문자열의 해시를 O(1)에 구할 수 있다

이분 탐색의 각 단계에서 O(N+M) 시간이 소요되므로, 전체 시간 복잡도는 O((N+M) log min(N,M))이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N+M) log min(N,M)) |
| 공간 복잡도 | O(N + M) |

N은 문자열 A의 길이, M은 문자열 B의 길이이다. 이분 탐색 O(log min(N,M))번 반복하며, 각 반복에서 해시 집합 구성과 탐색에 O(N+M)이 소요된다.

## 대안적 접근

- 접미사 배열 + LCP 배열: O((N+M) log(N+M))에 풀 수 있으며, 해시 충돌 걱정이 없다.
- DP 방법: O(NM) 시간과 공간이 필요하여 문자열이 길 때 비효율적이다.
- 접미사 트리: O(N+M)에 풀 수 있지만 구현이 복잡하다.
