# 문자열 해시 비교 - 해설

## 접근 방식

1. 각 문자열의 이중 해시 값을 계산한다
2. 해시 값을 키로 하는 맵에 문자열 개수를 누적한다
3. 같은 해시 값을 가진 문자열이 k개이면 쌍의 수는 k×(k-1)/2이다

## 핵심 아이디어

다항식 해싱을 사용하면 문자열을 정수 값으로 변환하여 O(1)에 비교할 수 있다. 이중 해싱(두 개의 서로 다른 기저와 모듈러)을 사용하면 해시 충돌 확률이 극히 낮아져 사실상 정확한 비교가 가능하다.

같은 해시 값을 가진 문자열의 개수를 맵으로 관리하면, 모든 쌍을 직접 비교하지 않고도 조합 공식 nC2 = k×(k-1)/2로 쌍의 수를 구할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × L) |
| 공간 복잡도 | O(N) |

N은 문자열 개수, L은 문자열의 평균 길이이다. 각 문자열의 해시를 O(L)에 계산하고, 맵 연산은 평균 O(1)이다.

## 대안적 접근

- 정렬 기반: 문자열을 사전순으로 정렬한 뒤 연속된 같은 문자열을 세면 O(NL log N)이다.
- 직접 비교: 모든 쌍을 직접 비교하면 O(N²L)로 N이 클 때 시간 초과가 발생한다.
