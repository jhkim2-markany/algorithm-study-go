# 가장 긴 고유 문자 부분 문자열 - 해설

## 접근 방식

1. 슬라이딩 윈도우를 사용하여 중복 없는 연속 부분 문자열을 관리한다
2. `left`와 `right` 두 포인터로 현재 윈도우의 범위를 나타낸다
3. `right`를 한 칸씩 확장하며 새 문자를 윈도우에 추가한다
4. 새로 추가한 문자가 윈도우 내에 이미 존재하면(중복), `left`를 이동하여 중복을 해소한다
5. 매 단계에서 현재 윈도우 길이와 최대 길이를 비교하여 갱신한다

## 핵심 아이디어

가변 크기 슬라이딩 윈도우 기법을 사용한다. 윈도우 내 각 문자의 등장 횟수를 크기 26인 배열로 관리하여, 중복 여부를 O(1)에 판단한다.

`right` 포인터가 새 문자를 추가했을 때 해당 문자의 카운트가 2 이상이 되면 중복이 발생한 것이다. 이때 `left`를 오른쪽으로 이동하면서 빠지는 문자의 카운트를 감소시켜, 중복이 해소될 때까지 윈도우를 축소한다.

각 문자가 윈도우에 들어오고 나가는 횟수가 최대 1번이므로, 전체 시간 복잡도는 O(N)이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(1) |

`left`와 `right` 포인터가 각각 최대 N번 이동하므로 시간 복잡도는 O(N)이다. 문자 카운트 배열의 크기가 26으로 고정이므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **해시맵으로 마지막 위치 저장**: 각 문자의 마지막 등장 위치를 해시맵에 저장하고, 중복 발생 시 `left`를 해당 위치 + 1로 점프하는 방법이 있다. 시간 복잡도는 동일하게 O(N)이지만, `left`의 이동 횟수를 줄일 수 있다.
