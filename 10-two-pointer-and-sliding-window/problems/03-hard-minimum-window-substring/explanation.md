# 최소 윈도우 부분 문자열 - 해설

## 접근 방식

1. 문자열 T에 포함된 각 문자의 필요 횟수를 배열에 기록한다
2. 슬라이딩 윈도우를 사용하여 S를 탐색한다
3. `right` 포인터를 확장하며 윈도우에 문자를 추가한다
4. 윈도우가 T의 모든 문자를 필요한 횟수만큼 포함하면, `left`를 축소하며 최소 길이를 갱신한다
5. 조건이 깨질 때까지 `left`를 이동한 후, 다시 `right`를 확장한다

## 핵심 아이디어

가변 크기 슬라이딩 윈도우와 문자 빈도 카운팅을 결합한다. 핵심은 "충족된 고유 문자 수(`formed`)"를 추적하는 것이다.

- `need[ch]`: T에서 문자 `ch`가 필요한 횟수
- `window[ch]`: 현재 윈도우에서 문자 `ch`의 등장 횟수
- `required`: T에 포함된 고유 문자의 수
- `formed`: 윈도우에서 필요 횟수를 충족한 고유 문자의 수

`formed == required`가 되면 윈도우가 T의 모든 문자를 포함한 것이다. 이때 `left`를 이동하여 윈도우를 최소화한다.

`right`가 문자를 추가할 때 `window[ch] == need[ch]`가 되면 `formed`를 증가시키고, `left`가 문자를 제거할 때 `window[ch] < need[ch]`가 되면 `formed`를 감소시킨다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(|S| + |T|) |
| 공간 복잡도 | O(1) |

`left`와 `right` 포인터가 각각 최대 |S|번 이동하므로 시간 복잡도는 O(|S|)이다. T의 문자 빈도를 세는 데 O(|T|)이 추가된다. 문자 빈도 배열의 크기가 128로 고정이므로 공간 복잡도는 O(1)이다.

## 대안적 접근

- **최적화된 윈도우**: S에서 T에 포함된 문자만 필터링하여 인덱스 리스트를 만들고, 이 리스트에 대해 슬라이딩 윈도우를 적용하면 S에 T의 문자가 드물게 분포할 때 성능이 향상된다.
