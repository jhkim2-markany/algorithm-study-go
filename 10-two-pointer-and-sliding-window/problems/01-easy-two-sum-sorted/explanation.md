# 정렬된 배열의 두 수의 합 - 해설

## 접근 방식

1. 배열이 오름차순으로 정렬되어 있으므로 투 포인터 기법을 적용한다
2. 왼쪽 포인터 `left`를 배열의 시작(인덱스 0)에, 오른쪽 포인터 `right`를 배열의 끝(인덱스 N-1)에 놓는다
3. 두 포인터가 가리키는 값의 합을 목표값 T와 비교한다
4. 합이 T보다 작으면 `left`를 오른쪽으로 이동하여 합을 증가시킨다
5. 합이 T보다 크면 `right`를 왼쪽으로 이동하여 합을 감소시킨다
6. 합이 T와 같으면 해당 인덱스를 출력한다

## 핵심 아이디어

배열이 정렬되어 있다는 성질을 활용한다. 현재 합이 목표값보다 작으면 더 큰 값이 필요하므로 왼쪽 포인터를 오른쪽으로 이동하고, 합이 목표값보다 크면 더 작은 값이 필요하므로 오른쪽 포인터를 왼쪽으로 이동한다.

이 방식이 정확한 이유는, `left`를 오른쪽으로 이동하면 합이 반드시 증가하고, `right`를 왼쪽으로 이동하면 합이 반드시 감소하기 때문이다. 정렬된 배열에서 이 단조성(monotonicity)이 보장되므로 최적의 쌍을 놓치지 않는다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

각 포인터가 최대 N번 이동하므로 시간 복잡도는 O(N)이다. 배열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **해시맵 활용**: 정렬 여부와 관계없이 해시맵에 값을 저장하며 `T - arr[i]`가 존재하는지 확인하는 방법이 있다. 시간 복잡도 O(N), 공간 복잡도 O(N)이다.
- **이진 탐색**: 각 원소에 대해 `T - arr[i]`를 이진 탐색으로 찾는 방법이 있다. 시간 복잡도 O(N log N)이다.
