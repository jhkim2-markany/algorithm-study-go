# 투 포인터와 슬라이딩 윈도우

## 개념

투 포인터(Two Pointer)는 배열이나 리스트에서 두 개의 포인터(인덱스)를 동시에 움직이며 원하는 조건을 만족하는 구간이나 쌍을 찾는 기법이다. 정렬된 배열에서 특히 효과적이며, 양쪽 끝에서 시작하여 중앙으로 좁혀가거나, 같은 방향으로 이동하며 탐색한다.

슬라이딩 윈도우(Sliding Window)는 배열이나 문자열에서 고정 또는 가변 크기의 연속 구간(윈도우)을 유지하면서, 윈도우를 한 칸씩 밀어가며 구간의 합, 최대값, 최소값 등을 효율적으로 계산하는 기법이다.

두 기법 모두 브루트포스로 O(N²)이 걸리는 문제를 O(N)으로 최적화할 수 있다는 공통점이 있다.

### 투 포인터의 두 가지 유형

1. **반대 방향 투 포인터**: 배열의 양쪽 끝에서 시작하여 중앙으로 이동한다. 정렬된 배열에서 두 수의 합, 물 담기 문제 등에 사용된다.
2. **같은 방향 투 포인터**: 두 포인터가 같은 방향으로 이동한다. 연속 부분 배열의 합, 중복 제거 등에 사용된다.

### 슬라이딩 윈도우의 두 가지 유형

1. **고정 크기 윈도우**: 윈도우 크기가 K로 고정되어 있다. 크기 K인 연속 구간의 최대 합 등에 사용된다.
2. **가변 크기 윈도우**: 조건에 따라 윈도우 크기가 변한다. 조건을 만족하는 최소/최대 길이 구간 탐색에 사용된다.

## 동작 원리

### 투 포인터 (반대 방향)

1. 왼쪽 포인터 `left`를 배열의 시작(0)에, 오른쪽 포인터 `right`를 배열의 끝(N-1)에 놓는다
2. 두 포인터가 가리키는 값을 기반으로 조건을 검사한다
3. 조건에 따라 `left`를 오른쪽으로 이동하거나 `right`를 왼쪽으로 이동한다
4. `left`와 `right`가 만날 때까지 반복한다

### 슬라이딩 윈도우 (가변 크기)

1. 왼쪽 포인터 `left`와 오른쪽 포인터 `right`를 모두 배열의 시작(0)에 놓는다
2. `right`를 오른쪽으로 확장하며 윈도우에 원소를 추가한다
3. 윈도우가 조건을 만족하면 결과를 갱신한다
4. 조건을 초과하면 `left`를 오른쪽으로 이동하여 윈도우를 축소한다
5. `right`가 배열 끝에 도달할 때까지 반복한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(N) |
| 시간 복잡도 (평균) | O(N) |
| 시간 복잡도 (최악) | O(N) |
| 공간 복잡도 | O(1) ~ O(K) |

각 포인터가 배열을 최대 한 번씩만 순회하므로 시간 복잡도는 O(N)이다. 공간 복잡도는 추가 자료구조(해시맵 등) 사용 여부에 따라 O(1)에서 O(K)까지 달라진다.

## 적합한 문제 유형

- 정렬된 배열에서 두 수의 합이 특정 값인 쌍 찾기
- 연속 부분 배열의 합이 특정 값 이상인 최소 길이 구간 찾기
- 중복 없는 가장 긴 부분 문자열 찾기
- 특정 문자를 모두 포함하는 최소 윈도우 찾기
- 고정 크기 구간의 최대/최소 합 구하기
- 물 담기(Container With Most Water) 문제
- 연속 구간에서 조건을 만족하는 최대/최소 길이 탐색


## 단계별 추적 (Trace)

### 예시 1: 투 포인터 — 정렬된 배열에서 합이 9인 쌍 찾기

입력: `arr = [1, 2, 4, 5, 7, 8, 11]`, `target = 9`

```
초기: left=0, right=6

=== 1단계 ===
arr[0]+arr[6] = 1+11 = 12 > 9 → right--
  [1, 2, 4, 5, 7, 8, 11]
   ↑L                ↑R

=== 2단계 ===
arr[0]+arr[5] = 1+8 = 9 == 9 → 찾음! (1, 8)
  [1, 2, 4, 5, 7, 8, 11]
   ↑L             ↑R

계속 탐색: left++, right--

=== 3단계 ===
arr[1]+arr[4] = 2+7 = 9 == 9 → 찾음! (2, 7)
  [1, 2, 4, 5, 7, 8, 11]
      ↑L       ↑R

=== 4단계 ===
arr[2]+arr[3] = 4+5 = 9 == 9 → 찾음! (4, 5)
  [1, 2, 4, 5, 7, 8, 11]
         ↑L ↑R

left=3, right=2 → left > right → 종료
결과: (1,8), (2,7), (4,5)
```

### 예시 2: 슬라이딩 윈도우 — 합이 7 이상인 최소 길이 부분 배열

입력: `arr = [2, 3, 1, 2, 4, 3]`, `target = 7`

```
초기: left=0, right=0, sum=0, minLen=∞

=== right 확장 ===
right=0: sum=2         < 7
right=1: sum=2+3=5     < 7
right=2: sum=5+1=6     < 7
right=3: sum=6+2=8     >= 7 → minLen=4
  [2, 3, 1, 2, 4, 3]
   ↑L         ↑R        sum=8, len=4

=== left 축소 ===
left=1: sum=8-2=6      < 7 → 축소 중단
  [2, 3, 1, 2, 4, 3]
      ↑L      ↑R        sum=6

=== right 확장 ===
right=4: sum=6+4=10    >= 7 → minLen=min(4,4)=4
  [2, 3, 1, 2, 4, 3]
      ↑L         ↑R     sum=10, len=4

=== left 축소 ===
left=2: sum=10-3=7     >= 7 → minLen=min(4,3)=3
left=3: sum=7-1=6      < 7 → 축소 중단
  [2, 3, 1, 2, 4, 3]
            ↑L   ↑R     sum=6

=== right 확장 ===
right=5: sum=6+3=9     >= 7 → minLen=min(3,3)=3
  [2, 3, 1, 2, 4, 3]
            ↑L      ↑R  sum=9, len=3

=== left 축소 ===
left=4: sum=9-2=7      >= 7 → minLen=min(3,2)=2 ★
left=5: sum=7-4=3      < 7 → 축소 중단

결과: 최소 길이 = 2 (부분 배열 [4, 3])
```

## 실전 팁

### 활용 노하우

- "연속 부분 배열"이라는 키워드가 나오면 슬라이딩 윈도우를 먼저 떠올려라
- "정렬된 배열에서 두 수의 합/차"가 나오면 반대 방향 투 포인터를 사용한다
- 슬라이딩 윈도우에서 윈도우 내 상태를 해시맵으로 관리하면 문자열 문제에 강력하다
- 투 포인터는 정렬이 전제되는 경우가 많으므로, 정렬 후 적용하는 패턴을 익혀두자

### 자주 하는 실수

- **포인터 이동 조건 혼동**: 합이 target보다 클 때 left를 이동할지 right를 이동할지 헷갈린다. 합을 줄이려면 left++, 늘리려면 right++
- **윈도우 축소 누락**: right를 확장한 후 조건을 만족하면 left를 축소하는 while 루프를 빠뜨리는 경우가 많다
- **같은 방향 투 포인터에서 left > right**: left가 right를 추월하지 않도록 조건을 확인한다
- **초기값 설정 오류**: minLen을 0으로 초기화하면 갱신이 안 된다. math.MaxInt로 초기화한다

### 엣지 케이스 주의사항

- 배열 길이가 0 또는 1인 경우
- 모든 원소가 동일한 값인 경우
- 조건을 만족하는 구간이 존재하지 않는 경우
- 전체 배열이 하나의 답인 경우 (윈도우 = 전체 배열)
- 음수가 포함된 배열에서는 투 포인터/슬라이딩 윈도우가 동작하지 않을 수 있다 (단조성 깨짐)

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 핵심 아이디어 | 적합한 상황 |
| --- | --- | --- | --- |
| 투 포인터 | O(N) | 두 포인터를 조건에 따라 이동 | 정렬된 배열의 쌍 탐색, 연속 구간 |
| 슬라이딩 윈도우 | O(N) | 고정/가변 크기 윈도우 이동 | 연속 부분 배열/문자열 최적화 |
| 브루트포스 | O(N²) | 모든 쌍/구간 탐색 | 제약 없는 소규모 데이터 |
| 누적합 | O(N) 전처리 + O(1) 쿼리 | 구간 합 전처리 | 구간 합 다수 쿼리 |
| 이진 탐색 | O(N log N) | 정렬 후 값 검색 | 특정 값 존재 여부 |
| 해시맵 | O(N) | 키-값 저장으로 빠른 조회 | 비연속 쌍 탐색 (Two Sum) |

**선택 기준:**
- 정렬된 배열에서 두 수의 합 → 반대 방향 투 포인터
- 연속 부분 배열의 조건 만족 최소/최대 길이 → 슬라이딩 윈도우
- 구간 합을 여러 번 질의 → 누적합
- 정렬되지 않은 배열에서 두 수의 합 → 해시맵
- 음수가 포함된 부분 배열 합 → 누적합 + 해시맵 (투 포인터 불가)
