# 포함 배제의 원리

## 개념

포함 배제의 원리(Inclusion-Exclusion Principle)는 여러 집합의 합집합 크기를 정확하게 계산하는 조합론적 기법이다. 각 집합의 크기를 단순히 더하면 교집합 부분이 중복 계산되므로, 교집합을 빼고 다시 더하는 과정을 반복하여 정확한 값을 구한다.

두 집합 A, B에 대해 |A ∪ B| = |A| + |B| - |A ∩ B|가 성립하며, 이를 N개 집합으로 일반화하면 홀수 개 집합의 교집합은 더하고 짝수 개 집합의 교집합은 빼는 규칙이 된다. 코딩테스트에서는 특정 조건을 만족하는 수의 개수를 세거나, 여러 조건 중 하나라도 만족하는 경우의 수를 구할 때 핵심적으로 사용된다.

## 동작 원리

### 1. 기본 공식

N개의 집합 A₁, A₂, ..., Aₙ의 합집합 크기는 다음과 같이 계산한다:

```
|A₁ ∪ A₂ ∪ ... ∪ Aₙ| =
  Σ|Aᵢ|                          (1개 집합)
- Σ|Aᵢ ∩ Aⱼ|                     (2개 집합의 교집합)
+ Σ|Aᵢ ∩ Aⱼ ∩ Aₖ|               (3개 집합의 교집합)
- ...
+ (-1)^(N+1) |A₁ ∩ A₂ ∩ ... ∩ Aₙ| (N개 집합의 교집합)
```

홀수 개 집합의 교집합은 더하고, 짝수 개 집합의 교집합은 빼는 규칙이다.

### 2. 비트마스크를 이용한 구현

N개의 집합이 있을 때, 비트마스크 0부터 2^N - 1까지 순회하며 모든 부분집합 조합을 열거한다.

1. 비트마스크 mask를 1부터 2^N - 1까지 순회한다
2. mask에서 켜진 비트에 해당하는 집합들의 교집합 크기를 계산한다
3. 켜진 비트의 수(popcount)가 홀수이면 더하고, 짝수이면 뺀다
4. 모든 mask에 대해 합산하면 합집합의 크기가 된다

예시 (3개 집합 A, B, C):
- mask=001(A): +|A|
- mask=010(B): +|B|
- mask=011(A∩B): -|A∩B|
- mask=100(C): +|C|
- mask=101(A∩C): -|A∩C|
- mask=110(B∩C): -|B∩C|
- mask=111(A∩B∩C): +|A∩B∩C|

### 3. 배수 문제에서의 활용

N 이하에서 주어진 소수들 p₁, p₂, ..., pₖ 중 하나 이상의 배수인 수의 개수를 구하는 문제:

1. Aᵢ = {pᵢ의 배수} 로 정의하면 |Aᵢ| = ⌊N/pᵢ⌋
2. Aᵢ ∩ Aⱼ = {pᵢ × pⱼ의 배수} 이므로 |Aᵢ ∩ Aⱼ| = ⌊N/(pᵢ × pⱼ)⌋
3. 비트마스크로 모든 부분집합을 열거하여 포함 배제를 적용한다
4. 서로소인 수의 개수 = N - |A₁ ∪ A₂ ∪ ... ∪ Aₖ|

### 4. 오일러 피 함수와의 관계

오일러 피 함수 φ(n)은 1부터 n까지의 수 중 n과 서로소인 수의 개수이다. 포함 배제의 원리를 적용하면:

1. n을 소인수분해한다: n = p₁^a₁ × p₂^a₂ × ... × pₖ^aₖ
2. n 이하에서 pᵢ의 배수인 수의 합집합을 포함 배제로 구한다
3. φ(n) = n × (1 - 1/p₁) × (1 - 1/p₂) × ... × (1 - 1/pₖ)

이 공식은 포함 배제의 원리를 정리한 결과이며, 오일러 피 함수를 O(√n)에 계산할 수 있게 한다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| N개 집합의 포함 배제 (비트마스크) | O(2^N) |
| 배수 문제 (K개 소인수) | O(2^K) |
| 오일러 피 함수 (단일 수) | O(√n) |
| 공간 복잡도 | O(N) 또는 O(K) |

비트마스크 기반 포함 배제는 집합의 수 N에 대해 지수적이므로, N이 20 이하일 때 실용적이다. 배수 문제에서 소인수의 개수 K는 보통 매우 작으므로(10^18 이하의 수도 소인수가 최대 15개) 충분히 빠르다.

## 적합한 문제 유형

- N 이하에서 특정 수들의 배수가 아닌 수의 개수를 세는 문제
- 여러 조건 중 하나라도 만족하는 경우의 수를 구하는 문제
- 오일러 피 함수 계산 문제
- 서로소인 수의 개수를 구하는 문제
- 정확히 K개의 조건을 만족하는 원소 수를 구하는 문제
- 뫼비우스 함수와 관련된 정수론 문제
- 교란 순열(완전 순열) 개수를 구하는 문제

## 단계별 추적 (Trace)

### 예시 1: 1~30에서 2, 3, 5의 배수가 아닌 수의 개수

```
집합 정의:
  A = {2의 배수}, B = {3의 배수}, C = {5의 배수}

[1단계] 각 집합의 크기
  |A| = ⌊30/2⌋ = 15
  |B| = ⌊30/3⌋ = 10
  |C| = ⌊30/5⌋ = 6

[2단계] 두 집합의 교집합
  |A∩B| = ⌊30/6⌋  = 5   (2×3=6의 배수)
  |A∩C| = ⌊30/10⌋ = 3   (2×5=10의 배수)
  |B∩C| = ⌊30/15⌋ = 2   (3×5=15의 배수)

[3단계] 세 집합의 교집합
  |A∩B∩C| = ⌊30/30⌋ = 1  (2×3×5=30의 배수)

[4단계] 포함 배제 적용
  |A∪B∪C| = (15+10+6) - (5+3+2) + 1
           = 31 - 10 + 1
           = 22

[5단계] 배수가 아닌 수의 개수
  30 - 22 = 8

검증: 1,7,11,13,17,19,23,29 → 8개 ✓
(이들은 30 이하에서 2,3,5와 모두 서로소인 수 = φ(30)과 일치)
```

### 예시 2: 비트마스크 구현 추적 (3개 집합)

```
소수 목록: primes = [2, 3, 5], N = 30

mask=001 (이진수, C만 선택 → 실제로는 primes[0]=2):
  popcount=1 (홀수) → +⌊30/2⌋ = +15

mask=010 (B만 선택 → primes[1]=3):
  popcount=1 (홀수) → +⌊30/3⌋ = +10

mask=011 (B,C 선택 → primes[0]*primes[1]=6):
  popcount=2 (짝수) → -⌊30/6⌋ = -5

mask=100 (A만 선택 → primes[2]=5):
  popcount=1 (홀수) → +⌊30/5⌋ = +6

mask=101 (A,C 선택 → primes[0]*primes[2]=10):
  popcount=2 (짝수) → -⌊30/10⌋ = -3

mask=110 (A,B 선택 → primes[1]*primes[2]=15):
  popcount=2 (짝수) → -⌊30/15⌋ = -2

mask=111 (A,B,C 모두 → 2*3*5=30):
  popcount=3 (홀수) → +⌊30/30⌋ = +1

합계: 15 + 10 - 5 + 6 - 3 - 2 + 1 = 22
```

## 실전 팁

### 활용 노하우

- 비트마스크로 구현하면 집합의 수 N에 관계없이 일관된 코드로 포함 배제를 적용할 수 있다
- "~가 아닌 것의 개수"를 구할 때는 전체에서 포함 배제 결과를 빼는 여사건 활용이 핵심이다
- 소인수의 개수는 매우 작다 (10^18 이하의 수도 서로 다른 소인수가 최대 15개). 따라서 2^15 = 32768번의 연산으로 충분하다
- 오일러 피 함수 공식 φ(n) = n × Π(1 - 1/pᵢ)는 포함 배제를 정리한 결과이므로, 단일 수의 서로소 개수는 이 공식이 더 빠르다

### 자주 하는 실수

- 부호 규칙을 반대로 적용하는 경우. 홀수 개 집합의 교집합은 더하고(+), 짝수 개는 빼야(-) 한다
- 비트마스크 순회 시 mask = 0을 포함하는 경우. mask는 1부터 시작해야 한다 (공집합 제외)
- 교집합 크기 계산에서 소수의 곱이 N을 초과하는 경우를 처리하지 않아 ⌊N/product⌋ = 0인데 불필요한 연산을 하는 경우
- 소인수분해를 잘못하여 중복 소인수를 포함하는 경우. 포함 배제에서는 서로 다른 소인수만 사용해야 한다
- int64 오버플로: 소수의 곱이 매우 커질 수 있으므로 곱셈 중간에 N을 초과하면 조기 종료하자

### 엣지 케이스

- 집합이 1개인 경우 — 포함 배제 없이 단순 계산
- 모든 집합의 교집합이 공집합인 경우
- N이 0인 경우 — 모든 집합의 크기가 0
- 소인수가 N보다 큰 경우 — 해당 집합의 크기가 0
- 소인수의 곱이 int64 범위를 초과하는 경우 — 곱셈 중 오버플로 체크 필요

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 포함 배제의 원리 | 합집합 크기를 교집합으로 정확히 계산, O(2^N) | 집합 수가 적고(≤20) 교집합 계산이 쉬울 때 |
| 뫼비우스 함수/반전 | 포함 배제를 약수 관계로 일반화 | 약수/배수 관계의 합산 문제, 정수론 문제 |
| 오일러 피 함수 | n과 서로소인 수의 개수를 O(√n)에 계산 | 단일 수에 대한 서로소 개수 문제 |
| 비트마스크 DP | 부분집합을 상태로 하는 DP | 포함 배제와 결합하여 복잡한 조건의 경우의 수를 셀 때 |
| 교란 순열 공식 | 포함 배제를 적용한 완전 순열 개수 공식 | 모든 원소가 원래 위치에 있지 않은 순열의 수를 구할 때 |
