## 오전: 핵심 패턴 속성 복습 (1시간)

각 패턴을 **빈 파일에 직접 타이핑**해서 손에 익히기. 보지 말고 쳐라.

### 반드시 칠 수 있어야 하는 10개 패턴

| # | 패턴 | 핵심 |
|---|---|---|
| 1 | 격자 BFS | dx/dy + queue + visited |
| 2 | 재귀 DFS | visited + 인접 리스트 순회 |
| 3 | 해시맵 빈도수 | `map[T]int` + `freq[v]++` |

| 5 | 이진 탐색 | `sort.Search` 또는 lo/hi/mid |
| 6 | 커스텀 정렬 | `sort.Slice` + 다중 조건 |
| 7 | 투 포인터 | 양끝 수렴 / 같은 방향 |
| 8 | 1차원 DP | 점화식 → 테이블 → 초기값 |
| 9 | 백트래킹 조합 | start 인덱스 + copy + append |
| 10 | 누적합 | `prefix[r+1] - prefix[l]` |

### Go 언어가 없을 경우 대비
HackerRank에서 Go를 지원하지 않는 경우가 있다. Python으로 전환할 준비:

```python
# Python 해시맵
freq = {}
for v in arr:
    freq[v] = freq.get(v, 0) + 1

# Python 정렬
arr.sort(key=lambda x: (-x.score, x.name))

# Python BFS
from collections import deque
queue = deque([(sr, sc, 0)])
```

## 시험 당일 전략 (최종 정리)

### 시간 배분
| 구간 | 시간 | 할 일 |
|---|---|---|
| 0~5분 | 5분 | 전체 문제 훑기, 난이도 파악, 풀이 순서 결정 |
| 5~25분 | 20분 | Easy 확실하게 풀기 |
| 25~55분 | 30분 | Medium 2개 도전 |
| 55~70분 | 15분 | Hard 시도 또는 Medium 마무리 |
| 70~75분 | 5분 | 엣지 케이스 점검, 제출 확인 |

### 제약 조건 → 풀이법 추론
| N 범위 | 복잡도 | 방향 |
|---|---|---|
| N ≤ 10 | O(N!) | 순열 전탐색 |
| N ≤ 20 | O(2^N) | 비트마스크, 브루트포스 |
| N ≤ 100 | O(N³) | 3중 루프, 플로이드 |
| N ≤ 10,000 | O(N²) | 2중 루프, DP |
| N ≤ 1,000,000 | O(N log N) | 정렬, 이분탐색, 힙 |
| N ≤ 10,000,000 | O(N) | 투 포인터, 해시, 누적합 |

### 절대 원칙
1. **Easy를 절대 놓치지 마라**
2. **문제를 끝까지 읽어라** — 제약 조건에 힌트가 있다
3. **막히면 넘겨라** — 20분 이상 한 문제에 쓰지 마라
4. **부분 점수를 노려라** — 최적해 못 찾으면 브루트포스라도 제출
5. **엣지 케이스 체크** — 빈 배열, 원소 1개, 모두 같은 값, 음수

### 자주 하는 실수
- `sort.Search`의 반환값이 배열 길이일 수 있음 (not found)
- slice append 후 원본이 변경될 수 있음 → `copy` 사용
- map 순회 순서는 보장되지 않음
- 재귀 DFS에서 visited 체크 위치 (호출 전 vs 함수 시작)
- 2차원 배열 초기화 시 행마다 `make` 필요
