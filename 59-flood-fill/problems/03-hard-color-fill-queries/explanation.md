# 색칠 쿼리 처리 - 해설

## 접근 방식

1. 유니온 파인드로 같은 색상의 인접 칸들을 하나의 집합으로 관리한다
2. 초기 상태에서 인접한 같은 색상 칸들을 모두 합친다
3. 각 쿼리마다:
   - 대상 칸의 대표 노드를 찾고 색상을 변경한다
   - 영역에 속한 모든 칸의 경계에서 인접한 같은 색상 영역과 합친다
4. 모든 쿼리 처리 후 각 칸의 대표 노드 색상을 출력한다

## 핵심 아이디어

단순히 매 쿼리마다 플러드 필을 수행하면 최악의 경우 O(Q × N × M)으로 시간 초과가 발생할 수 있다. 유니온 파인드를 사용하면 영역 병합을 효율적으로 처리할 수 있다.

핵심 포인트:
- 각 집합의 대표 노드에 색상 정보를 저장한다
- 색상 변경 시 대표 노드의 색상만 바꾸면 전체 영역이 변경된다
- 색상 변경 후 경계에서 같은 색상의 인접 영역과 합쳐야 한다

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q × N × M × α(NM)) |
| 공간 복잡도 | O(N × M) |

각 쿼리에서 영역의 멤버를 찾기 위해 격자를 스캔하므로 O(N×M)이 소요된다. 유니온 파인드 연산은 α(NM) ≈ O(1)이다. 제약 조건(N,M ≤ 500, Q ≤ 10,000) 내에서 충분히 동작한다.

## 대안적 접근

- 매 쿼리마다 BFS 플러드 필: 구현이 간단하지만 O(Q × N × M)으로 느릴 수 있다. 제약 조건이 작으면 충분하다
- 영역 인접 리스트 관리: 각 영역이 인접한 영역 목록을 관리하면 멤버 스캔 없이 경계 처리가 가능하지만 구현이 복잡하다
