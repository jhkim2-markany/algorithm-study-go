# LCA (최소 공통 조상)

## 개념

LCA(Lowest Common Ancestor, 최소 공통 조상)는 루트가 있는 트리에서 두 노드 u, v의 공통 조상 중 가장 깊은(루트에서 가장 먼) 노드를 찾는 알고리즘이다. 공통 조상이란 루트에서 u까지의 경로와 루트에서 v까지의 경로에 모두 포함되는 노드를 말한다.

LCA는 트리 위의 경로 문제를 해결하는 핵심 도구이다. 두 노드 사이의 거리, 경로 위의 가중치 합, 경로 위의 최댓값/최솟값 등 다양한 트리 쿼리 문제에서 LCA를 기반으로 풀이를 구성한다. 코딩테스트에서는 주로 희소 배열(Binary Lifting) 기법을 사용하여 O(log N) 시간에 LCA를 구한다.

## 선수 학습

LCA를 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [13-tree](../13-tree/) - 트리의 기본 개념, 루트, 부모, 깊이 등 트리 용어와 순회 방법
- [15-graph-dfs](../15-graph-dfs/) - DFS를 이용한 트리 탐색, 깊이 계산, 부모 배열 구성

## 동작 원리

### 1. 나이브 방법 (Naive)

가장 단순한 방법으로, 두 노드의 깊이를 맞춘 뒤 한 칸씩 올라가며 만나는 지점을 찾는다.

1. DFS로 각 노드의 깊이(depth)와 부모(parent)를 구한다
2. 두 노드 u, v 중 더 깊은 노드를 얕은 노드의 깊이까지 올린다
3. 두 노드가 같으면 그 노드가 LCA이다
4. 같지 않으면 두 노드를 동시에 한 칸씩 올리며 같아질 때까지 반복한다

이 방법은 구현이 간단하지만, 최악의 경우 O(N) 시간이 걸린다.

### 2. 희소 배열 (Binary Lifting)

부모를 1칸이 아닌 2^k칸씩 점프하여 LCA를 O(log N)에 구하는 기법이다.

1. 전처리: DFS로 각 노드의 깊이와 부모를 구한다
2. 희소 배열 구성: up[v][k] = v에서 2^k번째 조상을 저장한다
   - up[v][0] = parent[v] (직접 부모)
   - up[v][k] = up[up[v][k-1]][k-1] (2^k번째 조상 = 2^(k-1)번째 조상의 2^(k-1)번째 조상)
3. LCA 쿼리:
   - 깊이가 다르면 깊은 노드를 2의 거듭제곱 단위로 올려 깊이를 맞춘다
   - 두 노드가 같으면 LCA이다
   - 같지 않으면 k를 큰 값부터 줄여가며, up[u][k] ≠ up[v][k]인 경우에만 두 노드를 올린다
   - 최종적으로 up[u][0]이 LCA이다

### 3. 오일러 투어 + RMQ (Range Minimum Query)

트리를 오일러 투어로 선형화한 뒤, 구간 최솟값 쿼리로 LCA를 구하는 방법이다.

1. 오일러 투어: DFS 순서대로 방문하는 노드를 기록한다 (서브트리 진입/이탈 시 모두 기록)
2. 각 노드가 오일러 투어에서 처음 등장하는 위치(first)를 기록한다
3. LCA(u, v) = 오일러 투어 배열에서 first[u]와 first[v] 사이 구간에서 깊이가 최소인 노드
4. 구간 최솟값은 희소 테이블(Sparse Table)로 O(1)에 응답할 수 있다

전처리 O(N log N), 쿼리 O(1)로 쿼리가 매우 많을 때 유리하다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 나이브 방법 - 전처리 | O(N) |
| 나이브 방법 - 쿼리 | O(N) |
| Binary Lifting - 전처리 | O(N log N) |
| Binary Lifting - 쿼리 | O(log N) |
| 오일러 투어 + RMQ - 전처리 | O(N log N) |
| 오일러 투어 + RMQ - 쿼리 | O(1) |
| 공간 복잡도 (Binary Lifting) | O(N log N) |
| 공간 복잡도 (오일러 투어 + RMQ) | O(N log N) |

코딩테스트에서는 Binary Lifting이 구현 난이도와 성능의 균형이 좋아 가장 많이 사용된다. 쿼리 수가 매우 많은 경우 오일러 투어 + RMQ가 유리하다.

## 적합한 문제 유형

- 트리에서 두 노드의 최소 공통 조상을 구하는 문제
- 트리에서 두 노드 사이의 거리를 구하는 문제
- 트리 경로 위의 가중치 합/최댓값/최솟값을 구하는 문제
- 트리에서 여러 노드의 공통 조상을 구하는 문제
- 트리에서 K번째 조상을 구하는 문제
- 트리의 서브트리 관련 쿼리 문제
- 가중치 트리에서 경로 관련 쿼리 문제

## 단계별 추적 (Trace)

### 예시: Binary Lifting으로 LCA 구하기

트리 구조 (루트 = 1):

```
         1
        / \
       2   3
      / \   \
     4   5   6
    /
   7
```

#### 전처리: 깊이와 부모 배열 (DFS)

```
노드:   1  2  3  4  5  6  7
깊이:   0  1  1  2  2  2  3
부모:   -  1  1  2  2  3  4
```

#### 희소 배열 구성 (up[v][k] = v의 2^k번째 조상)

```
up[v][0] = parent[v]  (직접 부모)
  up[1][0]=-  up[2][0]=1  up[3][0]=1  up[4][0]=2
  up[5][0]=2  up[6][0]=3  up[7][0]=4

up[v][1] = up[up[v][0]][0]  (2번째 조상 = 부모의 부모)
  up[1][1]=-  up[2][1]=-  up[3][1]=-  up[4][1]=1
  up[5][1]=1  up[6][1]=1  up[7][1]=2

up[v][2] = up[up[v][1]][1]  (4번째 조상)
  up[7][2]=- (2의 2번째 조상 = 루트 위 → 없음)
```

#### LCA(7, 5) 쿼리 실행

```
[1단계] 깊이 맞추기
  depth[7]=3, depth[5]=2 → 7이 더 깊음
  차이 = 3 - 2 = 1 = 2^0
  7을 2^0칸 올림: up[7][0] = 4
  → u=4, v=5 (깊이 모두 2)

[2단계] 같은지 확인
  4 ≠ 5 → 같지 않으므로 계속 진행

[3단계] 함께 올리기 (k를 큰 값부터)
  k=1: up[4][1]=1, up[5][1]=1 → 같음! → 올리지 않음
  k=0: up[4][0]=2, up[5][0]=2 → 같음! → 올리지 않음

  두 노드가 이미 같은 부모를 가리키므로:
  LCA = up[4][0] = 2

결과: LCA(7, 5) = 2
```

#### LCA(7, 6) 쿼리 실행

```
[1단계] 깊이 맞추기
  depth[7]=3, depth[6]=2 → 차이 = 1
  7을 1칸 올림: up[7][0] = 4
  → u=4, v=6 (깊이 모두 2)

[2단계] 같은지 확인
  4 ≠ 6 → 계속 진행

[3단계] 함께 올리기
  k=0: up[4][0]=2, up[6][0]=3 → 다름! → 올림
  → u=2, v=3

  더 이상 올릴 수 없음 (k < 0)
  LCA = up[2][0] = 1

결과: LCA(7, 6) = 1
```

## 실전 팁

### 활용 노하우

- Binary Lifting이 구현 난이도와 성능의 균형이 가장 좋다. 코딩테스트에서는 이 방법을 추천
- 트리에서 두 노드 u, v 사이의 거리 = depth[u] + depth[v] - 2 × depth[LCA(u,v)]. 이 공식을 외워두자
- 경로 위의 가중치 합도 같은 원리로 구할 수 있다: dist[u] + dist[v] - 2 × dist[LCA(u,v)] (dist는 루트에서의 거리)
- K번째 조상 문제도 Binary Lifting으로 O(log N)에 풀 수 있다. K를 이진수로 분해하여 해당 비트가 켜진 만큼 점프
- LOG 값은 `log2(N) + 1` 정도로 설정하면 충분하다. N ≤ 10^5이면 LOG = 17, N ≤ 10^6이면 LOG = 20

### 자주 하는 실수

- 희소 배열의 LOG 크기를 너무 작게 잡아 배열 범위를 초과하는 경우. N의 log2 값보다 1~2 크게 잡자
- 깊이를 맞출 때 올리는 방향을 잘못 설정하는 경우. 항상 더 깊은 노드를 올려야 한다
- up 배열에서 루트의 조상을 0이나 -1로 설정하는데, 이 값이 유효한 노드 번호와 겹치는 경우
- DFS로 깊이/부모를 구할 때 루트 노드의 깊이를 0으로 초기화하지 않는 경우
- 함께 올리기 단계에서 `up[u][k] == up[v][k]`일 때 올리면 안 되는데 올려버리는 경우. 같으면 건너뛰어야 한다

### 엣지 케이스

- 두 노드가 같은 경우 — LCA는 자기 자신
- 한 노드가 다른 노드의 조상인 경우 — 깊이 맞추기만으로 LCA를 찾을 수 있음
- 루트 노드가 LCA인 경우
- 트리가 편향된 경우 (일직선 트리) — 깊이가 N까지 갈 수 있으므로 LOG 크기 확인
- 노드 번호가 0부터 시작하는지 1부터 시작하는지에 따른 인덱스 처리

## 관련 알고리즘 비교

| 알고리즘 | 전처리 | 쿼리 | 특징 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 나이브 LCA | O(N) | O(N) | 한 칸씩 올라가며 비교, 구현 매우 간단 | 쿼리 수가 적고 트리가 작을 때 |
| Binary Lifting | O(N log N) | O(log N) | 희소 배열로 2^k 점프, 구현과 성능의 균형 | 코딩테스트에서 가장 추천 |
| 오일러 투어 + Sparse Table | O(N log N) | O(1) | 트리를 선형화 후 RMQ로 LCA 응답 | 쿼리가 매우 많을 때 (10^6 이상) |
| Tarjan's Offline LCA | O(N + Q) | 오프라인 | Union-Find 기반, 모든 쿼리를 한 번에 처리 | 쿼리를 미리 알고 있고 오프라인 처리가 가능할 때 |
| HLD (Heavy-Light Decomposition) | O(N) | O(log N) | 트리를 체인으로 분할, 경로 쿼리와 결합 | 경로 위의 구간 쿼리(합, 최대 등)가 함께 필요할 때 |
