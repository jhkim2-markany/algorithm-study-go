# 트리 경로 위의 최댓값 쿼리 - 해설

## 접근 방식

1. 가중치 트리를 인접 리스트로 구성한다
2. DFS로 각 노드의 깊이, Binary Lifting 조상 배열, 경로 최대 가중치 배열을 구성한다
3. 각 쿼리에서 두 노드를 LCA까지 올리면서 경로 위 간선 가중치의 최댓값을 추적한다

## 핵심 아이디어

Binary Lifting에서 조상 정보뿐 아니라 경로 위의 추가 정보(최댓값, 최솟값, 합 등)도 함께 저장할 수 있다. maxW[v][k]는 노드 v에서 2^k번째 조상까지의 경로에 있는 간선 가중치의 최댓값을 저장한다.

희소 배열 구성 시 다음 점화식을 사용한다:
- maxW[v][k] = max(maxW[v][k-1], maxW[anc[v][k-1]][k-1])

이는 v에서 2^k번째 조상까지의 경로를 두 구간으로 나누어 각 구간의 최댓값 중 큰 값을 취하는 것이다. LCA를 구하는 과정에서 노드를 올릴 때마다 해당 구간의 최대 가중치를 결과에 반영한다.

이 기법은 LCA + Binary Lifting의 확장으로, 경로 위의 다양한 집계 연산(합, 최솟값, XOR 등)에도 동일하게 적용할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N + M log N) |
| 공간 복잡도 | O(N log N) |

전처리에 O(N log N), 각 쿼리에 O(log N)이 소요된다. 조상 배열과 최대 가중치 배열 모두 O(N log N) 공간을 사용한다.

## 대안적 접근

- Heavy-Light Decomposition (HLD): 트리를 체인으로 분해하고 세그먼트 트리를 올려 경로 쿼리를 처리한다. 간선 가중치 갱신이 필요한 경우에 유리하다. 전처리 O(N log N), 쿼리 O(log² N)이다.
- 오일러 투어 + 세그먼트 트리: 노드 가중치 문제에서 사용 가능하지만, 간선 가중치 문제에서는 변환이 필요하다.
