# 두 노드의 최소 공통 조상 - 해설

## 접근 방식

1. 트리를 인접 리스트로 구성한다
2. 루트(1번 노드)에서 DFS를 수행하여 각 노드의 깊이와 Binary Lifting 희소 배열을 구성한다
3. 각 쿼리에 대해 Binary Lifting으로 LCA를 O(log N)에 구한다

## 핵심 아이디어

Binary Lifting의 핵심은 up[v][k] = v에서 2^k번째 조상을 미리 계산해두는 것이다. 이를 통해 한 칸씩 올라가는 대신 2의 거듭제곱 단위로 점프할 수 있다.

LCA를 구하는 과정은 두 단계로 나뉜다:
1. 깊이 맞추기: 더 깊은 노드를 2의 거듭제곱 단위로 올려 같은 깊이로 맞춘다
2. 동시 올리기: 큰 점프부터 시도하며, 두 노드의 조상이 다른 경우에만 올린다. 최종적으로 한 칸 위가 LCA이다

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N + M log N) |
| 공간 복잡도 | O(N log N) |

전처리에 O(N log N), 각 쿼리에 O(log N)이 소요된다.

## 대안적 접근

- 나이브 방법: 깊이를 맞추고 한 칸씩 올라가는 방법으로 O(N) per query이다. N, M이 작을 때 사용 가능하다.
- 오일러 투어 + RMQ: 전처리 O(N log N), 쿼리 O(1)로 쿼리가 매우 많을 때 유리하다.
