# 공유기 설치 - 해설

## 접근 방식

1. 집의 좌표를 오름차순으로 정렬한다
2. 가장 인접한 두 공유기 사이의 최소 거리를 이진 탐색의 대상으로 설정한다
3. 탐색 범위는 1부터 (가장 먼 집 - 가장 가까운 집)까지이다
4. 결정 함수에서 그리디하게 공유기를 설치하여 C개 이상 설치 가능한지 판별한다
5. 조건을 만족하면 더 큰 거리를 시도하고, 불만족하면 작은 거리를 시도한다

## 핵심 아이디어

이 문제는 "가장 인접한 두 공유기 사이의 거리의 최댓값"을 구하는 문제이다. 이를 "최소 거리를 D 이상으로 유지하며 C개의 공유기를 설치할 수 있는가?"라는 결정 문제로 변환한다.

결정 함수는 그리디 알고리즘을 사용한다. 정렬된 집 목록에서 첫 번째 집에 공유기를 설치하고, 이후 마지막 설치 위치로부터 거리가 D 이상인 첫 번째 집에 다음 공유기를 설치한다. 이 방식이 최적인 이유는, 가능한 한 앞쪽에 설치해야 뒤쪽에 더 많은 공유기를 설치할 여지가 생기기 때문이다.

최소 거리 D가 커질수록 설치 가능한 공유기 수가 줄어드므로 단조성이 성립한다.

이 문제가 어려운 이유는 파라메트릭 서치와 그리디 알고리즘을 결합해야 하며, 결정 함수의 정당성을 증명해야 하기 때문이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N + N log D) |
| 공간 복잡도 | O(N) |

정렬에 O(N log N), 이진 탐색에 O(log D)번 반복하며 매 반복마다 O(N)으로 결정 함수를 수행하므로 O(N log D)이다. D는 좌표 범위(최대 10억)이므로 log D ≈ 30이다. 전체 시간 복잡도는 O(N log N + N log D)이다.

## 대안적 접근

- 좌표 압축 후 이진 탐색을 수행할 수도 있지만, 이 문제에서는 좌표 값 자체가 중요하므로 압축이 적합하지 않다.
- 결정 함수에서 그리디 대신 DP를 사용할 수도 있지만, 그리디가 더 효율적이고 구현이 간단하다.
