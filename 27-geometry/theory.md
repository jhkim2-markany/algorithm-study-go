# 기하학

## 개념

기하학(Computational Geometry)은 점, 선분, 다각형 등 기하학적 객체의 관계와 성질을 수학적으로 계산하는 알고리즘 분야이다. 코딩테스트에서는 주로 2차원 평면 위의 좌표를 다루며, 외적(Cross Product)을 활용한 방향 판별(CCW), 선분 교차 판정, 다각형 넓이 계산 등이 핵심 기법이다.

기하 알고리즘의 기본 도구는 벡터의 외적이다. 두 벡터의 외적 부호를 통해 세 점의 방향 관계(시계/반시계/일직선)를 판별할 수 있으며, 이를 기반으로 선분 교차, 볼록 껍질, 다각형 내부 판정 등 다양한 문제를 풀 수 있다.

## 동작 원리

기하 알고리즘의 핵심은 외적(Cross Product)을 활용한 방향 판별이다.

### 1. 외적 (Cross Product)

두 벡터 A = (x₁, y₁), B = (x₂, y₂)의 외적은 다음과 같이 정의한다:

```
A × B = x₁ * y₂ - x₂ * y₁
```

외적의 부호로 두 벡터의 방향 관계를 판별한다:
- 양수: B가 A의 반시계 방향에 위치
- 음수: B가 A의 시계 방향에 위치
- 0: A와 B가 평행 (일직선)

### 2. CCW (Counter-Clockwise) 판별

세 점 P₁, P₂, P₃가 주어졌을 때, P₁→P₂ 벡터와 P₁→P₃ 벡터의 외적으로 방향을 판별한다:

1. 벡터 A = P₂ - P₁ = (x₂ - x₁, y₂ - y₁) 을 구한다
2. 벡터 B = P₃ - P₁ = (x₃ - x₁, y₃ - y₁) 을 구한다
3. 외적 값 = A × B = (x₂ - x₁)(y₃ - y₁) - (x₃ - x₁)(y₂ - y₁) 을 계산한다
4. 결과 해석:
   - 양수 → 반시계 방향 (CCW)
   - 음수 → 시계 방향 (CW)
   - 0 → 일직선 (Collinear)

### 3. 선분 교차 판정

두 선분 AB와 CD의 교차 여부를 판정하는 과정:

1. CCW(A, B, C)와 CCW(A, B, D)의 부호가 다른지 확인한다
2. CCW(C, D, A)와 CCW(C, D, B)의 부호가 다른지 확인한다
3. 두 조건이 모두 만족하면 두 선분은 교차한다
4. 외적 값이 0인 경우(일직선)는 좌표 범위 겹침을 별도로 확인한다

### 4. 다각형 넓이 계산 (신발끈 공식)

꼭짓점이 순서대로 주어진 다각형의 넓이를 구하는 공식:

1. 각 꼭짓점 (xᵢ, yᵢ)에 대해 xᵢ * yᵢ₊₁ - xᵢ₊₁ * yᵢ 를 계산한다
2. 모든 값을 합산한다
3. 합의 절댓값을 2로 나누면 넓이가 된다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| CCW 판별 | O(1) |
| 선분 교차 판정 | O(1) |
| 다각형 넓이 (신발끈 공식) | O(N) |
| 볼록 껍질 (Graham Scan) | O(N log N) |
| 공간 복잡도 | O(N) |

기본 연산인 CCW와 선분 교차는 상수 시간에 수행된다. 다각형 넓이는 꼭짓점 수에 비례하며, 볼록 껍질은 정렬이 필요하므로 O(N log N)이다.

## 적합한 문제 유형

- 세 점의 방향 관계(CCW) 판별 문제
- 두 선분의 교차 여부 판정 문제
- 다각형의 넓이 계산 문제
- 점이 다각형 내부에 있는지 판정하는 문제
- 볼록 껍질(Convex Hull) 구하기 문제
- 가장 가까운 두 점 찾기 (Closest Pair) 문제
- 직선과 점 사이의 거리 계산 문제

## 단계별 추적 (Trace)

### 예시: CCW 판별과 선분 교차 판정

세 점 P₁(0, 0), P₂(4, 0), P₃(2, 3)의 CCW 판별과, 선분 AB와 CD의 교차 판정을 추적한다.

#### CCW 판별 과정

```
입력: P₁(0, 0), P₂(4, 0), P₃(2, 3)

1단계: 벡터 계산
  A = P₂ - P₁ = (4-0, 0-0) = (4, 0)
  B = P₃ - P₁ = (2-0, 3-0) = (2, 3)

2단계: 외적 계산
  A × B = 4 × 3 - 2 × 0 = 12

3단계: 부호 판별
  12 > 0 → 반시계 방향 (CCW)

시각화:
        P₃(2,3)
       /  ↑
      /   | 반시계
     /    |
  P₁(0,0)────→ P₂(4,0)
```

#### 선분 교차 판정 과정

```
입력: 선분 AB = (0,0)→(4,4), 선분 CD = (0,4)→(4,0)

1단계: CCW(A, B, C) 계산
  벡터 AB = (4, 4), 벡터 AC = (0, 4)
  외적 = 4×4 - 0×4 = 16 > 0 → CCW (반시계)

2단계: CCW(A, B, D) 계산
  벡터 AB = (4, 4), 벡터 AD = (4, 0)
  외적 = 4×0 - 4×4 = -16 < 0 → CW (시계)

3단계: CCW(C, D, A) 계산
  벡터 CD = (4, -4), 벡터 CA = (0, -4)
  외적 = 4×(-4) - 0×(-4) = -16 < 0 → CW

4단계: CCW(C, D, B) 계산
  벡터 CD = (4, -4), 벡터 CB = (4, 0)
  외적 = 4×0 - 4×(-4) = 16 > 0 → CCW

5단계: 교차 판정
  CCW(A,B,C) × CCW(A,B,D) = (+)(−) < 0 ✓
  CCW(C,D,A) × CCW(C,D,B) = (−)(+) < 0 ✓
  → 두 선분은 교차한다!

시각화:
  C(0,4) ╲     
          ╲    
           ╳ ← 교차점 (2,2)
          ╱    
  A(0,0) ╱     D(4,0)
         → B(4,4)
```

## 실전 팁

### 활용 노하우

- 기하 문제는 외적(CCW)을 기본 도구로 삼아라. 대부분의 기하 판별 문제는 CCW로 해결 가능하다
- 정수 좌표가 주어지면 실수 연산을 피하고 정수 외적만으로 판별하라. 부동소수점 오차를 원천 차단할 수 있다
- 다각형 넓이를 구할 때 신발끈 공식의 결과에 절댓값을 취하는 것을 잊지 마라
- 볼록 껍질 문제에서는 기준점 선택(가장 아래, 가장 왼쪽)과 각도 정렬이 핵심이다

### 자주 하는 실수

- 외적 계산 시 좌표 순서를 잘못 넣어 부호가 반대로 나오는 실수
- 선분 교차 판정에서 일직선(외적 = 0) 케이스를 처리하지 않는 실수
- `int` 범위 초과: 좌표가 10⁶ 수준이면 외적 결과가 10¹² 이상이 되므로 `int64`를 사용해야 한다
- 다각형 넓이 계산 시 마지막 꼭짓점과 첫 꼭짓점을 연결하는 항을 빠뜨리는 실수

### 엣지 케이스

- 세 점이 일직선 위에 있는 경우 (외적 = 0)
- 선분의 끝점이 다른 선분 위에 있는 경우 (접하는 교차)
- 좌표가 모두 같은 점 (퇴화 케이스)
- 다각형의 꼭짓점이 시계/반시계 순서가 아닌 경우

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| CCW (외적) | O(1), 세 점의 방향 판별 | 방향 판별, 선분 교차의 기본 도구 |
| 신발끈 공식 | O(N), 다각형 넓이 계산 | 꼭짓점 좌표가 순서대로 주어진 경우 |
| Graham Scan | O(N log N), 볼록 껍질 | 점 집합에서 볼록 껍질을 구할 때 |
| Andrew's Monotone Chain | O(N log N), 볼록 껍질 | Graham Scan 대안, 구현이 더 간단 |
| 분할 정복 Closest Pair | O(N log N), 최근접 점 쌍 | 가장 가까운 두 점을 찾을 때 |
| Ray Casting | O(N), 점의 다각형 내부 판정 | 볼록/오목 다각형 모두 적용 가능 |
