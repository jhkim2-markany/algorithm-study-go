# 삼분 탐색 (Ternary Search)

## 선수 학습

- [08-binary-search](../08-binary-search/) - 이진 탐색 (구간 축소 탐색의 기본 원리)

## 개념

삼분 탐색(Ternary Search)은 **유니모달(unimodal) 함수**에서 극값(최솟값 또는 최댓값)을 효율적으로 찾는 알고리즘이다.

### 유니모달 함수란?

구간 [lo, hi]에서 정의된 함수 f(x)가 다음 조건을 만족하면 유니모달이라 한다:

- **볼록 함수(아래로 볼록)**: 어떤 점 x*에서 최솟값을 가지며, x < x*이면 f는 단조 감소, x > x*이면 f는 단조 증가한다
- **오목 함수(위로 볼록)**: 어떤 점 x*에서 최댓값을 가지며, x < x*이면 f는 단조 증가, x > x*이면 f는 단조 감소한다

이진 탐색이 **단조 함수**에서 특정 값을 찾는 데 사용된다면, 삼분 탐색은 **유니모달 함수**에서 극값을 찾는 데 사용된다.

### 핵심 아이디어

구간 [lo, hi]를 3등분하는 두 점 m1, m2를 잡고:

```
m1 = lo + (hi - lo) / 3
m2 = hi - (hi - lo) / 3
```

- f(m1) > f(m2)이면 최솟값은 [m1, hi] 구간에 존재 → lo = m1
- f(m1) < f(m2)이면 최솟값은 [lo, m2] 구간에 존재 → hi = m2
- f(m1) == f(m2)이면 양쪽 모두 축소 가능 → lo = m1, hi = m2

매 반복마다 구간이 2/3로 줄어들므로, k번 반복 후 구간 크기는 (2/3)^k × (초기 구간)이 된다.

## 동작 원리

### 최솟값을 찾는 경우 (볼록 함수)

```
f(x)
 │
 │  ╲                    ╱
 │    ╲                ╱
 │      ╲            ╱
 │        ╲        ╱
 │          ╲    ╱
 │            ╲╱  ← 최솟값 (x*)
 │
 └──────────────────────── x
    lo    m1   x*  m2    hi
```

1. 구간 [lo, hi]에서 시작한다
2. 두 내부 점 m1 = lo + (hi-lo)/3, m2 = hi - (hi-lo)/3을 계산한다
3. f(m1)과 f(m2)를 비교한다:
   - f(m1) > f(m2): 최솟값은 m1 왼쪽에 없다 → lo = m1
   - f(m1) < f(m2): 최솟값은 m2 오른쪽에 없다 → hi = m2
   - f(m1) == f(m2): 양쪽 모두 축소 → lo = m1, hi = m2
4. 구간이 충분히 작아질 때까지 반복한다
5. lo (또는 hi)가 극값의 위치이다

### 최댓값을 찾는 경우 (오목 함수)

비교 방향만 반대로 하면 된다:
- f(m1) < f(m2): lo = m1
- f(m1) > f(m2): hi = m2

### 정수 도메인 vs 실수 도메인

- **실수 도메인**: `hi - lo > eps` (eps ≈ 1e-9) 조건으로 반복하거나, 고정 횟수(100~200회) 반복
- **정수 도메인**: `lo < hi` 조건으로 반복하며, 구간이 2~3 이하가 되면 남은 후보를 직접 비교

## 단계별 추적 (Trace)

### 예시 입력

볼록 함수 f(x) = (x - 3)² + 1 에서 구간 [0, 10]의 최솟값을 찾는다.

- 실제 최솟값: x = 3일 때 f(3) = 1

### 실행 과정

**1단계: 초기 상태**

```
lo = 0.000, hi = 10.000
구간 길이 = 10.000
```

**2단계: 1번째 반복**

```
m1 = 0 + (10-0)/3 = 3.333
m2 = 10 - (10-0)/3 = 6.667

f(m1) = (3.333-3)² + 1 = 1.111
f(m2) = (6.667-3)² + 1 = 14.449

f(m1) < f(m2) → hi = m2 = 6.667

구간: [0.000, 6.667], 길이 = 6.667
```

**3단계: 2번째 반복**

```
m1 = 0 + 6.667/3 = 2.222
m2 = 6.667 - 6.667/3 = 4.444

f(m1) = (2.222-3)² + 1 = 1.605
f(m2) = (4.444-3)² + 1 = 3.086

f(m1) < f(m2) → hi = m2 = 4.444

구간: [0.000, 4.444], 길이 = 4.444
```

**4단계: 3번째 반복**

```
m1 = 0 + 4.444/3 = 1.481
m2 = 4.444 - 4.444/3 = 2.963

f(m1) = (1.481-3)² + 1 = 3.304
f(m2) = (2.963-3)² + 1 = 1.001

f(m1) > f(m2) → lo = m1 = 1.481

구간: [1.481, 4.444], 길이 = 2.963
```

**5단계: 4번째 반복**

```
m1 = 1.481 + 2.963/3 = 2.469
m2 = 4.444 - 2.963/3 = 3.457

f(m1) = (2.469-3)² + 1 = 1.282
f(m2) = (3.457-3)² + 1 = 1.209

f(m1) > f(m2) → lo = m1 = 2.469

구간: [2.469, 4.444], 길이 = 1.975
```

**수렴 과정 요약**

```
반복  lo       hi       구간 길이   m1      m2      f(m1)    f(m2)
 1    0.000    10.000   10.000     3.333   6.667   1.111    14.449
 2    0.000    6.667    6.667      2.222   4.444   1.605    3.086
 3    0.000    4.444    4.444      1.481   2.963   3.304    1.001
 4    1.481    4.444    2.963      2.469   3.457   1.282    1.209
 5    2.469    4.444    1.975      3.128   3.786   1.016    1.617
 ...
 → x ≈ 3.000, f(x) ≈ 1.000 으로 수렴
```

매 반복마다 구간이 약 2/3로 줄어들어, 약 60~70회 반복이면 10^-9 정밀도에 도달한다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (실수) | O(log((hi-lo)/ε)) |
| 시간 복잡도 (정수) | O(log N) |
| 공간 복잡도 | O(1) |

- ε은 허용 오차 (보통 10⁻⁹)
- N은 탐색 구간의 크기
- 매 반복마다 구간이 2/3로 줄어들므로, log₃/₂(N) ≈ 1.71 × log₂(N) 회 반복

## 적합한 문제 유형

- **볼록/오목 함수의 극값 탐색**: f(x) = ax² + bx + c 같은 이차 함수의 최솟값/최댓값
- **거리 최적화**: 여러 점까지의 거리 합/최대 거리를 최소화하는 위치 찾기
- **넓이/부피 최적화**: 주어진 조건에서 넓이나 부피를 최대화하는 매개변수 찾기
- **비용 함수 최적화**: 볼록 비용 함수의 최솟값을 구하는 문제
- **기하 문제**: 두 도형 사이의 최소 거리, 최적 배치 등
- **실수 파라미터 최적화**: 파라메트릭 서치와 결합하여 실수 해를 구하는 문제

## 실전 팁

### 활용 노하우

- 문제에서 "최솟값", "최댓값"을 구하라고 하고, 함수가 볼록/오목임이 보장되면 삼분 탐색을 의심한다
- 이진 탐색으로 풀 수 없는 최적화 문제(단조 함수가 아닌 경우)에서 삼분 탐색을 시도한다
- 실수 도메인에서는 반복 횟수를 100~200회로 고정하는 것이 eps 비교보다 안전하다
- 정수 도메인에서는 구간이 3 이하가 되면 남은 후보를 직접 비교하는 것이 안전하다
- 함수가 유니모달인지 반드시 확인해야 한다. 유니모달이 아니면 삼분 탐색은 틀린 답을 줄 수 있다

### 자주 하는 실수

- **유니모달 조건 미확인**: 함수가 볼록/오목이 아닌데 삼분 탐색을 적용하면 지역 극값에 빠진다
- **비교 방향 혼동**: 최솟값을 찾을 때와 최댓값을 찾을 때 비교 방향이 반대이다. 헷갈리면 그림을 그려본다
- **정수 도메인에서 무한 루프**: lo + 1 == hi일 때 m1 == m2가 되어 구간이 줄지 않을 수 있다. 구간이 작아지면 직접 비교로 전환한다
- **eps 설정 오류**: eps가 너무 크면 정밀도 부족, 너무 작으면 부동소수점 오차로 무한 루프
- **구간 설정 오류**: 초기 구간 [lo, hi]가 극값을 포함하지 않으면 잘못된 답을 얻는다

### 엣지 케이스

- 극값이 구간의 양 끝점에 있는 경우: 삼분 탐색이 정상 동작하지만, 초기 구간 설정에 주의
- 함수가 상수인 경우: f(m1) == f(m2)가 항상 성립하여 양쪽 모두 축소된다
- 극값 부근에서 함수값 차이가 매우 작은 경우: 부동소수점 비교에 주의
- 정수 도메인에서 구간이 1~2인 경우: 직접 비교로 처리

## 관련 알고리즘 비교

| 알고리즘 | 적용 조건 | 시간 복잡도 | 적합한 상황 |
| --- | --- | --- | --- |
| 삼분 탐색 | 유니모달 함수 | O(log N) | 볼록/오목 함수의 극값 탐색 |
| 이진 탐색 | 단조 함수 | O(log N) | 정렬된 배열에서 값 찾기, 결정 문제 |
| 파라메트릭 서치 | 단조 판별 함수 | O(log N × 판별) | "X 이상 가능한가?" 형태의 최적화 |
| 황금 비율 탐색 | 유니모달 함수 | O(log N) | 삼분 탐색과 동일, 함수 호출 1회 절약 |
| 경사 하강법 | 미분 가능 함수 | 수렴 속도 다양 | 고차원 최적화, 머신러닝 |

### 삼분 탐색 vs 이진 탐색

- **이진 탐색**: 함수가 단조 증가/감소할 때 사용. 구간을 2등분하여 한쪽을 제거
- **삼분 탐색**: 함수가 유니모달(볼록/오목)일 때 사용. 구간을 3등분하여 1/3을 제거
- 삼분 탐색은 이진 탐색보다 수렴 속도가 약간 느리다 (log₃/₂ vs log₂)

### 삼분 탐색 vs 미분 기반 이진 탐색

볼록 함수의 도함수 f'(x)는 단조 증가하므로, f'(x) = 0인 점을 이진 탐색으로 찾을 수도 있다. 하지만:
- 도함수를 구하기 어려운 경우가 많다
- 삼분 탐색은 도함수 없이 함수값만으로 동작한다

## 변형 및 최적화

### 황금 비율 탐색 (Golden Section Search)

삼분 탐색의 개선 버전이다. 구간을 황금 비율(φ = (1+√5)/2 ≈ 1.618)로 분할하면, 매 반복에서 함수 호출을 1회만 하면 된다 (삼분 탐색은 2회).

```
φ = (1 + √5) / 2 ≈ 1.618
r = 1 / φ ≈ 0.618

m1 = lo + (1-r) × (hi-lo)
m2 = lo + r × (hi-lo)
```

다음 반복에서 이전 m1 또는 m2를 재활용할 수 있어 함수 호출 횟수가 절반으로 줄어든다.

### 정수 삼분 탐색

정수 도메인에서는 구간이 작아졌을 때 특별한 처리가 필요하다:

```go
for hi - lo > 2 {
    m1 := lo + (hi - lo) / 3
    m2 := hi - (hi - lo) / 3
    if f(m1) > f(m2) {
        lo = m1
    } else {
        hi = m2
    }
}
// 남은 후보 직접 비교
best := lo
for x := lo; x <= hi; x++ {
    if f(x) < f(best) {
        best = x
    }
}
```

### 2차원 삼분 탐색

2변수 볼록 함수 f(x, y)의 최솟값을 찾을 때, x에 대해 삼분 탐색하면서 각 x에 대해 y를 삼분 탐색하는 중첩 구조를 사용한다. 시간 복잡도는 O(log²N)이다.
