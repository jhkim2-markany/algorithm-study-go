# 오일러 경로 테크닉 (Euler Tour Technique)

## 개념

오일러 경로 테크닉(Euler Tour Technique, ETT)이란 트리를 DFS로 순회하면서 각 노드에 방문 시작 시각(in-time)과 종료 시각(out-time)을 부여하여, 트리 구조를 1차원 배열로 평탄화(flatten)하는 기법이다.

핵심 아이디어는 **서브트리가 연속 구간으로 매핑된다**는 것이다. 노드 v의 서브트리에 속하는 모든 노드는 배열에서 `[in[v], out[v]]` 구간에 연속으로 나타난다. 이 성질을 이용하면 서브트리에 대한 질의나 갱신을 세그먼트 트리, 펜윅 트리 등 구간 자료구조로 O(log N)에 처리할 수 있다.

대표적인 활용:
- **서브트리 합/최솟값/최댓값 질의**: 서브트리를 구간으로 변환하여 세그먼트 트리로 처리
- **서브트리 일괄 갱신**: 구간 갱신(lazy propagation)으로 서브트리 전체에 값을 더하거나 변경
- **경로 질의**: 오일러 투어와 LCA를 결합하여 경로 질의 처리
- **노드 값 변경 후 서브트리 질의**: 단일 노드 갱신 + 구간 질의

## 동작 원리

### 1. 오일러 투어 구성 (DFS 평탄화)

트리를 루트에서 DFS로 순회하면서 각 노드에 타임스탬프를 부여한다.

1. 전역 타이머 `timer = 0`을 준비한다
2. 루트에서 DFS를 시작한다
3. 노드 v에 처음 방문할 때 `in[v] = timer`를 기록하고 `timer++`
4. v의 모든 자식을 재귀적으로 방문한다
5. v의 모든 자식 방문이 끝나면 `out[v] = timer - 1`을 기록한다

```
핵심 성질:
- 노드 v의 서브트리 = 배열의 [in[v], out[v]] 구간
- u가 v의 서브트리에 속함 ⟺ in[v] ≤ in[u] ≤ out[v]
- 오일러 투어 배열의 크기 = N (노드 수)
```

### 2. 평탄화 배열 구성

DFS 방문 순서대로 노드를 배열에 배치한다. `euler[in[v]] = v`로 매핑하면, 서브트리 질의를 구간 질의로 변환할 수 있다.

```
euler 배열: [euler[0], euler[1], ..., euler[N-1]]
             DFS 방문 순서대로 노드 번호가 들어간다

값 배열:    [val[euler[0]], val[euler[1]], ..., val[euler[N-1]]]
             각 위치에 해당 노드의 값을 배치한다
```

### 3. 세그먼트 트리와 결합

평탄화된 배열 위에 세그먼트 트리를 구축하면:
- **서브트리 합 질의**: `query(in[v], out[v])` → O(log N)
- **노드 값 갱신**: `update(in[v], newVal)` → O(log N)
- **서브트리 일괄 갱신**: `rangeUpdate(in[v], out[v], delta)` → O(log N) (lazy propagation)

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 오일러 투어 구성 (DFS) | O(N) |
| 세그먼트 트리 구축 | O(N) |
| 서브트리 질의 (1회) | O(log N) |
| 노드 값 갱신 (1회) | O(log N) |
| 서브트리 일괄 갱신 (1회) | O(log N) |
| 공간 복잡도 | O(N) |

전처리 O(N) 후, 각 질의/갱신을 O(log N)에 처리한다.

## 적합한 문제 유형

- 서브트리에 속하는 노드들의 합, 최솟값, 최댓값을 반복적으로 질의하는 문제
- 특정 노드의 값을 변경한 뒤 서브트리 질의를 수행하는 문제
- 서브트리 전체에 값을 일괄 더하거나 변경하는 문제
- 트리에서 특정 노드가 다른 노드의 서브트리에 속하는지 O(1)에 판별하는 문제
- 트리 경로 질의를 오일러 투어 + LCA로 처리하는 문제
- 오프라인 쿼리에서 DFS 순서를 활용하는 문제

## 단계별 추적 (Trace)

### 예시: 오일러 투어 구성 및 서브트리 합 질의

다음 트리에서 오일러 투어를 수행하고, 서브트리 합을 구한다 (루트: 1).

```
        1 (val=3)
       / \
      2   3 (val=1)
(val=5) \
    / \   4 (val=2)
   5   6
(val=4)(val=7)
```

노드 값: val = [_, 3, 5, 1, 2, 4, 7] (1-indexed)
간선: (1,2), (1,3), (2,5), (2,6), (3,4)

**DFS 순회 과정 (timer = 0부터 시작):**

```
dfs(1, 0):
  in[1] = 0, timer = 1
  dfs(2, 1):
    in[2] = 1, timer = 2
    dfs(5, 2):
      in[5] = 2, timer = 3
      (리프) out[5] = 2
    dfs(6, 2):
      in[6] = 3, timer = 4
      (리프) out[6] = 3
    out[2] = 3
  dfs(3, 1):
    in[3] = 4, timer = 5
    dfs(4, 3):
      in[4] = 5, timer = 6
      (리프) out[4] = 5
    out[3] = 5
  out[1] = 5
```

**결과 타임스탬프:**

```
노드:    1  2  3  4  5  6
in:      0  1  4  5  2  3
out:     5  3  5  5  2  3
```

**평탄화 배열 (euler):**

```
인덱스:  0  1  2  3  4  5
노드:    1  2  5  6  3  4
값:      3  5  4  7  1  2
```

**서브트리 구간 확인:**

```
노드 1의 서브트리: [in[1], out[1]] = [0, 5] → 전체 배열 → 합 = 3+5+4+7+1+2 = 22
노드 2의 서브트리: [in[2], out[2]] = [1, 3] → [5, 4, 7]  → 합 = 5+4+7 = 16
노드 3의 서브트리: [in[3], out[3]] = [4, 5] → [1, 2]     → 합 = 1+2 = 3
노드 5의 서브트리: [in[5], out[5]] = [2, 2] → [4]         → 합 = 4 (리프)
```

**노드 값 갱신 예시:**

```
노드 6의 값을 7 → 10으로 변경:
  세그먼트 트리에서 인덱스 in[6] = 3의 값을 10으로 갱신

갱신 후 노드 2의 서브트리 합:
  query(1, 3) = 5 + 4 + 10 = 19
```

## 실전 팁

### 활용 노하우

- 오일러 투어의 핵심은 "서브트리 = 연속 구간"이라는 성질이다. 이 성질만 기억하면 다양한 문제에 적용할 수 있다
- `in[v]`와 `out[v]`를 구한 뒤, 서브트리 질의는 항상 `[in[v], out[v]]` 구간으로 변환한다
- 노드 값을 평탄화 배열에 매핑할 때 `euler[in[v]] = val[v]`로 저장하면 세그먼트 트리를 바로 구축할 수 있다
- 서브트리 포함 판별: `in[v] ≤ in[u] ≤ out[v]`이면 u는 v의 서브트리에 속한다. O(1)에 판별 가능하다
- 경로 질의가 필요하면 오일러 투어 + LCA를 결합하거나, HLD(Heavy-Light Decomposition)를 고려한다

### 자주 하는 실수

- `in`과 `out`의 인덱스 범위를 혼동하는 실수. `out[v]`가 마지막 자식의 `out` 값과 같은지 확인하자
- 0-indexed와 1-indexed를 혼용하는 실수. 타이머 시작값과 세그먼트 트리 인덱스를 통일하자
- 오일러 투어 배열에 노드 번호 대신 값을 넣어야 하는데 혼동하는 실수. `euler` 배열과 `val` 배열의 역할을 구분하자
- 양방향 간선에서 부모 방향으로 역행하는 실수. DFS 시 부모 노드를 인자로 전달하여 방지한다

### 엣지 케이스

- N = 1: 노드가 하나뿐인 트리. `in[1] = 0, out[1] = 0`, 서브트리 합 = val[1]
- 일직선 트리 (경로 그래프): 1-2-3-...-N 형태. `in[i] = i-1, out[i] = N-1-(깊이에 따라 다름)`
- 스타 그래프: 중심 노드의 `in = 0, out = N-1`, 리프 노드의 구간 길이 = 1
- 값이 0인 노드: 합 질의에서 0이 포함되어도 구간 합에 영향을 주므로 누락하지 않도록 주의
- 갱신 없이 질의만 있는 경우: 세그먼트 트리 대신 누적 합(prefix sum)으로 더 간단하게 처리 가능

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 오일러 투어 + 세그먼트 트리 | 서브트리를 구간으로 변환, O(log N) 질의 | 서브트리 질의/갱신 반복 |
| HLD (Heavy-Light Decomposition) | 경로를 O(log N)개 구간으로 분할 | 경로 질의/갱신 반복 |
| DFS 순서 + 누적 합 | 서브트리를 구간으로 변환, O(1) 질의 | 갱신 없이 서브트리 질의만 필요 |
| 트리 DP | 서브트리 단위 DP, O(N) | 한 번의 전처리로 결과 계산 |
| LCA + 경로 쿼리 | 두 노드 간 경로, O(log N) | 특정 두 노드 간 경로 질의 |
| 센트로이드 분할 | 트리 분할 정복, O(N log N) | 경로 조건 쌍 세기 |

- 서브트리 질의/갱신 반복 → 오일러 투어 + 세그먼트 트리
- 경로 질의/갱신 반복 → HLD
- 갱신 없이 서브트리 질의 → DFS 순서 + 누적 합
- 한 번의 전처리 → 트리 DP
- 두 노드 간 경로 → LCA
