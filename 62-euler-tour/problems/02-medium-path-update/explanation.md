# 경로 값 갱신 - 해설

## 접근 방식

1. 트리를 인접 리스트로 저장하고 오일러 투어를 수행한다
2. "루트→v 경로에 x를 더한다"를 오일러 투어 배열 위의 구간 연산으로 변환한다
3. 펜윅 트리에 차분 배열(difference array) 기법을 적용하여 구간 갱신 + 점 질의를 처리한다

## 핵심 아이디어

루트에서 노드 v까지의 경로에 x를 더하는 연산을 생각하자. 노드 u의 값은 "u의 조상 중 갱신된 노드들의 기여 합"이다.

오일러 투어에서 노드 v의 서브트리는 구간 `[in[v], out[v]]`에 매핑된다. 이 구간에 x를 더하면, 평탄화 배열에서 `in[v]` 이후에 위치한 모든 노드(= v의 서브트리)에 x가 반영된다.

노드 u의 현재 값을 구할 때는 평탄화 배열에서 `[0, in[u]]` 구간의 합을 구한다. 이는 u의 모든 조상에서 발생한 갱신의 합과 같다.

```
갱신: rangeAdd(in[v], out[v], x)   → v의 서브트리 구간에 x를 더한다
질의: query(in[u])                  → u까지의 누적 합 = u의 현재 값
```

차분 배열 기법을 사용하면:
- 구간 [l, r]에 x를 더하기: `update(l, +x)`, `update(r+1, -x)`
- 점 i의 값 구하기: `prefixSum(i)` = [0, i] 구간 합

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 오일러 투어 (전처리) | O(N) |
| 경로 갱신 (1회) | O(log N) |
| 노드 값 질의 (1회) | O(log N) |
| 전체 시간 복잡도 | O(N + Q log N) |
| 공간 복잡도 | O(N) |
