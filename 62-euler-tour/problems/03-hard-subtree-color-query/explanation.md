# 서브트리 색상 종류 쿼리 - 해설

## 접근 방식

1. 오일러 투어로 트리를 평탄화하여 서브트리를 연속 구간으로 변환한다
2. 색상 변경 질의가 있으므로 Mo's Algorithm with Updates를 적용한다
3. 구간 내 서로 다른 색상의 수를 색상별 등장 횟수 배열로 관리한다

## 핵심 아이디어

이 문제는 두 가지 기법을 결합한다:

**1. 오일러 투어 (서브트리 → 구간 변환)**

노드 v의 서브트리에 속하는 모든 노드는 평탄화 배열에서 `[in[v], out[v]]` 구간에 연속으로 나타난다. 따라서 "서브트리 내 서로 다른 색상 수"는 "구간 내 서로 다른 값의 수"로 변환된다.

**2. Mo's Algorithm with Updates (구간 + 갱신 처리)**

일반 Mo's Algorithm은 갱신이 없는 오프라인 구간 질의를 O(N√N)에 처리한다. 갱신이 있는 경우 시간 축을 추가하여 3차원으로 확장하며, 블록 크기를 N^(2/3)으로 설정하면 O((N+Q) × N^(2/3))에 처리할 수 있다.

```
질의 정렬 기준:
  1차: l / block
  2차: r / block
  3차: t (시간순)

구간 확장/축소 시:
  add(pos): color[euler[pos]]의 등장 횟수 증가, 0→1이면 종류 수 +1
  remove(pos): color[euler[pos]]의 등장 횟수 감소, 1→0이면 종류 수 -1

시간 갱신 적용/되돌리기:
  현재 구간에 포함된 위치면 remove → 색상 변경 → add
  포함되지 않으면 색상만 변경
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 오일러 투어 (전처리) | O(N) |
| Mo's Algorithm 정렬 | O(Q log Q) |
| Mo's Algorithm 실행 | O((N + Q) × N^(2/3)) |
| 전체 시간 복잡도 | O((N + Q) × N^(2/3)) |
| 공간 복잡도 | O(N + Q) |
