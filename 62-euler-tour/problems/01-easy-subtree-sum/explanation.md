# 서브트리 합 구하기 - 해설

## 접근 방식

1. 트리를 인접 리스트로 저장한다
2. 루트(1번 노드)에서 DFS를 수행하여 오일러 투어를 구성한다 (in/out 타임스탬프 기록)
3. 평탄화 배열에 각 노드의 값을 DFS 순서대로 배치한다
4. 펜윅 트리(BIT)를 구축하여 구간 합 질의와 단일 갱신을 지원한다
5. 서브트리 합 질의: `query(in[v], out[v])`로 구간 합을 구한다
6. 노드 값 변경: 기존 값과의 차이를 펜윅 트리에 반영한다

## 핵심 아이디어

오일러 투어의 핵심 성질은 **노드 v의 서브트리가 평탄화 배열에서 연속 구간 `[in[v], out[v]]`에 매핑된다**는 것이다.

이 성질을 이용하면 서브트리 합 질의를 단순한 구간 합 질의로 변환할 수 있다. 펜윅 트리를 사용하면 갱신과 질의 모두 O(log N)에 처리된다.

```
서브트리 합 = rangeQuery(in[v], out[v])
노드 값 변경 = update(in[v], newVal - oldVal)
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 오일러 투어 (전처리) | O(N) |
| 펜윅 트리 구축 | O(N log N) |
| 서브트리 합 질의 (1회) | O(log N) |
| 노드 값 갱신 (1회) | O(log N) |
| 전체 시간 복잡도 | O(N log N + Q log N) |
| 공간 복잡도 | O(N) |
