# 트리

## 개념

트리(Tree)는 노드(Node)와 간선(Edge)으로 이루어진 계층적 자료구조이다. N개의 노드와 N-1개의 간선으로 구성되며, 임의의 두 노드 사이에 정확히 하나의 경로만 존재하는 사이클 없는 연결 그래프이다.

트리의 핵심 용어는 다음과 같다:
- **루트(Root)**: 트리의 최상위 노드. 루트 트리에서는 하나의 노드를 루트로 지정한다.
- **부모(Parent)**: 특정 노드의 바로 위에 연결된 노드
- **자식(Child)**: 특정 노드의 바로 아래에 연결된 노드
- **리프(Leaf)**: 자식이 없는 노드 (단말 노드)
- **깊이(Depth)**: 루트에서 해당 노드까지의 간선 수
- **높이(Height)**: 해당 노드에서 가장 먼 리프까지의 간선 수. 트리의 높이는 루트의 높이이다.
- **서브트리(Subtree)**: 특정 노드를 루트로 하는 부분 트리

트리는 파일 시스템, 조직도, HTML DOM 등 계층적 데이터를 표현하는 데 널리 사용되며, 코딩테스트에서도 매우 빈번하게 출제된다.

## 동작 원리

트리 문제를 풀 때 가장 기본이 되는 것은 트리의 표현과 순회이다.

### 트리 표현

1. 인접 리스트로 트리를 표현한다. 각 노드에 연결된 노드 목록을 저장한다.
2. 루트가 주어지면 BFS 또는 DFS로 부모-자식 관계를 결정한다.

### 트리 순회

트리 순회는 모든 노드를 체계적으로 방문하는 방법이다. 대표적인 4가지 순회 방법이 있다:

1. **전위 순회(Preorder)**: 현재 노드 → 왼쪽 서브트리 → 오른쪽 서브트리 순서로 방문한다. 루트를 가장 먼저 처리할 때 사용한다.
2. **중위 순회(Inorder)**: 왼쪽 서브트리 → 현재 노드 → 오른쪽 서브트리 순서로 방문한다. 이진 탐색 트리에서 정렬된 순서를 얻을 때 사용한다.
3. **후위 순회(Postorder)**: 왼쪽 서브트리 → 오른쪽 서브트리 → 현재 노드 순서로 방문한다. 자식을 먼저 처리한 뒤 부모를 처리할 때 사용한다.
4. **레벨 순회(Level Order)**: 루트부터 깊이 순서대로 같은 레벨의 노드를 왼쪽에서 오른쪽으로 방문한다. BFS(큐)를 사용한다.

### 트리의 높이/깊이 계산

1. DFS로 리프 노드까지 재귀적으로 탐색한다.
2. 각 노드의 높이는 자식 노드 높이의 최댓값 + 1이다.
3. 깊이는 루트에서 해당 노드까지의 거리로, DFS 탐색 시 누적하여 계산한다.

### 부모-자식 관계 구축

1. 루트 노드에서 DFS 또는 BFS를 시작한다.
2. 방문하지 않은 인접 노드를 자식으로 설정한다.
3. 부모 배열(parent[])에 각 노드의 부모를 기록한다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(N) |
| 시간 복잡도 (평균) | O(N) |
| 시간 복잡도 (최악) | O(N) |
| 공간 복잡도 | O(N) |

트리 순회와 대부분의 트리 연산은 모든 노드를 한 번씩 방문하므로 O(N)이다. 인접 리스트 저장에 O(N) 공간이 필요하며, 재귀 호출 스택은 트리의 높이에 비례하여 최악의 경우 O(N)이다.

## 적합한 문제 유형

- 트리 순회(전위/중위/후위/레벨) 결과 출력
- 트리의 높이, 깊이, 지름 계산
- 부모-자식 관계 파악 및 조상 찾기
- 최소 공통 조상(LCA) 문제
- 서브트리 크기, 서브트리 합 계산
- 트리에서의 경로 탐색
- 트리 DP (트리 위에서의 동적 프로그래밍)


## 단계별 추적 (Trace)

### 예시: 트리 순회

다음 트리에서 4가지 순회를 수행한다.

```
입력: 노드 7개, 간선 [(1,2),(1,3),(2,4),(2,5),(3,6),(3,7)]
루트: 1

트리 구조:
         1
        / \
       2   3
      / \ / \
     4  5 6  7
```

### 실행 과정

```
=== 전위 순회 (Preorder): 현재 → 왼쪽 → 오른쪽 ===

방문(1) → 방문(2) → 방문(4) → 방문(5) → 방문(3) → 방문(6) → 방문(7)

추적:
  visit(1): 출력 1, 왼쪽 자식 2로 이동
    visit(2): 출력 2, 왼쪽 자식 4로 이동
      visit(4): 출력 4, 리프 → 복귀
    오른쪽 자식 5로 이동
      visit(5): 출력 5, 리프 → 복귀
  오른쪽 자식 3으로 이동
    visit(3): 출력 3, 왼쪽 자식 6으로 이동
      visit(6): 출력 6, 리프 → 복귀
    오른쪽 자식 7로 이동
      visit(7): 출력 7, 리프 → 복귀

결과: [1, 2, 4, 5, 3, 6, 7]

=== 중위 순회 (Inorder): 왼쪽 → 현재 → 오른쪽 ===
결과: [4, 2, 5, 1, 6, 3, 7]

=== 후위 순회 (Postorder): 왼쪽 → 오른쪽 → 현재 ===
결과: [4, 5, 2, 6, 7, 3, 1]

=== 레벨 순회 (Level Order): BFS ===

큐: [1]
Level 0: 출력 1, 자식 2,3 추가 → 큐: [2, 3]
Level 1: 출력 2, 자식 4,5 추가 → 큐: [3, 4, 5]
         출력 3, 자식 6,7 추가 → 큐: [4, 5, 6, 7]
Level 2: 출력 4, 5, 6, 7

결과: [1, 2, 3, 4, 5, 6, 7]
```

### 트리 높이 계산 추적

```
         1 (높이=2)
        / \
       2   3 (높이=1)
      / \ / \
     4  5 6  7 (높이=0, 리프)

height(4) = 0
height(5) = 0
height(2) = max(height(4), height(5)) + 1 = 1
height(6) = 0
height(7) = 0
height(3) = max(height(6), height(7)) + 1 = 1
height(1) = max(height(2), height(3)) + 1 = 2

트리의 높이 = 2
```

## 실전 팁

### 활용 노하우

- 트리 문제의 80%는 DFS(재귀)로 풀린다. 재귀 함수의 매개변수와 반환값을 명확히 설계하라
- "서브트리"라는 키워드가 나오면 후위 순회(자식 먼저 처리)를 떠올려라
- 트리의 부모-자식 관계를 구축할 때, 양방향 인접 리스트 + visited 배열 패턴을 사용한다
- 트리 지름(가장 먼 두 노드 사이 거리)은 임의의 노드에서 가장 먼 노드를 찾고, 그 노드에서 다시 가장 먼 노드를 찾는 2번 BFS/DFS로 구한다

### 자주 하는 실수

- **부모 방문 방지 누락**: 인접 리스트로 트리를 표현할 때, DFS에서 부모 노드를 다시 방문하면 무한 루프에 빠진다. `parent` 매개변수나 `visited` 배열로 방지한다
- **루트 노드 가정 오류**: 문제에서 루트가 1이 아닐 수 있다. 루트 노드를 문제 조건에서 확인한다
- **리프 노드 처리 누락**: 리프 노드에서의 기저 조건(base case)을 빠뜨리면 잘못된 결과가 나온다
- **0-indexed vs 1-indexed**: 노드 번호가 0부터인지 1부터인지에 따라 배열 크기가 달라진다

### 엣지 케이스 주의사항

- 노드가 1개뿐인 트리 (루트만 존재)
- 한쪽으로 치우친 트리 (사실상 연결 리스트, 높이 = N-1)
- 루트가 리프인 경우 (자식이 없는 루트)
- 간선 정보가 부모→자식이 아닌 양방향으로 주어지는 경우
- 노드 번호가 연속적이지 않은 경우

## 관련 알고리즘 비교

| 자료구조/알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 일반 트리 | N개 노드, N-1개 간선, 사이클 없음 | 계층 구조 표현, 기본 트리 문제 |
| 이진 트리 | 자식이 최대 2개 | 수식 트리, BST, 힙 |
| 이진 탐색 트리 (BST) | 왼쪽 < 루트 < 오른쪽 | 정렬된 데이터 검색/삽입/삭제 |
| 힙 | 완전 이진 트리 + 힙 속성 | 우선순위 큐, Top-K |
| 트라이 | 문자열 접두사 트리 | 문자열 검색, 자동완성 |
| 세그먼트 트리 | 구간 쿼리용 이진 트리 | 구간 합/최솟값/최댓값 쿼리 |
| 그래프 (일반) | 사이클 가능, 다중 경로 | 네트워크, 관계 표현 |

**선택 기준:**
- 계층적 데이터 + 사이클 없음 → 트리
- 정렬된 데이터의 동적 검색 → BST
- 최솟값/최댓값 반복 추출 → 힙
- 구간 쿼리 → 세그먼트 트리
- 문자열 접두사 검색 → 트라이
- 사이클이 있거나 다중 경로 → 그래프
