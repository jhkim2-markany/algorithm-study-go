# LCA 쿼리 - 해설

## 접근 방식

1. 트리를 인접 리스트로 표현하고, 루트(1번)에서 DFS를 수행하여 각 노드의 깊이와 직접 부모를 구한다
2. 희소 테이블(Sparse Table)을 구축한다. `up[k][v]`는 노드 v의 2^k번째 조상을 저장한다
3. 각 쿼리에 대해 이진 리프팅(Binary Lifting)으로 LCA를 구한다:
   - 두 노드의 깊이를 맞춘다
   - 두 노드를 동시에 올려 LCA를 찾는다

## 핵심 아이디어

이진 리프팅의 핵심은 "2의 거듭제곱 단위로 조상을 빠르게 탐색"하는 것이다.

희소 테이블 `up[k][v]`는 다음 점화식으로 구축한다:
- `up[0][v]` = v의 직접 부모
- `up[k][v]` = `up[k-1][up[k-1][v]]` (2^k번째 조상 = 2^(k-1)번째 조상의 2^(k-1)번째 조상)

LCA를 구할 때는 두 단계를 거친다:
1. 깊이 맞추기: 깊이 차이를 이진수로 분해하여 2의 거듭제곱 단위로 올린다
2. 동시 올리기: 큰 k부터 시작하여, 두 노드의 2^k번째 조상이 다르면 함께 올린다. 최종적으로 한 칸 더 올리면 LCA이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (전처리) | O(N log N) |
| 시간 복잡도 (쿼리당) | O(log N) |
| 시간 복잡도 (전체) | O((N + M) log N) |
| 공간 복잡도 | O(N log N) |

희소 테이블 구축에 O(N log N) 시간과 공간이 필요하다. 각 쿼리는 O(log N)에 처리되므로, M개의 쿼리를 처리하는 전체 시간은 O((N + M) log N)이다.

## 대안적 접근

- 오일러 경로(Euler Tour) + 희소 테이블(RMQ)을 사용하면 전처리 O(N log N), 쿼리 O(1)에 LCA를 구할 수 있다. 쿼리 수가 매우 많을 때 유리하다.
- 나이브한 방법으로 두 노드의 깊이를 맞춘 뒤 한 칸씩 올리면 쿼리당 O(N)이다. N과 M이 작을 때는 이 방법으로도 충분하다.
- Tarjan의 오프라인 LCA 알고리즘은 유니온 파인드를 활용하여 O(N + M × α(N))에 모든 쿼리를 처리한다.
