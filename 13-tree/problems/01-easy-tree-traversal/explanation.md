# 트리 순회 - 해설

## 접근 방식

1. 트리를 인접 리스트로 표현한다
2. 자식 노드를 번호 순서대로 방문하기 위해 각 인접 리스트를 정렬한다
3. 루트(1번 노드)에서 DFS를 시작하여 전위 순회를 수행한다
4. 전위 순회는 현재 노드를 먼저 방문한 뒤 자식 노드를 재귀적으로 방문한다
5. 부모 노드로 되돌아가지 않도록 이전 노드(부모)를 매개변수로 전달한다

## 핵심 아이디어

전위 순회의 핵심은 "현재 노드를 먼저 처리하고, 그 다음 자식을 순서대로 처리"하는 것이다. 루트 트리에서는 부모-자식 관계가 명시적으로 주어지지 않으므로, DFS 탐색 시 이전에 방문한 노드(부모)를 기록하여 역방향 탐색을 방지한다.

인접 리스트를 정렬하면 자식 노드를 항상 번호가 작은 순서대로 방문할 수 있어, 일관된 순회 결과를 보장한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

인접 리스트 정렬에 O(N log N), DFS 순회에 O(N)이 소요된다. 인접 리스트와 결과 배열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- 정렬 없이 자식 노드를 입력 순서대로 방문할 수도 있다. 문제에서 특정 순서를 요구하지 않는 경우 정렬을 생략하여 O(N)으로 해결할 수 있다.
- 재귀 대신 스택을 사용한 반복적 전위 순회도 가능하다. 재귀 깊이가 깊어질 수 있는 경우(편향 트리) 스택 오버플로를 방지할 수 있다.
