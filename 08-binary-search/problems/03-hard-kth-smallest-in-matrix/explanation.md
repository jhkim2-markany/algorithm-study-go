# 행렬에서 K번째 수 - 해설

## 접근 방식

1. 행렬의 최솟값(좌상단)과 최댓값(우하단)을 이진 탐색의 범위로 설정한다
2. 중간 값 mid에 대해 "행렬에서 mid 이하인 원소가 몇 개인가?"를 계산한다
3. 그 개수가 K보다 작으면 답은 mid보다 크고, K 이상이면 답은 mid 이하이다
4. 범위가 하나의 값으로 수렴하면 그 값이 K번째 수이다

## 핵심 아이디어

이 문제의 핵심은 "값에 대한 이진 탐색"이다. 인덱스가 아닌 값의 범위에서 이진 탐색을 수행한다.

어떤 값 X에 대해 "행렬에서 X 이하인 원소의 개수"는 X가 커질수록 단조 증가한다. 이 단조성을 이용하여 이진 탐색을 적용한다.

mid 이하인 원소의 개수를 세는 방법: 각 행이 정렬되어 있으므로, 행마다 이진 탐색(upper_bound)을 수행하면 O(N × log N)에 계산할 수 있다. 또는 행과 열이 모두 정렬되어 있는 성질을 이용하여 우상단에서 시작하는 계단식 탐색으로 O(N)에 계산할 수도 있다.

이진 탐색의 결과가 반드시 행렬에 존재하는 값으로 수렴하는 이유는, lo < hi 조건에서 count < k일 때 lo = mid + 1로 올리고, count >= k일 때 hi = mid로 내리기 때문이다. 최종적으로 lo == hi가 되는 시점의 값은 "K개 이상의 원소가 그 값 이하인 최소 값"이며, 이는 반드시 행렬에 존재하는 값이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × log N × log V) |
| 공간 복잡도 | O(N²) |

V는 값의 범위(최댓값 - 최솟값)이다. 이진 탐색이 O(log V)번 반복되고, 각 반복에서 N개 행에 대해 O(log N) 이진 탐색을 수행하므로 전체 O(N × log N × log V)이다. N = 1,500, V = 2 × 10⁹일 때 약 1,500 × 11 × 31 ≈ 511,500 연산으로 충분히 빠르다.

## 대안적 접근

- **계단식 카운팅**: mid 이하인 원소를 셀 때, 우상단(0행 N-1열)에서 시작하여 값이 mid 이하이면 아래로, 초과이면 왼쪽으로 이동하는 방식으로 O(N)에 계산할 수 있다. 전체 O(N × log V).
- **힙 활용**: 최소 힙에 각 행의 첫 원소를 넣고 K번 추출하는 방식. O(K × log N)이지만 K가 N²에 가까우면 비효율적이다.
- **전체 정렬**: 모든 원소를 배열에 넣고 정렬 후 K번째를 찾는 방식. O(N² log N²)으로 N이 클 때 느리다.
