# 값 찾기 - 해설

## 접근 방식

1. N개의 정수를 배열에 저장한다 (이미 오름차순 정렬되어 있음)
2. 각 질의마다 이진 탐색을 수행한다
3. 탐색 범위의 중간 값과 목표 값을 비교하여 범위를 절반씩 줄인다
4. 값을 찾으면 인덱스를, 찾지 못하면 -1을 출력한다

## 핵심 아이디어

배열이 이미 정렬되어 있으므로 이진 탐색을 바로 적용할 수 있다. 매 단계에서 중간 값과 목표 값을 비교하여 탐색 범위를 절반으로 줄이면, 최대 log₂N번의 비교만으로 결과를 얻는다.

선형 탐색(O(N))으로도 풀 수 있지만, N과 M이 최대 100,000이므로 최악의 경우 O(N×M) = 10¹⁰ 연산이 필요하여 시간 초과가 발생한다. 이진 탐색을 사용하면 O(M × log N)으로 충분히 빠르다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M × log N) |
| 공간 복잡도 | O(N) |

M개의 질의 각각에 대해 O(log N) 이진 탐색을 수행하므로 전체 시간 복잡도는 O(M × log N)이다. 배열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **해시맵 사용**: 배열의 값을 키, 인덱스를 값으로 하는 해시맵을 만들면 각 질의를 O(1)에 처리할 수 있다. 전체 O(N + M)이지만 추가 메모리가 필요하다.
- **Go의 sort.SearchInts 활용**: 표준 라이브러리의 이진 탐색 함수를 사용할 수도 있다.
