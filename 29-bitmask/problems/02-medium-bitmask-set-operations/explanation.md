# 비트마스크 집합 연산 - 해설

## 접근 방식

1. 집합 S를 하나의 정수(비트마스크)로 표현한다
2. 원소 x는 (x-1)번째 비트에 대응시킨다
3. 각 연산을 비트 연산으로 변환하여 O(1)에 처리한다
4. union과 intersect는 상대 집합도 비트마스크로 변환한 뒤 OR, AND 연산을 수행한다
5. 최종 마스크에서 1인 비트에 해당하는 원소를 오름차순으로 출력한다

## 핵심 아이디어

N ≤ 20이므로 집합의 모든 상태를 하나의 정수(최대 20비트)로 표현할 수 있다. 비트마스크를 사용하면 집합 연산을 단일 비트 연산으로 처리할 수 있어 매우 효율적이다.

- add: OR 연산으로 비트를 설정한다
- remove: AND NOT 연산으로 비트를 해제한다
- toggle: XOR 연산으로 비트를 반전한다
- union: OR 연산으로 합집합을 구한다
- intersect: AND 연산으로 교집합을 구한다

이미 포함된 원소에 add를 하거나, 없는 원소에 remove를 해도 비트 연산의 특성상 자연스럽게 무시된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q × K) |
| 공간 복잡도 | O(1) |

각 연산은 O(1)이지만, union과 intersect에서 K개의 원소를 읽어야 하므로 최악의 경우 O(Q × N)이다. N ≤ 20이므로 실질적으로 O(Q)에 가깝다.

## 대안적 접근

- 배열이나 해시셋으로 집합을 관리할 수도 있으나, 합집합과 교집합 연산에 O(N) 시간이 필요하다. 비트마스크는 이를 O(1)로 줄여준다. N이 작을 때(N ≤ 64) 비트마스크가 가장 효율적인 방법이다.
