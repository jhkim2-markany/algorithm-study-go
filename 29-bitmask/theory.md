# 비트마스킹

## 개념

비트마스킹(Bitmask)은 정수의 이진 표현(비트)을 활용하여 집합의 상태를 효율적으로 표현하고 조작하는 기법이다. N개의 원소로 이루어진 집합의 부분집합을 하나의 정수로 나타낼 수 있으며, 비트 연산(AND, OR, XOR, NOT, 시프트)을 통해 집합 연산을 O(1)에 수행할 수 있다.

예를 들어, 5개의 원소 {0, 1, 2, 3, 4}가 있을 때 부분집합 {0, 2, 4}는 이진수 10101, 즉 정수 21로 표현한다. 각 비트의 위치가 원소의 포함 여부를 나타내므로, 비트 연산만으로 원소 추가, 삭제, 포함 여부 확인, 합집합, 교집합 등을 빠르게 처리할 수 있다.

## 동작 원리

비트마스킹의 핵심은 6가지 기본 비트 연산이다.

### 1. AND 연산 (&)

두 비트가 모두 1일 때만 1을 반환한다. 집합의 교집합에 해당한다.

```
  1 0 1 1 0  (집합 A = {1, 3, 4})
& 1 1 0 1 0  (집합 B = {1, 3, 4} → 실제 {1, 3, 4})
-----------
  1 0 0 1 0  (A ∩ B)
```

### 2. OR 연산 (|)

두 비트 중 하나라도 1이면 1을 반환한다. 집합의 합집합에 해당한다.

```
  1 0 1 1 0  (집합 A)
| 1 1 0 1 0  (집합 B)
-----------
  1 1 1 1 0  (A ∪ B)
```

### 3. XOR 연산 (^)

두 비트가 다를 때 1을 반환한다. 집합의 대칭 차집합에 해당한다.

```
  1 0 1 1 0  (집합 A)
^ 1 1 0 1 0  (집합 B)
-----------
  0 1 1 0 0  (A △ B)
```

### 4. NOT 연산 (^, Go에서는 ^)

모든 비트를 반전한다. 집합의 여집합에 해당한다.

### 5. 왼쪽 시프트 (<<)

비트를 왼쪽으로 이동한다. `1 << i`는 i번째 비트만 1인 값을 만든다.

### 6. 비트마스크 기본 조작

| 연산 | 코드 | 설명 |
| --- | --- | --- |
| i번째 비트 설정 (Set) | `mask \| (1 << i)` | 원소 i를 집합에 추가 |
| i번째 비트 확인 (Check) | `mask & (1 << i)` | 원소 i가 집합에 있는지 확인 |
| i번째 비트 해제 (Clear) | `mask &^ (1 << i)` | 원소 i를 집합에서 제거 |
| i번째 비트 토글 (Toggle) | `mask ^ (1 << i)` | 원소 i의 포함 여부를 반전 |
| 전체 집합 | `(1 << n) - 1` | n개 원소 전체를 포함하는 집합 |
| 공집합 | `0` | 아무 원소도 포함하지 않는 집합 |
| 부분집합 열거 | `for sub := mask; sub > 0; sub = (sub - 1) & mask` | mask의 모든 부분집합을 순회 |

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 비트 연산 (Set/Check/Clear/Toggle) | O(1) |
| 집합 연산 (합집합/교집합/차집합) | O(1) |
| 전체 부분집합 열거 | O(2^N) |
| 특정 마스크의 부분집합 열거 | O(2^K) (K = 마스크의 비트 수) |
| 공간 복잡도 | O(1) (마스크 하나당) |

비트 연산 자체는 상수 시간이지만, 모든 부분집합을 열거하면 지수 시간이 소요된다. 따라서 비트마스킹은 N이 작을 때(보통 N ≤ 20) 효과적이다.

## 적합한 문제 유형

- 부분집합을 열거하거나 탐색하는 문제 (N ≤ 20)
- 상태를 비트로 압축하여 DP에 활용하는 문제 (비트필드 DP)
- 집합 연산(합집합, 교집합, 차집합)을 빠르게 수행해야 하는 문제
- 외판원 문제(TSP) 등 방문 상태를 관리하는 문제
- 플래그나 옵션을 비트로 관리하는 문제
- 원소 수가 적은 집합의 모든 조합을 탐색하는 문제
- 비트 연산을 활용한 수학적 성질을 이용하는 문제

## 단계별 추적 (Trace)

### 예시: 비트마스크로 부분집합 열거

집합 {0, 1, 2}의 모든 부분집합을 비트마스크로 열거하는 과정을 추적한다.

```
N = 3, 전체 집합 = (1 << 3) - 1 = 7 = 0b111

mask = 0 (0b000) → 공집합 {}
mask = 1 (0b001) → {0}
mask = 2 (0b010) → {1}
mask = 3 (0b011) → {0, 1}
mask = 4 (0b100) → {2}
mask = 5 (0b101) → {0, 2}
mask = 6 (0b110) → {1, 2}
mask = 7 (0b111) → {0, 1, 2}

총 2³ = 8개의 부분집합
```

### 예시: 특정 마스크의 부분집합 열거

mask = 5 (0b101, 집합 {0, 2})의 모든 부분집합을 열거한다.

```
mask = 5 = 0b101

반복 과정:
  sub = 5 (0b101) → {0, 2}
  sub = (5-1) & 5 = 4 & 5 = 4 (0b100) → {2}
  sub = (4-1) & 5 = 3 & 5 = 1 (0b001) → {0}
  sub = (1-1) & 5 = 0 & 5 = 0 (0b000) → {} (종료)

결과: {0,2}, {2}, {0}, {} (4개 = 2² 개)
```

### 예시: 비트 연산으로 집합 조작

```
A = 0b1010 (집합 {1, 3})
B = 0b1100 (집합 {2, 3})

합집합:   A | B  = 0b1110 = {1, 2, 3}
교집합:   A & B  = 0b1000 = {3}
차집합:   A &^ B = 0b0010 = {1}
대칭차:   A ^ B  = 0b0110 = {1, 2}

원소 2 추가: A | (1<<2) = 0b1010 | 0b0100 = 0b1110 = {1, 2, 3}
원소 1 제거: A &^ (1<<1) = 0b1010 &^ 0b0010 = 0b1000 = {3}
원소 3 확인: A & (1<<3) = 0b1010 & 0b1000 = 0b1000 ≠ 0 → 포함됨
```

## 실전 팁

### 활용 노하우

- N ≤ 20이면 비트마스크로 모든 부분집합을 열거할 수 있다 (2²⁰ ≈ 10⁶)
- 비트마스크 DP와 결합하면 순열 탐색(N!)을 부분집합 탐색(2^N)으로 줄일 수 있다
- `bits.OnesCount(uint(mask))`로 집합의 원소 수를 O(1)에 구할 수 있다 (Go의 math/bits 패키지)
- 부분집합 열거 패턴 `for sub := mask; sub > 0; sub = (sub - 1) & mask`는 반드시 외워두라

### 자주 하는 실수

- `1 << i`에서 i가 32 이상일 때 오버플로우 발생. `int64(1) << i`로 캐스팅해야 한다
- Go에서 NOT 연산은 `^mask`이지만, 이는 모든 비트를 반전하므로 `mask ^ ((1<<n)-1)`로 N비트 내에서만 반전해야 한다
- 부분집합 열거 시 공집합(sub = 0)을 별도로 처리해야 한다 (루프 조건이 sub > 0이므로)
- `&^` 연산자(AND NOT)를 `&(^x)`로 잘못 쓰는 실수

### 엣지 케이스

- 공집합 (mask = 0): 부분집합은 공집합 하나뿐
- 전체 집합 (mask = (1<<N)-1): 부분집합은 2^N개
- N = 0인 경우: 전체 집합이 0, 부분집합은 공집합 하나
- N = 1인 경우: 부분집합은 {}, {0} 두 개

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 비트마스킹 | O(1) 집합 연산, O(2^N) 열거 | 집합 상태 표현, N ≤ 20 |
| 비트필드 DP | O(2^N × N) 이상 | 순열/배정 최적화, N ≤ 20 |
| 백트래킹 | 가지치기로 탐색 공간 축소 | 제약 조건이 많아 가지치기가 효과적일 때 |
| 브루트포스 순열 | O(N!) | N ≤ 10, 모든 순열을 직접 탐색 |
| 해시 집합 | O(1) 평균 삽입/조회 | 원소 수가 많고 동적 집합 관리 필요 시 |
| 비트셋 (bitset) | 대량 비트 연산 최적화 | N이 크지만 비트 연산이 필요한 경우 |
