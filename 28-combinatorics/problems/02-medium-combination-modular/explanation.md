# 조합의 수 (모듈러 연산) - 해설

## 접근 방식

1. 팩토리얼 배열 fact[i] = i! mod p를 전처리한다 (0부터 N까지)
2. 역팩토리얼 배열 invFact[i] = (i!)⁻¹ mod p를 전처리한다
3. 역원은 페르마의 소정리를 이용한다: a⁻¹ ≡ a^(p-2) mod p
4. 각 쿼리에 대해 C(n, r) = fact[n] × invFact[r] × invFact[n-r] mod p로 계산한다

## 핵심 아이디어

N이 최대 1,000,000이므로 파스칼의 삼각형(O(N²))은 메모리와 시간 모두 부족하다. 대신 팩토리얼 전처리 + 모듈러 역원 방식을 사용한다.

핵심은 페르마의 소정리이다. p가 소수일 때 a^(p-1) ≡ 1 (mod p)이므로, a⁻¹ ≡ a^(p-2) (mod p)가 성립한다. 이를 이용하면 나눗셈을 곱셈으로 변환할 수 있다.

역팩토리얼은 invFact[N]을 먼저 구한 뒤, invFact[i] = invFact[i+1] × (i+1) 관계를 이용하여 역순으로 O(N)에 전처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + log P + T) |
| 공간 복잡도 | O(N) |

팩토리얼 전처리 O(N), 모듈러 거듭제곱 O(log P), 각 쿼리 O(1)이므로 전체 시간 복잡도는 O(N + log P + T)이다.

## 대안적 접근

- 뤼카의 정리(Lucas' Theorem)를 사용하면 N이 매우 크고 p가 작은 소수일 때 효율적이다. 하지만 p = 10⁹ + 7처럼 큰 소수에서는 팩토리얼 전처리 방식이 더 적합하다.
