# 조합론

## 개념

조합론(Combinatorics)은 유한 집합에서 원소를 선택하거나 배열하는 경우의 수를 체계적으로 계산하는 수학 분야이다. 코딩테스트에서는 순열(Permutation), 조합(Combination), 이항 계수(Binomial Coefficient) 계산이 핵심이며, 큰 수에 대한 모듈러 연산과 결합하여 출제되는 경우가 많다.

기본 도구로는 팩토리얼, 파스칼의 삼각형, 페르마의 소정리를 이용한 모듈러 역원 등이 있다. 이를 통해 nCr, nPr 등의 값을 효율적으로 계산할 수 있다.

## 동작 원리

### 1. 순열 (Permutation)

n개의 원소에서 r개를 순서를 고려하여 선택하는 경우의 수이다.

```
nPr = n! / (n-r)!
```

1. n부터 (n-r+1)까지의 수를 곱한다
2. 또는 n!을 구한 뒤 (n-r)!로 나눈다

### 2. 조합 (Combination)

n개의 원소에서 r개를 순서 없이 선택하는 경우의 수이다.

```
nCr = n! / (r! × (n-r)!)
```

1. 순열 nPr을 구한다
2. 중복 순서를 제거하기 위해 r!로 나눈다

### 3. 파스칼의 삼각형

이항 계수의 점화식을 이용한 계산 방법이다.

```
C(n, r) = C(n-1, r-1) + C(n-1, r)
```

1. 2차원 배열 dp를 생성한다
2. dp[i][0] = dp[i][i] = 1로 초기화한다
3. dp[i][j] = dp[i-1][j-1] + dp[i-1][j]로 채운다
4. dp[n][r]이 nCr의 값이다

### 4. 모듈러 역원을 이용한 이항 계수

큰 수의 이항 계수를 소수 p로 나눈 나머지를 구하는 방법이다.

1. 팩토리얼 배열 fact[i] = i! mod p를 전처리한다
2. 역팩토리얼 배열 inv_fact[i] = (i!)⁻¹ mod p를 전처리한다
3. nCr mod p = fact[n] × inv_fact[r] × inv_fact[n-r] mod p로 계산한다
4. 역원은 페르마의 소정리(a⁻¹ ≡ a^(p-2) mod p)로 구한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 파스칼의 삼각형 (전처리) | O(N²) |
| 파스칼의 삼각형 (쿼리) | O(1) |
| 팩토리얼 전처리 | O(N) |
| 모듈러 역원 (거듭제곱) | O(log P) |
| 이항 계수 쿼리 (전처리 후) | O(1) |
| 공간 복잡도 (파스칼) | O(N²) |
| 공간 복잡도 (팩토리얼) | O(N) |

파스칼의 삼각형은 N이 작을 때(약 5,000 이하) 적합하고, 팩토리얼 + 모듈러 역원 방식은 N이 클 때(최대 10⁶ 이상) 적합하다.

## 적합한 문제 유형

- 경우의 수를 구하는 문제 (nCr, nPr 계산)
- 격자 경로의 수를 구하는 문제
- 이항 계수를 모듈러 연산으로 구하는 문제
- 중복 조합, 중복 순열 문제
- 카탈란 수, 스털링 수 등 특수 조합 수열 문제
- 확률 계산에서 경우의 수가 필요한 문제
- 파스칼의 삼각형 성질을 활용하는 문제

## 단계별 추적 (Trace)

### 예시: 파스칼의 삼각형으로 C(5, 2) 계산

```
입력: n=5, r=2

1단계: dp 테이블 초기화 및 채우기

행 0: [1]
행 1: [1, 1]
행 2: [1, 2, 1]        ← C(2,1) = C(1,0)+C(1,1) = 1+1 = 2
행 3: [1, 3, 3, 1]     ← C(3,1) = 1+2 = 3, C(3,2) = 2+1 = 3
행 4: [1, 4, 6, 4, 1]  ← C(4,2) = C(3,1)+C(3,2) = 3+3 = 6
행 5: [1, 5, 10, 10, 5, 1]

2단계: 결과 읽기
  C(5, 2) = dp[5][2] = 10

시각화 (파스칼의 삼각형):
          1
         1 1
        1 2 1
       1 3 3 1
      1 4 6 4 1
     1 5 [10] 10 5 1   ← C(5,2) = 10
```

### 예시: 모듈러 역원으로 C(10, 3) mod 1000000007 계산

```
입력: n=10, r=3, p=1000000007

1단계: 팩토리얼 전처리
  fact[0] = 1
  fact[1] = 1
  fact[2] = 2
  fact[3] = 6
  ...
  fact[10] = 3628800

2단계: 역팩토리얼 계산 (페르마의 소정리)
  inv_fact[3] = 6^(p-2) mod p
  inv_fact[7] = 5040^(p-2) mod p

3단계: 이항 계수 계산
  C(10,3) = fact[10] × inv_fact[3] × inv_fact[7] mod p
          = 3628800 × inv(6) × inv(5040) mod p
          = 120

결과: C(10, 3) = 120
```

## 실전 팁

### 활용 노하우

- N이 작으면(≤ 5000) 파스칼의 삼각형, N이 크면(≤ 10⁶) 팩토리얼 + 모듈러 역원 방식을 사용하라
- 모듈러가 소수인지 확인하라. 소수이면 페르마의 소정리로 역원을 구하고, 소수가 아니면 확장 유클리드를 사용해야 한다
- 격자 경로 문제는 대부분 C(n+m, n)으로 변환 가능하다
- 중복 조합 H(n, r) = C(n+r-1, r)임을 기억하라

### 자주 하는 실수

- 팩토리얼 전처리 범위를 부족하게 잡는 실수 (n까지가 아니라 n+1까지 필요할 수 있음)
- 모듈러 연산을 중간 과정에서 빠뜨려 오버플로우가 발생하는 실수
- C(n, r)에서 r > n인 경우를 처리하지 않는 실수 (결과는 0)
- 역원 계산 시 거듭제곱의 지수를 p-2가 아닌 p-1로 잘못 쓰는 실수

### 엣지 케이스

- r = 0 또는 r = n인 경우 (결과는 항상 1)
- r > n인 경우 (결과는 0)
- n = 0인 경우
- 모듈러 값이 매우 작은 경우 (결과가 0이 될 수 있음)

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 파스칼의 삼각형 | O(N²) 전처리, 모듈러 불필요 | N ≤ 5000, 여러 쿼리 |
| 팩토리얼 + 모듈러 역원 | O(N) 전처리, 소수 모듈러 필요 | N ≤ 10⁶, 큰 수 |
| 뤼카 정리 (Lucas) | O(P² + log_P(N)) | 모듈러 P가 작은 소수일 때 |
| 포함 배제 원리 | 조건부 경우의 수 계산 | 제약 조건이 있는 조합 문제 |
| 카탈란 수 | C(2n,n)/(n+1), 특수 조합 수열 | 괄호 쌍, 이진 트리 수 등 |
| 스털링 수 | 분할/순환 순열의 수 | 집합 분할, 순환 순열 문제 |
