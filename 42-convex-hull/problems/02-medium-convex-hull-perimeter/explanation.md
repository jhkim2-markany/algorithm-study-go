# 볼록 껍질 둘레 계산 - 해설

## 접근 방식

1. N개의 점을 입력받아 Andrew's Monotone Chain으로 볼록 껍질을 구한다.
2. 볼록 껍질의 꼭짓점을 순서대로 순회하며 인접한 두 점 사이의 유클리드 거리를 계산한다.
3. 마지막 점과 첫 번째 점 사이의 거리도 포함하여 모든 변의 길이를 합산한다.
4. 결과를 소수점 둘째 자리까지 출력한다.

## 핵심 아이디어

볼록 껍질을 구한 뒤 둘레를 계산하는 것은 단순히 인접 꼭짓점 간 거리의 합이다. 핵심은 볼록 껍질을 정확하게 구하는 것이며, Monotone Chain 알고리즘이 꼭짓점을 반시계 방향 순서로 반환하므로 순서대로 거리를 더하면 된다.

주의할 점은 일직선 위의 점을 껍질에 포함하지 않아야 한다는 것이다. CCW 값이 0인 경우도 제거 대상에 포함하여 변 위의 중간 점을 제외한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

볼록 껍질 구성에 O(N log N), 둘레 계산에 O(H) (H: 껍질 꼭짓점 수)가 소요된다. H ≤ N이므로 전체 시간 복잡도는 O(N log N)이다.

## 대안적 접근

볼록 껍질의 둘레는 모든 점을 감싸는 최소 울타리 길이와 같다. 만약 울타리와 점 사이에 일정 거리 L을 두어야 한다면, 볼록 껍질 둘레에 2πL을 더하면 된다. 이는 "성벽 문제"로 알려진 응용이다.
