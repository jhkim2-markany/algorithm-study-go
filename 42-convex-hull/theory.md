# 볼록 껍질

## 개념

볼록 껍질(Convex Hull)은 2차원 평면 위에 주어진 점 집합을 모두 포함하는 가장 작은 볼록 다각형이다. 직관적으로, 모든 점에 못을 박고 고무줄로 감쌌을 때 고무줄이 이루는 다각형이 볼록 껍질이다.

볼록 껍질을 구하는 대표적인 알고리즘:

- **Graham Scan**: 기준점에서 각도 정렬 후 스택을 이용하여 볼록 껍질을 구성한다
- **Andrew's Monotone Chain**: x좌표 기준 정렬 후 상부 껍질과 하부 껍질을 각각 구성하여 합친다
- **Jarvis March (Gift Wrapping)**: 가장 외곽 점에서 시작하여 반시계 방향으로 다음 점을 하나씩 찾아간다

코딩테스트에서는 구현이 간결한 Andrew's Monotone Chain이 가장 많이 사용된다.

## 선수 학습

볼록 껍질을 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [27-geometry](../27-geometry/) - 외적(Cross Product), CCW 판별 등 기하학의 기본 개념
- [03-sorting](../03-sorting/) - 좌표 정렬, 각도 정렬 등 볼록 껍질 알고리즘에서 필수적인 정렬 기법

## 동작 원리

### 1. CCW (Counter-Clockwise) 판별

볼록 껍질 알고리즘의 핵심 연산이다. 세 점 P₁, P₂, P₃의 방향을 외적으로 판별한다:

```
CCW = (P₂.x - P₁.x) * (P₃.y - P₁.y) - (P₃.x - P₁.x) * (P₂.y - P₁.y)
```

- 양수: 반시계 방향 (좌회전)
- 음수: 시계 방향 (우회전)
- 0: 일직선 (직진)

### 2. Graham Scan

1. y좌표가 가장 작은 점(같으면 x좌표가 가장 작은 점)을 기준점으로 선택한다
2. 기준점에서 나머지 점들을 각도(atan2) 기준으로 정렬한다
3. 스택에 처음 두 점을 넣는다
4. 나머지 점을 순서대로 처리하며:
   - 스택의 상위 두 점과 현재 점이 좌회전(CCW > 0)이 아니면 스택에서 제거한다
   - 현재 점을 스택에 추가한다
5. 스택에 남은 점들이 볼록 껍질의 꼭짓점이다

### 3. Andrew's Monotone Chain

1. 모든 점을 x좌표 기준으로 정렬한다 (같으면 y좌표 기준)
2. 하부 껍질(Lower Hull) 구성:
   - 왼쪽에서 오른쪽으로 점을 순서대로 처리한다
   - 마지막 두 점과 현재 점이 우회전(CCW < 0)이거나 직진(CCW = 0)이면 마지막 점을 제거한다
   - 현재 점을 추가한다
3. 상부 껍질(Upper Hull) 구성:
   - 오른쪽에서 왼쪽으로 점을 순서대로 처리한다
   - 같은 방식으로 우회전/직진이면 제거하고 추가한다
4. 하부 껍질과 상부 껍질을 합쳐 볼록 껍질을 완성한다

### 4. 회전하는 캘리퍼스 (Rotating Calipers)

볼록 껍질 위에서 가장 먼 두 점(지름)을 O(N)에 구하는 기법:

1. 볼록 껍질을 먼저 구한다
2. 가장 왼쪽 점과 가장 오른쪽 점에서 시작한다
3. 두 평행선(캘리퍼스)을 동시에 회전시키며 대척점 쌍을 탐색한다
4. 각 대척점 쌍의 거리를 비교하여 최대 거리를 갱신한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| Graham Scan | O(N log N) |
| Andrew's Monotone Chain | O(N log N) |
| Jarvis March | O(NH) (H: 껍질 꼭짓점 수) |
| 회전하는 캘리퍼스 | O(N) (볼록 껍질 구성 후) |
| 공간 복잡도 | O(N) |

정렬이 O(N log N)으로 병목이며, 껍질 구성 자체는 O(N)이다. Jarvis March는 출력 민감(output-sensitive) 알고리즘으로, 껍질 꼭짓점 수 H가 작으면 효율적이다.

## 적합한 문제 유형

- 주어진 점 집합의 볼록 껍질 꼭짓점을 구하는 문제
- 볼록 껍질의 둘레 또는 넓이를 계산하는 문제
- 가장 먼 두 점의 거리(지름)를 구하는 문제
- 모든 점을 감싸는 최소 울타리 길이를 구하는 문제
- 볼록 다각형 내부 점 판정 문제
- 두 볼록 다각형의 교차 판정 문제

## 단계별 추적 (Trace)

### 예시: Andrew's Monotone Chain으로 볼록 껍질 구하기

입력 점: (0,0), (1,1), (2,2), (3,1), (4,0), (2,3), (1,2)

```
[1단계] x좌표 기준 정렬 (같으면 y좌표)
  정렬 결과: (0,0) (1,1) (1,2) (2,2) (2,3) (3,1) (4,0)

[2단계] 하부 껍질 (Lower Hull) — 왼쪽→오른쪽
  추가 (0,0): hull = [(0,0)]
  추가 (1,1): hull = [(0,0), (1,1)]
  추가 (1,2): CCW((0,0),(1,1),(1,2)) > 0 (좌회전) → 유지
              hull = [(0,0), (1,1), (1,2)]
  추가 (2,2): CCW((1,1),(1,2),(2,2)) < 0 (우회전) → (1,2) 제거
              CCW((0,0),(1,1),(2,2)) = 0 (직진) → (1,1) 제거
              hull = [(0,0), (2,2)]
  추가 (2,3): CCW((0,0),(2,2),(2,3)) > 0 (좌회전) → 유지
              hull = [(0,0), (2,2), (2,3)]
  추가 (3,1): CCW((2,2),(2,3),(3,1)) < 0 (우회전) → (2,3) 제거
              CCW((0,0),(2,2),(3,1)) < 0 (우회전) → (2,2) 제거
              hull = [(0,0), (3,1)]
  추가 (4,0): CCW((0,0),(3,1),(4,0)) < 0 (우회전) → (3,1) 제거
              hull = [(0,0), (4,0)]

  하부 껍질: [(0,0), (4,0)]

[3단계] 상부 껍질 (Upper Hull) — 오른쪽→왼쪽
  추가 (4,0): hull = [(4,0)]
  추가 (3,1): hull = [(4,0), (3,1)]
  추가 (2,3): CCW((4,0),(3,1),(2,3)) > 0 → 유지
              hull = [(4,0), (3,1), (2,3)]
  추가 (2,2): CCW((3,1),(2,3),(2,2)) < 0 → (2,3) 제거
              hull = [(4,0), (3,1), (2,2)]
              CCW((4,0),(3,1),(2,2)) ... 계속 진행
  ... (생략)
  추가 (0,0): 최종 상부 껍질 완성

[4단계] 합치기
  하부 + 상부 (중복 제거) → 볼록 껍질 완성

최종 볼록 껍질 꼭짓점 (반시계):
  (0,0) → (4,0) → (2,3) → (0,0)
```

### CCW 판별 시각화

```
  P3          P3
   \         /
    \  CCW  /
     \ > 0 /
  P1──P2    P1──P2
  (좌회전)    (우회전, CCW < 0)

  P1──P2──P3
  (일직선, CCW = 0)
```

## 실전 팁

### 활용 노하우

- Andrew's Monotone Chain이 Graham Scan보다 구현이 간결하고 실수가 적다. 코딩테스트에서는 Monotone Chain을 추천
- CCW 함수는 정수 연산으로 구현하여 부동소수점 오차를 피하자. 좌표가 정수이면 외적도 정수로 계산 가능
- 볼록 껍질 위의 점 개수가 필요한 문제에서는 일직선 위의 점(CCW = 0)을 포함할지 제외할지 문제 조건을 잘 확인하자
- 회전하는 캘리퍼스는 볼록 껍질을 먼저 구한 뒤 O(N)으로 최대 거리를 구하므로, 전체 O(N log N)이다

### 자주 하는 실수

- CCW 판별에서 등호(= 0, 일직선) 처리를 잘못하여 볼록 껍질에 불필요한 점이 포함되거나 필요한 점이 빠지는 경우
- 좌표 범위가 클 때(10^5 이상) 외적 계산에서 int32 오버플로 발생. int64를 사용해야 한다
- 모든 점이 일직선 위에 있는 경우 볼록 껍질이 선분이 되는 예외 처리 누락
- 정렬 기준을 잘못 설정하여 동일 x좌표에서 y좌표 순서가 뒤바뀌는 경우
- 상부/하부 껍질을 합칠 때 시작점과 끝점이 중복되는 것을 처리하지 않는 경우

### 엣지 케이스

- 점이 1개 또는 2개인 경우 — 볼록 껍질이 점 또는 선분
- 모든 점이 일직선 위에 있는 경우
- 동일한 좌표의 점이 여러 개 있는 경우 (중복 점)
- 좌표 값이 매우 큰 경우 (외적 계산 시 오버플로 주의)
- 볼록 껍질 꼭짓점이 3개 미만인 경우 넓이가 0

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| Andrew's Monotone Chain | O(N log N) | x좌표 정렬 후 상/하부 껍질 구성, 구현 간결 | 코딩테스트에서 가장 추천 |
| Graham Scan | O(N log N) | 기준점에서 각도 정렬 후 스택 사용 | 각도 기반 접근이 필요할 때 |
| Jarvis March (Gift Wrapping) | O(NH) | 출력 민감, 껍질 꼭짓점 수 H가 작으면 빠름 | 껍질 꼭짓점이 매우 적을 때 |
| 분할 정복 볼록 껍질 | O(N log N) | 점 집합을 분할하여 각각 볼록 껍질을 구한 뒤 병합 | 이론적 관심, 실전에서는 거의 사용 안 함 |
| 회전하는 캘리퍼스 | O(N) (껍질 구성 후) | 볼록 껍질 위에서 최대 거리 등을 효율적으로 계산 | 가장 먼 두 점, 최소 폭 등 |
