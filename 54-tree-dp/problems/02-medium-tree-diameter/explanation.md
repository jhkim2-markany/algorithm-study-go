# 트리의 지름 - 해설

## 접근 방식

1. 트리를 가중치 포함 인접 리스트로 저장한다
2. 루트(1번)에서 DFS 후위 순회를 수행한다
3. 각 노드에서 서브트리 내 가장 먼 리프까지의 거리(`depth[v]`)를 계산한다
4. 각 노드를 "꺾는 점"으로 하는 경로의 길이 = 가장 긴 두 자식 경로의 합
5. 모든 노드에 대해 이 값의 최댓값이 트리의 지름이다

## 핵심 아이디어

트리의 지름은 반드시 어떤 노드를 "꺾는 점(LCA)"으로 하는 경로이다. 각 노드 v에서 자식 방향으로 내려가는 경로 중 가장 긴 두 경로를 합하면 v를 꺾는 점으로 하는 최장 경로가 된다.

```
diameter = max(max1[v] + max2[v])  (모든 노드 v에 대해)
```

여기서 `max1[v]`는 v에서 서브트리 내 가장 먼 리프까지의 거리, `max2[v]`는 두 번째로 먼 리프까지의 거리이다.

DFS 한 번으로 모든 노드의 `depth`와 지름 후보를 동시에 계산할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

DFS 한 번으로 모든 노드를 방문하므로 O(N)이다.

## 대안적 접근

BFS를 2번 수행하는 방법도 있다. 임의의 노드에서 BFS로 가장 먼 노드 A를 찾고, A에서 다시 BFS로 가장 먼 노드 B를 찾으면 A-B 거리가 지름이다. 이 방법도 O(N)이지만, 트리 DP 방식이 더 일반적이고 확장성이 좋다.
