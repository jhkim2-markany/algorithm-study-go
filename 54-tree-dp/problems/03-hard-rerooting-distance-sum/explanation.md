# 모든 노드의 거리 합 - 해설

## 접근 방식

나이브하게 각 노드에서 BFS를 수행하면 O(N²)이다. 리루팅(Rerooting) 기법을 사용하면 O(N)에 해결할 수 있다.

1. **1단계 (Bottom-Up DFS):** 루트를 1번으로 잡고 DFS를 수행한다. 각 노드의 서브트리 크기 `sz[v]`와 서브트리 내 거리 합 `dp[v]`를 계산한다.
2. **2단계 (Top-Down 리루팅):** 루트에서 다시 DFS를 수행한다. 부모의 전체 거리 합으로부터 자식의 전체 거리 합을 O(1)에 계산한다.

## 핵심 아이디어

리루팅의 핵심 공식:

```
ans[child] = ans[parent] - sz[child] + (N - sz[child])
```

루트를 parent에서 child로 옮기면:
- child의 서브트리에 속한 `sz[child]`개의 노드는 child에 1씩 가까워진다 → `-sz[child]`
- 나머지 `N - sz[child]`개의 노드는 child에서 1씩 멀어진다 → `+(N - sz[child])`

이 공식을 통해 부모의 결과로부터 자식의 결과를 O(1)에 유도할 수 있으므로, 전체 시간 복잡도는 O(N)이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

DFS를 2번 수행하므로 O(2N) = O(N)이다.

## 대안적 접근

각 노드에서 BFS를 수행하는 O(N²) 방법이 있지만, N이 크면 시간 초과가 발생한다. 리루팅은 "모든 노드에 대해 ~를 구하라"는 유형의 트리 문제에서 핵심적인 최적화 기법이다.
