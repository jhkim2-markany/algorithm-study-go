# 트리 DP

## 개념

트리 DP(Tree Dynamic Programming)란 트리 구조에서 동적 프로그래밍을 적용하는 기법이다. 트리는 사이클이 없는 연결 그래프이므로, 임의의 노드를 루트로 잡으면 부모-자식 관계가 결정되고, 각 서브트리가 독립적인 부분 문제가 된다.

핵심 아이디어는 **리프 노드에서 루트 방향으로(Bottom-Up)** DP 값을 계산하는 것이다. DFS 후위 순회를 통해 자식 노드의 결과를 먼저 구한 뒤, 부모 노드의 DP 값을 자식들의 결과로부터 합산한다.

대표적인 트리 DP 유형:
- **서브트리 합/크기**: 각 노드를 루트로 하는 서브트리의 크기, 합, 최댓값 등을 구한다
- **트리의 지름**: 트리에서 가장 먼 두 노드 사이의 거리를 구한다
- **리루팅(Rerooting)**: 모든 노드를 루트로 삼았을 때의 결과를 O(N)에 구한다
- **독립 집합/매칭**: 트리에서 최대 독립 집합이나 최대 매칭을 구한다

## 동작 원리

### 1. 기본 트리 DP (Bottom-Up)

트리를 루트 있는 트리로 변환한 뒤, DFS 후위 순회로 DP 값을 계산한다.

1. 임의의 노드를 루트로 선택한다 (보통 노드 1)
2. 루트에서 DFS를 시작한다
3. 각 노드에서 모든 자식 노드를 먼저 방문한다 (후위 순회)
4. 자식 노드의 DP 값이 모두 계산된 후, 현재 노드의 DP 값을 계산한다
5. 점화식: `dp[v] = f(dp[child1], dp[child2], ...)`

```
점화식 예시 (서브트리 크기):
dp[v] = 1 + Σ dp[child]  (자기 자신 + 모든 자식 서브트리 크기)

점화식 예시 (최대 독립 집합):
dp[v][0] = Σ max(dp[child][0], dp[child][1])  (v를 선택하지 않음)
dp[v][1] = weight[v] + Σ dp[child][0]          (v를 선택함)
```

### 2. 리루팅 (Rerooting) 기법

모든 노드를 루트로 삼았을 때의 DP 값을 구하는 기법이다. 나이브하게 N번 DFS를 하면 O(N²)이지만, 리루팅을 사용하면 O(N)에 해결할 수 있다.

**2단계 DFS 접근:**

**1단계 (Bottom-Up DFS):**
1. 노드 1을 루트로 잡고 DFS를 수행한다
2. 각 노드에 대해 서브트리 기준 DP 값 `dp[v]`를 계산한다

**2단계 (Top-Down DFS - 리루팅):**
1. 루트에서 다시 DFS를 수행한다
2. 부모 노드의 "전체 트리 기준" 결과에서 현재 자식의 기여분을 빼면, 자식을 루트로 삼았을 때의 결과를 얻는다
3. 점화식: `ans[child] = dp[child] + (ans[parent] - dp[child]에 대한 기여분)`

핵심 원리: 루트를 v에서 자식 u로 옮기면, u의 서브트리는 그대로이고 v의 나머지 부분이 u의 새로운 서브트리가 된다.

### 3. 트리의 지름 (DP 방식)

트리에서 가장 먼 두 노드 사이의 거리를 DP로 구한다.

1. 각 노드 v에 대해 `depth[v]` = v에서 서브트리 내 가장 먼 리프까지의 거리
2. 트리의 지름 후보 = 각 노드에서 가장 긴 두 경로의 합
3. `diameter = max(depth[child1] + depth[child2] + 2)` (모든 노드에 대해)

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 기본 트리 DP (Bottom-Up) | O(N) |
| 리루팅 (Rerooting) | O(N) |
| 트리의 지름 (DP) | O(N) |
| 최대 독립 집합 | O(N) |
| 공간 복잡도 | O(N) |

모든 트리 DP는 각 노드와 간선을 정확히 한 번씩 방문하므로 O(N)이다.

## 적합한 문제 유형

- 각 노드의 서브트리에 대한 통계(크기, 합, 최댓값 등)를 구하는 문제
- 트리의 지름, 중심, 무게 중심을 구하는 문제
- 모든 노드를 루트로 삼았을 때의 결과를 구하는 문제 (리루팅)
- 트리에서 최대 독립 집합, 최소 버텍스 커버를 구하는 문제
- 트리에서 특정 조건을 만족하는 경로의 수를 세는 문제
- 트리의 간선을 끊어 서브트리를 분할하는 문제
- 트리에서 각 노드까지의 거리 합을 구하는 문제

## 단계별 추적 (Trace)

### 예시 1: 서브트리 크기 구하기

다음 트리에서 각 노드의 서브트리 크기를 구한다 (루트: 1).

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

간선: (1,2), (1,3), (2,4), (2,5), (3,6)

**DFS 후위 순회 과정:**

```
dfs(1)
  dfs(2)
    dfs(4) → 리프, dp[4] = 1
    dfs(5) → 리프, dp[5] = 1
    dp[2] = 1 + dp[4] + dp[5] = 1 + 1 + 1 = 3
  dfs(3)
    dfs(6) → 리프, dp[6] = 1
    dp[3] = 1 + dp[6] = 1 + 1 = 2
  dp[1] = 1 + dp[2] + dp[3] = 1 + 3 + 2 = 6
```

**결과:**

```
노드:  1  2  3  4  5  6
dp:    6  3  2  1  1  1
```

### 예시 2: 리루팅 - 모든 노드의 거리 합

다음 트리에서 각 노드로부터 다른 모든 노드까지의 거리 합을 구한다.

```
    1 --- 2 --- 3
          |
          4
```

N = 4, 간선: (1,2), (2,3), (2,4)

**1단계: Bottom-Up (루트 = 1)**

```
서브트리 크기:
  sz[4] = 1, sz[3] = 1, sz[2] = 1+1+1 = 3, sz[1] = 4

서브트리 내 거리 합 (dp[v] = 서브트리 내 모든 노드까지의 거리 합):
  dp[4] = 0, dp[3] = 0
  dp[2] = dp[3] + sz[3] + dp[4] + sz[4] = 0+1+0+1 = 2
  dp[1] = dp[2] + sz[2] = 2+3 = 5
```

**2단계: Top-Down 리루팅**

```
ans[1] = dp[1] = 5

루트를 1→2로 옮길 때:
  ans[2] = ans[1] - sz[2] + (N - sz[2])
         = 5 - 3 + (4 - 3) = 5 - 3 + 1 = 3

루트를 2→3으로 옮길 때:
  ans[3] = ans[2] - sz[3] + (N - sz[3])
         = 3 - 1 + (4 - 1) = 3 - 1 + 3 = 5

루트를 2→4로 옮길 때:
  ans[4] = ans[2] - sz[4] + (N - sz[4])
         = 3 - 1 + (4 - 1) = 3 - 1 + 3 = 5
```

**리루팅 핵심 공식:**

```
ans[child] = ans[parent] - sz[child] + (N - sz[child])
```

루트를 parent에서 child로 옮기면:
- child의 서브트리 노드들은 1씩 가까워진다 → -sz[child]
- 나머지 노드들은 1씩 멀어진다 → +(N - sz[child])

**결과:**

```
노드:  1  2  3  4
ans:   5  3  5  5
```

검증: 노드 2에서 다른 노드까지 거리 = 1(→1) + 1(→3) + 1(→4) = 3 ✓

## 실전 팁

### 활용 노하우

- 트리 DP의 첫 단계는 항상 "루트를 잡고 DFS 후위 순회"이다. 루트는 보통 노드 1로 잡는다
- 점화식을 설계할 때 "현재 노드의 DP 값 = 자식들의 DP 값으로부터 계산"이라는 패턴을 따른다
- 리루팅이 필요한 문제의 신호: "모든 노드에 대해 ~를 구하라"는 문구가 있으면 리루팅을 의심하자
- 인접 리스트로 트리를 저장하고, `parent` 배열이나 DFS 시 부모를 인자로 전달하여 역방향 탐색을 방지한다
- 재귀 깊이가 깊을 수 있으므로 (N ≤ 10⁵~10⁶), Go에서는 스택 오버플로우 걱정이 적지만 다른 언어에서는 반복적 DFS를 고려한다

### 자주 하는 실수

- 루트를 잡지 않고 양방향 간선을 그대로 사용하여 무한 루프에 빠지는 실수. DFS 시 부모 노드를 기록하여 역방향 탐색을 방지하자
- 리루팅에서 부모의 기여분을 잘못 계산하는 실수. 공식을 정확히 유도하고 작은 예시로 검증하자
- 리프 노드의 초기값을 잘못 설정하는 실수. 리프 노드는 자식이 없으므로 DP 초기값을 명확히 정의하자
- 간선 가중치가 있는 트리에서 가중치를 누락하는 실수. 간선 가중치를 인접 리스트에 함께 저장하자

### 엣지 케이스

- N = 1: 노드가 하나뿐인 트리. 서브트리 크기 = 1, 거리 합 = 0
- 일직선 트리 (경로 그래프): 1-2-3-...-N 형태. 리루팅에서 양 끝 노드의 값이 가장 크다
- 스타 그래프: 중심 노드에 N-1개의 리프가 연결된 형태. 중심 노드의 서브트리 크기 = N
- 간선 가중치가 0인 경우: 거리 계산에서 0 가중치 간선을 빠뜨리지 않도록 주의

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 트리 DP (Bottom-Up) | 서브트리 단위 DP, O(N) | 서브트리 통계, 최대 독립 집합 |
| 리루팅 (Rerooting) | 모든 노드를 루트로, O(N) | 모든 노드에 대한 결과 계산 |
| 트리 BFS (2회) | 지름 구하기, O(N) | 트리의 지름 (가중치 없는 경우) |
| LCA + 경로 쿼리 | 두 노드 간 경로, O(log N) 쿼리 | 특정 두 노드 간 경로 질의 |
| 센트로이드 분할 | 트리 분할 정복, O(N log N) | 경로 조건 만족 쌍 세기 |
| 오일러 투어 + 세그먼트 트리 | 서브트리 쿼리, O(log N) 쿼리 | 서브트리 갱신/질의 반복 |

- 서브트리 단위 한 번 계산 → 트리 DP
- 모든 노드에 대한 결과 → 리루팅
- 두 노드 간 경로 질의 → LCA
- 경로 조건 쌍 세기 → 센트로이드 분할
- 서브트리 갱신/질의 반복 → 오일러 투어 + 세그먼트 트리

## 변형 및 최적화

### 다중 상태 트리 DP

각 노드에 여러 상태를 두는 트리 DP이다. 예를 들어 최대 독립 집합에서 `dp[v][0]` (v를 선택하지 않음)과 `dp[v][1]` (v를 선택함)으로 나눈다.

```
dp[v][0] = Σ max(dp[child][0], dp[child][1])
dp[v][1] = weight[v] + Σ dp[child][0]
```

### 간선 기반 트리 DP

노드가 아닌 간선에 DP 값을 정의하는 변형이다. 간선을 끊었을 때 생기는 두 서브트리에 대한 정보를 관리한다.
