# DAG 경로 수와 최대 비용 - 해설

## 접근 방식

1. 그래프를 인접 리스트로 구성하면서 진입 차수를 계산한다
2. Kahn's Algorithm으로 위상 정렬을 수행한다
3. 세 가지 DP 배열을 초기화한다:
   - totalPaths[v]: S에서 v까지의 전체 경로 수
   - maxDist[v]: S에서 v까지의 최장 거리
   - maxPaths[v]: 최장 거리를 달성하는 경로 수
4. 위상 정렬 순서대로 각 정점을 처리하며 세 배열을 동시에 갱신한다
5. 도착 정점 T의 값을 출력한다

## 핵심 아이디어

이 문제는 DAG DP의 세 가지 변형을 하나의 위상 정렬 순회에서 동시에 처리하는 것이 핵심이다.

경로 수 계산은 단순 덧셈이다. 정점 u에서 v로 가는 간선이 있으면 totalPaths[v] += totalPaths[u]로 갱신한다.

최장 경로와 최장 경로 수는 함께 관리해야 한다. 간선 (u, v, w)를 처리할 때:
- maxDist[u] + w > maxDist[v]이면 더 긴 경로를 발견한 것이므로 거리와 경로 수를 모두 갱신한다
- maxDist[u] + w == maxDist[v]이면 같은 길이의 다른 경로를 발견한 것이므로 경로 수만 추가한다

경로 수가 매우 커질 수 있으므로 MOD(1,000,000,007)로 나눈 나머지를 관리한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(V + E) |
| 공간 복잡도 | O(V + E) |

위상 정렬에 O(V + E), DP 갱신에 O(V + E)가 소요된다. 세 가지 DP를 동시에 수행하지만 각각 O(V + E)이므로 전체 시간 복잡도는 O(V + E)이다.

## 대안적 접근

DFS + 메모이제이션으로 각 DP를 별도로 계산할 수도 있다. 하지만 세 가지 값을 한 번의 위상 정렬 순회에서 동시에 계산하는 것이 코드가 간결하고 효율적이다. 또한 역방향 그래프를 구성하여 T에서 역추적하는 방법도 가능하지만, 정방향 위상 정렬이 더 직관적이다.
