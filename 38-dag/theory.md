# DAG

## 개념

DAG(Directed Acyclic Graph, 방향 비순환 그래프)는 방향 간선을 가지면서 사이클이 존재하지 않는 그래프이다. 사이클이 없다는 성질 덕분에 위상 정렬이 가능하고, 위상 정렬 순서를 따라 동적 프로그래밍(DP)을 효율적으로 수행할 수 있다.

DAG의 핵심 특성은 다음과 같다:
- 모든 간선이 방향을 가진다 (u → v)
- 어떤 정점에서 출발하여 자기 자신으로 돌아오는 경로가 존재하지 않는다
- 위상 정렬이 항상 가능하다
- 위상 정렬 순서대로 DP를 수행하면 모든 부분 문제가 이미 해결된 상태에서 현재 문제를 풀 수 있다

DAG DP는 그래프 위에서의 최단 경로, 최장 경로, 경로 수 세기 등 다양한 문제를 선형 시간에 해결할 수 있게 해준다. Dijkstra나 Bellman-Ford와 달리 음수 가중치가 있어도 문제없이 동작하며, 시간 복잡도도 O(V + E)로 매우 효율적이다.

## 동작 원리

### 1. DAG 판별

주어진 방향 그래프가 DAG인지 확인하는 방법:

1. DFS를 수행하면서 각 정점의 상태를 관리한다 (미방문, 방문 중, 방문 완료)
2. DFS 도중 "방문 중"인 정점을 다시 만나면 사이클이 존재한다
3. 사이클이 없으면 DAG이다
4. 또는 위상 정렬을 시도하여, 모든 정점이 정렬에 포함되면 DAG이다

### 2. DAG에서의 위상 정렬

DAG의 정점을 선형 순서로 나열하는 과정:

1. 각 정점의 진입 차수(in-degree)를 계산한다
2. 진입 차수가 0인 정점을 큐에 넣는다
3. 큐에서 정점을 꺼내 결과에 추가하고, 해당 정점에서 나가는 간선을 제거한다
4. 간선 제거로 진입 차수가 0이 된 정점을 큐에 넣는다
5. 큐가 빌 때까지 반복한다

### 3. DAG 최단 경로

위상 정렬 순서를 이용한 최단 경로 계산:

1. 위상 정렬을 수행한다
2. 시작 정점의 거리를 0, 나머지를 무한대로 초기화한다
3. 위상 정렬 순서대로 각 정점 u를 처리한다
4. u에서 나가는 모든 간선 (u, v, w)에 대해 dist[v] = min(dist[v], dist[u] + w)로 갱신한다
5. 모든 정점 처리가 끝나면 dist 배열이 최단 거리이다

### 4. DAG 최장 경로

최단 경로와 동일한 방식이지만 갱신 조건만 반대:

1. 위상 정렬을 수행한다
2. 시작 정점의 거리를 0, 나머지를 음의 무한대로 초기화한다
3. 위상 정렬 순서대로 각 정점 u를 처리한다
4. u에서 나가는 모든 간선 (u, v, w)에 대해 dist[v] = max(dist[v], dist[u] + w)로 갱신한다
5. 모든 정점 처리가 끝나면 dist 배열이 최장 거리이다

### 5. DAG DP (DAG 위에서의 동적 프로그래밍)

DAG의 구조를 활용한 일반적인 DP 패턴:

1. 위상 정렬로 정점의 처리 순서를 결정한다
2. 각 정점에 대한 DP 값을 초기화한다
3. 위상 정렬 순서대로 각 정점을 처리한다
4. 현재 정점의 DP 값을 이전 정점(선행 정점)들의 DP 값으로부터 계산한다
5. 이 방식으로 경로 수 세기, 최대/최소 비용 계산 등 다양한 문제를 풀 수 있다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| DAG 판별 (DFS) | O(V + E) |
| 위상 정렬 (BFS/Kahn) | O(V + E) |
| DAG 최단/최장 경로 | O(V + E) |
| DAG DP | O(V + E) |
| 공간 복잡도 | O(V + E) |

DAG에서의 모든 기본 연산은 정점 수 V와 간선 수 E에 비례하는 선형 시간에 수행된다. 이는 Dijkstra의 O((V + E) log V)나 Bellman-Ford의 O(VE)보다 효율적이다.

## 적합한 문제 유형

- DAG에서 시작점부터 끝점까지의 최단/최장 경로를 구하는 문제
- DAG에서 두 정점 사이의 경로 수를 세는 문제
- 작업 스케줄링에서 전체 소요 시간(크리티컬 패스)을 구하는 문제
- 의존 관계가 있는 작업들의 최소 완료 시간을 구하는 문제
- DAG 위에서 최대/최소 비용 경로를 구하는 문제
- 위상 정렬 순서를 활용한 DP 문제
- 사이클 없는 그래프에서의 도달 가능성 판별 문제


## 단계별 추적 (Trace)

### 예시: DAG 최장 경로

```
그래프 (정점 5개, 간선 7개):
  0 → 1 (가중치 3)
  0 → 2 (가중치 6)
  1 → 2 (가중치 4)
  1 → 3 (가중치 4)
  1 → 4 (가중치 11)
  2 → 3 (가중치 8)
  2 → 4 (가중치 2)

시각화:
       3       4
  0 ------→ 1 ------→ 3
  |          |  \       ↑
  | 6        |4  \11    | 8
  ↓          ↓    ↓    |
  2 ------→ 2 ------→ 4
       (자기자신 아님, 1→2)

1단계: 진입 차수 계산
  정점:    0  1  2  3  4
  진입차수: 0  1  2  2  2

2단계: 위상 정렬 (BFS/Kahn)
  큐: [0]
  → 0 처리: 간선 0→1, 0→2 제거
    진입차수: 0  0  1  2  2
    큐: [1]
  → 1 처리: 간선 1→2, 1→3, 1→4 제거
    진입차수: 0  0  0  1  1
    큐: [2]
  → 2 처리: 간선 2→3, 2→4 제거
    진입차수: 0  0  0  0  0
    큐: [3, 4]
  → 3 처리, → 4 처리

  위상 정렬 결과: [0, 1, 2, 3, 4]

3단계: 최장 경로 계산 (시작점 = 0)
  초기: dist = [0, -∞, -∞, -∞, -∞]

  정점 0 처리:
    0→1: dist[1] = max(-∞, 0+3) = 3
    0→2: dist[2] = max(-∞, 0+6) = 6
    dist = [0, 3, 6, -∞, -∞]

  정점 1 처리:
    1→2: dist[2] = max(6, 3+4) = 7
    1→3: dist[3] = max(-∞, 3+4) = 7
    1→4: dist[4] = max(-∞, 3+11) = 14
    dist = [0, 3, 7, 7, 14]

  정점 2 처리:
    2→3: dist[3] = max(7, 7+8) = 15
    2→4: dist[4] = max(14, 7+2) = 14
    dist = [0, 3, 7, 15, 14]

  정점 3 처리: 나가는 간선 없음
  정점 4 처리: 나가는 간선 없음

결과: 최장 경로
  0→3: 15 (경로: 0→1→2→3, 비용 3+4+8=15)
  0→4: 14 (경로: 0→1→4, 비용 3+11=14)
```

## 실전 팁

### 활용 노하우

- DAG인지 확인하는 가장 쉬운 방법은 위상 정렬을 시도하는 것이다. 모든 정점이 정렬에 포함되면 DAG이다
- DAG 최단/최장 경로는 Dijkstra보다 빠르고 음수 가중치도 처리 가능하다
- "작업 순서가 정해진 스케줄링" 문제는 대부분 DAG + 위상 정렬 + DP로 풀린다
- 크리티컬 패스(최장 경로)는 프로젝트 관리 문제에서 자주 출제된다
- 경로 수 세기 문제에서는 DP 값이 매우 커질 수 있으므로 모듈러 연산을 잊지 말자

### 자주 하는 실수

- 그래프에 사이클이 있는데 DAG로 가정하고 풀어서 무한 루프에 빠지는 실수
- 위상 정렬 결과에 모든 정점이 포함되었는지 확인하지 않는 실수 (사이클 존재 시 일부 정점 누락)
- 최장 경로 초기값을 0으로 설정하여 시작점이 아닌 정점에서도 경로가 시작되는 실수 → -∞로 초기화해야 한다
- 최단 경로에서 초기값을 -∞로 설정하는 실수 → +∞로 초기화해야 한다
- 간선 방향을 반대로 저장하는 실수

### 엣지 케이스

- 정점이 1개인 DAG: 자기 자신까지의 거리는 0
- 간선이 없는 DAG: 모든 정점이 독립적, 시작점 외 거리는 ∞ (또는 -∞)
- 시작점에서 도달할 수 없는 정점: 거리가 초기값(∞ 또는 -∞) 그대로 유지
- 가중치가 모두 0인 경우: 최단/최장 경로 모두 0
- 음수 가중치가 있는 경우: DAG에서는 문제없이 처리 가능 (Bellman-Ford 불필요)

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 음수 가중치 | 사이클 허용 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| DAG 최단/최장 경로 | O(V + E) | 가능 | 불가 (DAG만) | DAG에서의 경로 문제 |
| Dijkstra | O((V+E) log V) | 불가 | 가능 | 양수 가중치 그래프 |
| Bellman-Ford | O(VE) | 가능 | 가능 | 음수 가중치 그래프 |
| Floyd-Warshall | O(V³) | 가능 | 가능 | 모든 쌍 최단 경로 |
| 위상 정렬 | O(V + E) | - | 불가 (DAG만) | 순서 결정 문제 |

**선택 기준:**
- 그래프가 DAG이면 → DAG DP (가장 효율적)
- 양수 가중치 + 사이클 가능 → Dijkstra
- 음수 가중치 + 사이클 가능 → Bellman-Ford
- 모든 쌍 최단 경로 → Floyd-Warshall (50번 참조)
- 순서만 필요 (가중치 없음) → 위상 정렬 (23번 참조)
