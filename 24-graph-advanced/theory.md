# 그래프 알고리즘 기타

## 개념

그래프 알고리즘 기타는 기본적인 DFS/BFS 탐색, 최단 경로, 최소 신장 트리, 위상 정렬을 넘어서는 고급 그래프 알고리즘들을 포괄한다. 대표적으로 이분 그래프 판별, 강한 연결 요소(SCC), 네트워크 플로우가 있다.

주요 알고리즘의 핵심 특성은 다음과 같다:

- **이분 그래프 판별(Bipartite Check)**: 그래프의 정점을 두 그룹으로 나누어, 같은 그룹 내 정점끼리는 간선이 없도록 할 수 있는지 판별한다. BFS 또는 DFS로 정점을 2가지 색으로 칠하며, 인접한 정점이 같은 색이면 이분 그래프가 아니다.
- **강한 연결 요소(Strongly Connected Components, SCC)**: 방향 그래프에서 서로 도달 가능한 정점들의 최대 부분 집합이다. Kosaraju 알고리즘은 DFS 두 번으로, Tarjan 알고리즘은 DFS 한 번으로 모든 SCC를 구한다.
- **네트워크 플로우(Network Flow)**: 소스(source)에서 싱크(sink)로 보낼 수 있는 최대 유량을 구하는 문제이다. Ford-Fulkerson 방법은 증가 경로(augmenting path)를 반복적으로 찾아 유량을 늘리며, Edmonds-Karp 알고리즘은 BFS로 증가 경로를 찾아 O(VE²) 시간에 최대 유량을 보장한다.

이 알고리즘들은 코딩테스트에서 중상급 난이도로 출제되며, 그래프의 구조적 성질을 깊이 이해해야 풀 수 있다.

## 동작 원리

### 이분 그래프 판별 (BFS 색칠)

1. 모든 정점을 미색칠 상태로 초기화한다.
2. 미색칠 정점을 하나 선택하여 색 0을 칠하고 BFS 큐에 넣는다.
3. 큐에서 정점을 꺼내 인접한 모든 정점을 확인한다.
4. 인접 정점이 미색칠이면 현재 정점과 반대 색을 칠하고 큐에 넣는다.
5. 인접 정점이 현재 정점과 같은 색이면 이분 그래프가 아니다.
6. 모든 정점을 처리할 때까지 반복한다 (비연결 그래프 고려).
7. 충돌 없이 모든 정점을 칠했으면 이분 그래프이다.

### 강한 연결 요소 - Kosaraju 알고리즘

1. 원본 그래프에서 모든 정점에 대해 DFS를 수행하며, 완료 순서대로 스택에 push한다.
2. 모든 간선의 방향을 뒤집은 역방향 그래프를 생성한다.
3. 스택에서 정점을 하나씩 pop하며, 역방향 그래프에서 DFS를 수행한다.
4. 한 번의 DFS로 방문되는 정점들이 하나의 SCC를 구성한다.
5. 스택이 빌 때까지 반복하여 모든 SCC를 구한다.

### 강한 연결 요소 - Tarjan 알고리즘

1. 각 정점에 방문 순서(discovery time)를 부여하며 DFS를 수행한다.
2. 각 정점의 low-link 값을 관리한다 (도달 가능한 가장 작은 방문 순서).
3. DFS 트리에서 자식 정점의 low-link 값으로 현재 정점의 low-link를 갱신한다.
4. 역방향 간선(back edge)을 통해 도달 가능한 정점의 방문 순서로도 low-link를 갱신한다.
5. DFS가 끝난 후 정점의 low-link가 자신의 방문 순서와 같으면, 해당 정점이 SCC의 루트이다.
6. 스택에서 해당 정점까지의 모든 정점을 pop하여 하나의 SCC로 묶는다.

### 네트워크 플로우 - Edmonds-Karp (BFS 기반 Ford-Fulkerson)

1. 잔여 그래프(residual graph)를 초기화한다 (용량 = 원래 용량, 역방향 간선 용량 = 0).
2. BFS로 소스에서 싱크까지의 증가 경로(augmenting path)를 찾는다.
3. 증가 경로가 존재하면, 경로상 간선들의 최소 잔여 용량(bottleneck)을 구한다.
4. 경로상 각 간선의 잔여 용량을 bottleneck만큼 감소시키고, 역방향 간선의 잔여 용량을 bottleneck만큼 증가시킨다.
5. 총 유량에 bottleneck을 더한다.
6. 증가 경로가 더 이상 존재하지 않을 때까지 2~5를 반복한다.
7. 최종 총 유량이 최대 유량이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 이분 그래프 판별 시간 | O(V + E) |
| 이분 그래프 판별 공간 | O(V + E) |
| Kosaraju SCC 시간 | O(V + E) |
| Kosaraju SCC 공간 | O(V + E) |
| Tarjan SCC 시간 | O(V + E) |
| Tarjan SCC 공간 | O(V + E) |
| Edmonds-Karp 시간 | O(V × E²) |
| Edmonds-Karp 공간 | O(V + E) |

이분 그래프 판별과 SCC 알고리즘은 모든 정점과 간선을 한 번씩 처리하므로 O(V + E)이다. Edmonds-Karp 알고리즘은 BFS로 증가 경로를 찾으므로 최대 O(VE)번의 증가가 발생하고, 각 BFS에 O(E)가 걸려 전체 O(VE²)이다.

## 적합한 문제 유형

- 이분 그래프 판별 (그래프 2-색칠 가능 여부)
- 이분 매칭 (최대 매칭 문제)
- 강한 연결 요소 분해 (방향 그래프 구조 분석)
- 2-SAT 문제 (SCC 기반 해결)
- 최대 유량 / 최소 컷 문제
- 이분 그래프 최대 매칭 (네트워크 플로우 변환)
- 간선 연결성 / 정점 연결성 판별
- 방향 그래프의 DAG 축약 (SCC 축약)

## 단계별 추적 (Trace)

### 예시 1: 이분 그래프 판별 (BFS)

정점 4개 (0~3), 간선: 0-1, 1-2, 2-3, 3-0

```
초기: color = [-1, -1, -1, -1]  (-1 = 미색칠)

정점 0에 색 0 칠하기, 큐에 추가
  color = [0, -1, -1, -1],  큐: [0]

Step 1: 큐에서 0 꺼냄 (색 0)
  인접 정점 1: 미색칠 → 색 1 칠하기
  인접 정점 3: 미색칠 → 색 1 칠하기
  color = [0, 1, -1, 1],  큐: [1, 3]

Step 2: 큐에서 1 꺼냄 (색 1)
  인접 정점 0: 색 0 (다른 색) → OK
  인접 정점 2: 미색칠 → 색 0 칠하기
  color = [0, 1, 0, 1],  큐: [3, 2]

Step 3: 큐에서 3 꺼냄 (색 1)
  인접 정점 2: 색 0 (다른 색) → OK
  인접 정점 0: 색 0 (다른 색) → OK

Step 4: 큐에서 2 꺼냄 (색 0)
  인접 정점 1: 색 1 (다른 색) → OK
  인접 정점 3: 색 1 (다른 색) → OK

충돌 없음 → 이분 그래프!
그룹 A = {0, 2}, 그룹 B = {1, 3}
```

### 예시 2: SCC - Kosaraju 알고리즘

정점 5개 (0~4), 간선: 0→1, 1→2, 2→0, 1→3, 3→4

```
1단계: 원본 그래프에서 DFS (완료 순서 기록)
  DFS(0) → DFS(1) → DFS(2) → 0 이미 방문
    2 완료 (스택 push 2)
    → DFS(3) → DFS(4)
      4 완료 (스택 push 4)
    3 완료 (스택 push 3)
  1 완료 (스택 push 1)
  0 완료 (스택 push 0)
  스택: [0, 1, 3, 4, 2] (top = 0)

2단계: 역방향 그래프 생성
  0←1, 1←2, 2←0, 1←3, 3←4
  즉: 1→0, 2→1, 0→2, 3→1, 4→3

3단계: 스택에서 pop하며 역방향 DFS
  pop 0: DFS(0) → 2 → 1 → 방문 완료
    SCC 1: {0, 2, 1} = {0, 1, 2}
  pop 1: 이미 방문
  pop 3: DFS(3) → 방문 완료
    SCC 2: {3}
  pop 4: DFS(4) → 3 이미 방문
    SCC 3: {4}

결과: SCC = [{0,1,2}, {3}, {4}]
```

## 실전 팁

### 활용 노하우

- 이분 그래프 판별은 BFS 색칠이 가장 직관적이고 구현이 쉽다
- SCC를 구한 뒤 각 SCC를 하나의 노드로 축약하면 DAG가 되어 위상 정렬을 적용할 수 있다
- 2-SAT 문제는 SCC로 해결하는 것이 정석이다 (같은 SCC에 x와 ¬x가 있으면 불가능)
- 네트워크 플로우 문제는 Edmonds-Karp(BFS 기반)가 구현이 간단하고 안정적이다
- 이분 매칭은 네트워크 플로우로 변환하거나 헝가리안 알고리즘을 사용한다

### 자주 하는 실수

- 이분 그래프 판별에서 비연결 그래프의 모든 컴포넌트를 처리하지 않는 경우
- Kosaraju에서 역방향 그래프를 잘못 구성하는 경우 (간선 방향 반대로)
- Tarjan에서 low-link 갱신 시 이미 SCC에 포함된 정점을 고려하는 실수
- 네트워크 플로우에서 역방향 간선(residual edge)을 추가하지 않는 경우
- SCC 결과를 DAG로 축약할 때 중복 간선을 처리하지 않는 경우

### 엣지 케이스

- 정점이 1개인 그래프 (자기 자신이 하나의 SCC)
- 간선이 없는 그래프 (각 정점이 독립 SCC, 이분 그래프)
- 완전 그래프에서의 이분 그래프 판별 (정점 3개 이상이면 이분 그래프 아님)
- 자기 자신으로의 간선이 있는 경우 (이분 그래프 아님, SCC에 영향)
- 소스에서 싱크로 경로가 없는 네트워크 (최대 유량 = 0)

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| 이분 그래프 판별 (BFS) | O(V + E) | 2-색칠, 간단한 구현 | 그래프 2분할 가능 여부 |
| Kosaraju SCC | O(V + E) | DFS 2번, 직관적 | SCC 분해, 구현 용이 |
| Tarjan SCC | O(V + E) | DFS 1번, low-link 활용 | SCC 분해, 메모리 효율 |
| Edmonds-Karp | O(V × E²) | BFS 기반 최대 유량 | 네트워크 플로우, 안정적 |
| Dinic | O(V² × E) | 레벨 그래프 + 블로킹 플로우 | 대규모 네트워크 플로우 |
| 헝가리안 알고리즘 | O(V³) | 이분 매칭 전용 | 이분 그래프 최대 매칭 |
