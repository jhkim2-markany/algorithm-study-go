# 강한 연결 요소 - 해설

## 접근 방식

1. 방향 그래프를 인접 리스트로 구성하면서, 동시에 역방향 그래프도 생성한다.
2. Kosaraju 알고리즘을 적용한다: 원본 그래프에서 DFS를 수행하며 완료 순서를 기록한다.
3. 완료 순서의 역순으로 역방향 그래프에서 DFS를 수행한다.
4. 역방향 그래프에서 한 번의 DFS로 방문되는 정점들이 하나의 SCC를 구성한다.
5. DFS 시작 횟수가 SCC의 개수이다.

## 핵심 아이디어

Kosaraju 알고리즘의 핵심은 "DFS 완료 순서"와 "역방향 그래프"의 조합이다.

원본 그래프에서 DFS를 수행하면, SCC 간의 위상 순서에서 나중에 완료되는 SCC가 스택의 위쪽에 위치한다. 역방향 그래프에서는 SCC 간의 간선 방향이 뒤집히므로, 스택 순서대로 DFS를 수행하면 각 SCC 내부만 탐색하게 된다.

예제 1에서 정점 {1,2,3}은 1→2→3→1 사이클로 하나의 SCC를 형성하고, 정점 {4,5,6}은 4→5→6→4 사이클로 또 하나의 SCC를 형성한다. 2→4와 5→1 간선은 SCC 간 간선이다. 예제 2에서는 모든 정점이 하나의 사이클(1→2→3→4→1)에 포함되므로 SCC가 1개이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

DFS를 두 번 수행하므로 O(2(V + E)) = O(V + E)이다. 원본 인접 리스트와 역방향 인접 리스트에 각각 O(V + E), 방문 배열과 스택에 O(V)의 공간이 필요하다.

## 대안적 접근

- Tarjan 알고리즘을 사용하면 DFS 한 번으로 모든 SCC를 구할 수 있다. 각 정점의 방문 순서와 low-link 값을 관리하며, low-link가 자신의 방문 순서와 같은 정점이 SCC의 루트가 된다.
- Tarjan 알고리즘은 역방향 그래프를 생성할 필요가 없어 메모리를 절약할 수 있지만, 구현이 다소 복잡하다.
