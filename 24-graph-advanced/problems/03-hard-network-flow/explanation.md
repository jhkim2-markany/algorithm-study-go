# 네트워크 플로우 - 해설

## 접근 방식

1. 방향 그래프를 잔여 용량 행렬(capacity matrix)로 표현한다. 중복 간선은 용량을 합산한다.
2. Edmonds-Karp 알고리즘을 적용한다: BFS로 소스에서 싱크까지의 증가 경로를 찾는다.
3. 증가 경로가 존재하면, 경로상 간선들의 최소 잔여 용량(bottleneck)을 구한다.
4. 경로상 각 간선의 잔여 용량을 bottleneck만큼 감소시키고, 역방향 간선의 잔여 용량을 bottleneck만큼 증가시킨다.
5. 증가 경로가 더 이상 존재하지 않을 때까지 반복하며, 누적된 유량이 최대 유량이다.

## 핵심 아이디어

Edmonds-Karp 알고리즘은 Ford-Fulkerson 방법에서 증가 경로를 BFS로 찾는 구체적인 구현이다. BFS를 사용하면 항상 최단 증가 경로(간선 수 기준)를 찾으므로, 최대 O(VE)번의 증가로 최대 유량에 도달함이 보장된다.

역방향 간선은 "유량 취소"를 가능하게 하는 핵심 장치이다. 이전에 보낸 유량을 되돌려 더 나은 경로로 재배분할 수 있게 해준다. 이 메커니즘이 없으면 탐욕적 선택에 의해 최적해를 놓칠 수 있다.

예제 1에서 소스 1에서 싱크 6으로의 최대 유량은 5이다. 예를 들어 경로 1→2→4→6으로 2, 경로 1→3→5→6으로 2, 경로 1→2→3→4→5→6 등의 조합으로 1을 추가하여 총 5의 유량을 보낼 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(V × E²) |
| 공간 복잡도 | O(V²) |

BFS로 증가 경로를 찾으므로 최대 O(VE)번의 증가가 발생하고, 각 BFS에 O(E)가 걸려 전체 O(VE²)이다. 용량 행렬에 O(V²), 인접 리스트에 O(V + E)의 공간이 필요하다. N ≤ 500이므로 용량 행렬 방식이 적합하다.

## 대안적 접근

- Dinic 알고리즘은 레벨 그래프와 블로킹 플로우를 사용하여 O(V²E) 시간에 최대 유량을 구한다. 일반적으로 Edmonds-Karp보다 빠르다.
- 간선 수가 많고 정점 수가 적은 경우 용량 행렬 대신 간선 리스트 기반 구현이 메모리 효율적이다.
- 최소 비용 최대 유량(MCMF) 문제로 확장할 경우 SPFA 기반 알고리즘을 사용한다.
