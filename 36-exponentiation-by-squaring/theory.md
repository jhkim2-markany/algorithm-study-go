# 분할 정복 거듭제곱

## 개념

분할 정복 거듭제곱(Exponentiation by Squaring)은 밑(base) a를 지수 n만큼 거듭제곱할 때, 지수를 반씩 나누어 O(log n)에 계산하는 기법이다. 단순 반복으로 a^n을 구하면 O(n)이 걸리지만, 분할 정복을 적용하면 O(log n)으로 줄일 수 있다.

핵심 아이디어는 다음 성질을 이용하는 것이다:
- n이 짝수일 때: a^n = (a^(n/2))^2
- n이 홀수일 때: a^n = a × a^(n-1)

이 기법은 정수 거듭제곱뿐 아니라 행렬 거듭제곱에도 적용할 수 있다. 행렬 거듭제곱을 활용하면 피보나치 수열, 선형 점화식 등을 O(k³ log n)에 계산할 수 있다(k는 행렬 크기). 또한 모듈러 연산과 결합하여 큰 수의 거듭제곱을 오버플로 없이 처리할 수 있다.

## 동작 원리

### 1. 빠른 거듭제곱 (Fast Power)

a^n mod m을 구하는 과정:

1. 결과값 result를 1로 초기화한다
2. 지수 n이 0보다 큰 동안 반복한다
3. n이 홀수이면 result에 a를 곱하고 mod m을 취한다
4. a를 제곱하고 mod m을 취한다 (a = a × a mod m)
5. n을 2로 나눈다 (n = n / 2)
6. 반복이 끝나면 result가 최종 답이다

예시: 3^13 mod 1000 계산

```
n=13(홀수): result = 1 × 3 = 3,     a = 3² = 9,     n = 6
n=6(짝수):                            a = 9² = 81,    n = 3
n=3(홀수):  result = 3 × 81 = 243,   a = 81² = 6561, n = 1
n=1(홀수):  result = 243 × 6561 = 1594323, n = 0
3^13 = 1594323
```

### 2. 모듈러 거듭제곱 (Modular Exponentiation)

큰 수의 거듭제곱에서 오버플로를 방지하기 위해 매 곱셈마다 모듈러 연산을 적용한다:

1. 곱셈 결과가 매우 커질 수 있으므로, 각 단계에서 mod m을 취한다
2. (a × b) mod m = ((a mod m) × (b mod m)) mod m 성질을 이용한다
3. 이를 통해 중간 결과가 항상 m 미만으로 유지된다

### 3. 행렬 거듭제곱 (Matrix Exponentiation)

행렬에도 동일한 분할 정복 원리를 적용할 수 있다:

1. 행렬 곱셈을 정의한다: C[i][j] = Σ(A[i][k] × B[k][j])
2. 단위 행렬 I를 result로 초기화한다
3. 정수 거듭제곱과 동일한 방식으로 행렬을 제곱해 나간다
4. n이 홀수이면 result에 행렬 A를 곱한다
5. A를 제곱하고 n을 반으로 나눈다

행렬 거듭제곱의 대표적 활용: 피보나치 수열

```
| F(n+1) |   | 1 1 |^n   | F(1) |
| F(n)   | = | 1 0 |   × | F(0) |
```

행렬 {{1,1},{1,0}}을 n번 거듭제곱하면 F(n)을 O(log n)에 구할 수 있다.

### 4. 선형 점화식의 행렬 거듭제곱 변환

k차 선형 점화식 f(n) = c₁f(n-1) + c₂f(n-2) + ... + cₖf(n-k)는 k×k 행렬의 거듭제곱으로 변환할 수 있다:

1. 상태 벡터 [f(n), f(n-1), ..., f(n-k+1)]을 정의한다
2. 전이 행렬을 구성한다
3. 행렬 거듭제곱으로 n번째 항을 O(k³ log n)에 계산한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 정수 거듭제곱 | O(log n) |
| 모듈러 거듭제곱 | O(log n) |
| 행렬 거듭제곱 (k×k 행렬) | O(k³ log n) |
| 공간 복잡도 (정수) | O(1) |
| 공간 복잡도 (행렬) | O(k²) |

정수 거듭제곱은 지수를 반씩 줄이므로 O(log n)이다. 행렬 거듭제곱은 행렬 곱셈 한 번에 O(k³)이 걸리므로 전체 O(k³ log n)이다.

## 적합한 문제 유형

- 큰 수의 거듭제곱을 모듈러 연산과 함께 구하는 문제
- 피보나치 수열의 n번째 항을 빠르게 구하는 문제 (n이 매우 큰 경우)
- 선형 점화식의 n번째 항을 구하는 문제
- 행렬 거듭제곱을 활용한 그래프 경로 수 계산 문제
- 모듈로 곱셈 역원 계산 (페르마 소정리 활용)
- 거듭제곱 합 S = a¹ + a² + ... + aⁿ을 빠르게 구하는 문제
- 큰 지수의 조합론 계산 문제


## 단계별 추적 (Trace)

### 예시 1: 정수 거듭제곱 2^10 mod 1000

```
입력: a = 2, n = 10, mod = 1000
초기: result = 1, base = 2

반복 1: n = 10 (짝수)
  base = 2 × 2 = 4 (mod 1000)
  n = 10 / 2 = 5
  상태: result=1, base=4, n=5

반복 2: n = 5 (홀수)
  result = 1 × 4 = 4 (mod 1000)
  base = 4 × 4 = 16 (mod 1000)
  n = 5 / 2 = 2
  상태: result=4, base=16, n=2

반복 3: n = 2 (짝수)
  base = 16 × 16 = 256 (mod 1000)
  n = 2 / 2 = 1
  상태: result=4, base=256, n=1

반복 4: n = 1 (홀수)
  result = 4 × 256 = 1024 mod 1000 = 24
  base = 256 × 256 = 65536 (mod 1000) = 536
  n = 1 / 2 = 0
  상태: result=24, base=536, n=0

결과: 2^10 mod 1000 = 24  (검증: 2^10 = 1024, 1024 mod 1000 = 24 ✓)
```

### 예시 2: 행렬 거듭제곱으로 피보나치 F(5) 계산

```
행렬 A = | 1 1 |    단위행렬 I = | 1 0 |
         | 1 0 |                  | 0 1 |

목표: A^5 계산 (F(5) = 5)

초기: result = I, base = A, n = 5

반복 1: n = 5 (홀수)
  result = I × A = | 1 1 |
                    | 1 0 |
  base = A × A = | 1 1 | × | 1 1 | = | 2 1 |
                  | 1 0 |   | 1 0 |   | 1 1 |
  n = 2

반복 2: n = 2 (짝수)
  base = | 2 1 | × | 2 1 | = | 5 3 |
         | 1 1 |   | 1 1 |   | 3 2 |
  n = 1

반복 3: n = 1 (홀수)
  result = | 1 1 | × | 5 3 | = | 8 5 |
           | 1 0 |   | 3 2 |   | 5 3 |
  n = 0

결과: result[0][1] = 5 → F(5) = 5 ✓

행렬 결과 해석:
| F(6) F(5) |   | 8 5 |
| F(5) F(4) | = | 5 3 |
```

## 실전 팁

### 활용 노하우

- 모듈러 거듭제곱은 `a^(p-2) mod p`로 모듈러 역원을 구할 때 필수이다 (페르마 소정리)
- 행렬 거듭제곱은 선형 점화식을 O(k³ log n)에 풀 수 있게 해준다
- 코딩테스트에서 "10^9 + 7로 나눈 나머지"가 나오면 모듈러 거듭제곱을 떠올리자
- 행렬 크기 k가 작을수록(보통 2~4) 행렬 거듭제곱이 효율적이다
- 비트 연산으로 구현하면 코드가 간결해진다: `n & 1`로 홀짝 판별, `n >>= 1`로 반 나누기

### 자주 하는 실수

- 모듈러 곱셈에서 오버플로: `(a * b) % mod`에서 a, b가 int64 범위 근처이면 곱셈 결과가 넘칠 수 있다
- 지수가 0인 경우 처리 누락: `a^0 = 1`이다 (a가 0이어도)
- 행렬 곱셈 구현 시 임시 배열을 사용하지 않아 값이 덮어씌워지는 실수
- 모듈러 연산을 곱셈 후에만 적용하고 중간 단계에서 빠뜨리는 실수
- 밑(base)이 mod보다 클 때 초기에 `base %= mod`를 하지 않는 실수

### 엣지 케이스

- 지수 n = 0: 결과는 항상 1 (행렬의 경우 단위행렬)
- 밑 a = 0, 지수 n > 0: 결과는 0
- 밑 a = 0, 지수 n = 0: 수학적으로 미정의이지만 보통 1로 처리
- mod = 1: 결과는 항상 0
- 지수가 매우 큰 경우 (10^18): 반드시 분할 정복 거듭제곱 사용

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 적합한 상황 | 특징 |
| --- | --- | --- | --- |
| 단순 반복 거듭제곱 | O(n) | n이 작을 때 | 구현 간단 |
| 분할 정복 거듭제곱 | O(log n) | n이 클 때 | 비트 연산 활용 |
| 행렬 거듭제곱 | O(k³ log n) | 선형 점화식 | 피보나치, 점화식 문제 |
| 모듈러 역원 (페르마) | O(log p) | p가 소수일 때 | a^(p-2) mod p |
| 모듈러 역원 (확장 유클리드) | O(log min(a,m)) | m이 소수가 아닐 때 | gcd(a,m)=1 필요 |

**선택 기준:**
- 단순 거듭제곱 + 지수가 작으면 → 반복문
- 큰 지수의 거듭제곱 → 분할 정복 거듭제곱
- 선형 점화식의 n번째 항 → 행렬 거듭제곱
- 모듈러 역원 (p가 소수) → 페르마 소정리 + 분할 정복 거듭제곱
- 모듈러 역원 (일반) → 확장 유클리드 호제법 (41번 참조)
