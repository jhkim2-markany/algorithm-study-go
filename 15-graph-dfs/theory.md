# 그래프 탐색 DFS

## 개념

깊이 우선 탐색(Depth-First Search, DFS)은 그래프의 한 정점에서 시작하여 가능한 한 깊이 들어가며 탐색하는 알고리즘이다. 현재 정점에서 방문하지 않은 인접 정점이 있으면 그 정점으로 이동하고, 더 이상 방문할 인접 정점이 없으면 이전 정점으로 되돌아간다(백트래킹). 이 과정을 모든 정점을 방문할 때까지 반복한다.

DFS의 핵심 특성은 다음과 같다:
- **깊이 우선**: 한 경로를 끝까지 탐색한 뒤 다른 경로를 탐색한다
- **백트래킹**: 더 이상 진행할 수 없으면 이전 분기점으로 되돌아간다
- **방문 배열**: 이미 방문한 정점을 다시 방문하지 않도록 기록한다
- **재귀/스택 기반**: 재귀 호출의 콜 스택 또는 명시적 스택을 사용한다

DFS는 그래프 이론에서 가장 기본적인 탐색 알고리즘 중 하나이며, 연결 요소 탐색, 사이클 판별, 위상 정렬, 강연결 요소 분해 등 다양한 그래프 알고리즘의 기반이 된다.

## 동작 원리

### 재귀 DFS

1. 시작 정점을 방문 처리한다.
2. 현재 정점의 인접 정점 목록을 순회한다.
3. 방문하지 않은 인접 정점이 있으면 해당 정점으로 재귀 호출한다.
4. 모든 인접 정점을 확인했으면 함수가 반환되며 이전 정점으로 돌아간다.
5. 그래프가 비연결 그래프인 경우, 모든 정점에 대해 방문 여부를 확인하고 미방문 정점에서 DFS를 다시 시작한다.

### 스택 DFS

1. 시작 정점을 스택에 넣는다.
2. 스택에서 정점을 꺼낸다.
3. 해당 정점이 미방문이면 방문 처리하고, 인접한 미방문 정점을 모두 스택에 넣는다.
4. 스택이 빌 때까지 2~3을 반복한다.

### DFS 트리와 간선 분류

DFS를 수행하면 탐색 과정에서 트리 구조가 형성된다. 이때 그래프의 간선은 다음과 같이 분류된다:

- **트리 간선(Tree Edge)**: DFS 트리를 구성하는 간선. 새로운 정점을 발견할 때 사용된 간선이다.
- **역방향 간선(Back Edge)**: 현재 정점에서 DFS 트리의 조상으로 향하는 간선. 사이클의 존재를 나타낸다.
- **순방향 간선(Forward Edge)**: 조상에서 자손으로 향하지만 트리 간선이 아닌 간선 (방향 그래프에서만 존재).
- **교차 간선(Cross Edge)**: 조상-자손 관계가 아닌 정점 사이의 간선 (방향 그래프에서만 존재).

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(V + E) |
| 시간 복잡도 (평균) | O(V + E) |
| 시간 복잡도 (최악) | O(V + E) |
| 공간 복잡도 | O(V) |

V는 정점의 수, E는 간선의 수이다. 모든 정점과 간선을 한 번씩 확인하므로 시간 복잡도는 O(V + E)이다. 방문 배열에 O(V), 재귀 호출 스택(또는 명시적 스택)에 최악의 경우 O(V) 공간이 필요하다.

## 적합한 문제 유형

- 그래프의 연결 요소(Connected Component) 개수 구하기
- 그래프에서 사이클 존재 여부 판별
- 위상 정렬 (방향 비순환 그래프)
- 단절점(Articulation Point)과 단절선(Bridge) 찾기
- 강연결 요소(SCC) 분해
- 경로 존재 여부 확인
- 이분 그래프 판별
- 트리에서의 DFS 기반 문제 (서브트리 크기, 오일러 투어 등)


## 단계별 추적 (Trace)

### 예시 입력

```
정점 수: 6, 간선: (1,2), (1,3), (2,4), (2,5), (3,6)
시작 정점: 1

인접 리스트:
1: [2, 3]
2: [1, 4, 5]
3: [1, 6]
4: [2]
5: [2]
6: [3]
```

### 실행 과정

```
호출 스택        방문 순서     visited
─────────────────────────────────────────
dfs(1)           [1]          {1}
├─ dfs(2)        [1,2]        {1,2}
│  ├─ dfs(4)     [1,2,4]      {1,2,4}
│  │  └─ 인접: 2 → 이미 방문 → 복귀
│  └─ dfs(5)     [1,2,4,5]    {1,2,4,5}
│     └─ 인접: 2 → 이미 방문 → 복귀
└─ dfs(3)        [1,2,4,5,3]  {1,2,3,4,5}
   └─ dfs(6)     [1,2,4,5,3,6] {1,2,3,4,5,6}
      └─ 인접: 3 → 이미 방문 → 복귀
```

**그래프와 DFS 트리**

```
원래 그래프:          DFS 트리:
  1 ─ 2                1
  |   |\               ├─ 2
  3   4  5             │  ├─ 4
  |                    │  └─ 5
  6                    └─ 3
                          └─ 6
```

**방문 순서: 1 → 2 → 4 → 5 → 3 → 6**

## 실전 팁

### 활용 노하우

- 연결 요소 개수를 구할 때는 모든 정점을 순회하며 미방문 정점에서 DFS를 시작하고, DFS 호출 횟수를 세면 된다
- 사이클 판별은 방문 상태를 3단계(미방문/진행중/완료)로 관리하면 방향 그래프에서도 정확히 판별할 수 있다
- 2차원 격자 문제에서 영역 개수를 세거나 영역 크기를 구할 때 DFS가 직관적이다. 4방향 또는 8방향 이동을 dx, dy 배열로 처리한다
- 재귀 깊이가 깊어질 수 있는 경우(정점 수 10만 이상) 스택 기반 DFS로 전환하거나, Go에서는 goroutine 스택이 자동 확장되므로 대부분 문제없다

### 자주 하는 실수

- 방문 체크를 재귀 호출 전에 하지 않고 함수 진입 후에 하면, 같은 정점이 스택에 여러 번 쌓여 시간 초과가 발생할 수 있다
- 비연결 그래프에서 시작 정점 하나에서만 DFS를 수행하면 모든 정점을 방문하지 못한다. 모든 정점에 대해 미방문 여부를 확인해야 한다
- 양방향 간선을 단방향으로만 추가하여 탐색이 불완전해지는 실수가 빈번하다

### 엣지 케이스 주의사항

- 정점이 1개인 그래프: 간선이 없으므로 시작 정점만 방문하고 종료
- 자기 자신으로의 간선(self-loop): 사이클 판별 시 별도 처리 필요
- 완전 그래프: 모든 정점이 연결되어 있어 인접 리스트가 매우 클 수 있다
- 정점 번호가 0부터 시작하는 경우와 1부터 시작하는 경우: 배열 크기와 인덱스 주의

## 관련 알고리즘 비교

| 알고리즘 | 탐색 방식 | 자료구조 | 최단 거리 보장 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| DFS | 깊이 우선 | 스택/재귀 | ✗ | 연결 요소, 사이클 판별, 위상 정렬 |
| BFS | 너비 우선 | 큐 | ✓ (가중치 없는 그래프) | 최단 거리, 레벨별 탐색 |
| 백트래킹 | 깊이 우선 + 가지치기 | 스택/재귀 | ✗ | 조합 탐색, 제약 조건 만족 문제 |
| 다익스트라 | 최소 비용 우선 | 우선순위 큐 | ✓ (양의 가중치) | 가중치 그래프 최단 경로 |
| 위상 정렬 | DFS 후위 순서 역순 | 스택/큐 | ✗ | DAG에서 작업 순서 결정 |
