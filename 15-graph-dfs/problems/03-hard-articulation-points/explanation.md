# 단절점 찾기 - 해설

## 접근 방식

1. 그래프를 인접 리스트로 표현한다.
2. DFS를 수행하며 각 정점에 방문 순서(discovered)를 부여한다.
3. 각 정점에서 서브트리를 통해 도달 가능한 가장 빠른 방문 순서(low value)를 계산한다.
4. 루트가 아닌 정점 v에 대해, v의 자식 u의 low 값이 v의 방문 순서 이상이면 v는 단절점이다.
5. 루트 정점은 DFS 트리에서 자식이 2개 이상이면 단절점이다.
6. 단절점을 오름차순으로 정렬하여 출력한다.

## 핵심 아이디어

이 문제는 Tarjan의 단절점 알고리즘을 사용한다. 핵심은 DFS 트리에서 각 정점의 "low value"를 계산하는 것이다.

low value란 해당 정점의 서브트리에서 역방향 간선(back edge)을 통해 도달할 수 있는 가장 빠른 방문 순서이다. 정점 v의 자식 u에 대해 low(u) >= discovered(v)이면, u의 서브트리에서 v를 거치지 않고는 v의 조상으로 갈 수 없다는 뜻이다. 따라서 v를 제거하면 u의 서브트리가 분리되므로 v는 단절점이다.

루트 정점은 특별히 처리한다. 루트는 부모가 없으므로 low value 조건을 적용할 수 없다. 대신 DFS 트리에서 자식이 2개 이상이면 루트를 제거했을 때 각 자식의 서브트리가 분리되므로 단절점이 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

DFS를 한 번 수행하므로 O(N + M)이다. 인접 리스트에 O(N + M), 방문 순서 배열과 단절점 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- 단순한 방법으로는 각 정점을 하나씩 제거한 뒤 연결 요소 개수를 확인하는 O(V × (V + E)) 방법이 있으나, 정점 수가 많으면 시간 초과가 발생한다.
- 단절선(Bridge)을 찾는 문제도 유사한 Tarjan 알고리즘으로 해결할 수 있다. 단절선의 조건은 low(u) > discovered(v)로, 등호가 포함되지 않는 점이 다르다.
