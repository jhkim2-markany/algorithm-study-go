# 연결 요소의 개수 - 해설

## 접근 방식

1. 그래프를 인접 리스트로 표현한다.
2. 방문 배열을 초기화한다.
3. 1번 정점부터 N번 정점까지 순회하며, 미방문 정점을 발견하면 해당 정점에서 DFS를 시작한다.
4. DFS는 현재 정점에서 도달 가능한 모든 정점을 방문 처리한다.
5. DFS를 시작한 횟수가 곧 연결 요소의 개수이다.

## 핵심 아이디어

DFS는 시작 정점에서 연결된 모든 정점을 빠짐없이 방문한다. 따라서 한 번의 DFS가 끝나면 하나의 연결 요소에 속한 모든 정점이 방문 처리된다. 아직 방문하지 않은 정점이 남아 있다면 그 정점은 다른 연결 요소에 속하므로, 새로운 DFS를 시작할 때마다 연결 요소 카운트를 1 증가시키면 된다.

간선이 0개인 경우 모든 정점이 독립적인 연결 요소이므로 답은 N이 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

모든 정점과 간선을 한 번씩 확인하므로 O(N + M)이다. 인접 리스트 저장에 O(N + M), 방문 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- BFS를 사용해도 동일하게 연결 요소를 탐색할 수 있다.
- 유니온 파인드(Union-Find) 자료구조를 사용하면 간선을 하나씩 처리하며 연결 요소를 관리할 수 있다.
