# 사이클 판별 - 해설

## 접근 방식

1. 방향 그래프를 인접 리스트로 표현한다.
2. 각 정점에 3가지 상태(색)를 부여한다: WHITE(미방문), GRAY(탐색 중), BLACK(탐색 완료).
3. 모든 정점에 대해 WHITE 상태인 정점에서 DFS를 시작한다.
4. DFS 진입 시 정점을 GRAY로 표시하고, DFS가 완료되면 BLACK으로 표시한다.
5. 탐색 중 GRAY 상태의 정점을 만나면 역방향 간선이 존재하는 것이므로 사이클이 있다.

## 핵심 아이디어

방향 그래프에서 사이클 판별의 핵심은 "3색 DFS(Three-Color DFS)"이다. 단순히 방문 여부만 기록하면 방향 그래프에서는 사이클이 아닌 경우도 사이클로 오판할 수 있다.

GRAY 상태는 "현재 DFS 경로에 포함된 정점"을 의미한다. DFS 탐색 중 GRAY 정점을 다시 만난다는 것은 현재 경로에서 출발한 정점으로 되돌아올 수 있다는 뜻이므로 사이클이 존재한다. BLACK 정점은 이미 탐색이 완료된 정점이므로, 이 정점을 만나는 것은 사이클을 의미하지 않는다.

자기 자신으로의 간선(self-loop)도 GRAY 상태에서 자기 자신을 만나는 것이므로 사이클로 올바르게 판별된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M) |
| 공간 복잡도 | O(N + M) |

모든 정점과 간선을 한 번씩 확인하므로 O(N + M)이다. 인접 리스트에 O(N + M), 색상 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- 위상 정렬(Kahn's Algorithm)을 사용하여 진입 차수 기반으로 사이클을 판별할 수 있다. 모든 정점을 처리할 수 없으면 사이클이 존재한다.
- 무방향 그래프의 경우 단순히 방문 배열과 부모 추적만으로 사이클을 판별할 수 있다. 방향 그래프에서는 반드시 3색 DFS 또는 위상 정렬을 사용해야 한다.
