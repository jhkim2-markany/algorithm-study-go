# 최장 공통 부분 수열 (LCS) - 해설

## 접근 방식

1. 두 문자열 A(길이 N)와 B(길이 M)에 대해 2차원 DP 테이블을 정의한다.
2. `dp[i][j]`를 A의 처음 i글자와 B의 처음 j글자 사이의 LCS 길이로 정의한다.
3. 점화식을 적용하여 테이블을 채운 뒤, `dp[N][M]`이 LCS 길이가 된다.
4. 역추적(backtracking)으로 DP 테이블을 거슬러 올라가며 실제 LCS 문자열을 복원한다.

## 핵심 아이디어

LCS 문제는 동적 프로그래밍의 대표적인 2차원 DP 문제이다. 점화식은 다음과 같다:

- `A[i] == B[j]`이면: `dp[i][j] = dp[i-1][j-1] + 1` (두 문자가 같으면 LCS에 포함)
- `A[i] != B[j]`이면: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])` (한쪽을 제외한 경우 중 큰 값)

LCS 문자열 복원은 `dp[N][M]`에서 시작하여 역추적한다. 현재 위치에서 두 문자가 같으면 해당 문자를 LCS에 추가하고 대각선으로 이동한다. 다르면 `dp[i-1][j]`와 `dp[i][j-1]` 중 큰 쪽으로 이동한다. 이 과정을 통해 LCS를 역순으로 구성한 뒤 뒤집으면 된다.

이 문제가 어려운 이유는 단순히 길이를 구하는 것에서 나아가 실제 문자열을 복원해야 하기 때문이다. 역추적 과정에서 DP 테이블의 값을 올바르게 따라가는 것이 핵심이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(N × M) |

N×M 크기의 DP 테이블을 채우므로 시간 복잡도는 O(N × M)이다. 역추적에 O(N + M) 시간이 추가로 소요된다. 공간 복잡도는 DP 테이블 크기인 O(N × M)이다. 길이만 구하는 경우 1차원 배열로 O(min(N, M))으로 최적화할 수 있지만, 문자열 복원이 필요하면 2차원 테이블이 필요하다.

## 대안적 접근

- **공간 최적화 (길이만 구하는 경우)**: 1차원 배열 두 개를 번갈아 사용하면 O(min(N, M)) 공간으로 LCS 길이를 구할 수 있다.
- **Hirschberg 알고리즘**: 분할 정복 + DP를 결합하여 O(min(N, M)) 공간으로 LCS 문자열까지 복원할 수 있다.
- **비트 병렬 알고리즘**: 비트 연산을 활용하여 상수 배 속도 향상을 얻을 수 있다.
