# 피보나치 수 - 해설

## 접근 방식

1. 피보나치 수열의 점화식 F(N) = F(N-1) + F(N-2)를 그대로 활용한다.
2. 단순 재귀로 구현하면 동일한 하위 문제를 반복 계산하여 O(2^N)이 걸린다.
3. DP 테이블을 사용하여 각 값을 한 번만 계산하면 O(N)으로 해결할 수 있다.
4. 바텀업 방식으로 dp[0], dp[1]부터 시작하여 dp[N]까지 순서대로 채운다.

## 핵심 아이디어

동적 프로그래밍의 가장 기본적인 예제이다. 피보나치 수열은 중복 부분 문제(Overlapping Subproblems)의 대표적인 사례로, F(5)를 구하기 위해 F(3)이 여러 번 계산되는 구조를 가진다.

DP 테이블에 이전 계산 결과를 저장하면 각 상태를 정확히 한 번만 계산하므로, 지수 시간에서 선형 시간으로 개선된다. 상태 정의는 `dp[i] = i번째 피보나치 수`이고, 점화식은 `dp[i] = dp[i-1] + dp[i-2]`이다.

N이 최대 45이므로 결과값이 약 11억으로 32비트 정수 범위 내에 들어간다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

DP 테이블을 0부터 N까지 한 번 순회하므로 시간 복잡도는 O(N)이다. 크기 N+1의 배열을 사용하므로 공간 복잡도도 O(N)이다. 이전 두 값만 유지하면 공간을 O(1)로 최적화할 수 있다.

## 대안적 접근

- **탑다운 메모이제이션**: 재귀 + 메모 테이블로 동일한 O(N) 시간에 해결 가능하다.
- **공간 최적화**: 변수 두 개만 사용하여 O(1) 공간으로 구현할 수 있다.
- **행렬 거듭제곱**: 2×2 행렬의 거듭제곱을 이용하면 O(log N)에 구할 수 있다.
