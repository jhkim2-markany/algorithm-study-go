# 배낭 문제 (0/1 Knapsack) - 해설

## 접근 방식

1. 각 물건에 대해 "넣는다" 또는 "넣지 않는다" 두 가지 선택이 있다.
2. 상태를 `dp[i][j] = i번째 물건까지 고려했을 때 용량 j에서의 최대 가치`로 정의한다.
3. 점화식: `dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])` (w[i] ≤ j인 경우).
4. 1차원 배열로 공간을 최적화하되, 역순으로 순회하여 같은 물건의 중복 사용을 방지한다.

## 핵심 아이디어

0/1 배낭 문제는 동적 프로그래밍의 대표적인 최적화 문제이다. 핵심은 각 물건을 순서대로 고려하면서, 현재 물건을 배낭에 넣을지 말지를 결정하는 것이다.

2차원 DP 테이블 `dp[i][j]`에서 i번째 물건을 넣지 않으면 `dp[i-1][j]`와 같고, 넣으면 `dp[i-1][j-w[i]] + v[i]`가 된다. 두 값 중 큰 것을 선택한다.

공간 최적화의 핵심은 역순 순회이다. `dp[j]`를 갱신할 때 `dp[j-w[i]]`는 아직 이전 행(i-1번째 물건까지의 결과)의 값이어야 한다. 역순으로 순회하면 아직 갱신되지 않은 이전 값을 참조할 수 있다. 정순으로 순회하면 이미 갱신된 값을 참조하게 되어 같은 물건을 여러 번 넣는 결과가 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × K) |
| 공간 복잡도 | O(K) |

N개의 물건에 대해 각각 용량 K까지 순회하므로 시간 복잡도는 O(N × K)이다. 1차원 배열 최적화로 공간 복잡도는 O(K)이다. 2차원 배열을 사용하면 O(N × K) 공간이 필요하다.

## 대안적 접근

- **2차원 DP**: `dp[i][j]` 테이블을 그대로 사용하면 구현이 직관적이지만 메모리를 더 사용한다.
- **탑다운 메모이제이션**: 재귀 + 메모로 구현할 수 있으며, 필요한 상태만 계산하는 장점이 있다.
- **분기 한정법(Branch and Bound)**: 최적해를 보장하면서 탐색 공간을 줄일 수 있다.
