# XOR 연립방정식 - 해설

## 접근 방식

1. 각 XOR 방정식을 GF(2) 위의 선형 방정식으로 모델링한다
2. M×(N+1) 행렬을 구성한다 (계수 + 결과값)
3. XOR 가우스 소거법으로 기약 행사다리꼴(RREF)을 만든다
4. 모순이 있으면 "IMPOSSIBLE", 아니면 해를 출력한다

## 핵심 아이디어

GF(2)에서는 덧셈과 뺄셈이 모두 XOR 연산이다. 따라서 행 소거 시 일반적인 나눗셈이 필요 없고, 단순히 XOR만 수행하면 된다.

일반 가우스 소거법과의 차이점:

- 곱셈 = AND, 덧셈 = XOR
- 피벗의 값이 항상 1이므로 나눗셈이 불필요하다
- 행 소거: row[i] ^= row[pivot] (해당 열이 1인 모든 행에 대해)

자유 변수가 존재하면 0으로 설정하여 아무 해 하나를 구한다.

```
소거 후 행렬 형태:
[1 0 * | b₁]    (* = 자유 변수에 의한 값)
[0 1 * | b₂]
[0 0 0 | 0 ]    ← 자유 변수 존재

모순 판별: [0 0 0 | 1] 형태의 행이 있으면 해 없음
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M × N) = O(N²M) |
| 공간 복잡도 | O(M × N) |

비트마스크를 사용하면 행 XOR 연산이 O(N/64)로 줄어 O(NM × N/64)가 된다.
