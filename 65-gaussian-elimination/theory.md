# 가우스 소거법 (Gaussian Elimination)

## 개념

가우스 소거법(Gaussian Elimination)은 연립일차방정식을 행렬의 기본 행 연산(Elementary Row Operations)을 통해 체계적으로 푸는 알고리즘이다.

N개의 미지수와 N개의 방정식으로 이루어진 연립일차방정식:

```
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ = b₁
a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ = b₂
...
aₙ₁x₁ + aₙ₂x₂ + ... + aₙₙxₙ = bₙ
```

이를 확대 행렬(Augmented Matrix) [A|b]로 표현한 뒤, 행 연산으로 상삼각 행렬(Upper Triangular Matrix)을 만들고 후진 대입으로 해를 구한다.

핵심 아이디어:

- **전진 소거(Forward Elimination)**: 피벗을 기준으로 아래 행의 해당 열을 0으로 만든다
- **후진 대입(Back Substitution)**: 마지막 행부터 역순으로 미지수를 하나씩 구한다
- **부분 피벗팅(Partial Pivoting)**: 수치 안정성을 위해 절댓값이 가장 큰 원소를 피벗으로 선택한다

대표적인 활용:

- 연립일차방정식의 해 구하기 (유일해, 무한해, 해 없음 판별)
- 행렬의 랭크(Rank) 계산
- 행렬식(Determinant) 계산
- GF(2) 위의 XOR 연립방정식 (비트 연산 최적화)
- 모듈러 연산 기반 연립방정식

## 동작 원리

### 1. 전진 소거 (Forward Elimination)

확대 행렬 [A|b]를 상삼각 형태로 변환하는 과정이다.

1. 첫 번째 열부터 순서대로 피벗 열을 선택한다
2. 해당 열에서 절댓값이 가장 큰 원소가 있는 행을 피벗 행으로 선택한다 (부분 피벗팅)
3. 피벗 행을 현재 행과 교환한다
4. 피벗 아래의 모든 행에 대해, 피벗 열의 원소가 0이 되도록 행 연산을 수행한다
5. 다음 열로 이동하여 반복한다

```
행 연산: row[i] = row[i] - (a[i][col] / a[pivotRow][col]) * row[pivotRow]
```

### 2. 후진 대입 (Back Substitution)

상삼각 행렬에서 마지막 행부터 역순으로 미지수를 구한다.

1. 마지막 행에서 xₙ을 직접 구한다: xₙ = bₙ / aₙₙ
2. 그 위 행에서 이미 구한 미지수를 대입하여 xₙ₋₁을 구한다
3. 첫 번째 행까지 반복한다

```
x[i] = (b[i] - Σ(j=i+1 ~ n-1) a[i][j] * x[j]) / a[i][i]
```

### 3. 부분 피벗팅 (Partial Pivoting)

수치 안정성을 위해 각 단계에서 피벗 열의 절댓값이 가장 큰 원소를 피벗으로 선택한다.

- 피벗이 0이거나 매우 작으면 나눗셈에서 큰 오차가 발생한다
- 절댓값이 가장 큰 원소를 선택하면 오차 전파를 최소화할 수 있다
- 실수 연산에서는 거의 필수적인 기법이다

### 4. 모듈러 가우스 소거법

소수 p에 대한 모듈러 연산에서의 가우스 소거법이다.

- 나눗셈 대신 모듈러 역원을 사용한다: a/b mod p = a × b⁻¹ mod p
- 페르마 소정리로 역원 계산: b⁻¹ ≡ b^(p-2) mod p
- 정수 연산만 사용하므로 실수 오차가 없다

### 5. XOR 가우스 소거법 (GF(2))

GF(2) (갈루아 체, 원소가 {0, 1}인 체)에서의 가우스 소거법이다.

- 덧셈과 뺄셈이 모두 XOR 연산이다
- 곱셈은 AND 연산이다
- 나눗셈이 필요 없다 (1의 역원은 1)
- 비트 연산으로 매우 효율적으로 구현할 수 있다
- 각 행을 하나의 정수(비트마스크)로 표현하면 한 번의 XOR로 행 연산이 완료된다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (일반) | O(N³) |
| 시간 복잡도 (XOR/비트마스크) | O(N² × M / W) (W=워드 크기, 보통 64) |
| 공간 복잡도 | O(N × M) |

- N: 방정식(행)의 수, M: 미지수(열)의 수
- XOR 가우스 소거법은 비트 연산 병렬화로 상수 배 빨라진다

## 적합한 문제 유형

- 연립일차방정식의 해를 구하는 문제
- 행렬의 랭크를 구하는 문제
- 행렬식(Determinant)을 구하는 문제
- XOR 조건이 주어지는 문제 (스위치 토글, 라이트 온/오프 등)
- 선형 독립인 벡터의 최대 개수를 구하는 문제
- 모듈러 연산 기반의 연립방정식 문제
- 선형 기저(Linear Basis)를 구하는 문제

## 단계별 추적 (Trace)

### 예시: 3×3 연립방정식 풀기

다음 연립방정식을 가우스 소거법으로 푼다:

```
 2x + 1y - 1z =  8
-3x - 1y + 2z = -11
-2x + 1y + 2z = -3
```

확대 행렬 [A|b]:

```
[  2   1  -1 |  8 ]
[ -3  -1   2 | -11]
[ -2   1   2 | -3 ]
```

**전진 소거 1단계: 피벗 열 = 0 (x의 계수)**

피벗 선택: 절댓값이 가장 큰 원소 = |-3| = 3 (행 1) → 행 0과 행 1 교환

```
행 교환 후:
[ -3  -1   2 | -11]
[  2   1  -1 |  8 ]
[ -2   1   2 | -3 ]
```

행 1 소거: row[1] = row[1] - (2/(-3)) × row[0] = row[1] + (2/3) × row[0]

```
row[1]: [2+(-3)×(2/3), 1+(-1)×(2/3), -1+2×(2/3), 8+(-11)×(2/3)]
      = [0, 1/3, 1/3, 2/3]
```

행 2 소거: row[2] = row[2] - ((-2)/(-3)) × row[0] = row[2] - (2/3) × row[0]

```
row[2]: [-2-(-3)×(2/3), 1-(-1)×(2/3), 2-2×(2/3), -3-(-11)×(2/3)]
      = [0, 5/3, 2/3, 13/3]
```

1단계 결과:

```
[ -3   -1     2   | -11  ]
[  0   1/3   1/3  |  2/3 ]
[  0   5/3   2/3  | 13/3 ]
```

**전진 소거 2단계: 피벗 열 = 1 (y의 계수)**

피벗 선택: |1/3| vs |5/3| → 5/3이 더 크므로 행 1과 행 2 교환

```
행 교환 후:
[ -3   -1     2   | -11  ]
[  0   5/3   2/3  | 13/3 ]
[  0   1/3   1/3  |  2/3 ]
```

행 2 소거: row[2] = row[2] - ((1/3)/(5/3)) × row[1] = row[2] - (1/5) × row[1]

```
row[2]: [0, 0, 1/3 - (2/3)×(1/5), 2/3 - (13/3)×(1/5)]
      = [0, 0, 1/5, -1/5]
```

2단계 결과 (상삼각 행렬 완성):

```
[ -3   -1     2   | -11  ]
[  0   5/3   2/3  | 13/3 ]
[  0    0    1/5  | -1/5 ]
```

**후진 대입:**

```
행 2: (1/5)z = -1/5  →  z = -1

행 1: (5/3)y + (2/3)(-1) = 13/3
      (5/3)y = 13/3 + 2/3 = 15/3 = 5
      y = 3

행 0: -3x + (-1)(3) + 2(-1) = -11
      -3x - 3 - 2 = -11
      -3x = -6
      x = 2
```

**결과: x = 2, y = 3, z = -1**

검증:

```
2(2) + 1(3) - 1(-1) = 4 + 3 + 1 = 8   ✓
-3(2) - 1(3) + 2(-1) = -6 - 3 - 2 = -11  ✓
-2(2) + 1(3) + 2(-1) = -4 + 3 - 2 = -3   ✓
```

## 실전 팁

### 활용 노하우

- 문제에서 "연립방정식", "선형 시스템", "행렬 랭크" 등의 키워드가 나오면 가우스 소거법을 의심한다
- XOR 조건이 여러 개 주어지는 문제(스위치 토글 등)는 GF(2) 가우스 소거법으로 모델링할 수 있다
- 실수 연산에서는 반드시 부분 피벗팅을 사용하고, EPS(예: 1e-9) 이하의 값은 0으로 처리한다
- 모듈러 연산 문제에서는 페르마 소정리로 역원을 구해 나눗셈을 대체한다
- XOR 가우스 소거법은 비트마스크로 행을 표현하면 상수 배 빨라진다

### 자주 하는 실수

- 피벗이 0인 경우를 처리하지 않아 0으로 나누는 실수. 피벗이 0이면 다른 행과 교환하거나, 해가 없음/무한해를 판별해야 한다
- 부분 피벗팅을 생략하여 실수 오차가 누적되는 실수. 실수 연산에서는 항상 부분 피벗팅을 사용하자
- 후진 대입에서 인덱스 범위를 잘못 설정하는 실수. 마지막 행부터 0번 행까지 역순으로 순회해야 한다
- 모듈러 연산에서 음수 처리를 빠뜨리는 실수. (a % p + p) % p로 항상 양수를 유지하자

### 엣지 케이스

- 해가 없는 경우 (Inconsistent): 소거 후 "0 = 상수(≠0)" 형태의 행이 존재
- 무한해 (Underdetermined): 자유 변수가 존재하여 해가 무한히 많음
- 피벗이 모두 0인 열: 해당 미지수는 자유 변수가 된다
- N = 1: 단일 방정식, 직접 나눗셈으로 해결
- 계수가 모두 0인 행: 랭크 계산 시 무시해야 한다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| 가우스 소거법 | O(N³) | 범용적, 행 연산 기반 | 일반적인 연립방정식, 랭크, 행렬식 |
| 크라메르 공식 (Cramer's Rule) | O(N × N!) 나이브 | 행렬식 비율로 해 계산 | 이론적 이해용, 실전에서는 비효율적 |
| LU 분해 (LU Decomposition) | O(N³) | 하삼각×상삼각 분해 | 같은 계수 행렬로 여러 우변을 풀 때 |
| XOR 가우스 소거법 | O(N²M/W) | GF(2), 비트 연산 최적화 | XOR 조건 문제, 선형 기저 |
| 가우스-조르단 소거법 | O(N³) | 기약 행사다리꼴까지 소거 | 역행렬 계산, 완전 소거 필요 시 |

- 일반 연립방정식 → 가우스 소거법 (부분 피벗팅)
- XOR 조건 문제 → XOR 가우스 소거법
- 같은 A로 여러 b를 풀기 → LU 분해
- 역행렬 계산 → 가우스-조르단 소거법
- 소규모 이론 문제 → 크라메르 공식
