# 다리 놓기 - 해설

## 접근 방식

1. dp[i]를 i번 기둥을 남기고 도달하는 최소 비용으로 정의한다
2. 점화식: `dp[i] = min(dp[j] + (h[i]-h[j])² + (sumW[i-1] - sumW[j]))` (j < i, j번 기둥도 남김)
3. 점화식을 전개하여 일차 함수 형태로 변환한다
4. 높이 h가 정렬되지 않아 기울기가 단조가 아니므로 Li Chao Tree를 사용한다

## 핵심 아이디어

점화식을 전개하면:

```
dp[i] = h[i]² + sumW[i-1] + min((-2·h[j])·h[i] + (dp[j] + h[j]² - sumW[j]))
```

일차 함수 `f_j(x) = (-2·h[j])·x + (dp[j] + h[j]² - sumW[j])`를 정의하고, x = h[i]에서의 최솟값을 구한다.

기울기 `-2·h[j]`는 h가 정렬되지 않으므로 단조가 아니다. 따라서 단조 CHT를 사용할 수 없고, Li Chao Tree를 사용하여 임의 순서의 기울기와 쿼리를 처리한다.

Li Chao Tree는 세그먼트 트리 구조로, 각 노드에 해당 구간에서 우세한 직선을 저장한다. 삽입과 쿼리 모두 O(log X)이며, X는 좌표 범위이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log X) |
| 공간 복잡도 | O(N log X) |

X는 h의 최댓값 (≤ 10⁶)이다. 나이브 O(N²) DP 대비 크게 개선된다.
