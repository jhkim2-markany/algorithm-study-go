# 직선 컨테이너 - 해설

## 접근 방식

1. 직선 추가와 쿼리가 임의 순서로 섞여 있으므로, 기울기 정렬 기반 CHT를 사용할 수 없다
2. Li Chao Tree를 사용하여 삽입과 쿼리를 모두 O(log X)에 처리한다
3. 동적 노드 할당으로 메모리를 절약한다

## 핵심 아이디어

Li Chao Tree는 세그먼트 트리 구조에서 각 노드가 해당 x 구간의 "우세 직선"을 저장하는 자료구조이다.

**삽입 알고리즘:**

1. 현재 노드의 구간 [lo, hi)에서 중간점 mid를 구한다
2. 새 직선과 기존 직선을 mid에서 비교한다
3. mid에서 더 작은 값을 주는 직선을 현재 노드에 저장한다
4. 밀려난 직선은 한쪽 자식에서만 기존 직선보다 좋을 수 있으므로, 해당 자식으로 재귀한다
5. 왼쪽 끝에서 밀려난 직선이 더 좋으면 왼쪽 자식으로, 아니면 오른쪽 자식으로 내려보낸다

**쿼리 알고리즘:**

1. 루트에서 x가 속하는 리프까지 내려간다
2. 경로상 각 노드의 직선에 x를 대입한다
3. 모든 값 중 최솟값을 반환한다

이 방식의 핵심은 각 노드에 하나의 직선만 저장하면서도, 쿼리 경로의 O(log X)개 노드만 확인하면 전체 최솟값을 구할 수 있다는 것이다.

**왜 정확한가?**

삽입 시 밀려난 직선은 구간의 절반에서만 우세할 수 있다. 이를 재귀적으로 처리하면, 모든 직선의 정보가 트리에 보존된다. 쿼리 시 루트에서 리프까지의 경로에 있는 모든 직선을 확인하므로, 해당 x에서 최적인 직선을 반드시 만나게 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 삽입 시간 | O(log X) |
| 쿼리 시간 | O(log X) |
| 전체 시간 | O(Q log X) |
| 공간 복잡도 | O(Q log X) |

X는 x 좌표의 범위 (≤ 2 × 10⁶)이다. 동적 노드 할당을 사용하면 실제 방문한 노드만 생성되므로 메모리가 효율적이다.
