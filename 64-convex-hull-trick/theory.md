# 볼록 껍질 트릭 (Convex Hull Trick)

## 개념

볼록 껍질 트릭(Convex Hull Trick, CHT)은 DP 최적화 기법 중 하나로, 일차 함수(선형 함수) 집합에서 특정 x 좌표에서의 최솟값 또는 최댓값을 효율적으로 구하는 자료구조이다.

DP 점화식이 다음과 같은 형태일 때 적용할 수 있다:

```
dp[i] = min(dp[j] + b[j] * a[i] + c[i])  (j < i)
```

이를 정리하면 `dp[i] = min(b[j] * a[i] + dp[j]) + c[i]` 형태가 되고, `f_j(x) = b[j] * x + dp[j]`라는 일차 함수들의 x = a[i]에서의 최솟값을 구하는 문제로 변환된다.

핵심 아이디어는 이 일차 함수들을 **하볼록 껍질(lower convex hull)** 형태로 관리하여, 불필요한 함수를 제거하고 쿼리를 빠르게 처리하는 것이다.

대표적인 CHT 유형:
- **단조 CHT**: 기울기와 쿼리 x가 모두 단조 증가/감소할 때, 스택 + 포인터로 O(N) 처리
- **비단조 CHT**: 기울기는 정렬 가능하지만 쿼리 x가 임의 순서일 때, 이분 탐색으로 O(N log N) 처리
- **Li Chao Tree**: 기울기와 쿼리 모두 임의 순서일 때, 세그먼트 트리 기반으로 O(N log N) 처리

## 동작 원리

### 1. 기본 원리 - 일차 함수의 하볼록 껍질

여러 일차 함수 f₁(x) = m₁x + b₁, f₂(x) = m₂x + b₂, ... 가 있을 때, 각 x에서의 최솟값 min(fᵢ(x))를 구하면 그 결과는 **하볼록 껍질** 형태가 된다.

```
y
│  \      /
│   \  f₂/
│ f₁ \/  /  f₃
│     \/ /
│   ──*──*──  ← 하볼록 껍질 (최솟값 envelope)
│    /  \/
│   /    \
└──────────── x
```

불필요한 직선은 다른 두 직선의 교점보다 위에 있는 직선이다. 이런 직선을 제거하면 볼록 껍질만 남는다.

### 2. 직선 추가 (Insert)

새로운 직선 f(x) = mx + b를 추가할 때, 기존 볼록 껍질의 마지막 두 직선과 비교하여 불필요한 직선을 제거한다.

직선 l₁, l₂, l₃가 순서대로 있을 때, l₂가 불필요한 조건:

```
l₁과 l₃의 교점의 y값 ≤ l₂와 l₃의 교점의 y값
```

즉, l₁과 l₃의 교점에서 l₂가 이미 l₁이나 l₃보다 위에 있으면 l₂는 최솟값에 기여하지 않으므로 제거한다.

교점 계산: 직선 y = m₁x + b₁과 y = m₂x + b₂의 교점 x좌표는:

```
x = (b₂ - b₁) / (m₁ - m₂)
```

**불필요 판정 (정수 연산):**

직선 A(기울기 m₁, 절편 b₁), B(기울기 m₂, 절편 b₂), C(기울기 m₃, 절편 b₃)에서 B가 불필요한 조건:

```
(b₃ - b₁) * (m₁ - m₂) ≤ (b₂ - b₁) * (m₁ - m₃)
```

이 조건을 만족하면 B를 제거한다.

### 3. 쿼리 (Query)

특정 x에서의 최솟값을 구하는 방법:

**단조 쿼리 (x가 증가 순서):** 포인터를 유지하며, 현재 직선보다 다음 직선이 더 작은 값을 주면 포인터를 전진시킨다. 각 포인터는 최대 한 번만 전진하므로 전체 O(N).

**비단조 쿼리 (x가 임의 순서):** 볼록 껍질에서 이분 탐색으로 최적 직선을 찾는다. 인접한 두 직선의 교점 x좌표를 기준으로 이분 탐색하여 O(log N).

### 4. Li Chao Tree

기울기와 쿼리 순서 모두 임의일 때 사용하는 세그먼트 트리 기반 자료구조이다.

**구조:** x좌표 범위 [lo, hi]에 대한 세그먼트 트리. 각 노드는 해당 구간의 "우세 직선"을 저장한다.

**삽입 (Insert):**

1. 현재 노드의 구간 [lo, hi]에서 중간점 mid = (lo + hi) / 2를 구한다
2. 새 직선과 기존 직선을 mid에서 비교한다
3. mid에서 더 작은 값을 주는 직선을 현재 노드에 저장한다
4. 나머지 직선은 왼쪽 또는 오른쪽 자식으로 재귀적으로 내려보낸다

**쿼리 (Query):**

1. 루트에서 리프까지 내려가며 각 노드의 직선에 x를 대입한다
2. 모든 노드의 값 중 최솟값을 반환한다

```
Li Chao Tree 삽입 예시:

구간 [0, 8], 직선 f(x) = 2x + 1 삽입

        [0,8] f=2x+1
       /          \
    [0,4]        [4,8]
    /   \        /   \
 [0,2] [2,4] [4,6] [6,8]

새 직선 g(x) = -x + 10 삽입:
mid=4에서 f(4)=9, g(4)=6 → g가 더 작으므로 g를 노드에 저장
f는 오른쪽 구간 [4,8]에서 g보다 작을 수 있으므로 오른쪽으로 내려보냄
```

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 단조 CHT (삽입 + 쿼리) | O(N) |
| 비단조 CHT (삽입 O(N) + 쿼리 O(log N)) | O(N log N) |
| Li Chao Tree (삽입 O(log X) + 쿼리 O(log X)) | O(N log X) |
| 공간 복잡도 (스택 기반) | O(N) |
| 공간 복잡도 (Li Chao Tree) | O(X) 또는 동적 할당 시 O(N log X) |

여기서 X는 쿼리 x좌표의 범위이다.

## 적합한 문제 유형

- DP 점화식이 `dp[i] = min/max(a[i] * b[j] + c[j] + d[i])` 형태인 문제 (i와 j의 항이 곱으로 엮임)
- O(N²) DP를 O(N log N) 또는 O(N)으로 최적화해야 하는 문제
- 수열 분할 비용 최소화 문제 (구간 비용이 일차 함수로 표현 가능)
- 기울기가 단조인 경우 스택 기반 O(N) CHT 적용 가능
- 쿼리 순서가 임의인 경우 Li Chao Tree 또는 이분 탐색 CHT 적용
- 기하학적으로 직선 집합의 하한/상한 envelope를 구하는 문제

## 단계별 추적 (Trace)

### 예시: 최소 비용 수열 분할

수열 a = [3, 1, 4, 1, 5]가 있고, 분할 비용이 다음과 같다:

```
dp[0] = 0 (초기값)
dp[i] = min(dp[j] + (sum[i] - sum[j])²)  (0 ≤ j < i)
```

여기서 sum[i]는 a[1..i]의 누적합이다.

전개하면:

```
dp[i] = min(dp[j] + sum[i]² - 2·sum[i]·sum[j] + sum[j]²)
      = sum[i]² + min((-2·sum[j])·sum[i] + dp[j] + sum[j]²)
```

일차 함수: `f_j(x) = (-2·sum[j])·x + (dp[j] + sum[j]²)`, 쿼리 x = sum[i]

**누적합 계산:**

```
sum[0] = 0
sum[1] = 3
sum[2] = 4
sum[3] = 8
sum[4] = 9
sum[5] = 14
```

**단계별 진행:**

```
초기: dp[0] = 0, 직선 f₀(x) = 0·x + 0 추가
      볼록 껍질: {f₀: y = 0}

i=1: x = sum[1] = 3
     f₀(3) = 0 → dp[1] = 9 + 0 = 9
     직선 f₁(x) = (-6)·x + (9 + 9) = -6x + 18 추가
     볼록 껍질: {f₀: y = 0, f₁: y = -6x + 18}

i=2: x = sum[2] = 4
     f₀(4) = 0, f₁(4) = -6·4 + 18 = -6
     min = -6 → dp[2] = 16 + (-6) = 10
     직선 f₂(x) = (-8)·x + (10 + 16) = -8x + 26 추가
     f₁ 불필요 판정: f₀, f₁, f₂에서
       (26 - 0)·(0 - (-6)) ≤ (18 - 0)·(0 - (-8))
       26·6 ≤ 18·8 → 156 ≤ 144? 아니오 → f₁ 유지
     볼록 껍질: {f₀, f₁, f₂}

i=3: x = sum[3] = 8
     f₀(8) = 0, f₁(8) = -30, f₂(8) = -38
     min = -38 → dp[3] = 64 + (-38) = 26
     직선 f₃(x) = (-16)·x + (26 + 64) = -16x + 90 추가
     볼록 껍질 정리 후: {f₀, f₂, f₃} (f₁ 제거됨)

i=4: x = sum[4] = 9
     f₀(9) = 0, f₂(9) = -46, f₃(9) = -54
     min = -54 → dp[4] = 81 + (-54) = 27
     직선 f₄(x) = (-18)·x + (27 + 81) = -18x + 108 추가
     볼록 껍질 정리 후: {f₀, f₃, f₄}

i=5: x = sum[5] = 14
     f₀(14) = 0, f₃(14) = -134, f₄(14) = -144
     min = -144 → dp[5] = 196 + (-144) = 52
```

**결과:**

```
dp = [0, 9, 10, 26, 27, 52]

검증 (dp[5]):
  j=0: (14-0)² = 196
  j=1: 9 + (14-3)² = 9 + 121 = 130
  j=2: 10 + (14-4)² = 10 + 100 = 110
  j=3: 26 + (14-8)² = 26 + 36 = 62
  j=4: 27 + (14-9)² = 27 + 25 = 52  ← 최솟값 ✓
```

**볼록 껍질 변화 시각화:**

```
y
│
│ f₀ (y=0, 수평선)
│──────────────────────
│         \  f₂
│          \/
│     f₃   /\  f₄
│      \  /  \
│       \/    \
│       /\     \
└──────────────────── x
     최솟값 envelope
```

## 실전 팁

### 활용 노하우

- DP 점화식을 전개했을 때 `min(f(j) * g(i) + h(j))` 형태가 보이면 CHT를 의심한다. i에 의존하는 항과 j에 의존하는 항이 곱으로 엮여야 한다
- 기울기가 단조 감소/증가하고 쿼리 x도 단조이면 스택 + 포인터로 O(N) 구현이 가능하다. 이 경우가 가장 간단하다
- 최솟값이 아닌 최댓값을 구해야 하면 부호를 뒤집어 최솟값 문제로 변환하거나, 상볼록 껍질을 관리한다
- Li Chao Tree는 구현이 직관적이고 기울기/쿼리 순서에 제약이 없어 실전에서 범용적으로 사용하기 좋다
- 좌표 범위가 크면 Li Chao Tree에 좌표 압축을 적용하거나 동적 세그먼트 트리를 사용한다

### 자주 하는 실수

- DP 점화식을 일차 함수 형태로 변환할 때 부호를 잘못 정리하는 실수. 전개 과정을 꼼꼼히 확인하자
- 최솟값과 최댓값을 혼동하여 볼록 껍질 방향을 반대로 관리하는 실수. 하볼록 껍질(최솟값)인지 상볼록 껍질(최댓값)인지 명확히 구분하자
- 불필요 직선 판정에서 정수 오버플로우. 기울기와 절편의 곱이 int64 범위를 초과할 수 있으므로 주의한다
- 단조 CHT에서 기울기 정렬 조건을 확인하지 않고 사용하는 실수. 기울기가 단조가 아니면 이분 탐색 또는 Li Chao Tree를 사용해야 한다

### 엣지 케이스

- N = 1: 분할할 원소가 하나뿐이므로 직선이 하나만 추가된다
- 모든 기울기가 동일한 경우: 절편이 가장 작은 직선만 남는다
- 쿼리 x가 매우 크거나 작은 경우: 기울기가 가장 작은(또는 큰) 직선이 최적이 된다
- 기울기와 절편이 모두 0인 직선: 정상적으로 처리되지만 불필요 판정에서 0 나눗셈에 주의
- int64 오버플로우: 기울기 × x + 절편 계산 시 값이 매우 커질 수 있다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 적용 조건 | 적합한 상황 |
| --- | --- | --- | --- |
| Convex Hull Trick (단조) | O(N) | 기울기 단조, 쿼리 단조 | 가장 빠른 CHT, 조건 만족 시 최우선 |
| Convex Hull Trick (이분 탐색) | O(N log N) | 기울기 단조, 쿼리 임의 | 기울기 정렬 가능하지만 쿼리 순서 임의 |
| Li Chao Tree | O(N log X) | 제약 없음 | 기울기/쿼리 모두 임의, 범용적 |
| 분할 정복 최적화 (D&C Opt.) | O(N log N) | opt[i] ≤ opt[i+1] 단조성 | CHT 형태가 아닌 DP 최적화 |
| Knuth 최적화 | O(N²) → O(N²) 상수 개선 | 사각 부등식 성립 | 구간 DP에서 최적 분할점 단조성 |

- 점화식이 일차 함수 형태 → CHT 또는 Li Chao Tree
- 최적 분할점이 단조 → 분할 정복 최적화
- 구간 DP + 사각 부등식 → Knuth 최적화
- 조건이 복잡하면 Li Chao Tree가 가장 안전한 선택
