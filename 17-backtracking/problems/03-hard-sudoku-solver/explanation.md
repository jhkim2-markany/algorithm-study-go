# 스도쿠 풀기 - 해설

## 접근 방식

1. 입력을 읽으면서 빈 칸(0)의 좌표를 리스트에 저장하고, 이미 채워진 숫자의 제약 조건을 등록한다.
2. 빈 칸 리스트의 첫 번째 칸부터 순서대로 채워나간다.
3. 각 빈 칸에 1~9를 시도하며, 행/열/3×3 박스 제약 조건을 만족하는 숫자만 배치한다.
4. 숫자를 배치한 후 다음 빈 칸으로 재귀 호출한다.
5. 모든 숫자가 실패하면 현재 칸을 비우고 이전 칸으로 되돌아간다.
6. 모든 빈 칸을 채우면 성공이다.

## 핵심 아이디어

스도쿠는 대표적인 제약 조건 만족 문제(CSP)이다. 핵심은 세 가지 제약 조건(행, 열, 3×3 박스)을 O(1)에 검사하는 것이다.

세 개의 2차원 불리언 배열을 사용한다:
- `row[r][num]`: r행에 num이 존재하는지
- `colB[c][num]`: c열에 num이 존재하는지
- `box[b][num]`: b번 박스에 num이 존재하는지

3×3 박스의 번호는 `(r/3)*3 + c/3`으로 계산한다. 이 공식은 9개의 박스를 0~8로 매핑한다.

빈 칸을 미리 리스트로 수집하면 매번 격자를 순회하며 빈 칸을 찾을 필요가 없어 효율적이다. 가지치기가 효과적으로 작동하여 빈 칸이 60개인 경우에도 실용적인 시간 내에 해를 찾을 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(9^K) (K: 빈 칸 수, 가지치기로 실제로는 훨씬 적음) |
| 공간 복잡도 | O(K) |

최악의 경우 각 빈 칸에 9개의 숫자를 시도하므로 O(9^K)이지만, 제약 조건에 의한 가지치기가 매우 효과적이어서 실제 탐색 노드 수는 이보다 훨씬 적다. 재귀 깊이는 빈 칸의 수 K이며, 제약 조건 배열에 O(1) 상수 공간이 필요하다.

## 대안적 접근

- 빈 칸 중 후보 숫자가 가장 적은 칸부터 채우는 MRV(Minimum Remaining Values) 휴리스틱을 적용하면 탐색 속도를 크게 향상시킬 수 있다.
- 비트마스크를 사용하면 후보 숫자 관리와 제약 조건 검사를 더 효율적으로 수행할 수 있다.
- Dancing Links(DLX) 알고리즘을 사용하면 스도쿠를 정확 피복(Exact Cover) 문제로 변환하여 매우 빠르게 풀 수 있다.
