# N과 M (순열) - 해설

## 접근 방식

1. 1부터 N까지의 숫자 중 M개를 중복 없이 선택하는 순열을 구하는 문제이다.
2. 백트래킹을 사용하여 숫자를 하나씩 선택하고, M개를 모두 선택하면 출력한다.
3. 이미 사용한 숫자는 `used` 배열로 추적하여 중복 선택을 방지한다.
4. 1부터 N까지 순서대로 시도하므로 자연스럽게 사전 순으로 출력된다.

## 핵심 아이디어

백트래킹의 가장 기본적인 형태인 순열 생성 문제이다. 핵심은 "선택 → 재귀 → 되돌리기" 패턴이다.

각 단계에서 아직 사용하지 않은 숫자를 하나 선택하고, 재귀적으로 다음 숫자를 선택한다. 재귀가 끝나면 선택을 취소하여 다른 숫자를 시도할 수 있도록 한다. `used` 배열을 통해 O(1)에 사용 여부를 확인하는 것이 효율적이다.

N이 최대 8이므로 최악의 경우 8! = 40,320개의 순열을 생성하며, 이는 충분히 빠르게 처리된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N! / (N-M)!) |
| 공간 복잡도 | O(N) |

N개에서 M개를 뽑는 순열의 수는 P(N, M) = N! / (N-M)!이다. 각 순열을 생성하는 데 O(M) 시간이 걸리므로 전체 시간 복잡도는 O(M × N! / (N-M)!)이다. 재귀 깊이와 사용 배열에 O(N) 공간이 필요하다.

## 대안적 접근

- 조합을 먼저 구한 뒤 각 조합의 순열을 생성하는 방법도 가능하지만, 직접 순열을 생성하는 것이 더 간단하다.
- `next_permutation` 방식으로 반복적으로 다음 순열을 구하는 방법도 있다.
