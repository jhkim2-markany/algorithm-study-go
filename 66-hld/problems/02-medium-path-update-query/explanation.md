# 경로 갱신과 질의 - 해설

## 접근 방식

1. HLD로 트리를 전처리한다 (서브트리 크기, Heavy Chain, DFS 번호)
2. Lazy Propagation이 적용된 세그먼트 트리를 구축한다 (초기값 0)
3. 경로 갱신: 두 노드가 같은 체인에 속할 때까지 체인을 올리며 구간 갱신을 수행한다
4. 경로 질의: 동일한 체인 분해 과정으로 구간 합을 누적한다

## 핵심 아이디어

기본 경로 합 질의에 구간 갱신(Lazy Propagation)이 추가된 형태이다. 경로 갱신과 경로 질의 모두 동일한 체인 분해 과정을 거친다.

```
경로 갱신 과정:
1. top[u] ≠ top[v]인 동안:
   - 깊은 쪽 체인의 [pos[top[u]], pos[u]] 구간에 w를 더함
   - u를 parent[top[u]]로 올림
2. 같은 체인에서 [pos[u], pos[v]] 구간에 w를 더함

Lazy Propagation:
- 구간 갱신 시 해당 노드에 lazy 값을 저장
- 질의 시 lazy 값을 자식에게 전파 (pushDown)
- seg[node] += lazy * 구간 길이
```

경로 갱신 연산 `1 1 4 3`의 추적 (트리: 1-2, 1-3, 2-4, 2-5):

```
경로 1→4: 1→2→4
체인 분해 후 구간 갱신:
  [pos[1], pos[4]] 구간에 3을 더함 (같은 체인이면 한 번에 처리)
결과: 노드 1, 2, 4의 값이 각각 3씩 증가
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 전처리 | O(N) |
| 경로 갱신 1회 | O(log²N) |
| 경로 질의 1회 | O(log²N) |
| 전체 시간 복잡도 | O(N + Q log²N) |
| 공간 복잡도 | O(N) |

Lazy Propagation으로 구간 갱신을 O(log N)에 처리하고, 경로당 O(log N)개의 체인을 순회한다.
