# 경로 최대 간선 가중치 - 해설

## 접근 방식

1. HLD로 트리를 전처리한다
2. 간선 가중치를 자식 노드에 매핑한다 (간선 (a,b)에서 깊이가 더 깊은 노드가 자식)
3. 최댓값 세그먼트 트리를 구축한다
4. 간선 갱신: 해당 간선의 자식 노드 위치에서 세그먼트 트리를 갱신한다
5. 경로 질의: 체인 분해 후 구간 최댓값을 구하되, LCA 노드는 제외한다

## 핵심 아이디어

간선 가중치 문제에서 가장 중요한 포인트는 **간선을 자식 노드에 매핑**하는 것이다. 간선 (u, v)에서 v가 자식이면, 이 간선의 가중치를 `pos[v]` 위치에 저장한다.

경로 질의 시 같은 체인 내에서 LCA 노드를 제외해야 한다. LCA 노드에 저장된 값은 LCA와 그 부모를 잇는 간선의 가중치이므로, 현재 경로와 무관하다.

```
간선 매핑:
  간선 (a, b, w) → 자식 노드 child에 w를 저장
  child = dep[a] > dep[b] ? a : b

경로 질의 시 LCA 처리:
  같은 체인 내: segQuery(pos[u]+1, pos[v])  ← pos[u]+1부터 시작
  (u가 LCA, 즉 깊이가 더 얕은 노드)
```

예제 1의 추적 (`2 4 5`, 경로: 4→2→5):

```
간선 매핑:
  간선 1 (1,2,3) → child=2, pos[2]에 3 저장
  간선 2 (1,3,5) → child=3, pos[3]에 5 저장
  간선 3 (2,4,2) → child=4, pos[4]에 2 저장
  간선 4 (2,5,7) → child=5, pos[5]에 7 저장

pathMax(4, 5):
  top[4]과 top[5]가 다를 수 있음 → 체인 분해
  LCA = 2
  경로 간선: (2,4) 가중치 2, (2,5) 가중치 7
  최댓값 = 7 ✓
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 전처리 | O(N) |
| 간선 갱신 1회 | O(log N) |
| 경로 최댓값 질의 1회 | O(log²N) |
| 전체 시간 복잡도 | O(N + Q log²N) |
| 공간 복잡도 | O(N) |

간선 갱신은 세그먼트 트리의 단일 위치 갱신이므로 O(log N)이다.
