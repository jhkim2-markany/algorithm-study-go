# 경로 합 구하기 - 해설

## 접근 방식

1. 트리를 인접 리스트로 저장한다
2. 1차 DFS로 서브트리 크기, 깊이, 부모, Heavy Child를 계산한다
3. 2차 DFS로 Heavy Child를 우선 방문하여 DFS 번호(pos)와 체인 정보(top)를 부여한다
4. DFS 번호 순서대로 노드 값을 배열에 배치하고 세그먼트 트리를 구축한다
5. 각 질의에 대해 두 노드가 같은 체인에 속할 때까지 체인을 올리며 구간 합을 누적한다

## 핵심 아이디어

HLD의 가장 기본적인 형태이다. 트리의 임의 경로를 최대 O(log N)개의 Heavy Chain 구간으로 분해하고, 각 구간의 합을 세그먼트 트리로 O(log N)에 구한다.

```
경로 질의 과정:
1. top[u] ≠ top[v]인 동안:
   - 깊은 쪽 체인의 [pos[top[u]], pos[u]] 구간을 질의
   - u를 parent[top[u]]로 올림
2. 같은 체인에서 [pos[u], pos[v]] 구간을 질의 (얕은 쪽이 왼쪽)
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 전처리 (DFS + 세그먼트 트리) | O(N) |
| 질의 1회 | O(log²N) |
| 전체 시간 복잡도 | O(N + Q log²N) |
| 공간 복잡도 | O(N) |

경로당 최대 O(log N)개의 체인을 순회하고, 각 체인에서 세그먼트 트리 질의 O(log N)을 수행한다.
