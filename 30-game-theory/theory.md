# 게임 이론

## 개념

게임 이론(Game Theory)은 두 플레이어가 번갈아 수를 두는 조합 게임(Combinatorial Game)에서 최적 전략과 승패를 판별하는 알고리즘 분야이다. 코딩테스트에서 다루는 게임 이론은 주로 다음 조건을 만족하는 공정 게임(Impartial Game)이다:

- 두 플레이어가 번갈아 행동한다
- 양쪽 플레이어가 동일한 수를 둘 수 있다 (공정 게임)
- 더 이상 수를 둘 수 없는 플레이어가 패배한다 (정상 승리 조건)
- 게임은 유한 시간 내에 반드시 종료된다
- 무작위 요소가 없다 (완전 정보 게임)

핵심 도구는 Sprague-Grundy 정리이다. 모든 공정 게임의 상태에 그런디 수(Grundy Number)를 부여할 수 있으며, 여러 독립적인 게임이 동시에 진행될 때 각 게임의 그런디 수를 XOR하여 전체 승패를 판별한다.

## 동작 원리

### 1. 승리/패배 포지션 (Win/Lose Position)

게임의 각 상태를 승리 포지션(W) 또는 패배 포지션(L)으로 분류한다:

1. 더 이상 수를 둘 수 없는 상태는 패배 포지션(L)이다
2. 패배 포지션으로 이동할 수 있는 상태는 승리 포지션(W)이다
3. 모든 이동이 승리 포지션으로만 가는 상태는 패배 포지션(L)이다

즉, 현재 플레이어가 상대를 패배 포지션으로 보낼 수 있으면 승리하고, 어떤 수를 두어도 상대가 승리 포지션에 놓이면 패배한다.

### 2. Nim 게임

N개의 돌 더미가 있고, 각 플레이어는 자기 차례에 하나의 더미에서 1개 이상의 돌을 가져간다. 마지막 돌을 가져가는 플레이어가 승리한다.

Nim 게임의 승패 판별:

1. 모든 더미의 돌 개수를 XOR한다: `x = a₁ ⊕ a₂ ⊕ ... ⊕ aₙ`
2. XOR 결과가 0이 아니면 선수(첫 번째 플레이어) 승리
3. XOR 결과가 0이면 후수(두 번째 플레이어) 승리

### 3. Sprague-Grundy 정리

모든 공정 게임의 상태에 그런디 수(Grundy Number, nimber)를 부여할 수 있다:

1. 종료 상태(더 이상 이동 불가)의 그런디 수는 0이다
2. 상태 s에서 이동 가능한 상태들의 그런디 수 집합을 S라 하면, `G(s) = mex(S)`이다
3. mex(Minimum Excludant)는 집합에 포함되지 않는 가장 작은 음이 아닌 정수이다
   - 예: mex({0, 1, 3}) = 2, mex({1, 2}) = 0, mex({}) = 0

여러 독립적인 게임이 동시에 진행될 때:

1. 각 게임의 그런디 수를 구한다: G₁, G₂, ..., Gₖ
2. 전체 게임의 그런디 수 = G₁ ⊕ G₂ ⊕ ... ⊕ Gₖ (XOR)
3. 전체 그런디 수가 0이 아니면 선수 승리, 0이면 후수 승리

### 4. 그런디 수 계산 과정

상태 공간이 작은 경우 DP로 그런디 수를 계산한다:

1. 종료 상태부터 시작하여 그런디 수 0을 부여한다
2. 각 상태에서 이동 가능한 모든 상태의 그런디 수를 수집한다
3. mex 함수를 적용하여 현재 상태의 그런디 수를 결정한다
4. 모든 상태의 그런디 수를 구할 때까지 반복한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| Nim 게임 승패 판별 | O(N) |
| 그런디 수 계산 (상태 수 S, 이동 수 M) | O(S × M) |
| mex 계산 | O(M) |
| 공간 복잡도 (그런디 수 저장) | O(S) |

Nim 게임은 XOR 연산만으로 O(N)에 판별 가능하다. 일반적인 게임의 그런디 수 계산은 상태 수와 각 상태에서의 이동 수에 비례한다.

## 적합한 문제 유형

- 두 플레이어가 번갈아 돌을 가져가는 Nim 게임 문제
- 최적 전략으로 플레이할 때 선수/후수 승패를 판별하는 문제
- 여러 독립적인 게임이 동시에 진행되는 복합 게임 문제
- 그런디 수를 계산하여 승패를 결정하는 문제
- 게임 상태를 DP로 분석하는 문제
- 돌 가져가기, 칸 이동 등 조합 게임 문제

## 단계별 추적 (Trace)

### 예시 1: Nim 게임 승패 판별

```
입력: 돌 더미 3개, 각각 [3, 4, 5]개

1단계: 각 더미의 이진 표현
  3 = 011
  4 = 100
  5 = 101

2단계: XOR 계산
  3 ⊕ 4 = 011 ⊕ 100 = 111 = 7
  7 ⊕ 5 = 111 ⊕ 101 = 010 = 2

3단계: 결과 판별
  XOR = 2 ≠ 0 → 선수 승리!

선수의 최적 전략:
  XOR 결과를 0으로 만들어야 한다.
  더미 3(=5)에서 2개를 가져가면 → [3, 4, 3]
  3 ⊕ 4 ⊕ 3 = 011 ⊕ 100 ⊕ 011 = 100 ⊕ 011 = 111... 
  → 더미 2(=4)에서 2개를 가져가면 → [3, 2, 5]
  3 ⊕ 2 ⊕ 5 = 011 ⊕ 010 ⊕ 101 = 001 ⊕ 101 = 100...
  → 더미 3(=5)에서 3개를 가져가면 → [3, 4, 2]
  3 ⊕ 4 ⊕ 2 = 011 ⊕ 100 ⊕ 010 = 101 ≠ 0
  → 더미 2(=4)에서 1개를 가져가면 → [3, 3, 5]
  3 ⊕ 3 ⊕ 5 = 000 ⊕ 101 = 5 ≠ 0
  → 더미 3(=5)에서 4개를 가져가면 → [3, 4, 1]
  3 ⊕ 4 ⊕ 1 = 011 ⊕ 100 ⊕ 001 = 110 = 6 ≠ 0
  → 실제 최적: 더미 1(=3)에서 1개 → [2, 4, 5] → 2⊕4⊕5 = 3 ≠ 0
  → 실제 최적: 더미 3(=5)에서 5개 → [3, 4, 0] → 3⊕4 = 7 ≠ 0
  → 실제 최적: 더미 2(=4)에서 3개 → [3, 1, 5] → 3⊕1⊕5 = 7 ≠ 0
  → 더미 3에서 2개 → [3, 4, 3] → 3⊕4⊕3 = 4 ≠ 0
  → 더미 1에서 2개 → [1, 4, 5] → 1⊕4⊕5 = 0 ✓ 선수 최적!
```

### 예시 2: 그런디 수 계산

```
입력: 돌 1개에서 1개 또는 3개를 가져갈 수 있는 게임, 돌 0~5개

상태별 그런디 수 계산:

G(0) = 0  (종료 상태, 패배)

G(1): 이동 가능 → {G(0)} = {0}
      mex({0}) = 1

G(2): 이동 가능 → {G(1)} = {1}
      mex({1}) = 0

G(3): 이동 가능 → {G(2), G(0)} = {0, 0} = {0}
      mex({0}) = 1

G(4): 이동 가능 → {G(3), G(1)} = {1, 1} = {1}
      mex({1}) = 0

G(5): 이동 가능 → {G(4), G(2)} = {0, 0} = {0}
      mex({0}) = 1

결과 테이블:
  돌 수:    0  1  2  3  4  5
  그런디 수: 0  1  0  1  0  1
  승패:     L  W  L  W  L  W

패턴: 홀수 → 선수 승리, 짝수 → 후수 승리
```

## 실전 팁

### 활용 노하우

- Nim 게임은 XOR만으로 O(N)에 판별 가능하므로, 문제가 Nim으로 변환 가능한지 먼저 확인하라
- 그런디 수에 패턴이 있는지 작은 값부터 계산하여 관찰하라. 많은 게임에서 주기적 패턴이 나타난다
- "선수 필승/후수 필승"을 묻는 문제는 대부분 게임 이론 문제이다
- 여러 독립 게임의 합은 각 게임의 그런디 수를 XOR하면 된다 (Sprague-Grundy 정리)

### 자주 하는 실수

- Nim 게임에서 XOR 결과가 0이면 "선수 승리"로 잘못 판단하는 실수 (0이면 후수 승리)
- mex 함수에서 집합에 없는 "가장 작은 음이 아닌 정수"를 구해야 하는데, 최솟값을 구하는 실수
- 게임 상태가 독립적이지 않은데 XOR을 적용하는 실수
- 종료 조건(더 이상 수를 둘 수 없는 상태)을 잘못 정의하는 실수

### 엣지 케이스

- 돌이 0개인 초기 상태 (선수가 바로 패배)
- 더미가 1개뿐인 Nim 게임 (선수가 전부 가져가면 승리)
- 모든 더미의 돌 수가 같은 경우 (짝수 개 더미면 후수 승리)
- 이동 가능한 수가 1가지뿐인 게임

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| Nim 게임 (XOR) | O(N), 단순 XOR 판별 | 돌 가져가기 게임, 표준 Nim |
| Sprague-Grundy | O(S × M), 범용 게임 분석 | 임의의 공정 게임 승패 판별 |
| 미니맥스 (Minimax) | O(b^d), 게임 트리 탐색 | 비공정 게임, 점수 최적화 |
| 알파-베타 가지치기 | 미니맥스 최적화 | 탐색 공간이 큰 게임 |
| DP 승패 판별 | O(상태 수), 메모이제이션 | 상태 공간이 작은 게임 |
| 게임 DP + 그런디 | O(S × M), 복합 게임 | 여러 독립 게임의 합 |
