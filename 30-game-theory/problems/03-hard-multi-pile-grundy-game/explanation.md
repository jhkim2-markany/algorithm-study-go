# 다중 파일 그런디 게임 - 해설

## 접근 방식

1. 가져갈 수 있는 돌의 개수 집합 S와 각 게임의 돌 개수를 입력받는다
2. 0부터 최대 돌 개수까지 각 상태의 그런디 수를 DP로 계산한다
3. 각 상태에서 이동 가능한 상태들의 그런디 수를 수집하고 mex를 적용한다
4. Sprague-Grundy 정리에 따라 모든 게임의 그런디 수를 XOR한다
5. XOR 결과가 0이 아니면 선수 승리, 0이면 후수 승리를 출력한다

## 핵심 아이디어

이 문제는 Sprague-Grundy 정리의 핵심 응용이다. 여러 독립적인 게임이 동시에 진행될 때, 전체 게임의 승패는 각 개별 게임의 그런디 수를 XOR하여 결정할 수 있다.

그런디 수 계산 과정:
- 돌 0개의 그런디 수는 0이다 (종료 상태)
- 돌 i개의 그런디 수는 mex({G(i-s) | s ∈ S, i ≥ s})이다
- mex(Minimum Excludant)는 집합에 포함되지 않는 가장 작은 음이 아닌 정수이다

예제 1에서 moves = {1, 3, 4}일 때:
- G(0)=0, G(1)=1, G(2)=0, G(3)=1, G(4)=2, G(5)=3, G(6)=2, G(7)=0
- 게임 (5, 3, 7)의 그런디 수: G(5) ⊕ G(3) ⊕ G(7) = 3 ⊕ 1 ⊕ 0 = 2 ≠ 0 → 선수 승리

예제 2에서 moves = {1, 2}일 때:
- G(0)=0, G(1)=1, G(2)=2, G(3)=0, G(4)=1
- 게임 (4, 4)의 그런디 수: G(4) ⊕ G(4) = 1 ⊕ 1 = 0 → 후수 승리

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(maxN × K + M) |
| 공간 복잡도 | O(maxN) |

maxN까지 그런디 수를 계산하는 데 O(maxN × K), M개 게임의 XOR을 구하는 데 O(M)이 소요된다.

## 대안적 접근

- 가져갈 수 있는 수가 {1, 2, ..., K}처럼 연속적인 경우, 그런디 수는 n mod (K+1)이라는 패턴이 존재하여 O(1)에 계산할 수 있다. 하지만 일반적인 이동 집합에서는 DP로 그런디 수를 계산해야 한다.
- 메모이제이션을 사용한 탑다운 방식으로도 구현할 수 있으나, 여러 게임에서 같은 그런디 수를 참조하므로 바텀업 방식이 더 효율적이다.
