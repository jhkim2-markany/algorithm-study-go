# 모든 부분집합 생성 - 해설

## 접근 방식

1. 재귀 함수에서 현재 선택 가능한 최소 원소(start)와 현재까지 선택한 원소 목록(current)을 관리한다.
2. 매 재귀 호출 시 현재 상태를 결과 목록에 추가한다 (빈 집합 포함).
3. start부터 N까지 각 원소를 하나씩 선택하고 재귀 호출한다.
4. 재귀 호출 후 선택을 취소(백트래킹)하여 다른 경우를 탐색한다.
5. 모든 부분집합을 생성한 후 크기순, 사전순으로 정렬하여 출력한다.

## 핵심 아이디어

부분집합 생성은 각 원소에 대해 "포함/미포함"을 결정하는 이진 선택의 연속이다. 재귀 함수에서 start 매개변수를 사용하여 이미 고려한 원소를 다시 선택하지 않도록 하면, 중복 없이 모든 부분집합을 생성할 수 있다.

이 문제는 재귀와 백트래킹의 결합을 보여준다. 원소를 선택한 후 재귀 호출하고, 돌아와서 선택을 취소하는 패턴은 순열, 조합, 부분집합 등 경우의 수 열거 문제의 기본 틀이다.

N이 최대 15이므로 부분집합의 수는 최대 2^15 = 32,768개로, 모든 부분집합을 생성하고 정렬해도 시간 내에 처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^N * N) |
| 공간 복잡도 | O(2^N * N) |

부분집합의 수가 2^N개이고, 각 부분집합을 저장하는 데 최대 O(N)이 필요하다. 정렬에 O(2^N * N * log(2^N))이 추가로 소요되지만, N ≤ 15이므로 충분히 빠르다.

## 대안적 접근

비트마스크를 사용하면 반복문으로도 구현할 수 있다. 0부터 2^N - 1까지의 정수를 순회하며, 각 비트가 1인 위치의 원소를 부분집합에 포함시키는 방식이다. 비트마스크 방식은 재귀 호출 스택을 사용하지 않아 메모리 효율적이지만, 재귀 방식이 백트래킹과의 연결을 이해하는 데 더 유용하다.
