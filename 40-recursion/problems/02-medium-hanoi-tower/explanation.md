# 하노이의 탑 - 해설

## 접근 방식

1. N개의 원판을 기둥 A에서 기둥 C로 옮기는 문제를 세 단계로 분할한다.
2. 위의 N-1개 원판을 기둥 A에서 보조 기둥 B로 옮긴다 (재귀 호출).
3. 가장 큰 원판을 기둥 A에서 기둥 C로 옮긴다 (1회 이동).
4. 보조 기둥 B의 N-1개 원판을 기둥 C로 옮긴다 (재귀 호출).
5. 기저 조건은 원판이 1개일 때 바로 옮기는 것이다.

## 핵심 아이디어

하노이의 탑은 재귀의 고전적 예제이다. 핵심은 "N개의 원판을 옮기는 문제"를 "N-1개의 원판을 옮기는 문제 2번 + 1번 이동"으로 분할하는 것이다.

이 분할 구조에서 총 이동 횟수의 점화식은 T(N) = 2 * T(N-1) + 1이며, 이를 풀면 T(N) = 2^N - 1이 된다. 따라서 이동 횟수는 원판 수에 대해 지수적으로 증가한다.

재귀 호출에서 from, to, aux 기둥의 역할이 매 호출마다 바뀌는 것이 이 문제의 핵심 패턴이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^N) |
| 공간 복잡도 | O(N) |

총 2^N - 1번의 이동이 필요하므로 시간 복잡도는 O(2^N)이다. 재귀 깊이는 N이므로 호출 스택의 공간 복잡도는 O(N)이다.

## 대안적 접근

반복적(iterative) 방법으로도 풀 수 있다. 원판 번호의 홀짝에 따라 이동 방향을 결정하는 패턴을 이용하면 스택 없이 구현할 수 있지만, 재귀 방식이 훨씬 직관적이고 이해하기 쉽다.
