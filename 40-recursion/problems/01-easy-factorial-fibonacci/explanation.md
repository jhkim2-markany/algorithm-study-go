# 팩토리얼과 피보나치 - 해설

## 접근 방식

1. 팩토리얼은 기본 재귀로 구현한다. N!은 N * (N-1)!로 정의되며, 기저 조건은 0! = 1이다.
2. 피보나치는 단순 재귀로 구현하면 O(2^N)의 시간이 걸리므로, 메모이제이션을 적용한다.
3. 메모이제이션은 맵을 사용하여 이미 계산한 값을 저장하고, 동일한 입력에 대해 캐시된 값을 반환한다.
4. 두 함수 모두 기저 조건을 먼저 확인하고, 재귀 단계에서 문제 크기를 줄여나간다.

## 핵심 아이디어

팩토리얼은 선형 재귀의 가장 기본적인 예시이다. 한 번의 재귀 호출로 문제 크기가 1씩 줄어들어 N번 호출 후 기저 조건에 도달한다.

피보나치는 이진 재귀(두 번의 재귀 호출)의 대표적 예시이다. 단순 재귀는 동일한 부분 문제를 반복 계산하여 지수 시간이 걸리지만, 메모이제이션을 적용하면 각 부분 문제를 한 번만 계산하여 O(N)으로 줄일 수 있다. 이것이 재귀와 동적 프로그래밍의 연결 고리이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (팩토리얼) | O(N) |
| 시간 복잡도 (피보나치, 메모이제이션) | O(M) |
| 공간 복잡도 | O(N + M) |

팩토리얼은 N번의 재귀 호출, 피보나치는 메모이제이션 덕분에 M번의 계산만 수행한다. 공간은 호출 스택과 메모이제이션 테이블에 사용된다.

## 대안적 접근

팩토리얼과 피보나치 모두 반복문(Bottom-up)으로 구현할 수 있다. 반복문 방식은 호출 스택을 사용하지 않아 스택 오버플로 위험이 없지만, 재귀 방식이 점화식을 더 직관적으로 표현한다.
