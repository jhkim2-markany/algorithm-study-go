# 그리디

## 개념

그리디(Greedy) 알고리즘은 매 단계에서 현재 상황에서 가장 좋아 보이는 선택(지역 최적해)을 반복하여 전체 문제의 최적해를 구하는 방법이다. 한 번 내린 선택을 번복하지 않으며, 이전 선택이 이후 선택에 영향을 주지 않는다는 가정 하에 동작한다.

그리디 알고리즘이 최적해를 보장하려면 다음 두 가지 조건을 만족해야 한다:

- **탐욕적 선택 속성(Greedy Choice Property)**: 매 단계에서 지역 최적 선택이 전체 최적해로 이어진다. 즉, 현재 최선의 선택이 나중에 더 나은 선택을 배제하지 않는다.
- **최적 부분 구조(Optimal Substructure)**: 전체 문제의 최적해가 부분 문제의 최적해를 포함한다. 큰 문제의 최적해를 작은 문제의 최적해로부터 구성할 수 있다.

이 두 조건이 성립하지 않는 문제에 그리디를 적용하면 최적해가 아닌 근사해만 얻을 수 있으므로, 문제 분석 단계에서 그리디 적용 가능 여부를 반드시 검증해야 한다.

## 동작 원리

그리디 알고리즘은 일반적으로 다음 단계를 따른다:

1. 문제의 해를 구성하는 요소들을 정렬하거나 우선순위를 매긴다
2. 가장 유리한 요소부터 순서대로 선택한다
3. 선택한 요소가 제약 조건을 만족하는지 확인한다
4. 조건을 만족하면 해에 포함시키고, 그렇지 않으면 건너뛴다
5. 모든 요소를 검토할 때까지 2~4단계를 반복한다

### 대표적인 그리디 문제 유형

- **거스름돈 문제**: 가장 큰 단위의 동전부터 사용하여 최소 동전 수로 거스름돈을 만든다
- **활동 선택 문제**: 종료 시간이 빠른 활동부터 선택하여 최대한 많은 활동을 수행한다
- **분할 가능 배낭 문제**: 단위 무게당 가치가 높은 물건부터 담아 배낭의 가치를 최대화한다
- **허프만 코딩**: 빈도가 낮은 문자부터 합쳐 최적 접두사 코드를 생성한다

### 그리디 vs 동적 프로그래밍

그리디와 동적 프로그래밍(DP)은 모두 최적 부분 구조를 활용하지만, 핵심 차이가 있다:

- 그리디는 매 단계에서 하나의 선택만 하고 되돌아가지 않는다 (탑다운 방식)
- DP는 모든 가능한 선택을 고려하고 그 중 최적을 선택한다 (바텀업 또는 메모이제이션)
- 그리디가 적용 가능한 문제는 DP로도 풀 수 있지만, 그 역은 성립하지 않는다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(N) |
| 시간 복잡도 (평균) | O(N log N) |
| 시간 복잡도 (최악) | O(N log N) |
| 공간 복잡도 | O(1) ~ O(N) |

그리디 알고리즘 자체의 선택 과정은 O(N)이지만, 대부분의 경우 정렬이 선행되므로 전체 시간 복잡도는 O(N log N)이 된다. 정렬 없이 순회만으로 해결되는 문제(예: 거스름돈)는 O(N)이다. 공간 복잡도는 문제에 따라 다르며, 추가 자료구조 없이 해결 가능한 경우 O(1)이다.

## 적합한 문제 유형

- 거스름돈, 동전 교환 등 최소 개수 문제
- 활동 선택, 회의실 배정 등 스케줄링 문제
- 분할 가능 배낭 문제 (Fractional Knapsack)
- 최소 신장 트리 (Kruskal, Prim)
- 최단 경로 (Dijkstra)
- 허프만 코딩 등 최적 인코딩 문제
- 구간 관련 문제 (구간 합치기, 최소 구간 커버)
- 문자열/배열에서 사전순 최소/최대 구성 문제


## 단계별 추적 (Trace)

### 예시: 활동 선택 문제 (Activity Selection)

활동 목록이 `[(시작, 종료)]` 형태로 주어진다. 겹치지 않게 최대한 많은 활동을 선택한다.

입력: `[(1,4), (3,5), (0,6), (5,7), (3,9), (5,9), (6,10), (8,11), (8,12), (2,14), (12,16)]`

### 실행 과정

```
1단계: 종료 시간 기준으로 정렬
  (1,4) (3,5) (0,6) (5,7) (3,9) (5,9) (6,10) (8,11) (8,12) (2,14) (12,16)

시간축:
0    2    4    6    8   10   12   14   16
|----|----|----|----|----|----|----|----|
[=======]                                (1,4)
     [=======]                           (3,5)
[==============]                         (0,6)
          [=======]                      (5,7)
     [================]                  (3,9)
          [============]                 (5,9)
               [===========]            (6,10)
                    [=========]         (8,11)
                    [============]      (8,12)
  [============================]        (2,14)
                         [=========]   (12,16)

2단계: 탐욕적 선택 (종료 시간이 빠른 것부터)

선택 1: (1,4) ✓  → 현재 종료 시간 = 4
  (3,5): 시작=3 < 4 → 겹침, 건너뜀 ✗
  (0,6): 시작=0 < 4 → 겹침, 건너뜀 ✗
선택 2: (5,7) ✓  → 현재 종료 시간 = 7
  (3,9): 시작=3 < 7 → 겹침, 건너뜀 ✗
  (5,9): 시작=5 < 7 → 겹침, 건너뜀 ✗
  (6,10): 시작=6 < 7 → 겹침, 건너뜀 ✗
선택 3: (8,11) ✓ → 현재 종료 시간 = 11
  (8,12): 시작=8 < 11 → 겹침, 건너뜀 ✗
  (2,14): 시작=2 < 11 → 겹침, 건너뜀 ✗
선택 4: (12,16) ✓ → 현재 종료 시간 = 16

결과: 4개 활동 선택 → (1,4), (5,7), (8,11), (12,16)

시간축 (선택된 활동):
0    2    4    6    8   10   12   14   16
|----|----|----|----|----|----|----|----|
[=======]                                (1,4) ✓
          [=======]                      (5,7) ✓
                    [=========]         (8,11) ✓
                         [=========]   (12,16) ✓
```

## 실전 팁

### 활용 노하우

- 그리디 문제는 "정렬 기준"을 찾는 것이 핵심이다. 종료 시간, 단위 가치, 마감 기한 등 무엇을 기준으로 정렬할지 결정하라
- 그리디가 최적해를 보장하는지 확신이 없으면, 작은 반례를 직접 만들어 검증하라
- 코딩테스트에서 그리디 문제는 보통 정렬 + 한 번 순회(O(N log N))로 풀린다
- "~의 최소 개수", "~의 최대 개수" 문제에서 그리디를 먼저 시도해 보라

### 자주 하는 실수

- **정렬 기준 오류**: 시작 시간이 아닌 종료 시간으로 정렬해야 하는 문제에서 기준을 잘못 잡는 경우
- **그리디 적용 불가 문제에 적용**: 0-1 배낭 문제처럼 그리디가 최적해를 보장하지 않는 문제에 그리디를 적용하면 오답이 된다
- **동점 처리 누락**: 정렬 기준이 같은 원소들의 2차 정렬 기준을 고려하지 않는 경우
- **탐욕적 선택의 정당성 미검증**: "이게 맞겠지"라는 직감만으로 구현하면 반례에 걸린다

### 엣지 케이스 주의사항

- 입력이 1개뿐인 경우
- 모든 활동/구간이 겹치는 경우
- 모든 활동/구간이 겹치지 않는 경우
- 시작 시간과 종료 시간이 같은 활동 (길이 0인 구간)
- 정렬 기준 값이 동일한 원소가 다수인 경우

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 핵심 아이디어 | 적합한 상황 |
| --- | --- | --- | --- |
| 그리디 | O(N log N) | 지역 최적 선택 반복 | 탐욕적 선택 속성 + 최적 부분 구조 |
| 동적 프로그래밍 | O(N²) ~ O(N × W) | 모든 경우 탐색 + 메모이제이션 | 그리디 불가, 부분 문제 중복 |
| 브루트포스 | O(2^N) | 모든 조합 탐색 | 소규모 데이터, 검증용 |
| 백트래킹 | O(2^N) (최악) | 가지치기 포함 전수 탐색 | 제약 조건이 강한 탐색 |

**선택 기준:**
- 탐욕적 선택 속성이 증명 가능 → 그리디
- 탐욕적 선택이 최적해를 보장하지 않음 → DP
- 분할 가능 배낭 → 그리디, 0-1 배낭 → DP
- 스케줄링(종료 시간 기준) → 그리디
- 최소 동전 수(임의 단위) → DP, 배수 관계 단위 → 그리디
