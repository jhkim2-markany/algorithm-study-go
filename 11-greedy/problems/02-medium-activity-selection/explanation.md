# 활동 선택 - 해설

## 접근 방식

1. N개의 회의 정보(시작 시간, 종료 시간)를 입력받는다
2. 종료 시간을 기준으로 오름차순 정렬한다
3. 첫 번째 회의를 선택하고, 이후 회의들을 순서대로 검토한다
4. 현재 회의의 시작 시간이 마지막으로 선택한 회의의 종료 시간 이상이면 선택한다
5. 선택된 회의의 총 개수를 출력한다

## 핵심 아이디어

이 문제는 대표적인 그리디 문제인 활동 선택 문제(Activity Selection Problem)이다. 핵심 전략은 종료 시간이 빠른 회의를 우선 선택하는 것이다.

종료 시간이 빠른 회의를 선택하면 남은 시간이 최대한 많아지므로, 이후에 더 많은 회의를 배치할 수 있다. 이것이 탐욕적 선택 속성을 만족하는 이유이다.

정렬 시 종료 시간이 같은 경우 시작 시간이 빠른 것을 우선하면, 동일 종료 시간 내에서도 올바른 선택이 보장된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

정렬에 O(N log N), 선택 과정에 O(N)이 소요되므로 전체 시간 복잡도는 O(N log N)이다. N개의 회의 정보를 저장하는 배열이 필요하므로 공간 복잡도는 O(N)이다.

## 대안적 접근

- **시작 시간 기준 정렬**: 시작 시간 기준으로 정렬하면 최적해를 보장하지 않는다. 시작이 빠르지만 매우 긴 회의가 여러 짧은 회의를 차단할 수 있기 때문이다.
- **동적 프로그래밍**: 가중치가 있는 활동 선택 문제(각 회의에 가치가 다른 경우)에는 DP를 사용해야 한다. 이 문제처럼 모든 회의의 가치가 동일한 경우에는 그리디가 최적이다.
