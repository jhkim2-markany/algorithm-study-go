# 작업 스케줄링 - 해설

## 접근 방식

1. N개의 작업 정보(마감 기한, 보상)를 입력받는다
2. 보상이 큰 작업부터 처리하기 위해 보상 기준 내림차순으로 정렬한다
3. 각 작업에 대해 마감 기한 당일부터 1일차까지 역순으로 빈 날짜를 찾는다
4. 빈 날짜가 있으면 해당 날짜에 작업을 배정하고 보상을 누적한다
5. 빈 날짜가 없으면 해당 작업은 건너뛴다
6. 모든 작업을 검토한 후 총 보상을 출력한다

## 핵심 아이디어

이 문제의 핵심은 보상이 큰 작업을 우선 배정하되, 가능한 한 늦은 날짜에 배정하는 것이다.

보상이 큰 작업부터 처리하는 이유는 탐욕적 선택 속성 때문이다. 보상이 큰 작업을 먼저 확보하면 전체 보상이 최대가 된다.

마감 기한 당일부터 역순으로 빈 날짜를 찾는 이유는, 가능한 한 늦은 날짜에 배정해야 앞쪽 날짜를 마감이 빠른 다른 작업에 양보할 수 있기 때문이다.

빈 날짜를 효율적으로 찾기 위해 유니온 파인드(Union-Find) 자료구조를 사용할 수 있지만, 이 풀이에서는 단순 배열을 사용하여 구현의 명확성을 우선한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × D) |
| 공간 복잡도 | O(N) |

정렬에 O(N log N), 각 작업마다 최대 D일을 탐색하므로 최악의 경우 O(N × D)이다. D ≤ N이므로 최악 O(N²)이지만, 실제로는 빈 날짜를 빠르게 찾으므로 평균적으로 훨씬 빠르다. 유니온 파인드를 사용하면 O(N log N)으로 최적화할 수 있다.

## 대안적 접근

- **유니온 파인드 최적화**: 각 날짜의 "다음 빈 날짜"를 유니온 파인드로 관리하면 빈 날짜 탐색을 거의 O(1)로 줄일 수 있다. 전체 시간 복잡도가 O(N log N)이 된다.
- **우선순위 큐 활용**: 마감 기한 순으로 정렬한 뒤, 현재까지의 작업 중 보상이 가장 작은 것을 교체하는 방식으로도 풀 수 있다.
