# 거스름돈 - 해설

## 접근 방식

1. 거슬러 줘야 할 금액 N을 입력받는다
2. 가장 큰 단위의 동전(500원)부터 차례로 사용한다
3. 각 동전에 대해 현재 남은 금액을 해당 동전으로 나눈 몫이 사용할 동전 수이다
4. 나머지를 다음 단위 동전으로 넘긴다
5. 모든 동전 종류를 처리하면 총 동전 수를 출력한다

## 핵심 아이디어

이 문제는 그리디 알고리즘의 가장 기본적인 예시이다. 큰 단위의 동전부터 최대한 많이 사용하면 전체 동전 수가 최소가 된다.

이 전략이 최적해를 보장하는 이유는 동전 단위가 배수 관계(500 = 5×100, 100 = 2×50, 50 = 5×10)이기 때문이다. 큰 동전 1개는 항상 작은 동전 여러 개로 대체 가능하므로, 큰 동전을 먼저 사용하는 것이 항상 유리하다.

만약 동전 단위가 배수 관계가 아닌 경우(예: 1원, 7원, 10원)에는 그리디가 최적해를 보장하지 않으며, 동적 프로그래밍을 사용해야 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(K) |
| 공간 복잡도 | O(1) |

K는 동전 종류의 수이다. 이 문제에서 K=4로 고정되어 있으므로 사실상 O(1)이다. 각 동전에 대해 나눗셈과 나머지 연산만 수행하므로 매우 효율적이다.

## 대안적 접근

- **동적 프로그래밍**: 동전 단위가 배수 관계가 아닌 일반적인 경우에는 DP로 풀어야 한다. 이 문제의 동전 단위에서는 그리디가 최적이므로 DP는 불필요하게 복잡하다.
