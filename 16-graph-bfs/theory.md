# 그래프 탐색 BFS

## 개념

너비 우선 탐색(Breadth-First Search, BFS)은 그래프의 한 정점에서 시작하여 인접한 정점을 먼저 모두 방문한 뒤, 방문한 정점들의 인접 정점을 다시 방문하는 방식으로 탐색하는 알고리즘이다. 시작 정점으로부터의 거리가 가까운 정점부터 차례대로 방문하므로, 가중치가 없는 그래프에서 최단 경로를 구하는 데 적합하다.

BFS의 핵심 특성은 다음과 같다:
- **너비 우선**: 시작 정점에서 가까운 정점부터 먼 정점 순서로 탐색한다
- **레벨 단위 탐색**: 같은 거리에 있는 정점들을 한 레벨로 묶어 처리할 수 있다
- **큐 기반**: FIFO(선입선출) 큐를 사용하여 방문 순서를 관리한다
- **최단 거리 보장**: 가중치가 없는(또는 모든 가중치가 동일한) 그래프에서 시작 정점으로부터의 최단 거리를 보장한다

BFS는 DFS와 함께 그래프 탐색의 양대 기본 알고리즘이며, 최단 거리 문제, 레벨별 탐색, 이분 그래프 판별, 다차원 격자 탐색 등에 널리 사용된다.

## 동작 원리

### 기본 BFS

1. 시작 정점을 방문 처리하고 큐에 넣는다.
2. 큐에서 정점을 하나 꺼낸다.
3. 꺼낸 정점의 인접 정점 목록을 순회한다.
4. 방문하지 않은 인접 정점이 있으면 방문 처리하고 큐에 넣는다.
5. 큐가 빌 때까지 2~4를 반복한다.

### 최단 거리 BFS

1. 시작 정점의 거리를 0으로 설정하고 큐에 넣는다.
2. 큐에서 정점을 하나 꺼낸다.
3. 꺼낸 정점의 인접 정점 중 미방문 정점에 대해 거리를 (현재 정점의 거리 + 1)로 설정하고 큐에 넣는다.
4. 큐가 빌 때까지 반복하면, 모든 도달 가능한 정점까지의 최단 거리가 계산된다.

### 다차원 격자 BFS

2차원 격자(미로, 지도 등)에서의 BFS는 그래프 BFS의 특수한 형태이다:

1. 시작 좌표를 큐에 넣고 방문 처리한다.
2. 큐에서 좌표를 꺼내고, 상하좌우(4방향 또는 8방향) 인접 칸을 확인한다.
3. 격자 범위 안에 있고, 이동 가능하며, 미방문인 칸을 큐에 넣고 방문 처리한다.
4. 큐가 빌 때까지 반복한다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(V + E) |
| 시간 복잡도 (평균) | O(V + E) |
| 시간 복잡도 (최악) | O(V + E) |
| 공간 복잡도 | O(V) |

V는 정점의 수, E는 간선의 수이다. 모든 정점과 간선을 한 번씩 확인하므로 시간 복잡도는 O(V + E)이다. 방문 배열에 O(V), 큐에 최악의 경우 O(V) 공간이 필요하다. 2차원 격자의 경우 V = 행 × 열, E = O(V)이므로 시간 복잡도는 O(행 × 열)이다.

## 적합한 문제 유형

- 가중치 없는 그래프에서 최단 거리 구하기
- 2차원 격자(미로)에서 최단 경로 탐색
- 레벨별(거리별) 탐색이 필요한 문제
- 이분 그래프 판별
- 다중 시작점 BFS (여러 시작점에서 동시에 탐색)
- 연결 요소 탐색
- 0-1 BFS (가중치가 0 또는 1인 그래프에서의 최단 경로)
- 상태 공간 탐색 (퍼즐, 게임 등의 최소 이동 횟수)


## 단계별 추적 (Trace)

### 예시 입력

```
정점 수: 6, 간선: (1,2), (1,3), (2,4), (2,5), (3,6)
시작 정점: 1

인접 리스트:
1: [2, 3]
2: [1, 4, 5]
3: [1, 6]
4: [2]
5: [2]
6: [3]
```

### 실행 과정

```
단계   큐 상태          꺼낸 정점   거리 갱신           visited
──────────────────────────────────────────────────────────────
초기   [1]              -          dist[1]=0           {1}
1      [2, 3]           1          dist[2]=1, dist[3]=1  {1,2,3}
2      [3, 4, 5]        2          dist[4]=2, dist[5]=2  {1,2,3,4,5}
3      [4, 5, 6]        3          dist[6]=2             {1,2,3,4,5,6}
4      [5, 6]           4          (인접: 2 이미 방문)    {1,2,3,4,5,6}
5      [6]              5          (인접: 2 이미 방문)    {1,2,3,4,5,6}
6      []               6          (인접: 3 이미 방문)    {1,2,3,4,5,6}
```

**레벨별 탐색 시각화**

```
레벨 0:     1           dist = 0
           / \
레벨 1:   2   3         dist = 1
         / \   \
레벨 2: 4   5   6       dist = 2
```

**방문 순서: 1 → 2 → 3 → 4 → 5 → 6**

**최단 거리 결과: dist = [_, 0, 1, 1, 2, 2, 2]** (인덱스 0은 미사용)

## 실전 팁

### 활용 노하우

- 가중치 없는 그래프에서 최단 거리를 구할 때는 BFS가 정답이다. 다익스트라를 쓸 필요 없다
- 2차원 격자 문제에서 최단 경로를 구할 때 BFS를 사용한다. `dx, dy` 배열로 4방향 이동을 처리하고, `dist[r][c]`로 거리를 기록한다
- 다중 시작점 BFS: 여러 시작점을 큐에 동시에 넣고 BFS를 수행하면, 각 칸에서 가장 가까운 시작점까지의 거리를 한 번에 구할 수 있다
- 레벨별 처리가 필요하면 큐의 현재 크기만큼만 꺼내는 방식으로 레벨을 구분한다

### 자주 하는 실수

- 방문 체크를 큐에서 꺼낼 때 하면 같은 정점이 큐에 여러 번 들어가 메모리 초과나 시간 초과가 발생한다. 큐에 넣을 때 방문 체크를 해야 한다
- 2차원 격자에서 범위 체크를 빠뜨려 인덱스 초과 에러가 발생하는 경우가 많다. 이동 후 좌표가 격자 범위 안에 있는지 먼저 확인한다
- 시작 정점의 거리를 0으로 초기화하지 않거나, 방문 배열을 초기화하지 않는 실수

### 엣지 케이스 주의사항

- 시작 정점과 도착 정점이 같은 경우: 거리 0을 반환해야 한다
- 도달 불가능한 정점: 거리를 -1 또는 무한대로 처리해야 한다
- 격자 크기가 1×1인 경우: 시작점이 곧 도착점
- 모든 칸이 벽인 경우: 이동 불가능 처리

## 관련 알고리즘 비교

| 알고리즘 | 탐색 방식 | 자료구조 | 최단 거리 보장 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| BFS | 너비 우선 | 큐 | ✓ (가중치 없는 그래프) | 최단 거리, 레벨별 탐색, 격자 탐색 |
| DFS | 깊이 우선 | 스택/재귀 | ✗ | 연결 요소, 사이클 판별, 경로 존재 |
| 다익스트라 | 최소 비용 우선 | 우선순위 큐 | ✓ (양의 가중치) | 가중치 그래프 최단 경로 |
| 0-1 BFS | 너비 우선 변형 | 덱 | ✓ (가중치 0 또는 1) | 가중치 0/1 그래프 최단 경로 |
| 벨만-포드 | 간선 완화 반복 | 배열 | ✓ (음의 가중치 허용) | 음의 가중치 그래프 최단 경로 |
