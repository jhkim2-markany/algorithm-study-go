# 2D 좌표 압축 활용 - 해설

## 접근 방식

1. 모든 직사각형의 x 좌표와 y 좌표를 각각 수집한다
2. x 좌표와 y 좌표를 각각 정렬 후 중복 제거하여 압축한다
3. 압축된 좌표로 격자(grid)를 구성한다
4. 각 격자 셀에 대해 어떤 직사각형이 해당 셀을 덮는지 확인한다
5. 덮이는 셀의 실제 넓이(압축 전 좌표 기준)를 합산한다

## 핵심 아이디어

좌표 범위가 -10⁹ ~ 10⁹으로 매우 크지만, 실제 직사각형의 꼭짓점 좌표는 최대 2N개이다. 좌표 압축을 통해 최대 2N × 2N 크기의 격자로 변환하면, 모든 셀을 순회하며 넓이를 계산할 수 있다.

핵심은 압축된 격자의 각 셀이 원래 좌표계에서 서로 다른 크기의 직사각형 영역을 나타낸다는 점이다. 따라서 셀이 덮이는지 확인한 후, 실제 넓이(xs[i+1] - xs[i]) × (ys[j+1] - ys[j])를 더해야 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N³) |
| 공간 복잡도 | O(N²) |

좌표 압축에 O(N log N), 격자 순회에 O(N²), 각 셀마다 직사각형 확인에 O(N)이 소요되어 전체 O(N³)이다. N ≤ 1,000이므로 충분히 빠르다.

## 대안적 접근

스위핑(Sweeping) + 세그먼트 트리를 사용하면 O(N² log N) 또는 O(N log N)으로 더 효율적으로 풀 수 있다. x 좌표 기준으로 이벤트를 정렬하고, y 좌표에 대해 세그먼트 트리로 구간 업데이트를 수행하는 방식이다. N이 매우 큰 경우에는 이 방법이 필요하지만, N ≤ 1,000인 이 문제에서는 좌표 압축 + 격자 순회가 더 직관적이다.
