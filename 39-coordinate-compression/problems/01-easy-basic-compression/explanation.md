# 기본 좌표 압축 - 해설

## 접근 방식

1. N개의 정수를 입력받는다
2. 원본 배열을 복사한 뒤 정렬한다
3. 정렬된 배열에서 중복을 제거하여 고유 좌표 배열을 만든다
4. 원본 배열의 각 원소에 대해 고유 좌표 배열에서 이진 탐색으로 인덱스를 찾는다
5. 찾은 인덱스가 곧 해당 값보다 작은 서로 다른 수의 개수이다

## 핵심 아이디어

좌표 압축의 가장 기본적인 형태이다. "해당 수보다 작은 서로 다른 수의 개수"는 곧 정렬 후 중복 제거한 배열에서의 인덱스와 같다.

예를 들어 {2, 4, -10, 4, -9}를 정렬하면 {-10, -9, 2, 4}(중복 제거 후)이고, 2는 인덱스 2에 위치하므로 2보다 작은 서로 다른 수는 2개(-10, -9)이다.

Go의 `sort.SearchInts()` 함수는 정렬된 배열에서 이진 탐색을 수행하여 O(log N)에 위치를 찾아준다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

정렬에 O(N log N), 이진 탐색 N회에 O(N log N)이 소요되어 전체 O(N log N)이다.

## 대안적 접근

맵(해시맵)을 사용하여 각 값에 순위를 매핑하는 방법도 있다. 정렬 후 중복 제거한 배열을 순회하면서 맵에 {값: 인덱스}를 저장하면, 이후 O(1)에 조회할 수 있다. 전체 시간 복잡도는 동일하게 O(N log N)이지만, 이진 탐색 대신 해시맵 조회를 사용하므로 상수 시간이 더 빠를 수 있다.
