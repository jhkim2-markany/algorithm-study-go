# 좌표 압축 + 구간 합 - 해설

## 접근 방식

1. N개의 점의 좌표를 수집하여 좌표 압축을 수행한다
2. 압축된 좌표 기준으로 각 위치에 가중치를 합산한다
3. 누적합(Prefix Sum) 배열을 구성한다
4. 각 쿼리 [L, R]에 대해 이진 탐색으로 압축된 범위를 찾고, 누적합으로 O(1)에 답을 구한다

## 핵심 아이디어

좌표 범위가 -10⁹ ~ 10⁹으로 매우 크기 때문에 배열 인덱스로 직접 사용할 수 없다. 좌표 압축을 통해 실제 사용되는 좌표만 추출하면 최대 N개의 고유 좌표로 줄어든다.

압축 후 핵심은 쿼리의 구간 [L, R]을 압축된 인덱스 범위로 변환하는 것이다:

- L 이상인 첫 번째 압축 인덱스: `sort.SearchInts(unique, L)` (lower bound)
- R 이하인 마지막 압축 인덱스 + 1: `sort.SearchInts(unique, R+1)` (upper bound)

이렇게 변환된 범위에 대해 누적합을 사용하면 O(1)에 구간 합을 구할 수 있다.

같은 좌표에 여러 점이 있는 경우 가중치를 합산하여 처리한다. int64를 사용하여 오버플로우를 방지한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + Q) log N) |
| 공간 복잡도 | O(N + Q) |

좌표 압축에 O(N log N), 누적합 구성에 O(N), 각 쿼리 처리에 O(log N)이 소요되어 전체 O((N + Q) log N)이다.

## 대안적 접근

세그먼트 트리나 BIT(펜윅 트리)를 사용하면 점 업데이트와 구간 쿼리를 동시에 지원할 수 있다. 이 문제처럼 모든 점이 미리 주어지고 쿼리만 처리하는 경우에는 누적합이 더 간단하고 효율적이지만, 점이 동적으로 추가/삭제되는 경우에는 세그먼트 트리가 필요하다. 오프라인 쿼리로 처리하는 방법도 가능하다.
