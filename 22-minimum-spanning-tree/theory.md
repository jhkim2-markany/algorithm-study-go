# 최소 신장 트리

## 개념

최소 신장 트리(Minimum Spanning Tree, MST)는 가중치 무방향 그래프에서 모든 정점을 연결하는 트리 중 간선 가중치의 합이 최소인 트리이다. 신장 트리는 그래프의 모든 정점을 포함하면서 사이클이 없는 부분 그래프로, 정점이 V개이면 정확히 V-1개의 간선을 갖는다.

MST의 핵심 성질은 다음과 같다:

- **컷 성질(Cut Property)**: 그래프를 두 집합으로 나누는 컷에서 가중치가 최소인 간선은 반드시 MST에 포함된다.
- **사이클 성질(Cycle Property)**: 사이클에서 가중치가 최대인 간선은 MST에 포함되지 않는다.
- **유일성**: 모든 간선의 가중치가 서로 다르면 MST는 유일하다.

MST를 구하는 대표적인 알고리즘으로 Kruskal 알고리즘과 Prim 알고리즘이 있다.

- **Kruskal 알고리즘**: 간선을 가중치 순으로 정렬한 뒤, Union-Find를 사용하여 사이클을 형성하지 않는 간선을 하나씩 선택한다.
- **Prim 알고리즘**: 임의의 정점에서 시작하여 우선순위 큐로 현재 트리에 인접한 간선 중 최소 가중치 간선을 선택하며 트리를 확장한다.

## 동작 원리

### Kruskal 알고리즘

1. 그래프의 모든 간선을 가중치 오름차순으로 정렬한다.
2. 각 정점을 독립된 집합으로 초기화한다 (Union-Find 초기화).
3. 정렬된 간선을 순서대로 확인한다.
4. 현재 간선의 두 끝점이 서로 다른 집합에 속하면 (사이클이 생기지 않으면) 해당 간선을 MST에 추가하고 두 집합을 합친다 (Union).
5. 두 끝점이 같은 집합에 속하면 (사이클이 생기면) 해당 간선을 건너뛴다.
6. MST에 V-1개의 간선이 추가되면 종료한다.

Kruskal 알고리즘은 그리디 방식으로 동작하며, 컷 성질에 의해 정확성이 보장된다. Union-Find에 경로 압축과 랭크 기반 합치기를 적용하면 거의 상수 시간에 집합 연산을 수행할 수 있다.

### Prim 알고리즘

1. 임의의 시작 정점을 선택하고 MST 집합에 추가한다.
2. 시작 정점에 연결된 모든 간선을 우선순위 큐(최소 힙)에 넣는다.
3. 큐에서 가중치가 가장 작은 간선을 꺼낸다.
4. 해당 간선의 도착 정점이 이미 MST에 포함되어 있으면 건너뛴다.
5. 포함되어 있지 않으면 도착 정점을 MST에 추가하고, 해당 정점에 연결된 간선들을 큐에 넣는다.
6. MST에 V개의 정점이 모두 포함되면 종료한다.

Prim 알고리즘은 Dijkstra 알고리즘과 구조가 유사하다. 차이점은 Dijkstra는 출발점으로부터의 누적 거리를 기준으로 하지만, Prim은 현재 트리까지의 간선 가중치만을 기준으로 한다.

## 복잡도

| 알고리즘 | 시간 복잡도 | 공간 복잡도 | 비고 |
| --- | --- | --- | --- |
| Kruskal | O(E log E) | O(V + E) | 간선 정렬이 지배적 |
| Prim (우선순위 큐) | O((V + E) log V) | O(V + E) | 희소 그래프에 유리 |
| Prim (배열) | O(V²) | O(V + E) | 밀집 그래프에 유리 |

- V: 정점의 수, E: 간선의 수
- Kruskal은 간선 수가 적은 희소 그래프에서 효율적이며, Union-Find 연산은 거의 O(1)이므로 정렬 비용 O(E log E)가 지배적이다.
- Prim은 우선순위 큐를 사용할 때 Dijkstra와 동일한 복잡도를 가지며, 밀집 그래프에서는 배열 기반 구현이 더 효율적일 수 있다.
- 간선 수 E는 최대 V(V-1)/2이므로 O(E log E) = O(E log V)이다.

## 적합한 문제 유형

- 모든 도시를 최소 비용으로 연결하는 도로/네트워크 설계
- 통신 네트워크에서 최소 비용 케이블 배선
- 클러스터링 (MST에서 가장 비용이 큰 간선을 제거하여 그룹 분리)
- 전력망, 수도관 등 인프라 최소 비용 구축
- 차선 최소 신장 트리 (Second-best MST) 문제
- 간선 추가/제거 시 MST 변화 분석
- 가중치 그래프의 연결 비용 최적화

## 단계별 추적 (Trace)

### 예시 입력 - Kruskal

정점 5개 (0~4), 간선: (0-1, 2), (0-3, 6), (1-2, 3), (1-3, 8), (1-4, 5), (2-4, 7), (3-4, 9)

### 실행 과정

```
간선 정렬 (가중치 오름차순):
  (0-1, 2), (1-2, 3), (1-4, 5), (0-3, 6), (2-4, 7), (1-3, 8), (3-4, 9)

초기 Union-Find: 각 정점이 독립 집합
  {0} {1} {2} {3} {4}

간선 (0-1, 2): Find(0)=0, Find(1)=1 → 다른 집합 → 선택! Union(0,1)
  MST 간선: [(0-1, 2)]
  {0,1} {2} {3} {4}

간선 (1-2, 3): Find(1)=0, Find(2)=2 → 다른 집합 → 선택! Union(0,2)
  MST 간선: [(0-1, 2), (1-2, 3)]
  {0,1,2} {3} {4}

간선 (1-4, 5): Find(1)=0, Find(4)=4 → 다른 집합 → 선택! Union(0,4)
  MST 간선: [(0-1, 2), (1-2, 3), (1-4, 5)]
  {0,1,2,4} {3}

간선 (0-3, 6): Find(0)=0, Find(3)=3 → 다른 집합 → 선택! Union(0,3)
  MST 간선: [(0-1, 2), (1-2, 3), (1-4, 5), (0-3, 6)]
  {0,1,2,3,4}

V-1 = 4개 간선 선택 완료 → 종료
MST 총 비용: 2 + 3 + 5 + 6 = 16
```

### Prim 알고리즘 추적 (시작 정점 0)

```
초기: MST = {0}, PQ = [(2, 0→1), (6, 0→3)]

Step 1: PQ에서 (2, 0→1) 꺼냄 → 정점1 추가
  MST = {0, 1}
  PQ에 추가: (3, 1→2), (8, 1→3), (5, 1→4)
  PQ = [(3,1→2), (5,1→4), (6,0→3), (8,1→3)]

Step 2: PQ에서 (3, 1→2) 꺼냄 → 정점2 추가
  MST = {0, 1, 2}
  PQ에 추가: (7, 2→4)
  PQ = [(5,1→4), (6,0→3), (7,2→4), (8,1→3)]

Step 3: PQ에서 (5, 1→4) 꺼냄 → 정점4 추가
  MST = {0, 1, 2, 4}
  PQ에 추가: (9, 4→3)

Step 4: PQ에서 (6, 0→3) 꺼냄 → 정점3 추가
  MST = {0, 1, 2, 3, 4} → 모든 정점 포함, 종료

MST 총 비용: 2 + 3 + 5 + 6 = 16 (Kruskal과 동일)
```

## 실전 팁

### 활용 노하우

- 간선 수가 적은 희소 그래프에서는 Kruskal이 유리하고, 밀집 그래프에서는 Prim이 유리하다
- Kruskal은 Union-Find 구현이 필수이므로, Union-Find 템플릿을 미리 준비해두자
- "모든 정점을 최소 비용으로 연결"이라는 키워드가 나오면 MST를 떠올리자
- MST에서 가장 비용이 큰 간선을 제거하면 2개의 클러스터로 분리할 수 있다 (클러스터링 응용)
- 간선 가중치가 모두 같으면 아무 신장 트리나 MST이다

### 자주 하는 실수

- Kruskal에서 Union-Find 초기화를 빠뜨리는 경우
- Prim에서 이미 MST에 포함된 정점을 다시 처리하여 중복 간선을 추가하는 경우
- 그래프가 비연결(disconnected)인 경우를 처리하지 않는 경우 (MST가 존재하지 않음)
- 간선 정렬 시 가중치가 같은 간선의 순서를 고려하지 않는 경우 (결과는 달라도 비용은 동일)
- 방향 그래프에 MST 알고리즘을 적용하는 실수 (MST는 무방향 그래프 전용)

### 엣지 케이스

- 정점이 1개인 경우 (MST 비용 = 0, 간선 0개)
- 그래프가 비연결인 경우 (MST 불가능 → 최소 신장 포레스트)
- 모든 간선의 가중치가 동일한 경우
- 다중 간선(같은 두 정점 사이에 여러 간선)이 있는 경우
- 자기 자신으로의 간선(self-loop)이 있는 경우 (무시해야 함)

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| Kruskal | O(E log E) | 간선 중심, Union-Find 사용 | 희소 그래프, 간선 리스트 입력 |
| Prim (힙) | O((V+E) log V) | 정점 중심, 우선순위 큐 사용 | 밀집 그래프, 인접 리스트 입력 |
| Prim (배열) | O(V²) | 정점 중심, 단순 배열 사용 | 밀집 그래프, V가 작을 때 |
| Borůvka | O(E log V) | 병렬화 가능, 각 컴포넌트에서 최소 간선 선택 | 병렬 처리가 필요한 경우 |
| Dijkstra | O((V+E) log V) | 최단 경로 (MST 아님) | 단일 출발점 최단 경로 |
