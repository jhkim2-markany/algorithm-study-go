# 차선 최소 신장 트리 - 해설

## 접근 방식

1. 먼저 크루스칼 알고리즘으로 MST를 구한다.
2. MST에 포함되지 않은 간선 (u, v, w)를 하나 추가하면 사이클이 생긴다.
3. 이 사이클에서 MST에 속한 간선 중 가중치가 최대인 간선을 제거하면 새로운 신장 트리가 된다.
4. 모든 비-MST 간선에 대해 이 과정을 수행하고, 가중치 합이 가장 작은 것이 차선 MST이다.

## 핵심 아이디어

차선 MST는 MST와 정확히 하나의 간선만 다르다는 성질을 이용한다.

MST에 포함되지 않은 간선 (u, v, w)를 추가하면 MST에 사이클이 생긴다. 이 사이클에서 가중치가 가장 큰 MST 간선을 제거하면 비용 증가가 최소인 새로운 신장 트리를 얻는다. 비용 변화는 `w - max_on_path(u, v)`이다.

MST 위에서 두 정점 사이 경로의 최대 가중치를 구하기 위해 DFS를 사용한다. 각 비-MST 간선마다 DFS를 수행하므로 전체 시간 복잡도는 O(M × N)이다.

예제 1에서 MST는 (1,2,1), (2,3,2), (3,4,3)이고 총 가중치는 6이다. 비-MST 간선 (1,3,3)을 추가하면 1-2-3 경로의 최대 가중치는 2이므로 비용 변화는 3-2=1, 새 가중치는 7이다. 비-MST 간선 (2,4,5)를 추가하면 2-3-4 경로의 최대 가중치는 3이므로 비용 변화는 5-3=2, 새 가중치는 8이다. 따라서 차선 MST의 가중치는 7이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M log M + M × N) |
| 공간 복잡도 | O(N + M) |

크루스칼 알고리즘에 O(M log M), 각 비-MST 간선에 대한 경로 최대값 계산에 O(N)이 소요되어 전체 O(M × N)이다. N ≤ 1,000이고 M ≤ 100,000이므로 시간 내에 충분히 처리 가능하다.

## 대안적 접근

- LCA(최소 공통 조상)와 희소 테이블을 사용하면 경로 최대값 쿼리를 O(log N)에 처리할 수 있어 전체 O(M log N)으로 개선된다. 하지만 구현이 복잡하다.
- N이 작은 경우(N ≤ 1,000) DFS 기반 접근이 구현이 간단하고 충분히 빠르다.
