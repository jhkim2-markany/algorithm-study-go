# 최소 비용 연결 - 해설

## 접근 방식

1. 모든 도시를 최소 비용으로 연결하는 문제는 최소 신장 트리(MST)를 구하는 전형적인 문제이다.
2. 크루스칼 알고리즘을 사용하여 간선을 가중치 오름차순으로 정렬한다.
3. Union-Find 자료구조를 사용하여 사이클 여부를 판별하며 간선을 선택한다.
4. N-1개의 간선을 선택하면 모든 정점이 연결된 MST가 완성된다.

## 핵심 아이디어

크루스칼 알고리즘의 기본 적용 문제이다. 핵심은 "가중치가 작은 간선부터 선택하되, 사이클이 생기지 않는 간선만 선택"하는 것이다.

Union-Find에 경로 압축(Path Compression)과 랭크 기반 합치기(Union by Rank)를 적용하면 Find와 Union 연산이 거의 O(1)에 수행된다. 따라서 전체 시간 복잡도는 간선 정렬에 의해 결정된다.

예제 1에서 간선을 가중치순으로 정렬하면 (1,2,1), (2,3,2), (3,4,3), (1,3,4), (2,4,5)이다. 처음 세 간선을 선택하면 모든 도시가 연결되며 총 비용은 1+2+3=6이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M log M) |
| 공간 복잡도 | O(N + M) |

간선 정렬에 O(M log M), Union-Find 연산에 O(M × α(N)) ≈ O(M)이 소요된다. 전체 시간 복잡도는 정렬이 지배적이므로 O(M log M)이다.

## 대안적 접근

- Prim 알고리즘을 사용할 수도 있다. 우선순위 큐 기반 Prim은 O((V+E) log V)이며, 밀집 그래프에서는 배열 기반 Prim(O(V²))이 더 효율적일 수 있다.
- 이 문제는 간선 수가 많지 않으므로 크루스칼이 구현이 간단하고 효율적이다.
