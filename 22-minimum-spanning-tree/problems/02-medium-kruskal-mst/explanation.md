# 크루스칼 MST - 해설

## 접근 방식

1. 크루스칼 알고리즘의 핵심은 간선을 가중치 오름차순으로 정렬한 뒤, 사이클을 형성하지 않는 간선을 순서대로 선택하는 것이다.
2. 가중치가 같은 간선이 여러 개 있을 때 정렬 기준을 추가하여 결정적인 출력을 보장한다.
3. Union-Find 자료구조로 두 정점이 같은 집합에 속하는지 빠르게 판별한다.
4. MST에 포함된 간선을 선택 순서대로 저장하여 출력한다.

## 핵심 아이디어

기본 크루스칼 알고리즘에 간선 출력 기능을 추가한 문제이다. 핵심은 두 가지이다:

첫째, 간선 정렬 시 가중치가 같은 경우의 타이브레이킹 규칙을 정확히 구현해야 한다. 가중치 → 첫 번째 정점 → 두 번째 정점 순으로 오름차순 정렬한다.

둘째, 선택된 간선을 별도 배열에 저장하여 선택 순서대로 출력한다. 크루스칼 알고리즘에서 간선은 가중치가 작은 것부터 선택되므로, 저장 순서가 곧 가중치 오름차순이다.

예제 1에서 정렬된 간선은 (1,2,1), (2,3,2), (3,4,2), (2,4,3), (1,3,4), (4,5,5), (3,5,6)이다. 순서대로 선택하면 (1,2,1), (2,3,2), (3,4,2)까지 선택된다. (2,4,3)은 2와 4가 이미 연결되어 있으므로 건너뛴다. (1,3,4)도 마찬가지로 건너뛰고, (4,5,5)를 선택하면 MST가 완성된다. 총 가중치는 1+2+2+5=10이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M log M) |
| 공간 복잡도 | O(N + M) |

간선 정렬에 O(M log M), Union-Find 연산에 O(M × α(N))이 소요된다. 정렬이 지배적이므로 전체 시간 복잡도는 O(M log M)이다. 간선 배열과 Union-Find 배열에 O(N + M) 공간이 필요하다.

## 대안적 접근

- Prim 알고리즘으로도 MST를 구할 수 있지만, 간선을 가중치 순서대로 출력하려면 별도의 정렬이 필요하다.
- 크루스칼 알고리즘은 간선 중심으로 동작하므로 간선 출력이 자연스럽다.
