# 색칠된 경로 쿼리 - 해설

## 접근 방식

1. 센트로이드 분할을 수행한다
2. 각 센트로이드에서 서브트리 내 모든 노드의 거리와 색상 정보를 수집한다
3. 같은 색상인 노드 쌍에 대해 거리 합별로 쌍의 수를 센다
4. 포함-배제로 같은 서브트리 내의 쌍을 빼준다
5. 전처리 결과를 `ans[d]` 배열에 저장하여 각 쿼리에 O(1)로 답한다

## 핵심 아이디어

기본 센트로이드 분할에 "같은 색상" 조건이 추가된 문제이다. 센트로이드를 지나는 경로에서 양 끝 노드의 색상이 같은 쌍만 세야 한다.

```
센트로이드 c를 지나는 경로 (u, v):
  조건 1: dist(u, c) + dist(c, v) = K
  조건 2: color[u] = color[v]
```

같은 색상의 노드끼리만 쌍을 이루므로, 색상별로 노드를 그룹화한 뒤 각 그룹 내에서 거리 합이 K인 쌍을 센다.

포함-배제 원리:
- 전체 노드에서 같은 색상 + 거리 합 d인 쌍 수를 구한다 (포함)
- 각 서브트리 내에서 같은 색상 + 거리 합 d인 쌍 수를 빼준다 (배제)

쿼리가 여러 개이므로, 모든 거리에 대해 한 번에 전처리하고 `ans[d]` 배열에 저장한다. 이후 각 쿼리는 O(1)에 답할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (전처리) | O(N² log N) (최악) |
| 시간 복잡도 (쿼리) | O(1) per query |
| 공간 복잡도 | O(N) |

최악의 경우 같은 색상의 노드가 많으면 쌍 열거에 O(N²)이 소요될 수 있다. 그러나 센트로이드 분할의 재귀 구조 덕분에 실제로는 훨씬 빠르게 동작한다. 색상의 종류가 많으면 각 그룹의 크기가 작아져 효율적이다.

## 대안적 접근

색상 수가 적은 경우, 각 색상별로 별도의 센트로이드 분할을 수행하는 방법도 있다. 또는 센트로이드 트리를 구축한 뒤 각 쿼리마다 센트로이드 트리를 순회하는 온라인 방식도 가능하다.
