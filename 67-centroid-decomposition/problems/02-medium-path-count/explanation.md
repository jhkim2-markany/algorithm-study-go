# 특정 거리 경로 수 세기 - 해설

## 접근 방식

1. 센트로이드 분할을 수행한다
2. 각 센트로이드에서 서브트리 내 모든 노드까지의 가중치 거리를 수집한다
3. 해시맵을 이용하여 합이 정확히 K인 쌍을 센다
4. 같은 서브트리 내의 쌍은 포함-배제로 빼준다
5. 거리가 K를 초과하면 탐색을 중단하는 가지치기를 적용한다

## 핵심 아이디어

문제 01과 유사하지만, "K 이하"가 아닌 "정확히 K"를 구해야 하므로 투 포인터 대신 해시맵을 사용한다.

```
센트로이드 c를 지나는 경로:
  dist(u, v) = dist(u, c) + dist(c, v) = K
  → dist(c, v) = K - dist(u, c)
```

거리 배열을 순회하면서 해시맵에 이전까지의 거리 빈도를 저장하고, 현재 거리 d에 대해 `K - d`의 빈도를 더하면 정확히 K인 쌍의 수를 구할 수 있다.

포함-배제 원리:
- 전체 거리 배열에서 합이 K인 쌍 수 (포함)
- 각 서브트리의 거리 배열에서 합이 K인 쌍 수를 빼기 (배제)

가중치가 양수이므로 거리가 K를 초과하면 더 이상 탐색할 필요가 없다. 이 가지치기로 실제 수행 시간을 크게 줄일 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 재귀 레벨에서 해시맵 연산이 O(N)이고, 재귀 깊이가 O(log N)이므로 전체 O(N log N)이다. 가지치기를 적용하면 실제로는 더 빠르다.

## 대안적 접근

거리 배열을 정렬한 뒤 투 포인터로 "정확히 K"인 쌍을 셀 수도 있다. 이 경우 시간 복잡도는 O(N log² N)이지만, 구현이 약간 더 복잡하다.
