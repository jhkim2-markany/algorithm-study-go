# 센트로이드 분할 (Centroid Decomposition)

## 개념

센트로이드 분할(Centroid Decomposition)이란 트리에서 **센트로이드(무게 중심)**를 반복적으로 찾아 제거하면서 분할 정복을 수행하는 기법이다.

트리의 센트로이드란, 해당 노드를 제거했을 때 생기는 모든 서브트리의 크기가 전체 노드 수의 절반 이하가 되는 노드를 말한다. 모든 트리에는 반드시 하나 이상의 센트로이드가 존재한다.

핵심 아이디어는 **센트로이드를 기준으로 트리를 분할하면 재귀 깊이가 O(log N)으로 보장된다**는 것이다. 센트로이드를 제거하면 각 서브트리의 크기가 원래의 절반 이하이므로, 분할 정복의 깊이가 log₂N을 넘지 않는다.

대표적인 활용:

- **경로 조건 쌍 세기**: 트리에서 거리가 K 이하(또는 정확히 K)인 노드 쌍의 수를 센다
- **경로 위 쿼리**: 특정 조건을 만족하는 경로를 효율적으로 탐색한다
- **센트로이드 트리 구축**: 센트로이드 분할 과정에서 만들어지는 새로운 트리 구조를 활용한다
- **오프라인/온라인 경로 쿼리**: 센트로이드 트리를 이용한 다양한 쿼리 처리

## 선수 학습

센트로이드 분할을 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [13-tree](../13-tree/) - 트리의 기본 개념, 순회, 부모-자식 관계, 서브트리 크기 계산
- [18-divide-and-conquer](../18-divide-and-conquer/) - 분할 정복의 기본 원리, 재귀적 문제 분할

## 동작 원리

### 1. 센트로이드 찾기

트리의 센트로이드는 다음 조건을 만족하는 노드 c이다:

```
모든 서브트리 크기 ≤ N/2  (c를 제거했을 때 생기는 각 서브트리)
```

센트로이드를 찾는 과정:
1. 임의의 노드를 루트로 잡고 DFS로 각 노드의 서브트리 크기 `sz[v]`를 계산한다
2. 루트에서 시작하여, 서브트리 크기가 N/2를 초과하는 자식 방향으로 이동한다
3. 더 이상 이동할 수 없는 노드가 센트로이드이다

또는 단순하게 모든 노드를 순회하면서 조건을 확인한다:

```
센트로이드 조건:
  max(sz[child1], sz[child2], ..., N - sz[v]) ≤ N/2
  (N - sz[v]는 v의 부모 방향 서브트리 크기)
```

### 2. 센트로이드 분할 과정

1. 현재 트리에서 센트로이드 c를 찾는다
2. c를 "제거"한다 (방문 표시)
3. c를 제거하면 여러 개의 서브트리가 생긴다
4. 각 서브트리에 대해 재귀적으로 1~3을 반복한다
5. 재귀 과정에서 센트로이드 트리가 구축된다

### 3. 센트로이드 트리

분할 과정에서 만들어지는 새로운 트리이다:
- 첫 번째로 찾은 센트로이드가 루트가 된다
- 각 서브트리에서 찾은 센트로이드가 부모 센트로이드의 자식이 된다
- 센트로이드 트리의 높이는 O(log N)이다

센트로이드 트리의 핵심 성질:
- 원래 트리에서 두 노드 u, v의 경로는 반드시 센트로이드 트리에서 u와 v의 LCA를 지난다
- 센트로이드 트리의 높이가 O(log N)이므로, 각 노드에서 루트까지 올라가며 처리하면 O(log N)번의 연산으로 충분하다

### 4. 경로 문제 해결 패턴

센트로이드 분할로 경로 문제를 해결하는 일반적인 패턴:

1. 센트로이드 c를 찾는다
2. c를 지나는 모든 경로를 처리한다 (c에서 각 서브트리로의 거리를 계산)
3. c를 제거하고 각 서브트리에 대해 재귀한다
4. "c를 지나는 경로" = "c에서 서브트리 A의 노드까지" + "c에서 서브트리 B의 노드까지"

이때 같은 서브트리 내의 두 노드를 잘못 합치지 않도록 주의해야 한다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 센트로이드 찾기 (1회) | O(N) |
| 센트로이드 분할 전체 | O(N log N) |
| 센트로이드 트리 높이 | O(log N) |
| 공간 복잡도 | O(N) |

각 재귀 레벨에서 모든 노드를 한 번씩 방문하고, 재귀 깊이가 O(log N)이므로 전체 O(N log N)이다.

## 적합한 문제 유형

- 트리에서 거리가 K 이하(또는 정확히 K)인 노드 쌍의 수를 세는 문제
- 트리에서 특정 조건을 만족하는 경로의 수를 세는 문제
- 트리 위의 경로에 대한 오프라인/온라인 쿼리 문제
- 트리에서 가장 가까운 특정 색상 노드까지의 거리를 구하는 문제
- 트리 경로 위의 값의 합, XOR, GCD 등 조건을 만족하는 쌍을 세는 문제
- 트리에서 분할 정복이 필요한 문제 전반

## 단계별 추적 (Trace)

### 예시: 센트로이드 분할 과정

다음 트리에서 센트로이드 분할을 수행한다.

```
원래 트리 (N=7):

        1
       / \
      2   3
     / \   \
    4   5   6
        |
        7
```

간선: (1,2), (1,3), (2,4), (2,5), (3,6), (5,7)

**1단계: 전체 트리에서 센트로이드 찾기 (N=7)**

```
서브트리 크기 (루트=1):
  sz[4]=1, sz[7]=1, sz[5]=2, sz[2]=4, sz[6]=1, sz[3]=2, sz[1]=7

각 노드를 제거했을 때 최대 서브트리 크기:
  노드 1: max(sz[2], sz[3]) = max(4, 2) = 4 > 7/2=3.5 → ✗
  노드 2: max(sz[4], sz[5], 7-sz[2]) = max(1, 2, 3) = 3 ≤ 3.5 → ✓ 센트로이드!
```

센트로이드 = 노드 2

```
센트로이드 트리: [2]

노드 2를 제거하면 3개의 서브트리가 생긴다:
  서브트리 A: {4}         (노드 4)
  서브트리 B: {5, 7}      (노드 5, 7)
  서브트리 C: {1, 3, 6}   (노드 1, 3, 6)
```

**2단계: 서브트리 A에서 센트로이드 찾기 (N=1)**

```
서브트리 A: {4}
  노드가 1개이므로 센트로이드 = 노드 4

센트로이드 트리: 2 → 4
```

**3단계: 서브트리 B에서 센트로이드 찾기 (N=2)**

```
서브트리 B:
    5
    |
    7

sz[7]=1, sz[5]=2
  노드 5: max(sz[7], 2-sz[5]) = max(1, 0) = 1 ≤ 1 → ✓ 센트로이드!

센트로이드 = 노드 5

센트로이드 트리: 2 → 4, 2 → 5

노드 5를 제거하면:
  서브트리: {7}
  센트로이드 = 노드 7

센트로이드 트리: 2 → 4, 2 → 5 → 7
```

**4단계: 서브트리 C에서 센트로이드 찾기 (N=3)**

```
서브트리 C:
    1
    |
    3
    |
    6

sz[6]=1, sz[3]=2, sz[1]=3
  노드 1: max(sz[3], 3-sz[1]) = max(2, 0) = 2 > 1.5 → ✗
  노드 3: max(sz[6], 3-sz[3]) = max(1, 1) = 1 ≤ 1.5 → ✓ 센트로이드!

센트로이드 = 노드 3

센트로이드 트리: 2 → {4, 5, 3}, 5 → 7

노드 3을 제거하면:
  서브트리: {1}, {6}
  각각 센트로이드 = 노드 1, 노드 6

센트로이드 트리: 2 → {4, 5, 3}, 5 → 7, 3 → {1, 6}
```

**최종 센트로이드 트리:**

```
        2           ← 루트 (전체 트리의 센트로이드)
      / | \
     4  5   3       ← 각 서브트리의 센트로이드
        |  / \
        7 1   6     ← 재귀적으로 찾은 센트로이드
```

센트로이드 트리의 높이 = 3 (≤ log₂7 ≈ 2.8 → 올림 3)

**성질 확인:**
- 원래 트리에서 노드 4와 노드 6의 경로: 4-2-1-3-6
- 센트로이드 트리에서 4와 6의 LCA: 노드 2
- 노드 2는 원래 경로 4-2-1-3-6 위에 있다 ✓

## 실전 팁

### 활용 노하우

- 센트로이드 분할의 핵심 패턴은 "센트로이드를 지나는 경로를 처리하고, 나머지는 재귀"이다
- 경로 쌍을 셀 때, 센트로이드에서 각 서브트리까지의 거리 배열을 구한 뒤 정렬 + 투 포인터 또는 해시맵으로 합산한다
- 같은 서브트리 내의 두 노드를 잘못 합치는 것을 방지하려면, 서브트리별로 따로 처리한 뒤 빼는 "포함-배제" 기법을 사용한다
- 센트로이드 트리를 미리 구축해두면, 온라인 쿼리에서 각 노드에서 센트로이드 트리의 루트까지 올라가며 O(log N)번 처리할 수 있다
- `removed[]` 배열로 이미 센트로이드로 선택된 노드를 표시하여 재귀 시 해당 노드를 건너뛴다

### 자주 하는 실수

- 서브트리 크기를 계산할 때 이미 제거된 노드를 포함하는 실수. `removed[]` 배열을 반드시 확인하자
- 센트로이드를 찾을 때 전체 크기 N 대신 현재 서브트리 크기를 사용해야 하는데 혼동하는 실수. 재귀 호출 시 현재 서브트리의 크기를 정확히 전달하자
- 같은 서브트리 내의 경로를 중복 계산하는 실수. 포함-배제 원리를 적용하거나, 서브트리별로 분리하여 처리하자
- 센트로이드 트리와 원래 트리를 혼동하는 실수. 두 트리의 인접 리스트를 별도로 관리하자

### 엣지 케이스

- N = 1: 노드가 하나뿐인 트리. 센트로이드 = 유일한 노드, 경로 쌍 = 0
- N = 2: 두 노드가 하나의 간선으로 연결. 어느 노드든 센트로이드가 될 수 있다
- 일직선 트리 (경로 그래프): 1-2-3-...-N 형태. 센트로이드는 중앙 노드, 센트로이드 트리 높이 = O(log N)
- 스타 그래프: 중심 노드가 항상 센트로이드. 센트로이드 트리 높이 = 2
- 모든 간선 가중치가 0인 경우: 거리 조건 문제에서 모든 쌍이 조건을 만족할 수 있다

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 센트로이드 분할 | 트리 분할 정복, O(N log N) | 경로 조건 쌍 세기, 경로 쿼리 |
| HLD (Heavy-Light Decomposition) | 경로를 O(log N)개 구간으로 분할 | 경로 질의/갱신 반복 (세그먼트 트리 결합) |
| 오일러 투어 + 세그먼트 트리 | 서브트리를 구간으로 변환 | 서브트리 질의/갱신 반복 |
| 트리 DP | 서브트리 단위 DP, O(N) | 한 번의 전처리로 결과 계산 |
| LCA + 경로 쿼리 | 두 노드 간 경로, O(log N) | 특정 두 노드 간 경로 질의 |

- 경로 조건 쌍 세기 → 센트로이드 분할
- 경로 질의/갱신 반복 → HLD
- 서브트리 질의/갱신 반복 → 오일러 투어 + 세그먼트 트리
- 한 번의 전처리 → 트리 DP
- 두 노드 간 경로 → LCA
