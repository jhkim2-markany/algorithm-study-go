# 아나그램 그룹화 - 해설

## 접근 방식

1. 각 문자열을 정렬하여 아나그램의 대표 키(canonical key)를 생성한다
2. 해시맵에 대표 키를 기준으로 문자열들을 그룹화한다
3. 각 그룹 내 문자열을 사전순으로 정렬한다
4. 그룹들을 첫 번째 문자열 기준 사전순으로 정렬하여 출력한다

## 핵심 아이디어

아나그램의 핵심 성질은 "같은 문자를 같은 개수만큼 포함한다"는 것이다. 따라서 문자열의 문자를 정렬하면 아나그램끼리 동일한 문자열이 된다. 예를 들어 "eat", "tea", "ate"는 모두 정렬하면 "aet"가 된다.

이 정렬된 문자열을 해시맵의 키로 사용하면, 같은 키를 가진 문자열들이 자연스럽게 하나의 그룹으로 묶인다. Go의 `map[string][]string`을 활용하면 간결하게 구현할 수 있다.

대표 키 생성 방법으로 정렬 대신 문자 빈도 배열을 사용할 수도 있다. 각 문자의 등장 횟수를 26자리 배열로 만들어 문자열로 변환하면 정렬 없이 O(L) 시간에 키를 생성할 수 있다 (L: 문자열 길이).

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × L log L) |
| 공간 복잡도 | O(N × L) |

N은 문자열 개수, L은 문자열의 최대 길이이다. 각 문자열을 정렬하는 데 O(L log L), N개의 문자열을 처리하므로 전체 O(N × L log L)이다. 해시맵에 모든 문자열을 저장하므로 공간 복잡도는 O(N × L)이다.

## 대안적 접근

- **빈도 배열 키**: 정렬 대신 각 문자의 빈도를 26자리 배열로 만들어 키로 사용하면 O(N × L)에 해결할 수 있다. 문자열 길이가 길 때 정렬 방식보다 효율적이다.
- **소수 곱 해싱**: 각 알파벳에 고유한 소수를 대응시키고, 문자열의 각 문자에 해당하는 소수를 모두 곱한 값을 키로 사용하는 방법이 있다. 해시 충돌 가능성이 있어 주의가 필요하다.
