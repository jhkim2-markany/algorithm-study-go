# 빈도수 세기 - 해설

## 접근 방식

1. 해시맵을 이용하여 각 정수의 등장 횟수를 센다
2. 해시맵의 키-값 쌍을 슬라이스로 변환한다
3. 등장 횟수 내림차순, 같으면 값 오름차순으로 정렬한다
4. 정렬된 결과를 출력한다

## 핵심 아이디어

해시맵(Go의 `map`)을 사용하면 각 원소의 등장 횟수를 O(1)에 갱신할 수 있다. 배열을 한 번 순회하면서 `map[값]++`로 빈도를 누적하면 전체 O(N)에 빈도수 계산이 완료된다.

정렬 조건이 두 가지(빈도수 내림차순, 값 오름차순)이므로 `sort.Slice`에 커스텀 비교 함수를 전달하여 처리한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + K log K) |
| 공간 복잡도 | O(K) |

N은 배열 크기, K는 서로 다른 원소의 개수이다. 배열 순회에 O(N), 정렬에 O(K log K)가 소요된다. K ≤ N이므로 전체 시간 복잡도는 O(N + K log K)이다. 해시맵에 최대 K개의 항목이 저장되므로 공간 복잡도는 O(K)이다.

## 대안적 접근

- **정렬 후 카운팅**: 배열을 먼저 정렬하면 같은 값이 연속으로 나타나므로 순회하며 빈도를 셀 수 있다. 시간 복잡도는 O(N log N)으로 해시맵 방식보다 느리다.
- **카운팅 배열**: 값의 범위가 제한적이면 배열 인덱스를 직접 사용하여 빈도를 셀 수 있다. 이 문제에서는 음수가 포함되고 범위가 넓어 해시맵이 더 적합하다.
