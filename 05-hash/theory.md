# 해시

## 개념

해시(Hash)는 해시 함수를 사용하여 키(key)를 고정 크기의 값(해시 값)으로 변환하고, 이를 인덱스로 활용하여 데이터를 저장·조회하는 자료구조이다. 대표적인 구현체로 해시 테이블(Hash Table), 해시 맵(Hash Map), 해시 셋(Hash Set)이 있다.

해시의 핵심은 임의의 크기를 가진 데이터를 고정 크기의 값으로 매핑하는 해시 함수에 있다. 좋은 해시 함수는 입력이 고르게 분포되도록 하여 충돌을 최소화한다.

### 해시 함수

해시 함수는 임의의 길이의 입력을 받아 고정 길이의 출력(해시 값)을 생성하는 함수이다. 이상적인 해시 함수의 조건은 다음과 같다:

- **결정성**: 같은 입력에 대해 항상 같은 출력을 반환한다
- **균일 분포**: 출력 값이 해시 테이블 전체에 고르게 분포한다
- **빠른 계산**: 해시 값을 상수 시간에 계산할 수 있다

대표적인 해시 함수 기법:
- **나눗셈법(Division Method)**: `h(k) = k mod m` (m은 테이블 크기)
- **곱셈법(Multiplication Method)**: `h(k) = floor(m * (k * A mod 1))` (A는 0~1 사이 상수)
- **문자열 해시**: 각 문자의 ASCII 값에 소수를 곱하여 누적 합산

### 주요 연산

- **Put/Insert**: 키-값 쌍을 해시 테이블에 저장한다
- **Get/Search**: 키를 이용하여 값을 조회한다
- **Delete**: 키에 해당하는 항목을 삭제한다
- **Contains**: 특정 키가 존재하는지 확인한다

## 동작 원리

### 해시 테이블의 동작

1. 키를 해시 함수에 입력하여 해시 값을 계산한다
2. 해시 값을 테이블 크기로 나눈 나머지를 인덱스로 사용한다
3. 해당 인덱스 위치에 키-값 쌍을 저장한다
4. 조회 시 동일한 과정으로 인덱스를 계산하여 값을 찾는다

### 충돌 해결 (Collision Resolution)

서로 다른 키가 같은 해시 값을 가지는 경우를 충돌(Collision)이라 한다. 충돌 해결 방법은 크게 두 가지이다:

**1. 체이닝(Chaining)**
- 같은 인덱스에 해당하는 항목들을 연결 리스트로 연결한다
- 충돌이 발생해도 같은 버킷에 여러 항목을 저장할 수 있다
- 최악의 경우 한 버킷에 모든 항목이 몰리면 O(N)이 된다

**2. 개방 주소법(Open Addressing)**
- 충돌 시 다른 빈 슬롯을 찾아 저장한다
- 선형 탐사(Linear Probing): 다음 슬롯을 순차적으로 탐색
- 이차 탐사(Quadratic Probing): 제곱수만큼 건너뛰며 탐색
- 이중 해싱(Double Hashing): 두 번째 해시 함수로 탐색 간격 결정

### Golang에서의 해시

Go에서는 내장 `map` 타입이 해시 맵을 제공한다:
- `map[KeyType]ValueType`으로 선언한다
- `make(map[KeyType]ValueType)`으로 초기화한다
- 키의 존재 여부는 `value, ok := m[key]` 패턴으로 확인한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (삽입, 평균) | O(1) |
| 시간 복잡도 (조회, 평균) | O(1) |
| 시간 복잡도 (삭제, 평균) | O(1) |
| 시간 복잡도 (최악) | O(N) |
| 공간 복잡도 | O(N) |

평균적으로 모든 연산이 O(1)에 수행된다. 최악의 경우(모든 키가 같은 해시 값을 가질 때) O(N)이 되지만, 좋은 해시 함수를 사용하면 실질적으로 O(1)에 가깝다. N개의 키-값 쌍을 저장하므로 공간 복잡도는 O(N)이다.

## 적합한 문제 유형

- 원소의 빈도수 세기, 최빈값 찾기
- 배열에서 두 수의 합이 특정 값이 되는 쌍 찾기 (Two Sum)
- 중복 원소 검사, 유일한 원소 찾기
- 문자열에서 반복되지 않는 첫 번째 문자 찾기
- 아나그램 판별 및 그룹화
- 부분 문자열에서 중복 없는 가장 긴 길이 찾기
- 캐시 구현 (LRU Cache 등)
- 그래프의 인접 리스트 표현


## 단계별 추적 (Trace)

### 예시: 빈도수 세기

입력: `arr = ["apple", "banana", "apple", "cherry", "banana", "apple"]`

```
해시맵을 이용한 빈도수 카운팅:

단계 1: "apple"  → map에 없음 → map["apple"] = 1
  map: {"apple": 1}

단계 2: "banana" → map에 없음 → map["banana"] = 1
  map: {"apple": 1, "banana": 1}

단계 3: "apple"  → map에 있음 → map["apple"] = 2
  map: {"apple": 2, "banana": 1}

단계 4: "cherry" → map에 없음 → map["cherry"] = 1
  map: {"apple": 2, "banana": 1, "cherry": 1}

단계 5: "banana" → map에 있음 → map["banana"] = 2
  map: {"apple": 2, "banana": 2, "cherry": 1}

단계 6: "apple"  → map에 있음 → map["apple"] = 3
  map: {"apple": 3, "banana": 2, "cherry": 1}

최종 결과: apple=3, banana=2, cherry=1
```

### 예시: Two Sum (합이 K인 쌍 찾기)

입력: `arr = [2, 7, 11, 15]`, `target = 9`

```
해시맵으로 O(N) 풀이:

단계 1: num=2, 필요한 값=9-2=7
  map에 7 없음 → map[2] = 0 (인덱스 저장)
  map: {2: 0}

단계 2: num=7, 필요한 값=9-7=2
  map에 2 있음! → 인덱스 (map[2], 현재) = (0, 1) ✓ 정답!
  map: {2: 0}

  ┌──────────────────────────────────┐
  │  arr:  [2,  7,  11,  15]        │
  │         ↑   ↑                    │
  │       idx=0 idx=1                │
  │       2 + 7 = 9 = target  ✓     │
  └──────────────────────────────────┘

결과: (0, 1)
브루트포스 O(N²) 대비 O(N)으로 해결!
```

## 실전 팁

### 활용 노하우

- **존재 여부 확인 패턴**: `if _, ok := m[key]; ok { ... }` 패턴은 Go 해시맵의 핵심이다. 반드시 숙지하자
- **기본값 활용**: Go map은 존재하지 않는 키에 접근하면 제로값을 반환한다. 빈도수 세기에서 `m[key]++`만으로 초기화 없이 카운팅이 가능하다
- **해시셋 대용**: Go에는 Set이 없으므로 `map[Type]bool` 또는 `map[Type]struct{}`를 사용한다. `struct{}`는 메모리를 차지하지 않아 더 효율적이다
- **문자열 키 생성**: 아나그램 그룹화 시 문자열을 정렬하여 키로 사용하거나, 문자 빈도 배열을 문자열로 변환하여 키로 사용한다

### 자주 하는 실수

- **map 초기화 누락**: `var m map[string]int`로 선언만 하면 nil map이다. 반드시 `m = make(map[string]int)` 또는 `m := map[string]int{}`로 초기화해야 한다. nil map에 쓰기를 시도하면 런타임 패닉이 발생한다
- **map 순회 순서**: Go의 map은 순회 순서가 보장되지 않는다. 정렬된 순서가 필요하면 키를 별도로 정렬해야 한다
- **슬라이스를 키로 사용**: Go에서 슬라이스는 map의 키로 사용할 수 없다. 배열(`[N]int`)이나 문자열로 변환하여 사용하자
- **동시 접근**: Go map은 동시 읽기/쓰기에 안전하지 않다. 고루틴에서 사용할 때는 `sync.Map`이나 뮤텍스를 사용하자

### 엣지 케이스 주의사항

- **빈 배열**: 빈 배열에서 빈도수를 세면 빈 map이 반환된다
- **키가 0인 경우**: 정수 map에서 `m[0]`은 키 0의 값과 존재하지 않는 키의 기본값(0)이 구분되지 않는다. 반드시 `val, ok` 패턴을 사용하자
- **해시 충돌**: Go 내장 map은 충돌을 자동 처리하지만, 직접 해시 테이블을 구현할 때는 충돌 해결 로직이 필수이다
- **대소문자 구분**: 문자열 키에서 "Apple"과 "apple"은 다른 키이다. 필요시 `strings.ToLower()`로 통일하자

## 관련 알고리즘 비교

| 자료구조/기법 | 조회 시간 | 삽입 시간 | 순서 보장 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| 해시맵 (map) | O(1) 평균 | O(1) 평균 | 없음 | 빈도수 세기, 존재 여부 확인, 키-값 매핑 |
| 정렬 + 이진 탐색 | O(log N) | O(N) | 있음 | 정렬된 데이터에서 반복 조회 |
| 배열 인덱싱 | O(1) | O(1) | 있음 | 키가 작은 범위의 정수인 경우 |
| 트라이 (Trie) | O(L) | O(L) | 사전순 | 문자열 접두사 검색, 자동완성 |
| BST (이진 탐색 트리) | O(log N) | O(log N) | 있음 | 순서가 필요한 동적 데이터 |

**선택 기준**:
- 단순 존재 여부/빈도수 → 해시맵
- 키가 0~10^6 범위의 정수 → 배열 인덱싱 (더 빠름)
- 정렬된 순서가 필요 → 정렬 + 이진 탐색 또는 BST
- 문자열 접두사 검색 → 트라이
- 범위 쿼리가 필요 → BST 또는 세그먼트 트리
