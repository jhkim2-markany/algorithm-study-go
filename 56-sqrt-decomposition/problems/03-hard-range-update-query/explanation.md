# 구간 갱신과 구간 합 쿼리 - 해설

## 접근 방식

1. 배열을 √N 크기의 블록으로 분할하고, 각 블록의 합과 lazy 값을 관리한다
2. 구간 갱신 [l, r]에 v를 더할 때:
   - 완전히 포함되는 블록은 lazy 값에 v를 더한다 (O(1))
   - 양 끝의 부분 블록은 개별 원소에 v를 더하고 블록 합을 갱신한다 (O(√N))
3. 구간 쿼리 [l, r]의 합을 구할 때:
   - 완전히 포함되는 블록은 블록합 + lazy × 블록크기를 사용한다
   - 양 끝의 부분 블록은 개별 원소 + lazy를 합산한다

## 핵심 아이디어

세그먼트 트리의 Lazy Propagation과 유사한 개념을 블록 단위로 적용한다. 완전히 포함되는 블록에 대한 갱신은 lazy 값만 변경하여 O(1)에 처리하고, 쿼리 시 lazy 값을 반영하여 정확한 합을 계산한다.

블록의 실제 합은 `blockSum[i] + lazy[i] × blockSize[i]`이다. blockSum은 개별 원소의 합이고, lazy는 블록 전체에 균일하게 더해진 값이다.

부분 블록을 처리할 때는 개별 원소에 직접 접근하므로, 해당 원소의 실제 값은 `a[i] + lazy[i/b]`이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (구간 갱신) | O(√N) |
| 시간 복잡도 (구간 쿼리) | O(√N) |
| 시간 복잡도 (전체) | O(N + Q√N) |
| 공간 복잡도 | O(N) |

양 끝 부분 블록 처리에 O(√N), 중간 블록 순회에 O(√N)이므로 각 연산은 O(√N)이다.

## 대안적 접근

세그먼트 트리 + Lazy Propagation을 사용하면 구간 갱신 O(log N), 구간 쿼리 O(log N)으로 더 빠르게 처리할 수 있다. BIT(펜윅 트리)를 두 개 사용하는 방법으로도 구간 갱신 + 구간 쿼리를 O(log N)에 처리할 수 있다. 다만 Sqrt Decomposition + Lazy 블록은 구현이 직관적이고, 더 복잡한 연산(예: 구간 대입)에도 쉽게 확장할 수 있다.
