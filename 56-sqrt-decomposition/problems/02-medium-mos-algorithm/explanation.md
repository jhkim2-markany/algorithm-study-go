# 구간 서로 다른 수의 개수 (Mo's Algorithm) - 해설

## 접근 방식

1. 모든 쿼리를 미리 받아 Mo's Algorithm 기준으로 정렬한다
2. 현재 구간 [curL, curR]을 유지하며, 쿼리 순서대로 구간을 확장/축소한다
3. 원소를 추가할 때 빈도가 0→1이 되면 서로 다른 수의 개수를 증가시킨다
4. 원소를 제거할 때 빈도가 1→0이 되면 서로 다른 수의 개수를 감소시킨다
5. 각 쿼리의 답을 원래 인덱스에 저장하고, 원래 순서대로 출력한다

## 핵심 아이디어

구간 내 서로 다른 수의 개수는 세그먼트 트리로 온라인 처리하기 어렵다. Mo's Algorithm을 사용하면 원소 하나의 추가/제거를 O(1)에 처리할 수 있으므로, 전체 O((N+Q)√N)에 해결할 수 있다.

쿼리 정렬 시 홀짝 최적화를 적용하면 R 포인터의 이동 거리가 줄어들어 상수 배 빨라진다. 짝수 블록에서는 R 오름차순, 홀수 블록에서는 R 내림차순으로 정렬한다.

구간 확장을 먼저 하고 축소를 나중에 하는 순서가 중요하다. 이렇게 해야 구간이 비어있는 상태가 발생하지 않는다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + Q)√N) |
| 공간 복잡도 | O(N + Q) |

L 포인터는 같은 블록 내에서 최대 √N 이동하고, 블록이 바뀔 때 최대 2√N 이동한다. R 포인터는 같은 블록의 쿼리들에 대해 총 O(N) 이동한다. 블록이 √N개이므로 R의 총 이동은 O(N√N)이다.

## 대안적 접근

오프라인 쿼리를 정렬 후 세그먼트 트리와 결합하는 방법도 있다. 쿼리를 R 기준으로 정렬하고, 각 값의 마지막 등장 위치를 세그먼트 트리에 기록하면 O(N log N + Q log N)에 해결할 수 있다. 다만 구현이 더 복잡하다.
