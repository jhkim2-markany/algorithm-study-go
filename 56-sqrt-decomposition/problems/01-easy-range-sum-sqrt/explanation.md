# 구간 합 쿼리 (Sqrt Decomposition) - 해설

## 접근 방식

1. 배열을 √N 크기의 블록으로 분할하고, 각 블록의 합을 미리 계산한다
2. 점 갱신 시 해당 원소와 블록 합만 갱신한다
3. 구간 합 쿼리 시 완전히 포함되는 블록은 블록 합을 사용하고, 양 끝의 부분 블록은 개별 원소를 순회한다

## 핵심 아이디어

배열을 √N 크기의 블록으로 나누면, 임의의 구간 [L, R]에 대해:
- 양 끝의 부분 블록 원소 수: 최대 2 × √N
- 완전히 포함되는 중간 블록 수: 최대 √N

따라서 구간 쿼리를 O(√N)에 처리할 수 있다. 세그먼트 트리의 O(log N)보다 느리지만, 구현이 매우 간단하다는 장점이 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (점 갱신) | O(1) |
| 시간 복잡도 (구간 쿼리) | O(√N) |
| 시간 복잡도 (전체) | O(N + Q√N) |
| 공간 복잡도 | O(N) |

## 대안적 접근

세그먼트 트리를 사용하면 점 갱신 O(log N), 구간 쿼리 O(log N)으로 더 빠르게 처리할 수 있다. BIT(펜윅 트리)도 동일한 복잡도로 가능하며 구현이 더 간결하다. 다만 제곱근 분할법은 구현이 가장 직관적이고, 복잡한 쿼리 함수에도 쉽게 적용할 수 있다는 장점이 있다.
