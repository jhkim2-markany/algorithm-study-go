# 제곱근 분할법 (Sqrt Decomposition)

## 개념

제곱근 분할법(Sqrt Decomposition)은 크기 N인 배열을 약 √N 크기의 블록으로 나누어, 구간 쿼리와 점 갱신(또는 구간 갱신)을 O(√N)에 처리하는 기법이다.

핵심 아이디어:
- 배열을 `B = ⌈√N⌉` 크기의 블록으로 분할한다
- 각 블록에 대해 미리 계산된 요약 정보(합, 최솟값 등)를 유지한다
- 쿼리 시 완전히 포함되는 블록은 요약 정보를 사용하고, 양 끝의 부분 블록만 개별 원소를 순회한다

세그먼트 트리(O(log N))보다 느리지만 구현이 훨씬 간단하며, Mo's Algorithm과 결합하면 오프라인 구간 쿼리를 효율적으로 처리할 수 있다.

## 동작 원리

### 1. 블록 분할 (기본 Sqrt Decomposition)

1. 블록 크기 `B = ⌈√N⌉`을 결정한다
2. 인덱스 `i`가 속하는 블록 번호는 `i / B`이다
3. 각 블록의 요약 정보(예: 블록 합)를 미리 계산한다
4. **점 갱신**: 원소 값을 변경하고, 해당 블록의 요약 정보를 갱신한다 → O(1)
5. **구간 쿼리 [L, R]**:
   - L과 R이 같은 블록에 있으면 개별 원소를 순회한다 → O(√N)
   - 다른 블록에 있으면:
     - L이 속한 블록의 나머지 원소를 순회한다
     - 완전히 포함되는 중간 블록들은 요약 정보를 사용한다
     - R이 속한 블록의 앞부분 원소를 순회한다
   - 양 끝 블록 순회 O(√N) + 중간 블록 O(√N) = 총 O(√N)

### 2. Mo's Algorithm (오프라인 구간 쿼리)

Mo's Algorithm은 제곱근 분할법을 쿼리 정렬에 적용한 오프라인 기법이다.

1. 모든 쿼리 [L, R]을 미리 받는다
2. 쿼리를 `(L / B, R)` 기준으로 정렬한다 (L의 블록 번호 기준 1차 정렬, R 기준 2차 정렬)
3. 현재 구간 [curL, curR]을 유지하며, 쿼리 순서대로 구간을 확장/축소한다
4. 원소를 하나씩 추가(add)/제거(remove)하며 답을 관리한다

**정렬 기준의 핵심:**
- 같은 블록 내의 쿼리들은 R이 오름차순이므로, R 포인터의 이동이 최소화된다
- L 포인터는 같은 블록 내에서 최대 √N만 이동한다
- 전체 포인터 이동 횟수: O((N + Q)√N)

### 3. 구간 갱신 + 구간 쿼리 (Lazy 블록)

구간 갱신이 필요한 경우 각 블록에 lazy 값을 두어 처리한다.

1. **구간 갱신 [L, R]에 값 v 더하기**:
   - 완전히 포함되는 블록: 블록의 lazy 값에 v를 더한다 → O(1)
   - 부분 블록: 개별 원소에 v를 더하고 블록 요약 정보를 재계산한다 → O(√N)
2. **구간 쿼리 [L, R]**:
   - 완전히 포함되는 블록: 블록 합 + lazy × 블록 크기를 사용한다
   - 부분 블록: 개별 원소 + lazy를 합산한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 점 갱신 | O(1) |
| 구간 쿼리 (기본) | O(√N) |
| 구간 갱신 (lazy) | O(√N) |
| Mo's Algorithm (Q개 쿼리) | O((N + Q)√N) |
| 공간 복잡도 | O(N) |

## 적합한 문제 유형

- 점 갱신 + 구간 합/최솟값/최댓값 쿼리 (세그먼트 트리 대안)
- 오프라인 구간 쿼리: 구간 내 서로 다른 수의 개수, 구간 내 특정 조건 만족 원소 수
- 구간 갱신 + 구간 쿼리 (lazy 블록)
- Mo's Algorithm 기반 문제: 쿼리를 정렬하여 처리할 수 있는 경우
- 세그먼트 트리로 풀기 어려운 복잡한 구간 쿼리 (원소 추가/제거가 O(1)인 경우)

## 단계별 추적 (Trace)

### 예시 1: 기본 Sqrt Decomposition (구간 합)

배열: `[1, 3, 5, 2, 7, 6, 3, 8]`, N = 8, B = ⌈√8⌉ = 3

```
블록 분할:
  블록 0: [1, 3, 5]  → 합 = 9
  블록 1: [2, 7, 6]  → 합 = 15
  블록 2: [3, 8]     → 합 = 11

인덱스:   0  1  2 | 3  4  5 | 6  7
배열:     1  3  5 | 2  7  6 | 3  8
블록:     [  0   ] [  1   ] [ 2  ]
블록합:      9        15       11
```

**쿼리: 구간 합 [1, 6]**

```
L=1, R=6
L의 블록 = 1/3 = 0, R의 블록 = 6/3 = 2

1) L이 속한 블록 0의 나머지: 인덱스 1, 2 → 3 + 5 = 8
2) 완전히 포함되는 블록 1: 블록합 = 15
3) R이 속한 블록 2의 앞부분: 인덱스 6 → 3

합계 = 8 + 15 + 3 = 26
```

**점 갱신: a[4] = 10 (기존 7)**

```
a[4] = 10
블록 1의 합: 15 - 7 + 10 = 18

갱신 후:
인덱스:   0  1  2 | 3   4  5 | 6  7
배열:     1  3  5 | 2  10  6 | 3  8
블록합:      9        18       11
```

### 예시 2: Mo's Algorithm

배열: `[1, 2, 1, 3, 2]`, N = 5, B = ⌈√5⌉ = 3

쿼리: 구간 내 서로 다른 수의 개수
- Q1: [0, 3]
- Q2: [1, 4]
- Q3: [0, 1]

```
쿼리 정렬 (L/B, R) 기준:
  Q3: [0, 1] → (0/3=0, R=1)
  Q1: [0, 3] → (0/3=0, R=3)
  Q2: [1, 4] → (1/3=0, R=4)

처리 순서: Q3 → Q1 → Q2

--- Q3: [0, 1] ---
curL=0, curR=0: add(a[0]=1) → cnt={1:1}, distinct=1
curR=1: add(a[1]=2) → cnt={1:1, 2:1}, distinct=2
답: 2

--- Q1: [0, 3] ---
curR=2: add(a[2]=1) → cnt={1:2, 2:1}, distinct=2
curR=3: add(a[3]=3) → cnt={1:2, 2:1, 3:1}, distinct=3
답: 3

--- Q2: [1, 4] ---
curL=1: remove(a[0]=1) → cnt={1:1, 2:1, 3:1}, distinct=3
curR=4: add(a[4]=2) → cnt={1:1, 2:2, 3:1}, distinct=3
답: 3
```

## 실전 팁

### 활용 노하우

- 블록 크기 B는 보통 `⌈√N⌉`으로 설정하지만, Mo's Algorithm에서는 `⌈N / √Q⌉`로 설정하면 더 최적이다
- Mo's Algorithm에서 같은 블록 내 쿼리의 R 정렬을 홀수 블록은 오름차순, 짝수 블록은 내림차순으로 하면 상수 배 빨라진다 (Hilbert curve 정렬도 유사한 효과)
- 세그먼트 트리로 풀기 어려운 문제(예: 구간 내 서로 다른 수의 개수)에서 Mo's Algorithm이 강력하다
- Go에서 블록 크기는 `int(math.Ceil(math.Sqrt(float64(n))))`으로 계산하거나, 간단히 정수 연산으로 구한다

### 자주 하는 실수

- 블록 크기를 0으로 설정하는 실수. N = 0인 경우를 처리하지 않으면 나눗셈 오류가 발생한다
- Mo's Algorithm에서 쿼리 정렬 후 원래 쿼리 순서를 잃어버리는 실수. 쿼리에 원래 인덱스를 저장해야 한다
- 구간 [L, R]에서 L > R인 경우를 처리하지 않는 실수
- Mo's Algorithm에서 add/remove 순서를 잘못 구현하는 실수. 확장은 먼저, 축소는 나중에 해야 한다
- 마지막 블록의 크기가 B보다 작을 수 있음을 간과하는 실수

### 엣지 케이스

- N = 1: 블록이 1개, 모든 쿼리가 단일 원소 접근
- 쿼리 범위가 하나의 블록 안에 완전히 포함되는 경우
- L = 0, R = N-1: 전체 배열에 대한 쿼리
- 모든 원소가 같은 값인 경우
- Q = 0: 쿼리가 없는 경우 (Mo's Algorithm에서 블록 크기 계산 주의)

## 관련 알고리즘 비교

| 알고리즘 | 점 갱신 | 구간 쿼리 | 구간 갱신 | 구현 난이도 | 적합한 상황 |
| --- | --- | --- | --- | --- | --- |
| Sqrt Decomposition | O(1) | O(√N) | O(√N) | 쉬움 | 간단한 구간 쿼리, 구현 시간 절약 |
| 세그먼트 트리 | O(log N) | O(log N) | O(log N) | 보통 | 빠른 쿼리 필요, 온라인 처리 |
| BIT (펜윅 트리) | O(log N) | O(log N) | O(log N) | 쉬움 | 누적합 기반 쿼리 |
| Mo's Algorithm | - | O((N+Q)√N) | - | 보통 | 오프라인 구간 쿼리, 복잡한 쿼리 함수 |

- 온라인 쿼리 + 빠른 속도 → 세그먼트 트리 또는 BIT
- 오프라인 쿼리 + 복잡한 쿼리 함수 → Mo's Algorithm
- 간단한 구현 + 적당한 속도 → Sqrt Decomposition
- 구간 갱신 + 구간 쿼리 → 세그먼트 트리 with Lazy Propagation 또는 Sqrt with Lazy 블록

## 변형 및 최적화

### Mo's Algorithm

Mo's Algorithm은 제곱근 분할법의 가장 대표적인 응용이다. 오프라인으로 모든 쿼리를 받아 정렬한 뒤, 투 포인터처럼 구간을 확장/축소하며 답을 구한다.

**적용 조건:**
- 모든 쿼리를 미리 알 수 있어야 한다 (오프라인)
- 원소 하나를 추가/제거할 때 답을 O(1) 또는 O(log N)에 갱신할 수 있어야 한다

**정렬 최적화 (홀짝 정렬):**

```
블록 번호가 짝수이면 R 오름차순
블록 번호가 홀수이면 R 내림차순
```

이렇게 하면 R 포인터가 지그재그로 이동하여 총 이동 거리가 줄어든다.

### 블록 크기 최적화

- 기본: `B = √N`
- Mo's Algorithm: `B = N / √Q` (Q가 쿼리 수)
- 갱신이 있는 Mo's Algorithm: `B = N^(2/3)`

블록 크기를 문제 특성에 맞게 조절하면 상수 배 성능 향상을 얻을 수 있다.

### Mo's Algorithm with Updates (Mo's Algorithm + 갱신)

점 갱신이 포함된 경우, 쿼리를 `(L/B, R/B, 시간)` 기준으로 정렬하여 처리한다. 블록 크기는 `B = N^(2/3)`으로 설정하며, 전체 복잡도는 O(N^(5/3))이다.
