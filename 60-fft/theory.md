## 개념

FFT(Fast Fourier Transform, 고속 푸리에 변환)는 이산 푸리에 변환(DFT)을 O(N log N)에 계산하는 알고리즘이다.

### DFT와 IDFT

다항식 A(x) = a₀ + a₁x + a₂x² + ... + aₙ₋₁xⁿ⁻¹이 있을 때:

- **DFT**: 계수 표현 [a₀, a₁, ..., aₙ₋₁]을 점값 표현 [A(ω⁰), A(ω¹), ..., A(ωⁿ⁻¹)]으로 변환한다
- **IDFT**: 점값 표현을 다시 계수 표현으로 복원한다

여기서 ω = e^(2πi/N)은 N차 복소수 단위근(primitive N-th root of unity)이다.

### 다항식 곱셈과 FFT

두 다항식 A(x)와 B(x)의 곱 C(x) = A(x) · B(x)를 구하려면:

1. 나이브하게 계수끼리 곱하면 O(N²)이 걸린다
2. FFT를 사용하면 O(N log N)에 가능하다:
   - A, B를 각각 DFT로 점값 표현으로 변환 → O(N log N)
   - 점값끼리 곱셈 → O(N)
   - 결과를 IDFT로 계수 표현으로 복원 → O(N log N)

### 핵심 수식

DFT 공식:

```
X[k] = Σ(j=0 ~ N-1) x[j] · ω^(jk),  ω = e^(2πi/N)
```

IDFT 공식:

```
x[j] = (1/N) · Σ(k=0 ~ N-1) X[k] · ω^(-jk)
```

IDFT는 DFT와 동일한 구조이며, ω 대신 ω⁻¹을 사용하고 결과를 N으로 나누면 된다. 따라서 같은 FFT 함수를 재활용할 수 있다.

## 동작 원리

### 쿨리-튜키(Cooley-Tukey) 알고리즘

FFT의 가장 대표적인 구현 방식이다. N이 2의 거듭제곱일 때 분할 정복으로 동작한다.

**핵심 아이디어**: 짝수 인덱스 항과 홀수 인덱스 항으로 분리한다.

```
A(x) = A_even(x²) + x · A_odd(x²)
```

여기서:
- A_even(y) = a₀ + a₂y + a₄y² + ...  (짝수 인덱스 계수)
- A_odd(y)  = a₁ + a₃y + a₅y² + ...  (홀수 인덱스 계수)

**버터플라이 연산**:

N차 단위근의 성질 ω^(N/2) = -1을 이용하면:

```
A(ωᵏ)       = A_even(ω²ᵏ) + ωᵏ · A_odd(ω²ᵏ)
A(ωᵏ⁺ᴺ/²)  = A_even(ω²ᵏ) - ωᵏ · A_odd(ω²ᵏ)
```

이렇게 크기 N 문제를 크기 N/2 문제 2개로 분할하면, T(N) = 2T(N/2) + O(N) = O(N log N)이 된다.

### 알고리즘 단계

1. **패딩**: 두 다항식의 차수 합 이상이 되도록 N을 2의 거듭제곱으로 올림
2. **FFT(정방향)**: 계수 배열 → 점값 배열 (DFT)
3. **점별 곱셈**: 두 점값 배열의 같은 인덱스끼리 곱셈
4. **FFT(역방향)**: 점값 배열 → 계수 배열 (IDFT)
5. **반올림**: 실수 오차를 보정하여 정수 계수 복원

## 단계별 추적 (Trace)

### 예시 입력

다항식 A(x) = 1 + 2x + 3x² 와 B(x) = 4 + 5x 의 곱을 구한다.

- A의 계수: [1, 2, 3]
- B의 계수: [4, 5]
- 결과 C(x) = A(x)·B(x) = 4 + 13x + 22x² + 15x³
- 기대 결과 계수: [4, 13, 22, 15]

### 실행 과정

**1단계: 패딩**

```
결과 다항식 차수 = 2 + 1 = 3 → 계수 4개 필요
N = 4 (2의 거듭제곱으로 올림)
A = [1, 2, 3, 0]  (0으로 패딩)
B = [4, 5, 0, 0]  (0으로 패딩)
```

**2단계: FFT(정방향) - A의 DFT 계산**

N=4일 때 ω = e^(2πi/4) = i (허수 단위)

```
ω⁰ = 1,  ω¹ = i,  ω² = -1,  ω³ = -i

재귀 분할:
  A_even = [1, 3] → DFT → [4, -2]
  A_odd  = [2, 0] → DFT → [2, 2]

버터플라이 연산:
  k=0: A[0] = 4 + ω⁰·2 = 4 + 1·2 = 6
       A[2] = 4 - ω⁰·2 = 4 - 1·2 = 2
  k=1: A[1] = -2 + ω¹·2 = -2 + i·2 = -2+2i
       A[3] = -2 - ω¹·2 = -2 - i·2 = -2-2i

DFT(A) = [6, -2+2i, 2, -2-2i]
```

**3단계: FFT(정방향) - B의 DFT 계산**

```
재귀 분할:
  B_even = [4, 0] → DFT → [4, 4]
  B_odd  = [5, 0] → DFT → [5, 5]

버터플라이 연산:
  k=0: B[0] = 4 + ω⁰·5 = 9
       B[2] = 4 - ω⁰·5 = -1
  k=1: B[1] = 4 + ω¹·5 = 4+5i
       B[3] = 4 - ω¹·5 = 4-5i

DFT(B) = [9, 4+5i, -1, 4-5i]
```

**4단계: 점별 곱셈**

```
C[0] = 6 · 9           = 54
C[1] = (-2+2i)·(4+5i)  = -8-10i+8i+10i² = -18-2i
C[2] = 2 · (-1)        = -2
C[3] = (-2-2i)·(4-5i)  = -8+10i-8i+10i² = -18+2i
```

**5단계: IDFT (역방향 FFT)**

```
IDFT 결과를 N=4로 나눔:
c[0] = (54 + (-18-2i) + (-2) + (-18+2i)) / 4 = 16/4 = 4
c[1] = (54 + (-18-2i)·(-i) + (-2)·(-1) + (-18+2i)·(i)) / 4
     = (54 + (-2i+18i²·(-1)... )) / 4 = 52/4 = 13
c[2] = (54 + (-18-2i)·(-1) + (-2)·(1) + (-18+2i)·(-1)) / 4
     = (54 + 18+2i - 2 + 18-2i) / 4 = 88/4 = 22
c[3] = (54 + (-18-2i)·(i) + (-2)·(-1) + (-18+2i)·(-i)) / 4
     = 60/4 = 15

결과: [4, 13, 22, 15]
```

**6단계: 검증**

```
C(x) = 4 + 13x + 22x² + 15x³

검산: (1 + 2x + 3x²)(4 + 5x)
    = 4 + 5x + 8x + 10x² + 12x² + 15x³
    = 4 + 13x + 22x² + 15x³  ✓
```

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

- N은 결과 다항식의 계수 개수 (2의 거듭제곱으로 올림)
- 나이브 다항식 곱셈 O(N²) 대비 획기적으로 빠르다

## 적합한 문제 유형

- **다항식 곱셈**: 두 다항식의 곱을 구하는 문제
- **큰 수 곱셈**: 매우 큰 정수의 곱셈 (각 자릿수를 계수로 취급)
- **합성곱(Convolution)**: c[k] = Σ a[i]·b[k-i] 형태의 합을 구하는 문제
- **문자열 매칭**: 와일드카드 패턴 매칭 등
- **카운팅 문제**: 두 집합의 원소 합/차의 빈도 계산
- **주파수 분석**: 신호 처리 관련 문제

## 실전 팁

### 활용 노하우

- 문제에서 "합성곱", "다항식 곱", "두 배열의 원소 합 빈도" 등의 키워드가 나오면 FFT를 의심한다
- N이 10⁵ 이상이고 O(N²)이 시간 초과인 곱셈/합성곱 문제는 FFT가 거의 확정이다
- 결과 계수가 정수인 경우 `math.Round()`로 반올림하여 실수 오차를 보정한다
- 배열 크기를 2의 거듭제곱으로 맞추는 것을 잊지 말아야 한다

### 자주 하는 실수

- **배열 크기 부족**: 결과 다항식의 차수는 (deg A + deg B)이므로, N은 (len A + len B) 이상의 2의 거듭제곱이어야 한다. 부족하면 결과가 잘린다
- **실수 오차**: 복소수 FFT는 부동소수점 오차가 누적된다. 결과가 정수여야 하면 반드시 반올림한다
- **IDFT에서 N으로 나누기 누락**: IDFT 결과를 N으로 나누지 않으면 값이 N배가 된다
- **비재귀 구현 시 비트 반전 순서 오류**: 반복적 FFT 구현에서 인덱스 비트 반전(bit-reversal permutation)을 빠뜨리면 결과가 뒤섞인다

### 엣지 케이스

- 한쪽 다항식이 상수(차수 0)인 경우: 단순 스칼라 곱이지만 FFT로도 정상 동작한다
- 계수가 모두 0인 경우: 결과도 모두 0이다
- 계수가 매우 큰 경우 (10⁹ 이상): 실수 FFT의 정밀도 한계에 주의. NTT 사용을 고려한다
- N=1인 경우: FFT 없이 직접 곱셈하면 된다

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| FFT (복소수) | O(N log N) | 복소수 연산, 실수 오차 존재 | 일반적인 다항식 곱셈, 실수 계수 |
| NTT (정수론적 변환) | O(N log N) | 모듈러 연산, 오차 없음 | 결과가 정수이고 모듈러 연산이 필요한 경우 |
| 카라츠바 곱셈 | O(N^1.585) | 분할 정복, 구현 간단 | N이 작거나 FFT/NTT 구현이 어려운 경우 |
| 나이브 곱셈 | O(N²) | 가장 단순 | N ≤ 1000 정도의 작은 입력 |

### FFT vs NTT 선택 기준

- **FFT**: 범용적, 실수 계수 가능, 구현이 직관적. 단, 실수 오차 주의
- **NTT**: 모듈러 소수(예: 998244353) 하에서 정확한 정수 연산. 코딩테스트에서 모듈러 출력이 요구되면 NTT가 유리

## 변형 및 최적화

### NTT (Number Theoretic Transform, 정수론적 변환)

복소수 단위근 대신 모듈러 원시근(primitive root)을 사용하는 FFT이다.

- 소수 p에 대해 원시근 g를 사용: ω = g^((p-1)/N) mod p
- 대표적인 NTT 친화 소수: 998244353 = 119 × 2²³ + 1 (원시근 g=3)
- 장점: 정수 연산만 사용하므로 오차가 없다
- 단점: 모듈러 소수의 제약이 있고, 결과가 해당 소수 범위 내여야 한다

### 실수 FFT 정밀도 개선

- `float64` 기본 정밀도로 계수가 10⁹ 이하이고 N ≤ 2²⁰ 정도면 충분하다
- 더 큰 값이 필요하면 분할 기법을 사용한다: 각 계수를 상위/하위 비트로 나누어 여러 번 FFT를 수행
- 또는 NTT로 전환하여 정확한 결과를 얻는다

### 비재귀(반복적) FFT

재귀 호출 오버헤드를 줄이기 위해 반복문으로 구현하는 방식이다.

1. 비트 반전 순열(bit-reversal permutation)로 배열을 재배치
2. 아래에서 위로(bottom-up) 버터플라이 연산 수행
3. 실전에서는 재귀보다 빠르고 스택 오버플로 위험이 없다
