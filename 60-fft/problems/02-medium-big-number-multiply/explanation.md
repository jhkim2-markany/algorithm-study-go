# 큰 수 곱셈 - 해설

## 접근 방식

1. 두 큰 수를 문자열로 입력받는다
2. 각 자릿수를 다항식의 계수로 취급한다 (낮은 자릿수 = 낮은 차수)
   - 예: "123" → 3 + 2x + 1x² → 계수 [3, 2, 1]
3. FFT를 사용하여 두 다항식을 곱한다
4. 결과 계수에서 10 이상인 값은 올림(carry) 처리한다
5. 최종 결과를 역순으로 출력한다

## 핵심 아이디어

큰 수 곱셈은 본질적으로 다항식 곱셈과 동일하다. 정수 N = dₖdₖ₋₁...d₁d₀ 은 다항식 D(10) = d₀ + d₁·10 + d₂·10² + ... + dₖ·10ᵏ 으로 표현할 수 있다.

따라서 두 정수의 곱은 두 다항식의 곱을 구한 뒤, x=10을 대입하는 것과 같다. 다항식 곱셈 후 각 계수가 10 이상이면 올림(carry)을 전파하면 된다.

**주의사항:**
- 문자열을 역순으로 저장하여 인덱스 0이 1의 자리가 되도록 한다
- 올림 처리 시 순서대로 (낮은 자릿수부터) 처리해야 한다
- 결과 출력 시 선행 0을 제거한다

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N), N = len(A) + len(B) |
| 공간 복잡도 | O(N) |

나이브 큰 수 곱셈은 O(N²)이므로, 자릿수가 10만 이상일 때 FFT가 필수적이다.

## 대안적 접근

- 카라츠바 알고리즘: O(N^1.585), FFT보다 느리지만 구현이 간단하다
- NTT: 정수 연산만 사용하므로 실수 오차가 없다. 자릿수가 매우 클 때 더 안정적이다
