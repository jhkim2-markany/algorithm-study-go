# 다항식 곱셈 기본 - 해설

## 접근 방식

1. 두 다항식의 계수 배열을 입력받는다
2. 결과 다항식의 크기(len(A) + len(B) - 1)를 계산하고, 2의 거듭제곱으로 올림한다
3. 각 계수 배열을 복소수 배열로 변환하고 0으로 패딩한다
4. 정방향 FFT로 두 배열을 점값 표현으로 변환한다
5. 점값끼리 곱셈한다
6. 역방향 FFT(IDFT)로 결과를 계수 표현으로 복원한다
7. 실수부를 반올림하여 정수 계수를 얻는다

## 핵심 아이디어

다항식 곱셈의 나이브 구현은 O(N²)이지만, FFT를 사용하면 O(N log N)에 수행할 수 있다. 핵심은 계수 표현을 점값 표현으로 변환하면 곱셈이 O(N)이 된다는 점이다.

- 계수 표현에서의 곱셈: O(N²) (각 계수 쌍을 모두 곱해야 함)
- 점값 표현에서의 곱셈: O(N) (같은 점에서의 값끼리 곱하면 됨)
- FFT/IFFT 변환: O(N log N)

따라서 전체 과정은 O(N log N) + O(N) + O(N log N) = O(N log N)이다.

비재귀(반복적) FFT 구현을 사용하여 재귀 호출 오버헤드를 줄였다. 비트 반전 순열로 배열을 재배치한 후, 아래에서 위로 버터플라이 연산을 수행한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N), N = len(A) + len(B) |
| 공간 복잡도 | O(N) |

## 대안적 접근

- N이 작으면 (N ≤ 1000) 나이브 O(N²) 곱셈으로도 충분하다
- 결과를 모듈러 소수로 출력해야 하면 NTT를 사용하는 것이 정확하다
