# 합성곱 카운팅 - 해설

## 접근 방식

1. 배열 A와 B 각각에 대해 값의 빈도 배열(frequency array)을 구한다
   - freqA[v] = A에서 값 v가 등장하는 횟수
   - freqB[v] = B에서 값 v가 등장하는 횟수
2. 빈도 배열을 다항식의 계수로 취급한다
   - PA(x) = freqA[0] + freqA[1]·x + freqA[2]·x² + ...
   - PB(x) = freqB[0] + freqB[1]·x + freqB[2]·x² + ...
3. 두 다항식의 곱 PC = PA × PB를 FFT로 계산한다
4. PC의 계수 PC[s]가 곧 합 a+b=s를 만족하는 쌍의 개수이다

## 핵심 아이디어

이 문제의 핵심은 "두 배열에서 원소를 하나씩 골라 합이 S인 쌍의 개수"가 합성곱(Convolution)과 동일하다는 점이다.

다항식 곱셈에서:

```
PC[s] = Σ(i+j=s) PA[i] · PB[j]
```

이는 정확히 "A에서 값 i를 고르는 경우의 수 × B에서 값 j를 고르는 경우의 수"를 i+j=s인 모든 쌍에 대해 합한 것이다.

나이브하게 모든 (i, j) 쌍을 확인하면 O(N²)이지만, FFT를 사용하면 O(M log M)에 해결할 수 있다 (M = 값의 최대 범위).

**빈도 배열 → 다항식 변환이 핵심 테크닉이다.** 카운팅 문제에서 FFT를 적용하려면 항상 이 변환을 먼저 생각해야 한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(M log M), M = max(A) + max(B) |
| 공간 복잡도 | O(M) |

N ≤ 200,000이고 값의 범위도 N 이하이므로, M ≤ 400,000이다. FFT로 충분히 시간 내에 해결 가능하다.

## 대안적 접근

- 나이브 O(N²): 모든 쌍을 직접 확인. N ≤ 5,000 정도에서만 가능
- NTT: 결과가 정수이고 모듈러 출력이 필요한 경우 더 정확하다
- 값의 범위가 작으면 (≤ 5,000) 나이브 합성곱으로도 가능하다
