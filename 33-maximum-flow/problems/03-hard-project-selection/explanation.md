# 프로젝트 선택 문제 - 해설

## 접근 방식

1. 프로젝트 선택 문제를 최소 컷 문제로 변환한다
2. 소스, 싱크를 추가하고 이익/비용에 따라 간선을 구성한다
3. 의존 관계는 무한 용량 간선으로 표현한다
4. Edmonds-Karp 알고리즘으로 최대 유량(= 최소 컷)을 구한다
5. 최대 순이익 = 양수 이익의 합 - 최소 컷

## 핵심 아이디어

프로젝트 선택 문제는 최소 컷으로 모델링하는 대표적인 문제이다. 모델링 방법은 다음과 같다:

- 소스(S)와 싱크(T)를 추가한다
- 이익이 양수인 프로젝트 i: S → i 간선 (용량 = p_i)
- 이익이 음수인 프로젝트 i: i → T 간선 (용량 = |p_i|)
- 의존 관계 (a, b): a → b 간선 (용량 = ∞)

최소 컷에서 소스 쪽에 남은 프로젝트가 "선택한 프로젝트"이다:
- 양수 이익 프로젝트를 선택하지 않으면 S→i 간선이 컷된다 (이익 손실)
- 음수 이익 프로젝트를 선택하면 i→T 간선이 컷된다 (비용 발생)
- 의존 관계를 위반하면 ∞ 용량 간선이 컷되어야 하므로 불가능하다

따라서 최소 컷은 "포기하는 양수 이익 + 감수하는 음수 비용"의 최솟값이며, 최대 순이익 = 양수 이익 합 - 최소 컷이 된다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(V × E²) |
| 공간 복잡도 | O(V + E) |

정점 수 V = N + 2, 간선 수 E = N + M이다. N ≤ 200이므로 Edmonds-Karp 알고리즘으로 충분히 해결할 수 있다.

## 대안적 접근

- 프로젝트 수가 적으면 비트마스크 DP로 모든 부분집합을 탐색할 수 있지만, N ≤ 200에서는 불가능하다
- Dinic 알고리즘을 사용하면 더 빠르게 해결할 수 있다
- 이 모델링 기법은 "최대 가중 폐합(Maximum Weight Closure)" 문제로도 알려져 있다
