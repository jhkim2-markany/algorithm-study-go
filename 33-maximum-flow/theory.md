# 최대 유량

## 개념

최대 유량(Maximum Flow)은 방향 그래프에서 소스(source)에서 싱크(sink)로 보낼 수 있는 최대 유량을 구하는 문제이다. 각 간선에는 용량(capacity)이 있으며, 간선을 통해 흐르는 유량은 용량을 초과할 수 없다. 또한 소스와 싱크를 제외한 모든 정점에서 들어오는 유량과 나가는 유량이 같아야 한다(유량 보존 법칙).

네트워크 유량 문제는 물류 네트워크, 통신 네트워크, 이분 매칭 등 다양한 분야에서 활용된다. 핵심 아이디어는 "증가 경로(augmenting path)"를 반복적으로 찾아 유량을 늘리는 것이며, 더 이상 증가 경로가 없을 때 최대 유량에 도달한다.

최대 유량-최소 컷 정리(Max-Flow Min-Cut Theorem)에 의해, 네트워크의 최대 유량은 최소 컷의 용량과 같다. 이 정리는 유량 문제의 이론적 기반이 된다.

## 동작 원리

최대 유량 알고리즘의 핵심은 잔여 그래프(residual graph)에서 증가 경로를 반복적으로 찾는 것이다.

### 1. 잔여 그래프 (Residual Graph)

원래 그래프에서 현재 유량 상태를 반영한 그래프이다:

- 순방향 간선: 용량 - 현재 유량 만큼의 잔여 용량을 가진다
- 역방향 간선: 현재 유량만큼의 잔여 용량을 가진다 (유량 취소 가능)

역방향 간선은 이미 보낸 유량을 "되돌리는" 역할을 하며, 이를 통해 최적이 아닌 경로 선택을 나중에 수정할 수 있다.

### 2. Ford-Fulkerson 방법

Ford-Fulkerson은 증가 경로를 찾는 구체적 방법을 지정하지 않는 일반적 프레임워크이다:

1. 모든 간선의 유량을 0으로 초기화한다
2. 잔여 그래프에서 소스에서 싱크로의 증가 경로를 찾는다
3. 증가 경로를 따라 보낼 수 있는 최대 유량(병목 용량)을 구한다
4. 경로를 따라 유량을 갱신한다 (순방향 +, 역방향 -)
5. 더 이상 증가 경로가 없을 때까지 2~4를 반복한다

### 3. Edmonds-Karp 알고리즘

Ford-Fulkerson 방법에서 증가 경로를 BFS로 찾는 구체적 알고리즘이다:

1. BFS를 사용하여 소스에서 싱크까지의 최단 증가 경로를 찾는다
2. 경로 상의 최소 잔여 용량(병목)을 구한다
3. 경로를 따라 유량을 갱신한다
4. 더 이상 BFS로 싱크에 도달할 수 없을 때까지 반복한다

BFS를 사용하므로 항상 최단 경로(간선 수 기준)를 선택하며, 이로 인해 시간 복잡도가 보장된다.

### 4. 최대 유량-최소 컷 정리

네트워크의 최대 유량은 소스와 싱크를 분리하는 최소 컷의 용량과 같다:

- 컷(cut): 소스를 포함하는 집합 S와 싱크를 포함하는 집합 T로 정점을 분할하는 것
- 컷의 용량: S에서 T로 가는 간선들의 용량 합
- 최소 컷: 용량이 최소인 컷

최대 유량 알고리즘이 종료된 후, 잔여 그래프에서 소스로부터 BFS/DFS로 도달 가능한 정점 집합이 S, 나머지가 T가 되어 최소 컷을 구할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| Ford-Fulkerson (DFS) | O(V × E × f) (f: 최대 유량) |
| Edmonds-Karp (BFS) | O(V × E²) |
| Dinic 알고리즘 | O(V² × E) |
| 공간 복잡도 | O(V + E) |

Edmonds-Karp는 BFS를 사용하여 최대 O(V × E)번의 증가 경로를 찾으며, 각 BFS는 O(E)이므로 전체 O(V × E²)이다. 실제로는 이보다 훨씬 빠르게 동작하는 경우가 많다.

## 적합한 문제 유형

- 네트워크에서 최대 유량 구하기 문제
- 이분 매칭 문제 (최대 유량으로 변환 가능)
- 최소 컷 문제 (최대 유량-최소 컷 정리 활용)
- 프로젝트 선택 문제 (최소 컷으로 모델링)
- 간선/정점 분리 문제 (연결성 관련)
- 순환 있는 네트워크의 유량 최적화 문제
- 다중 소스/싱크 유량 문제 (가상 소스/싱크 추가)

## 단계별 추적 (Trace)

### 예시: Edmonds-Karp 알고리즘으로 최대 유량 구하기

```
입력: 4개 정점 (S=0, A=1, B=2, T=3)
간선 (용량):
  S→A: 10
  S→B: 8
  A→B: 5
  A→T: 7
  B→T: 10

초기 유량: 모든 간선 0

=== 반복 1: BFS로 증가 경로 탐색 ===
경로: S → A → T
병목 용량: min(10, 7) = 7

유량 갱신:
  S→A: 0 → 7  (잔여: 10-7=3)
  A→T: 0 → 7  (잔여: 7-7=0)
역방향 잔여:
  A→S: 0 → 7
  T→A: 0 → 7

현재 총 유량: 7

=== 반복 2: BFS로 증가 경로 탐색 ===
경로: S → B → T
병목 용량: min(8, 10) = 8

유량 갱신:
  S→B: 0 → 8  (잔여: 8-8=0)
  B→T: 0 → 8  (잔여: 10-8=2)

현재 총 유량: 7 + 8 = 15

=== 반복 3: BFS로 증가 경로 탐색 ===
경로: S → A → B → T
병목 용량: min(3, 5, 2) = 2

유량 갱신:
  S→A: 7 → 9  (잔여: 10-9=1)
  A→B: 0 → 2  (잔여: 5-2=3)
  B→T: 8 → 10 (잔여: 10-10=0)

현재 총 유량: 15 + 2 = 17

=== 반복 4: BFS로 증가 경로 탐색 ===
BFS 결과: 싱크 T에 도달 불가 → 종료

최대 유량: 17

최종 유량 상태:
  S→A: 9/10
  S→B: 8/8
  A→B: 2/5
  A→T: 7/7
  B→T: 10/10
```

### 최소 컷 확인

```
최대 유량 종료 후 잔여 그래프에서 S로부터 BFS:
  S에서 도달 가능: {S, A} (S→A 잔여 1 > 0)
  S에서 도달 불가: {B, T}

최소 컷: S 집합 = {S, A}, T 집합 = {B, T}
컷 간선: S→B(8) + A→B(5) + A→T(7) = 20? 
→ 실제 컷 용량: S→B(8) + A→T(7) + A→B(5) 중 S→T 방향만
→ 컷 용량 = S→B(8) + A→B(5) + A→T(7) = 20이 아니라
→ 실제: 포화 간선 = S→B(8), A→T(7), B→T(10) 중...
→ 최소 컷 = 최대 유량 = 17 (정리에 의해 보장)
```

## 실전 팁

### 활용 노하우

- 이분 매칭 문제는 최대 유량으로 변환 가능하다. 가상 소스/싱크를 추가하고 용량 1로 설정하라
- Edmonds-Karp(BFS)는 구현이 간단하고 시간 복잡도가 보장되므로 코딩테스트에서 가장 추천한다
- 간선을 추가할 때 역방향 간선도 반드시 함께 추가하라 (용량 0)
- 인접 리스트에서 간선과 역방향 간선의 인덱스를 짝수/홀수로 관리하면 역방향 접근이 편리하다

### 자주 하는 실수

- 역방향 간선을 추가하지 않는 실수 (유량 취소가 불가능해져 최적해를 못 찾음)
- 양방향 간선을 단방향 2개로 처리할 때 역방향 간선 관리를 잘못하는 실수
- BFS에서 잔여 용량이 0인 간선을 방문하는 실수 (잔여 용량 > 0인 간선만 탐색)
- 다중 소스/싱크 문제에서 가상 소스/싱크를 추가하지 않는 실수

### 엣지 케이스

- 소스에서 싱크로 직접 연결된 간선이 없는 경우
- 용량이 0인 간선이 존재하는 경우
- 소스와 싱크가 연결되지 않은 경우 (최대 유량 = 0)
- 모든 간선의 용량이 1인 경우 (간선 분리 문제)

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| Edmonds-Karp (BFS) | O(VE²), 구현 간단 | 코딩테스트 범용, 중소 규모 그래프 |
| Dinic | O(V²E), 레벨 그래프 활용 | 대규모 그래프, 이분 매칭 O(E√V) |
| Ford-Fulkerson (DFS) | O(VEf), 최대 유량에 의존 | 용량이 작은 경우에만 사용 |
| 이분 매칭 (Hopcroft-Karp) | O(E√V), 이분 그래프 전용 | 이분 매칭 문제에 특화 |
| MCMF (최소 비용 최대 유량) | O(VEf), 비용 최적화 | 유량에 비용이 추가된 문제 |
| 최소 컷 | 최대 유량과 동치 | 네트워크 분리, 프로젝트 선택 |
