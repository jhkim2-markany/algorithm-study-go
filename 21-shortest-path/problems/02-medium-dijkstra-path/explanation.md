# 다익스트라 경로 추적 - 해설

## 접근 방식

1. Dijkstra 알고리즘으로 도시 1에서 모든 도시까지의 최단 거리를 구한다.
2. 최단 거리를 갱신할 때마다 이전 정점(prev)을 기록하여 경로를 추적할 수 있게 한다.
3. 도시 N에 도달할 수 없으면 -1을 출력한다.
4. 도달 가능하면 도시 N에서 prev 배열을 따라 역추적하여 경로를 복원한다.
5. 역추적한 경로를 뒤집어서 출발점부터 순서대로 출력한다.

## 핵심 아이디어

Dijkstra 알고리즘에서 경로를 복원하려면 각 정점의 "이전 정점"을 기록하면 된다. 완화(relaxation) 과정에서 dist[v]가 갱신될 때 prev[v] = u로 설정하면, 나중에 도착점에서 출발점까지 prev를 따라가며 최단 경로를 역추적할 수 있다.

경로 복원 과정:
1. v = N에서 시작
2. prev[v]를 따라 이전 정점으로 이동
3. v = 1(출발점)에 도달하면 종료
4. 수집한 정점들을 뒤집으면 출발점→도착점 순서의 경로가 된다

우선순위 큐에서 이미 확정된 거리보다 큰 값이 나오면 건너뛰는 최적화(lazy deletion)를 적용하여 불필요한 연산을 줄인다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O((N + M) log N) |
| 공간 복잡도 | O(N + M) |

Dijkstra 알고리즘의 시간 복잡도는 우선순위 큐를 사용할 때 O((V + E) log V)이다. 경로 복원은 O(N)으로 전체 복잡도에 영향을 주지 않는다. 인접 리스트, 거리 배열, prev 배열에 O(N + M) 공간이 필요하다.

## 대안적 접근

- A* 알고리즘을 사용하면 휴리스틱을 통해 탐색 공간을 줄일 수 있다.
- 간선 가중치가 작은 정수라면 다이얼(Dial) 알고리즘으로 O(V + E + W) 시간에 풀 수 있다 (W: 최대 가중치).
- BFS 기반 역추적으로 모든 최단 경로를 구할 수도 있다.
