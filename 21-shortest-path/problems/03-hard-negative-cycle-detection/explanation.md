# 음수 사이클 탐지 - 해설

## 접근 방식

1. Bellman-Ford 알고리즘을 사용하여 음수 사이클을 탐지한다.
2. 그래프가 연결되지 않을 수 있으므로, 가상의 시작점에서 모든 정점으로 가중치 0인 간선이 있다고 가정한다 (모든 정점의 초기 거리를 0으로 설정).
3. (N-1)번 모든 간선에 대해 완화(relaxation)를 수행한다.
4. N번째 반복에서 여전히 거리가 갱신되는 간선이 있으면 음수 사이클이 존재한다.

## 핵심 아이디어

Bellman-Ford 알고리즘의 핵심 성질을 활용한다: 음수 사이클이 없는 그래프에서 최단 경로는 최대 (V-1)개의 간선을 포함한다. 따라서 (V-1)번 완화를 수행하면 모든 최단 거리가 확정된다.

만약 V번째 반복에서도 거리가 줄어드는 간선이 존재한다면, 이는 경로를 한 바퀴 더 돌 때마다 거리가 계속 줄어든다는 의미이므로 음수 사이클이 존재하는 것이다.

비연결 그래프에서도 음수 사이클을 탐지하기 위해 모든 정점의 초기 거리를 0으로 설정한다. 이는 가상의 초출발점(super source)에서 모든 정점으로 가중치 0인 간선을 추가한 것과 동일한 효과이다. 이렇게 하면 어떤 컴포넌트에 있는 음수 사이클이든 탐지할 수 있다.

풀이 코드에는 SPFA(Shortest Path Faster Algorithm) 기반 탐지 방법도 포함되어 있다. SPFA는 큐를 사용하여 갱신이 필요한 정점만 처리하므로 평균적으로 더 빠르다. 어떤 정점이 큐에 N번 이상 들어가면 음수 사이클이 존재한다고 판단한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (Bellman-Ford) | O(V × E) |
| 시간 복잡도 (SPFA 평균) | O(E) |
| 시간 복잡도 (SPFA 최악) | O(V × E) |
| 공간 복잡도 | O(V + E) |

Bellman-Ford는 V번 반복하며 매번 E개의 간선을 검사하므로 O(V × E)이다. SPFA는 평균적으로 O(E)에 가깝지만 최악의 경우 O(V × E)이다. 간선 리스트와 거리 배열에 O(V + E) 공간이 필요하다.

## 대안적 접근

- Floyd-Warshall 알고리즘으로도 음수 사이클을 탐지할 수 있다. dist[i][i] < 0인 정점이 있으면 음수 사이클이 존재한다. 단, O(V³) 시간이 걸린다.
- DFS 기반으로 각 SCC(강연결 요소) 내에서 음수 사이클을 탐지하는 방법도 있다.
- Johnson 알고리즘은 Bellman-Ford로 음수 가중치를 제거한 후 Dijkstra를 적용하는 방식이다.
