# 최단 경로

## 개념

최단 경로(Shortest Path)는 그래프에서 두 정점 사이를 연결하는 경로 중 간선 가중치의 합이 최소인 경로를 찾는 문제이다. 네트워크 라우팅, 지도 내비게이션, 스케줄링 등 다양한 분야에서 핵심적으로 활용된다.

최단 경로 알고리즘은 문제 조건에 따라 크게 세 가지로 분류된다:

- **Dijkstra 알고리즘**: 음이 아닌 가중치 그래프에서 단일 출발점 최단 경로를 구한다. 우선순위 큐를 사용하여 효율적으로 동작한다.
- **Bellman-Ford 알고리즘**: 음수 가중치가 있는 그래프에서도 단일 출발점 최단 경로를 구할 수 있다. 음수 사이클 탐지도 가능하다.
- **Floyd-Warshall 알고리즘**: 모든 정점 쌍 사이의 최단 경로를 구한다. 동적 프로그래밍 기반으로 동작한다.

## 동작 원리

### Dijkstra 알고리즘

1. 출발 정점의 거리를 0으로, 나머지 정점의 거리를 무한대로 초기화한다.
2. 우선순위 큐(최소 힙)에 출발 정점을 넣는다.
3. 큐에서 거리가 가장 작은 정점 u를 꺼낸다.
4. u의 인접 정점 v에 대해, u를 거쳐 v로 가는 거리가 기존 거리보다 짧으면 갱신한다 (완화, relaxation).
5. 갱신된 정점을 큐에 넣는다.
6. 큐가 빌 때까지 3~5를 반복한다.

Dijkstra는 그리디 방식으로 동작하며, 한 번 확정된 정점의 최단 거리는 변하지 않는다. 단, 음수 가중치가 있으면 이 성질이 깨지므로 사용할 수 없다.

### Bellman-Ford 알고리즘

1. 출발 정점의 거리를 0으로, 나머지 정점의 거리를 무한대로 초기화한다.
2. 모든 간선에 대해 완화(relaxation)를 수행한다.
3. 2번 과정을 (V-1)번 반복한다. V는 정점의 수이다.
4. V번째 반복에서도 거리가 갱신되는 간선이 있으면 음수 사이클이 존재한다.

Bellman-Ford는 매 반복마다 최단 경로에 포함되는 간선 수가 1개씩 늘어나는 것을 보장한다. V-1번 반복하면 최대 V-1개의 간선으로 이루어진 최단 경로를 모두 찾을 수 있다.

### Floyd-Warshall 알고리즘

1. 거리 행렬 dist를 초기화한다. dist[i][j]는 i에서 j로의 직접 간선 가중치이며, 간선이 없으면 무한대, i==j이면 0이다.
2. 중간 정점 k를 0부터 V-1까지 순회한다.
3. 모든 정점 쌍 (i, j)에 대해, k를 경유하는 경로가 더 짧으면 갱신한다: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]).
4. 모든 k에 대해 반복하면 모든 쌍의 최단 거리가 구해진다.

Floyd-Warshall은 동적 프로그래밍의 원리를 사용한다. "정점 0~k만 경유할 때의 최단 거리"를 점진적으로 확장하여 최종적으로 모든 정점을 경유할 수 있는 최단 거리를 구한다.

## 복잡도

| 알고리즘 | 시간 복잡도 | 공간 복잡도 | 비고 |
| --- | --- | --- | --- |
| Dijkstra (우선순위 큐) | O((V + E) log V) | O(V + E) | 음수 가중치 불가 |
| Dijkstra (배열) | O(V²) | O(V + E) | 밀집 그래프에 유리 |
| Bellman-Ford | O(V × E) | O(V) | 음수 가중치 허용, 음수 사이클 탐지 |
| Floyd-Warshall | O(V³) | O(V²) | 모든 쌍 최단 경로 |

- V: 정점의 수, E: 간선의 수
- Dijkstra는 우선순위 큐를 사용할 때 희소 그래프에서 가장 효율적이다.
- Bellman-Ford는 Dijkstra보다 느리지만 음수 가중치를 처리할 수 있다.
- Floyd-Warshall은 정점 수가 적을 때(V ≤ 500 정도) 모든 쌍 최단 경로를 구하는 데 적합하다.

## 적합한 문제 유형

- 가중치 그래프에서 단일 출발점 최단 경로 (Dijkstra, Bellman-Ford)
- 모든 정점 쌍 사이의 최단 경로 (Floyd-Warshall)
- 음수 가중치가 있는 그래프의 최단 경로 (Bellman-Ford)
- 음수 사이클 탐지 (Bellman-Ford)
- 네트워크 라우팅, 지도 경로 탐색
- 비용 최소화 문제 (최소 비용으로 목적지 도달)
- 경로 복원 (최단 경로의 실제 경유 정점 추적)
- 비가중치 그래프의 최단 경로 (BFS)

## 단계별 추적 (Trace)

### 예시 입력 - Dijkstra

정점 5개 (0~4), 출발점 0, 간선: (0→1, 4), (0→2, 1), (1→3, 1), (2→1, 2), (2→3, 5), (3→4, 3)

### 실행 과정

```
초기: dist = [0, ∞, ∞, ∞, ∞],  PQ = [(0, 0)]

Step 1: PQ에서 (0, 정점0) 꺼냄
  → 정점1: dist[1] = min(∞, 0+4) = 4  갱신
  → 정점2: dist[2] = min(∞, 0+1) = 1  갱신
  dist = [0, 4, 1, ∞, ∞],  PQ = [(1,2), (4,1)]

Step 2: PQ에서 (1, 정점2) 꺼냄
  → 정점1: dist[1] = min(4, 1+2) = 3  갱신!
  → 정점3: dist[3] = min(∞, 1+5) = 6  갱신
  dist = [0, 3, 1, 6, ∞],  PQ = [(3,1), (4,1), (6,3)]

Step 3: PQ에서 (3, 정점1) 꺼냄
  → 정점3: dist[3] = min(6, 3+1) = 4  갱신!
  dist = [0, 3, 1, 4, ∞],  PQ = [(4,1), (6,3), (4,3)]

Step 4: PQ에서 (4, 정점1) 꺼냄 → 이미 처리됨, 스킵
        PQ에서 (4, 정점3) 꺼냄
  → 정점4: dist[4] = min(∞, 4+3) = 7  갱신
  dist = [0, 3, 1, 4, 7],  PQ = [(6,3), (7,4)]

Step 5: PQ에서 (6, 정점3) 꺼냄 → 이미 처리됨, 스킵
        PQ에서 (7, 정점4) 꺼냄 → 인접 간선 없음

최종 dist = [0, 3, 1, 4, 7]
최단 경로: 0→2→1→3→4 (비용 7)
```

### 예시 입력 - Bellman-Ford

정점 4개 (0~3), 출발점 0, 간선: (0→1, 1), (0→2, 4), (1→2, -2), (2→3, 3)

```
초기: dist = [0, ∞, ∞, ∞]

반복 1 (모든 간선 완화):
  (0→1): dist[1] = min(∞, 0+1) = 1
  (0→2): dist[2] = min(∞, 0+4) = 4
  (1→2): dist[2] = min(4, 1+(-2)) = -1  갱신!
  (2→3): dist[3] = min(∞, -1+3) = 2
  dist = [0, 1, -1, 2]

반복 2 (모든 간선 완화):
  변화 없음 → 조기 종료 가능

반복 3 (V번째 검사):
  변화 없음 → 음수 사이클 없음

최종 dist = [0, 1, -1, 2]
```

## 실전 팁

### 활용 노하우

- 가중치가 모두 양수이면 Dijkstra, 음수 가중치가 있으면 Bellman-Ford를 선택한다
- Dijkstra에서 `dist[u] < 현재 꺼낸 거리`이면 이미 처리된 정점이므로 스킵하는 최적화가 필수이다
- 정점 수가 적고(V ≤ 500) 모든 쌍 최단 경로가 필요하면 Floyd-Warshall이 간결하다
- "최단 경로의 간선 수"를 구할 때는 BFS(가중치 없는 경우)가 가장 효율적이다
- 경로 복원이 필요하면 `prev[]` 배열을 유지하여 역추적한다

### 자주 하는 실수

- Dijkstra에서 음수 가중치 그래프에 적용하여 오답을 내는 경우
- 무한대 초기값을 너무 작게 설정하여 오버플로우 발생 (Go에서 `math.MaxInt64/2` 권장)
- Dijkstra에서 방문 체크 없이 같은 정점을 여러 번 처리하여 TLE 발생
- Bellman-Ford에서 반복 횟수를 V-1이 아닌 V로 설정하여 불필요한 연산 수행
- Floyd-Warshall에서 k 루프를 가장 바깥에 두지 않으면 오답 발생 (k-i-j 순서 필수)

### 엣지 케이스

- 출발점과 도착점이 같은 경우 (거리 = 0)
- 도달 불가능한 정점이 있는 경우 (거리 = ∞ 유지)
- 자기 자신으로의 간선(self-loop)이 있는 경우
- 동일한 두 정점 사이에 여러 간선이 있는 경우 (다중 간선)
- 음수 사이클이 존재하는 경우 (Bellman-Ford로 탐지)
- 정점이 1개뿐인 경우

## 관련 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 음수 가중치 | 적합한 상황 |
| --- | --- | --- | --- |
| Dijkstra (힙) | O((V+E) log V) | 불가 | 양수 가중치, 단일 출발점, 희소 그래프 |
| Bellman-Ford | O(V × E) | 가능 | 음수 가중치, 음수 사이클 탐지 |
| Floyd-Warshall | O(V³) | 가능 | 모든 쌍 최단 경로, V가 작을 때 |
| BFS | O(V + E) | 불가 (가중치 없음) | 가중치 없는 그래프의 최단 경로 |
| 0-1 BFS | O(V + E) | 불가 (0/1만) | 가중치가 0 또는 1인 그래프 |
| SPFA | O(V × E) 최악 | 가능 | Bellman-Ford 최적화, 평균적으로 빠름 |
