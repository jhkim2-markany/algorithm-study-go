# 순열 검사 - 해설

## 접근 방식

1. N개의 정수 배열과 K를 입력받는다
2. 배열을 오름차순으로 정렬하여 사전순으로 가장 앞서는 순열부터 탐색한다
3. 재귀적으로 순열을 생성하면서, 인접 원소의 차이가 K를 초과하면 가지치기한다
4. 조건을 만족하는 첫 번째 순열을 찾으면 출력하고 종료한다
5. 모든 순열을 탐색해도 조건을 만족하는 순열이 없으면 "NO"를 출력한다

## 핵심 아이디어

이 문제는 순열 생성과 가지치기(pruning)를 결합한 브루트포스이다.

1. **사전순 보장**: 배열을 먼저 정렬하고, 작은 값부터 순서대로 순열에 추가하면 자연스럽게 사전순으로 탐색된다. 조건을 만족하는 첫 번째 순열이 곧 사전순으로 가장 앞서는 답이다.

2. **가지치기**: 순열을 한 원소씩 추가할 때, 직전 원소와의 차이가 K를 초과하면 해당 가지를 더 이상 탐색하지 않는다. 이를 통해 불필요한 탐색을 줄여 실행 시간을 단축한다.

3. **조기 종료**: 조건을 만족하는 순열을 하나 찾으면 즉시 탐색을 중단한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N!) |
| 공간 복잡도 | O(N) |

최악의 경우 N!개의 모든 순열을 탐색해야 하므로 시간 복잡도는 O(N!)이다. N이 최대 8이므로 8! = 40,320으로 시간 내에 충분히 처리할 수 있다. 재귀 호출 스택과 순열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **next_permutation 방식**: 정렬된 배열에서 다음 순열을 반복적으로 생성하며 조건을 검사하는 방식이다. 재귀 없이 구현할 수 있지만 가지치기가 어렵다.
- **그래프 모델링**: 원소를 노드로, 차이가 K 이하인 쌍을 간선으로 연결한 그래프에서 해밀턴 경로를 찾는 문제로 변환할 수 있다. N이 작으므로 실질적인 이점은 크지 않다.
