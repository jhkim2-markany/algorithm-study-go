# 두 수의 합 - 해설

## 접근 방식

1. N개의 정수 배열과 목표 합 M을 입력받는다
2. 2중 반복문으로 모든 서로 다른 두 원소의 쌍 (i, j)를 탐색한다
3. 각 쌍의 합이 M과 같은지 검사한다
4. 조건을 만족하는 첫 번째 쌍을 출력하고 종료한다
5. 모든 쌍을 검사해도 조건을 만족하는 쌍이 없으면 "NO"를 출력한다

## 핵심 아이디어

가장 기본적인 브루트포스 패턴인 2중 반복문을 사용한다. 바깥 반복문의 인덱스 i와 안쪽 반복문의 인덱스 j를 i+1부터 시작하여, 같은 원소를 두 번 선택하거나 같은 쌍을 중복 검사하는 것을 방지한다.

N이 최대 1000이므로 모든 쌍의 수는 약 500,000개로, 시간 제한 내에 충분히 처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N²) |
| 공간 복잡도 | O(N) |

N개 원소에서 2개를 선택하는 모든 조합 N(N-1)/2개를 검사하므로 시간 복잡도는 O(N²)이다. 배열 저장에 O(N) 공간이 필요하다.

## 대안적 접근

- **해시맵 활용**: 배열을 순회하면서 각 원소 x에 대해 M-x가 해시맵에 존재하는지 확인하면 O(N)에 풀 수 있다. 다만 이 문제의 N 범위에서는 브루트포스로도 충분하다.
- **정렬 + 투 포인터**: 배열을 정렬한 뒤 양쪽 끝에서 포인터를 좁혀가며 탐색하면 O(N log N)에 풀 수 있다.
