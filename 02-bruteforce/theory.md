# 브루트포스

## 개념

브루트포스(Brute Force)는 가능한 모든 경우의 수를 하나씩 탐색하여 정답을 찾는 방법이다. 완전 탐색(Exhaustive Search)이라고도 부른다.

이 방법은 문제의 해가 될 수 있는 모든 후보를 체계적으로 나열하고, 각 후보가 조건을 만족하는지 확인하여 정답을 구한다. 알고리즘 설계가 단순하고 정확성이 보장되지만, 경우의 수가 많아지면 시간이 급격히 증가하므로 입력 크기가 작은 문제에 주로 적용한다.

## 동작 원리

브루트포스는 일반적으로 다음 단계를 따른다:

1. 문제에서 가능한 모든 경우의 수를 파악한다
2. 각 경우를 체계적으로 나열한다 (반복문, 재귀 등)
3. 각 경우가 문제의 조건을 만족하는지 검사한다
4. 조건을 만족하는 경우 중 최적의 답을 선택한다

### 주요 탐색 방식

- **반복문 중첩**: 2중, 3중 반복문으로 모든 조합을 탐색한다. 구현이 가장 단순하다.
- **순열/조합 생성**: N개 원소의 모든 순열(N!)이나 조합(2^N)을 생성하여 탐색한다.
- **비트마스크**: 부분집합을 비트로 표현하여 2^N개의 모든 부분집합을 탐색한다.
- **재귀 호출**: 선택/비선택 구조로 모든 경우를 재귀적으로 탐색한다.

### 적용 판단 기준

브루트포스를 적용하기 전에 경우의 수를 먼저 계산해야 한다. 일반적으로 1초에 약 1억(10^8)번의 연산이 가능하므로, 전체 경우의 수가 이 범위 안에 들어오면 브루트포스를 적용할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (최선) | O(N) |
| 시간 복잡도 (평균) | O(N²) ~ O(2^N) |
| 시간 복잡도 (최악) | O(N!) |
| 공간 복잡도 | O(N) ~ O(2^N) |

브루트포스의 복잡도는 탐색 방식에 따라 크게 달라진다. 2중 반복문은 O(N²), 부분집합 탐색은 O(2^N), 순열 탐색은 O(N!)이 일반적이다. 입력 크기 N이 20 이하이면 O(2^N), 10 이하이면 O(N!)도 시간 내에 처리할 수 있다.

## 적합한 문제 유형

- 배열에서 특정 조건을 만족하는 쌍이나 부분집합을 찾는 문제
- 모든 순열이나 조합을 생성하여 최적해를 구하는 문제
- 입력 크기가 작아(N ≤ 20) 완전 탐색이 가능한 문제
- 비트마스크로 상태를 표현할 수 있는 문제
- 최적화 문제에서 정확한 해를 보장해야 하는 문제
- 다른 알고리즘의 정답 검증용 기준 풀이가 필요한 경우


## 단계별 추적 (Trace)

### 예시: 배열에서 합이 K인 쌍 찾기

입력: `arr = [2, 7, 11, 15]`, `K = 9`

```
2중 반복문으로 모든 쌍을 탐색:

i=0, j=1: arr[0]+arr[1] = 2+7  = 9  ✓ 정답 발견! → (0, 1)
i=0, j=2: arr[0]+arr[2] = 2+11 = 13 ✗
i=0, j=3: arr[0]+arr[3] = 2+15 = 17 ✗
i=1, j=2: arr[1]+arr[2] = 7+11 = 18 ✗
i=1, j=3: arr[1]+arr[3] = 7+15 = 22 ✗
i=2, j=3: arr[2]+arr[3] = 11+15= 26 ✗

총 탐색 횟수: C(4,2) = 6번
결과: 인덱스 (0, 1) → 값 (2, 7)
```

### 예시: 부분집합 합 (Subset Sum)

입력: `arr = [3, 1, 4]`, `target = 5`

```
비트마스크로 모든 부분집합 탐색 (2^3 = 8가지):

mask=000 (0): 공집합        → 합 = 0  ✗
mask=001 (1): {3}           → 합 = 3  ✗
mask=010 (2): {1}           → 합 = 1  ✗
mask=011 (3): {3, 1}        → 합 = 4  ✗
mask=100 (4): {4}           → 합 = 4  ✗
mask=101 (5): {3, 4}        → 합 = 7  ✗
mask=110 (6): {1, 4}        → 합 = 5  ✓ 정답!
mask=111 (7): {3, 1, 4}     → 합 = 8  ✗

비트마스크 해석:
  mask=110 → 비트 1(=1), 비트 2(=4) 선택
  ┌─────┬─────┬─────┐
  │ arr[0]=3 │ arr[1]=1 │ arr[2]=4 │
  │  비트 0  │  비트 1  │  비트 2  │
  │    0     │    1     │    1     │  ← mask=110
  └─────┴─────┴─────┘
  선택된 원소: {1, 4} → 합 = 5 = target ✓
```

## 실전 팁

### 활용 노하우

- **경우의 수 먼저 계산**: 코드를 작성하기 전에 전체 경우의 수를 계산하자. N≤20이면 2^N ≈ 100만, N≤10이면 N! ≈ 360만으로 시간 내 처리 가능하다
- **브루트포스를 기준 풀이로 활용**: 최적화된 풀이를 검증할 때 작은 입력에 대해 브루트포스 결과와 비교하면 디버깅에 효과적이다
- **비트마스크 활용**: 부분집합 탐색 시 `for mask := 0; mask < (1 << n); mask++` 패턴을 사용하면 재귀보다 간결하다
- **가지치기(Pruning)**: 순수 브루트포스에 간단한 조건 검사를 추가하면 실행 시간을 크게 줄일 수 있다

### 자주 하는 실수

- **경우의 수 과소평가**: N=20일 때 2^20 = 약 100만이지만, 각 경우에서 O(N) 작업을 하면 총 O(N × 2^N) = 약 2000만이 된다. 내부 연산 비용도 함께 계산해야 한다
- **중복 탐색**: 순서가 중요하지 않은 조합 문제에서 순열처럼 탐색하면 불필요한 중복이 발생한다. 조합은 `j = i+1`부터 시작하자
- **오버플로**: 합이나 곱을 계산할 때 int32 범위를 초과할 수 있다. Go에서는 `int`가 64비트이지만, 명시적으로 확인하자
- **재귀 깊이**: N이 크면 재귀 호출 스택이 깊어져 스택 오버플로가 발생할 수 있다

### 엣지 케이스 주의사항

- **빈 배열**: 입력 배열이 비어 있으면 부분집합은 공집합 하나뿐이다
- **원소가 1개**: 부분집합은 공집합과 자기 자신 2가지이다
- **모든 원소가 동일**: 중복 원소가 있으면 결과에서 중복을 제거해야 하는지 확인하자
- **음수 포함**: 합 문제에서 음수가 포함되면 가지치기 조건이 달라진다
- **target = 0**: 공집합의 합이 0이므로 공집합을 포함할지 문제 조건을 확인하자

## 관련 알고리즘 비교

| 알고리즘 | 핵심 특징 | 시간 복잡도 | 적합한 상황 |
| --- | --- | --- | --- |
| 브루트포스 | 모든 경우의 수를 탐색 | O(2^N) ~ O(N!) | N이 작고 정확한 해가 필요한 경우 |
| 백트래킹 | 브루트포스 + 가지치기 | O(2^N) 이하 | 조건 위반 시 조기 종료 가능한 경우 |
| 동적 프로그래밍 | 부분 문제의 최적해를 재활용 | O(N×W) 등 | 최적 부분 구조와 중복 부분 문제가 있는 경우 |
| 그리디 | 매 단계 최선의 선택 | O(N log N) | 지역 최적 = 전역 최적이 보장되는 경우 |
| 중간에서 만나기 | 탐색 공간을 반으로 분할 | O(2^(N/2)) | N이 30~40 정도로 2^N은 불가하지만 2^(N/2)은 가능한 경우 |

**선택 기준**:
- N ≤ 10 → 순열 기반 브루트포스 (O(N!))
- N ≤ 20 → 비트마스크 브루트포스 (O(2^N))
- N ≤ 40 → 중간에서 만나기 (O(2^(N/2)))
- N이 더 크면 → DP, 그리디, 이분 탐색 등 최적화 기법 필요
