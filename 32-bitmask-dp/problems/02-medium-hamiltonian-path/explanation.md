# 최소 비용 해밀턴 경로 - 해설

## 접근 방식

1. 방문한 도시의 집합을 비트마스크로 표현한다
2. dp[mask][i]를 "방문한 도시 집합이 mask이고 현재 도시가 i일 때의 최소 비용"으로 정의한다
3. 출발 도시가 자유이므로, 모든 도시를 출발점으로 초기화한다: dp[1 << i][i] = 0
4. 모든 상태에 대해, 아직 방문하지 않은 도시로 이동하는 전이를 수행한다
5. 모든 도시를 방문한 상태(mask = 2^N - 1)에서 모든 도시의 dp 값 중 최솟값이 답이다

## 핵심 아이디어

해밀턴 경로 문제는 TSP(외판원 문제)의 변형이다. TSP와의 차이점은 두 가지이다:

- 출발 도시로 돌아올 필요가 없다 → 최종 상태에서 귀환 비용을 더하지 않는다
- 출발 도시를 자유롭게 선택할 수 있다 → 모든 도시를 출발점으로 초기화한다

비트필드 DP의 핵심은 "어떤 도시들을 방문했는가"와 "현재 어디에 있는가"만으로 상태를 정의하는 것이다. 방문 순서가 달라도 이 두 정보가 같으면 이후 최적 경로가 동일하므로, DP가 적용 가능하다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^N × N²) |
| 공간 복잡도 | O(2^N × N) |

상태 수가 2^N × N이고, 각 상태에서 N개의 도시로 전이를 시도하므로 O(2^N × N²)이다. N=16일 때 약 2^16 × 256 ≈ 1,677만으로 시간 내에 충분하다.

## 대안적 접근

- 백트래킹으로 모든 순열을 탐색하면 O(N!)이므로 N=16일 때 시간 초과이다
- 탑다운 메모이제이션 방식으로도 구현할 수 있다. 재귀 함수 solve(mask, cur)를 정의하고 캐시를 활용하는 방식이다
- 출발 도시가 고정된 경우, 초기 상태를 하나만 설정하면 된다
