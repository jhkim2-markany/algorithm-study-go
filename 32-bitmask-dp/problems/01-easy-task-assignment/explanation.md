# 작업 배정 문제 - 해설

## 접근 방식

1. N명의 작업자에게 N개의 작업을 1:1로 배정하는 문제이다
2. 비트마스크로 "어떤 작업들이 이미 배정되었는가"를 표현한다
3. dp[mask]를 "mask에 해당하는 작업들이 배정된 상태에서의 최소 비용"으로 정의한다
4. mask에서 1인 비트의 개수가 k이면, 0번부터 k-1번 작업자까지 배정이 완료된 상태이다
5. k번째 작업자에게 아직 배정되지 않은 작업을 하나씩 시도하며 최솟값을 갱신한다

## 핵심 아이디어

작업 배정 문제는 비트필드 DP의 가장 기본적인 형태이다. 핵심 관찰은 다음과 같다:

- 작업자를 0번부터 순서대로 배정하면, mask에서 1인 비트의 개수가 곧 "다음에 배정할 작업자의 번호"가 된다
- 따라서 별도로 작업자 번호를 DP 상태에 포함할 필요가 없다
- 이 관찰 덕분에 DP 테이블이 1차원(dp[mask])으로 충분하다

순열을 모두 탐색하면 O(N!)이지만, 비트필드 DP를 사용하면 "어떤 작업들이 배정되었는가"만 중요하므로 상태 수가 2^N으로 줄어든다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(2^N × N) |
| 공간 복잡도 | O(2^N) |

상태 수가 2^N이고, 각 상태에서 최대 N개의 작업을 시도하므로 O(2^N × N)이다. N=20일 때 약 2^20 × 20 ≈ 2,097만으로 시간 내에 충분하다.

## 대안적 접근

- 브루트포스로 모든 순열을 탐색하면 O(N!)이므로 N=20일 때 시간 초과이다
- 헝가리안 알고리즘(Hungarian Algorithm)을 사용하면 O(N³)에 해결할 수 있지만, 구현이 복잡하다
- N이 작은 경우(N ≤ 20) 비트필드 DP가 구현이 간단하고 충분히 빠르다
