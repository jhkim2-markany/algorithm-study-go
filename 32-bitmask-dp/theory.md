# 비트필드 DP

## 개념

비트필드 DP(Bitmask DP)는 비트마스크를 이용한 상태 압축과 동적 프로그래밍(DP)을 결합한 기법이다. N개의 원소로 이루어진 집합의 부분집합 상태를 하나의 정수(비트마스크)로 표현하고, 이를 DP 테이블의 인덱스로 사용하여 최적화 문제를 해결한다.

일반적인 DP에서는 상태를 1차원 또는 2차원 배열로 표현하지만, 비트필드 DP에서는 "어떤 원소들이 선택/방문/배정되었는가"라는 집합 상태를 비트마스크로 압축한다. 이를 통해 N!개의 순열을 모두 탐색하는 대신 2^N개의 부분집합 상태만 고려하면 되므로, 지수적으로 탐색 공간을 줄일 수 있다.

예를 들어, 4명의 작업자에게 4개의 작업을 배정하는 문제에서 mask=0b1010(=10)은 "작업 1과 작업 3이 이미 배정된 상태"를 의미한다. 이 상태에서 다음 작업자에게 아직 배정되지 않은 작업(0번 또는 2번)을 배정하는 전이를 수행한다.

## 동작 원리

비트필드 DP의 핵심은 집합 상태를 비트마스크로 압축하여 DP 테이블의 차원으로 사용하는 것이다.

### 1. 상태 정의

DP 테이블을 다음과 같이 정의한다:

- `dp[mask]`: 선택된 원소 집합이 mask일 때의 최적값
- `dp[mask][i]`: 선택된 원소 집합이 mask이고 마지막으로 선택한 원소가 i일 때의 최적값

mask는 N비트 정수로, 각 비트가 해당 원소의 선택 여부를 나타낸다.

### 2. 초기 상태 설정

문제에 따라 초기 상태를 설정한다:

- 아무것도 선택하지 않은 상태: `dp[0] = 0`
- 하나의 원소만 선택한 상태: `dp[1 << i] = cost[i]`

### 3. 상태 전이

현재 상태 mask에서 아직 선택되지 않은 원소 j를 추가하는 전이를 수행한다:

```
if mask & (1 << j) == 0:  // j가 아직 선택되지 않았으면
    nextMask = mask | (1 << j)  // j를 선택한 새로운 상태
    dp[nextMask] = min(dp[nextMask], dp[mask] + cost[j])
```

### 4. 최종 답 계산

모든 원소를 선택한 상태 `(1 << N) - 1`에서의 DP 값이 최종 답이 된다.

### 5. 비트마스크 DP의 전형적 패턴

배정 문제(Assignment Problem)를 예로 들면:

1. N명의 작업자에게 N개의 작업을 1:1로 배정한다
2. `dp[mask]`를 "mask에 해당하는 작업들이 배정된 상태에서의 최소 비용"으로 정의한다
3. mask에서 1인 비트의 개수가 k이면, k번째 작업자까지 배정이 완료된 상태이다
4. k번째 작업자에게 아직 배정되지 않은 작업 j를 배정하는 전이를 수행한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (배정 문제) | O(2^N × N) |
| 시간 복잡도 (TSP) | O(2^N × N²) |
| 시간 복잡도 (부분집합 합) | O(3^N) |
| 공간 복잡도 | O(2^N) 또는 O(2^N × N) |

비트필드 DP의 시간 복잡도는 상태 수 2^N에 각 상태에서의 전이 비용을 곱한 값이다. N ≤ 20 정도까지 실용적으로 사용할 수 있으며, N ≤ 15~16이 가장 일반적이다.

## 적합한 문제 유형

- 작업 배정 문제 (N명에게 N개 작업을 1:1 배정)
- 외판원 문제 (TSP, 모든 도시를 방문하는 최소 비용 경로)
- 해밀턴 경로/순환 문제
- 집합을 여러 그룹으로 분할하는 문제
- 순열 최적화 문제 (순서가 중요한 배치 문제)
- 상태 압축이 필요한 격자 DP (프로필 DP)
- 부분집합에 대한 최적화 문제 (부분집합 합 DP)

## 단계별 추적 (Trace)

### 예시: 작업 배정 문제 (Assignment Problem)

3명의 작업자에게 3개의 작업을 배정할 때 최소 비용을 구한다.

```
입력: 비용 행렬
         작업0  작업1  작업2
작업자0:   9     2     7
작업자1:   6     4     3
작업자2:   5     8     1

상태: dp[mask] = mask에 해당하는 작업들이 배정된 상태의 최소 비용
      mask의 1인 비트 수 = 배정 완료된 작업자 수

초기: dp[0b000] = 0 (아무도 배정 안 됨)

작업자 0 배정 (mask에서 비트 수 = 0 → 작업자 0):
  작업 0 배정: dp[0b001] = dp[0b000] + cost[0][0] = 0 + 9 = 9
  작업 1 배정: dp[0b010] = dp[0b000] + cost[0][1] = 0 + 2 = 2
  작업 2 배정: dp[0b100] = dp[0b000] + cost[0][2] = 0 + 7 = 7

작업자 1 배정 (mask에서 비트 수 = 1 → 작업자 1):
  dp[0b001]에서:
    작업 1 배정: dp[0b011] = min(∞, 9 + 4) = 13
    작업 2 배정: dp[0b101] = min(∞, 9 + 3) = 12
  dp[0b010]에서:
    작업 0 배정: dp[0b011] = min(13, 2 + 6) = 8
    작업 2 배정: dp[0b110] = min(∞, 2 + 3) = 5
  dp[0b100]에서:
    작업 0 배정: dp[0b101] = min(12, 7 + 6) = 12
    작업 1 배정: dp[0b110] = min(5, 7 + 4) = 5

작업자 2 배정 (mask에서 비트 수 = 2 → 작업자 2):
  dp[0b011]=8:  작업 2 배정 → dp[0b111] = min(∞, 8 + 1) = 9
  dp[0b101]=12: 작업 1 배정 → dp[0b111] = min(9, 12 + 8) = 9
  dp[0b110]=5:  작업 0 배정 → dp[0b111] = min(9, 5 + 5) = 10

결과: dp[0b111] = 9
최적 배정: 작업자0→작업1(2), 작업자1→작업2(3), 작업자2→작업0(5) 가 아니라
          작업자0→작업1(2), 작업자1→작업2(3), 작업자2→작업0(5) → 비용 10
          실제 최적: dp[0b011]=8 → 작업자0→작업1(2), 작업자1→작업0(6) → +작업2→작업자2(1) = 9
```

### 예시: TSP (외판원 문제) 추적

```
입력: 4개 도시, 거리 행렬
     0  1  2  3
  0: -  10 15 20
  1: 10 -  35 25
  2: 15 35 -  30
  3: 20 25 30 -

상태: dp[mask][i] = mask에 포함된 도시를 방문하고 현재 도시 i에 있을 때의 최소 비용
시작: 도시 0

초기: dp[0b0001][0] = 0

mask=0b0001 (도시 0 방문):
  → 도시 1: dp[0b0011][1] = 0 + 10 = 10
  → 도시 2: dp[0b0101][2] = 0 + 15 = 15
  → 도시 3: dp[0b1001][3] = 0 + 20 = 20

... (중간 과정 생략)

최종: dp[0b1111][i] + dist[i][0] 의 최솟값
```

## 실전 팁

### 활용 노하우

- N ≤ 20이면 비트필드 DP를 고려하라. N ≤ 15~16이 가장 일반적인 범위이다
- `bits.OnesCount(uint(mask))`로 현재 몇 번째 작업자/단계인지 O(1)에 알 수 있다
- 부분집합 합 DP (SOS DP)는 O(3^N) 대신 O(2^N × N)으로 최적화 가능하다
- TSP에서 시작점을 고정하면 상태 수를 줄일 수 있다

### 자주 하는 실수

- DP 테이블 초기화를 잘못하는 실수 (INF로 초기화해야 하는데 0으로 하는 경우)
- mask에서 비트 수를 세어 현재 단계를 결정할 때 off-by-one 에러
- 전이 시 이미 선택된 원소를 다시 선택하는 실수 (`mask & (1<<j) != 0` 체크 누락)
- 메모리 초과: 2^20 × 20 × 8바이트 ≈ 160MB이므로 N이 크면 메모리 제한 확인 필요

### 엣지 케이스

- N = 1인 경우 (mask = 1, 답이 바로 결정)
- 모든 비용이 동일한 경우
- 비용이 0인 간선이 존재하는 경우
- 도달 불가능한 상태가 있는 경우 (INF 전파 주의)

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 비트필드 DP | O(2^N × N), 상태 압축 | 순열/배정 최적화, N ≤ 20 |
| 백트래킹 | O(N!), 가지치기 가능 | 제약 조건이 많아 가지치기가 효과적일 때 |
| 헝가리안 알고리즘 | O(N³), 배정 문제 전용 | N이 커서 비트필드 DP가 불가능할 때 |
| 일반 DP | O(N²) ~ O(N³) | 상태가 집합이 아닌 경우 |
| 분할 정복 | O(N log N) | 독립적인 부분 문제로 분할 가능할 때 |
| SOS DP (부분집합 합) | O(2^N × N) | 모든 부분집합에 대한 합/최적값 |
