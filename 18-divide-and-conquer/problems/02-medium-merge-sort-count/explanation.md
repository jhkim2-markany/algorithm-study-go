# 병합 정렬 역전 카운트 - 해설

## 접근 방식

1. 브루트포스로 모든 쌍을 확인하면 O(N²)이므로 N이 큰 경우 시간 초과가 발생한다.
2. 병합 정렬의 병합 과정에서 역전의 수를 효율적으로 셀 수 있다.
3. 배열을 반으로 나누어 왼쪽 내부의 역전, 오른쪽 내부의 역전, 왼쪽-오른쪽 간의 교차 역전을 각각 구한다.
4. 교차 역전은 병합 과정에서 오른쪽 원소가 왼쪽 원소보다 먼저 선택될 때 발생한다.

## 핵심 아이디어

병합 정렬의 병합 단계에서 역전을 세는 것이 핵심이다. 두 정렬된 부분 배열을 병합할 때, 오른쪽 부분의 원소 arr[j]가 왼쪽 부분의 원소 arr[i]보다 작으면, arr[i]부터 왼쪽 부분의 끝까지 모든 원소가 arr[j]보다 크므로 (mid - i + 1)개의 역전이 한 번에 카운트된다.

이 방법은 병합 정렬의 구조를 그대로 활용하면서 추가적인 카운팅만 수행하므로, 정렬과 동일한 O(N log N) 시간 복잡도를 유지한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

병합 정렬과 동일한 시간 복잡도 O(N log N)이다. 임시 배열을 위한 O(N) 추가 공간이 필요하다.

## 대안적 접근

- BIT(Binary Indexed Tree) 또는 세그먼트 트리를 사용하여 O(N log N)에 해결할 수도 있다.
- 브루트포스로 모든 쌍을 확인하면 O(N²)이지만, N ≤ 500,000에서는 시간 초과가 발생한다.
