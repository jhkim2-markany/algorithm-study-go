# 분할 정복

## 개념

분할 정복(Divide and Conquer)은 큰 문제를 더 작은 부분 문제로 분할하고, 각 부분 문제를 재귀적으로 해결한 뒤, 그 결과를 합쳐서 원래 문제의 해를 구하는 알고리즘 설계 기법이다. 부분 문제가 충분히 작아지면 직접 해결(base case)한다.

분할 정복의 핵심 특성은 다음과 같다:

- **분할(Divide)**: 원래 문제를 같은 유형의 더 작은 부분 문제로 나눈다
- **정복(Conquer)**: 부분 문제를 재귀적으로 해결한다. 부분 문제가 충분히 작으면 직접 해결한다
- **결합(Combine)**: 부분 문제의 해를 합쳐서 원래 문제의 해를 구한다
- **독립성**: 분할된 부분 문제들은 서로 독립적이어서 개별적으로 해결할 수 있다

분할 정복은 병합 정렬, 퀵 정렬, 이진 탐색, 거듭제곱 계산, 최근접 점 쌍, 카라츠바 곱셈 등에 널리 사용된다.

## 동작 원리

### 기본 분할 정복

1. 현재 문제의 크기가 기저 조건(base case)에 해당하는지 확인한다.
2. 기저 조건이면 직접 해를 구하여 반환한다.
3. 기저 조건이 아니면 문제를 2개 이상의 부분 문제로 분할한다.
4. 각 부분 문제에 대해 재귀적으로 같은 과정을 반복한다.
5. 부분 문제의 결과를 결합하여 현재 문제의 해를 구한다.

### 병합 정렬 (Merge Sort)

분할 정복의 대표적인 예시인 병합 정렬의 과정:

1. 배열의 길이가 1 이하이면 이미 정렬된 상태이므로 반환한다 (기저 조건).
2. 배열을 중간 지점을 기준으로 왼쪽과 오른쪽 두 부분으로 나눈다 (분할).
3. 왼쪽 부분과 오른쪽 부분을 각각 재귀적으로 정렬한다 (정복).
4. 정렬된 두 부분 배열을 하나의 정렬된 배열로 합친다 (결합).

### 최대 부분 배열 합 (Maximum Subarray)

분할 정복으로 최대 부분 배열 합을 구하는 과정:

1. 배열의 길이가 1이면 해당 원소를 반환한다 (기저 조건).
2. 배열을 중간 지점을 기준으로 왼쪽과 오른쪽으로 나눈다 (분할).
3. 왼쪽 부분의 최대 부분 배열 합을 재귀적으로 구한다.
4. 오른쪽 부분의 최대 부분 배열 합을 재귀적으로 구한다.
5. 중간 지점을 걸치는 최대 부분 배열 합을 구한다.
6. 세 값 중 최댓값을 반환한다 (결합).

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 (병합 정렬) | O(N log N) |
| 시간 복잡도 (이진 탐색) | O(log N) |
| 시간 복잡도 (최대 부분 배열) | O(N log N) |
| 시간 복잡도 (최근접 점 쌍) | O(N log² N) 또는 O(N log N) |
| 공간 복잡도 | O(N) 또는 O(log N) (재귀 깊이) |

분할 정복의 시간 복잡도는 마스터 정리(Master Theorem)로 분석할 수 있다. T(N) = aT(N/b) + O(N^d) 형태의 점화식에서 a는 부분 문제의 수, b는 분할 비율, d는 결합 단계의 복잡도 지수이다. 공간 복잡도는 재귀 호출 스택의 깊이 O(log N)에 추가 배열 사용 여부에 따라 달라진다.

## 적합한 문제 유형

- 병합 정렬, 퀵 정렬 등 정렬 알고리즘
- 이진 탐색 (정렬된 배열에서 원소 찾기)
- 최대 부분 배열 합 (Maximum Subarray)
- 최근접 점 쌍 (Closest Pair of Points)
- 거듭제곱 빠르게 계산 (Fast Exponentiation)
- 카라츠바 곱셈 (큰 수의 곱셈)
- 역전 카운트 (Inversion Count)
- 행렬 곱셈 (슈트라센 알고리즘)
- 볼록 껍질 (Convex Hull)


## 단계별 추적 (Trace)

### 예시 입력

병합 정렬로 배열 `[38, 27, 43, 3, 9, 82, 10]`을 정렬하는 과정을 추적한다.

### 실행 과정

**분할 단계**

```
                [38, 27, 43, 3, 9, 82, 10]
                /                         \
        [38, 27, 43, 3]            [9, 82, 10]
         /           \              /         \
     [38, 27]     [43, 3]       [9, 82]      [10]
      /    \       /    \        /    \
    [38]  [27]  [43]   [3]    [9]   [82]
```

**결합(병합) 단계**

```
단계 1: 리프 노드 병합
  [38] + [27] → [27, 38]    (27 < 38)
  [43] + [3]  → [3, 43]     (3 < 43)
  [9] + [82]  → [9, 82]     (9 < 82)
  [10]        → [10]         (원소 1개)

단계 2: 중간 노드 병합
  [27, 38] + [3, 43]:
    3 < 27 → [3]
    27 < 43 → [3, 27]
    38 < 43 → [3, 27, 38]
    남은 43 → [3, 27, 38, 43]

  [9, 82] + [10]:
    9 < 10 → [9]
    10 < 82 → [9, 10]
    남은 82 → [9, 10, 82]

단계 3: 최종 병합
  [3, 27, 38, 43] + [9, 10, 82]:
    3 < 9   → [3]
    9 < 27  → [3, 9]
    10 < 27 → [3, 9, 10]
    27 < 82 → [3, 9, 10, 27]
    38 < 82 → [3, 9, 10, 27, 38]
    43 < 82 → [3, 9, 10, 27, 38, 43]
    남은 82 → [3, 9, 10, 27, 38, 43, 82]
```

**최종 결과: [3, 9, 10, 27, 38, 43, 82]**

## 실전 팁

### 활용 노하우

- 분할 정복 문제를 만나면 "분할 기준", "기저 조건", "결합 방법" 세 가지를 먼저 정의한다
- 마스터 정리를 활용하면 시간 복잡도를 빠르게 계산할 수 있다: T(N) = aT(N/b) + O(N^d)에서 log_b(a)와 d를 비교한다
- 역전 카운트(Inversion Count) 문제는 병합 정렬의 병합 단계에서 역전 수를 세는 방식으로 O(N log N)에 해결할 수 있다
- 최근접 점 쌍 문제는 x좌표 기준 분할 후, 경계 영역의 점들만 추가 확인하는 방식으로 효율적으로 풀 수 있다

### 자주 하는 실수

- 기저 조건을 잘못 설정하여 무한 재귀에 빠지는 경우: 배열 크기가 0 또는 1일 때 반환하도록 한다
- 분할 시 중간 인덱스 계산에서 오버플로: `mid = lo + (hi - lo) / 2`를 사용한다
- 병합 시 임시 배열을 매번 새로 할당하면 성능이 저하된다. 미리 할당해두고 재사용한다
- 분할이 균등하지 않으면 최악의 경우 O(N²)이 될 수 있다 (퀵 정렬의 경우)

### 엣지 케이스 주의사항

- 빈 배열 또는 원소가 1개인 배열: 기저 조건으로 즉시 반환
- 모든 원소가 동일한 경우: 정렬은 문제없지만, 분할 기준 선택에 주의 (퀵 정렬)
- 배열 크기가 2인 경우: 분할 후 각각 크기 1이 되어 바로 병합
- 매우 큰 배열: 재귀 깊이 O(log N)이므로 스택 오버플로 위험은 낮지만, 메모리 사용량에 주의

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 분할 정복 | 독립적 부분 문제로 분할 후 결합 | 병합 정렬, 최근접 점 쌍, 거듭제곱 |
| 동적 프로그래밍 | 중복 부분 문제를 메모이제이션으로 해결 | 부분 문제가 겹치는 최적화 문제 |
| 그리디 | 매 단계 최선의 선택, 전체 최적 보장 | 최적 부분 구조 + 탐욕 선택 성질 |
| 백트래킹 | DFS + 가지치기 | 조합 탐색, 제약 조건 만족 문제 |
| 브루트포스 | 모든 경우 탐색 | 탐색 공간이 작은 문제 |

**분할 정복 vs 동적 프로그래밍 핵심 차이:**
- 분할 정복: 부분 문제가 서로 독립적 (겹치지 않음)
- 동적 프로그래밍: 부분 문제가 서로 겹침 (중복 계산 발생)
