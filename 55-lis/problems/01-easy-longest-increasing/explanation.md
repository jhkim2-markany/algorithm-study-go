# 최장 증가 부분 수열 길이 - 해설

## 접근 방식

1. `tails` 배열을 유지한다. `tails[k]`는 길이 `k+1`인 증가 부분 수열의 마지막 원소 최솟값이다
2. 수열의 각 원소 `x`에 대해:
   - `x`가 `tails`의 마지막 원소보다 크면 `tails` 끝에 추가한다
   - 그렇지 않으면 `tails`에서 `x` 이상인 첫 위치를 이분 탐색으로 찾아 교체한다
3. 최종 `tails` 배열의 길이가 LIS 길이이다

## 핵심 아이디어

`tails` 배열은 항상 정렬된 상태를 유지한다. 각 위치의 값을 가능한 한 작게 유지하면, 이후에 더 많은 원소가 뒤에 붙을 가능성이 높아진다.

Go의 `sort.SearchInts`를 사용하면 lower_bound를 간편하게 구현할 수 있다. 이 함수는 정렬된 슬라이스에서 주어진 값 이상인 첫 번째 인덱스를 반환한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

각 원소마다 이분 탐색 O(log N)을 수행하므로 전체 O(N log N)이다.

## 대안적 접근

O(N²) DP 방식도 가능하다. `dp[i]`를 `a[i]`를 마지막으로 하는 LIS 길이로 정의하고, 이전 원소들을 순회하며 갱신한다. N ≤ 5,000 정도까지는 사용 가능하지만, N ≤ 100,000에서는 시간 초과이다.
