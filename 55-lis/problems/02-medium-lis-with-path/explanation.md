# LIS 경로 역추적 - 해설

## 접근 방식

1. O(N log N) 이분 탐색으로 LIS 길이를 구하면서, 각 원소가 `tails`에서 들어간 위치 `pos[i]`를 기록한다
2. 역추적: 배열 끝에서부터 앞으로 순회하며, `pos[i]`가 현재 찾는 위치와 일치하는 원소를 LIS에 추가한다
3. LIS 길이 `L`에서 시작하여 `L-1, L-2, ..., 0` 순서로 원소를 찾는다

## 핵심 아이디어

`pos[i]`는 원소 `a[i]`가 LIS에서 차지할 수 있는 위치(0-indexed)를 나타낸다. 역순으로 순회하면서 `pos[i] == target`인 원소를 찾으면, 그 원소가 LIS의 해당 위치에 올 수 있는 가장 마지막(오른쪽) 원소이다.

역순 순회가 정확한 이유: 같은 `pos` 값을 가진 원소가 여러 개일 수 있지만, 뒤에서부터 찾으면 가장 나중에 갱신된 값을 선택하게 되어 유효한 LIS를 복원할 수 있다.

```
pos[i] = a[i]가 tails에서 들어간 인덱스
역추적: i를 n-1부터 0까지 순회하며 pos[i] == target인 원소 선택
```

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N) |

LIS 계산 O(N log N) + 역추적 O(N) = O(N log N)이다.

## 대안적 접근

O(N²) DP 방식에서는 `dp[i]`와 함께 이전 원소의 인덱스 `prev[i]`를 기록하면 역추적이 더 직관적이다. `dp` 최댓값 위치에서 `prev`를 따라가면 LIS를 복원할 수 있다. 다만 N이 크면 시간 초과이다.
