# LIS 개수 세기 - 해설

## 접근 방식

1. O(N²) DP를 사용하여 각 위치에서 끝나는 LIS의 길이 `dp[i]`와 개수 `cnt[i]`를 동시에 계산한다
2. 각 `i`에 대해 `j = 0 ~ i-1`을 순회하며:
   - `a[j] < a[i]`이고 `dp[j]+1 > dp[i]`이면: 더 긴 LIS 발견, `dp[i]` 갱신 및 `cnt[i] = cnt[j]`
   - `a[j] < a[i]`이고 `dp[j]+1 == dp[i]`이면: 같은 길이, `cnt[i] += cnt[j]`
3. 전체 LIS 길이 `L`을 구한 뒤, `dp[i] == L`인 모든 `cnt[i]`를 합산한다

## 핵심 아이디어

LIS 길이뿐 아니라 개수를 세려면, 각 위치에서 끝나는 LIS의 경우의 수를 추적해야 한다. `cnt[i]`는 `a[i]`를 마지막 원소로 하는 LIS의 가짓수이다.

점화식:

```
dp[i] = max(dp[j] + 1)  (a[j] < a[i], j < i)
cnt[i] = Σ cnt[j]       (a[j] < a[i], j < i, dp[j]+1 == dp[i])
```

`dp[j]+1 > dp[i]`인 경우 더 긴 LIS를 발견한 것이므로 `cnt[i]`를 `cnt[j]`로 초기화한다. `dp[j]+1 == dp[i]`인 경우 같은 길이의 다른 경로이므로 `cnt[i]`에 `cnt[j]`를 누적한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N²) |
| 공간 복잡도 | O(N) |

N ≤ 5,000이므로 O(N²)이 충분하다.

## 대안적 접근

N이 더 큰 경우(N ≤ 100,000), 세그먼트 트리나 BIT(Binary Indexed Tree)를 활용하여 O(N log N)에 LIS 개수를 구할 수 있다. 좌표 압축 후 세그먼트 트리에 (최대 길이, 해당 길이의 개수) 쌍을 저장하고 구간 쿼리로 갱신한다.
