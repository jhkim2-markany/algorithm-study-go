# LIS (최장 증가 부분 수열)

## 개념

LIS(Longest Increasing Subsequence, 최장 증가 부분 수열)란 주어진 수열에서 원소의 순서를 유지하면서 값이 순증가하는 부분 수열 중 가장 긴 것을 말한다.

예를 들어 수열 `[10, 20, 10, 30, 20, 50]`에서 LIS는 `[10, 20, 30, 50]`이고 길이는 4이다.

핵심 정의:
- **부분 수열(Subsequence)**: 원래 수열에서 일부 원소를 골라 순서를 유지한 수열. 연속일 필요는 없다
- **증가(Increasing)**: 선택한 원소들이 순증가(strictly increasing)한다. 즉 `a[i] < a[j]` (i < j)
- **최장(Longest)**: 가능한 증가 부분 수열 중 길이가 가장 긴 것

LIS는 코딩테스트의 클래식 유형으로, DP와 이분 탐색의 결합을 묻는 대표적인 문제이다.

## 동작 원리

### 1. O(N²) DP 방식

가장 직관적인 방법이다. `dp[i]`를 `a[i]`를 마지막 원소로 하는 LIS의 길이로 정의한다.

**점화식:**

```
dp[i] = max(dp[j] + 1)  (0 ≤ j < i, a[j] < a[i])
dp[i] = 1               (조건을 만족하는 j가 없는 경우)
```

**알고리즘:**

1. 모든 `dp[i]`를 1로 초기화한다 (자기 자신만으로 길이 1인 LIS)
2. 각 `i`에 대해, `j = 0 ~ i-1`을 순회하며 `a[j] < a[i]`인 경우 `dp[i] = max(dp[i], dp[j]+1)`
3. 전체 `dp` 배열의 최댓값이 LIS 길이이다

### 2. O(N log N) 이분 탐색 방식

`tails` 배열을 유지하며, `tails[k]`는 길이 `k+1`인 증가 부분 수열의 마지막 원소의 최솟값을 저장한다.

**핵심 관찰:** `tails` 배열은 항상 정렬된 상태를 유지한다.

**알고리즘:**

1. 빈 `tails` 배열을 준비한다
2. 수열의 각 원소 `x`에 대해:
   - `x`가 `tails`의 마지막 원소보다 크면, `tails` 끝에 `x`를 추가한다
   - 그렇지 않으면, `tails`에서 `x` 이상인 첫 번째 위치를 이분 탐색(lower_bound)으로 찾아 그 위치의 값을 `x`로 교체한다
3. 최종 `tails` 배열의 길이가 LIS 길이이다

**왜 동작하는가:**

`tails[k]`를 가능한 한 작게 유지하면, 이후에 더 많은 원소가 뒤에 붙을 수 있는 가능성이 높아진다. 이분 탐색으로 교체할 위치를 O(log N)에 찾으므로 전체 O(N log N)이다.

**주의:** 최종 `tails` 배열은 실제 LIS가 아니다. LIS의 길이만 정확하다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| O(N²) DP 방식 - 시간 | O(N²) |
| O(N log N) 이분 탐색 방식 - 시간 | O(N log N) |
| 공간 복잡도 (기본) | O(N) |
| 공간 복잡도 (경로 역추적 포함) | O(N) |

## 적합한 문제 유형

- 수열에서 가장 긴 증가/감소 부분 수열의 길이를 구하는 문제
- 상자 쌓기, 전깃줄 교차 최소화 등 LIS로 모델링되는 문제
- 수열을 최소 개수의 감소하지 않는 부분 수열로 분할하는 문제 (딜워스 정리)
- 2차원 좌표에서 특정 조건을 만족하는 최장 체인을 구하는 문제
- 순열에서 정렬에 필요한 최소 연산 수를 구하는 문제

## 단계별 추적 (Trace)

### 예시 1: O(N²) DP 방식

수열: `[10, 20, 10, 30, 20, 50]`

```
초기: dp = [1, 1, 1, 1, 1, 1]

i=0: a[0]=10
  조건 만족하는 j 없음
  dp[0] = 1

i=1: a[1]=20
  j=0: a[0]=10 < 20 → dp[1] = max(1, dp[0]+1) = 2
  dp[1] = 2

i=2: a[2]=10
  j=0: a[0]=10, 10 < 10? NO
  j=1: a[1]=20, 20 < 10? NO
  dp[2] = 1

i=3: a[3]=30
  j=0: a[0]=10 < 30 → dp[3] = max(1, 1+1) = 2
  j=1: a[1]=20 < 30 → dp[3] = max(2, 2+1) = 3
  j=2: a[2]=10 < 30 → dp[3] = max(3, 1+1) = 3
  dp[3] = 3

i=4: a[4]=20
  j=0: a[0]=10 < 20 → dp[4] = max(1, 1+1) = 2
  j=1: a[1]=20, 20 < 20? NO
  j=2: a[2]=10 < 20 → dp[4] = max(2, 1+1) = 2
  j=3: a[3]=30, 30 < 20? NO
  dp[4] = 2

i=5: a[5]=50
  j=0: a[0]=10 < 50 → dp[5] = max(1, 1+1) = 2
  j=1: a[1]=20 < 50 → dp[5] = max(2, 2+1) = 3
  j=2: a[2]=10 < 50 → dp[5] = max(3, 1+1) = 3
  j=3: a[3]=30 < 50 → dp[5] = max(3, 3+1) = 4
  j=4: a[4]=20 < 50 → dp[5] = max(4, 2+1) = 4
  dp[5] = 4

최종: dp = [1, 2, 1, 3, 2, 4]
LIS 길이 = max(dp) = 4
```

### 예시 2: O(N log N) 이분 탐색 방식

수열: `[10, 20, 10, 30, 20, 50]`

```
tails = []

x=10: tails 비어있음 → 추가
  tails = [10]

x=20: 20 > tails[-1]=10 → 추가
  tails = [10, 20]

x=10: 10 ≤ tails[-1]=20 → lower_bound(10) = 인덱스 0 → 교체
  tails = [10, 20]  (tails[0]=10을 10으로 교체, 변화 없음)

x=30: 30 > tails[-1]=20 → 추가
  tails = [10, 20, 30]

x=20: 20 ≤ tails[-1]=30 → lower_bound(20) = 인덱스 1 → 교체
  tails = [10, 20, 30]  (tails[1]=20을 20으로 교체, 변화 없음)

x=50: 50 > tails[-1]=30 → 추가
  tails = [10, 20, 30, 50]

최종: len(tails) = 4
LIS 길이 = 4
```

### 예시 3: tails 배열 변화가 뚜렷한 경우

수열: `[3, 5, 6, 2, 5, 4, 19, 5, 6, 7, 12]`

```
tails = []

x=3:  추가         → tails = [3]
x=5:  추가         → tails = [3, 5]
x=6:  추가         → tails = [3, 5, 6]
x=2:  교체 idx=0   → tails = [2, 5, 6]
x=5:  교체 idx=1   → tails = [2, 5, 6]  (변화 없음)
x=4:  교체 idx=1   → tails = [2, 4, 6]
x=19: 추가         → tails = [2, 4, 6, 19]
x=5:  교체 idx=2   → tails = [2, 4, 5, 19]
x=6:  교체 idx=3   → tails = [2, 4, 5, 6]
x=7:  추가         → tails = [2, 4, 5, 6, 7]
x=12: 추가         → tails = [2, 4, 5, 6, 7, 12]

LIS 길이 = 6
```

## 실전 팁

### 활용 노하우

- 코딩테스트에서 LIS 문제는 거의 항상 O(N log N) 풀이를 요구한다. N ≤ 100,000이면 O(N²)은 시간 초과이다
- Go에서 `sort.SearchInts(tails, x)`를 사용하면 lower_bound를 간편하게 구현할 수 있다
- LIS 문제인지 판별하는 핵심 키워드: "가장 긴 증가", "최소 개수의 감소 수열로 분할", "전깃줄 교차 최소화"
- 딜워스 정리: 수열을 감소하지 않는 부분 수열로 분할하는 최소 개수 = LIS 길이. 이를 활용하면 분할 문제를 LIS로 변환할 수 있다
- 2차원 LIS (좌표 정렬 후 한 축에 대해 LIS)는 정렬 기준에 주의하자. 첫 번째 키 오름차순, 두 번째 키 내림차순으로 정렬하면 순수 LIS로 환원된다

### 자주 하는 실수

- `tails` 배열을 실제 LIS로 착각하는 실수. `tails`는 LIS 길이만 정확하며, 실제 LIS 원소를 구하려면 별도의 역추적이 필요하다
- 순증가(strictly increasing)와 비감소(non-decreasing)를 혼동하는 실수. 문제 조건을 정확히 확인하자. 비감소 LIS는 lower_bound 대신 upper_bound를 사용한다
- 이분 탐색에서 lower_bound와 upper_bound를 혼동하는 실수. 순증가 LIS는 lower_bound, 비감소 LIS는 upper_bound이다
- 경로 역추적 시 인덱스 관리를 잘못하는 실수. 각 원소가 `tails`에서 어느 위치에 들어갔는지를 기록해야 한다

### 엣지 케이스

- N = 1: LIS 길이는 항상 1이다
- 이미 정렬된 수열: LIS 길이 = N
- 역순 정렬된 수열: LIS 길이 = 1
- 모든 원소가 같은 수열: 순증가 LIS 길이 = 1
- 원소 값이 음수인 경우: 알고리즘에 영향 없음, 비교 연산만 사용하므로 정상 동작

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| LIS (O(N²) DP) | 직관적, 경로 역추적 용이 | N ≤ 5,000, 실제 LIS 원소 필요 |
| LIS (O(N log N) 이분 탐색) | 빠름, 길이만 정확 | N ≤ 100,000, 길이만 필요 |
| LCS (최장 공통 부분 수열) | 두 수열의 공통 부분 수열 | 두 수열 간 유사도 비교 |
| LIS + 역추적 | O(N log N)에 실제 LIS 복원 | 길이와 실제 수열 모두 필요 |
| 세그먼트 트리 LIS | 좌표 압축 + 구간 최댓값 | 2차원 LIS, 조건부 LIS |

- 단순 LIS 길이 → O(N log N) 이분 탐색
- LIS 원소 복원 → O(N log N) + 역추적 또는 O(N²) DP
- 두 수열의 공통 부분 수열 → LCS
- 조건부 LIS (가중치, 2차원) → 세그먼트 트리 활용

## 변형 및 최적화

### LDS (최장 감소 부분 수열)

수열을 뒤집거나 비교 연산을 반대로 하면 LIS와 동일한 알고리즘으로 구할 수 있다.

```
LDS(a) = LIS(reverse(a))
또는
LDS(a) = LIS(-a[0], -a[1], ..., -a[n-1])
```

### LNDS (최장 비감소 부분 수열)

순증가 대신 비감소(같은 값 허용)를 구하는 변형이다. 이분 탐색에서 lower_bound 대신 upper_bound를 사용한다.

```
순증가 LIS: tails에서 x 이상인 첫 위치 (lower_bound)
비감소 LNDS: tails에서 x 초과인 첫 위치 (upper_bound)
```

### LIS 경로 역추적

O(N log N) 방식에서 실제 LIS 원소를 복원하는 방법이다.

1. 각 원소 `a[i]`가 `tails`에서 들어간 위치 `pos[i]`를 기록한다
2. LIS 길이 `L`부터 역순으로, `pos[i] == L-1`인 가장 마지막 `i`를 찾아 LIS에 추가한다
3. `L`을 1 감소시키고 반복한다

### 딜워스 정리 활용

수열을 최소 개수의 감소하지 않는 부분 수열로 분할하는 문제는 LIS 길이와 같다. 이는 딜워스 정리(Dilworth's theorem)에 의한 것이다.

```
최소 분할 수 = LIS 길이
```
