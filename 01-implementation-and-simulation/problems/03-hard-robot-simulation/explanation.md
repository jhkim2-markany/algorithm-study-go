# 로봇 시뮬레이션 - 해설

## 접근 방식

1. R×C 격자와 N개 로봇의 초기 상태를 입력받는다
2. 격자 배열에 각 로봇의 위치를 기록하여 충돌 감지에 활용한다
3. 로봇 번호 순서대로, 각 로봇이 M개의 명령을 모두 수행한다
4. 각 명령에 따라 회전 또는 전진을 처리한다
5. 전진 시 격자 범위와 다른 로봇과의 충돌을 검사한다
6. 모든 로봇의 명령 수행이 끝나면 최종 상태를 출력한다

## 핵심 아이디어

이 문제의 핵심은 여러 조건을 정확히 처리하는 구현 능력이다:

1. **방향 회전 처리**: 방향을 1~4 숫자로 표현하고, 좌회전과 우회전을 모듈러 연산으로 처리한다. 우회전은 `d%4 + 1`, 좌회전은 `(d+2)%4 + 1`로 계산한다.

2. **충돌 감지**: 격자 배열에 로봇 번호를 기록하여 O(1)에 특정 위치에 로봇이 있는지 확인한다. 로봇이 이동할 때 이전 위치를 비우고 새 위치에 기록하는 것을 잊지 않아야 한다.

3. **순차 처리**: 로봇이 동시에 움직이는 것이 아니라 번호 순서대로 모든 명령을 수행한다. 따라서 앞 번호 로봇의 이동 결과가 뒷 번호 로봇의 이동에 영향을 준다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × M) |
| 공간 복잡도 | O(R × C) |

N개의 로봇이 각각 M개의 명령을 수행하므로 시간 복잡도는 O(N × M)이다. 격자 배열을 저장하기 위해 O(R × C)의 공간이 필요하다.

## 대안적 접근

- **해시맵 기반 충돌 감지**: 격자 배열 대신 해시맵에 로봇 위치를 저장하면 격자가 매우 클 때 메모리를 절약할 수 있다. 다만 이 문제의 제약 조건(R, C ≤ 100)에서는 배열이 더 효율적이다.
- **동시 이동 변형**: 만약 모든 로봇이 동시에 이동하는 변형 문제라면, 각 단계에서 모든 로봇의 다음 위치를 먼저 계산한 뒤 충돌을 일괄 처리해야 한다. 이 경우 구현 난이도가 크게 올라간다.
