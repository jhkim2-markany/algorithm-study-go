# 나선형 행렬 - 해설

## 접근 방식

1. N×N 크기의 빈 행렬을 생성한다
2. 시작 위치 (0, 0)에서 오른쪽 방향으로 출발한다
3. 1부터 N²까지 순서대로 현재 위치에 값을 채운다
4. 다음 위치가 범위를 벗어나거나 이미 채워져 있으면 방향을 시계 방향으로 전환한다
5. 모든 칸을 채울 때까지 반복한다

## 핵심 아이디어

방향 배열(dx, dy)을 사용하여 오른쪽 → 아래 → 왼쪽 → 위 순서로 이동 방향을 관리한다. 현재 방향으로 진행할 수 없을 때(경계 초과 또는 이미 채워진 칸) 다음 방향으로 전환하는 것이 핵심이다.

이미 채워진 칸을 판별하기 위해 행렬의 초기값 0을 활용한다. 값이 1부터 시작하므로 0인 칸은 아직 채워지지 않은 칸이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N²) |
| 공간 복잡도 | O(N²) |

N×N 행렬의 모든 칸을 정확히 한 번씩 방문하므로 시간 복잡도는 O(N²)이다. N×N 행렬을 저장하기 위한 공간이 필요하므로 공간 복잡도도 O(N²)이다.

## 대안적 접근

- **경계 변수 방식**: top, bottom, left, right 네 개의 경계 변수를 두고, 한 방향의 순회가 끝날 때마다 해당 경계를 줄여나가는 방식이다. 방향 전환 로직이 더 명시적이지만 코드가 다소 길어진다.
- **레이어 방식**: 바깥 테두리부터 안쪽 테두리까지 한 겹씩 채워나가는 방식이다. 각 레이어를 독립적으로 처리할 수 있어 직관적이다.
