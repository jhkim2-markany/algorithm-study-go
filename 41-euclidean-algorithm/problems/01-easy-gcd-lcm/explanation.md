# GCD와 LCM 계산 - 해설

## 접근 방식

1. 유클리드 호제법을 사용하여 두 수의 GCD를 구한다.
2. GCD(a, b)는 a를 b로 나눈 나머지를 반복적으로 구하여, 나머지가 0이 되는 시점의 나누는 수가 GCD이다.
3. LCM은 GCD를 이용하여 LCM(a, b) = a / GCD(a, b) * b로 계산한다.
4. 오버플로를 방지하기 위해 a * b를 먼저 계산하지 않고, a를 GCD로 나눈 뒤 b를 곱한다.

## 핵심 아이디어

유클리드 호제법의 핵심은 GCD(a, b) = GCD(b, a mod b)라는 성질이다. a를 b로 나눈 나머지로 대체해도 GCD가 변하지 않으므로, 나머지가 0이 될 때까지 반복하면 GCD를 얻을 수 있다. 이 알고리즘은 단순 약수 열거(O(min(a,b)))보다 훨씬 효율적인 O(log(min(a,b)))의 시간 복잡도를 가진다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(log(min(A, B))) |
| 공간 복잡도 | O(1) |

유클리드 호제법의 반복 횟수는 최악의 경우에도 log에 비례하며, 추가 메모리를 사용하지 않는다.

## 대안적 접근

1부터 min(A, B)까지 모든 수로 나누어 보는 방법도 있지만, O(min(A, B))의 시간이 걸려 비효율적이다. 유클리드 호제법은 이보다 훨씬 빠르게 동작한다.
