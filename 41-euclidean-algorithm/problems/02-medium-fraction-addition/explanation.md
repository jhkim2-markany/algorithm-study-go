# 분수의 덧셈 - 해설

## 접근 방식

1. 결과 분수를 0/1로 초기화한다.
2. 각 분수를 순서대로 읽으며 현재 결과에 더한다.
3. 두 분수를 더할 때 LCM을 이용하여 통분한다: a/b + c/d에서 공통 분모를 LCM(b, d)로 설정한다.
4. 매 덧셈 후 GCD로 약분하여 중간 결과의 오버플로를 방지한다.
5. 최종 결과를 기약분수로 출력한다.

## 핵심 아이디어

분수의 덧셈에서 유클리드 호제법은 두 가지 역할을 한다. 첫째, LCM을 구하여 효율적으로 통분한다. 단순히 두 분모를 곱하면 수가 급격히 커지지만, LCM을 사용하면 최소한의 공통 분모를 얻을 수 있다. 둘째, 매 단계마다 GCD로 약분하여 수의 크기를 관리한다.

음수 분자를 처리할 때 GCD 함수에서 절댓값을 사용해야 한다는 점에 주의한다. 또한 분모가 음수가 되는 경우 부호를 분자로 이동시켜 분모를 항상 양수로 유지한다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N × log(D)) |
| 공간 복잡도 | O(1) |

N은 분수의 개수, D는 분모의 최댓값이다. 각 분수를 더할 때마다 GCD/LCM 연산에 O(log D)가 소요된다.

## 대안적 접근

모든 분모의 LCM을 먼저 구한 뒤 한 번에 통분하는 방법도 있다. 이 경우 LCM이 매우 커질 수 있어 큰 수 처리가 필요할 수 있다. 순차적으로 더하면서 매번 약분하는 방식이 오버플로에 더 안전하다.
