# 확장 유클리드와 모듈러 역원 - 해설

## 접근 방식

1. 각 쿼리에 대해 확장 유클리드 알고리즘을 사용하여 모듈러 역원을 구한다.
2. 먼저 GCD(A, M)을 확인한다. GCD가 1이 아니면 역원이 존재하지 않으므로 -1을 출력한다.
3. GCD가 1이면 확장 유클리드 알고리즘으로 Ax + My = 1을 만족하는 x를 구한다.
4. x가 음수일 수 있으므로 (x % M + M) % M으로 0 이상 M 미만의 값으로 변환한다.

## 핵심 아이디어

확장 유클리드 알고리즘은 GCD를 구하는 과정에서 베주 항등식 ax + by = GCD(a, b)의 계수 x, y를 동시에 구한다. 기저 조건(b = 0)에서 x = 1, y = 0으로 시작하여, 재귀 호출이 반환될 때마다 x = y₁, y = x₁ - (a/b) × y₁로 역추적한다.

모듈러 역원은 Ax ≡ 1 (mod M), 즉 Ax + My = 1을 만족하는 x이다. 이 식은 GCD(A, M) = 1일 때만 정수 해가 존재한다(베주 항등식). 확장 유클리드 알고리즘으로 구한 x가 바로 모듈러 역원이 된다.

대량의 쿼리를 처리해야 하므로 bufio를 사용한 빠른 입출력이 필수적이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(Q × log(min(A, M))) |
| 공간 복잡도 | O(log(min(A, M))) |

각 쿼리당 확장 유클리드 알고리즘의 재귀 깊이는 O(log(min(A, M)))이다. Q개의 쿼리를 처리하므로 전체 시간 복잡도는 O(Q × log(min(A, M)))이다.

## 대안적 접근

M이 소수인 경우 페르마의 소정리를 이용하여 a^(M-2) mod M으로 역원을 구할 수 있다. 이 방법은 분할 정복 거듭제곱(36-exponentiation-by-squaring)을 사용하며, 시간 복잡도는 O(log M)이다. 그러나 M이 소수가 아닌 일반적인 경우에는 확장 유클리드 알고리즘이 필요하다.
