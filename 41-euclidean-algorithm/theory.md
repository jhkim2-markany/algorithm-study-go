# 유클리드 호제법

## 개념

유클리드 호제법(Euclidean Algorithm)은 두 정수의 최대공약수(GCD, Greatest Common Divisor)를 효율적으로 구하는 알고리즘이다. 기원전 300년경 유클리드가 제안한 것으로, 현존하는 가장 오래된 알고리즘 중 하나이다.

핵심 원리는 다음과 같다: 두 수 a, b (a > b)에 대해 GCD(a, b) = GCD(b, a mod b)가 성립한다. 이 성질을 반복 적용하면 나머지가 0이 되는 시점에서 GCD를 얻을 수 있다.

유클리드 호제법의 주요 확장 개념:

- **최소공배수(LCM)**: GCD를 이용하여 LCM(a, b) = a * b / GCD(a, b)로 계산한다
- **확장 유클리드 알고리즘**: GCD뿐 아니라 ax + by = GCD(a, b)를 만족하는 정수 x, y도 함께 구한다
- **모듈러 역원**: 확장 유클리드 알고리즘을 활용하여 모듈러 연산에서의 곱셈 역원을 구할 수 있다

## 동작 원리

### 1. 기본 유클리드 호제법

두 수 a, b의 GCD를 구하는 과정:

1. a를 b로 나눈 나머지 r을 구한다 (r = a mod b)
2. 나머지 r이 0이면 b가 GCD이다
3. 나머지 r이 0이 아니면 a를 b로, b를 r로 대체한다
4. 1~3 과정을 나머지가 0이 될 때까지 반복한다

예시: GCD(48, 18)
- 48 mod 18 = 12 → GCD(18, 12)
- 18 mod 12 = 6 → GCD(12, 6)
- 12 mod 6 = 0 → GCD = 6

### 2. 최소공배수(LCM) 계산

GCD를 이용하여 LCM을 구하는 과정:

1. 두 수 a, b의 GCD를 구한다
2. LCM(a, b) = a / GCD(a, b) * b로 계산한다
3. 오버플로를 방지하기 위해 a * b를 먼저 계산하지 않고, a를 GCD로 나눈 뒤 b를 곱한다

### 3. 확장 유클리드 알고리즘

ax + by = GCD(a, b)를 만족하는 정수 x, y를 구하는 과정:

1. 기저 조건: b = 0이면 GCD = a이고, x = 1, y = 0이다
2. 재귀적으로 GCD(b, a mod b)에 대해 x₁, y₁을 구한다
3. 역추적하여 x = y₁, y = x₁ - (a/b) * y₁로 계산한다
4. 이 과정을 통해 GCD와 함께 베주 항등식의 계수를 얻는다

### 4. 모듈러 역원

a의 모듈러 m에 대한 역원(a⁻¹ mod m)을 구하는 과정:

1. GCD(a, m) = 1인지 확인한다 (서로소일 때만 역원이 존재)
2. 확장 유클리드 알고리즘으로 ax + my = 1을 만족하는 x를 구한다
3. x를 m으로 나눈 나머지가 a의 모듈러 역원이다
4. x가 음수일 수 있으므로 (x % m + m) % m으로 양수로 변환한다

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 기본 유클리드 호제법 | O(log(min(a, b))) |
| 확장 유클리드 알고리즘 | O(log(min(a, b))) |
| LCM 계산 | O(log(min(a, b))) |
| 공간 복잡도 (반복) | O(1) |
| 공간 복잡도 (재귀) | O(log(min(a, b))) |

유클리드 호제법의 반복 횟수는 피보나치 수열과 관련이 있으며, 최악의 경우에도 O(log(min(a, b)))번의 나머지 연산으로 GCD를 구할 수 있다.

## 적합한 문제 유형

- 두 수 또는 여러 수의 최대공약수/최소공배수를 구하는 문제
- 분수의 약분, 통분이 필요한 문제
- 모듈러 역원을 구해야 하는 문제 (조합론, 확률 등에서 빈출)
- 베주 항등식(ax + by = c)의 정수 해를 구하는 문제
- 서로소 판별 문제
- 나머지 연산과 관련된 정수론 문제

## 단계별 추적 (Trace)

### 예시 1: GCD(252, 105) 계산

```
단계 1: GCD(252, 105)
  252 = 105 × 2 + 42
  → GCD(105, 42)

단계 2: GCD(105, 42)
  105 = 42 × 2 + 21
  → GCD(42, 21)

단계 3: GCD(42, 21)
  42 = 21 × 2 + 0
  → 나머지 = 0, GCD = 21

추적 요약:
  252 mod 105 = 42
  105 mod  42 = 21
   42 mod  21 =  0  ← 종료
                 ↑
              GCD = 21
```

### 예시 2: 확장 유클리드 알고리즘 — 30x + 21y = GCD(30, 21)

```
[순방향: GCD 계산]
  GCD(30, 21): 30 = 21×1 + 9   → GCD(21, 9)
  GCD(21,  9): 21 =  9×2 + 3   → GCD(9, 3)
  GCD( 9,  3):  9 =  3×3 + 0   → GCD = 3

[역방향: 계수 x, y 복원]
  기저: GCD(3, 0) → x=1, y=0  (3×1 + 0×0 = 3)

  GCD(9, 3): a=9, b=3, q=3
    x = y_prev = 0
    y = x_prev - q × y_prev = 1 - 3×0 = 1
    검증: 9×0 + 3×1 = 3 ✓

  GCD(21, 9): a=21, b=9, q=2
    x = y_prev = 1
    y = x_prev - q × y_prev = 0 - 2×1 = -2
    검증: 21×1 + 9×(-2) = 21 - 18 = 3 ✓

  GCD(30, 21): a=30, b=21, q=1
    x = y_prev = -2
    y = x_prev - q × y_prev = 1 - 1×(-2) = 3
    검증: 30×(-2) + 21×3 = -60 + 63 = 3 ✓

결과: 30×(-2) + 21×3 = 3 = GCD(30, 21)
```

### 예시 3: LCM(12, 18) 계산

```
1단계: GCD(12, 18) 계산
  18 mod 12 = 6  → GCD(12, 6)
  12 mod  6 = 0  → GCD = 6

2단계: LCM 계산 (오버플로 방지)
  LCM = 12 / GCD(12,18) × 18
      = 12 / 6 × 18
      = 2 × 18
      = 36

검증: 36 / 12 = 3 (정수), 36 / 18 = 2 (정수) ✓
```

## 실전 팁

### 활용 노하우

- LCM 계산 시 `a * b / gcd`가 아닌 `a / gcd * b`로 계산하여 오버플로를 방지하자
- 여러 수의 GCD/LCM은 두 수씩 순차적으로 계산하면 된다: GCD(a, b, c) = GCD(GCD(a, b), c)
- 모듈러 역원이 필요한 문제에서 모듈러가 소수이면 페르마 소정리(a^(p-2) mod p)가 더 간단하고, 소수가 아니면 확장 유클리드를 사용하자
- 분수를 다루는 문제에서는 분자/분모를 GCD로 나누어 항상 기약분수로 유지하자

### 자주 하는 실수

- LCM 계산 시 `a * b`를 먼저 계산하여 int64 오버플로가 발생하는 경우. 반드시 `a / gcd * b` 순서로 계산
- GCD(0, 0)의 처리를 잊는 경우. 수학적으로 GCD(0, 0) = 0으로 정의한다
- 음수 입력에 대해 GCD를 구할 때 절댓값을 취하지 않는 경우
- 확장 유클리드에서 반환된 x가 음수일 때 모듈러 역원으로 변환하는 과정에서 `(x % m + m) % m`을 빠뜨리는 경우
- GCD(a, 0) = a라는 기저 조건을 잘못 구현하는 경우

### 엣지 케이스

- 두 수 중 하나가 0인 경우: GCD(a, 0) = a, GCD(0, b) = b
- 두 수가 같은 경우: GCD(a, a) = a
- 두 수가 서로소인 경우: GCD(a, b) = 1
- 매우 큰 수(10^18 수준)에서의 GCD 계산 — 유클리드 호제법은 문제없이 동작
- LCM이 int64 범위를 초과하는 경우 — 문제 조건을 확인하고 필요시 큰 수 처리

## 관련 알고리즘 비교

| 알고리즘 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 유클리드 호제법 (기본) | 나머지 연산 반복으로 GCD 계산, O(log min(a,b)) | 두 수의 GCD/LCM을 빠르게 구할 때 |
| 확장 유클리드 알고리즘 | GCD와 함께 베주 계수 x, y를 구함 | 모듈러 역원, 일차 부정방정식의 해를 구할 때 |
| 소인수분해 기반 GCD | 두 수를 소인수분해하여 공통 소인수의 최소 지수를 취함 | 여러 수의 GCD를 구조적으로 이해할 때 (실전에서는 비효율적) |
| 페르마 소정리 | a^(p-1) ≡ 1 (mod p)를 이용한 모듈러 역원 | 모듈러가 소수일 때 모듈러 역원을 구하는 간단한 방법 |
| 스타인 알고리즘 (Binary GCD) | 나눗셈 대신 뺄셈과 시프트 연산 사용 | 나눗셈이 느린 환경에서 GCD를 구할 때 (실전에서는 거의 사용 안 함) |
