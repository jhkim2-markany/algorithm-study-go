# 구간 최솟값 쿼리 - 해설

## 접근 방식

1. 배열을 입력받는다
2. Sparse Table을 O(N log N)에 전처리한다
3. 각 쿼리에 대해 O(1)에 구간 최솟값을 응답한다

## 핵심 아이디어

Sparse Table의 핵심은 임의의 구간 [L, R]을 길이가 2의 거듭제곱인 두 구간으로 덮는 것이다. 구간 길이 len = R - L + 1에 대해 k = floor(log₂(len))을 구하면, [L, L+2^k-1]과 [R-2^k+1, R] 두 구간이 [L, R] 전체를 덮는다.

두 구간이 겹칠 수 있지만, min 연산은 멱등(idempotent)이므로 같은 원소를 여러 번 비교해도 결과가 변하지 않는다. 따라서 겹치는 부분이 있어도 정확한 최솟값을 구할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N + M) |
| 공간 복잡도 | O(N log N) |

전처리에 O(N log N), 각 쿼리에 O(1)이므로 총 O(N log N + M)이다.

## 대안적 접근

- 세그먼트 트리: 전처리 O(N), 쿼리 O(log N)으로 값 갱신이 필요한 경우 유리하다.
- 나이브 방법: 매 쿼리마다 구간을 순회하면 O(NM)으로 N, M이 클 때 시간 초과가 발생한다.
