# 슬라이딩 윈도우 최솟값 - 해설

## 접근 방식

1. 배열을 입력받는다
2. 최솟값과 최댓값 각각에 대해 Sparse Table을 O(N log N)에 전처리한다
3. 각 윈도우 위치 [i, i+K-1]에 대해 O(1)에 최솟값과 최댓값을 쿼리한다
4. 총 N-K+1개의 윈도우 위치에 대해 결과를 출력한다

## 핵심 아이디어

슬라이딩 윈도우 최솟값/최댓값 문제는 본질적으로 고정 크기 구간에 대한 RMQ(Range Minimum/Maximum Query)이다. 배열이 정적이므로 Sparse Table을 사용하면 전처리 후 각 쿼리를 O(1)에 응답할 수 있다.

모든 윈도우의 크기가 K로 동일하므로, k = floor(log₂(K))를 한 번만 계산하면 된다. 각 윈도우 위치 i에서:
- 최솟값 = min(sparse_min[k][i], sparse_min[k][i+K-2^k])
- 최댓값 = max(sparse_max[k][i], sparse_max[k][i+K-2^k])

이 문제의 난이도가 높은 이유는 N이 최대 1,000,000으로 크고, 최솟값과 최댓값을 동시에 구해야 하며, 효율적인 입출력 처리가 필요하기 때문이다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N log N) |
| 공간 복잡도 | O(N log N) |

전처리에 O(N log N), 전체 쿼리에 O(N)이므로 총 O(N log N)이다.

## 대안적 접근

- 덱(Deque): 모노톤 덱을 사용하면 O(N)에 풀 수 있다. 각 원소가 덱에 최대 한 번 삽입/삭제되므로 전체 O(N)이다. 공간도 O(N)으로 Sparse Table보다 효율적이다.
- 세그먼트 트리: 전처리 O(N), 각 쿼리 O(log N)으로 총 O(N log N)이다. 값 갱신이 필요한 변형 문제에 유리하다.
- Sparse Table은 구현이 간단하고 쿼리가 O(1)이라는 장점이 있지만, 이 문제에서는 덱이 시간/공간 모두 최적이다.
