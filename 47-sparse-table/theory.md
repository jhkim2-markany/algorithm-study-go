# 희소 배열 (Sparse Table)

## 개념

희소 배열(Sparse Table)은 정적 배열에서 구간 쿼리를 효율적으로 처리하기 위한 자료구조이다. 전처리에 O(N log N) 시간과 공간을 사용하며, 이후 각 쿼리를 O(1)에 응답할 수 있다. 단, 배열의 값이 변경되지 않는(정적인) 경우에만 사용할 수 있다.

Sparse Table이 O(1) 쿼리를 지원하는 핵심 조건은 연산이 멱등(idempotent)해야 한다는 것이다. 멱등 연산이란 같은 원소를 여러 번 적용해도 결과가 변하지 않는 연산을 말한다. 예를 들어 min(a, a) = a, max(a, a) = a, gcd(a, a) = a이므로 최솟값, 최댓값, GCD는 멱등 연산이다. 반면 합(sum)은 같은 원소를 두 번 더하면 결과가 달라지므로 멱등 연산이 아니다.

코딩테스트에서 Sparse Table은 RMQ(Range Minimum/Maximum Query) 문제, LCA의 오일러 투어 기반 풀이, 구간 GCD 쿼리 등에 활용된다.

## 선수 학습

희소 배열을 학습하기 전에 다음 내용을 먼저 학습하는 것을 권장한다:

- [06-prefix-sum](../06-prefix-sum/) - 누적합을 이용한 구간 쿼리의 기본 개념, 전처리 후 O(1) 쿼리 응답 패턴
- [25-segment-tree](../25-segment-tree/) - 구간 쿼리를 처리하는 트리 기반 자료구조, Sparse Table과의 비교를 위한 배경 지식

## 동작 원리

### 1. 테이블 구성 (전처리)

길이 N인 배열 A에 대해 2차원 테이블 sparse[k][i]를 구성한다. sparse[k][i]는 인덱스 i에서 시작하여 길이 2^k인 구간의 쿼리 결과를 저장한다.

1. 로그 값 전처리: log[i] = floor(log₂(i))를 미리 계산한다
2. 길이 1인 구간 초기화: sparse[0][i] = A[i] (또는 인덱스 i)
3. 점화식으로 길이 2^k인 구간을 채운다:
   - sparse[k][i] = op(sparse[k-1][i], sparse[k-1][i + 2^(k-1)])
   - 즉, 길이 2^k인 구간을 길이 2^(k-1)인 두 구간으로 나누어 합친다
4. k는 1부터 floor(log₂(N))까지 반복한다

### 2. 쿼리 응답 (O(1))

구간 [L, R]에 대한 쿼리를 응답하는 과정:

1. 구간 길이 len = R - L + 1을 구한다
2. k = floor(log₂(len))을 구한다
3. 길이 2^k인 두 구간 [L, L+2^k-1]과 [R-2^k+1, R]을 겹쳐서 합친다:
   - 결과 = op(sparse[k][L], sparse[k][R - 2^k + 1])
4. 두 구간이 겹치더라도 멱등 연산이므로 결과가 정확하다

이것이 Sparse Table의 핵심 아이디어이다. 임의의 구간을 2의 거듭제곱 길이의 두 구간으로 덮을 수 있고, 멱등 연산이므로 겹치는 부분이 있어도 결과에 영향을 주지 않는다.

### 3. 멱등 연산 vs 비멱등 연산

| 연산 | 멱등 여부 | O(1) 쿼리 가능 |
| --- | --- | --- |
| min | 멱등 | 가능 |
| max | 멱등 | 가능 |
| gcd | 멱등 | 가능 |
| bitwise AND | 멱등 | 가능 |
| bitwise OR | 멱등 | 가능 |
| sum | 비멱등 | 불가 (O(log N)) |

비멱등 연산(합 등)의 경우에도 Sparse Table을 사용할 수 있지만, 겹치지 않는 구간으로 분할해야 하므로 쿼리가 O(log N)이 된다. 이 경우 세그먼트 트리와 성능이 같으므로 세그먼트 트리를 사용하는 것이 일반적이다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 전처리 시간 | O(N log N) |
| 쿼리 시간 (멱등 연산) | O(1) |
| 쿼리 시간 (비멱등 연산) | O(log N) |
| 공간 복잡도 | O(N log N) |

Sparse Table은 전처리 후 O(1) 쿼리가 가능하다는 점에서 세그먼트 트리(O(log N) 쿼리)보다 빠르다. 그러나 값 갱신이 불가능하므로 정적 배열에서만 사용할 수 있다.

## 적합한 문제 유형

- 정적 배열에서 구간 최솟값/최댓값을 구하는 문제 (RMQ)
- 정적 배열에서 구간 GCD를 구하는 문제
- LCA를 오일러 투어 + RMQ로 구하는 문제
- 슬라이딩 윈도우에서 최솟값/최댓값을 구하는 문제
- 값이 변하지 않는 배열에서 다수의 구간 쿼리를 처리하는 문제
- 구간에서 비트 연산(AND, OR) 결과를 구하는 문제

## 단계별 추적 (Trace)

### 예시 입력

배열 A = [2, 5, 1, 4, 3, 7, 6] (N=7)에 대해 구간 최솟값(RMQ) Sparse Table을 구성하고 쿼리를 처리한다.

### 전처리 과정

**1단계: 로그 값 전처리**

```
log[1]=0, log[2]=1, log[3]=1, log[4]=2, log[5]=2, log[6]=2, log[7]=2
```

**2단계: k=0 (길이 1 구간) 초기화**

```
sparse[0] = [2, 5, 1, 4, 3, 7, 6]
             i=0 i=1 i=2 i=3 i=4 i=5 i=6
```

**3단계: k=1 (길이 2 구간) 계산**

```
sparse[1][i] = min(sparse[0][i], sparse[0][i+1])

sparse[1][0] = min(2, 5) = 2    구간 [0,1]
sparse[1][1] = min(5, 1) = 1    구간 [1,2]
sparse[1][2] = min(1, 4) = 1    구간 [2,3]
sparse[1][3] = min(4, 3) = 3    구간 [3,4]
sparse[1][4] = min(3, 7) = 3    구간 [4,5]
sparse[1][5] = min(7, 6) = 6    구간 [5,6]
```

**4단계: k=2 (길이 4 구간) 계산**

```
sparse[2][i] = min(sparse[1][i], sparse[1][i+2])

sparse[2][0] = min(2, 1) = 1    구간 [0,3]
sparse[2][1] = min(1, 3) = 1    구간 [1,4]
sparse[2][2] = min(1, 3) = 1    구간 [2,5]
sparse[2][3] = min(3, 6) = 3    구간 [3,6]
```

### 쿼리 처리

**쿼리: 구간 [1, 5]의 최솟값**

```
len = 5 - 1 + 1 = 5
k = log[5] = 2  (2^2 = 4)

왼쪽 구간: sparse[2][1] = 1   → 구간 [1, 4]
오른쪽 구간: sparse[2][2] = 1  → 구간 [2, 5]

결과 = min(1, 1) = 1 ✓ (실제 최솟값: A[2]=1)
```

**쿼리: 구간 [3, 6]의 최솟값**

```
len = 6 - 3 + 1 = 4
k = log[4] = 2  (2^2 = 4)

왼쪽 구간: sparse[2][3] = 3   → 구간 [3, 6]
오른쪽 구간: sparse[2][3] = 3  → 구간 [3, 6]

결과 = min(3, 3) = 3 ✓ (실제 최솟값: A[4]=3)
```

## 실전 팁

### 활용 노하우

- Sparse Table은 정적 배열 + 멱등 연산 조합에서 가장 빠른 구간 쿼리 자료구조이다. 값 갱신이 없다면 세그먼트 트리 대신 사용하자
- 로그 배열을 미리 전처리하면 쿼리마다 `math.Log2`를 호출하는 오버헤드를 피할 수 있다
- LCA 문제에서 오일러 투어 + Sparse Table 조합은 O(N log N) 전처리, O(1) 쿼리로 매우 효율적이다
- 구간 GCD 문제가 나오면 Sparse Table을 먼저 떠올리자. 세그먼트 트리보다 구현이 간단하고 빠르다

### 자주 하는 실수

- k 루프를 가장 바깥에 두지 않는 실수: 전처리 시 k를 먼저 순회해야 이전 단계 결과를 올바르게 참조한다
- 인덱스 범위 초과: `sparse[k][i + (1 << (k-1))]`에서 `i + (1 << k) - 1 < N` 조건을 확인해야 한다
- 로그 값 계산 오류: `log[1] = 0`부터 시작해야 하며, 정수 로그를 사용해야 한다 (부동소수점 오차 주의)
- 비멱등 연산에 O(1) 쿼리를 적용하는 실수: 합(sum) 연산은 겹치는 구간 때문에 O(1) 쿼리가 불가능하다

### 엣지 케이스

- 구간 길이가 1인 경우: `sparse[0][L]`을 바로 반환한다
- 구간 길이가 정확히 2의 거듭제곱인 경우: 두 구간이 완전히 겹치지만 멱등 연산이므로 정확하다
- N = 1인 경우: 전처리 시 k=0만 존재하며, 모든 쿼리가 단일 원소를 반환한다
- 모든 원소가 동일한 경우: 정상 동작하지만, 테스트 시 다양한 값으로 검증하자

## 관련 알고리즘 비교

| 알고리즘 | 전처리 | 쿼리 | 갱신 | 적합한 상황 |
| --- | --- | --- | --- | --- |
| Sparse Table | O(N log N) | O(1) | 불가 | 정적 배열, 멱등 연산 (RMQ, GCD) |
| 세그먼트 트리 | O(N) | O(log N) | O(log N) | 동적 배열, 임의 연산 (합, 최솟값 등) |
| 누적합 (Prefix Sum) | O(N) | O(1) | 불가 | 정적 배열, 구간 합 전용 |
| 펜윅 트리 (BIT) | O(N) | O(log N) | O(log N) | 동적 배열, 구간 합/점 갱신 |
| 모노톤 덱 | O(N) | 슬라이딩 O(1) | - | 고정 크기 슬라이딩 윈도우 최솟값/최댓값 |

- 값 갱신이 필요하면 → 세그먼트 트리 또는 펜윅 트리
- 정적 배열 + 멱등 연산이면 → Sparse Table (O(1) 쿼리)
- 정적 배열 + 구간 합이면 → 누적합 (가장 단순)
- 슬라이딩 윈도우 최솟값/최댓값이면 → 모노톤 덱 (공간 효율적)
