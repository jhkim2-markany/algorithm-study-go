# 구간 최솟값 쿼리 - 해설

## 접근 방식

1. 세그먼트 트리를 구간 합이 아닌 구간 최솟값을 저장하도록 변형한다.
2. 트리 구축 시 내부 노드에 두 자식 중 작은 값을 저장한다.
3. 질의 시 구간이 겹치지 않으면 무한대(math.MaxInt64)를 반환하고, 완전히 포함되면 노드 값을 반환한다.
4. 갱신 시 리프 노드를 변경한 뒤 부모 노드를 min 연산으로 재계산한다.

## 핵심 아이디어

세그먼트 트리의 연산을 합(+)에서 최솟값(min)으로 바꾸는 것이 핵심이다. 합의 항등원이 0인 것처럼, 최솟값의 항등원은 무한대(∞)이다. 구간이 겹치지 않을 때 무한대를 반환하면, min 연산에서 다른 쪽 결과에 영향을 주지 않는다.

구간 합 세그먼트 트리와의 차이점:
- 내부 노드 계산: `tree[node] = tree[left] + tree[right]` → `tree[node] = min(tree[left], tree[right])`
- 항등원: 0 → math.MaxInt64
- 나머지 구조(구축, 질의, 갱신의 재귀 흐름)는 동일하다

이처럼 세그먼트 트리는 결합 법칙이 성립하는 임의의 연산(합, 최솟값, 최댓값, GCD 등)에 적용할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M log N) |
| 공간 복잡도 | O(N) |

구간 합 세그먼트 트리와 동일한 복잡도이다. 트리 구축에 O(N), 각 질의와 갱신에 O(log N)이 소요된다.

## 대안적 접근

- 희소 테이블(Sparse Table)을 사용하면 질의를 O(1)에 처리할 수 있지만, 갱신이 불가능하다. 갱신이 없는 정적 배열에서는 희소 테이블이 더 효율적이다.
- 제곱근 분할(Sqrt Decomposition)을 사용하면 구현이 간단하지만, 질의와 갱신 모두 O(√N)이다.
