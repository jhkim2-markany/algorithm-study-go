# 느리게 갱신되는 세그먼트 트리 - 해설

## 접근 방식

1. 구간 갱신(구간에 값 더하기)과 구간 질의(구간 합)를 모두 효율적으로 처리해야 한다.
2. 일반 세그먼트 트리로 구간 갱신을 하면 최악 O(N log N)이 걸리므로, Lazy Propagation을 적용한다.
3. 구간 갱신 시 해당 구간을 완전히 포함하는 노드에서 멈추고, 갱신 정보를 lazy 배열에 저장한다.
4. 이후 질의나 갱신으로 해당 노드의 자식을 방문할 때 lazy 값을 전파(push down)한다.

## 핵심 아이디어

Lazy Propagation의 핵심은 "필요할 때까지 갱신을 미루는 것"이다.

구간 [l, r]에 v를 더하는 연산에서, 현재 노드의 구간이 [l, r]에 완전히 포함되면:
- `tree[node]`에 `v × (구간 길이)`를 더한다
- `lazy[node]`에 v를 더한다 (자식에게 전파할 값 저장)

자식 노드를 방문해야 할 때(부분적으로 겹치는 경우) push down을 수행한다:
- 왼쪽 자식의 tree 값에 `lazy[node] × (왼쪽 구간 길이)`를 더한다
- 오른쪽 자식의 tree 값에 `lazy[node] × (오른쪽 구간 길이)`를 더한다
- 양쪽 자식의 lazy 값에 `lazy[node]`를 더한다
- `lazy[node]`를 0으로 초기화한다

이를 통해 구간 갱신과 구간 질의 모두 O(log N)에 처리할 수 있다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M log N) |
| 공간 복잡도 | O(N) |

트리 구축에 O(N), 각 구간 갱신과 구간 질의에 O(log N)이 소요된다. tree 배열과 lazy 배열 각각 O(4N) 공간이 필요하므로 전체 공간 복잡도는 O(N)이다.

## 대안적 접근

- 펜윅 트리(BIT)를 두 개 사용하여 구간 갱신 + 구간 질의를 처리할 수 있다. 구현이 더 간결하지만 이해하기 어려울 수 있다.
- 제곱근 분할(Sqrt Decomposition)로도 구간 갱신과 질의를 처리할 수 있으나, 시간 복잡도가 O(√N)으로 느리다.
