# 구간 합 쿼리 - 해설

## 접근 방식

1. 세그먼트 트리의 가장 기본적인 활용인 구간 합 질의 + 점 갱신 문제이다.
2. 배열을 기반으로 세그먼트 트리를 구축한다.
3. 갱신 연산(op=1)이 들어오면 해당 인덱스의 리프 노드를 갱신하고 부모 노드를 재계산한다.
4. 질의 연산(op=2)이 들어오면 구간 [l, r]에 해당하는 노드들의 합을 반환한다.

## 핵심 아이디어

세그먼트 트리는 배열의 구간 정보를 이진 트리 형태로 저장한다. 각 노드는 특정 구간의 합을 미리 계산해 두므로, 질의 시 전체 구간을 순회하지 않고 O(log N)개의 노드만 방문하면 된다.

트리의 노드 번호를 1부터 시작하면, 노드 i의 왼쪽 자식은 2i, 오른쪽 자식은 2i+1이 된다. 이를 통해 배열 하나로 트리를 표현할 수 있다. 크기 4N의 배열이면 충분하다.

질의 시 현재 노드의 구간과 질의 구간의 관계에 따라 세 가지로 분기한다:
- 완전히 포함: 현재 노드의 값을 바로 반환
- 전혀 겹치지 않음: 0(항등원)을 반환
- 부분적으로 겹침: 양쪽 자식에 재귀 질의 후 합산

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(N + M log N) |
| 공간 복잡도 | O(N) |

트리 구축에 O(N), 각 질의와 갱신에 O(log N)이 소요되므로 전체 시간 복잡도는 O(N + M log N)이다. 세그먼트 트리 배열에 O(4N) = O(N) 공간이 필요하다.

## 대안적 접근

- 펜윅 트리(Binary Indexed Tree, BIT)를 사용하면 구현이 더 간결하고 상수 배 빠르다. 다만 세그먼트 트리가 더 범용적이다.
- 누적합 배열을 사용하면 질의는 O(1)이지만 갱신이 O(N)이므로, 갱신이 빈번한 경우 세그먼트 트리가 유리하다.
