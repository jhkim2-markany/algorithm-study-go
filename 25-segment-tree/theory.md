# 세그먼트 트리

## 개념

세그먼트 트리(Segment Tree)는 배열의 특정 구간에 대한 질의(query)와 갱신(update)을 효율적으로 처리하기 위한 이진 트리 기반 자료구조이다. 일반 배열에서 구간 합이나 구간 최솟값을 구하려면 O(N)이 걸리지만, 세그먼트 트리를 사용하면 O(log N)에 처리할 수 있다.

세그먼트 트리의 핵심 특성은 다음과 같다:
- **완전 이진 트리 구조**: 리프 노드는 원본 배열의 각 원소를 저장하고, 내부 노드는 자식 노드들의 구간 정보를 합산하여 저장한다
- **구간 분할**: 각 노드는 배열의 특정 구간을 담당하며, 자식 노드는 부모 구간을 절반으로 나눈 구간을 담당한다
- **빠른 질의와 갱신**: 구간 질의와 점 갱신 모두 O(log N)에 수행된다
- **느리게 갱신(Lazy Propagation)**: 구간 갱신이 필요한 경우, 갱신을 지연시켜 O(log N)에 처리할 수 있다

세그먼트 트리는 구간 합, 구간 최솟값/최댓값, 구간 GCD 등 결합 법칙이 성립하는 연산에 활용된다.

## 동작 원리

### 트리 구축 (Build)

1. 배열의 전체 구간 [0, N-1]을 루트 노드에 할당한다.
2. 각 노드의 구간을 절반으로 나누어 왼쪽 자식과 오른쪽 자식에 할당한다.
3. 리프 노드(구간 길이가 1인 노드)에 도달하면 원본 배열의 값을 저장한다.
4. 내부 노드는 두 자식 노드의 값을 합산(또는 최솟값, 최댓값 등)하여 저장한다.
5. 재귀적으로 모든 노드를 채워 트리를 완성한다.

### 구간 질의 (Query)

1. 루트 노드부터 시작하여 질의 구간 [L, R]과 현재 노드의 구간을 비교한다.
2. 현재 노드의 구간이 질의 구간에 완전히 포함되면 해당 노드의 값을 반환한다.
3. 현재 노드의 구간이 질의 구간과 전혀 겹치지 않으면 항등원(합: 0, 최솟값: ∞)을 반환한다.
4. 부분적으로 겹치면 왼쪽 자식과 오른쪽 자식에 대해 재귀적으로 질의한다.
5. 두 자식의 결과를 합산하여 최종 결과를 반환한다.

### 점 갱신 (Point Update)

1. 루트 노드부터 시작하여 갱신할 인덱스가 포함된 자식으로 내려간다.
2. 리프 노드에 도달하면 값을 갱신한다.
3. 리프에서 루트까지 올라가며 각 노드의 값을 자식 노드의 값으로 다시 계산한다.

### 느리게 갱신 (Lazy Propagation)

구간 전체에 동일한 값을 더하거나 변경해야 할 때 사용한다:

1. 갱신 구간이 현재 노드의 구간을 완전히 포함하면, 현재 노드의 값을 갱신하고 자식에게 전파할 값을 lazy 배열에 저장한다.
2. 부분적으로 겹치면 먼저 현재 노드의 lazy 값을 자식에게 전파(push down)한 뒤, 왼쪽과 오른쪽 자식에 대해 재귀적으로 갱신한다.
3. 질의 시에도 lazy 값이 남아 있으면 먼저 전파한 뒤 질의를 수행한다.
4. 이를 통해 구간 갱신과 구간 질의 모두 O(log N)에 처리할 수 있다.

## 복잡도

| 구분 | 복잡도 |
| --- | --- |
| 트리 구축 | O(N) |
| 구간 질의 | O(log N) |
| 점 갱신 | O(log N) |
| 구간 갱신 (Lazy) | O(log N) |
| 공간 복잡도 | O(N) |

세그먼트 트리는 크기 4N의 배열로 구현하며, 구축에 O(N), 질의와 갱신에 O(log N)이 소요된다. Lazy Propagation을 사용하면 추가로 크기 4N의 lazy 배열이 필요하지만, 시간 복잡도는 동일하게 O(log N)을 유지한다.

## 적합한 문제 유형

- 구간 합 질의 (Range Sum Query)
- 구간 최솟값/최댓값 질의 (Range Min/Max Query)
- 구간 GCD/LCM 질의
- 구간 갱신 + 구간 질의 (Lazy Propagation 활용)
- 좌표 압축 후 구간 연산
- 역전 카운트 (Inversion Count)
- 구간에서 K번째 원소 찾기
- 히스토그램에서 최대 직사각형 (세그먼트 트리 활용 풀이)

## 단계별 추적 (Trace)

### 예시 입력

배열: [1, 3, 5, 7, 9, 11], 구간 합 세그먼트 트리

### 트리 구축 과정

```
배열 인덱스:  0  1  2  3  4  5
배열 값:     [1, 3, 5, 7, 9, 11]

세그먼트 트리 (노드 번호: [구간] = 값):

              1: [0,5] = 36
            /                \
      2: [0,2] = 9         3: [3,5] = 27
      /        \            /         \
  4: [0,1]=4  5:[2,2]=5  6:[3,4]=16  7:[5,5]=11
  /     \                 /      \
8:[0,0]=1 9:[1,1]=3   12:[3,3]=7 13:[4,4]=9
```

### 구간 질의: Query(1, 4) → 구간 [1,4]의 합

```
노드 1: [0,5] → 부분 겹침 → 왼쪽/오른쪽 자식으로
  노드 2: [0,2] → 부분 겹침 → 왼쪽/오른쪽 자식으로
    노드 4: [0,1] → 부분 겹침 → 왼쪽/오른쪽 자식으로
      노드 8: [0,0] → 질의 범위 밖 → 0 반환
      노드 9: [1,1] → 완전 포함 → 3 반환
    노드 4 결과: 0 + 3 = 3
    노드 5: [2,2] → 완전 포함 → 5 반환
  노드 2 결과: 3 + 5 = 8
  노드 3: [3,5] → 부분 겹침 → 왼쪽/오른쪽 자식으로
    노드 6: [3,4] → 완전 포함 → 16 반환
    노드 7: [5,5] → 질의 범위 밖 → 0 반환
  노드 3 결과: 16 + 0 = 16

최종 결과: 8 + 16 = 24  (= 3 + 5 + 7 + 9)
```

### 점 갱신: Update(2, 10) → 인덱스 2의 값을 10으로 변경

```
변경 전: arr[2] = 5 → 변경 후: arr[2] = 10 (차이 = +5)

노드 5: [2,2] = 5 → 10  (리프 갱신)
노드 2: [0,2] = 9 → 14  (자식 합 재계산: 4 + 10)
노드 1: [0,5] = 36 → 41 (자식 합 재계산: 14 + 27)

갱신 경로: 노드 5 → 노드 2 → 노드 1 (O(log N) = 3단계)
```

## 실전 팁

### 활용 노하우

- 세그먼트 트리 배열 크기는 `4 * N`으로 잡으면 안전하다 (정확히는 2^(ceil(log2(N))+1))
- 구간 합, 구간 최솟값, 구간 최댓값, 구간 GCD 등 결합 법칙이 성립하는 연산에 사용 가능하다
- Lazy Propagation은 "구간 갱신 + 구간 질의"가 동시에 필요할 때만 사용한다
- 좌표 압축 + 세그먼트 트리 조합은 역전 카운트, 구간 내 원소 개수 등에 자주 사용된다
- 재귀 구현이 직관적이지만, 반복(iterative) 구현이 상수 배 빠르다

### 자주 하는 실수

- 배열 크기를 `2 * N`으로 잡아 인덱스 초과 오류 발생 (반드시 `4 * N`)
- Lazy Propagation에서 push down을 질의/갱신 전에 하지 않아 오답 발생
- 구간 경계를 잘못 설정하여 off-by-one 에러 발생 (0-indexed vs 1-indexed)
- 항등원을 잘못 설정하는 경우 (합: 0, 최솟값: INF, 최댓값: -INF, GCD: 0)
- 점 갱신과 구간 갱신을 혼동하여 Lazy 없이 구간 갱신을 시도하는 경우

### 엣지 케이스

- 배열 크기가 1인 경우 (루트 = 리프)
- 전체 구간을 질의하는 경우 (루트 노드 값 반환)
- 질의 구간이 단일 원소인 경우 (리프 노드 직접 반환)
- 모든 원소가 0인 경우 (합 트리에서 모든 노드가 0)
- 갱신 값이 매우 크거나 음수인 경우 (오버플로우 주의)

| 세그먼트 트리 | O(N) | O(log N) | O(log N) Lazy | O(log N) | 범용 구간 연산, 갱신+질의 혼합 |
| 펜윅 트리 (BIT) | O(N log N) | O(log N) | O(log N) | O(log N) | 구간 합 특화, 구현 간결 |
