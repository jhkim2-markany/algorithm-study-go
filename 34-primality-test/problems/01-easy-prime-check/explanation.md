# 소수 판별 - 해설

## 접근 방식

1. 2 미만의 수는 소수가 아니므로 바로 제외한다
2. 2는 유일한 짝수 소수이므로 별도 처리한다
3. 짝수를 먼저 제외하여 검사 범위를 절반으로 줄인다
4. 3부터 √N까지 홀수로만 나누어 보며 약수가 있는지 확인한다
5. 약수가 없으면 소수, 있으면 합성수로 판정한다

## 핵심 아이디어

N이 합성수라면 N = a × b (a ≤ b)로 표현할 수 있고, 이때 a ≤ √N이 성립한다. 따라서 √N까지만 나누어 보면 소수 여부를 판정할 수 있다. 추가로 짝수를 먼저 제외하면 홀수만 검사하므로 실제 검사 횟수가 절반으로 줄어든다.

## 복잡도 분석

| 구분 | 복잡도 |
| --- | --- |
| 시간 복잡도 | O(T × √N) |
| 공간 복잡도 | O(1) |

각 테스트 케이스마다 √N까지 나눗셈을 수행하므로 O(√N)이고, T개의 케이스에 대해 O(T × √N)이다. N ≤ 10^6이므로 √N ≤ 1,000으로 충분히 빠르다.

## 대안적 접근

- 에라토스테네스의 체를 사용하여 1~10^6 범위의 소수를 미리 구해두면, 각 쿼리를 O(1)에 처리할 수 있다. 다만 이 문제에서는 시행 나눗셈으로도 충분하다.
